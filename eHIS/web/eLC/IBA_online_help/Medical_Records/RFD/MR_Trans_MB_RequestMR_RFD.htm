Enter the following details:

 

Request Tab

  1. Patient ID - Enter the Patient ID in the format as specified in the MP Parameter. The Patient ID entered has to be unique and valid. On entering the Patient ID, all the demographics of the patient are displayed automatically in the screen. The Patient ID field can be left blank, if the Patient has not been registered with the Facility. In such cases, the minimum demographic details have to be entered while registering the patient. Alternatively, click [?] function to invoke the Search Patient function whereby specifying the appropriate search criteria, retrieves the patient records. You can select the required Patient ID.

  2. Citizen ID - Enter the Citizen ID in alphanumeric format of maximum length as specified in the MP Parameter. The 'Prompt' specified as Citizen ID in MP Parameter appears as the field legend. The Citizen ID is displayed automatically on entering the Patient ID of the registered patient. For a previously unregistered patient, this needs to be entered.

Enter the values in the fields 'Previous HN', 'Government No', 'Driving License No' and 'Expatriate No' as specified in the MP Parameter under 'Alternate ID Checks' in the 'Master Patient Index' module. The fields and their order displayed in the screen may vary depending on the MP Parameter Setup.

  1. Other Alternate Type - Select the other Alternate Type from the list. This displays all the Alternate IDs that have been defined in the MP Parameter setup other than for the default Alternate IDs.

  2. Other Alt ID. - Enter the other alternate ID in numeric format of maximum length as specified in the MP Parameter.

  3. Title - Select a Title of the name from the list if valid values. If the patient ID is selected, the name prefix is defaulted which cannot be changed. This field is mandatory.

  4. Patient Name - Enter the patient Name in alphanumeric format of  maximum length as specified in MP Parameter. If the Patient ID is selected, the patient name is displayed and it cannot be changed. The system allows entry of patient name for only unregistered patients. This field is mandatory.

The multi parts of Patient Name are the Prefix, First Name, Second Name, Third Name, Family Name and Suffix. Based on the setup in MP Parameter (Legal Names), the multi parts of patient name are displayed. The displayed titles and order of appearance are specified in the parameter function. If 'Accept Patient Name in Local Language' is set in the MP Parameter, the patient name can be entered in local language. If a "*" sign appears next to the name part, it is mandatory to enter the data. The field is mandatory if 'Reqd' check box is selected in Legal Names tab in MP Parameter.

  1. Suffix - Select the suffix for the name from the list of valid values. If Patient ID is entered, suffix for the name of the patient is displayed automatically which cannot be altered.

  2. Gender - Select the Gender of the Patient from the list for an unregistered patient. The listed values are Male, Female and Unknown. This field is mandatory. If Patient ID is entered, the gender of the patient is displayed automatically.

  3. Age - For an unregistered patient, enter an approximate Age of the Patient in Years, Months and Days, if the exact Date of Birth is not known. If Age is entered, the system automatically calculates the Date of Birth and displays it in the 'Birth Date' field. Either Age or Date of Birth is entered. On entering one of them, the other field is displayed. For a registered patient, details in the age fields get displayed (and disabled) automatically on entry of Patient ID.

  4. Birth Date - For an unregistered patient enter the Birth Date in dd/mm/yyyy format. If it is entered, the system calculates the Patient's age and displays it in the 'Age' field. Either the Age or Date of Birth is entered. On entering one of them, the other field gets displayed. This field is mandatory. For a registered patient, Date of Birth field gets displayed (and disabled, based on existing registration information) automatically on entry of Patient ID.

  5. Birth Place - For an unregistered patient enter the Birthplace in alphanumeric format of 30 characters maximum length, or use [?] function to select the required birthplace. If Citizen ID is entered, birthplace is displayed based on the birthplace logic defined in the routine associated with the Citizen ID. If the required birth place is not available in the search list, enter the birth place in alphanumeric format of 30 characters maximum length in the text box provided next to [?] function. This field is mandatory.

  6. Residency - While creating a booking for an unregistered patient, select either 'Citizen' or 'Non-citizen'. The 'Citizen' is selected and Legal is selected and disabled, by default. If Citizen ID is entered for the patient, 'Citizen' is selected and disabled, or both the values are available for selection. For a registered patient, these details get displayed (and disabled) automatically on entry of Patient ID.

  7. Status - Select either 'Legal' or 'Illegal' as citizen status. 'Legal' is selected, by default. If the patient is a 'Citizen', 'Legal' is selected and disabled. If the patient is a 'Non-Citizen', the status can be either 'Legal' or 'Illegal'. For a registered patient, these details get displayed (and disabled) automatically on entry of Patient ID.

 

Requester Details Tab

Enter the details of the requester requesting for the Medical Report.

  1. Registration No - This field is disabled. The Registration Number or Request ID has to be generated.

  2. Registration Date/Time - The current system Date and Time is displayed as the Registration Date and Time.

  3. Application Type - Select the type of application from the list. The listed values are populated from the 'Application Type' function. This field is mandatory.

  4. Letter Date/time - Select the Date and Time of the letter from the Calendar function. The current system date and time is displayed, by default.  This field is mandatory.

  5. Applicant Name - Enter the name of the applicant requesting the Medical Board in alphanumeric format of 30 characters maximum length.

  6. Reference No - Enter the Reference Number for the application in numeric format of 15 characters maximum length.

  7. Relationship Type - Select the type of relationship of the requester to the patient from the list of valid values. The listed values are 'Next of Kin', 'First to Notify', 'Employer', 'Self' and 'Others'. On selecting 'Next of Kin', 'First to Notify' or 'Employer', the 'Applicant Name' field is displayed and disabled if the details were recorded while registering the patient. On selecting the value as 'Self', the 'Applicant Name' field is disabled.

  8. Relationship - Select the Relationship of the requester to the patient from the list. This field is enabled and becomes mandatory only if the 'Relationship Type' is selected as 'Others' or this field is disabled.

  9. Requester Type - Select the type of requester from the list. The values are predefined. This field is disabled if any value is selected in the 'Relationship Type' field.

  10. Requester - Enter the requester in alphanumeric format of 15 characters maximum length or select the name of the requester who requested for the medical report from [?] function. This field becomes mandatory on selecting the 'Requester Type'. This field is disabled if any value is selected in the 'Relationship Type' field.

  11. Consent letter - Select the availability of the consent letter for the request as 'Available', 'Not Available' or 'Not Required' from the list of valid values.

  12. Attention To - Enter the name of the person whose attention is needed in the preparation of the Medical Report in alphanumeric format of 50 characters maximum length.

  13. Type of Medical Board - Select a value from the list. The listed values are defined in the function 'Medical Board Type'. This field is mandatory.

  14. Application No - Enter the Application Number in numeric format of 15 characters maximum length.

Click [Applicant Address] to enter the contact address of the applicant of the Medical Report request. Enter the Contact Address of the applicant and click [Close] to save the details.

The applicant address cannot be entered if the Requester Type and Requester are selected.

 

Other Details Tab

Payment Details

  1. Charges Applicable - Select Yes or No from the list indicating whether charges are applicable to the applicant for the Medical Report of the patient.

  2. Receipt No - Enter the number of the receipt in alphanumeric format of 15 characters maximum length. This field is enabled only if the 'Charges Applicable' field is selected as 'Yes'.

  3. Amount - Enter the amount to be charged in numeric format of 15 characters maximum length. This field is enabled only if the 'Charges Applicable' field is selected as 'Yes'.

  4. Receipt Date - The current date and time is displayed as the Receipt Date and this field is disabled if the Receipt No is entered.

  5. Additional Charges - Select Yes or No from the list indicating whether any additional charges are applicable to the applicant for the medical report of the patient.

  6. Additional Receipt No - Enter the number of the receipt in alphanumeric format of 15 characters maximum length. This field is enabled only if the 'Additional Charges' field is selected as 'Yes'.

  7. Additional Amount - Enter the amount to be charged in numeric format of 15 characters maximum length. This field is enabled only if the 'Additional Charges' field is selected as 'Yes'.

  8. Additional Receipt Date - The current date and time is displayed as the Additional Receipt Date and this field is disabled if the Receipt No is entered.
