Chart Summary (V)

From this function you can View the patient’s chart summary.  

Chart Summary provides the practitioners a summary of clinical events from the patient’s history. Although, Chart Summary is part of the sub menu Chart History, this function is the first to be displayed on invoking a patient. The format and layout of the chart summary is customized to suit practitioner preferences and varies based on the Patient Class.

Viewing Patient Chart Summary

From the CA Transactions > Clinician Access > Desktop Menu > select required patient, Patient Chart.

The View Chart Summary screen gets displayed.

The Patient Chart displays the following:

  1. The Patient details which include the Name, Age, Gender, Patient ID, Encounter ID, Location and Attending Practitioner of the patient are displayed in the Patient Line header.

  2. If the Patient has Allergies and High Risk conditions recorded, the Patient Line header displays Allergies and High Risk. Profile is displayed in the Patient Line header on selecting any patient.

    1. Select the Profile icon from the Patient Line header to view the profile of the patient in the Patient Profile screen.

    2. The Name, Age, Gender and Patient ID is displayed.

    3. The contents of the screen are those enabled from the Patient Profile Content function in CA Code Setup. The description of the field is also as defined in the same function of CA Code Setup.

    4. Select the required check box against the Profile Content and click [Show] to view the details. The details of the selected Profile Content are displayed on the right hand side of the screen.

    5. Click [Print] to print details or click [Close] to exit from the screen.

    6. Select the check box against Patient Allergy Signs and the password has to be entered to view the high risk conditions recorded for the patient.

    7. Enter the password as defined in the User Access Rights function in Terminology and Coding Management module. Click [OK] and then click [Show]. The High Risk conditions recorded for the patient is displayed.

    Select the High Risk icon from the Patient Line header and the screen to enter the password and then view the High Risk condition recorded for the patient is displayed.

    Enter the correct password and click [OK] to view the High Risk Conditions and the Onset Date.

    Select the Allergies icon from the Patient Line header to view the allergies recorded for the patient.

    The Allergies screen displays the details ‘Allergic To’ and ‘Description’ of the Allergy.

  3. The History icon is displayed on selecting a patient. Click the History icon to navigate to Clinical Event History menu.

  4. The Quick Links list box lists the Flow Sheet defined for the Practitioner. Select the required Quick Link from the list to display the Flow Sheet details.

  5. Blood Group, RH Factor, Antibody, Triage Detail (for Emergency Patients), Research Info , Growth Chart, View Future Bookings/Appointments are also displayed if recorded for the patient.

  6. Click [Refresh] to refresh the screen. If there is any new Clinical data recorded for the patient, it is reflected in the corresponding section of the Chart Summary.

  7. MDS Criteria Met’ or ‘MDS Criteria Not Met’ condition is displayed next to 'Refresh' link if MDS Data set is selected as mandatory for all the patient class in ‘MDS for Patient Class’ screen. This indicates Physician that all the activities configured for Minimum Data Set (MDS) are captured or not. Click ‘MDS Criteria Not Met’ hyperlink to view the list of pending MDS activities for the patient.

 

The Patient chart Sections are displayed as defined in the Chart Summary Template function associated to the Patient Class, Specialty and Practitioner in Assign Chart Summary function.

 

Lab results received from external labs are uploaded using attach image function in the result entry function in labs. You can view the uploaded image by clicking the View Image hyperlink.

Current Medications

The details of the Medication that the patient is currently using are displayed. The details displayed are the Generic Name (original name of the drug) of the drug prescribed, the Dosage and the Duration for which the Drug has been taken in terms of the start date and the end date of the prescription. The drug is displayed in this section as long as the end date is well within the system date time. Once the prescription expires, i.e. the end date exceeds the system date then the drug does appear in this section.

  1. Click the  icon, Pharmacy Orders dialog gets displayed. You can place orders for the respective patient in the dialog.

  2. Click the  icon, Medication Administration Details dialog gets displayed. You can place orders for the respective patient in the dialog.

  3. Click the  icon, Patient Drug Profile dialog gets displayed. You can place orders for the respective patient in the dialog.

  4. Click the  icon to view all the current medications of the selected patient in a modal window.

  5. If you select the Allow Renew from Chart Summary check box, a new check box is displayed next to the drugs. You can select the new check box against the drugs that need to be renewed. If you mouse over the new check box, a tool tip ‘Renew Order’ is displayed. You can renew the order by selecting the drugs that need to be renewed and clicking on the existing Pharmacy Orders icon.

  6. Upon clicking the Pharmacy Orders icon, the Pharmacy Orders screen is displayed with the details of the renewed drug(s) defaulted. The End Date of the drug before renew plus one minute will be defaulted as the Start Date of the renewed drug. If the user clicks on the existing Pharmacy Orders icon without selecting the new check box, the existing functionality will continue.

This is applicable only for Normal Rx and for Pharmacy drugs within the Chart Summary.

Diagnosis

The active Diagnosis recorded for the patient is displayed, if any. The details like Terminology Set, Diagnosis/Problem and Description are displayed.

If there is any High Risk Condition associated with the sensitive diagnosis after recording, a High Risk icon gets displayed in the View Chart Summary screen in Diagnosis widget.

This icon gets displayed when a practitioner records a sensitive diagnosis and if there is any high risk condition associated with the sensitive diagnosis in Terminology Code function in Medical Records module.

Encounters

The details of the last few encounters that the Patient had in any of the facilities in the enterprise are displayed. The details include the Encounter Date, nature of encounter, Facility, service provided, specialty and attending practitioner. The encounter details are displayed as a hyperlink. On clicking the hyperlink, the system shifts the control to the ‘Clinical Events’ function where the clinical details recorded for the encounter is displayed in a tree structure with the dates when the events were performed forming the nodes of the tree. On clicking each of the nodes, the details recorded on the particular date is displayed. The details are populated from the clinical repository (history) of the patient but are restricted only to the selected encounter. The events can also be seen in the flow sheet format.

 

The encounters related to current speciality are displayed under ‘Current Speciality Related’ header while encounters related to other speciality are displayed under ‘Other Speciality Related’ header in Encounter widget if Grouping by Current Speciality check box is selected in the Widget Parameter screen.

Results

All patient results (that is, resulted status as completed including amended results) are displayed under results in the patient chart summary.

The Result details of the selected patient are displayed under respective History Types. The Resulted Date, Event Description, Resulted Time and indicator icons are displayed. Place the cursor over an Event Description to highlight the event in another colour. Click the Event description to display details of the result. Result details displayed for the History Type Laboratory are Event description, Normal Range, Reported by, Patient class, Resulted date/time. Click image icons to display the image and click text icon to display the text document. Click Monitoring icon to display the graph of the monitored data at different dates and times. Result details displayed for the History Type Radiology are Event description, Reported by, Patient class, Resulted date/time. Pending for Review results are displayed in bold. Click 'More Dtl' hyperlink to view all the results in a new window. Click [Close] to exit from the more details screen.

The results are displayed in the Chart Summary component based on factors such as Patient Class, Duration, Number of events, Categories.

The Patient Chart Summary lists down the current and the previous encounter irrespective of the Patient Class. The patient chart lists those Result types based on Order Category setup in Order Entry and Result Reporting module.

If there are no encounters then the Patient Chart Summary lists down the last 14 results irrespective of the encounter. Date/Time the specimen is collected, is displayed by default, along with Date/Time the specimen sample was is resulted and recorded.

Click the Component Header 'Results' which is a hyperlink to display the Results function.

  1. Click the  icon, Lab Results dialog gets displayed. You can view and record results for respective patient in the dialog.

  2. Click the  icon Radiology Results dialog gets displayed. You can view and record results for respective patient in the dialog.

  3. Click this  icon to view the operation theatre results. Click the icon, OT Results dialog gets displayed. You can record results for respective patient in the dialog.

  4. Click this  icon to view all the results of selected patient in a modal window.

Critical Lab Alert

This widget displays all the lab investigation in a specific order where investigation marked as critical will be displayed first followed by Notification – alert result and finally Rejected investigations.

Click the  icon to view the critical lab alerts in a modal window.

 

Audit Trail hyperlink ‘A’

  1. Click audit trail hyperlink ‘A’ to view the audit history of the lab investigations in a ‘View Audit Trail’ dialog.

 

Confirm hyperlink

  1. Click Confirm hyperlink to review as well as forward the lab investigation to single or multiple practitioners for review. On clicking Confirm hyperlink, the ‘Review Result’ dialog gets displayed.

  2. Enter the Remarks and click [Complete Review] to mark the respective investigation as reviewed. The lab investigation status gets changed as reviewed and appears in disabled mode without any Confirm link in the widget.

  3. If you want to forward the lab investigation to other practitioner for review, click [Forward]. The Forward to Practitioner dialog gets displayed.

  4. Select single or multiple practitioners and click [OK] to forward the review request. After forwarding the review request, the system also prompts you to complete review for selected investigation.

  5. Click [OK] to complete the review or [Cancel] to complete it later. If you click [Cancel], the status of the lab investigation does not change and the system displays “Confirm” hyperlink for review.

The system shows the Tool tip “FORWARD/COMPLETE REVIEW” when you mouse hover the “Confirm” hyperlink.
