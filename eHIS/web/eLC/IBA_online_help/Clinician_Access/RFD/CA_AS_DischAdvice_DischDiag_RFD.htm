On invoking the function, the diagnosis profile of the patient is displayed, which includes Diagnosis Description, Accuracy, Encounter Stage, onset date of the diagnosis, Status and Detail. This section is empty, if the patient has no diagnosis (problems) recorded in the facility or enterprise. The diagnoses displayed are only the active diagnosis for the patient that may have been recorded across encounters. The records displayed are grouped by the diagnosis set.

 

Update Diagnosis Profile

  1. Diagnosis Set - Select the Diagnosis Set from the list. The list displays the Terminology Sets as defined in the Terminology Set function that is associated for a practitioner, facility and specialty in Medical Record module. On selecting the Diagnosis Set, the screen as shown is displayed.

 

Enter the following details:

  1. Diagnosis – Enter the Diagnosis code or click [?] function to search and select the desired diagnosis. This field is mandatory. On selecting [?] function to display the Search Tab of the screen, by default.

On selecting the Search By criteria, enter Search Text and click [Search] to display diagnosis codes. Based on the nature of the diagnosis selected, the system displays a hyperlink to select from the associated supporting diagnoses. Based on the nature of the diagnosis selected (Dagger Code or Consequence) the system enables you to associate the Manifestation/External Cause.

Click the Select Tab to select codes from the classifications defined in the Terminology Code function in Terminology & Coding Management module.

The codes as defined for the terminology sets in the function Terminology Codes are displayed. Select Code hyperlink. Based on the nature of the diagnosis selected the system displays a hyperlink for the user to select from the associated supporting diagnoses. Based on the nature of the diagnosis selected (Dagger Code or Consequence) the system enables associating the Manifestation/External Cause.

[+] is enabled, if the chosen diagnosis code has an indicator set as either ‘D’ or ‘C’ which indicate ‘Dagger’ and ‘Consequence’ respectively. This setup is done in the ‘ICD Code’ master function in the Terminology and Coding Management module. Select the supporting diagnosis to be associated with the main diagnosis and click [OK].

The selected code is displayed in the Diagnosis Field.

Click [!] corresponding to the diagnosis field to view the description of the diagnosis as set in MR Terminology Code setup. On viewing the description, click [Close].

  1. Encounter Stage – Encounter Stage field is displayed, if you are recording diagnosis for an inpatient. Select the desired encounter stage from the list. The valid values are ‘Admission’, ’Intermediate’ and ‘Discharge’.

  2. Nature - Select the nature of diagnosis as Actual, Potential or Wellness. This field is mandatory.

  3. Accuracy – Select Accuracy from the list. The Accuracies are defined in the Accuracy function in the MR Terminology Set. On selecting the Accuracy, Differential Group hyperlink is displayed. Select the hyperlink to view a screen. Select the Group 1 radio button or if the diagnosis is to be recorded under a new group, select New Group radio button. On selecting New Group radio button, Group 2 is displayed, which can be selected. Click [Close] to exit from the screen.

  4. Severity – Select Moderate, Mild, Severe or Extreme the severity from the list. This field is mandatory.

  5. Priority – Select priority as Normal, High or Critical. This field is mandatory.

Select ‘As Evidenced By’ hyperlink.

The Defining Characteristics are displayed, which can be selected. The characteristics are defined as displayed in ‘Factors for Terminology Code’ function. On selecting the Defining Characteristics, click [OK] or click [Close] to exit from the screen.

  1. Onset Type – Select the Onset Type as Acute or Chronic from the list. This field is mandatory. Not Applicable is displayed, by default.

  2. Onset Date – This field denotes the date from which the Patient has the problem. The current system date is displayed by default, which can be changed to an earlier date. To change the date, choose the date form the Calendar function. This field is mandatory.

  3. Since – This field is updated automatically, on entering the Onset Date.

  4. Status – The Status displayed is Active by default. You  cannot modify the status while recording the current diagnosis. Other available statuses are 'Resolved' and 'In Error'.

  5. As On – The current date is displayed for the status.

  6. Include in Current Encounter – The check box is selected and disabled, which indicates that the diagnosis is included in the current encounter.

  7. Remarks – Enter the remarks, if any.

On recording the diagnosis, the data entry part of the screen is cleared and the screen is refreshed to include the new problem assessment by the practitioner as part of the ‘Current Problems’ list.
