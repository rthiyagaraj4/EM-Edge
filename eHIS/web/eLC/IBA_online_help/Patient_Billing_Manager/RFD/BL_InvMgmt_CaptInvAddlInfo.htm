Enter the following details to capture additional information:

 

  1. Episode Type – The Episode Type of the patient gets displayed, by default.

  2. Episode ID – The Episode ID of the patient gets listed, by default.

  3. Visit ID – The Visit ID of the patient gets displayed, by default.

  4. Patient ID – Enter the Patient ID in alphanumeric format with a maximum of 10 characters. The patient name, gender, age, the clinic to which the patient is admitted, admission date, department, admission type, name of the attending practitioner for the selected Episode gets displayed.

 

The Customer, Policy Type, Policy Number, referral sources and the claim code associated to the patient gets displayed. The Claim Code and the amount approved by the Insurance Company for the selected patient is also displayed along with the number of days until when the amount is valid. The referral source details are displayed, if the referral source is set as mandatory for the respective Billing Group and the referral details are captured. Place the mouse on the    icon to display the long name of the External or Referral Source recorded in ‘External/Referral Source’ function of Application Masters module. The  icon appears on the screen if the long name for the External Referral Source is recorded in ‘External/Referral Source’ function of Application Masters module.

 

Additional Invoice Info

You have to capture the following details before the invoice generation. If the details are already captured, [V] is displayed adjacent to the selected record indicating that the additional details are successfully captured for the selected record. If the details are not captured [C] is displayed indicating that the details are to be captured before generating the invoice.

  1. [Populate Additional Info] – Click [Populate Additional Info] to populate the additional details setup for the Payer Group, Payer and the Policy Type combination defined using the ‘Setup Additional Info by Payer/Policy’ tab in the ‘Setup Invoice Additional Info’ function. You can associate additional data, which is not associated to the Payer Group, Payer and the Policy Type combination by selecting the values from the list.

  2. SL No – Enter the Serial Number in numeric format in four digits of maximum length. This identifies the order in which the additional information has to appear in the invoice generated.

  3. Info ID - Click [List of Values] to select an Information ID. The Info ID’s created using the ‘Setup Data for Additional Info Identifiers’ tab using the ‘Setup Invoice Additional Info’ function gets listed.

  4. Code – Click [List of Values] to select a Code. The Codes defined for the selected Info ID gets listed.

  5. Description – The description for the selected Code gets displayed.

  6. Data – The data set while defining the Info ID using the ‘Additional Info Identifiers’ tab in the ‘Setup Invoice Additional Info’ with the ‘Data Set Available’ check box as cleared gets displayed.

  7. Mandatory – Select the check box, if the additional invoice detail is mandatory while generating the invoice. The additional details are not mandatory, if the check box is cleared.
