Interface to GL

The GL Interface code serves as a link between the Patient Billing Manager module and the General Ledger module. The Bill Type has to be interfaced to the General Ledger module for the transaction to take place.

 

Enter the following details to interface a Bill Type to the GL codes:

 

  1. Bill Type - The selected Bill Type appears automatically when the Bill Type is defined under the Bill Type tab.

 

Control A/C Codes for

  1. Bill - Click [List of Values] to view and to select a valid GL Interface code. The values beginning with 'R' only gets listed. 'R' indicates that the account code is a receivable account. The posting account defined against this GL interface code is debited while posting billing charges to General Ledger module. The selected posting account is credited while settling the bill or transferring the bill to Accounts Receivable module.

  2. A/R - Click [List of Values] to view and to select a valid GL Interface code. The values beginning with 'R' only get listed. 'R' indicates that the account code is a receivable account. The posting account defined against this GL interface code is debited while transferring the deferred patient's bill from Billing to Credit Management module and the Billing Receivable control account will be credited.

  3. DCP - Click [List of Values] to view and to select a valid GL Interface code. The GL Interface code signifies the Billing Receivable Control Account for Deferred Cash Patients "R" indicates that it is a receivable account. The posting account defined against this GL interface code is debited while transferring the Cash patient's bill from Billing to Deferred Cash Patient and the Billing Receivable control account is credited.

  4. Bill Rounding - Click [List of Values] to view and to select a valid GL Interface code. The Control Account signifies the bill rounding. This is a mandatory field.

  5. Discount – Click [List of Values] to select a Discount Account. The values defined in the General Ledger module get listed. This is a mandatory field.

  6. Adjustment - Click [List of Values] to select an Adjustment Account. The values defined in the General Ledger module get listed. This is a mandatory field.

  7. Exemption - Click [List of Values] to select an Exemption Account. The values defined in the General Ledger module get listed. This is a mandatory field.

  8. Write-Off - Click [List of Values] to select a Write-Off account. The values defined in the General Ledger module get listed. This is a mandatory field.

  9. Eff. From Date – Click Calendar function to select a date. The date signifies the day from when the GL interface codes are effective for the bill type. The date selected has to be greater than the system date and should not overlap with the existing definitions.

  10. Eff. To Date - Enter the date until when the GL Interface code is effective for the bill type. The date entered has to be greater than the effective from date.

 

Control Account Descriptions for

The description for all the control accounts appears in the appropriate text box when the Control codes are set.

 

Control A/C by Payers

You have to associate Control Accounts to individual Payers.

 

  1. Payer Code – Click [List of Values] to select a Payer. This is a mandatory field.

  2. Payer Name – The description for the selected Payer gets displayed.

  3. Discount – Click [List of Values] to associate a Discount accounting code to the Payer. This is a mandatory field.

  4. Adjustment - Click [List of Values] to associate an Adjustment code to the Payer. This is a mandatory field.

  5. Exemption - Click [List of Values] to associate a valid Exemption code to the Payer. This is a mandatory field.

  6. Write Off - Click [List of Values] to associate a valid Write-Off accounting code to the Payer. This is a mandatory field.

 

Control A/C Description

The description for the selected Control Accounts associated with to Payers gets displayed.
