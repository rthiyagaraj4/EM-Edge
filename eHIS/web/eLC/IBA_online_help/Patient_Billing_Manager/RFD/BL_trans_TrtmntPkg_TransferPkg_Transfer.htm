Enter the following details to transfer a package:

 

Package Details

The packages that are subscribed for the patient and are open are listed.

  1. Package Code - Click [List of Values] to select and transfer a package. The selected package details and package service details are displayed.

  2. Cancel Remarks - Click [List of Values] to select remarks for cancellation. The description of the selected remark is displayed. The reasons that are defined through Reasons for Billing Actions function are listed.

  3. Cancel Unprocessed Orders - Select the check box 'Cancel Unprocessed Orders' to cancel the orders that are not processed.

  4. [Exc] - Click [Exc] to view the services that are excluded from the package. The services that are excluded through the 'Maintain Packages' function for the respective package are listed.

 

Transfer Details

  1. Package Code - Click [List of Values] to select a package that has to be transferred. The description of the selected package is displayed. The values that are defined through the 'Package Definition' function are listed. The package that is subscribed or associated to the patient encounter are not listed. Cross-transfer of package for episode restricted package are not listed. Press [Tab] to record Effective Date range.

  2. Effective Date [From - To] - Click the Calendar function to select the effective date. The 'To' is computed and displayed based on the 'valid days' specified during package definition. This is a mandatory field.

  3. The Billing Class, Customer Group and Customer code and description, package amount, episode type and number are displayed. Press [Tab] to record remarks.

  4. Remarks - Click [List of Values] to record the remarks for package transfer. The values recorded through the 'Actions for Billing Actions' function with the 'Cancel Package' check box as selected are listed. This is a mandatory field.

  5. Click [Commit Form] or press [F10] to save the records or click [Clear Record] to clear the data.

  6. Enter Receipts/Refunds screen is displayed for an inpatient or outpatient encounter in the facility. The deposit amount is mandatory for a package if it is recorded as mandatory during package definition. You can close the screen and proceed with the transaction if deposit amount is not mandatory for the selected package. If the deposit amount is mandatory, you cannot proceed with the transaction. Refund screen is displayed only if the deposit amount is collected against the package. The 'Receipt/Refund' field value is defaulted to 'Receipt'. The package sequence number and the amount settled are displayed. Select the Receipt Type and enter the details. Amount given cannot be lesser than the settled amount. Allocated amount must be equal to the settled amount.

If the deposit that has to be accepted from the patient is less than the minimum deposit of the transferred package, the ‘Enter OP Receipts/Refunds’ or the ‘Enter IP Receipts/Refunds’ functional screen is displayed based on the patient class to accept patient deposit. The ‘Nature’ value is defaulted to ‘Patient Deposit’ and the ‘Episode Type’ is defaulted to ‘External’. The ‘Receipt Type’ value is defaulted based on the ‘Package Parameter’.

=> System displays an alert message ‘Minimum Deposit insufficient for transfer package’, if the available deposit is less than the defined minimum deposit. You cannot transfer the package, if the minimum amount is not collected.

  1. You can also issue a refund through the Enter Receipts/Refunds screen for an inpatient or outpatient encounter in the facility. The 'Receipt/Refund' field value is defaulted to 'Refund'. The package sequence number and the amount settled are displayed. Select the Receipt Type and enter the details. Amount given cannot be lesser than the settled amount.

You have to log-in to a valid cash counter to accept deposits and issue refund for a package.

=> An alert message ‘Do you want to refund the deposit’ is displayed, if the minimum deposit is not applicable for the transferred package or if the deposit amount is more than the minimum deposit amount specified for the respective package. Click [Yes] to issue a refund or [No] to internally transfer the excess amount as the deposit amount against the patient ID that can be utilized for settlements in the future encounters. The ‘Nature’ value is defaulted to ‘Patient Deposit’ and the ‘Episode Type’ is defaulted to ‘External’. The ‘Receipt Type’ value is defaulted based on the ‘Package Parameter’.

  1. A receipt number is generated for the deposit transaction.
