Enter the following to generate discharge bill cum receipt:

 

  1. Admission No – Enter the Admission Number in numeric format of eight digits maximum length. This field is mandatory. The episode type is displayed.

  2. Patient No. – Enter the patient number in alphanumeric format of 20 characters maximum length. On entering a valid patient number, patient name, Gender and age will be displayed. Patient lookup is provided.

  3. Facility – Select the value from the list. For the logged in user ID, associated facilities will be displayed as list items.

  4. Inpatient Bill Type – The values are system defined – “Pending Interim Bill”, “Pending Discharge Bill” and “Pending Undercharge Bill”. Select one of the Bill Types to generate the bill.

  5. Customer – If any Customer is associated with the Patient during registration, the details are displayed. Else, you can click [List of Values] to set the Customer with the Patient record. You can select a Customer from the list when the Bill Generation Option is set to Customer.

  6. Facility - The facility id must be selected for the patient who has undertaken the service in the concerned facility.

  7. Age - The age of the patient will be shown in years, months and the days.

  8. Sex - The Gender of the patient will be indicated in the radio button as Male/Female.

  9. User – The name of the user logged-in to the Cash Counter is displayed.

  10. Cash Counter/Shift – The Cash Counter code along with the Shift details is displayed.

 

Search Patient Visit

This tab is used to search the inpatient admission for selection. The following is the selection criteria:

  1. Location Type – The values are system defined – “Clinic” and “Ward”. Select one of the values from the list. The details are displayed for the Location Type selected.

  2. Location Code – Select a Location Code from [List of Values]. The values are populated in the list based on the Location Type selected.

  3. Episode Type – The values are system defined. Select a value from the list.

  4. Option – The values selected for the ‘Inpatient Bill Type’ is defaulted.

  5. Encounter ID – Enter the Encounter ID of the respective patient. The visit details for the specific encounter are displayed.

  6. Encounter Date Range – Click Calendar function to set the date. This field is mandatory. The details for the encounter between the specified date range are displayed.

  7. Patient No – Enter the Patient Number in alphanumeric format of 20 characters, to see the admission of particular patient.

  8. NRIC – Enter the NRIC number for the respective Patient ID.

  9. Ward Code – Enter the Ward code in alphanumeric format of four characters maximum length. This can be selected from the list of valid values. Leave blank if all the records need to be displayed.

  10. [Search] – Click this button to display the details in the detail block. The details displayed will be of Admission Date, Patient No, Ward Code and description, episode type, Admission No and Unbilled amount. To select a particular admission number for raising receipt, double-click on the specific record from the details displayed.

 

Generate Bill Cum Receipt

In this tab page only the services that are not billed in the current admission can be processed.

  • Enter the Admission number to be billed, if the Admission number is known. If the facility Id is not selected, the system will force the user to select the facility. For the first time of opening the function/when [Clear] is clicked in this page, the system will default to the current facility id in the ‘Facility name’ list.

  • The services associated to the admission to be billed are listed in the detail block. If any one of the service is unconfirmed, then the system will throw an error for the ‘Unconfirmed service’ and will not allow for billing.

  • All the patients outstanding/unbilled amount will be shown in the top frame of the ‘Patients outstanding’ across all the facilities.

  • After pressing the tab key/enter key in the patient ID field, the system will display all the services that are to be billed.

  • Select the include indicator (check box) in the details, whether the service has to be included for the billing/not. Select the check box to include billing. Clear the check box, if billing is not involved.

  • All the patient’s portion/Customers portions are shown and the net payable amount by the patient is shown in the separate field.

  • Enter the amount that has to be paid by the patient. The amount entered here will be validated against the ‘Total. Net amount payable by the patient’. If the amount is greater than the above indicated field, the system will raise warning and it will not allow for billing.

  • Enter the settlement type field. The settlement type may be in liquid Cash or through the Cash instruments like Cheque/Credit Card.

  • Click on the process button. The system will pop-up the message that the bill has been processed and prompts the cashier to note the bill number/receipt number. When clicked ok, the system will clear off the screen.

  • Patient Outstanding details of Cash Bills (to be payable by the patient) Amount, Unbilled Amount and Deposit Amount will be displayed.

  • On entering the Valid Admission and Patient no, the service availed details is displayed. The Fields that are displayed are Service, Description, Discount, Net Amount, Bill Amount and Deposits. You can also select the service location from the list provided.

 

Patient Previous Outstanding

This frame consists of the following fields:

  1. Cash Bills - The Patient’s outstanding Bill amount is displayed for user verification. You cannot change this value. It will display the only bills that are payable by the patient.

  2. Customer [From-To] – Click [List of Values] to select the customers range. The search visits of the customers will be displayed.

  3. Unbilled - The Patient’s outstanding Unbilled amount is displayed for user verification. You cannot change this value.

  4. Deposits - The Patient’s outstanding Deposit amount is displayed for user verification. You cannot change this value.

  5. Payable Amount – The amount payable by the patient is displayed and cannot be changed.

  6. Deposits – The unadjusted deposits are displayed for the visit and cannot be changed.

 

Settlement Details Frame

  1. Amount Settled - Enter the document amount for this transaction. The amount should be positive only. the amount should not be greater than the original document's outstanding amount. For this comparison, the absolute value of the amounts is considered. This field is mandatory.

  2. Settlement Type - Enter a valid settlement type code. Click [List of Values] to view and select a valid settlement type code. If the code entered is valid, then the short description is displayed for user verification. If settlement type is Cash settlement, then the settlement reference and remarks are blanked out and not accepted. If settlement type is not Cash oriented, then settlement reference and remarks are accepted.

  3. Cheque/Card No - Enter the Cheque number or card number in alphanumeric format of 20 characters maximum length. If settlement type is non Cash, then these are mandatory. Otherwise, this should be left blanks. Enter the details of the Cheque or a Credit Card. For example, if the settlement is by Credit Card, enter the Credit Card identity, and expiry date of the Credit Card. If the settlement type is by Cheque, enter the Cheque details and the date.

  4. Date – Click Calendar function to set a valid date. Enter the date of the Cheque, if the settlement type is Cheque or if the settlement type is Credit Card, enter the date of the expiry of the Credit Card.

  5. Bank/Card Co. Name/Branch – Enter the Bank or Credit Card company name in alphanumeric format of 30 characters maximum length. If settlement type is non Cash, then this is mandatory. Otherwise, this should be left blanks. For example, if the settlement is by Credit Card, enter the Credit Card company name. If the settlement is by Cheque enter the banks name or banks’ branch name. Click [List of Values] to view and select an approved Credit Card company name or an approved bank or branch.

  6. Online Approval – If the check box is selected, Credit Card entered receipt will be forwarded to the Credit Card Company immediately. If the check box is cleared, Credit Card receipts will be forwarded as batch. In case of Online Approval, Batch No. information to be filled manually.

  7. Batch No – Enter the batch number if settlement type is of Credit Card.

  8. Approval Code – Enter the Approval Code with a maximum of 30 characters. If the settlement is made through Credit Card, Approval code of the Credit Card to be entered here.

  9. Sale Draft No. – Enter the Sale Draft Number with a maximum of 20 characters. When settlement is made by Credit Card, sale draft number details captured at the time of swiping to be recorded here.

  10. Payer’s Name - Enter the payer name with a maximum of 40 characters. This field is to be entered if the receipt nature code is ‘MI’.

  11. Payer’s Ref Id – Enter the Payer’s Reference Id with a maximum of 30 characters. When Payer name is captured, is having any reference ID, can be entered as input for this field.

  12. Through Mail – The receipts received through post, for the settlement of bill or deposit can be recorded by selecting the check box. If the check box is selected, Medical Report Payment receipt is printed.

  13. Remarks – Enter the remarks with a maximum of 30 characters.

  14. Receipt Type - Enter a valid receipt type code. Click [List of Values] to view and select a valid receipt type code. If the code entered is valid, then the short description is displayed for user verification. The account code and department is displayed.

 

Unbilled & Outstanding Details

Tot Outstanding & Unbilled

After entering the patient ID and admission number, if there are any patients outstanding/unbilled exist, this tab page will be enabled. The aim the tab page is for the information purpose only. This gives the information like patients total outstanding bills/unbilled. Click on the tab “Outstanding bills/Unbilled Admission Details” to see the details.

 

  1. Outstanding bills - The patients total outstanding amount gets displayed.

  2. Unbilled - The patients total unbilled gets displayed.

 

Outstanding bills

This page lists all the details of the outstanding bills and generates the consolidated receipts. The outstanding records are displayed. On double-click, all the bills are considered for generating a consolidated receipt.

 

Unbilled Admission

The aim of the tab page is to provide the information for the unbilled episodes across the facilities and to bill the corresponding episode ‘Unbilled services’. Click on the Unbilled admission to list the ‘Unbilled admission’. In the first tab page under the ‘Unbilled admission details’ will be ‘Unbilled Admission’, which list all the admissions which are not billed.

Select the admission to view all the services for that admission. Click on the sibling tab ‘Unbilled service request of an unbilled admission’. The sibling tab will display all the services for that selected admission. When clicking on the sibling tab ‘Unbilled service request of an unbilled admission’, if there is any ‘Unconfirmed services’, the system will generate the warning and never allow to bill the services for that selected admission.

  1. Facility Id - The facility id where the patient has undergone the services is displayed.

  2. Episode date - The facility id where the patient has undergone the services is displayed.

  3. Episode type - The episode type will be ‘O’ for the outpatients, ‘E’ for Emergency and ‘R’ for the ‘External patients’.

  4. Episode no. - The episode number of the patients encounter gets displayed.

  5. Visit no. - The visit number of the patients encounter gets displayed.

  6. Gross Amt - Total gross amount for all the services gets displayed.

  7. Discount Amt - Total discount amount for all the services gets displayed.

  8. Net. Amt - Total Net. Amount for all the services gets displayed.

 

Unbilled service request of an unbilled admission

The aim of the tab page is to provide the information for the unbilled services for the selected episode in the previous sibling tab page.

  1. Module Id - The module id, which has posted the transaction in billing, is displayed.

  2. Service date - The date on which the service is availed.

  3. Transaction date - The date on which the transaction is entered.

  4. Billing Service - The service availed by the patient. The service description is also displayed.

  5. Service Qty - The service quantity provided to the patient gets displayed.

  6. Gross Amt - Gross amount for all the service gets displayed.

  7. Discount Amt - Discount amount for all the service gets displayed.

  8. Net. Amt - Net Amount for all the service gets displayed.

 

Unconfirmed Services

After entering the patient ID/episode id/visit id if there are any services, which are not confirmed, this tab page will be enabled. This gives the information of the services, which are not confirmed. The unconfirmed services are listed.

 

Future Orders

If any request for future orders by the patient, the details can be viewed here.
