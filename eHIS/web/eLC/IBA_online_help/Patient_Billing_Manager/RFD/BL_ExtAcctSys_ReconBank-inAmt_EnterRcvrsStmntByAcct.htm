Enter the following details to record receiver's statement:

 

Statement Header

  1. Statement Ref – Enter the Statement Reference in alphanumeric format of 20 characters maximum length. A Statement Reference number has to be unique and it identifies each individual transaction. This is a mandatory field.

  2. Statement Date – Click the Calendar function to select a date. This identifies the date when the Statement is generated. This is a mandatory field.

  3. Account Code – Click Account Code for which the Statement is prepared. This is a mandatory field.

  4. Cut Off Date – Click the Calendar function to select the Cut-Off Date. The cut-off date indicates the date upto, which the statement is received. The Cut Off Date selected has to be equal to or lesser than the Statement Date. This is a mandatory field.

  5. Total No of Entries – Enter the Total number of entries in the numeric format. This indicates the number of individual statement details, which can be entered in a single statement. This will be validated with the count of detailed records entered in “Statement Details” while committing the statement.

  6. Opening Bal – Enter the Opening Balance in numeric format. The Opening Balance identifies the cash collected at the beginning of the day for the respective Account Code. System defaults Previous Closing Balance as the Opening Balance for entries following the first entry.

  7. Closing Bal – Enter the Closing Balance in numeric format. The Closing Balance identifies the amount of cash collected at the end of the day for the respective Account Code. System calculates the Closing Balance automatically based on the Net Amount for each statement. System verifies the amount while committing the statement. Closing Balance is calculated as the [Sum of Opening Balance + Transaction Total]. A warning message ‘Closing Balance does not match with the Sum of Opening Balance and Transaction Total’ is raised and the statements are reconciled, if the closing balance does not match. Click [OK] to recalculate and display the Closing Balance.

  8. Cash Book Bal - The previous month account balance is displayed from cash book for the account selected in Account Code based on the statement date selected. If the amount is negative in cash book bal field, the text ‘CR’ will be displayed next to the cash book balance amount. If the amount is positive, the text ‘DR’ will be displayed next to cash book balance amount. If the amount is zero, the text ‘DR’ will be displayed next to the cash book balance amount. You cannot edit the value in this text box.

Statement Details

  1. SI – Enter the Serial Number in numeric format of three digits maximum length. This is a mandatory field.

  2. Bank Name – Right click on this field and select ‘Bank-in Documents’ option to invoke the BANK_IN_DOC screen . A window pops-up with multiple transaction details in the associated banks. This indicates that the transaction is processed in the respective Bank. The document reference and transaction date gets displayed automatically, if the value is selected from the list. The name of the Bank is displayed.

    Click [Execute Query] or press [F8] to view the details.

    1. Bank Code - The Bank code is listed indicating the transactions processed.

    2. Document Ref - The document reference number is displayed for the respective bank.

    3. Trx Date - The date when the transaction is processed gets displayed.

    4. Mode of Payment - The mode of payment used is displayed.  

    5. Credit - The credit amount is displayed.

    6. Select - Select the check box adjacent to the record to associate it. You can associate multiple records to the statement reference by selecting the check boxes.

    Note: You can only associate banks with the deposit facility. The banks are set as valid for deposit by selecting the check box 'Use for Deposit Bank-in' in the 'Banks and Branches' function.

    The total number of individual records has to match with the 'Total No of Entries'.

    The following details are displayed:

    1. Bank - The Bank name is listed indicating the transactions processed.

    2. Bank-in Doc – The Bank-in Doc name in which the transaction has been processed is displayed.

    3. Bank-in Date - The date in which the transaction has been processed is displayed.

    4. Receipt Date – The date of receipt that is received from the bank is displayed.

    5. Bank-in Amount - The total document amount is displayed.

    6. Mode of Payment - The mode of payment used is displayed.

    7. Select – Select this check box against the transaction to be reconciled. You can select more than one record.

    8. Click [OK] to display the selected records in the Enter Receiver’s Statement by Account screen or click [Cancel] to close the screen.

  3. Document Ref – Enter the Document Reference Details in alphanumeric format of 20 characters maximum length. The Reference details are displayed when generating the Bank-in Batch. The same reference number can be specified here. The value gets displayed automatically, if the Bank Name is selected from the [List of Values]. This is a mandatory field.

  4. Trx Date – Click the Calendar function to select a transaction date. This indicates the date when the transaction is processed in the respective Bank. The transaction date has to be lesser than the Cut-off Date. This is a mandatory field.

  5. Mode of Payment – Click [List of Values] to select a Mode of Payment. This identifies the settlement type for the relevant transaction in the respective Bank. The description of the selected Payment Mode is displayed.

  6. Debit – Enter the amount deducted from the relevant Bank after the transactions.

  7. Credit – Enter the amount credited to the Bank after the transactions.

  8. Net Amt – The net amount is displayed automatically. You cannot update the value. Negative sign is  displayed to indicate credit transactions and debit transactions are displayed using positive signs. This is a mandatory field.

  9. Reconciled Date – The date when the individual statements are reconciled is displayed. Status of the record is displayed here as reconciled or not yet reconciled, when the data is viewed in query mode. The check box is selected, if the record is reconciled.

  10. Total – The Total amount is displayed.
