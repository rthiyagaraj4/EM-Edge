Enter the following details to define a Billing Group:

 

  1. Billing Group - Enter the Billing Group in alphanumeric format of two characters maximum length. A Billing Group Code has to be unique and cannot be changed after creation. This is a mandatory field.

  2. Long Description - Enter the Long Description in alphanumeric format of 60 characters maximum length. This is a mandatory field.

  3. Short Desc - Enter the Short Description in alphanumeric format of 15 characters maximum length. This is a mandatory field.

  4. Explanatory Note - Enter an explanatory note for the Billing Group in alphanumeric format with a maximum of 2000 characters. The explanatory note is displayed on the screen. This is a mandatory field.

  5. Settlement Indicator - Select a Settlement Indicator from the list. The values are system defined - 'Cash Patient', 'Credit Patient' and 'Sponsored Patient'.

  6. Cash Settlement - Select Cash Settlement, if the Billing Group is applicable for Cash settlements.

  7. Credit Patient – Select Credit Patient, if the Billing Group is applicable for Credit settlements.

  8. Sponsored Patient – Select Sponsored Patient, if the Billing Group is applicable for Insurance policy holders or patients availing sponsorship from the Employer.

  1. Payer Settlement Type – The Payer Settlement Type indicates the mode of payment chosen by the payer to settle the service amount. You have an option to associate a Settlement Type to the Service. The values are system defined - 'Cash', 'Payment by Approved Company' and 'Payment by Insurance Company'.

  2. Cash – Select Cash, if the payer has to settle the amount by making a Cash payment.

  3. Payment by Approved Company – Select Payment by Approved Company, if the Employer or an Approved Company settles the amount.

  4. Payment by Insurance Company - Select Payment by Insurance Company, if the payment towards the service is settled by the Insurance Company.

  1. Credit Doc Ref required - Select the check box, if the Credit Document Reference is mandatory during patient registration. This is applicable only if the Settlement Indicator is set to Sponsor or Credit and not applicable if the Settlement Indicator is Cash.

  2. Basic Classification – Select the check box, if the Billing Group has to be listed during patient registration in the Financial Details tab. Clear the check box and the Billing Group is not listed for selection during patient registration.

  3. Assign during Admission – Select the check box, if the respective Billing Group has to be listed during inpatient admission to the Facility.

  4. Assign during Visit – Select the check box, if the Billing Group has to be during Outpatient Visit registration.

  5. Allow Discounts at Billing – Select the check box, if discounts are allowed for the respective Billing Group. Clear the check box and the bill cannot be modified for any discounts.

  6. Payer Employer ID Mandatory – Select the check box, if the Employer Name has to be recorded during patient registration, patient admission and visit registration in the Master Patient Index, Inpatient and Day Care Management and the Outpatient Management modules for an Insured patient. The Employer Name is made mandatory under the ‘Financial Details’ tab in the Register Patient, Admit Patient and Register Visit functions in the Master Patient Index, Inpatient and Day Care Management and the Outpatient Management modules. Clear the check box, if the Employer Name is not mandatory for an Insured Patient.

  7. Capture Referral Source – Select the check box, if the Referral Source details have to be captured for the Billing Group. If the check box is selected, the fields ‘Primary Referral Source’ and the ‘Secondary Referral Source’ Details have to be entered while registering the Visit in Outpatient Management module, patient registration in the Master Patient Index module or admitting patient in the Inpatient and Day Care Management module. Clear the check box to disable the fields during the transaction in the respective modules.

  8. Ref Source Main Mandatory – Select the check box, if primary referral source details are mandatory during patient registration. If the check box is cleared, primary referral source is not mandatory.

  9. Ref Source Sub Mandatory - Select the check box, if the secondary referral source details has to be captured during patient registration. If the check box is cleared, secondary referral source is not mandatory.

  10. Default Settlement Type - Click [List of Values] to associate a default Settlement Type to the Billing Group. On selecting the Billing Group during transaction in the external modules like during visit registration in Outpatient Management module, patient admission in the Inpatient and Day Care Management module, or patient registration in the Master Patient Index module, the Settlement Type is set automatically to the one selected. You have an option to select any other default Settlement Type during transaction.

  11. Assign during Patient Regn - Select the check box to assign the Billing Group to a patient during registration. Clear the check box and the respective Billing Group is not listed during patient registration.

  12. Assign during Day Care Admission – Select the check box to enable the respective Billing Group during Daycare Admission. The respective Billing Group gets listed during modification of financial details for a Daycare Admission. Clear the check box and the relevant Billing Group does not get listed during Daycare admission.

  13. Assign during Accident & Emergency Visit – Select the check box, if the relevant Billing Group has to get listed only for Accident and Emergency visits. Clear the check box to disable the Billing Group from the list for Accident and Emergency visits.

  14. Check for Cost Availability – Select the check box to enable the Cost indicator option. Clear the check box and you cannot specify the Cost Amount and the Reimbursable Amount towards the relevant Billing Group.

  15. Discount Applicable if Used as Non-Ins - Select the check box to define the Billing Group as 'Non-Insurance Billing Group'. You can also define the discount amount applicable for the specific Billing Group. The discount amount defined for the Billing Group is deducted from the standard price, when the Billing Group is used as a 'Non-Insurance Billing Group' in patient transactions.

  16. Accident Classification - Select a value from the list. The values are system defined – 'Not Applicable', 'Injury on Duty' and 'Motor Vehicle Accident'.This value is used to generate the reports having different classifications in circumstances of injury as indicated by the doctor in Ambulatory Care Management and Inpatient and Day Care Management modules.

  17. Not Applicable – Select Not Applicable, if the Billing Group is not applicable for Accident Classification.

  18. Injury on Duty – Select Injury on Duty to categorize the Billing Group as applicable for accident case, where the patient is injured on duty.

  19. Motor Vehicle Accident – Select Motor Vehicle Accident, if the Billing Group is used to classify the accident cases as Motor Vehicle Accident.

  1. Category – Select a Category from the list. The values are system defined. The Billing Group is categorized based on the entries set for Category.

  2. Unrestricted – Select Unrestricted, if the Billing Group is applicable to all the patients visiting the facility. The patient can be a valid citizen of the country or a non-national.

  3. Nationals – Select Nationals to enable the Billing Group to the valid citizens of the nation. The Billing Group is not applicable to other patients visiting the facility.

  4. Others – Select Others, if the relevant Billing Group is applicable to non-national patients.

  1. Level of Service Elig for Inpatient Only – Enter the level of service in numeric format with a maximum of two digits. The level of service indicates the priority of the selected Billing Group for Inpatients. The Bed Class gets listed in the ‘Hospital’s Request’ field while transferring or admitting the patient in the Inpatient and Day Care Management module, based on the level of service defined here and in the Billing Class function.

  2. Service Price in Currency – Click [List of Values] to associate a Foreign Currency to the Billing Group. The Currency associated to the respective Billing Group gets displayed in the ‘Price Class – Billing Group Definition’ tab in the ‘Price Class Specific Prices’ function.

  3. Package Patient – Select the check box, if the Billing Group is valid for package subscription. You can associate the Billing Group to a Price Class using the ‘Price Class Specific Prices’ function, if the check box is selected. Clear the check box, if the Billing Group is not package related.
