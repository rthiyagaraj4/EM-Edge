Enter the following details to create a Credit Card Batch:

 

  1. Batch No – The Document Type selected for ‘Manual Batches’ under the tab Credit Card Cycle in the BL parameters function gets displayed, by default. If the document number is system generated for the selected document type, you can navigate to Credit Card Type directly. If the check box ‘System Generated Document Number’ is selected in ‘Document Type Details’ function in the Financials Base Manager module, the document number is system generated for the selected Document Type. The document number is generated automatically by the system during batch generation. If the check box is cleared, the document number has to be generated manually and you have to enter a unique number for the respective Credit Card batch. This is a mandatory field.

  2. Credit Card Type – Click [List of Values] to select a Credit Card Type. The description for the selected Credit Card Type gets displayed. The batch is generated for the selected Credit Card Type.

  3. Batch Date – The system defaults the  current date as the Batch Date. You cannot update the Batch Date. This is a mandatory field.

  4. Batch Remarks – Enter the Batch Remarks in alphanumeric format of 100 characters maximum length. This is a mandatory field.

 

Credit Card Transactions for the Batch  

  1. Select All – Select the check box to select all the records populated in the detail block to generate a Credit Card Batch. Clear the check box and the records marked for selection are reverted to the status of unselected. If the check box is selected, the selected records amount is accumulated as the batch amount, which is displayed as “Total”.

  2. Select – Select the records, which are to be associated for, batch processing. The unselected records are not considered towards batch processing. If the check box is cleared, the document amount is accumulated as the batch amount, which is displayed as the “Total”.

  3. Doc Amount – The amount generated from the Credit Card transaction is populated in the column. You cannot change the amount. The amount collected from the patient and the receipt generated for the selected transaction gets displayed.

  4. Approval Ref No – The system defaults the entry to the Approval  number entered during receipt generation for Credit Card settlement. You cannot alter the data in the field.

  5. Credit Card Document Reference - The Credit Card receipt details like Document Type, Document Reference Number and the date are displayed.

  6. Patient No – The system defaults the patient ID  from whom the payment is collected.

  7. Patient Name - The system defaults the respective patient’s  name from whom the payment is collected. This is a display field and you cannot alter the data in the field.

  8. Total – The total amount is calculated automatically based on the Document Amount for the selected records. The Credit Card Batch is generated for the selected Credit Card Type and the Document Amount.
