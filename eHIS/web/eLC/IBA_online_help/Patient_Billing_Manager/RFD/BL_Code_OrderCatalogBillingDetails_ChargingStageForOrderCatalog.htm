Charging Stage for Order Catalog

The Charging Stage for Order Catalog tab enables you to select a charging stage for an Order Catalog. The charging stage defines when a patient has to be charged when an order is placed. The charges are computed at the selected charging stage.

 

Enter the following details to define the Charging Stage:

  1. Order Catalog Code - The Order Catalog Code selected under the tab ‘Services for Order Catalog’ gets displayed with the description.

 

All

The selected charge type with the description gets displayed.

  1. Order – Select the check box to charge all the patient class when the order is placed. Clear the check box to select each patient class manually to charge the patient when the order is being placed.

  2. Charge Future Order Later – Select the check box, if there are several orders placed for future visits and if the orders have to be charged only during the respective visit. If selected, the rule is applicable for all the patient class. Clear the check box, if the orders have to be charged during current visit.

  3. Regn – Select the check box to charge the order amount for all the patient class during order registration. Clear the check box to select each patient class manually to charge the patient when the order is being registered.

  4. Compl – Select the check box to charge the orders for all the patient class only when the orders are processed completely.

  5. Result – Select the check box, if the selected patient class has to be charged during result generation for an order placed. The orders are charged during result generation for all the patient class. Clear the check box to select each patient class manually to charge the patient when the result is reported.

  6. Patient Class - The patient class gets listed, by default.

  7. Description - The description for the individual patient class gets displayed.

  8. Order – Select the check box to charge the respective patient class when the orders are placed. If the check box is cleared, the patient is not charged for the orders when the order is placed.

  9. Charge Future Order Later – Select the check box to charge the future orders when the patient visits for future orders. Clear the check box to charge the orders when the order is placed.

  10. Charge on Regn – Select the check box to charge the patient during order during registration level. If the check box is cleared, the patient is not charged during order registration.

  11. Check for Payer – Select a value from the list. The values are system defined – ‘NA’ and ‘Billed’.  A prompt gets displayed indicating that the bill is not settled in the previous stages for the order by the payer, if the value is selected as ‘Billed’. If you select ‘NA’, the prompt is not generated even though the bill is not settled for the order registration. This is valid for the Payers settling the bill amount on behalf of the patient.

  12. Check For Patient – Select a value from the list. The values are system defined – ‘NA’, ‘Paid’ and ‘Billed’. You can select the stage when the alert message has to be displayed for the patient if the bill is not settled for the orders placed. This is valid for the patients’ settling the bill amount.

  13. Charge on Compl – Select the check box to charge the respective patient class when the order is processed completely. Clear the check box, if the charges are to be computed at any other stage of order processing.

  14. Check for Payer - Select a value from the list. The values are system defined – ‘NA’ and ‘Billed’. You can select the stage when the prompt has to be displayed, if the amount for the orders placed is not settled. This is valid for the Payers settling the bill amount on behalf of the patient.

  15. Check for Patient - Select a value from the list. The values are system defined – ‘NA’, ‘Paid’ and ‘Billed’. You can select the stage when the prompt has to be displayed if the amount for the orders placed is not settled. This is valid for the patients’ settling the bill amount.

  16. Charge on Result – Select the check box to charge the respective patient class only when the result is generated. Clear the check box, if the charges are to be computed at any other stage of order processing.

  17. Check for Payer - Select a value from the list. The values are system defined – ‘NA’ and ‘Billed’.  You can select the stage when the prompt has to be displayed, if the amount for the orders placed are not settled. This is valid for the Payers settling the bill amount on behalf of the patient.

  18. Check for Patient - Select a value from the list. The values are system defined – ‘NA’, ‘Paid’ and ‘Billed’. You can select the stage when the prompt has to be displayed if the amount for the orders placed is not settled. This is valid for the patients’ settling the bill amount.
