Settlement Type

Enter the following details to create the Settlement Types:

 

Settlement Type

  1. Settlement Type - Enter the Settlement Type code in alphanumeric format of two characters maximum length. This is a mandatory field.

  2. Description - Enter the Description in alphanumeric format of 30 characters maximum length. This is a mandatory field.

  3. Short Description - Enter the Short Description in alphanumeric format of 15 characters maximum length. This is a mandatory field.

  4. Settlement Nature - Select the Settlement Nature from the list. The values are system defined - ‘Cash’, ‘Credit Card’, ‘Cheque’, ‘Autopay’ and ‘Others’.

  5. Cash – Select Cash, if the mode of payment is Cash.

  6. Credit Card – Select Credit Card, if the mode of payment is Credit Card.

  7. Cheque – Select Cheque, if the payment is cleared through Cheque.

  8. Autopay – Select Autopay, if the payment is cleared through Autopay.

  9. Others – Select Others, if the payment is made through any other modes.

  1. Cashier Cheque – Select the check box, if the mode of payment is a Cashier Cheque. A Cashier Cheque is considered to be in of the form of ‘Cheque’, but the Settlement Nature for a Cashier Cheque is selected as ‘Cash’. You can accept the excess amount using a Cashier Cheque for the services availed from the facility and return the remaining amount as a refund.  

  2. Use by Cashier – Select the check box, if the Cashier is able to use the Settlement Type.

  3. Use by Accounts – Select the check box, if the accounts personnel is able to use the Settlement Type.

  4. Start Digit Check Enabled - Select the check box to permit verification of starting digits of a Credit Card number. The starting digits of the credit card number can be recorded in the text box ‘Credit Card No’, only if the check box is selected. This verification is valid only if the Settlement Nature for the settlement mode is selected as Credit Card.

  5. Credit Card No – Enter the first four digits of the credit card number. The first four digits of the credit card number recorded is are verified while entering the credit card number in the transaction functions. If the first four digits entered in the transaction function does not match with the numbers recorded using this function, an error message ‘Invalid Credit Card Number’ gets displayed.

  6. Sale Draft No – Select the check box, if the Sale Draft number has to be entered for the selected Settlement Type in the Settle Outstanding Bill function in the transactions. Clear the check box, if the Sale Draft number is not mandatory for the selected Settlement Type. This is valid only when the selected Settlement Nature is Credit Card.

  7. Payer Name – Select the check box, if the Payer Name has to be entered for the selected Settlement Type in the Settle Outstanding Bill function in the transactions. Clear the check box, if the Payer Name is not mandatory for the selected Settlement Type. This is valid only when the selected Settlement Nature is Credit Card.

  8. Payer Ref ID - Select the check box, if the Payer Reference ID has to be entered for the selected Settlement Type in the Settle Outstanding Bill function in the transactions. Clear the check box, if the Payer Reference ID is not mandatory for the selected Settlement Type. This is valid only when the selected Settlement Nature is Credit Card.

  9. Other Counter - Select the check box, if the Settlement Type is valid for any other Counter. For example, a patient or payer can settle the bill amount in a Counter Service which will send a text file to the facility indicating the amount it has received. Clear the check box, if the Settlement Type is not valid for settlement at any other Counter.

  10. Bank Counter - Select the check box, if the Settlement Type is valid for Bank Counters. Clear the check box if the Settlement Type is not valid for Bank Counters.

  11. Prepaid - Select the check box, if the settlement type is valid for prepaid management. Clear the check box if the settlement type is not valid for prepaid management.

  12. Account Code For Credit Card Commission - Enter Account Code For Credit Card Commission or click [List of values] to select the account code for credit card commission.

  13. Credit Card Commission Percentage - The credit card commission percentage is displayed in Credit Card Commission Percentage field based on selection of account code.

  14. Warning Message - Enter the Warning Message in alphanumeric format for a Settlement Type. The warning message recorded is displayed in the transaction functions when respective Settlement Type is selected.

  15. Consider TCS As Cash for TCS - Select this check box to display the tax amount when the receipt (against one or more bills) collected in cash is more than the amount defined in the setup.

  16. Cheque Type – Select the required Cheque Type from the drop down list. The listed values are ‘Blank’, ‘In-house’, ‘Local’ and ‘Outstation’. This field  is disabled if the Settlement Nature is set to a value other than “cheque”. This is one-time setup”.

 

The below mentioned fields are enabled for entry when the 'Settlement Modewise Receipt' check box is selected in the 'BL Parameters' function. The associated receipt types are generated when the respective Settlement Type is selected during bill settlement.

Receipt Doc Types

  1. GST percentage – You can enter numeric values. The maximum number of digits allowed in this field is 2 digits before decimal point and 2 digits after decimal point. This field is disabled by default and enabled only when the “Settlement Nature” is “Credit card”. You cannot enter values in this field for all the other settlement nature values except credit card. If you enter the account code for GST, the GST percentage is conditional mandatory. If you enter the account code for GST and does not enter the GST percentage, the message ‘Please enter the GST percentage’ is displayed on clicking on the Save button.

  2. Account code for GST – You can select the account code for GST. When you click on the search and select button within account code for GST, the Account code for GST dialog gets displayed.

    • The code and description is displayed.

    • You can search the required code using the Find text box. Once the value is selected, you can click OK to select the value. The selected value is displayed in the text box within settlement type function. This is a one-time setup. If you enter the GST percentage and not enter account code for GST, the message ‘Please enter the account code for GST‘ is displayed

IP/OP/Ext - Click [List of Values] to select a default document type for receipt generation when the respective Settlement Type is selected for an Inpatient, Outpatient and External patient transaction. The selected document type is considered during receipt generation for the selected patient class. The description of the respective Document Type is displayed. Refund Doc Types

  1. IP/OP/Ext - Click [List of Values] to select a default document type for receipt generation for refund transactions of an Inpatient, Outpatient and External patient. The selected document type is considered during receipt generation for the selected patient class. The description of the respective Document Type is displayed. The values defined through the 'Document Types' function with the 'Transaction Type' as 'Payment' are listed.

 

Deposit Receipt Doc Types

  1. IP/OP/Ext - Click [List of Values] to select a default document type for deposit receipt generation when the respective Settlement Type is selected for an Inpatient, Outpatient and External patient transaction. The selected document type is considered during receipt generation for the deposit related transactions of the respective patient class. The description of the respective Document Type is displayed. The values defined through the 'Document Types' function with the 'Transaction Type' as 'Receipts' are listed.

 

Deposit Refund Doc Types

  1. IP/OP/Ext - Click [List of Values] to select a default document type for refund receipt generation when the respective Settlement Type is selected for an Inpatient, Outpatient and External patient transaction. The selected document type is considered during receipt generation for the selected patient class. The description of the respective Document Type is displayed. The values defined through the 'Document Types' function with the 'Transaction Type' as  'Payment' are listed.

 

Misc Receipt/Refund Doc Types

  1. Receipts - Click [List of Values] to select a default document type for miscellaneous receipt generation when the respective Settlement Type is selected. The description of the respective Document Type is displayed. The values defined through the 'Document Types' function with the 'Transaction Type' as 'Receipt' are listed.

  2. Refunds - Click [List of Values] to select a default document type for miscellaneous refund transactions  when the respective Settlement Type is selected. The description of the respective Document Type is displayed. The values defined through the 'Document Types' function with the 'Transaction Type' as  'Payment' are listed.
