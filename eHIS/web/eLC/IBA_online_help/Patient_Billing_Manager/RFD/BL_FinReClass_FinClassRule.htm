Enter the following details to define Financial Classification Rule:

 

  1. Classification Code – Enter the Classification Code in alphanumeric format of six characters maximum length. The Classification Code is unique and cannot be changed after creation. This is a mandatory field.

  2. Long Description – Enter the Long Description in alphanumeric format of 100 characters maximum length. This is a mandatory field.

  3. Short Description - Enter the Short Description in alphanumeric format of 30 characters maximum length. This is a mandatory field.

  4. Classification Type – Select a value from the list to identify the Classification Type. The system-defined values are - Social Grant, Exempted, Provisional, Income/Asset Based, Visitors, Special Case, Credit, Sponsored and Insurance.

  5. Include Family Members – Select the check box if the family members of the patient can be grouped under the financial classification rule. This check box is enabled only if the ‘Classification Type’ is selected as ‘Income/Asset Based’.

 

Valid for registration

Select the check boxes against the respective encounter, if the financial classification code is valid for the respective patient encounter. The ‘Patient Registration’ check box is disabled, if the ‘Classification Type’ is selected as ‘Special Case’.

  1. Patient Registration – Select the check box, if the financial classification code is applicable for patient registration. Clear the check box, if the financial classification code is not valid for patient registration.

  2. Emergency Encounter - Select the check box, if the financial classification code is applicable for emergency encounters. Clear the check box, if the financial classification code is not valid for emergency encounters.

  3. Outpatient Encounter - Select the check box, if the financial classification code is applicable for Outpatient Encounters. Clear the check box, if the financial classification code is not valid for Outpatient Encounters.

  4. Daycare Encounter - Select the check box, if the financial classification code is applicable for Daycare Encounters. Clear the check box, if the financial classification code is not valid for Daycare Encounters.

  5. Inpatient Encounter - Select the check box, if the financial classification code is applicable for Inpatient Encounters. Clear the check box, if the financial classification code is not valid for Inpatient Encounters.

  6. Valid Settlement Type - The Settlement Types are displayed and disabled based on the Classification Type selected. The listed items for List Items Settlement Type are Cash, Credit and Sponsored.

  7. Valid Settlement Type should be Cash only when the classification type is either Social Grant, Exempted, Income/Asset Based, Provisional, Visitors.

  8. If the classification type is either Special Case/Credit, Settlement Type has to be credit.

  9. Settlement Type has to be Sponsored, if the classification type is either Sponsored/Insurance.

  1. Payer Settlement Type - The Payer Settlement Types are displayed and disabled based on the Classification Type selected.

  2. The values in the list are Cash, Payment by Approved by Company, Payment by Insurance by Payer.

  3. The value has to be cash, if the valid settlement type is Cash. The value has to be either Payment by Approved by Company or Payment by Insurance by Payer only when the valid settlement type is either Special Case or Credit.

  4. The value has to be Payment by Payer only when the valid settlement type is either Sponsored or Insurance.

  1. Applicable to Patient Sex – Select the gender from the list. The system-defined values are  Male, Female and Both.

  2. Applicable to Child - Select the check box, if the financial reclassification is applicable to a child. On selecting the check box and using the respective financial classification code in the transaction functions while updating the financial details for a child, system compares the child’s age with the age defined in the ‘Financial Reclassification Parameters’ function.

 

The following validations are required only when the classification type is other than Income/Asset Based:

  1. Applicable Billing Group – Click [List of Values] to select the Billing Group. The values defined in the Billing Groups function get listed. You can associate a Billing Group to only one Financial Classification code. The Billing Group selected should match with the Valid Settlement Type.

  2. Valid For [IP, DC, OP, AE] – The check boxes are selected for the respective patient class based on the Billing Group definition. You cannot clear or select the check box.

  3. Payer Group - Click [List of Values] to select the Payer Group. The values defined in the Customer Groups function get listed. The list of values is available for selection only if the Classification Type is selected as ‘Credit’ or ‘Sponsored’.  

  4. Customer - Click [List of Values] to select the Customer. The values defined in the Customer function get listed. The list of values is available for selection only if the Classification Type is selected as ‘Credit’ or ‘Sponsored’.  

  5. Validity Check for Basic classification – Select the check box, if the Billing Group is valid for reclassification. Clear the check box, if the Billing Group is not valid for financial reclassification.

  6. Applicable for Basic Billing groups during Encounter Registration - Click [List of Values] to select a cash, credit or sponsored Billing Group.

 

The following validations are required only when the classification type is Income/Asset Based:

  1. Income/Asset Indicator – Select the value as ‘Income’ or ‘Asset’. The values are system-defined. The Billing Group is selected for the income/asset.

  2. Lower Limit – Enter the lower limit in numeric format. This indicates the lower limit valid for income/asset range when the financial reclassification code is used.

  3. Higher Limit – Enter the higher limit in numeric format. This indicates the higher limit valid for income/asset range when the financial reclassification code is used.

  4. Applicable Billing Group - Click [List of Values] to select a Cash Billing Group. The values defined in the Billing Groups function get listed. You can associate a Billing Group to only one Financial Classification code. The Billing Group selected should match with the Valid Settlement Type.

  5. Valid For [IP, DC, OP, AE] – The check boxes are selected for the respective patient class based on the Billing Group definition. You cannot clear or select the check box.
