Enter the following details to define a Cash Counter:

 

  1. Cash Counter Code - Enter the alphanumeric code with two characters maximum length. This is a mandatory field.

  2. Facility – The Facility is defaulted to the logged in Facility.

  3. Cash Delivery Doc Type - Click [List of Values] to select the Cash Delivery Document Type. This is selected as the default document type for cash delivery.

  4. Long Description - Enter a Long Description of 30 characters maximum length. This is a mandatory field.

  5. Short Description - Enter a Short Description of 15 characters maximum length. This is a mandatory field.

  6. Company Code - Enter the Company Code in alphanumeric format of five characters maximum length.

  7. External System Specific Refund Counter Only - Select the check box if the Cash Counter is valid to generate slips for the refunds issued through external systems. Clear the check box, if the Cash Counter is not valid to generate slips for refunds issued through external systems. If the check box is cleared, you cannot use the function 'Generate Refund Slip for Ext. System' to generate slips.

  8. Special Clinic - Select the check box, if the Cash Counter is valid for special Outpatient Clinics.

  9. Building Code -  Click [List of Values] to select a Building Code. The description of the Building Code is displayed on selecting the code.  The Cash Counter defined is valid for the selected Building Code. The values defined through 'Building in Facility' function in the Application Masters module are listed.

  10. Special and Normal Clinic - Select this checkbox to generate bills/receipts/refunds of normal cash counter from special clinic cash counter. This check box is disabled by default. It is enabled only if the Special Clinic check box is selected. If the Special Clinic check box is selected, the check box is enabled. After selecting both check boxes, if you unselect the Special Clinic check box, the check box is also unselected and disabled.

  11. Floor Code - Click [List of Values] to select a Floor Code. The description of the Building Code is displayed on selecting the code. The Cash Counter defined is valid for the selected Floor Code. The values defined through 'Floor in Facility' function in the Application Masters module are listed.

 

Nature

  1. Accounts – Select Accounts, if the Accounts personnel have to use the Cash Counter.

  2. Cashier – Select Cashier, if the Cashier uses the Cash Counter. The Cashier transactions only are processed in the selected Cash Counter.

 

Default Credit Card Machine

  1. Workstation ID – Enter the Workstation ID in alphanumeric format of 30 characters maximum length. The Credit Card Machine is set as the default machine to accept the payments when the Credit Card is processed from the respective Workstation.

  2. Credit Card M/c ID – Click [List of Values] to select a Credit Card Machine. The ID of the Credit Card Machines associated to the facility while defining the Banks and Branches get listed. The Credit Card for the Workstation is processed at the associated Machine interface.

 

Process Receipts

  1. IP/Day Care – Select the check box to process the receipts for Inpatient or Daycare patients in the respective Cash Counter.

  2. OP/Emg - Select the check box to process the receipts for Outpatient or Emergency patients in the respective Cash Counter.

  3. Ext – Select the check box to process the receipts for referral patients in the respective Cash Counter.

  4. Allow to Change Amt – Select the check box to permit the Cashier to update the receipt amount in the respective Cash Counter.

 

Process Refunds

  1. IP/Day Care – Select the check box to process refunds in the Cash Counter for Inpatient or Daycare patients.

  2. OP/Emg – Select the check box to process refunds in the Cash Counter for Outpatients or Emergency patients.

  3. Ext - Select the check box to process refunds in the Cash Counter for referral patients.

  4. Allow to Change Amt – Select the check box to permit the Cashier in the Cash Counter to update the refund amount.

  5. Allow Refund only in same shift – Select the check box to permit the Cashier to refund the amount during the same shift in the Cash Counter. Clear the check box and the Cashier is able to issue a refund during any other shift at the Cash Counter.

 

Shift Time

A Cash Counter can operate in three different shifts. The timings for each of these shifts have to be defined.

Shift 1

  1. Start Time – Click Calendar function to select a date and time. The date and time indicates the start time for the first shift. The date has to be lesser or equal to the system date.

  2. End Time - Click Calendar function to select a date and time. The date and time indicates the end of Shift 1. The End Time has to be greater than the Start Time.

  3. Logout Required – Select the check box, if the Cashier has to logout of the Cash Counter at the end of the shift.

 

Shift 2

  1. Start Time - Click Calendar function to select a date and time. The date and time indicates the start time for the second shift. If the Cash Counter operates in a single shift, do not enter a Start Time.

  2. End Time - Click Calendar function to select a date and time. The date and time indicates the end of Shift 2. The End Time has to be greater than the Start Time.

  3. Logout Required - Select the check box, if the Cashier has to logout of the Cash Counter at the end of the shift.

 

Shift 3

  1. Start Time - Click Calendar function to select a date and time. The date and time indicates the start time for the third shift. If the Cash Counter operates in a single shift, do not enter a Start Time.

  2. End Time - Click Calendar function to select a date and time. The date and time indicates the end of shift 3. The End Time has to be greater than the Start Time.

  3. Logout Required - Select the check box, if the Cashier has to logout of the Cash Counter at the end of the shift.

 

Handover & Grace Limit

Cash and Cheque

  1. Text - Enter the Cash limit in numeric format of 13 digits maximum length. Cashier is either alerted or a warning message is displayed when the mentioned amount is reached for Cash or Cheque transactions.

  2. Action - Select an Action from the list. The system-defined values are 'Alert User', 'Block User' and 'NA'.

  3. Alert User - Select 'Alert User' to alert the Cashier when the Cash or Cheque limit is reached. You cannot select a Limit Indicator or specify the 'Grace Limit' when the 'Action' is selected as 'Alert User'. Limit Indicator is set as 'NA' automatically. On selecting 'Alert User' an alert message is displayed for every transaction when the handover limit is crossed. The transactions are blocked when the grace limit is reached.

  4. Block User - Select 'Block User' to block the Cashier transactions when the Cash or Cheque limit is reached. You can set the 'Limit Indicator' as '%' or 'Amount' and specify the 'Grace Limit' when the 'Action' is selected as 'Block User'.

  5. No Action - Select 'No Action' if action type is not valid. An alert or warning message is not displayed when the action is not selected.

  1. Limit Indicator - Select a Limit Indicator from the list. The system-defined values are '%', "Amount' and 'N/A'. You can select the value as '%' or "Amount' only when the 'Action' is selected as 'Block User'.

  2. % - Select '%' to block the cashier transactions when a percentage of the Cash or Cheque limit is reached.

  3. Amount - Select 'Amount' to block the cashier transactions when the Cash or Cheque limit reaches the mentioned amount.

  4. N/A - Select the value as 'N/A', if limit indicator is not required. The value is automatically set to 'N/A' when the 'Action' is selected as 'Alert User' or 'N/A'. You can select the value as 'N/A' when you select the 'Action' as 'Block User'.

  1. Grace Limit - Enter the Grace Limit in numeric format of 12 digits maximum length. You can specify the Grace Limit when the 'Action' is selected as 'Block User'. The Grace Limit should not exceed the Cash Limit.

  2. Grace Limit has to be within the range '0-100' when the 'Limit Indicator' is selected as '%'.

  3. Grace Limit should not be greater than the Cash or Cheque limit when the 'Limit Indicator' is selected as 'Amount'.

  4. Grace Limit cannot be recorded when the 'Limit Indicator' is selected as 'N/A'.

An error message 'Grace Limit cannot be greater than Cheque Limit' is displayed if the Grace Limit is greater than the Cheque Limit.

An error message 'Grace Limit cannot be greater than Cash Limit' is displayed if the Grace Limit is greater than the Cash Limit.
