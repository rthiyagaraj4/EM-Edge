Enter the following details to update the financial details of an Outpatient, Emergency patient or an External patient:

 

  1. Billing Group – The Billing Group code for the relevant patient gets displayed. If the Billing Group has to be updated, click [List of Values] to view and to select a valid Billing Group code. If the code is valid, the short description is displayed for user verification. If the Billing Group changes, you have to take care to correct all the unbilled patient charges for that episode, if any present. This is because change in the Billing Group might have an effect on the patient's charges.

The Settlement Indicator and the Customer Code are displayed based on the Billing Group Code selected.

  1. [Expl.Note] - Click [Expl.Note] to view the explanation on the selected Billing Group. The explanatory note recorded while defining the Billing Group gets displayed.

  2. Responsibility for Bill Payment – Enter the name of the person responsible towards the payment of the bill.

  3. Relationship - The relationship of the patient with the Responsible Person is displayed. Click [List of Values] to associate a relationship of the bill payer with the patient.

  4. GL Holder Name – The GL holder name associated to the patient while registration gets displayed, if any.

  5. Relation with GL Holder – The relationship of the patient with the GL Holder gets displayed.

  6. Annual Family Income – The Family’s annual income recorded during patient registration gets displayed. Enter the annual income of the family in numeric format.

  7. Number of Dependents – The number of members directly dependent on the patient gets displayed. Enter the value in numeric format with a maximum of two digits.

  8. Family Assets Value – The approximate value of family’s asset gets displayed. Enter the value in numeric format.

  9. Responsibility for Bill Payment - Enter the name of the person who is responsible for the payment of bills.

  10. Billing Class – Click [List of Values] to set a valid Billing Class.

  11. Black Listed - If the check box is selected, the patient has to be financially blacklisted. If the check box is cleared, the patient should not have been financially blacklisted.

  12. Customer - The Customer code cannot be recorded, if the Settlement Indicator is set as Cash. If the settlement indicator is not cash, click [List of Values] to select a valid Customer code. If the Settlement Indicator is Credit, the Customer has to be a credit patient. If Settlement Indicator is Sponsored Patient, the Customer has to be a sponsored patient. If the code is valid, the customer's name is displayed for user verification. Billing Group for the Customer has to be the same as that of the displayed Billing Group.

  13. Employer - Click [List of Values] to view and to select an Employer code. If the code is valid, the short description is displayed for user verification. The Employer code is mandatory, if the associated Billing Group has the check box ‘Payer Employer ID Mandatory’.

  14. Claim Code – Enter the Claim Code in alphanumeric format of 20 characters maximum length.

 

Policy Details

You can enter the details in the fields below only if the selected Billing Group is Insurance.

  1. Additional Billing Group – Click [List of Values] to associate an Additional Billing Group to the patient record along with the Policy Details. The Financial Details associated to the patient during registration gets displayed. You can associate multiple policies to a relevant patient record.

  2. Blng Grp - Click [List of Values] to select a Billing Group. The selected Billing Group is applicable for multiple policy holders where the amount is being settled by the multiple insurance companies.

  3. Customer - Click [List of Values] to associate a Customer to the patient record. The Customer associated to the selected Billing Group is listed.

  4. Priority – Enter the priority in numeric format of two digits maximum length to indicate the precedence of each Customer.

  5. Policy Type - Click [List of Values] to select a Policy Type. The details are validated in the list from the Insurance Management module.

  6. Policy No – Enter the Policy Number in alphanumeric format of 30 characters maximum length.

  7. Policy Start Date - Click Calendar function to select the date. The date indicates from when is the policy valid for the selected patient.  

  8. Policy Expiry Date – Click Calendar function to select the date. The date indicates until when the policy is valid for the selected patient.

  9. Credit Auth Ref – Enter the Credit Authorization Reference details in alphanumeric format. If the Credit Authorization Reference details are left blank, the first 20 characters of the Policy Number are considered, by default.

  10. Credit Auth Date – Click Calendar function to select the date. The date indicates when was the credit authorized.

  11. Effective From – Click Calendar function to select the date. The Policy is effective from the date selected.  

  12. Effective To – Click Calendar function to select the date. The Policy is effective until the date selected.

  13. Approved Amt – Enter the amount approved by the relevant Customer towards the patient for the selected Policy.

  14. Approved Days – Enter the number of days until when the Policy is valid for the relevant patient.

  15. Primary Referral Source – Click [List of Values] to view and to select a Primary Referral Source. The referral source details have to captured, if the referral source is set as mandatory for the respective Billing Group. Place the mouse on the    icon to display the long name of the External or Referral Source recorded in ‘External/Referral Source’ function of Application Masters module. The  icon appears on the screen if the long name for the External Referral Source is recorded in ‘External/Referral Source’ function of Application Masters module.

  16. Secondary Referral Source – Click [List of Values] to view and to select a Secondary Referral Source. The referral source details have to captured, if the referral source is set as mandatory for the respective Billing Group. An icon adjacent to the secondary referral source is displayed, if the long description of the referral source exceeds 100 characters. The additional details are displayed on mouse over.

  17. Claim Code – Enter the Claim Code in alphanumeric format of 20 characters maximum length.

  18. Billing Group Description – The Description for the selected Billing Group is displayed.

  19. Customer Name – The name of the selected Customer is displayed.

  20. Policy Description – The Description for the selected Policy Type is displayed.

  21. Non Ins. Billing Grp - Click [List of Values] to select a Billing Group. The selected Billing Group should be non-insurance Billing Group.

  22. Non Ins. Customer - Click [List of Values] to select a Customer. The selected Customer should not belong to the Insurance Group.

  23. [Expl.Note] - Click [Expl.Note] to check the explanatory note associated with the selected Billing Group.

 

Credit Details

  1. Document Ref – You cannot record the Credit Document Reference, if the Settlement Indicator is selected as Cash. The document reference is mandatory for credit patients.

  2. Doc Start - End Date – Click Calendar function to specify the Start and the End date for the document reference.
