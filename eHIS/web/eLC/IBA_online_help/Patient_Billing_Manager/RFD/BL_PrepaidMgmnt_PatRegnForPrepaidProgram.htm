Enter the following details to register a patient for prepaid program:

 

  1. Prepaid Code - Click [List of Values] to select a Prepaid Code. The values defined through the 'Prepaid Code Setup' tab in 'Prepaid Payment Setup Rule' function are listed. This is a mandatory field.

  2. Register Date [From - To] - Click the Calendar function to select a registration date range. The 'From' date is mandatory. The 'To' date has to be within the range specified for protocol.

  3. Policy Number - Enter the Policy Number in alphanumeric format of 30 characters maximum length. This is a mandatory field.

  4. Treatment Date - Click the Calendar function to select the treatment date.

  5. Regn Number - Enter the registration number in alphanumeric format of 20 characters maximum length. This is a mandatory field.

  6. Primary Hospital - Click [List of Values] to select the Primary Hospital. The values defined through the 'Referral Source' function are listed.

 

Protocols Details

  1. Protocol Code - Click [List of Values] to select a Protocol Code. The values defined through the 'Prepaid Group/Protocols By Prepaid' tab in 'Prepaid Payment Setup Rule' function are listed. The Protocol Codes that are associated to the selected Prepaid Code are listed. If 'Lump Sum Applicable' check box is selected for the selected Prepaid code, an error message 'Amount is not Available To Allocate, Cannot Assign The Protocol' is displayed. You have to record the prepaid amount and generate a receipt for the selected Prepaid code in order to register the patient with the selected prepaid code. This is a mandatory field.

  2. Eff Date [From - To] - Click the Calendar function to select the effective date range. The 'effective to' date has to be greater than the registration period. This is a mandatory field.

  3. Prepaid Amt - The Prepaid Amount recorded for the respective Prepaid and Protocol Code combination is displayed, by default. You cannot update the amount.

  4. Num of Payments - The number of payments recorded for the respective Prepaid and Protocol Code combination through the 'Prepaid Payment Rule' tab in 'Prepaid Payment Setup Rule' function is displayed. You cannot update the value.

  5. Payer Group - The Payer Group associated to the selected Prepaid and Protocol Code combination through the 'Prepaid Payment Rule' tab in 'Prepaid Payment Setup Rule' function is displayed. The description of the selected Payer Group is displayed.

  6. Payer - The Payer associated to the selected Payer Group, Prepaid and Protocol Code combination through the 'Prepaid Payment Rule' tab in 'Prepaid Payment Setup Rule' function is displayed.

  7. Description - The description of the selected Payer is displayed.

  8. Policy Type - The Policy Type associated to the selected Payer Group, Payer, Prepaid and Protocol Code combination through the 'Prepaid Payment Rule' tab in 'Prepaid Payment Setup Rule' function is displayed.

  9. Policy - The description of the associated Policy Type is displayed.
