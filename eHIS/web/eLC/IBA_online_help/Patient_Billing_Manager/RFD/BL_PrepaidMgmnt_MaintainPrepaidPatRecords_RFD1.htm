Enter the following details to update the prepaid patient records:

 

  1. Effective To - Click the Calendar function to select the date. The date selected has to be greater than the system date.

  2. Status - Select a value from the list. The system-defined values are 'Inactive', 'Active' and 'Cancel'. The value is displayed as 'Active', by default. Select 'Cancel' to cancel the prepaid registration. You cannot cancel the protocol, if it is under process. Select 'Inactive' to set the status as 'Inactive' for the selected patient.

  3. Treatment Date - Click the Calendar function to select a treatment date.

  4. Primary Hospital - Click [List of Values] to select a Primary Hospital. The values defined through the 'Referral Source' function are listed. The description of the selected hospital is displayed.

  5. Reason - Enter the reason in alphanumeric format. The reason is mandatory only if the 'Status' is selected as 'Inactive' or 'Cancel'.

 

Protocol Details

The protocol details that are associated to the patient through the 'Patient Registration For Prepaid Program' function are listed. Click [Create Record] to associate any other protocol to the patient.

  1. Protocol Code - The Protocol Code associated to the patient is displayed. Click [List of Values] to associate any other Protocol Code. You cannot associate the same Protocol Code to the patient. The description of the selected Protocol Code is displayed. This is a mandatory field.

  2. Effective [From - To] - The Effective date range selected during registration is displayed. Click the Calendar function to define the Effective Date range for the Protocol. The 'Effective From' date is mandatory. The 'Effective To' date has to be within the registration period.

  3. Prepaid Amt - The Prepaid Amount recorded for the respective Protocol is displayed. You cannot update the prepaid amount associated to a Protocol.

  4. No of Pmnts - The number of payments defined for the protocol is displayed. You cannot update the value.

  5. Amt Received - The Amount received for the respective Protocol is displayed.

  6. Adjsted Amt - The amount adjusted for the protocol is displayed.

  7. Remaining Balance - The balance amount in the total prepaid amount deposited by the Payer after deducting the Protocol amount is displayed.

  8. Completed - Select the check box to mark the protocol as complete. You cannot select the check box if the Protocol period is valid.

  9. Payer Group - The Payer Group associated to the respective Protocol is displayed. The description of the Payer Group is displayed.

  10. Payer - The Payer associated to the respective Protocol is displayed. The description of the Payer is displayed.

  11. Policy Type - The Policy Type associated to the respective Protocol is displayed. The description of the Policy Type is displayed.

 

Protocol Payment Details

The Protocol Payment Details are displayed based on the definition in 'Prepaid Payment Setup Rule' function.
