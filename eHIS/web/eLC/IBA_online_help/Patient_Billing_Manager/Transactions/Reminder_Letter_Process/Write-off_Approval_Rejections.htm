Write-off Approval/Rejections

From this function you can either Approve or Reject the write-off request raised for the documents, where the outstanding amount is not settled. You can Cancel the approved write-off requests.

Write-Off Approval/Rejections function is used to either reject or approve the request placed to write-off the unsettled bills. An authorized billing personnel is able to cancel the approved requests using this function. The records for which the write-off requests are generated get listed. The check box ‘Write-off Approval’ has to be selected in the function ‘Users for Authorization/Refund/Cash Counters’ in the Administration menu. A document reference number is generated on selecting the records. A screen pops-up requesting confirmation to generate the write for the number of records selected. You can either approve or reject the write-off request for all the outstanding documents or the write-off request generated for the outstanding documents associated to a selected Billing Group Category.

Reject or Approve the Write-Off requests

From the Transactions > Reminder Letter Process, double-click Write-Off Approval/Rejections function.

The Write-Off Approval/Rejections screen gets displayed.

  1. Select a Billing Group Category. The values defined using the Billing Group Category function get listed. This is a mandatory field.

  2. Select an Outstanding Amount range. The values defined using the Reminder Letter Setup function and associated to the selected Billing group Category get listed. This is a mandatory field.

  3. The system date is displayed as the Write-off Process Date, by default. You can update the date. The outstanding document details with the write-off approval for the selected date range get listed.

  4. Click [List of Values] to select a Level of Service. This is applicable for inpatient transactions. The write-off documents are filtered based on the level of service selected. You can write-off the outstanding documents or cancel the write-off.

  5. Click [Authorize] to login and either to cancel or to approve the write-off request. A screen appears requesting the details to login and to approve or reject the write-off request. Click [OK] to login and to authorize the request or click [Cancel] to exit from the screen.

    Enter the following details to login:

    1. User ID – Enter the User ID in alphanumeric format. Users who are authorized to approve the write-off process are only permitted to login and reject or approve the request. This is a mandatory field.

    2. Pin No – Enter the PIN number. The PIN Number recorded for the specific User during definition in the System Manager module has to be entered. This is a mandatory field.

    3. Approval Reason – Click [List of Values] to select an Approval Reason. The reasons defined using the function ‘Reason for Approval’ function get listed. This is a mandatory field.

    4. Rejection Reason – Click [List of Values] to select a reason for Rejection. The reasons defined using the function ‘Reason for Approval’ function get listed. This is a mandatory field.

  6. The document details for which the write-off request has been placed using the function Generate Write-off function get listed. The details are displayed under the following tabs:

Record Approval/Rejection

Cancel Approved Write-Off

  1. Select the records and click [Commit Form] or [Process] to approve or to cancel the write-off request.

  2. Click [Clear Record] to clear the entered data.
