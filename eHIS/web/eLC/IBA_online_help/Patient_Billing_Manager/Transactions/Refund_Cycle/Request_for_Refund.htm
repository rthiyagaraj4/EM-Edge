Request For Refund

From this function you can Process the refund request raised by the user. You can Query to view the refund request raised by the Cashier.

Request for Refund function is used to view and to process the refund request raised by the Cashiers. Authorized Billing Personnel can select to either view all refund requests or refund against Bill and DCP or the Refundable Deposit Requests. The patient records, where the refund amount is not processed are saved in the database. You have an option to filter the refund requests raised. You can choose to view all the refund requests placed by the ‘Cashier’, ‘Refund against Bill and DCP’, ‘Refundable Deposit’ and ‘Patient Deposit’. The Transactions, where the deposits are accepted from the patient and the amount has to be refunded gets listed. You have to select the record to approve the request. A document reference number is generated for the request processed against a record. The document type is displayed based on the value selected for ‘Approval Doc Type’ field under the ‘Refund Processing’ tab in the BL Parameters. The document number is system generated.

Request for Refund

From the Transactions > Refund Cycle, double-click Request for Refund function.

The Request For Refund screen gets displayed.

  1. The Request Date gets displayed, by default. You cannot update the date. This is a mandatory field.

  2. The name of the logged-in User gets displayed automatically on clicking the Request By field. You can enter the name of the Cashier who is generating the request for the refund. This is a mandatory field.

  3. Enter the Remarks in alphanumeric format of 100 characters maximum length. The details supporting the refund request documents can be entered as Remarks. This is a mandatory field.

  4. The refund requests raised by the selected Cashier get listed on entering the mandatory fields.

  5. Select the records, enter the refund details and the mailing address. <Related Field Description>

  6. Click [Commit Form] or [Process] to record the refund request or [Clear Record] or [Cancel] to clear the entered data. A reference document number is generated for the request placed.
