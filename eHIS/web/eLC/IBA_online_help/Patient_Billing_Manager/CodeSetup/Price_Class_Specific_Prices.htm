Price Class Specific Prices

From this function you can Define Price Class, Associate Customer with the Price Class, Associate Billing Groups with the Price Class and setup price and discounts for the Billing Services. You can Query to view the records created.

The Price Class identifies if the patient is a Citizen, a Non-Citizen or belongs to any special health care service groups created. The price and the discounts are defined against the Billing Class. You can define the Price Class Setup, Customer specific and Billing Group specific prices using the following tab pages:

Price Class Specific Prices

Price Class Specific Price is an option provided to define the price for a specific Price Class created using the function Price Class Setup. You can also Associate multiple billing services to a Price Class for a specified period. You have to associate billing services to the Price Class, so the charge for the billing service is calculated based on the price defined for a patient category.  You can define the price for a package within the facility and associate the Billing Services to the Price Class defined as a package. The services associated are valid for the respective package. You can also define the package price for each patient class in the facility. Select the Reimbursable Type to indicate whether the price is reimbursable for the selected patient class.

You can create, update and view the prices defined using the tab pages:

Maintain Prices

Create Prices

View Past Prices

Package Related other Setup

Price Class – Billing Group Specific Prices

Price Class - Billing Group Definition is used to  define the price for a specific Billing Group in a Facility. You have to associate billing services to a Billing Group, so the charge for the billing service is calculated based on the price class associated to a Billing Group. You can associate multiple services to a Billing Group using the function ‘Billing Group/Customer Specific Prices’. The rate defined for the Price Class is applicable to the Billing Group associated to it. You can associate a Billing Group that is in turn associated to a Currency Code. The patient associated to the selected Billing Group and a Price Class can settle the bills using the Currency associated to the Price Class. If Price Class is not associated to any Currency, the patient or the Customer has to settle the bills using the local currency. The Currency associated to the Billing Group, gets displayed in the Billing Groups function.

Enter the following details to associate Billing Group to a Price Class:

  1. Billing Group – Click [List of Values] to select a Billing Group. The values defined using the Billing Group function get listed. This is a mandatory field.

  2. Description - The description for the selected Billing Group gets displayed.  

  3. Associated Attribute - Select a value from the list. The values are system defined – Unrestricted, Nationals and Others.

  4. Unrestricted - Select Unrestricted, if the selected Billing Group and the Price Class combination is valid for any patient admitted to the Facility.

  5. Nationals – Select Nationals, if the selected Billing Group and the Price Class combination is valid for the patients who are valid citizens of the Country.

  6. Others – Select Others, if the selected Billing Group and the Price Class combination is valid for non-nationals.

  1. Current Price Class Code – The Price Class Code associated to the selected Billing Group initially gets displayed. If the Price Class is not associated, the text box is displayed blank.

  2. New Price Class Code - Click [List of Values] to select a New Price Class Code. The values defined using the Price Class tab get displayed. This is a mandatory field.

  3. Description - The Description for the New Price Class Code gets displayed.

  4. Price in Currency – The Currency Code associated to the selected Price Class while defining the Price Class under ‘Price Class Setup’ tab gets displayed along with the description.

  5. Valid From - Click Calendar function to select a date. The selected Price Class is valid for the Billing Group from the date selected. The date selected has to be greater than the system date. This is a mandatory field.

  6. Valid To - Click Calendar function to select a date. The selected Price Class is valid for the Billing Group until the date selected. The date selected has to be greater than the valid from date.

  7. [Prices for Associated Price Class] - Click [Prices for Associated Price Class] to create, maintain and view the past prices for a specific Price Class. The tab is shifted to Price Class Specific Prices on clicking [Prices for Associated Price Class].

Price Class – Customer Specific Prices

Price Class - Customer Group Definition is an option provided to define the price for a specific Customer Group in a Facility. You have to associate billing services to Customer Group, so the charge for the billing service is calculated based on the price class associated to a Customer Group. You can associate multiple services to a Customer Group using the function Billing Group/Customer Specific Prices. You can associate a Price Class that is in turn associated to a Currency Code. The Customer associated to a Price Class can settle the bills using the Currency associated to the Price Class. If Price Class is not associated to any Currency, the Customer has to settle the bills using the local currency.

Enter the following details to associate Customer to a Price Class:

  1. Facility – Select a Facility from the list. The Facilities associated to the logged in User get listed.

  2. Customer Code – Click [List of Values] to select an appropriate Customer Code. The values defined using the function Customer get listed. This is a mandatory field.

  3. Customer Name - The description for the selected Customer gets displayed.

  4. Associated Attribute - Select a value from the list. The values are system defined – Unrestricted, Nationals and Others.

  5. Unrestricted - Select Unrestricted, if the selected Customer and Price Class combination is valid for any patient admitted to the Facility.

  6. Nationals – Select Nationals, if the selected Customer and Price Class combination is valid for the patients who are valid citizens of the Country.

  7. Others – Select Others, if the selected Customer and Price Class combination is valid for non-nationals.

  1. Current Price Class Code – The Price Class Code associated to the Customer initially gets displayed. If the Price Class is not associated, the text box is displayed blank.

  2. New Price Class Code – Click [List of Values] to select a New Price Class Code. The values defined using the Price Class tab get displayed. This is a mandatory field.

  3. Description – The Description for the New Price Class Code gets displayed.

  4. Valid From – Click Calendar function to select a date. The selected Price Class is valid for the Customer from the date selected. The date selected has to be greater than the system date. This is a mandatory field.

  5. Valid To - Click Calendar function to select a date. The selected Price Class is valid for the Customer until the date selected. The date selected has to be greater than the valid from date.

  6. [Prices for Associated Price Class] - Click [Prices for Associated Price Class] to create, maintain and view the past prices for a specific Price Class. The tab is shifted to Price Class Specific Prices on clicking [Prices for Associated Price Class].

Price Class Setup

The Price Class tab enables you to create a Price Class pertinent to the Facility. A Price Class identifies the government/private organizations or the healthcare packages. You can Query to view the Price Class defined. Price class Setup is used to define the healthcare packages. It can be Government or Private Organizations. A patient associated to any of these organizations avails special prices or discounts.  

Enter the following details to define a Price Class:

  1. Facility – Select a Facility from the list box to define and associate a Price Class. The Facilities defined and associated to the user in the System Manager Module get listed.

  2. Price Class Code - Enter the Price Class Code in alphanumeric format with a maximum of eight characters. A Price Class Code is unique and cannot be changed after creation. This is a mandatory field.

  3. Long Description – Enter a Long Description for the Price Class Code in alphanumeric format with a maximum of 40 characters. This is a mandatory field.

  4. Short Description – Enter a Short Description for the Price Class Code in alphanumeric format with a maximum of 40 characters. This is a mandatory field.

  5. Price in Currency – Select a Price Currency from the list The values defined using the function ‘Foreign Currencies’ in the ‘Financials Base Manager’ module gets listed. The Foreign Currency is valid for the selected Price Class which is in turn associated to a Billing Group. A patient who is assigned the selected Billing Group can pay the bills using the currency associated to the selected Billing Group and the Price Class.

  6. Package – Select the check box to indicate that the Price Class recorded is a package. Clear the check box, if the Price Class is not a package. You can define the price and associate services to a Price Class defined as a package.

  7. Billing Class - Click [List of Values] to associate a Billing Class to define the package price. The description for the selected Billing Class gets displayed.

  8. Package Price - Enter the package price in numeric format. This indicates the total amount the patient has to pay during the  package subscription.

  9. Applicability - Select a value from the list. The system defined values are - 'All', 'Female' and 'Male'. The package is either available for all the patients or only to male or female patients based on the value selected.

  10. Subscription Age - Enter the age of the patient in yy-mm-dd format. The package is valid for the patients in the age group defined.

  11. [Prices for Price Class] - Click [Prices for Price Class] to create, maintain or to view the past prices for the selected Price Class. The screen gets shifted to Price Class Specific Prices tab.

  12. [Billing Groups for Price Class] - Click [Billing Groups for Price Class] to view the Billing Groups associated with a Price Class. The screen gets shifted to Price Class – Billing Group Definition tab.

  13. [Customers for Price Class] - Click [Customers for Price Class] to view the list of Customers associated with a Price Class. The screen gets shifted to Price Class – Customer Definition tab.

Packages by Billing Group

Packages By Billing Group

Packages by Billing Group tab is used to associate a Billing Group to multiple packages. The Billing Groups defined as package related appears in the Financial Details tab in the external modules. A package is valid for a Billing Group with the Settlement Type as ‘Cash’ or ‘Customer’. The Package is not valid for a ‘Sponsored Patient’.

Enter the following details to associate a Billing Group to packages:

  1. Facility – Select a Facility from the list. The Facilities associated to the logged in User get listed.

  2. Billing Group – Click [List of Values] to select a Billing Group. The values defined using the ‘Billing Groups’ function get listed. The description for the selected Billing Group gets displayed. This is a mandatory field.

  3. Associated Attribute - Select a value from the list. The system defined values are Unrestricted, Nationals and Others.

  4. Unrestricted - Select Unrestricted, if the selected Billing Group and Price Class combination are valid for any patient admitted to the Facility.

  5. Nationals – Select Nationals, if the selected Billing Group and Price Class combination are valid for the patients who are valid citizens of the Country.

  6. Others – Select Others, if the selected Billing Group and Price Class combination are valid for non-nationals.

  1. Package Price Class – Click [List of Values] to select and associate a Package Price Class to a Billing Group. The Price Classes defined as ‘Package’ under the ‘Price Class Setup’ tab get displayed. The description for the selected Package Price Class gets displayed. This is a mandatory field.

  2. Price in Currency – The ‘Price in Currency’ set for the selected package under the ‘Price Class Setup’ gets displayed.

  3. Service [From–To] Date – Click Calendar function to select the date. The package is valid for the selected date range. The effective date selected has to be greater than the system date.

  4. Standard Price Class – Click [List of Values] to select the Standard Price Class. The Price Classes defined under the ‘Price Class Setup’ tab with description get displayed. This is a mandatory field.

  5. Price in Currency – The ‘Price in Currency’ set for the selected package under the ‘Price Class Setup’ tab gets displayed.

  6. Pre Payment Required – Select the check box, if pre payment is applicable for the selected package. You can accept the pre payment amount for the respective package using the transaction functions.

  7. [Prices for Associated Price Class] - Click [Prices for Associated Price Class] to create, maintain and view the past prices for a specific Price Class. The tab is shifted to Price Class Specific Prices on clicking [Prices for Associated Price Class].

Defining Price Class Specific Prices

From the Masters > select Multi Pricing, double-click Price Class Specific Prices function.

The Price Class Specific Prices screen gets displayed with Price Class Specific Price tab enabled.

  1. Enter the details in the mandatory (*) fields found in the tab pages.

  2. Click [Commit Form] or press [F10] to save the records or [Clear Record] to clear the entered data.

  3. Click [Execute Query] or [F8] to view the records saved.
