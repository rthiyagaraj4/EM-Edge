Cash Counters for Settlement Type

From this function you can Associate the Cash Counters to relevant Settlement Types. The Cash Counters are restricted from processing the transactions using any other Settlement Types. You can Query to view the Cash Counters to which the Settlement Types are already associated and include new Settlement Types with the Cash Counters.

Cash Counters for Settlement Type function is used to associate the Settlement Types to a selected Cash Counter. A Cashier in the Cash Counter is restricted from accepting the payments using any other Settlement Type, which is not associated to the Cash Counter.

Associating Settlement Types to a Cash Counter

From the Masters menu, select Facility Specific Setup menu, double-click Cash Counters for Settlement Type function.

The Cash Counters for Settlement Type screen gets displayed.

  1. Click [List of Values] to select a Settlement Type. This is a mandatory field.

  2. The logged in Facility name is displayed, by default.

  3. Enter the details in the mandatory (*) fields found in the following tab pages:

Cash Counters

Enter the following details to associate Settlement Types to a Cash Counter:

  1. Settlement Type – Click [List of Values] to select a Settlement Type. The description for the selected Settlement Type is displayed automatically in the adjacent text box.

  2. Facility - The logged in Facility is set, by default and the field is disabled.

Cash Counters

  1. Cash Counter - Click [List of Values] to select a Cash Counter. The values defined using the Cash Counter function get listed. This is a mandatory field.

  2. Description - The description for the selected Cash Counter is displayed automatically.

  3. Credit Card Machine Interfaced – Select the check box to interface the Credit Card machine to the Cash Counter for the selected Settlement Type. The check box is enabled only if the Settlement Type is selected as Credit Card.

  4. Credit Card Machine ID – Enter the Credit Card Machine ID in alphanumeric format with a maximum of 30 digits. The Credit Card Machine ID field is enabled only if the Settlement Type is selected as Credit Card. These details are valid only when the transaction takes place in the respective Cash Counter for the selected Settlement Type. The credit card details for the relevant patient are to be captured when performing the transactions for the selected Settlement Type.

  5. [Interface to GL] – Click [Interface to GL] to switch to the next tab to associate GL interface codes to the Cash Counter.

Interface to GL

The Settlement Type and the Cash Counter has to be associated to the General Ledger module.

Enter the following details to associate the Cash Counter to the GL codes:

  1. Cash Counter - The Cash Counter selected under the Cash Counters tab gets displayed. The description for the Cash Counter appears automatically in the adjacent text box.

GL Interface

  1. GL Interface – Click [List of Values] to select a GL interface code. The values get populated only when the valid GL Interface Codes are set up under the Accounting Entity Setup. This is a mandatory field.

  2. Description - The description for the relevant GL interface code gets displayed.

  3. Effective From Date – Click Calendar function to select a date. The GL Interface Codes are applicable for Cash Counter and the selected Settlement Type from the date selected. The date selected has to be greater than the system date and cannot overlap with the existing definitions. This is a mandatory field.

  4. Effective to – Date - Click Calendar function to select a date. The GL Interface Codes are valid for the Cash Counter and the selected Settlement Type until the date selected. The date selected has to be greater than the effective from date.

  1. Click [Interface to GL] to associate the Cash Counters to GL interface codes.

  2. Click [Commit Form] or press [F10] to save or [Clear Record] to clear the entered data.

  3. Click [Execute Query] or press [F8] to search for the existing records within the facility.
