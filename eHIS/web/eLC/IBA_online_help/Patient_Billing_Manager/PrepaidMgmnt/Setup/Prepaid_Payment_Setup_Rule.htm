Prepaid Payment Setup Rule

From this function you can Define the Prepaid Payment Rule and Setup Prepaid Groups. You can also Define the Protocols for Prepaid Groups based on which the prepayment transactions are done in the facility. You can Query to view the existing records.

The Prepaid Payment Setup Rule function is used to setup prepaid payment rule. A Payer will have a tie-up with the facility based on which the payer will deposit an amount in the facility. The Payer pays the amount in advance as a prepaid amount to the facility. The prepaid amount is settled against the bill amount of the patient. Patients' who are covered by a Payer has to be registered with the Payer. Through this function you can setup prepaid program code, prepaid program name and link to an Insurance policy. You can define the Prepaid Program Group/Protocol and define the payment conditions.

Prepaid Payment Setup Rule

From the Prepaid Management > Setup, double-click on Prepaid Payment Setup Rule function.

The Prepaid Payment Setup Rule screen gets displayed.

  1. Enter the details in the mandatory (*) fields in the following tab pages:

Prepaid Code Setup

Prepaid Group/Protocols By Prepaid

Prepaid Payment Rule

  1. Click [Commit Form] or press [F10] to save the records or [Clear Record] to delete the data.

  2. Click [Execute Query] or press [F8] to query for the existing records.
