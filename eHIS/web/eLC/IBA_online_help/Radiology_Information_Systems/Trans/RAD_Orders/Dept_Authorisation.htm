Departmental Authorisation  

From this function you can Authorize pending radiology examinations. You can also Replace, Reject or Add exams to a pending authorization list of exams.

All the details of a pending examination like Examination description, Study Location where the examination is to be performed, Order Number, Ordered Date and Time and the name of the attending radiologist are displayed. A scheduled examination can be replaced or rejected by giving a valid reason. You can also add exams to the given list of exams. The instructions given to the patient, the examination history, the patient history and the drugs the patient is allergic to can be viewed. Only Radiologists can access this function.

Authorizing Pending Radiology examinations

From the Transactions > Orders, double click Department Authorisation function.

The Pending Exams for Authorisation screen gets displayed.

  1. The exams pending for authorisation are displayed or enter the search criteria and click [Search] to select a specific examination.

  2. On searching, examinations pending for authorization are displayed based on the entered criteria.

  3. Double click on the Patient ID and ‘ Authorise/Reject/Replace’ screen gets displayed. On double clicking Patient ID, the Authorise/Reject/Replace does not display for the examinations with the status ‘Reject’ and ‘Replace’. The displayed details are  Order Number, Order Date, Episode number, Patient ID, Ordering Facility ,Patient Name, Sex and Age of the Patient. If you can click ‘AUTH’ in the Status text field in Pending Exams for Authorisation screen, the Audit Trail screen gets invoked with Authorized by and Authorised Date details. The audit trail screen gets invoked with Added by, Added Date and Reason for addition If the exam is automatically authorised and manually added exam. If you can click ‘REJ’ in the Status text field in Pending Exams for Authorisation screen, the Audit Trail screen gets invoked with Rejected by and Rejected Date and Reason for rejection details. You cannot modify the displayed details.

    The exams that require authorization for the selected patient are displayed in the ‘Exams Requiring Authorization’ tab.  

    You can reject, replace the exam or add a new exam to the list of examinations. Right click the exam code to select the following options.

    Reject Request

    Enter the details to reject a Request:

    1. Exam – The selected examination gets displayed and defaulted.

    2. Reject Reason – Select the reason for rejecting the examination from the list of values. The code and the description of the selected reason gets displayed. The reasons for the action types are created in the ‘Reasons’ function in the ‘Examinations’ masters.

    Click [OK] to confirm the rejection of the exam or [Cancel] to cancel the rejection. After the exam is rejected, it is no longer available in the Pending Examination list and is not available for further transactions.

    Replace Exam

    Enter the details to replace the current exam with another exam:

    1. Wing – Select the required wing from the list. The Operating Unit/Wings defined in the ‘Operating Units/Wings’ function under the Administration menu is displayed.

    2. Section – Enter the section in the alphanumeric format of four characters maximum length or click [List of Values] to select the section. Sections are displayed based on the selected ‘Operating Unit/Wing’. Sections are associated to the Operating Unit/Wing in the ‘Sections for Operating Unit/Wing’ function under the Administration menu.

    3. Current Exam – The exam you have selected to replace gets displayed and defaulted.

    4. Replace Exam – Select the examination (with which you are going to replace the original exam) from the list of values. Exams are displayed based on the selected ‘Operating Unit/Wing’. Exams are associated to the Operating Unit/Wing in the ‘Exams for Operating Unit/Wing’ function under the Administration Menu.

    5. Laterality – Select the laterality from the list. The list of system-defined values is, ‘Right’, ‘Left’ and ‘Bilateral’.

    6. Replacement Reason – Select the reason for replacing the current examination with another examination.  Reasons for the various Action Types are defined in the ‘Reasons’ function in  ‘Others’ masters.

    7. Net Charge – The net charge defined in Patient Billing Manager module gets displayed for view and editing if ‘Interfaced to Patient Billing’ check box is selected in the ‘RD Parameters for Facility’ function.

    As you enter the laterality, the ‘Views’ tab gets enabled. Select the view from the list of values. This field is optional.

     

    Click [OK] to replace the current exam with another exam, or click [Cancel].

    Add New Exam

    Enter the details to add new exams to the order:

    1. Wing – Select the required wing from the list. The Operating Unit/Wings defined in the ‘Operating Units/Wings’ function under the Administration menu is displayed.

    2. Section – Enter the section in the alphanumeric format of four characters maximum length or click [List of Values] to select the section. Sections are displayed based on the selected ‘Operating Unit/Wing’. Sections are associated to the Operating Unit/Wing in the ‘Sections for Operating Unit/Wing’ function under the Administration menu.

    3. Exam – Select the exam to be added to the order from the list of values. Exams are displayed based on the selected ‘Operating Unit/Wing’. Exams are associated to the Operating Unit/Wing in the ‘Exams for Operating Unit/Wing’ function under the Administration Menu.

    4. Laterality – Select the laterality from the list. The list of system-defined values is, ‘Right’, ‘Left’ and ‘Bilateral’.

    5. Reason – Select the reason for adding the exam to the existing order from the list of values. The reasons for the action types are created in the ‘Reasons’ function in the ‘Examinations’ masters.

    6. Net Charge – The net charge defined in Patient Billing Manager module gets displayed for view and editing if ‘Interfaced to Patient Billing’ check box is selected in the ‘RD Parameters for Facility’ function.

    As you enter the laterality, the ‘Views’ tab gets enabled. Select the view from the list of values. This field is optional.

    Click [OK] to add the exam to the order, or click [Cancel].

    Select the check box corresponding to the examination and click [Authorize] to authorize the order. In the examinations setup, if the ‘Authorization Required’ check box is selected, the registration of the examination is not possible without first authorizing the examination.

  4. Click [T] to view the patient tracking location details.  The ‘ Change Patient Location’ screen is displayed on selecting a patient.

    The following details are displayed:

    1. Patient ID – The Patient ID and Patient Name is displayed.

     

    Current Location Details

    1. Current Location Status – The Current Location Status of the patient is displayed.

    2. Location Identity / Location – The location identity or the location of the patient is displayed.

    3. Movement Date Time – The movement date and time of the patient is displayed.

     

    Change Location

    1. Location Identity / Location – Select a location identity / location from the list. The locations created using ‘Study Location’ function are displayed for selection.

     

    Click [Change Location] to record the new location or [Cancel] to retain the old location.

     

  5. Click [C] to view the Clinical Events for the patient. The Clinical Events screen displays the order number of the examination, the relevant catalog code and description and the Patient Name and ID.

  6. Right click ‘Ordered By’ or ‘Attend By’ to view the contact details. The ‘Physician Contact Details’ screen gets displayed.

The color indicators for Priority [P] and Mobile Request [M] are displayed in the bottom of the Pending Exams for Authorization screen. The Patient Name is highlighted with the color indicator ‘Red’ for the high risk patient. You have to select the ‘Departmental Authorisation Required’ check box in the ‘RD Parameters for Facility’ function to enable this function.
