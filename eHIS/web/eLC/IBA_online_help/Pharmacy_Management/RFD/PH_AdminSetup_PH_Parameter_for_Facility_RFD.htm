Enter the following details to set the facility specific parameter values:

 

  1. Interface to Inventory Management - Select the check box to Interface Pharmacy Management module with Inventory Management for the logged in facility.

  2. Display Unallocated Orders at Delivery Stage – Select the check box to display orders at delivery stage for which batches have not been allocated during allocation stage.

  3. Duplicate check for ATC Chemical Main Class - Select the check box to Duplicate Check for ATC Chemical Main Class while dispensing.

  4. Display ATC Allergy Alert From - Select the value from the list to enable allergy alert to any of the drug generic names belonging to the same ATC. The listed values are ‘Anatomical Class’, ‘Therapeutic Main Class’, ‘Therapeutic Sub Class’, ‘Chemical Main Class’ and ‘Chemical Sub Class’. By default, ‘None’ is displayed.

  5. Display Charge Details in Drug Lookup – Select the check box to display charge details in the Drug Lookup.

  6. Display Charge Details in Prescriptions – Select the check box to display charge details in prescription function.

  7. Allow Copy Prescription Upto No Of Day(s) - Enter the number of days allowed to copy the existing prescriptions in numeric format of three digits maximum length. Based on the period specified in this field, the 'From' and 'To' Order Date in the Previous Orders screen is made default.

  8. Print Expiry Date for Expired Item on Label – Select the check box to print expiry date of item on the drug label while dispensing.

  9. Mandatory Remarks for PRN Frequency – Select the check box for Mandatory remarks for PRN Frequency.

  10. Tab Based Grouping and Sorting in Dispensing - Select the check box to enable grouping and sorting in Verification, Fill and Deliver stages of dispensing.

  11. Display Price Type in Drug Lookup – Select the type of price to be displayed in the Drug Lookup. The Price Types are ‘Patient Specific’ and ‘Base Price’. The value ‘Patient Specific’ is selected, by default.

  12. Allow Short Expiry Drugs - Select the check box to allow dispensing of Drugs, which is due to expire.

  13. Allow User Locks During Dispensing - Select the check box to allow user to lock the transactions while dispensing. The check box is cleared, by default.

  14. Narcotic Approval Form Required - Select the check box to generate the narcotic approval form in the facility for any narcotic drugs that are prescribed. If this check box is selected, the field 'Print Narcotic Form by Default' is enabled.

  15. Print Narcotic Form by Default - Select the check box to print the narcotic form, by default for any narcotic drugs that are prescribed.

  16. Display the UOM in Transactions By  - Select the value from the list. The listed values are Code and Description. Based on the values selected, the Code or Description of the UOM is displayed in the transactions.

  17. Default Privilege Drug Selection - Select the values from the list. The listed values are 'All Drugs' and 'Self Privilege Drugs'. Based on the value selected, the system will display all the Drugs or the Self Privilege Drugs.

  18. Edit Dispense Label after Dispensing / Delivery - Select the check box to edit the Dispense Label on Dispensing the Drug.

  19. Allow Medication Order Without Diagnosis – Select the check box to dispense the medication to the patient using the Dispense Medication function without capturing the diagnosis details. If this is unchecked and patient diagnosis is not already recorded, the following error message is displayed when you try to place a new Pharmacy Order for a patient from the CA module. If the Patient Diagnosis is not recorded, the  error message 'The Patient Diagnosis is not recorded. Do you want to record Diagnosis now?' is displayed.

  20. Alternate Drug Remarks – Select one of the available options: ‘Not Applicable’, ‘Applicable with Optional’ and ‘Applicable with Mandatory’

  21. Allow Amending of Alternate Drug Details - Select this check box to enable the ‘Amend Alternate Drug details’ button during Allocate, Fill without fill List and All Stages. This is applicable for Only Normal Rx. The system does not show Amend Alternate Drug details button in Drug Dosage Details drop down in Dispense Medication screen for any other Rx even if the check box is selected.

  22. Prescription Preview Mandatory - Select this checkbox to make the preview of prescription mandatory during pharmacy order for All Patients (OP/IP/Daycare/Emergency). By default this check is not selected.

  23. Display previous HT, WT, BSA, BMI details - Select this checkbox to display the multiple rows for HT, WT, BSA, BMI value in the prescription header of the Pharmacy Order screen.

Note:

    •  The default value of the drop-down list will be ‘Not Applicable’. If this default value is selected, the existing functionality will continue.

    • If the option ‘Applicable with Optional’ is selected, the Remarks window will appear in the Allocate and Fill stages when you select the Select checkbox within the Allocate/Reallocate/Fill/Refill screens respectively. However, the user can close the Remarks window without entering any remarks.

    • If the option ‘Applicable with Mandatory’ is selected, entering remarks for the alternate drug will be mandatory.

    • The recorded remark is displayed in the Dispense medication screen - Existing Order, Order Tracking, and Patient Drug Profile areas.

Prescription Rules

  1. Allow Refill – If ‘Allow Refill’ is applicable at enterprise level (that is ‘Allow Refill’ selected in PH Parameter). The check box is selected, by default. This indicates that prescription refill is allowed in the logged in facility. You can clear the check box, to indicate that Prescription Refill is not allowed in the logged in facility. If refill is not applicable at the enterprise level, the check box is cleared and disabled.

  2. Max No of Refills allowed – If ‘Allow Refill’ check box is selected, this field gets enabled. The value specified at the enterprise level is disabled, by default. You can change the displayed value, but cannot exceed the enterprise level setting. Enter the maximum number of refill allowed in numeric format of two digits maximum length. The defined value cannot be exceeded during ordering. This is a mandatory field.

  3. Max Duration for Refill Prescription () Days - If ‘Allow Refill’ check box is selected, this field gets enabled. The value specified at the enterprise level is disabled, by default. You can change the displayed value, but cannot exceed the enterprise level setting. Enter the maximum days for refill prescription in numeric format of three digits maximum length. The defined value cannot be exceeded during ordering. This is a mandatory field.

  4. Print Refill Prescriptions along with Main – Select the check box to Print Refill Prescriptions along with the main.

  5. Auto-transfer A&E Medication Orders to IP Admission - If you select this check box, the Medication Orders associated with a patient’s A&E encounter are automatically transferred to his/her IP encounter when a patient is transferred from A&E to IP. It is unchecked and enabled by default.  Auto-transfer A&E Medication Orders to IP is done if IP admission is made within the defined hours after creating the A&E encounter.

  6. Emergency Encounter Duration within –  This field is enabled and becomes mandatory only when Auto-transfer A&E Medication Orders to IP Admission check box is selected. .

  7. Split Orders By -

  8. Priority - If this check box is selected, 'Priority' type is considered for splitting while releasing the prescription.

  9. Drug Class - If this check box is selected, 'Drug Class' is considered for splitting while releasing the prescription.

For example:

Let us assume the following as PH Parameter for Facility.

Outpatient

      • Maximum Prescription Duration for outpatient/take home medication = 30 days (set at enterprise level)

      • Maximum number of refills allowed = 3 times

      • Maximum days for refill prescription = 50 days.

      • A practitioner prescribes Drug ‘B’ - 2 times a day for 20 days and refill 3 times.

In this scenario, medication order fails even though the prescription duration and number of refills have not exceeded the set parameter. The system multiplies the prescription duration (that is, 20 days in the example) with the number of refills (three times). The product of which is 60 days that is greater than the set maximum days for refill prescription at facility level (that is 50 days).

  • Authorized Drug - Select this checkbox to generate  separate order ID for authorize drugs for outpatient. This is  applicable only for new orders.

 

  1. Print Refill Prescriptions along with Main – Select the check box to print refill prescriptions along with the Main prescription. If the check box is cleared, only the main prescription is printed. This field gets enabled only when ‘Allow Refill’ is selected.

 

Default Frequency for IV Order

 

  1. Frequency – Displays all the continuous frequencies created within the Application Masters.  Select the required frequency from the drop-down list. The default value would be ‘Select’.

The frequency set in this new parameter will be defaulted within the screens mentioned below:

• Preview Rx– Frequency tab

• Existing Orders – Order Catalog tab and

• Patient Drug Profile – Dosage Details tab.
