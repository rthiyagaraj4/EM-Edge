Merge Duplicates

From this function you can Merge duplicate patient records identified in the ‘Identify Main/Duplicate Patients’ and ‘Enter Duplicate Patients’ function by searching the desired record(s).

The duplicate patient number record(s) is/are deleted from the Master Patient Index and an audit record is created linking the duplicate(s) and main Patient number.

The access to this function is determined by the User Access Rights set for the logged-in user. If the rights are not defined either for the logged in user or the right for the DR process, ‘Merge’ is set as ‘None’ for the logged in user and the access is denied for the user. If the privilege level is set as Logged-in facility, the user can perform the ‘Merge Duplicates’ DR process for the Logged-in facility. If the privilege level is set up as All Facilities, the user is allowed to access the function and can perform the ‘Merge Duplicates’ DR process for all the facilities.

The merging of two records results in an error, if the patient is an Inpatient.

Merging Duplicate Patient Records

From the DR Processes, select the Merge Duplicates function.

The Merge Duplicates screen gets displayed.

  1. Select the desired Main Patient ID from the Duplicate Groups frame to display the duplicate details in the Group Details frame. Click the + hyperlink from the Duplicate Groups frame to view the Main Patient Details recorded during registration. Select the check box against the required Main Patient ID to enable [Start Now] and [Start Later]. Click Select All check box to select all the Main Patient ID's under Duplicate Groups frame.

    1. Patient ID - The Main Patient ID and the Duplicate Patient ID's followed by the Name of the patient, Age and Gender are displayed.

    2. Main - A tick mark is displayed for the main patient record.

    3. Duplicate - A tick mark is displayed for the duplicate records.

    4. Do not consider - A tick mark is displayed for records not considered.

Or

  1. Enter the desired Patient ID or click the [?] function to search and select the required Patient ID. You can use the criteria to search for patients under the duplicate groups as an alternate method of searching. If the number of duplicate groups are more, you can narrow down the patient search by specifying the Patient ID as search criteria. The logged in facility is displayed and disabled in the Facility field. Click [Search] to view the search results.

    1. Patient ID - The Main Patient ID and the Duplicate Patient ID's followed by the Name of the patient, Age and Gender are displayed.

    2. Main - A tick mark is displayed for the main patient record.

    3. Duplicate - A tick mark is displayed for the duplicate records.

    4. Do not consider - A tick mark is displayed for records not considered.

  2. Click the + hyperlink from the Group Details frame to view the Main and Duplicate Patient Details recorded during registration.

  3. On selecting the check box against the Main Patient ID under the Duplicate Groups frame, click [Start Now] to start the merge process immediately or enter date and time and click [Start Later] to start the merge process later.

    Date & Time - Enter the date & time in dd/mm/yyyy hh:mm format to start the Merge Duplicate Process at the specified date & time. The date & time entered must be greater than or equal to current date & time.

  4. Click [Reset] to clear the details displayed in the Group Details frame.

 

On starting the merge process, the merge process is submitted as job. You can monitor the merge process using the ‘Administer Search Process’ function.
