Review Merge Process Errors

From this function you can View the list of duplicate groups that failed in the merge process and Reset the error status of transactions of Merge Duplicates process.

You can reset selected or all error transactions after taking appropriate action to rectify the error situations identified from the error report. The Merge Duplicates process when initiated subsequently processes such transactions, where error status is reset.

Reviewing Merge Process Errors

From the DR Processes, select the Review Merge Process Errors function.

The Review Merge Process Errors screen gets displayed.

  1. Select the desired Main Patient ID from the Duplicate Groups frame to display the duplicate details in the Group Details frame. Click the + hyperlink from the Duplicate Groups frame to view the Main Patient Details recorded during registration. Select the check box against the required Main Patient ID to enable [Reset to Merge Process]. Click Select All check box to select all the Main Patient ID's under Duplicate Groups frame.

    1. No of Groups - The total number of groups that cannot be merged is displayed.

    2. Main Patient ID - The Main Patient ID identified is displayed. The patient details displayed below the main Patient ID are Name, Age in Years and Months and Gender. Click the hyperlink of the Patient ID. The details about the main and duplicate patient records pertaining to that group is displayed in the Group Details area.

    3. Count - The total number of records identified as Main/Duplicate for each group is displayed.

    4. Select All - Select the check box to select all the groups identified as Main/Duplicate reviewing or select each Group individually.

Or

  1. Enter the desired Patient ID or click the [?] function to search and select the required Patient ID.  You can use the criteria to search for the patients under the duplicate groups as an alternate method for searching. If the number of duplicate groups are more, you can narrow down the patient search by specifying the Patient ID as search criteria. Click [Search] to view the search results.

    1. Patient ID - The Main Patient ID and the Duplicate Patient ID's followed by the Name of the patient, Age and Gender are displayed.

    2. Main - A tick mark is displayed for the main patient record.

    3. Duplicate - A tick mark is displayed for the duplicate records.

    4. Do not consider - A tick mark is displayed for records not considered.

     

  2. Click [Reset to Merge Process] to resubmit the failed process for merging using ‘Merge Duplicate’ function or [Error Details] to view the error details.
