Package Definition

From this function you can Define a Package applicable for a facility. You can Record the package discount amount that can be issued during transactions and Copy the services of a package to another. You can Query to view the existing package definitions.

Package Definition function is used to define a package valid for the facility. You can define the base price for a package and also the customer specific price for the package. The charges for a package are computed based on the price setup through this function. You can also specify the discount amount on a package for a specific Customer or Billing Group.

Package is a set of services, defined with a lump sum amount instead of normal individual billing services. With the Package Definition option you can frame the package based on the services applicable for Inpatient/Outpatient for the Cash/Credit Patients with different pricing and Package Services for different period. With definition of Package name, the set of services (Charge Based/Rate Based) are associated with the package with quantity/amount limit for the Services at Group/Classification/ Billing Service level.

 

Package Amount can be defined for different billing classes with Rate/Price factor and episodes applicable can be determined. Package can be defined exclusively for Outpatient and Inpatient episode. You have the option of charging the Package Service Price based on the Standard/User defined/As per rule/Price class definition. A new package can be created with the query of existing package and through the Copy function. You can modify the package definition through this function. Discount for the Package can be defined for a Billing Group/Customer with date range options.

Defining A Package

From the Setup > Package, select Package Definition Function.

The Package Definition screen gets displayed.

  1. Enter the details in the mandatory (*) fields in the following tabs:

Package

You can Define a Package through this tab and record the base price for a package. You can also associate rate or charge based services to the package. The details are recorded under three main tabs:

Enter the following details to define a Package:

  1. Code - Enter the Package code in alphanumeric format of eight characters maximum length. This is a mandatory field.

  2. [Copy] - Click [Copy] to copy the Billing Services from an existing package.

  3. Long Desc - Enter the long description in alphanumeric format of 100 digits maximum length. This is a mandatory field.

  4. Short Desc -  Enter the short description in alphanumeric format of 40 digits maximum length. This is a mandatory field.

  5. Gender - Select a gender from the list. The system-defined values are 'Male', 'Female' and 'All'. This is a mandatory field.

  6. Effective From - Click the Calendar function to select Effective From date. The restriction on the date selection is based on the number specified for 'Package Effective from Back dated Days' in 'Package Parameter' function. This is a mandatory field.

  7. Effective To - Click the Calendar function to select Effective To date. The package is available only till the selected date and will not be available for subscription after the date specified.

  8. Min Age - Enter the age in yyyy/mm/dd format. The corresponding package can be associated to the patient only if the age of the person is greater than the minimum age specified. This is a mandatory field.

  9. Max Age - Enter the age in yyyy/mm/dd format. The corresponding package can be associated to the patient only if the age of the person is lesser than the maximum age specified.

  10. Episode Type - Select an Episode Type for whom the package is valid.

  11. All - Select All, if the package is valid for any Episode Type. The check boxes IP, OP, DC and EM are selected automatically when you select 'All'.

  12. IP - Select the check box, if the package is available for inpatient subscription. Clear the check box, if the package is not available for inpatient subscription.

  13. DC - Select the check box, if the package is available for daycare patient subscription. Clear the check box, if the package is not available for daycare patient subscription.

  14. OP -  Select the check box, if the package is available for outpatient subscription. Clear the check box, if the package is not available for outpatient subscription.

  15. EM - Select the check box, if the package is available for emergency subscription. Clear the check box, if the package is not available for emergency subscription.

  16. Order Associated - Select the check box if the orders are associated to the package. The check box is enabled for selection only if the Outpatient Episode Type is selected. If the check box is selected, Practitioner selection becomes mandatory for the respective package during visit registration.

  17. Across Encounters - Select the check box, if the package is available for subscription for all the episode types and for order/non-order associated packages. The selected package is valid for multiple patient encounters if the check box is selected.

  18. Valid Days - Enter the number of days in three digits maximum length to indicate the number of days during which the package is available. The value is set to '1' automatically when the 'Episode Type' is selected as 'OP'. This is a mandatory field.

  19. Applicable To - Select a value from the list. The system-defined values are 'Cash', 'Credit' and 'Both (Cash and Credit)'. You can define Customer specific price for a package only if the values are selected as 'Credit' or 'Both (Cash and Credit). If the value is selected as 'Cash', the 'Customers' tab is disabled.

  20. Approved - Select the check box to approve the package. The check box is enabled for selection only if the user is authorized for package definition through the 'Package User Access' function. The package is available for subscription and association only if it is approved.

  21. Approved By - The name of the logged in user is displayed, by default on selecting the 'Approved' check box. You cannot update the entry.

  22. Approved Date  - System date is displayed, by default on selecting the 'Approved' check box. You cannot update the entry.

  23. From Date - The date selected for 'Effective From' is displayed, by default. Click the Calendar function to select a date. This is a mandatory field.  The date selected should not overlap with the Base Price and it has to be within the Effective date range selected.

  24. To Date - Click the Calendar function to select a date. The selected date has to be greater than the selected 'From Date'. If you do not specify the date, a day prior to the 'From Date' selected for the  subsequent record is displayed as the 'To Date'. The date selected should not overlap with the Base Price and it has to be within the Effective date range selected.

  25. Package Price Class - Click [?] function to select and associate a Package Price Class. The Package Price Class code and definition are listed based on the values defined through the 'Price Class Specific Price' function. The selected Price class acts as a base for the rate based Package Services and Package Price (charge based) amount indicator option when selected as Price Class. During transactions for individual services, the associated values are listed, if the 'Amount Indicator' is selected as 'Price Class'.

  26. Outside Package Price Class - Click [?] function to select and associate an active Price Class code. The services that are not a part of the package are charged based on the standard price defined for the selected Price Class.

  27. Accounting Code - Click [?] function to select and associate an Accounting Code. The package details are posted to the selected Accounting Code.

  28. VAT applicable –  Select this checkbox to enable VAT percentage text box.

  29. VAT percentage – If you select the VAT Applicable checkbox, the rules which are of type “Main” from “Rule for Additional Charge” screen, are displayed in the look up of this text box.

  30. Select - Select the radio button to associate Package price class for the respective encounter date.

  31. [Add] - Click [Add] to add a new record.

Customers

This tab is enabled for selection when the 'Applicable To' value is selected as 'Cash' or 'Both (Credit & Cash)'. The Package definition screen is invoked through which you can associate the services and record the price for a specific Customer. You can also associate rate or charge based services to the package and record price for each Billing Service that is associated.

Enter the following details:

  1. Customer Group - Click [?] function to select a Customer Group. The definition of the selected Customer Group is displayed.

  2. Customer - Click [?] function to select a Customer. The Customers' associated to the selected Customer Group are displayed. The definition of the selected Customer is displayed.

  3. Valid Days - Enter the valid days in numeric format to indicate the number of days during which the package is available for subscription for the selected Customer Group and Customer.

  4. Dtls - Click Dtls displayed as a hyperlink to define the package price class and associate accounting code for posting.

  5. [Add] - Click [Add] to associate the selected details to the package.

The following details are displayed on clicking Dtls hyperlink:

  1. Date [From - To] - Click the Calendar function to select the date range. The selected package is valid for the respective Customer Group and Customer during the period selected.  The date selected should not overlap with the Base Price and it has to be within the Effective date range selected.

  2. Package Price Class - Click [?] function to select the Package Price Class. The values defined through the Price Class Specific Price function with the Package check box as selected listed.

  3. Outside Package Price Class - Click [?] function to select the Package Price Class for the services that are excluded from the package. The values defined through the Price Class Specific Price function with the Package check box as selected listed.

  4. Accounting Code - Click [?] function to select and associate an Accounting Code. The package details are posted to the selected Accounting Code.

  1. Select - Select the radio button to associate Package price class for the respective encounter date.

  2. [Add] - Click [Add] to add a new record.

Package Discount

You can specify the discount for a Billing Group and Customer through this tab.  

Enter the following details:

  1. Customer Group - Click [?] function to select a Customer Group. The definition of the selected Customer Group is displayed.

  2. Customer - Click [?] function to select a Customer. The Customers' associated to the selected Customer Group are displayed. The definition of the selected Customer is displayed.

  3. Billing Class - Click [?] function to select a Billing Class. The description of the selected Billing Class is displayed.

  4. Effective[From - To] - Click the Calendar function to select the date range. The selected package is valid for the respective Customer Group, Customer and Billing Class during the period selected.  

  5. Discount Type - Select a value from the list. The system-defined values are 'Amount; and 'Percent'.

  6. Discount - Enter the value in numeric format to indicate the discount amount or discount percentage.

  7. Select - Select the check box to associate Package price class for the respective encounter date.

  8. [Add] - Click [Add] to record the discount details.

  9. [Delete] - Click [Delete] to delete the records that are associated.

  10. Billing Group - Click [?] function to select a Billing Group. The description of the selected Billing Group is displayed.

  11. Billing Class - Click [?] function to select a Billing Class. The description of the selected Billing Class is displayed.

  12. Effective[From - To] - Click the Calendar function to select the date range. The selected package is valid for the respective Billing Group and the Billing Class combination during the period selected.  

  13. Discount Type - Select a value from the list. The system-defined values are 'Amount; and 'Percent'.

  14. Discount - Enter the value in numeric format to indicate the discount amount or discount percentage.

  15. Select - Select the check box to associate Package price class for the respective encounter date.

  16. [Add] - Click [Add] to record the discount details.

  17. [Delete] - Click [Delete] to delete the records that are associated.

  1. Click [Copy] to copy the package details from an existing package and define a new package with the same services. A new pop-up screen is displayed. Enter the details in the mandatory (*) fields and click [OK] to create or [Cancel] to cancel the transaction. This button is enabled only during package definition.

  2. Click [Modify] to update the package details. Authorized users are only permitted to login and modify the details. An authorization screen pops-up requesting mandatory details. Enter the details and click [OK] to login or [Cancel] to cancel the transaction. This button is enabled only in query mode.

    Enter the following details:

    1. User ID - Enter the User ID in alphanumeric format. This is a mandatory field.

    2. PIN - Enter the PIN recorded for the respective user. This is a mandatory field.

    3. Remarks -  Enter the Remarks in alphanumeric format. This is a mandatory field.

  3. Click [Query] to view the details recorded.

  4. Click [Apply] to save the details or [Reset] to clear the details.
