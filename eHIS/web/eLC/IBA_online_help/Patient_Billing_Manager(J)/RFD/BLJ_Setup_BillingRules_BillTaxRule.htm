Enter the following Details

  1. Patient Class – Select the Patient class from the drop down. The Patient classes defined in the billing class are listed in the drop down list. This is mandatory fields.

  2. Billing Group – Select the Billing Group using lookup function. The Billing Groups defined in the billing masters are listed.

  3. Customer Group – Select the Billing Group using lookup function. The Customer Groups defined in the billing masters are listed. If a particular customer group is selected, the customers mapped to that customer group alone will be listed in customer lov.

  4. Customer – Select the customer using lookup function. The customers defined in the billing masters are listed.

  5. Rule Code – Select the Rule Code using lookup function. The Rule codes defined in the Rule for Additional Charge are listed. The Rule code is associated to patient class / billing group /customer group / customer code / policy combination.

  6. Policy Type – Select the Policy Type using lookup function. The Policy Type defined in the billing masters are listed.

  7. Effective [From-To] – Select the Date using calendar function. The Bill Tax Rule will be applicable based on the selected date.

 

Criteria for Tax

In this section, you can define the service tax rule that will be applied to the cosmetic services. If any of the codes defined in this block are present in the bill, the bill becomes eligible for cosmetic service tax calculation.

 

  1. Type – Select the Rule Type using lookup function. The Rule Type defined in the Rule for Additional Charge are listed.

  2. Service Code – Enter Service code in the alpha numeric character format. If billing service is selected in type, all the billing services will be listed in the lookup function.

  3. Add – Click this button to add a new row.

  4. Remove – Select this checkbox to remove the selected record.

  5. Enabled – Select this checkbox to enable the rule which has been created. Only if the rule is enabled it will be applied in the corresponding bill. If the rule is not needed to be applied in the bill, the user can clear the check box

 

If there are two rows in header block, one for OP patient class and another for ** patient class, and if OP visit is registered, priority will be given to OP patient class row setup. If the rule code created in Rule for additional charge is end dated, automatically the entry in Cosmetic Service Tax Applicability screen (new screen) will become obsolete. More than one combination of Billing Group, Customer Group, Customer, Rule code and Policy are not allowed in this screen. This section can be empty for the header block. If it is empty, the rule will not be applied in the bill.

 

Criteria For Tax

This section is used to to define the tax rule. If any of the codes defined in this section are present in the bill, the bill becomes eligible for cosmetic service tax calculation.

 

  1. Type – Select the Rule Type using lookup function. The Rule Type defined in the Rule for Additional Charge are listed.

  2. Service Code – Enter Service code in the alpha numeric character format. If billing service is selected in type, all the billing services will be listed in the lookup function.

  3. Inclusion/Exclusion – Select a value ‘Include’ or ‘Exclude’ from drop down list. Based on selection, the services are included or excluded.

  4. Add – Click this button to add a new row.

  5. Remove – Select this checkbox to remove the selected record.

 

The current excluded services will be applied only on services which are outside the packageBased on the bill tax rule master, the tax will be calculated on the bill amount. 0Payer coverage check box will not be considered for calculating the tax created in Bill Tax Rule master screen.
