Enter the following details to define Package Parameters:

 

  1. Package Effective from Back Dated (Days) - Enter the number of days in numeric format of three digits maximum length to record the number of days allowed for package creation from the current system date. You can specify the number of days if the check box 'Package Applicable' is selected.

  2. GL Posting by Summary - Select the check box to post summarized General Ledger details.

  3. Validity for Availing Package Service Orders (Days) - Enter the number of days to avail the package services in numeric format of three digits maximum length. You can specify the number of days if the check box 'Package Applicable' is selected. It indicates the grace period to utilize the package services.

  4. Variance Service Code - Enter the Billing Service Code for variance. Variance is computed as [Package Price - Actual Service Price].

  5. Auto Refund Applicable - Select the check box if automatic refund is applicable. The check box is selected in 'Package Definition' function enabling you to issue a refund, if you select the check box. You cannot issue a refund if the check box is cleared.

  6. Validity for Package Service Refund (Days) - Enter the number of days in numeric format of three digits maximum length. It indicates the grace number of days to issue a refund for the unavailed services in the package.

  7. Patient Deposit for Package - Click [?] function to select a patient deposit receipt type for package transactions. The Receipt Types that are defined through the 'Receipt Types' function with the 'Receipt Nature' as 'Patient Deposit' is listed. A receipt number is generated with the selected receipt type when patient deposit is recorded through the 'Enter OP Receipts/Refunds' or 'Enter IP Receipts/Refunds' function.
