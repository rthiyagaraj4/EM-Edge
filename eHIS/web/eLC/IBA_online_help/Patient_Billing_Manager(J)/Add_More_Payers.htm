Add More Payers

You can associate multiple Payers to a patient by clicking [Add More Payers]. The Billing Service price is computed and distributed among the Payers based on the coverage limit specified.

 

Enter the following details to associate additional Payers to a Billing Group:

 

  1. Encounter Date/Time -  Current date and time gets displayed as the Encounter date and time.  

  2. Search By - Select a value from the list to query Payer details based on the 'Country' or 'Region'. The values in the associated field gets updated based on the value selected for 'Search By'.

  3. Country/Region - Click [?] function to select a Country or Region. The legend changes based on the value selected for 'Search By'.

  4. Payer Group - Click [?] function to select a Payer Group. Customers defined in the Patient Billing Manager module gets listed. This is a mandatory field.

  5. Payer - The legend is displayed as a hyperlink. Click the hyperlink to view Customer details. Click [?] function to select a Customer. This is a mandatory field. The values defined through the 'Customer' function in Patient Billing Manager is displayed.

  6. Priority – Enter the Priority in numeric format with a maximum of two digits. The Priority indicates the precedence of the Payer in the list. This is a mandatory field.

  7. Initial credit Authorization - Enter the Initial credit Authorization in numeric format. This is a mandatory field.

  8. Initial Credit Auth Date - Click the calendar function to select the authorization date. This is a mandatory field.

  9. Approved Amount - Enter the coverage amount in numeric format. This is a mandatory field.

  10. Approved Days - Enter the number of days during which the approved amount can be availed by the patient. This is a mandatory field.

  11. Policy - Click [?] function to select a Policy. Policy Types defined in the Insurance Management module gets listed. This is a mandatory field.

  12. Policy No – Enter the Policy Number in numeric format with a maximum of 30 digits. This is a mandatory field.

If the patient continues to stay in the facility and continues to use the same policy for coverage, you can extend the coverage details by selecting a new effective date for the Payer, Payer Group and Policy Type combination. Following are the validations that will enable you to associate the same Payer, Payer Group and Policy Type combination to the patient for a different effective date.

=> You can associate the same Payer, Payer Group and Policy Type combination to the patient, if the Policy Numbers are different.

=> The effective dates for the selected combination has to be different, if the Payer, Payer Group and Policy Type combination are the same.

=> A single bill is generated for the same Payer, Payer Group and Policy Type combination with same policy numbers and different effective dates. Separate bills are generated for different Payer, Payer Group and Policy Type combination.

=> If you choose the same effective date for the Payer, Payer Group and Policy Type combination that is already associated to the patient, an error message 'Policy Eff. date cannot overlap for the same Payer Group, Payer, Policy Type and Policy Number combination.' is displayed.

=> You can associate any other Payer, Payer Group and Policy Type to the patient, if there is a gap in the effective dates. If the coverage details are not recorded, the amount for the services utilized by the patient are considered to be patient payable.

=> The Billing Group recorded during patient registration is displayed, by default, during the patient admission and visit registration, if the check boxes 'Assign During Admission' and 'Assign During Visit' are selected for the respective Billing Group.

=> System checks for valid flags at the Customer level based on the 'Customer' definition in Accounts Receivable module, if the check boxes 'Assign During  Admission' and 'Assign During Visit' are cleared for the respective Billing Group. System defaults the Customer instead of the Billing Group if any check box is selected in the 'Customer' function under the 'Customer Valid Flags' tab.

=> You have to enter the financial details manually, if the respective check boxes are cleared in 'Billing Groups' and 'Customer' functions.

  1. Policy Start Date – Click the Calendar function to select a date. The selected Policy Type is considered valid from the date selected. The system date is displayed as the default date. This is a mandatory field.

  2. Policy Expiry Date – Click the Calendar function to select a date. The selected Policy Type is considered valid until the date selected. If the check box ‘Default Policy Start Date’ is selected and the number of days recorded in ‘Default Policy Expiry Days’ field in the ‘Setup Service Coverage Detail by Policy’ function in ‘Patient Billing Manager’ module, then the expiry date is calculated based on the number of days specified. Click the Calendar function to select any other date as the expiry date. This is a mandatory field.

  3. Policy Effective From – Click the Calendar function to select the date beginning when the Policy is effective. The Policy Start Date is set, by default.

  4. Policy Effective To – Click the Calendar function to select the date until when the Policy is effective. The Policy Expiry Date is set, by default. The ‘Policy Effective To’ date has to be within the range specified for ‘Policy Start Date’ and ‘Policy Expiry Date’.

  5. Adjustment Rule Indicator - The adjustment rule indicator selected for the selected Payer Group, Payer and Policy combination in Insurance Management module is displayed.

  6. %Amount Indicator - The percentage amount indicator selected for the selected Payer Group, Payer and Policy combination in Insurance Management module is displayed.

  7. Adjustment Value - The adjustment value selected for the selected Payer Group, Payer and Policy combination in Insurance Management module is displayed.  

  8. Adjustment Interface Code - The adjustment interface code selected for the selected Payer Group, Payer and Policy combination in Insurance Management module is displayed.

  9. Click [Accept] to associate additional Payers to a Billing Group or [Cancel] to exit from the screen.
