Characteristics Tab

Synonyms Tab

Synonyms are the different names given to an Order Catalog to be used by various departments. These are basically secondary names categorized as Synonym Type. Primary synonym is the Short Description.

  1. Type - The types displayed are Direct Care and Ancillary Dept. Ancillary Department refers to Radiology and Pathology, where as Direct Care refers to Practitioner type like Physician, Dentist, Surgeon, etc. The name specified for Direct Care is available for users of Direct Care Department, while Ancillary Department users get the name defined for Ancillary Department while placing an order.

  2. Name - Enter the name for the type in alphanumeric 40 characters maximum length.

  3. Active – Select the check box and the Name is applicable for the selected department.  

 

Duplicate Check

The criteria for checking Duplicate Orders are defined in the tab.

  1. Required - Select the check box, if duplicate check is required for the defined catalog. If duplicate check is not required, clear the check box. On selecting the check box, other fields are enabled.

  2. Order Priority Applicable (Routine, Urgent, Stat) - Select the check box according to the Priority required for the order catalog.

  3. Status Considered - Select the value from the list. The values displayed are  'Active Orderable Status', 'Any Orderable Status' and 'Result Completed Order Status’. This field is enabled only when duplicate check is applicable (that is Required check box is selected).

  4. For status as 'Active Orderable Status', the duplicate check is performed on all the active orders of defined catalog (an order is active till it is registered).

  5. For status as 'Any Orderable Status', the duplicate check is performed for any order status of the catalog (that is from pending consent till fully resulted except for cancelled, hold or discontinued orders).

  6. For status as 'Result Completed Order Status', the duplicate check is performed based on order completed status only. The system either prompts a warning message or does not allow you to place same drug again for the patient in Place Order once the order is complete. In case if the previous Order is not completed, then system will allow you to place the order till the Order is completed.

  1. Action Type - Select the action type from the list. The listed values are 'By Orderable' and 'By Activity Type'.

Type

  1. Behind Action - Select the required value from the list. The listed values are Reject, Warn and No Check. This field is enabled only when 'Required' is selected. If previous orders exist for the Order Catalog and the order is placed for the same Order Catalog,  the action is performed, based on the Action Type selected. Select 'Reject' and duplicate orders are rejected. Select 'Warn' and a warning message is displayed while placing an order, indicating that the order is a duplicate. You can either proceed to place the order or cancel the duplicate order. On selecting ‘No Check’, duplicate check is not performed for the duplicate order placed.

  2. Period - Enter the period in numeric format of three characters maximum length. Duplicate check is performed for previous orders within the specified period.

  3. Unit - Select the unit from the list. The values displayed are Months, Weeks, Hours, Days and Minutes.

  4. Ahead Action - Select the required value from the list. The values displayed are Reject, Warn and No Check. This field is enabled only when 'Required' is selected. If future order exists for the Order Catalog and it is placed again, action is performed based on the type selected. If 'Reject' is selected, the ordering process is rejected. If 'Warn' is selected, you are warned. Duplicate check is not performed for the order placed again for value as 'No Check'.

  5. Period - Enter the period in numeric format of three characters maximum length. Duplicate check is performed for future orders within the specified period.

  6. Unit - Select the Unit from the list. The values displayed are Months, Weeks, Hours, Days and Minutes.

If the status considered is ‘Active Orderable Status’ and ‘Behind action’ is ‘Reject’ with period as ‘two weeks’, duplicate check for orders placed for the defined Order Catalog since last two weeks from the current system date is performed. If any order is placed within these two weeks, new order for the defined Order Catalog cannot be placed, as it is rejected. If behind action is ‘Warn’, you are warned that order is placed for the Order Catalog within the specified period. If behind action is ‘No Check’, you can place order without any hindrance, the same explanation holds for ‘Ahead Action’, but duplicate check is performed for future orders placed.

  1. Duplicate Override Reason Required – Select this check box to allow you to enter the reason to override the duplicate order while placing orders in Clinician Access (CA) module. When this check box is selected, it is mandatory to enter the reason for overriding duplicate orders.

 

Consent/Refusal

The Consent/Refusal tab helps you to attach a consent/refusal form to the Order Catalog.

  1. Consent Required - Select the check box, if consent is required from the patient. Clear the check box, if consent is not required. Consent Stage, Consent Form List and Consent Note text field are enabled, if the check box is selected.

  2. Consent Form List - Select the Consent Form List as 'Fixed' or 'Variable'. Fixed can be selected, if patient consent form has to be fixed while placing an order. Variable can be selected, if patient consent has to be variable while placing an order.

  3. Consent Stage - Select the Consent Stage as 'Before order Release' or 'Before Order Registration'. 'Before Order Release' can be selected, if patient consent is required before order is released by the Practitioner. 'Before Order Registration' can be selected, if consent is required before order is registered by the Practitioner.

  4. Multiple Consent - Select the check box to select the number of consent form required Before Order Release and Before Order Registration. Consent Note and Consent Stage gets disabled on selecting this field.

  5. Before Order Release – Enter the number of consent forms in numeric format with a maximum of two digits. The number indicates the consents required before order release.

  6. Before Order Registration – Enter the number of consent forms in numeric format with a maximum of two digits. The number indicates the consents required before order registration.

  7. Consent Note - Select the Consent Note text from the list. This field is enabled only when 'Consent Required' is selected. The Consent Notes defined using the function 'Consent/Refusal/Referral Notes' get listed. The record of type 'Consent Form' is populated.

  8. Refusal Note Text - Select the Refusal Note Text from the list. Refusal Note Text is defined using the function 'Consent/Refusal/Referral Notes'. The record of type 'Refusal Form' is populated.

 

Explanatory Text

The explanation for the Order Catalog has to be recorded. The Explanatory Text entered is displayed in Clinician Access module while the order is placed. The text is displayed, on mouse hover over the Order Catalog description.

  1. Text - Enter the Explanation for the Order Catalog in alphanumeric format of 255 characters maximum length. The explanatory text appears on moving the mouse over the Order Catalog in Clinician Access module during the transactions. A [?] is displayed adjacent to the Order Catalog name in the Orderable Column, if the Explanatory Text is recorded for the selected Order Catalog.
