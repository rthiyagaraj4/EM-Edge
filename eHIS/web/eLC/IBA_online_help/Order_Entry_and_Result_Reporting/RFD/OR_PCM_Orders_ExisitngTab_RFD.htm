Enter the following details:

 

  1. View - Select the value from the list provided. The values displayed are 'Current Encounter Orders' and 'All Orders'. If 'Current Encounter Orders' is selected, all existing orders of the current encounter are displayed. If 'All Orders' is selected, all the orders placed during various encounters (including the current one) of the patient are displayed.

  2. Status - Select the status from the list provided. The values displayed are All, Pending for Reviews, Outstanding, Active, Completed, Future Order, Discontinued/Cancelled, Not Done, and Hold

  3. Priority - Select the value from the list. The values displayed are Stat, Urgent and Routine.

  4. Period [From-To] - Enter the period in date format of dd/mm/yyyy. The result set displays orders placed within the specified period.

  5. Order Category - Select the order category from the list.

  6. Order Type - Select the order type from the list. The values displayed correspond to the selected order category. If order category is not selected, order types belonging to all the order categories are listed.

  7. Activity type - Select the Activity type from the list. The values displayed correspond to the selected order type.

  8. Catalog - Enter the order catalog in alphanumeric format of 40 characters maximum length.

  9. Search Criteria - Specify the search criteria from the list. The values listed are Starts with, Contains, and Ends with. Starts With is displayed, by default.

  10. Order By - Select the order by from the list. The values listed are Order date time, order category and order type. The result set displayed is sorted based on the selected value.

  11. View Order by - Select the value as either Header or Line. If the selected patient is an Inpatient, the Line appears, by default. If the selected patient is outpatient/emergency/day care/external patient, the header appears, by default. The default value can be changed.

  12. Group By  – Select the value from the list provided. The values displayed are ‘Order Date’, ‘Order Category’ and ‘Order Type’.

  13. If Order Date is selected, all orders are displayed as per the date of order.

  14. If Order Category is selected, all orders are displayed as per the order category.

  15. If Order Type is selected, all orders are displayed as per the type of order.

  16. All these orders are sorted in descending order.

  1. Order Frequency  - Select the value from the list provided. The values displayed are ‘Single Order’, ‘Continuous Order’and ‘All’.

  2. If Single Order is selected, non repetitive orders are sorted in ascending order.

  3. If Continuous Order is selected, orders that are repetitive and must be done at regular intervals are sorted in descending order.

  4. If All is selected, all the orders are displayed.

  1. As on - The current application server date and time are displayed.

  2. Start Date - The order Start date and time is displayed as read-only text.

  3. End Date - The order End date and time is displayed as read-only text.

  4. Priority - The priority of the order is displayed.

  5. Order Type - The order type of the orderable is displayed.

  6. Order Catalog - The displayed order catalog is hyperlinked. Click on the link and a screen appears displaying actions that can be performed on the order catalog. Actions like viewing, canceling, amending, order on hold, resume or discontinuing can be performed depending on the status of the order catalog (these tasks are to be defined for the logged in user in the function 'Tasks for Resp/Rln' of Clinician Access' Module). If the logged in user belongs to ancillary departments like radiology or pathology, the ancillary department synonym of the order catalog is displayed. For other users, direct care synonym or short description (if direct care synonym is not available) is displayed.

  7. If the remark for the alternate drug exists, a new icon  appears in the Order Catalog column for all the orders . Click and the system will launch the existing Dispensed Locations screen. Click the Remarks hyperlink in the Remarks column to view the existing remarks for the alternate drug in the Remarks dialog window.

  1. Ordered By - The ordering practitioner name is displayed.

  2. Status - The status of the order catalog is displayed. Both system and user defined status values are displayed. If the results are available for an order, the order status is displayed as a hyperlink. Click Status hyperlink to view the order details in the 'Result-Lab Order' dialog. Click [View Audit Trail] to view the audit trail details of the order. Click [Print] to print the order. Click [Close] to close the dialog.

This enables online printing of lab reports in same format in both 2-Tier and 3-Tier modules. Setup needs to be done for appropriate report, printer destination and facility in the Print Routing function in the System Manager module to submit the laboratory reports to online Report Printer. 2 Tier module should be installed and order category should be ‘Laboratory’.

 

You can discontinue the Order either at the Header level or Line level.

Select the system-defined 'Refund' values from the list. The listed values are ‘Yes’ or ‘No’. Select ‘Yes’ to refund the discontinue order or select ‘No’ if you do not want to refund the discontinued order.

The radiology icon is displayed and on mouse over the details Ordered from, Reason for ordering if recorded are displayed, these details are static.
