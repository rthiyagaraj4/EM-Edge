External Order

From this function you can Place orders for external patients who do not have an active encounter in the logged in facility.

The order is also placed for specimens that are received from external referral facilities that are send for processing. It is mandatory for the Patient ID to be identified in this function. If the patient has already been registered in the logged in facility then the same can be used to place the order else the patient has to be registered in the facility before placing the order. This function has provision for the practitioner to view the previous orders placed for the patient.

Placing External Orders

From the CA Transactions > Clinician Access > Desktop Menu > Order Management, select External Orders function.

The Place External Orders screen gets displayed.

  1. Select the Patient using [?] function. This field is mandatory.

  2. On selecting the required Patient, the Patient Name, Sex and Age are displayed.

The screen displays two tabs

New Orders

  1. Referral Facility – Select the Referral Facility from where the patient has been send from the list. The values in the list display all the referral facilities identified in the master ‘Referral Source’ in Application Masters module. This field is mandatory.

  2. Referral Practitioner – Select the name of the practitioner who referred the patient or the specimen to the logged in facility in alphanumeric format of 30 characters maximum length. This field is mandatory.

  3. Date of Registration – Enter the date and time on which the external order registration is done. The system date and time is displayed, by default. This field is mandatory.

  4. Remarks – Enter the Remarks in alphanumeric format of 500 characters maximum length. This field is mandatory.

  5. Order Category – Select the Order Category to which the orderable to be recorded is associated.  

  6. Order Type - Select the Order Type to which the orderable to be recorded is associated.

  7. Activity Type - Select the Activity Type to which the Orderable to be recorded is associated.

  8. Orderable – Enter the Orderable for which the order has to be placed or select the orderable by using [?] function.

  9. If [Place Order] is clicked without choosing the orderable in the ‘External Orders’ function the ‘New Orders’ function is invoked where the orderable can be chosen using the textual search or from the Tick sheet option. These search options are the same that can be found when the ‘New Orders’ tab is invoked from the ‘Order’ function in the patient chart menu.

Order Entry Preview screen is displayed on clicking [Place Order]. Charge details for the services associated to the Order Catalog are displayed. Click ‘View/Edit’ hyperlink to modify the charge details. Select the check box adjacent to a service to update or delete the records. Click [Modify] to update the details or [Remove] to dissociate a service. Click [Add] to associate services to the selected Order Catalog. Click [Accept] to save the details and place an order.  An authorised cashier logged into the Cash Counter can issue discount for the services associated to the Order Catalog. You can issue discount to a patient while placing an order and collect payment against the bill amount. On entering the discount amount or percentage, discount reason screen pops-up. Cashier has to specify the reason to issue discount for a service. On clicking [OK], Bill settlement screen is displayed using which a Cashier can specify the amount settled for Order placed.

The settlement window and the discount window is displayed, if the following criteria is met:

• User placing order is a valid Cashier with privilege rights to settle bill amount and issue discount.

• 'Billing' check box is selected in 'OP Parameter for Facility' function.

• Billing Service is associated to the Order Catalog.

• Charging Stage is defined for the Order Catalog.

• If the charging stage for the Order Catalog is defined as 'Order', the settlement and the discount screen gets displayed while placing orders.

• If the charging stage for the Order Catalog is defined as 'Charge on Regn' or 'Charge on Compl', the settlement and the discount screen gets displayed on order registration or order completion.

• If the charging stage for the Order Catalog is defined as  'Charge on Result', the settlement screen does not get displayed, but the bill is generated internally and can be taken through billing module.

• The discount to an Order Catalog can be given based on the maximum discount that can be given to the associated Billing Service as per billing service setup definition.

• The settlement screen is not displayed for future orders.

• The settlement screen is not displayed for Order Catalogs which require consent or co-sign or authorization.

Existing Orders

There is a provision to view all the orders that were previously placed for the patient. The list of previous orders placed for a patient can be viewed by clicking on the Existing Orders Tab in the ‘External Orders’ function.

Enter the search criteria such as view by, status, priority, order category, order type, Activity type, period, catalog, order by and view order by. Click [Search] to view the details.

The following Details are displayed:

  1. Column One – Future Order/Active (Renew)/Expiry renew is displayed.

  2. Active (Renew): Active orders are those orders which are active and may require renewal when the expiry date is coming close.

  3. Future orders: Future orders are those which are placed for future date.

  4. Expiry renew: Expiry renew are those are those orders where the expiry date has crossed but may have grace period before which it has to be renewed.

  1. Column Two – Secondary Orders Exists is displayed. The ‘Secondary order exists’ are those for which secondary orders exists and the secondary orders need to be performed prior to performing the existing order  

  2. Ordered At - The Ordered Date and Time is displayed.

  3. Priority - The Priority for the order as identified while placing the order is displayed.

  4. Order Type - The order type of the orderable for which the order has been placed is displayed.

  5. Order Catalog - The orderable for which the order has been placed is displayed. On clicking the hyperlink on the order catalog, it is possible to view, cancel or Amend the order.

  6. Ordered By – The name of the practitioner who placed the order is displayed.

  7. Status – The Status of the order placed is displayed. This field displays various color coding for Appointment Scheduling and Result Reporting. If Appointment Schedule is to be rescheduled, a button ‘S’ is displayed along with the color associated and on clicking the button, the Appointment Schedule screen where Appointment Scheduling can be done is displayed.

  8. Action – Select the check box to either print the selected order or copy the order. To print the selected order, click [Print].
