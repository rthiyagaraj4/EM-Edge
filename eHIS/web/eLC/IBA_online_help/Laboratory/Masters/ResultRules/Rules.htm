Rules

From this function you can Define rules specific to a department by associating attributes. You can Query for the existing records.

The rules necessary to perform tests in a department are defined. You can associate the test codes for which the rule is valid. Rules specific to a department are defined by associating the relevant attributes. The attributes are classified and defined in the following tab pages:

• Setup Rule Components - This tab enables you to select one or more predefined tests as criteria for the rule.

• Review – This tab enables you to set up technical or pathological review for the rule.

• Reflex Tests – This tab enables you to define rule for one or more reflex tests to be performed with the defined rule.

Defining Rules

From the Masters > Result Rules, double-click Rules function.

The Rules screen gets displayed.

  1. Enter the data in the mandatory (*) fields. <Related Field Description>

  2. The screen consists of the following tabs:

Setup Rule Components

Setup Rule Components Tab

The Setup Rule Components tab is selected, by default.

  1. Code - Enter the test code or select a Test Code from the [List of Values] function. The list displays the values as defined through the Test Codes masters function.

  2. Description - The corresponding description for the selected test code is displayed, by default.

  3. Component - Select a Component from the list the listed values are pre defined. This is a mandatory field.

  4. Operator - Select the desired Operator from the list. The valid values are 'Equal To', 'Not Equal To', 'Greater Than', and 'Lesser Than'.

  5. Value - Enter the location code or click [List of Values] function to select it from the list. The listed values are created in 'Outpatient Management' function.

  6. Operator - Select the desired Operator from the list. The valid values are 'And' and 'Or'.

Review

Review Tab

Click Review tab to invoke the corresponding tab frame. This tab is enabled on entering the data in the Rule Code field.

  1. Technical Review – Select the check box if Technical Review is required for this rule.

  2. Path Review Type – Select the Pathological Review Type from the list. The valid values are ‘Normal Review’ and ‘Abnormal Review’.

A warning message ‘Rule is set for Reflex Tests’ is displayed on clicking on ‘Review’ tab if you have selected Reflex Test for the Rule instead of Technical review.

Reflex Tests

Reflex Tests Tab

Click Reflex Tests tab to invoke the Reflex Tests tab frame.

  1. Test Code – Enter the Test Code or select it using [List of Values] function. The list displays the values as defined through the Test Codes function.

  2. Description – The corresponding description for the selected test code is displayed, by default.

  3. Bill – Select the check box if billing is required for the selected test. A warning message ‘Error getting Billing Service Code’ is displayed if you have not selected the billing check box in Test Code master for the selected test code.

  4. Click [Save] or press [F10] to save the created Rules records or click [Clear Record] to clear the entered data.

  5. Click [Execute Query] or press [F8] to search for existing Rules records within the department.
