Enter the following details:

  1. Department - The department name associated to the logged in user is displayed. If the user has access to multiple departments, select the desired department name from Department list.

  2. Perform Site – Enter the Perform Site or click [List of Values] functions to select the desired performance from the list of values. The list displays the values defined in the 'Perform Site' masters in Laboratory module. This field is functional on installation of the Laboratory module.

  3. Referral No – The number assigned by the external source for the patient is displayed.

  4. Patient ID – The Patient ID for the selected specimen number is displayed, by default.

  5. Patient Name – The short name for the patient is displayed, by default.

  6. Specimen No. - Enter the specimen number in alphanumeric format of 18 characters maximum length or select it using [List of Values] function. The number is made up of three components. The first two digits identify the laboratory department and are defined in the department setup. The next two digits identify the year of processing. The last eleven digits are running serial number but vary based on the parameter setup for Specimen Number length.

  7. Age/Sex – The patient’s Age and Gender are displayed.

  8. Source – The source from where the request originated. The valid available values are ‘Ward’, ‘Clinic’ and ‘External’ depending on the episode type. The description for the source code is displayed.

  9. Priority – The priority as selected while placing orders in Clinician Access module for selected patient is displayed and the field is disabled. The valid values are ‘Stat’, ‘Urgent’ and ‘Routine’. You cannot modify this value.

  10. Priority Specimen – Select the check box to consider the specimen as high priority.

  11. Registered Date – The specimen registration date and time is displayed. You cannot modify this value.

  12. Assign To – The name of the person for whom the selected specimen processing is assigned is displayed.
