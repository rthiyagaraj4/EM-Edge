Process Month End

From this function you can Process the transactions in the Accounts Receivable module. The documents have to be posted before executing this function.

This function is used to process the transactions for a given month. On successful completion of process, parameter month is incremented to next month. When last month of the year is reached, a year end process is executed instead of the month end process. System performs certain validations before processing a month end.

  • For a selected period, all the transactions are posted to General Ledger.

  • When interface option is set to “Batch”, General Ledger distributions and the export to General Ledger module has to be complete. On successful completion of process, the system increments the financial month by 1.

Month End Process

From the Period End, double-click Process Month End function.

The Process Month End screen gets displayed.

  1. The Current Accounting Period set in the ‘AR Parameters’ gets displayed, by default. You cannot update the Current Accounting Period.

  2. Select a value from the list to process the records. The options are “Online”, “Batch” and "No Interface". Select one of the options from the list. This indicates whether the interface method to General Ledger is “Online”, “Batch” for the next financial period. Select "No Interface", if the Accounts Receivable module is not interfaced to the General Ledger.

  3. Click [Run Process] to execute the month end process.

  4. A confirmation screen pops-up. Click [Yes] to process the month end or [No] to cancel the process.

  5. A log file with the output is generated. Click [OK] to view the log file or [Cancel] to exit from the screen.

 

The month end can be processed only when the unposted or uninterfaced transactions are posted. The credit note, debit note, adjustment, receipt, payment, invoice transactions have to be posted. If the details are not posted, you cannot process a month end.
