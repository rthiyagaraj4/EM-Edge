Enter the following details to generate an Invoice:

 

  1. Customer Code [From – To] – Enter the required Customer Code range in alphanumeric format of eight characters maximum length or select the required Customer Code from [List of Values]. The Customer Codes defined in the ‘Customer’ function are displayed for selection. If the code selected is valid, the Customer Name is displayed as read-only text for user verification. The Invoice is generated from that particular Customer. If ‘From’ value is not entered, the system assumes the lowest value for the ‘FROM’ value. If ‘To’ value is not entered, the system assumes the highest value for the ‘To’ value. When both values are entered, ‘To’ value must be greater than or equal to the ‘From’ value.

  2. Bill Date [ From – To] – Enter the Bill Date range in the format of dd/mm/yyyy or select the required date range using the Calendar function. The Invoice is generated beginning with the specified date. The current system date is displayed, by default. This is a mandatory field.

  3. Bill Type/Num [From – To] – Select the required Bill Type and Bill Number from [List of Values]. The entries with Bill Type and Bill Number get populated when the data is posted from the Patient Billing Manager module. If ‘From’ value is not entered, the system assumes the lowest value for the ‘FROM’ value.

 

Credit Notes

  1. Doc Type Code – Enter the Document Type Code in alphanumeric format of six characters maximum length or select the required Document Type Code from [List of Values]. The Document Type Code set for Credit Notes in AR Parameter is displayed, by default. This is a mandatory field.

  2. Start Doc No – On entering the Document Type Code, the Starting Document Number set for the selected Document Type in Financials Base module is displayed, by default.  

  3. Doc Date – Enter the Document Date in the format of mm/dd/yyyy or click the Calendar function to set a date on which the document is created. The current system date is displayed, by default. This is a mandatory field.

  4. Hospital File Reference – Enter the Hospital File Reference in alphanumeric format of 20 characters maximum length. The entered File Reference Number is unique and it identifies the files maintained manually in which the related data of customers specified while generating an Invoice is stored.

  5. Post Month/Year – The month and year set in AR Parameter is displayed as Current Accounting Year, by default. You can modify the data provided the Post Month and Year must not be less than Current Accounting Year. This is a mandatory field.

  6. Facility ID – Select the required Facility ID from the list. The Facilities defined in the ‘Facility’ function and associated to the ‘Responsibility’ of the user in Systems Manager module are displayed. The value ‘All Facilities’ is displayed, by default.

  7. Debit Control A/C – Select the check box, if the Invoice is to be generated for the amount, which is debited from the Bill Control Account. If the check box is cleared, the Invoice is generated for the amount debited in the Customer Control Account.
