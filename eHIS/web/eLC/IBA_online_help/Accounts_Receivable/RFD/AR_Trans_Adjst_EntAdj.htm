Enter the following details to generate an Adjustment Document:

 

  1. Doc Type/No – Click [List of Values] to select the required Document Type Code and Number. The Document Type Code set for Credit Notes in AR Parameter is displayed, by default. If the specification for the document type states that the document number is system generated, the document number is generated when the transaction is saved. When the system generates the document number, field is protected from entering the number. This is a mandatory field.

  2. Date – Enter the date in the format of dd/mm/yyyy or select the required date using the Calendar function. The system date is displayed as Invoice Date, by default. Document date has to be less than or equal to the system date. This is a mandatory field.

  3. Document Nature – Click [List of Values] to select the appropriate Document Nature. The values listed are Normal, Move to Bad Debit and Remove from Bad Debit. Select Normal to adjust the amount. Select Move to Bad Debit to transfer the amounts to bad debts maintained in the GL interface, which cannot be collected from the Customer due to certain valid reasons. Select Remove from Bad Debit to delete the bad debts recorded in the GL interface for the respective Customer, if the amount is settled by the Customer.

  4. Doc Amount – Enter the Document Amount in numeric format of nine digits maximum length. This is a mandatory field.

  5. Month/Year – Enter the Month/Year in numeric format of two and four digits each maximum length respectively.  The current processing month value mentioned in AR parameters is displayed, by default, but the value can be updated. This is a mandatory field.

  6. Balance Amt - The total balance amount the Customer owes to the facility gets displayed.

 

Adjustment Note Details

  1. Customer -  Click [List of Values] to select a valid Customer Code. The entries with code and its short description get populated. If the Customer is of miscellaneous type, a screen pops-up on clicking [Name_Addr]. Enter the Customer details in the mandatory fields.

  2.  Orig Doc Type/No - Enter the Document Type in alphanumeric format of six characters maximum length or click [List of Values] to select the required Document Type. If Original document & Number to be allocated is a consolidated Invoice document, bills need to be allocated from pop-up ‘Bill wise allocation (Consolidated Invoice)’. This is a mandatory field.

  3. Adj Type –  Click [List of Values] to select a valid Adjustment Type. The values listed are ‘Debit’, ‘Credit’ and ‘None’. Select Credit, if the amount being allocated is crediting the customer account. This is accepted since the amounts are all accepted as positive amount only irrespective of whether the adjustment type is debit or credit. Select Debit, if the Customer owes an additional amount to the facility. Select Normal, if the amount is settled.

  4. Reference – Enter the relevant Reference in alphanumeric format of 15 characters maximum length. The default value is populated by the system if it is of allocated entry.  

  5. Payment Terms - On selecting the Customer, payment terms details set for the selected Customer in Customer Master is displayed. This can be changed. You can also click [List of Values] to select the Payment Terms. This is a mandatory field.

  6. Base Date – Click the Calendar function to enter a valid Base date. Base date is used to calculate the due date and the settlement date. After calculation, both are displayed. This cannot be changed if allocated entry. This is a mandatory field.

  7. Due Date – On selecting the Base Date, the Due Date is calculated automatically based on the value for Due Date entered in “Payment Terms” screen, by default. You can set another date as Due Date.  

  8. Settlement Date – On selecting the Base Date, the Settlement Date is calculated automatically based on the value for Settlement Date entered in “Payment Terms” screen, by default.

  9. Settlement Percentage – On selecting the Settlement Date, the Settlement Percentage is displayed, by default. You can enter another Settlement Percentage in numeric format of three digits maximum length. It is mandatory to enter the Settlement Percentage if Settlement Date is entered.

  10. Gross Amount – Enter the Gross Amount in numeric format of nine digits maximum length. This is a mandatory field.

  11. Disc Amt – Enter the Discount Amount in numeric format of nine digits maximum length. The Payment Amount is calculated based on the gross amount and the discount amount specified.

  12. Markup Amt – This is a display field. The value for Markup Amount is derived from the Markup percentage entered in ‘Payment Terms’ screen. The Markup value is calculated on the amount derived from Gross Amount minus Discount Amount and displayed as a read-only text.

  13. Pay Amt - This is a display field. The value for Payment Amount is derived from Gross Amount less of Discount amount and addition of markup amount, that is,

Payment Amount = Gross Amount – Discount Amount + Markup Amount.

Adjustment Line

  1. Posting Account –  Click [List of Values] to select a Posting Account Code. This is a mandatory field.

  2. Amount – Enter the Amount in numeric format of nine digits maximum length. This is a mandatory field.

  3. Debit – This is a display field. When the value for amount entered is greater than zero, it is displayed as Debit.

  4. Credit – This is a display field. When the value for amount entered is less than zero, it is displayed as Credit.

  5. Narration - Enter the Narration in alphanumeric format of 50 characters maximum length. The description for invoice line can be entered as narration.

  6. [Patient Bills] - Click [Patient Bills] to view the Billing Details which are generated in the Patient Billing Manager module for the services.

You are not allowed to modify or delete any values for the Adjustments generated from external modules.
