Enter the following details to create a Customer:

 

  1. Customer Code - Enter the Customer Code in alphanumeric format of six characters maximum length. The Customer Code is unique and cannot be changed after creation. The length of the Customer Code has to match with the Customer Code Length specified in the AR parameter. This is a mandatory field.

  2. Alpha Code - Enter the Alpha Code in alphanumeric format of 10 characters maximum length. The Alpha Code can be the same as that of other Customer Code or it can be unique. This is a mandatory field.

  3. Long Name - Enter the long name of the Customer in alphanumeric format of 250 characters maximum length. This is a mandatory field.

  4. Short Name - Enter the short name of the Customer in alphanumeric format of 15 characters maximum length. This is a mandatory field.

  5. GSTIN - Enter the GST Identification Number (GSTIN) in alphanumeric format of 20 characters maximum length. GSTIN is unique for each customer.

  6. Explanatory Details - Enter the explanatory details in alphanumeric format. You can record additional details related to the Customer.

  7. Country Code – Click [List of Values] to select the Country Code. The values defined using the function ‘Country’ in the Master Patient Index module get listed.

  8. State - Click  [List of Values] to select a State. The entries with code and its description get populated only when the Regions are defined in the Master Patient Index Module.  

  9. Address [Line 1 – 5]  – Enter the address in alphanumeric format of 60 characters maximum length for each line. The first line is mandatory.

  10. Contact Name & Title [Line 1 –2] – Enter the Contact Name of the person and the Title in alphanumeric format of 40 characters maximum length for each line. This is a mandatory field.

  11. Fax No. - Enter the Fax Number in alphanumeric format of 30 characters maximum length.  

  12. Telephone No. – Enter the Telephone Number in alphanumeric format of 30 characters maximum length.  

  13. Valid From – Click the Calendar function to select a date. The date indicates from when the Customer is valid in the facility. The ‘Valid from’ field is visible only if the ‘Validity Period’ is set in AR Parameter. The date cannot be lesser than the backdate validity set in the AR Parameters. The back date validity is calculated from the current System Date.

  14. Valid To - Click the Calendar function to select a date. The date indicates until when the Customer is valid in the facility. The ‘Valid To’ field is visible only if the Validity Period is set in AR Parameter.

  15. Service Price in Currency – Click [List of Values] to associate a Foreign Currency to the Customer. The Currency associated to the respective Customer gets displayed in the ‘Price Class – Customer Definition’ tab in the ‘Price Class Specific Prices’ function.

  16. Patient Related – Select the check box, if the Customer is a patient. Clear the check box, if the Customer is miscellaneous (temporary) in nature.

  17. Credit Patients – Select the check box, if the Customer is a credit patient.

  18. Misc Customer – Select the check box, if the Customer is miscellaneous (temporary) in nature. Clear the check box, if the Customer is patient related.

  19. Loc Lang Reporting Reqd – Select the check box, if the Local Language Reporting is required. If the check box is selected, the ‘Local Address’ tab gets enabled accepting the name and address in local language.

  20. VAT No. - Enter the VAT number in the length of 20 alphanumeric characters.
