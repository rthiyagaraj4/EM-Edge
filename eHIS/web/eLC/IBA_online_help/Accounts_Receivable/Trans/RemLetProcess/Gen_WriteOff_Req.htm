Generate Write-Off Request

From this function you can Generate a write-off request for the documents where the outstanding bill amount is not settled.

This function is used to generate a Write-Off Request for a particular Customer based on the Outstanding Amount and the Period for a particular Document Type. The Write-Off Request can be based on the Nationality of the Customer also. Only certain users are authorized to generate Write-Off request. The reason for the Write-Off is to be mentioned while generating the report. Once the Write-Off Request is generated it has to be approved by an authorized user. You can generate any number of Write-Off Request before the authorized user approves the same.

Generating a Write-Off Request

From the Transactions > Reminder Letter Process, double-click Generate Write-off Request function.

The Generate Write-Off Request screen gets displayed.

  1. Enter the details in the mandatory (*) fields. <Related Field Description>

  2. Click [Authorize] to login and generate a write-off request. Authorized Users are only permitted to login to process the Write-off. User is permitted to process the Write-off in the Patient Billing Manager module using the function User Privileges and Cash Counter Access.

    The authorized users only can generate a Write-Off. For security reasons, the User ID, Pin Number and a valid reason has to be entered. User ID and Pin Number is confidential only to the valid users. All the fields are mandatory. Click [OK] to accept and proceed or [Cancel] to cancel the entries.

    On clicking [OK], related records like Reason Code, Original Document Number and Date, Outstanding Amount, Dispatched Date, Billing Group and Patient Number associated with the Customer for a specified Outstanding Period and Amounts range are populated for view and to generate Write-Off.

  3. The outstanding document details get displayed. <Related Field Description>

  4. Click [Process] to generate the write-off request for the selected outstanding records. Click [Yes] to confirm or [No] to cancel the Write-Off.

  5. A document reference number is generated for the transaction. Click [OK] to accept the processed records.
