Write-Off Approval/Rejection

From this function you can Approve or Reject write-off request placed by an authorized user for a Customer.

This function is used to generate a Write-Off Request for a particular Customer based on the Outstanding Amount and the Period for a particular Document Type. The Write-Off Request can be based on the Nationality of the Customer also. Only certain users are authorized to generate Write-Off request. The reason for the Write-Off is to be mentioned while generating the report. Once the Write-Off Request is generated it has to be approved by an authorized user. The user can generate any number of Write-Off Request before the authorized user approves the same.

Write-Off Request

From the Transactions > Reminder Letter Process, double-click Write-off Approval/Rejection function.

The Write-Off Approval /Rejection screen gets displayed.

  1. Click [List of Values] to select a valid Customer. On selecting a Customer code, the short description of the Customer is displayed. This is a mandatory field.

  2. Select an Outstanding Amount range. The values defined using the Reminder Letter Setup function and associated to the selected Customer get listed. This is a mandatory field.

  3. The system date is displayed as the Write-off Request Date, by default. You can update the date. The outstanding document details with the write-off approval for the selected date range get listed.

  4. Click [Authorize] to login and either to cancel or to approve the write-off request. A screen appears requesting the details to login and to approve or reject the write-off request. Click [OK] to login and to authorize the request or click [Cancel] to exit from the screen.

    Enter the following details to login:

    1. User ID – Enter the User ID in alphanumeric format. Users who are authorized to approve the write-off process are only permitted to login and reject or approve the request. This is a mandatory field.

    2. Pin No – Enter the PIN number. The PIN Number recorded for the specific User during definition in the System Manager module has to be entered. This is a mandatory field.

    3. Approval Reason – Click [List of Values] to select an Approval Reason. The reasons defined using the function Reason for Billing Actions get listed. This is a mandatory field.

    4. Rejection Reason – Click [List of Values] to select a reason for Rejection. The reasons defined using the function Reason for Billing Actions get listed. This is a mandatory field.

  5. The document details for which the write-off request has been placed using the function Generate Write-off Request function get listed. The details are displayed under the following tabs:

Write-off Approval

The related records like Invoice Document Number and Date, Serial Number and Request Amount associated with the Customer for the specified Outstanding Period and Amounts are populated for view and user can approve or reject the write-off requests.

  1. Write-off Approval Date – The system date is displayed as Write-off Approval Date, by default. Click the Calendar function to select a date.

  2. Sel – Select the check boxes to select the required documents for approval.

  3. Apprd – On selecting the “Select” check box, the associated “Approved” check box is selected automatically. The selected document is approved to be written-off.

On selecting approved for write-off, related records like Write-off Approved or Rejected Reason specified on Authorisation, Date and Amount associated with the Customer for the specified Outstanding Period and Amounts range are also populated.

  1. Write-off Aprr/Rejn Reason – On selecting the check box for approval the appropriate reason selected for Approval of the Write-off Document is displayed. This is a mandatory field.

  2. Approval/Rejection Remarks – Enter the Remarks for the Approval or Rejection in alphanumeric format of 200 characters maximum length. This is a mandatory field.

  3. Write-off Doc No – This is a display field. On selecting a particular record, related Document Type and Number is displayed.

  4. Billing Group – This is a display field. On selecting a particular record, related code along with its description for the Billing Group is displayed.

  5. Patient No – This is a display field. On selecting a particular record, related Patient ID along with the Patient Name is displayed.

  6. Episode – This is a display field. The type of patient along with the Visit Number is displayed. The data is populated from the AR Pending Documents.

Write-Off Rejection

Click Write-Off Rejection tab to cancel the Write-Off request. Click [Authorize] to cancel the Write-Off request.

  1. [Authorize] – The authorized users only can approve or reject the Write-Off requests. For security reasons, the User ID, Pin Number and the Rejection Reason have to be entered. User ID and Pin Number are confidential only to the valid users. All the fields are mandatory. Click [OK] to accept and proceed or click [Cancel] to cancel the entries.

After entering the fields and clicking [OK], the related records like Invoice Document Number and Date, Serial Number and Amount associated with the Customer for the specified Outstanding Amount range are populated for view and to cancel the Write-Off requests.

  1. Write-off Approval Date – The system date is displayed as Write-off Request Date, by default. Click the Calendar function to set a date.

  2. Cancel – Select the check box to cancel the Write-Off request for the selected Invoice Document.

On selecting cancel, related records like Reason associated with the Customer for a specified Outstanding Period and Amounts range are populated for view. You can select a relevant record to cancel the Write-Off request for the relevant record.

  1. Reason – On selecting the check box, the appropriate reason selected for Cancellation at the time of authorization is displayed. This is a mandatory field.

  2. Remarks – Enter the Remarks for cancelling a relevant document in alphanumeric format of 200 characters maximum length. This is a mandatory field.

  3. Billing Group – On selecting a particular record, related code along with its description for the Billing Group is displayed.

  4. Org Doc No – On selecting a particular record, related Original Document Number is displayed.

  5. Episode No – On selecting a particular record, the type of patient alongwith the Visit Number is displayed. The data is populated from the AR Pending Documents.

  6. Patient No – On selecting a particular record, related Patient ID alongwith the Patient Name is displayed.

  1. Select the records and click [Commit Form] or [Process] to approve or to cancel the write-off request.

  2. Click [Clear Record] to clear the entered data.

  3. Press [F8] or click [Execute Query] to view all the write-off transactions. Press upward arrow key or downward arrow key to navigate to the previous or the next records.

  4. Click [Execute Query] or press [F8] to query existing records.
