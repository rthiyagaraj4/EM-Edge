Process Outstanding Documents

From this function you can Generate a report to view the outstanding documents and the associated transactions.

This function is used to process the outstanding documents and the associated transactions for a range of Customers selected. This function has to be executed only during the initial setup phase of operations. Error free outstanding documents only are processed. If any errors are encountered, the process is executed again after rectifying the errors. The outstanding balance amount is updated on the Customer master. These transactions are transferred to the customer transaction history for future reference and use. The outstanding documents and the associated transactions are deleted once they are successfully processed. Erroneous transactions are printed from the same function using the "Report Error Transactions" option in Process/Report input parameter.

Processing a report for outstanding documents

From the Op - Balance, double-click Process Outstanding Documents function.

The Process Outstanding Documents screen gets displayed.

  1. Click [List of Values] to select a valid Customer Code. Both the values are optional. If ‘From’ value is not entered, the system assumes the lowest value for the ‘From’ and if ‘To’ value is not entered, the system assumes the highest value for the ‘To’ value. When both values are entered, ‘To’ value has to be greater than or equal to the ‘From’ value.

  2. Click on ‘Post Opening Transactions’ to process the outstanding documents. Click on ‘Report Error Transactions’ to print the list of errors encountered during posting.

  3. Click [Run Process] to either print, view or save the document in the local machine.

  4. After selecting the required preference, click [OK] to process or [Cancel] to re-enter the selection criteria for the report.
