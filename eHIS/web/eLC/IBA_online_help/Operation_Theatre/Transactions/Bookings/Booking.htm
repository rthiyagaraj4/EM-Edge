Booking

From this function you can View availability of schedules and Perform bookings for preferred dates. Booking function allows querying the earliest available schedule and supports over booking for fully booked operating rooms.

Booking is an option provided for a planned operation/procedure. The Booking function is used to reserve time slots from the available time slots for performing an operation on a specified date. A booking reference number is generated after the booking is confirmed.

Operations that were ordered from the Clinician Access module and not scheduled is displayed in the Pending Orders tab. You can book or waitlist these orders.

Operations that were Waitlisted form the Clinician Access module is displayed in the in the Waitlist tab. If an earlier booking is cancelled or transferred, the corresponding slots are free and the waitlisted operations are prompted for booking. You can cancel the waitlist and the orders are displayed in the Pending Orders tab.

Creating Booking

From the Bookings, select the Booking function.

The Booking screen gets displayed.

The screen displays 3 tabs:

Schedule

Pending Orders

Waitlist

The time slots for each day depends on the 'Unit Time for Booking' as set in the 'Setup Parameters for Facility' function. The system  displays a warning message about the existing scheduled operation when a patient already has an existing scheduled and/or unscheduled operation for a particular day and you try to check-in or book the patient for the same day. The warning message is ‘Patient has existing operations for this day, Do you want to continue?’
