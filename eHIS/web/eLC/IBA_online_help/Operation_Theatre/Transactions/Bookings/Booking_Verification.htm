Booking Verification

From this function you can Verify the booked cases and View or Modify details of the Operation.   

Booking verification has two tabs - Booked Cases and Emergency Cases. The 'Booked Cases' tab shows all the booked cases and the 'Emergency Cases' tab shows all the emergency cases. You can verify the Booked Cases at various levels based on the number of stages defined in the booking finalization stages function. Priority can be set for booked cases and they can be re-sequenced based on urgency. 'Emergency Cases' shows the unscheduled emergency waitlisted cases. Assigning the theatre and pre-Anaesthesia evaluation can be done for these Waitlisted cases. If the theatre has already been assigned, you can view the theatre details for these cases.

Verification of a Booking

From the Bookings, select the Booking Verification function.

The Booking Verification screen gets displayed.

The screen displays 2 tabs:

Booked Cases

Emergency Cases

After marking an operation for Reschedule in the Booking Verification function, the operation becomes disabled and the verification cannot be done for that operation. To reschedule the operation, go to 'To Be Rescheduled Cases' function.

  • If the future order exist for the selected patient, the Future Order screen is displayed alerting the user to link the previously placed order with future date.

  • If the future order is linked to the current encounter, a message 'This Booking is already linked to Current Encounter' is displayed to avoid repeated linking.
