Setup Parameter Tab

  1. Unit Time for Booking (in Hrs:Min) - Enter the minimum time for identifying the Booking slots in Operation Theatre. The booked time for an operation is a multiple of this Unit of Time. The working hours of the Operating Room is divided into a number of equal sized time slabs each equal to the Unit Time.

  2. Enable Multi Facility OT Booking - Select this checkbox to enable multi facility OT booking from OT and CA modules. By default this checkbox is not selected.

Booking Time

  1. Multi Facility – Select this check box to allow the user to perform transactions across facilities.

  2. Working Days Start - Enter the Start Time and End Time of a working day for creating an Operating Room. Based on the details entered, OT Booking screen displays the time slots for Working days.

  3. Holiday Start - Enter the Start Time and the End Time of a holiday for creating an Operating Room. Based on the details entered, OT Booking screen displays the time slots for holiday.

  4. Restrict Theatre Booking - Select the check box if the system has to check if the logged in user has rights to book an operating theatre. The system restricts the logged in user to proceed with the booking if the booking rights are not set for that user.

OT Notification (Within a Day)

  1. Section From-To Time - Enter the From-To time for Section1, Section2 and Section3. These timings are defaulted in the Notification List by OR function for listing operations booked between these timings. The timings are maintained in Date Time format to validate day crossovers.

Booking Numbering

  1. Next No. - Enter the next Booking Serial Number to be assigned by the system. The Booking Serial Number is a system-generated number common across all Operation Theatres. At the start of operations, the next serial number specified is the first Booking Serial Number generated by the system. This number is incremented by 1 automatically by the system, for every booking. The number must be greater than zero. If the check box to prefix the date is selected in Setup Parameter function, then the Booking Number is a combination of the date of Booking and the Next OT Booking Serial Number. Also, in this case, the system automatically sets the Booking Serial Number to 1 as the first booking made at the start of a new year.

  2. Max No. - Enter the maximum serial number for booking. After the system reaches the maximum serial number, no further Booking Numbers are generated. At this stage, the System Administrator should reset the Next number appropriately. This number cannot be less than the 'Next Number' entered.

Waitlist Numbering

  1. Next No. - Enter the next Waitlist Number to be assigned by the system. This number is incremented by one automatically by the system for every Waitlist.

  2. Max No. - Enter the maximum waitlist number for waitlisting. After the system has reaches the maximum waitlist number, no further Waitlist numbers are generated. At this stage, the System Administrator should reset the maximum number. This number cannot be less than the Waitlist Next Number.

Operation Numbering IP/DC

  1. Next No - Enter the Next Operation Serial Number that the system generates for Inpatients or Day Care patients. For operations, the next serial number specified is the first Operation Serial Number generated by the system. This number is incremented by one automatically by the system, for every Operation registered. The number must be greater than zero. If the check box to prefix the date is selected in Setup Parameter function, the Operation Number is a combination of the booking date and the Next Operation Serial Number. Also, in this case, the system automatically sets the Operation Serial Number to 1 as the first registration made at the start of a new year.

  2. Max No - Enter the maximum operation number for inpatients or daycare patients.  When the maximum serial number is reached, no further Operation Numbers are generated. At this stage the System Administrator should reset the Next number appropriately. This number cannot be less than the Next Number.

Operation numbering OP/EM

  1. Next No - Enter the Next Operation Serial Number that the system generates for Outpatients or Emergency Patients. For operations, the next serial number specified is the first Operation Serial Number generated by the system. This number is incremented by one automatically by the system, for every Operation registered. The number must be greater than zero. If the check box to prefix the date is selected in Setup Parameter function, the Operation Number is a combination of the booking date and the Next Operation Serial Number. Also, in this case, the system automatically sets the Operation Serial Number to 1 as the first registration made at the start of a new year.

  2. Max No - Enter the maximum operation serial number for outpatients or emergency patients.  When the maximum serial number is reached, no further Operation Numbers are generated. At this stage the System Administrator should reset the Next number appropriately. This number cannot be less than the Next Number.

Turnaround Time

  1. Turnaround time – Select this check box to enable the Start OT Status and End OT status drop-down lists.

  2. Start OT Status – Select the Start OT Status from the drop-down list. The listed values are ‘Call’, ‘Check-in to OR’, ‘Check-Out from OR’ and ‘Depart’. This is a mandatory field.

  3. End OT status - Select the End OT status from the drop-down list. The listed values are ‘Call’, ‘Check-in to OR’, ‘Check-Out from OR’ and ‘Depart’. This is a mandatory field.

If the value selected in  the Start OT Status and End OT Status are not sequential, the system invokes an error message ‘Please configure the status in the right order’.

 

Other Details Tab

 Booking/Waitlist

  1. Waitlist warning limit Days - Enter the number of days after which the waitlist should be shown in red in the Booking screen Waitlist tab. This indicates that the schedule is waitlisted for more than the acceptable number of days. All the waitlisted operations appear in the 'Waitlist' tab in the Booking function.

  2. Warn for Unscheduled cases only - Select the check box to indicate that the waitlist warning is for Unscheduled cases only.

  3. Initiate IP Booking - Select the check box to indicate that the IP Booking function should be initiated after completing the OT Booking.

  4. Create Diary Schedule By? - Select Surgeon or Theatre to create the diary schedule.

  5. Booking Cut-off Time for Elective Cases - Enter cut-off time for booking elective cases. You cannot book elective cases beyond the time specified in this field.

  6. Allow merge of pending orders into existing booking – Select this checkbox to allow you to choose one or more pending orders and perform a new booking or amend a booking to merge the operations. By default this checkbox is not selected.

  7. Timelimit for existing booking selection (In days) –Enter the maximum number of days in future within which existing bookings for a patient will be automatically retrieved by the system. The default value is ‘365’. You can enter values ranging from 1 to 365. This text box is enabled only if the parameter ‘Allow merge of pending orders into existing booking' is checked, otherwise this text box will be disabled and greyed.

  8. Include Blocked Slot in Available Slot Count – Select this checkbox to allow configuring whether a blocked slot will be included or excluded in the available slot count. By default this check box is not selected

Booking Verification

  1. Allow Outpatients while finalizing? - Select the action from the list to be taken when an Outpatient booking is being finalized. The listed values are Stop, Warn and Allow. If 'Allow' is selected, booking is allowed to be finalized without any alert. If 'Warn' is selected, a warning alert is given while finalizing the booking and if 'Stop' is selected, the booking cannot be finalized.

  2. Allow Emergency Patients while finalizing? - Select the action from the list to be taken when a booking of an Emergency Patient is being finalized. The listed values are Stop, Warn and Allow. If 'Allow' is selected, booking is allowed to be finalized without any alert. If 'Warn' is selected, a warning alert is given while finalizing the booking and if 'Stop' is selected, the booking cannot be finalized.

  3. PAC Review Orderable - Select the order catalog using the [?] function based on which the Pre Anaesthesia orderable is generated. Order catalogs created with Order Category 'Miscellaneous' in the Order Entry and Result Reporting module are displayed for selection.

  4. No. of Swab Counts Required (Including Initial and Final Count) - Select the number of swab counts required during the operation. The number of swab counts can be 1, 2, 3, 4, 5 or 6. The number of swab counts selected is displayed in the OT Slate function.

When an order for an operation is placed, two orders are generated. One is the Operation order and the other is an Anaesthesia order. While placing an order in the Clinician Access module, you have to select the 'Anaesthesia Required' check box to include the Anaesthesia order.

  1. No. of Swab Counts Required (Mandatory) - Select the number of swab counts required during the operation. This is a mandatory field. The number of swab counts can be 1, 2, 3 or ALL. The number of swab counts selected is displayed in the OT Slate function.

  2. Reason for Re Arrange Sequence (Mandatory) - Select the check box to mandatorily enter the 'Reason' for changing the sequence of the patient in Booking Verification function and clear the check box to change the sequence of the patient without entering the reason.

Documentation Completion Status Check

  1. Surgeon Doc - Select the check box to indicate that the surgeon notes have to be mandatorily entered in the OT Slate function.

  2. Nursing Doc - Select the check box to indicate that the nursing notes have to be mandatorily entered in the OT Slate function.

  3. Intra Anaesthesia Doc - Select the check box to indicate that the intra Anaesthesia document has to be marked as complete in the Anaesthesia function.

Slate

  1. Show last Antibiotic Admin Time - Select the check box if the Last Antibiotic Administered Time has to be displayed in OT Slate.

  2. Slate Login Reqd? - Select the check box to enter the login ID and password to record details in the OT Slate function.

  3. Include Cases for Past [] Days - Enter the number of days allowed to include the past cases in numeric format of two digits maximum length.

  4. Check-In Backdate Limit [ ] Days - Enter the number of days in numeric values of three digits maximum length. Based on the 'Check-In Backdate Limit' value recorded, you can back date the surgery date in direct check-in function. The Surgery Date will display the current system date and it is disabled if 'Check-In Backdate Limit' value zero or left blank.

  5. Display Hospital Personnel Assigned During Booking/Check-In – Select this check box to display the surgeon, assistant surgeon and anaesthetist details in the Record Surgeon Notes and Record Nursing Notes screens during Booking Verification or Check-In.

  6. Default User Role - Select Default User Role in the Slate section under Other Details tab in the Setup Parameters For Facility screen to display a Roles for User function in the Setup menu.

  7. Link Operation and Notes in Record Surgeon Notes – Select this check box to link the operation and notes details in the Record Surgeon Notes. If you select this check box the error message ‘‘Cancellation not allowed as Personnel details exists against this operation’ gets displayed when you try to cancel the operation in Record Surgeon Notes screen.

  8. Multiple Check In Allowed – Select this check box to allow multiple check in OT slate. By default, this check box is selected.

 

If you clear this check box, the error message "Notes already exists against this operation. Are you sure you wish to proceed?" gets displayed if one or more Notes exist for the selected operation in the Record Surgeon Notes screen. By default, this check box is selected.

 

Interfaces

  1. External Link - Select this check box to display the icon in the OT Slate and Patient Banner invoked from the OT Slate screen.

  2. Interface to Patient Billing - Select the check box to interface the Theatre Management module to Patient Billing Manager module. Clear the check box to disable the Patient Billing Manager module association to the Operation Theatre module.

  3. Interface to Oral Health – Select the check box to interface Oral Health to Operation Theatre module. On selecting the check box, the text box ‘Oral Health OT Procedure’ is enabled for selection.

  4. Interface to inventory - Select the check box if interface to inventory is required.

  5. Sale Doc Type - Enter the sales document type at the enterprise level or select the sales document type using the [?] function. The values are populated from the 'Document Type' function of Inventory Management module.

  6. Request Doc Type - Enter the request document type or select the request document type using the [?] function. The values are populated from the 'Document Type' function of Inventory Management module.

  7. CSSD Items Requested By - Select the CSSD Items Requested By using [?] function. The list of stores created and defined in the  Sterile Stock module is displayed.

  8. Requested On - Select the Requested On using [?] function. The list of stores created and defined in the Inventory module is displayed.

  9. Request Type – By default, ‘Normal’ is displayed and disabled.

 

Invoke Dental Chart

  1. Oral Health OT Procedure – Select a value using [?] function. The selected procedure is selected as dental OT procedure.

 

Instruments/CSSD Path

  1. Instrument/CSSD Image Location – Enter the location of the Instrument/CSSD Images in the server in alphanumeric format.

 

Billing Setup Tab

The Billing Setup tab is enabled when the field 'Interface to Billing' is selected at the facility level.

 

Operation Services

  1. Billing Sub Services - Select the 'Billing Sub Services' check box to control the Billing for surgical accessories. The check box is selected, by default.

  2. Admin Function for Charging - Select the check box to enable the admin function to display the charges individually. If this check box is cleared, the admin function is not enabled. This check box is selected, by default.

 

Room Service Usage

  1. Bill Holding Area Usage? - Select the check box, to charge the usage of the holding area. If 'Bill Holding Area Usage' is selected, capturing 'End Time' in the function 'Holding Area' function is mandatory before performing 'Check-in to OR' stage. The check box is selected, by default.

  2. Bill Operation Room Usage? - Select the check box to charge the usage of the Operating Room. This check box is selected, by default. If 'Bill Operation Room Usage' is selected, 'Check-Out from OR' stage is mandatory before 'Checking in to Recovery Room'.

  3. Bill Recovery Room Usage? - Select the check box to charge the usage of the Recovery Room. This check box is selected by default. If 'Bill Recovery Room Usage' is selected, the 'Recovery Room' details must be captured mandatorily in 'Check-in to Recovery Room' function.

  4. Charges Applicable from Check-In Date Time - Select this check box to consider the Check-In Date Time when the Completed check box is selected and Record button is clicked in Administrative Function for All Charges screen. If this check box is cleared, the System Date/Time will be considered.

Bill Finalization Stages

  1. Holding Area - Select the billing finalization stage for holding area from the list. The listed values are 'During Checking-Out Individual Rooms' and 'Before Check-Out/Documentation Completion'. This field is enabled if 'Bill Holding Area Usage' check box is selected. Select 'During Checking-Out Individual Rooms' from the list to charge the holding area usage in 'Check In/Out From Holding area/Document' function in OT Slate. Select 'Before Check-out/Documentation Completion' from the list to charge the holding area usage in 'Administrative Function for All Charges' function in OT Slate.

  2. Operating Room - Select the billing finalization stage for Operating Room from the list. The listed values are 'During Checking-Out Individual Rooms' and 'Before Check-Out/Documentation Completion'. This field is enabled if 'Bill Operating Room Usage' check box is selected. Select 'During Checking-Out Individual Rooms' from the list to charge the Operating Room usage in 'Check In/Out From Holding area/Document' function in OT Slate. Select 'Before Check-out/Documentation Completion' from the list to charge the Operating Room usage in 'Administrative Function for All Charges' function in OT Slate.

  3. Recovery Room - Select the billing finalization stage for Recovery Room from the list. The listed values are 'During Checking-Out Individual Rooms' and 'Before Check-Out/Documentation Completion'. This field is enabled if 'Bill Recovery Room Usage' check box is selected. Select 'During Checking-Out Individual Rooms' from the list to charge the Recovery Room usage in 'Check In/Out From Holding area/Document' function in OT Slate. Select 'Before Check-out/Documentation Completion' from the list to charge the Recovery Room usage in 'Administrative Function for All Charges' function in OT Slate.

  4. Operations/Procedures – Select the values from the list to define the Operation/Procedure charges in the finalisation stage. The listed values are ‘Admin Function’ and ‘Record Surgeon Notes’.

 

Surgical Accessories

  1. Accessory Type - The Accessory Types are displayed.

  2. Surgeon Notes - Select the check box against the Accessory Type to enter the Surgeon Notes in the selected finalization stage. The Surgeon Notes check box is disabled for Accessory Type 'Equipment', 'Instruments/CSSD Packs' and 'Consumable Packs'.

  3. Nursing Notes - Select the check box against the Accessory Type to enter the Nursing Notes in the selected finalization stage.

  4. Admin - Select the check box against the Accessory Type to finalize the billing in the finalization  stage selected. The Admin check box is enabled on selecting the 'Admin Function for Charging' check box.

  5. Finalization Stage - Select the finalization stage from the list to enter the data for the selected surgical accessories. The listed values are 'Record Surgeon Notes', 'Record Nursing Notes' and 'Admin' Function.
