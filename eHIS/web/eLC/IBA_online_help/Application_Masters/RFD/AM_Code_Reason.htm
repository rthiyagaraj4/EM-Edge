Enter the following details to create a Reason record:

  1. Code - Enter the Reason code in alphanumeric format of eight characters maximum length. A Reason Code is used to identify the Reasons. The Reason Code has to be unique and cannot be changed after creation. This field is mandatory.

  2. Description - Enter the description of the Reason in alphanumeric format of 30 characters maximum length. This field is mandatory.

  3. Enabled - Select the check box to enable the Reason record. The check box is selected, by default. Clear the check box to disable the Reason record.

 

Applicable To:

 

Appointment Scheduling

  1. Book Appointment – Select the check box to enable the reason under Book Appointments function in Appointment Scheduling module. Clear the check box to disable the reason to get displayed in Booking Appointments function. The check box is cleared, by default.

  2. Transfer Appointment – Select the check box to enable the reason under Transfer Appointments function in Appointment Scheduling module. Clear the check box to disable the reason to get displayed in Transfer Appointments function. The check box is cleared, by default.

  6. Cancel Schedule - Select the check box to enable the reason while canceling the schedule in Appointment Scheduling module. Clear the check box to disable the reason to get displayed while canceling the schedule. The check box is cleared, by default.

  8. Cancel Wait List - Select the check box to enable the reason while canceling wait list the schedule in Appointment Scheduling module. Clear the check box to disable the reason to get displayed while canceling wait list schedule. The check box is cleared, by default.

  10. Override Appointment Rule – Select this check box to enable the reason while selecting a practitioner other than the practitioner defined by the system while applying Rule Based Appointment Schedule. The check box is cleared by default.

  11. Translator Override Reason – Select this check box to enable the reason under Book Appointments function when you try to proceed with booking after the alert is displayed that translators are unavailable. Clear this check box to disable the reason to get displayed in the Book Appointments function. The check box is cleared, by default.

Note: This check box will be enabled in this screen only when the “Schedule Rule Applicable” check box is checked in OA Parameter screen.
