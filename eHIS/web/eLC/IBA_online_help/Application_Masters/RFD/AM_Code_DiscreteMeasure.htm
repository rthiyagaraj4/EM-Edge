Enter the following details to create a Discrete Measure record:

 

  1. ID - Enter the Discrete Measure ID in alphanumeric format of 15 characters maximum length. An ID is used to identify the Discrete Measure and it has to be unique. You cannot change it after creation. This is a mandatory field.

  2. Long Description – Enter the long description of the Discrete Measure in alphanumeric format of 60 characters maximum length. This is a mandatory field.

  3. Short Description – Enter the short description of the Discrete Measure in alphanumeric format of 30 characters maximum length. This is a mandatory field.

  4. Component Type – Select the required Component Type from the list of valid values. The listed values are Check Box, Date, Date/Time, Decimal Numeric, Embedded Component, Formula, Grid Component, Integer Numeric, List, Long Text, Matrix Component, Short Text, Summary Component and Time. The Component Type created is attached to any Service that is a part of a Healthcare Location Type (for example, Nursing Unit, Clinic, etc.). This is a mandatory field.

    Numeric Map Digits

    1. Max – Enter the maximum allowable digits in numeric format of one digit maximum. The maximum allowable digits must be greater than or equal to the minimum allowable digits.

    2. Min - Enter the minimum allowable digits in numeric format of one digit maximum length.

    3. Decimal Numeric – Enter the number of decimal positions in numeric format of two digits maximum length. The digits specified must be lesser than or equal to the difference of [Max - Min] digits specified.

    The above values must be in between the range 1-9.

    1. Reference Ranges – Select the reference ranges from the list of valid values. The valid values are ‘Single’ and ‘Multiple’. This field displays ‘Single’, by default.

    If the selected ‘Reference Range’ is ‘Single’, enter the following details:

    1. Reference (Low - High) – Enter the low value and the high value of the discrete measure. For example, for Blood Pressure, the low value can be set to 80 and the high value can be set to 110.

    2. Critical (Low - High) – Enter the Critical low value and Critical high value for the Discrete Measure. The Critical Low and High values are as indicators when such situation arises, for example, for Blood Pressure, the Critical low value can be set as 70 and Critical high value can be set as 120.

    3. Units – Enter the Unit of measurement of the Discrete Measure in alphanumeric format of 10 characters maximum. For example, the Unit for Blood Pressure can be set as mm HG.

    4. Numeric Prefix Required - Select the check box to enter Numeric prefixes to discrete measure template.

    The system displays [Numeric Details] if the selected ‘Reference Range’ is ‘Multiple’. An example is to record the weight or height of patients of various age groups.

    Click the [Numeric Details] to display a Reference Ranges - Numeric Details screen.

    1. Sex – Select the Gender from the list of valid values. The valid values are ‘Male’, ‘Female’ and ‘Unknown’. This field displays ‘All’, by default.

    Start/End

    1. Age – Enter the age in numeric format of three digits maximum.

    2. Units – Select the Units from the list of valid values. The list of valid values include ‘Days’, ‘Months’ and ‘Years’. ‘Days’ is displayed, by default.

    3. Sequence No – Enter the sequence number in numeric format of two digits maximum.

    4. Reference (Low - High) – Enter the low value and the high value of the Discrete Measure.

    5. Critical (Low - High) – Enter the Critical low value and Critical high value for the Discrete Measure. The Critical low and high values are as indicators, when the recorded value is same or exceeds the entered Critical Low/High value.

    6. UOM – Enter the Unit of Measurement of the Discrete Measure in alphanumeric format of 10 characters maximum length. For example, the Unit of Measurement for weight can be set as ‘KGs’ or ‘Pounds’.

    7. Default – Enter the default value, which has to appear while recording the vital signs.

    On entering the required details, click [Select] and the system displays the entered values in the top half of the numeric details  screen. Click [Clear] to clear the values entered and on the press of [OK], the control goes back to the Discrete Measure create screen. Click [Cancel] to cancel the operation.

    On selecting the ‘Component Type’ as ‘Matrix Component’,  [Define Matrix] is displayed. Click [Define Matrix] and Matrix Definition screen gets displayed.

    1. Matrix Columns - Enter the Matrix Columns in numeric format of two digits maximum length. Maximum number of columns is 10. This is a mandatory field.

    2. Matrix Column Span – Enter the Matrix column Span in numeric format of two digits maximum length. Maximum value has to be nine. This is a mandatory field.

    3. Matrix Rows – Enter the Matrix Rows in numeric format of one digit maximum length. Maximum number of columns is 10. This is a mandatory field.

    On entering the details, click [Show Matrix].

    1. Row Header/Column Header – Enter the Column and Row header in alphanumeric format of 25 characters maximum length.

    2. [?] - Click [?] function to select the component for each Matrix field.

    3. [Show Preview] - Click [Show Preview] to view the Matrix.

    Click [OK] to Save the Matrix component defined. Click [Cancel] to exit from the screen.

The templates defined through the master can be associated to the Long Text using this function. The long text templates associated in this setup are available for the user during transactions.

    1. Associate Score – Select the check box to associate numerical score for each value that user defines in the list item. If the check box is selected, the Score field gets displayed on clicking [Alpha Info].

    2. Link Standard Value – Select the check box to link the standard value to the discrete measure.

    3. Standard Value Ref – Click [?] to select the pre-defined standard value.

    Click [Alpha Info] to display the Alpha Info screen.

    1. ID – Enter the ID for the list in alphanumeric format of four characters maximum length.

    2. Description – Enter the description in alphanumeric format of 100 characters maximum length. This field is mandatory.

    3. Default – Select the check box to display the entered value as default, while making the selection or clear the check box. You can define one list item as default.

    On entering the required details, click [Select] and the entered values are displayed in the top half of the Alpha Info screen.

    Component Definition

    1. Component – Select the Component from the [?] function. The Component Lookup screen is displayed on clicking the [?] function. Enter the following to select a component.

    2. Component ID – Enter the Component ID in alphanumeric format of 15 digits maximum length.

    3. Component Type – Select the Type of Component from the list of values.

    4. Component Prompt – Enter the component prompt in alphanumeric format of 15 digits maximum length.

    On entering the above details, click [Search] to view the list of components.

    Click the Component ID hyperlink to select the component. On selecting the component, it appears in the Component field under Component Definition.

    1. Column Span - Enter the column span in alphanumeric format of two characters maximum length. The maximum value for column span is 10.

    2. Text Rows – Enter the row span in alphanumeric format of one character maximum length. The value of the row span can be maximum two. This field is enabled only when you select the components of type ‘Short Text’.

    Click [Add To Grid] to add the component to the List of Grid Components.

    Changing the column position of the component:

    To change the position of any component in the grid, use the Column Position hyperlink.

    If you want to change the position of the component, click the hyperlink of the desired component’s Column Position.

    The system allows you to select the Column Position to which the desired component has to be moved.

    ‘+’ is a hyperlink. This is given to add a duplicate grid to the existing row. On clicking the hyperlink, the system adds a grid.

    Note that there is a ‘-’ mark next to the ‘+’ mark in the newly added row. Click the ‘-’ mark to remove the added row.

    Similar operations are done on a grid when it is used in the transactions.

    Modifying the component

    To change the grid component, click the Column Header hyperlink to view the details of the component in the respective fields under Component Definition.

    To modify the component, enter the details such as Component, Column Span or Text Rows and click [Modify].

    To remove the component from the grid, click [Delete from Grid].

Applicable Categories

  1. All - Select the check box to select all applicable categories. The check box is selected, by default.

  2. Recording Vital – Select the check box to enable the Discrete Measure while recording Vital Signs. The check box is selected, by default. Clear the check box to disable discrete measure while recording Vital Signs.

  3. Recording Chart - Select the check box to enable the Discrete Measure while recording charts in ‘Section for Chart’ function in Clinician Access Code Setup. The check box is selected, by default. Clear the check box to disable discrete measure while recording Charts.

  4. Result Reporting - Select the check box to enable Discrete Measure in ‘Order Catalog’ master function. The check box is selected, by default. Clear the check box to disable the discrete measure while Result Reporting.

  5. Notes Template - Select the check box to enable Discrete Measure while recording Notes in Clinician Access Code Setup. The check box is selected, by default. Clear the check box to disable the discrete measure while creating Notes Template.
