Reason

From this function you can Create various Reasons and Query for a required Reason record.

The Reason for performing various transactions needs to be compulsorily recorded while managing Outpatients, Inpatients, Emergency Patients and Scheduling Appointments. The transaction to which the reason is applicable has to be indicated. If the defined reason is common for some of the patient related transactions, the same is indicated using this function.

You can indicate the applicability of a reason to the transactions of the following Enterprise Management modules.

• Appointment Scheduling

• Outpatient Management

• Inpatient and Day Care Management

• Accident and Emergency

• Mortuary Management

 

Creating a Reason

From the AM Other Setup > Other Master, select the Reason function.

The Reason screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the details in the mandatory (*) fields. <Related Field Descriptions>

  3. Click [Apply] to save a Reason created or [Reset] to clear the entered data.

  4. Click [Query] to search for an existing Reason record.
