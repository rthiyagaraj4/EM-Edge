Interface Gateway Servers

From this function you can Define Interface Gateway Servers and Query for existing records. You can Delete a queried record.

This function is used for setting up communication clients (these are clients which are used for running the outbound and inbound communication processes) details.

Defining Interface Gateway Servers

From Gateway Configuration, select Interface Gateway Servers.

The Interface Gateway Servers screen gets displayed.

  1. Click [Create] to invoke the Create Screen.

  2. Enter the ID in alphanumeric format of 60 characters maximum length. This is a mandatory field.

  3. Enter a suitable name for the client computer. This is a mandatory field.

  4. Select the Outbound check box to enable the client as a voluntary interface client.

  5. Select the Inbound check box to enable the client to receive query messages from the interfaced application(s).

  6. Select the Enabled check box to enable the interface client.

The screen displays two tabs:

Outbound

A list of the interfaced applications (with facility) is displayed. Select the corresponding check box to specify the applications as ‘Outbound’.

Inbound

Inbound Settings

Communication Settings

  1. TCP/IP Address – Enter the IP Address of the client in valid format.

  2. Port – Enter the port number in which the EM Interface listener is going to listen for queries from the interfaced applications.

Port number has to be cross checked with the interfaced application’s requirement. Some applications send the query in predefined port number and in that case, only that number can be defined as port number.

A list of the interfaced applications (with facility) is displayed. Select the corresponding check box to specify the applications as ‘Inbound’.

  1. Click [Apply] to save the record.

  2. Click [Query] to query an existing record.

  3. Click [Delete] to delete a queried record.
