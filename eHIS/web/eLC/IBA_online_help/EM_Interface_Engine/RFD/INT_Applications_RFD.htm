Enter the following details to create Applications:

 

  1. ID - Enter the Application ID in alphanumeric format of 10 characters maximum length. Each Application has a unique ID. This is a mandatory field.

  2. Name – Enter the descriptive Application Name of the interfaced application in the alphanumeric format of 50 characters maximum length. The Name of the Application has to be verified with the interfaced application’s setup. This is a mandatory field.

The name has to be defined based on the requirement of interfaced application. This field is case sensitive.

  1. External Facility Name – Enter the descriptive Facility Name of the interfaced application name in alphanumeric format of 50 characters maximum length. The name of the Facility has to be verified with the interfaced application’s setup. This is a mandatory field.

  2. HL7 Version – The version of the HL7 standard supported by the application is displayed as 23.1. The acronym HL stands for Health Level This is a mandatory field.

  3. Field Separator - As per HL7 2.3.1 specification, the usual field separator in the message segment is ‘|’ character and the system displays this field separator, by default. In case the interfaced application accepts a different field separator character, the value of this item can be changed as per the requirement. Considering HL7 2.3.1 specification, CSC recommends the field separator character as ‘|’. This is a mandatory field.

  4. Enabled – Select the check box to enable this application. Only the enabled applications are available for further transactions.
