Register Attendance

From this function, you can Register the arrival of a patient to the Emergency Location and Record Primary Triage details for the patient.

A Patient Identification Number is essential to register the attendance of a patient using this function. Register Attendance function is an option to register the patient attendance along with the Patient Identification Number when a patient arrives at the Emergency Location without a valid Identification Number. On successful patient registration, the system generates a valid Encounter ID to identify the emergency visit of the patient.

 

Hospital and Professional Charges are displayed on recording the financial details of a patient. The patient payable amount towards Hospital charge as well as Doctor’s charge is displayed. Professional Charge is displayed along with the Hospital Charge, only if the billing details and visit registration rules are defined for the Clinic. The doctor’s charge has to be a part of the panel, if the Professional charge has to be set as zero. If the Professional Charge is set as a Service, you cannot set the doctor’s fee as zero. You can instead associate the Hospital Charges to make up the Professional fees, when the doctor does not charge the patient for the services.

Registering Attendance

From the AE Transactions, select the Register Attendance function.

The Register Attendance screen gets displayed.

This screen allows registering the attendance from three different options.

Details Unknown

Patient ID

Referral ID

  1. Click [Contact Details] to record the contact details of the patient.

You can set the Race, Country, Region, and Mobile number to be mandatory or non-mandatory as required, by selecting or clearing the respective check boxes in the Mandatory fields during Registration field of the MP Parameter screen.

  1. Click [Previous Encounters] to view the previous encounter details of the patient, if the selected patient had previously received care from the facility.

    Click [Previous Encounter] to view the details of previous encounters of the patient. The screen displays the Encounter Date and Time, Encounter Type, Encounter ID, Location Type, Location, Service, Sub service, Room No, Bed No, Resource, Disposition Date and Time, and Disposition Type recorded for the patient.

  2. Click [Last Visit Within 24 Hrs. Details] to view the details of the patient’s last visit, if the patient has made any visit recorded within the last 24 hours. This is disabled if the patient has not made any visits to the facility within 24 hours from current date and time.

    Click [Visit Within 24 Hours] tab to view the details of the patient's last visit, if it is within the last 24 hours. This tab is disabled if patient has not made any visits to the facility within 24 hours from current date and time. User is able to view the Encounter details like Location and Disposition details like Disposition Type.

  3. Click [Apply] to register attendance or click [Clear] to clear the entered data.

  4. The Bill Settlement Details screen gets displayed to record the charge details. (This screen does not appear for emergency registration of unknown patients)

  5. The Create Patient File screen gets displayed, if the file management parameters are set for the facility. Record the patient file details. <Related Field Description>

  6. Click [OK] to generate file number or click [Cancel] to cancel file generation.

  7. If the internal reports are defined, the Internal Reports screen gets displayed. Click [OK] to generate Internal Reports or click [Cancel] to exit from the screen.

  8. The Encounter ID and File Number is generated and displayed.

 

The Attendance cannot be registered for patients who have open encounters. On selecting such patients an alert is displayed.

An alert message is displayed listing the outstanding, unbilled or any installment due amount for patient's previous visits to the facility. This message is displayed only if the Credit Authorization parameters are setup using the 'BL Parameters' function in Patient Billing Manager module.
