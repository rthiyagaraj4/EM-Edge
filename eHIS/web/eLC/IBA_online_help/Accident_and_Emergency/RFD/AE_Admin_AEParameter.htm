Enter the following details to record AE Parameter:

 

Billing Interface

 

  1. Billing Interfaced – Select the check box, if the Patient Billing Manager module is interfaced. Once the check box is selected, the visit charges are automatically generated during registration and updated in the Patient Billing Manager module. Clear the check box, if the Patient Billing Manager is not interfaced.

  2. Allow AE checkout with bill settlement - Select the check box to settle the bill amount during check out. A settlement screen is displayed in 'AE Checkout' function on selecting the check box, prompting the user to settle bill amount.

 

Visit Controls

 

  1. Allow back Date/Time of Start Consultation – Select the check box to allow back dating of start consultation. Once the check box is selected, the text field next to it gets enabled. Enter the number of hours in numeric format. The hours must be less than 23 and greater than 0.

  2. Allow backdating of Visit Date/Time – If the check box is selected, this function allows you to backdate the visit Date/Time within the range specified in the text box. You can enter up to four digits. If the check box is cleared, you are not allowed to backdate the visit Date/Time.  This field is mandatory on selecting the check box.

  3. Max No. of days allowed for revising attendance details - Enter the number of days until when the details of a visit can be revised after the visit registration has taken place, in numeric format of four digits maximum length. The period is considered based on the date and not on the date/time. This field is mandatory. The default value is one and can be changed if required.

  4. Cut off period for including previous day’s visits – Specify the time period in hours to indicate the cut off period for including previous day’s visit. This field is mandatory.

  5. Alert Re-attendance of Patient in Register Attendance Within - Specify the time period in hours to indicate the attendance of patient. This field can accept value from 1 to 999. It can neither be blank nor allow zero. By default, it is set to 24 hours. This is a mandatory field.

  6. Record Diagnosis not Mandatory  

• BID – Select the check box to make recording of patient diagnosis as non-mandatory if the patient is brought dead to the facility.

• Absconded - Select the check box to make recording of patient diagnosis as non-mandatory if the patient is absconded.  

• DID - Select the check box to make recording of patient diagnosis as non-mandatory if the patient is deceased in department.

• Call Not Around - Select the check box to make recording of patient diagnosis as non-mandatory if the patient is not around in the AE department when called by the practitioner for consultation.

 

The ‘Call Not Around’ case indicates the patient who is registered using Register Attendance function and Secondary Triage is done. The patient is then waiting to be called for consultation. However, when the practitioner calls the patient is not around in AE department. Therefore, the practitioner can check out the ‘Call Not Around’ patient without recording the diagnosis.

 

  1. Display AE Patients with attendance registered within ‘X’ Hours – Enter the number of hours based on which the system displays the AE patients who stay more than 24 hours and not yet checked out in the Emergency Current Patients and AE Check Out screens. You can enter maximum of 99 hours. By default, 24 hours is displayed. This field is mandatory. If you enter less than 24 hours, an error message ‘The value must be between 24 and 99 hours’ gets displayed.

  2. Invoke Recall option for AE – Select the check box to invoke the Recall option for AE patients. By default, the check box is cleared and enabled. If you select the check box, the ‘Follow-up Encounters’ dialog and ‘Link/Delink Follow-up Encounter’ field appears in the Register Attendance screen for the patient having follow-up for that day.

 

Age Limits

 

  1. Min. Age of the patient for capturing pregnancy details – Enter the minimum age in numeric format of two digits maximum length to capture the pregnancy details of the female patient registered in an AE location. This field is mandatory. The default value is nine and can be changed, if required.

  2. Max. Age of the patient for capturing pediatric details - Enter the maximum age in numeric format of two digits maximum length to capture the pediatric details of the child patient registered in an Accident and Emergency Location. This field is mandatory.

  3. Min Age of the patient for capturing LMP details – Enter the minimum age in numeric format of two digits maximum length to capture the LMP (Last Menstrual Period) of the female patient while recording the Secondary Triage Details of the patient even when the patient is not pregnant. The minimum age of the patient has to be more than seven years. The minimum age of the patient is set to eight, by default. This is a mandatory field.

 

Order Catalog

 

  1. Sp02 – Select the required type of Sp02 using the [?] function. The order catalog gets populated from the Order Entry and Result Reporting module.

  2. ECG - Select the required type of ECG using the [?] function. The order catalog gets populated from the Order Entry and Result Reporting module.

  3. FBC - Select the required type of FBC using the [?] function. The order catalog gets populated from the Order Entry and Result Reporting module.

  4. Dressing - Select the required type of Dressing using the [?] function. The order catalog gets populated from the Order Entry and Result Reporting module.

  5. Capillary Blood Sugar - Select the required type of Capillary Blood Sugar using the [?] function. The order catalog gets populated from the Order Entry and Result Reporting module.

 

Record Vital Signs

 

  1. Systolic BP – Select the Systolic BP using the [?] function. The values get populated from the Application Masters module.

  2. Respiratory Rate – Select the Respiratory Rate using the [?] function. The values get populated from the Application Masters module.

 

Queue Management

 

  1. Action when unsettled/unbilled amount for the encounter – Defines the action to be taken while assigning or reassigning treatment area for a patient, if the patient has any unsettled or unbilled amount in that encounter. You can select one of the following three options from the drop-down list:

 

• No action – The system will not display any warning or error messages even if the the patient has any unsettled or unbilled amount in that encounter and you can proceed with the Assign or Reassign Treatment area activities for the patient

• Warning – The system will display a warning message “Unsettled/Unbilled amount exists for the encounter. Do you want to continue?” when assigning or reassigning treatment area for the patient. You can click OK and still proceed with the activities.

• Error – The system will display an error message “Unsettled/Unbilled amount exists for the encounter. Cannot proceed.” You cannot assign or reassign treatment area for the patient.

These messages will be displayed when you try to select the Assign Treatment area or Reassign Treatment area options in the Current Emergency Patients  screen.

 

  1. Refresh Interval - Enter the refreshing interval in minutes for the queue management screen view in numeric format of four digits maximum length. Based on the time interval set in the AE Parameter, the system automatically refreshes the Current Emergency Patients screen. This field is mandatory.

  2. Invoke Vital Signs from Queue – The check box is selected, when in current emergency patient queue, the record vital sign option is displayed. If the check box is cleared, Assign Treatment Area option alone is displayed.

  3. Enable Access Rights for Emergency Queue Management - Select the check box to enable the user to access the 'Access Rights for Queue Management' function. Clear the check box to disable the user from accessing the 'Access Rights for Queue Management' function.

  4. Observation Ward Type - Select an Observation Ward Type from the list. The system-defined values are 'Clinic', 'Nursing Unit' and 'Treatment Area'. The Observation Ward Type defined is defaulted in the transactions.

  5. Observation Ward - Select the Observation Ward using [?] function. This field becomes mandatory, if the 'Observation Ward Type' is selected. The values displayed are based on the selected Observation Ward Type.

  6. If Observation Ward Type is selected as 'Clinic', clinics created in 'Clinic' function with 'Level of Care' selected as 'Emergency' in Outpatient Management module gets listed. The selected Clinic is displayed as the Observation Ward Tab in 'Current Emergency Patients' function.

  7. If Observation Ward Type is selected as 'Nursing Unit', the Nursing Units created in 'Nursing Unit' function with 'Level of Care' associated to 'Emergency' indicator in Inpatient and Daycare Management module is listed. The selected Nursing Unit gets displayed as the Observation Ward Tab in 'Current Emergency Patients' function.

  8. If Observation Ward Type is selected as 'Treatment Area', Treatment Areas created in 'Treatment Area for location' function is listed. The selected Treatment Area gets displayed as the Observation Ward Tab in 'Current Emergency Patients' function.

  1. Levels of Triage Applicable - Select this check box to enable five levels triage process in the Accident and Emergency module. By default, the check box is cleared.

  2. Invoke Register Newborn for AE – Select the check box to enable the Register Newborn for AE context sensitive link in Current Emergency Patients screen. By default, the check box is not selected.

 

Disaster Management

 

  1. Location - Select a Location from the list. Locations created in 'Clinic' function with the 'Applicable Visit Type Indicators' defined as 'Emergency' in Outpatient Management module are listed. The selected Location gets displayed and defaulted in 'Disaster Patient Registration' function.

  2. Patient Series – Select the required Patient Series from the drop-down list. The list includes all the Patient Series Types that are applicable to the Accident and Emergency encounter type. The value selected in the drop-down list appears a default value in the ‘Disaster Patient Registration’ screen.

If the default ‘Patient Series’ is not configured in AE Parameter screen for Disaster Patient Registration, then the default ‘Patient Series’ configured for Emergency Patients in MP Parameter is applicable. If default Emergency patient series is also not set in the MP Parameter, then ‘Select’ appears as the default option for the ‘Patient Series’ field in the ‘Disaster Patient Registration’ screen.

 

  1. Service - Select a Service from the list. The services associated to the selected location in 'Service for Location' function in Outpatient Management module are listed. The selected Service gets displayed and defaulted in 'Disaster Patient Registration' function.

  2. Visit Type - Select a Visit Type from the list. The Visit Types defined for the selected location in 'Visit Type for Location' function in Ambulatory Care Management module are listed. The selected Visit Type gets displayed and defaulted in 'Disaster Patient Registration' function'.

  3. Priority Zone - Select a Priority Zone from the list. The system defined Priority Zones are Green, Red, Yellow and Unassigned. The selected Priority Zone is defaulted in 'Disaster Patient Registration' function.

  4. Default Unknown Adult Age - Enter the default Unknown Adult Age of the patient in numeric format of three digits maximum length. The age is defaulted in 'Disaster Patient Registration' function under 'Estimated Age' for Unknown patients.

The Vital signs fields is not displayed if Clinician Access module is un-installed.
