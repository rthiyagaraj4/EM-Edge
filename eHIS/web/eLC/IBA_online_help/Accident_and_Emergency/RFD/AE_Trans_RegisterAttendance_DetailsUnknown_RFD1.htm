Enter the following details to perform Emergency Registration:

 

  1. Details Unknown – This field is disabled indicating that the registration is performed for a patient whose details are unknown.

  2. Patient Series – This field indicates the series to be used to generate the Identification Number of a patient. The series set in MPI Parameter Setup gets displayed and disabled, by default.

  3. National ID – The defaulted National ID in numeric format is displayed and disabled while registration. The ‘Prompt’ specified in MP Parameter appears as the field legend. The length and the format of National ID are based on the National ID checks defined in MP Parameter.

  4. Other Alternate No – Select the required Alternate Number from the list. The listed values are defined in the function 'Alternate ID Type’ of Master Patient Index module. On selecting the required Alternate ID type, enter the ID in alphanumeric format of 20 characters maximum length. The field ‘Other Alt ID’ is displayed provided ‘Accept Other Alt ID’ is set in Alt ID checks tab in MP Parameter in the Master Patient Index module.

  5. Membership - Select the Membership from the list. The listed values are Organization Member, Related Member - Family, Related Member - Non Family, Non-Organization Member. It is mandatory to select the Alternate ID as set in the field 'Alternate ID Type for Membership ID' in MP Parameter function. This field is displayed if 'Maintain Organization Membership' check box is selected in the MP Parameter function.

  6. Organization - Select the Organization using [?] function. The Organization Sub Unit is displayed on selecting the Organization. If more than one Sub Unit is associated to the Organization, select the Sub Unit from the list.

  7. Head of Family Patient ID - This field is displayed based on the ‘Maintain Family Link’ set in MP Parameter. If family link is to be maintained, enter the Head Patient ID in the required format as specified in the MP Parameter, or click on the [?] to select the required Head Patient ID. The Head patient name is displayed next to the field.

  8. Relationship - The Relationship is displayed as set in the 'Relationship for Organization Member' field in the MP Parameter function on selecting the Membership as 'Organization Member'. Select the Relationship to the displayed Head of Family. This field is mandatory.

  9. Relationship Level - This field is displayed on selecting the Relationship with a level defined in the Relationship function. Select the Relationship Level using [?] function. The Relationship Levels defined in the Relationship Level function is displayed. This field is mandatory.

The above field names are displayed based on the legends for Alternate ID defined in the MP Parameter.

  1. Patient Name – The multi parts of Patient Name are the Prefix, First Name, Second Name, Third Name, Family Name and Suffix. Based on the setup in MP Parameter (Legal Names), the multi parts of patient name are displayed. The displayed titles and order of appearance are specified in the MP Parameter function. If ‘Accept Patient Name in Local Language’ is set in the MP Parameter, the patient name can be entered in local language. If a ‘*’ sign appears next to the name, it is mandatory to enter the data.

On entering the patient’s name, the system builds the patient name based on the derivation logic defined in MP Parameter and displays them in the text area after the Patient ID value.

  1. Gender - Select the Gender of the Patient from valid list of values. The list of values are Male, Female and Unknown. This field is mandatory.

  2. Age - Select the check box, to enter the Age of the Patient in years or months or days, if date of birth is not known. If age is entered, system calculates the date of birth and displays it in 'Birth Date' field. Either 'Age' or 'Date of Birth' only can be entered. On entry of either one, the other field becomes read-only. This field is mandatory.

  3. Birth Date - Enter the Birth Date in dd/mm/yyyy format. On entering the Birth Date, the system calculates the Patient's age and displays it in the 'Age (Y/M/D) fields. This field is mandatory.

  4. Birth Place - The Birth Place of the patient is displayed and disabled.

  5. Marital Status - The Marital Status of the patient is displayed and disabled.

  6. Patient Category - Category of the Patient is displayed and disabled. The listed values are defined in the function ‘Patient Category’ and the patient category must be associated to the Membership using 'Pat Cat by Org. Membership' master function.

  7. Residency - Select either 'Citizen' or 'Non-citizen'. By default, 'Citizen' is selected. If National ID is entered for the patient, 'Citizen' is selected and disabled, or both the values are available for selection.

  8. Status - Select either 'Legal' or 'Illegal' as citizen status. By default, 'Legal' is selected. If residency is set as 'Citizen', 'Legal' is selected and disabled. If the residency is set as 'Non-Citizen', the status can be either 'Legal' or 'Illegal'.

  9. Nationality - The defaulted Nationality set in MP Parameter is displayed. Select the required value using the [?] function. This field is mandatory. If National ID is entered, the default Nationality set in MP Parameter is displayed and disabled.

  10. Race - Select the Race from the list. This field is mandatory. The listed values are defined in 'Race' function in MP Code Setup.

  11. Ethnicity - The Ethnicity of the patient is displayed and disabled.

  12. Religion - Select the Religion of the Patient from the list box. The listed values are defined in the Religion function.

  13. Language - Select the language of the patient from the [?] function.
