Request Details  

The Request Details section is displayed below the ‘Requested Queue’ and ‘Ambulance Chart’ section in collapsed mode.  The Request Details section is used to raise a new ambulance request or update status of the existing request.

 

To record a new ambulance request:

  1. Enter the fields in the Request Details section:

    1. Nature of Call – Enter the first few letters of the nature of the nature of call and select it from the drop-down list. The short description of the nature of call in the ‘Nature of Call’ function will be available in the drop-down list.  This field is mandatory.  

    2. Priority – Select the priority of the request from the drop-down list. The list values are Low, Medium and High.

    3. Pickup Location – Select the pickup location whether it is external or hospital.  By default the External option is selected.

    4. If you select External check box, on entering the characters, values will be populated based on the ‘Town’ value configured in MPI. You can also enter a value which is not available in Town function. The captured value will be displayed next to Country and State value displayed in the ‘Google Map’ field.

    5. If you select the Hospital check box, the values captured in "Town" for the facility will be defaulted here and you can change if required and the same will be available next to Country and state value displayed in ‘Google Map’ field.

    1. Address – Enter the pickup address in alphanumeric format of 2000 characters maximum length. This field can be viewed only in the expanded mode.

    2. Pickup Time – Select the Immediate check box if the pickup is to be done immediately. The system date and time gets defaulted in the text field. Or, click the Calendar to enter a future or past date.

    3. Dropping Location – Select whether the dropping location is external or hospital.

    4. By default, the External check box is selected.  Enter the dropping location; on typing the characters, the town value will be populated.

    5. If you select the Hospital check box, the address gets populated based on configuration in Facility parameter for "Town" and it can be changed if required.

    1. Patient Name – Enter the patient name manually or select the Patient Name using [?] function. This is a mandatory field.

    2. Gender and Age – If you select the Patient Name using search, the gender and age gets displayed automatically else you need to enter it manually.

    3. Escort – Enter if any escort (attendant) is accompanying the patient. This field can be viewed only in the expanded mode.

    4. Attended By – If the patient is attended by any nurse or practitioner, enter their name. This field can be viewed only in the expanded mode.

    5. Requester Name – Enter the name of the person requesting for the ambulance. This is a mandatory field.

    6. Contact – Enter the contact details in alphanumeric format of 30 characters maximum length.

    7. Remarks – Enter the remarks if any. You can enter up to 2000 characters maximum length.

    8. Google Map – Based on the selected pickup location, the values configured in Facility for State, Country will be displayed here. This field can be viewed only in the expanded mode.

    9. Print - On click of the print button, Google map will be opened to search the address. On click on print button in the map, the map gets printed.

    10. Ambulance – You have to drag and drop the ambulance from ambulance chart. You can drag and drop the ambulance in Available and Assigned status. After dragging it, the details of ambulance will be populated for Available ambulance. If ambulance is assigned status, already captured request details will be displayed in this section.

    11. Dispatched – Select this check box if the ambulance gets dispatched.  On selecting this checkbox, fields like Odometer Start, Expected Distance, Dispatch Remarks, Resource gets displayed.

    12. Ambulance Service Type – Enter the ambulance service type. When you enter the first letter of the service type, the available values will be populated for selection. This field can be viewed only in the expanded mode.

  2. Click [Record]. The request gets saved and appears in the Requested Queue section with the status as ‘Requested’.

  3. Double-click the requested record in the Requested Queue section, the request details gets displayed in the Request Details section.

If you want to cancel, the request, update the Status as ‘Cancelled’ and click [Record]. The request gets cancelled.

  1. Select the Ambulance by dragging from the Ambulance chart if you have not assigned it already. To dispatch the ambulance, select the Dispatch check box, the below fields appear.

i. Odometer Start – Enter the odometer start value in numeric format up to 12 characters maximum length.

ii. Expected Distance – Enter the expected distance of travel. You can enter maximum of 12 digits.

iii. Dispatch Remarks – Enter the dispatch remarks if any in the text box.

iv. Resource – Enter the resource. On typing first few letters, the resource name appears as a drop-down. Select the required resource. The selected resource will be displayed below the resource field with a check box selected.

You cannot change the Odometer Start and Dispatch Remarks after the ambulance is dispatched.

  1. After entering the above fields, click [Record] the request status gets changed from ‘Requested’ to ‘Assigned’.

  2. Double-click Assigned request, the request details appear in the Request Details section. Enter the Distance Covered value once the patient is pickup and dropped in the required location , the Odometer End, End Date Time appear automatically.

  3. Click [Record]. The request status gets changed to ‘Completed’.

• Ambulance can be requested without assigning ambulance but Dispatch can be performed only after assigning an ambulance. If you try to dispatch without Ambulance, an error message will be thrown “Ambulance cannot be blank” gets displayed.

• You can change the ‘Pickup Time’ and ‘Start Date Time’ in Requested status. Once the status is Assigned, you cannot change these values.

• Even though Odometer End Time is not entered, Last Reading will be displayed in Requested Queue, where the Last Reading will be taken from the last value captured for the Ambulance.

• Odometer Start will be calculated based on the last odometer end of particular ambulance.
