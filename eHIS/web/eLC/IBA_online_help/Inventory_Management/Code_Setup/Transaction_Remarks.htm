Transaction Remarks

This function is used to Group the Reason for each Transaction Type and display the reason, which relate to specific transaction.

The Remarks are defined in the Transaction Remarks function of Materials Base module of Enterprise Management. The listed Transaction Types are Vendor Return, Request, Request Issue, Urgent Issue, Manufacturing, Product Transfer, Issue Return, Adjustment, Variance, Opening Stock, Sales, Sales Return.  

Creating an association of Transaction Remarks to a Transaction Type

From the Code Setup, select the Transaction Remarks function.

The Transaction Remarks screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Select the required Transaction Type from the list. The Transaction Types applicable to Inventory Management as generated through scripts are listed. This is a mandatory field.

  3. On selecting the required Transaction Type, the list of Transaction Remarks defined in the Materials Base module are displayed.

  4. Select the check box against the required Remarks Description if the Transaction Remark is applicable to the selected Transaction Type. Click Select check box in the header to select all the Transaction Remarks.

  5. Click [Apply] to save the association of Remarks to a Transaction Type or [Reset] to clear the entered data.

A ‘Header Level Remarks Required’ check box is displayed if the values ‘Request’, ‘Req Issue’ and ‘Consumption’ are selected in the Transaction Type drop-down list. By default, this check box is cleared.

Select this check box to configure the ‘Remarks’ field as mandatory in ‘Create/Review Request’, ‘Issue (Req/Urg)’ and ‘Stock Item Consumption’ functions.
