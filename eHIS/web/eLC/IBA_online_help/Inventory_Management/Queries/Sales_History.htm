Sales History

From this function you can Query the Sales Transaction History performed with respect to the Items sold out to a patient.  

The Sales Types are ‘Inpatients’, ‘Outpatients’ ‘Referrals’ and ‘Discharge', 'Direct Sales', 'Emergency', 'Day Care' and 'All'. The store and item details, document and sales type, document date and number, sale quantity and value, cost value, requested quantity of an item by the user, pending sales quantity, patient ID and name are displayed based on the search criteria.

The value is listed as zero, if the actual sale quantity of the item is greater than the requested quantity. When the actual sale quantity of the item is lesser than the requested quantity, the difference of [Requested Quantity - Actual Sale Quantity] is displayed as the column value.

The quantity of items issued to the user against the request placed are listed.

Viewing the Sales Transaction History

From the ST Queries, select the Sales History function.

The Sales History screen gets displayed.

  1. Click [Search] to view all the Sales Transaction History or enter the search criteria and click [Search] to display the required Sales Transaction History.

    Enter the following search criteria to view un-posted transactions:

     

    1. Doc Type - Select the required Document Type from the list. The documents associated to the Transaction Type ‘Sales/Patient Issues’, as set in ‘ST Document Type’ function are displayed in the list.

    2. Doc No - Enter the Document Number in numeric format of eight digits maximum length.

    3. Item [From-To] - Enter the range of Item for which the batch details are to be searched in alphanumeric format or select the required Item using [?] function. Both the values are optional. If ‘From’ value is not entered, the system assumes a low value for the ‘From’ selected item code and if ‘To’ value is not entered, the system assumes a high value for the ‘To’ Item code. When both values are entered, ‘To’ value has to be greater than or equal to the ‘From’ value.

    4. Item Class [From - To] - Enter the range of Item Class for which the batch details are to be searched in alphanumeric format of four characters maximum length. Both the values are optional. If ‘From’ value is not entered, the system assumes a low value for the ‘From’ selected Item Class code and if ‘To’ value is not entered, the system assumes a high value for the ‘To’ Item Class code. When both values are entered, ‘To’ value should be greater than or equal to the ‘From’ value. Click [?] function to query the Item Class code and select required values.

    5. Store Code - Enter the Store Code required to be viewed in alphanumeric format of 15 characters maximum length or select the required Store Code using [?] function.

    6. Drug/Non Drug - Select the Drug/Non Drug value from the list. The listed values are ‘Drug’, ‘Non Drug’ and ‘Both’. The value ‘Both’ is displayed, by default.

    7. Sales Type – Select the required value for Sale type from the list. The listed values are ‘Inpatients’, ‘Outpatients’ ‘Referrals’ and ‘Discharge', 'Direct Sales', 'Emergency', 'Day Care' and 'All'.

    8. Date [From-To] – Enter the ‘From’ and ‘To’ date period to view the Sales History.

    9. Item Code - Enter the required Item in alphanumeric format of 60 characters maximum length or select the Item using [?] function. This field is mandatory for Query Type ‘Detail’.

    10. Patient ID - Enter the Patient ID as was generated while registration in length as specified in MP Parameter.

    11. Patient Name – The patient name for the Patient ID is displayed.

    12. Nationality – The Nationality for the selected Patient ID is displayed.

    13. Encounter ID – Enter the required Encounter ID to search for particular sales history records.

  2. On searching, the Sales Transaction History is displayed based on the entered criteria.
