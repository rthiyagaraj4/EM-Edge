Enter the following details:

  1. Transaction Type – The code of the Transaction Type applicable to Pharmacy Stock as generated through the script is displayed.

  2. Long Description – The long description of the Transaction Type applicable to Inventory Management as generated through the script is displayed.

  3. Short Description – The short description of the Transaction Type applicable to Inventory Management as generated through the script is displayed.

  4. No of records per Transaction - Enter the number of records per Transactions in numeric format of four digits maximum length. This option is not applicable for Sales Transactions.

  5. Item Class Based Y/N – Select the check box to make the Transaction Type specific to an Item Class. This option is not applicable for Manufacturing Receipts and Variance Transactions.

  6. Item Class Wise Documents - Select the ‘ Item Class Wise Documents ’ check box to generate the documents for the transaction types based on the Item Class. The check boxes are cleared, by default.

  7. Doc Date Modifiable Y/N - Select the check box, if the Document Date for the selected Transaction can be modified.

If 'Item Class Wise Documents' is selected for any transaction type and the transaction has items belonging to multiple item classes, the documents are generated for each Item Class by considering the number of records per transaction.

The documents cannot be generated based on the Item Class for the following transactions and the 'Item Class Wise Documents' check box is disabled for these transactions:

  • Condemn Expired Batches

  • Manufacturing Receipts

  • Variance

 

If 'Item Class Wise Documents' check box is selected for any transaction type, in the query mode of the un-finalized documents, user will not be allowed to add items which belong to different Item Classes. The Item lookup will display the Items belonging to particular Item Class to which the document is generated.

 

The system will not invoke the Bill Generation and Settlement Page ID  if multiple documents are created for 'Sales' and 'Sales Return' transactions. The Bill Generation and Settlement Page is invoked for the single document.
