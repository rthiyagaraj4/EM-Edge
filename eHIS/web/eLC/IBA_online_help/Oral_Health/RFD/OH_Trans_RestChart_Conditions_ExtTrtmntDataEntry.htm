Enter the following details to record the treatment:

 

  1. Clinical Group – Select a Clinical Group from the list. The Clinical Group defined in the Code setup is listed. It identifies the Clinical Group, where the patient is treated.

  2. Category – Select a value from the list to select a Category. The Treatment Categories are system defined. This is a mandatory field.

  3. Site – Based on the Category selected, the Site details are displayed. The values are system defined and differ based on the Condition Type. This is a mandatory field. The Site values are:

  4. Arch – Select Arch and the Condition Type is applicable for the Arch.

  5. Quadrant – Select Quadrant and the Condition Type is applicable for the Quadrant.

  6. Tooth – Select Tooth and the Condition Type is applicable for a Tooth.

  7. Tooth Range – Select Tooth Range and the Condition Type is applicable for a Tooth Range. You can select the tooth numbers for which the Condition Type is applicable.

  1. Area – The Area differs based on the Site selected. This is a mandatory field. The values are:

  2. If the Site is selected as ‘Arch’, the Area is set to ‘Arch’. You can select either the Upper Arch or the Lower Arch from the list. The values are system defined.

  3. If the Site is selected as ‘Quadrant’, the Area is set to ‘Quadrant’. You can select either the Upper Right, Upper Left, Lower Right or the Lower Left quadrants from the list. The values are system defined.

  4. If the Site is selected as ‘Tooth’, the Area is set to ‘Tooth No’. You can select a tooth from the list. The selected Condition Type is applicable for the tooth selected. The values are system defined.

  5. If the Site is selected as ‘Tooth Range’, the Area is set to ‘Tooth Nos’. You can select a range of tooth from the list using [Ctrl] key. The selected Condition Type is applicable for the selected tooth range. A minimum of two teeth should be selected for the tooth range. The values are system defined.

  1. Status – Select the Status from the list. The system defined values are ‘Complete’, ‘Incomplete’ and ‘Mark Error’. You can select the status as Complete, if the treatment is completed. Select the status as Incomplete, if the treatment is not completed. Select the status as Mark Error, if the treatment details are recorded by error. This is a mandatory field.

  2. Treatment – Select a Treatment from the list. The treatments defined in the code setup and associated to the selected Category are displayed. This is a mandatory field.

  3. Location - Enter the external Location, where the patient was treated initially. The Location identifies the external source, where the patient was treated for the condition recorded. This is a mandatory field.

  4. Surface – Select the Surface from the list on which the Dental Condition and the Condition Type is applicable. The values are system defined.

  5. Cusp Tip - Select a value from the list. The values differ based on the Treatment Category and Site selected. This is a mandatory field.

  6. Performed Date – Click the Calendar function to select a date. The date field indicates the date, when the treatment was performed for the condition recorded. The Date has to be equal to or less than the system date. This is a mandatory field.

  7. Outcome – Select the values from the list. The values are system defined – ‘Success’ and ‘Failure’.

  8. Success – Select ‘Success’ to indicate that the external treatment was a success.

  9. Failure - Select ‘Failure’ to indicate that the external treatment was a failure.

  1. Failure Reason – Click [?] function to select a reason for external treatment failure. The values recorded through the ‘Failure Reason’ function are listed. This is a mandatory field.

  2. Remarks – Enter the remarks in alphanumeric format.

  3. Click [Record] to save the record or [Reset] to clear the data and to enter new external treatment.
