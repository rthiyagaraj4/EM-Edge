1. Location Type – Select a Location Type from the list. ‘All’ is displayed, by default. Other values for the Location Type are Day Care, Emergency, Inpatient and Outpatient. On selecting Day Care, a Day care legend is displayed with all the Day care units that are associated to the logged in practitioner. Similarly, when Emergency is selected, a legend is displayed which opens up all the Emergency units associated to the logged in practitioner. When an Inpatient is selected, a legend is displayed which opens up with the list of all the nursing units and when Outpatient is selected, a legend is displayed with the list of all the Clinics.

  2. View List - Select one of the values ‘Self Related', or 'All' from the list. 'Self Related' is displayed, by default. Self-related displays the list of patients who are associated to the logged in practitioner. When ‘All’ is selected, the list of all the patients associated to other practitioners (to whom the logged in practitioner may not be primary) also gets listed in addition to the self-related patients.

  3. Patient ID – Select the required patient using [?] function.

  4. Attending Practitioner - Enter the Practitioner name or click [?] function to search for the Attending practitioner. This field is enabled only if the 'View List' field is selected as All. If View List is selected as Self Related, the logged in user is displayed and disabled. All Practitioners are displayed for selection on clicking the [?] function.

  5. Specialty - Enter the Specialty or click [?] function to select the specialty. The Specialties associated to the Practitioner is displayed in the search screen for selection.

  6. Visit Type - Select the Visit Type from the list. This field is enabled only if the Patient Class 'Outpatient'. The Visit Types listed for selection are defined in the Visit Type function of Outpatient Management module.

  7. Unassigned – When this check box is selected, the list of all the patients to whom no attending practitioner is assigned gets displayed. This field gets disabled if the Location Type selected is 'Day Care’ or 'Inpatient'.

  8. Group by Patient Class - The check box is selected and disabled, by default.  This facilitates the display of the patient list in the order of Patient Class. The check box is cleared to display without the Patient Class grouping.

  9. Group by Location - The check box is selected, by default. This facilitates the display of the patient list in the order of Location. The check box is cleared to display without the Location grouping.

  1. National ID (Alternate ID) - Enter the National ID in alphanumeric format as mentioned in MP Parameter of maximum length.

  2. Citizen ID (Prompt specified in MP Parameter) - Enter the patient Citizen ID in alphanumeric format as mentioned in MP Parameter of maximum length.

  3. Past Visits - This check box is enabled only on selecting Location Type as All, Emergency or Outpatients. Select the check box to list patients with any previous visits. The check box is cleared and disabled when the selected Location Type is Daycare or Inpatient.

  4. Period - The field is enabled only when 'Past Visits' check box is selected. The 'Date' value is defaulted as defined in the parameter. Enter the date in dd/mm/yyyy format within the allowable period as defined in the parameter. This is to filter patient listing with encounters within the specified period range.

  5. Discharged - This check box is enabled on selecting Location Type as All, Daycare or Nursing Unit. Select the check box to include the discharged patients in the list. The check box is cleared and disabled if the selected Location Type is Emergency or Outpatient.

  6. Period - The field is enabled only if 'Discharged' check box is selected. The 'Date' value is defaulted as defined in the parameter. Enter the date in dd/mm/yyyy format within the allowable period as defined in the parameter. This is to filter patient listing with encounters within the specified period range.

  7. Admitted Period – Enter the date range using the Calendar function. Enter the date in dd/mm/yyyy format within the allowable period as defined in the parameter. This is used to filter patient listing based on admission dates. This field is displayed if the Location Type is selected as All, Daycare or Nursing Unit.

  8. Queue Status – This check box is used to filter patients based on various queue statuses applicable to an Emergency or Outpatient encounter. The various statuses listed are ‘Checked In’, ‘Arrived at NS’, ‘VS Recorded’, ‘Cons in Progress’, ‘Treat/Invest’, ‘Return from Invest’ and ‘Checked Out’.

The referral details of the patient get displayed. On the top part of the screen the patient demographics is displayed. In the other part of the screen the details of Referred From and Referred To gets displayed under two tabs.

Referred From:

The Referred From details of the patient is displayed. Click [Clinical Details] to view the Exam and Clinical findings, Referral Diagnosis and Treatment recorded during the referral registration.

Click [Notes] to view the Referral Notes recorded during the referral registration.

Referred To:

The Referred To details of the patient is displayed.

 

 

Patient List by Location

From this function you can View patient list based on the primary care location assigned to patients during an active encounter.

The function lists all patients who are directly related to the logged in practitioner. Also, practitioner can view patients who are not directly related to the practitioner and patients who have not been assigned any practitioner. The function provides various query criteria, which is used to filter patient listing and view patients based on past encounters.

Viewing Patient List by Location

From the CA Transactions > Clinician Access > Desktop Menu > Patient List, select By Location.

The Patient List by Location screen gets displayed.

  1. Enter the search criteria.

  2. Click [More Criteria] to enter additional search details.

  3. Click [Less Criteria] to enter minimum search details.

  4. Click [Search] to display patients based on the entered criteria. You can view the Outpatient, Day Care, Inpatient and Referral details in the tab pages. Click Confirm hyperlink to view and Confirm Bedside Referral details. The Encounter Details, Nursing Unit Details and Referral Details are displayed. Enter the Bedside Referral Seen Date in dd/mm/yyyy format or select the Bedside Referral Seen Date using Calendar function. The Bedside Referral Seen Date cannot be greater than the current system date. This is a mandatory field. Click [Record] to save the record or click [Close] to exit.

  5. A ‘*’ symbol is displayed against the patient name, if relationship between the practitioner and patient does not exist. Click Patient Name hyperlink to display the Assign Relationship screen and assign relationship.

  6. For patients with relationship assigned, Patient Chart is displayed.
