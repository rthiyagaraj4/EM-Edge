Enter the following details to create a Relationship record:

 

  1. Relationship ID - Enter the Relationship ID in alphanumeric format of 20 characters maximum length. The Relationship ID must be unique and cannot be changed after creating. This field is mandatory.

  2. Relationship Name - Enter the Relationship Name in alphanumeric format of 60 characters maximum length. This field is mandatory.

  3. Sequence No - Enter the Sequence Number in numeric format of four digits maximum length. This value should be unique.

  4. Nature - Select the Nature of the Relationship as Lifetime or Encounter Specific. By default, Encounter Specific is selected.

  5. Enabled – Select the check box to enable the Relationship record.

 

Direct/Indirect Settings

The Direct/Indirect setting is applicable based on the Relationship. The condition that is applied is as follows:

  • If the nature selected is 'Lifetime', 'Allow direct' field is internally set to be enabled, whereas fields under the legend ‘Relationship’ are internally disabled. The Expiry Rule is optional in this case. Select the check box to enable the 'Expiry Rule Applicable', if it is enabled the Expiry Rule is selected from the list box which contains the values 'Day after effect' or 'Hour after effect', which is mandatory and enter the expiry period in numeric format of six digits maximum length. If the 'Expiry Rule Applicable' is disabled no rule is applicable. This indicates that all relationships set up as ‘Lifetime’ is user assigned.

  • If the nature selected is 'Encounter Specific', select the 'Allow direct' check box to enable or clear to disable the flags under the ‘Relationship’ legend.

  • If 'Allow Direct' field is enabled, all flags under the ‘Relationship’ legend are internally set to be disabled. The 'Expiry Rule Applicable' is set enabled internally. Select the appropriate expiry rule applicable from the list of values with an expiry period.

  • If the 'Allow Direct' is disabled, the flags under the ‘Relationship’ legend are enabled. Select the check box to enable or clear the check box to disable the flags. The 'Expiry Rule Applicable' is selected and set enabled internally to select the appropriate ‘Expiry Rule’ applicable from the list of values with an expiry period.

All Direct Relationships are available to assign relationships to other practitioners using the task ‘Assign Relationship’ provided in the Clinician Access module. All Indirect Relationships are assigned by the system to the various practitioners that are identified in various transactions of Enterprise Management. The details of when the system assigns an indirect relationship to a practitioner are detailed below.

 

Relationship

  1. IP Enc - Select the check box if applicable. The relationship defined with this flag is assigned to the ‘Attending Practitioner’ identified in the ‘Admit Patient’ function or the ‘Assign Bed/Mark Patient Arrival’ functions in the Inpatient and Day Care Management module.

  2. IP Admit - Select the check box if applicable. The relationship defined with this flag is assigned to the ‘Admitting Practitioner’ identified in the ‘Admit Patient’ function in the Inpatient and Day Care Management module.

  3. OP Visit - Select the check box if applicable. The relationship defined with this flag is assigned to the ‘Attending Practitioner’ identified in the ‘Register Visit’ function in the Outpatient Management module.

  4. Accident & Emergency - Select the check box if applicable. The relationship defined with this flag is assigned to the ‘Attending Practitioner’ identified in the ‘Register Attendance’ function in the Accident & Emergency module.

  5. Day Care - Select the check box if applicable. The relationship defined with this flag is assigned to the ‘Admitting Practitioner’ identified in the ‘Admit Patient’ function in the Inpatient and Day Care Management module for day care patients.

  6. Referral Reln - Select the check box if applicable. The relationship defined with this flag is assigned to the ‘Practitioner’ identified in the ‘Refer Patient’ function in the Clinician Access module.

  7. Nurse Relation - Select the check box if applicable. The relationship defined with this flag is assigned to any nurse picking up a patient from the Patient List in the Clinician Access module.

  8. Default - This flag can be selected against each of the relations mentioned above. This indicates that the relation set is defaulted by the system if there are multiple relationships defined for each of the flags.

 

Expiry Rule

  1. Expiry Based - Select the check box if the Relationship is expiry based. This field is selected and disabled if the Nature chosen is Encounter specific. If the Nature chosen is Lifetime, the check box is optional.

  2. Rule - Select a valid rule from the list. The value for this field is selected only if the Expiry Based check box is selected. This field is mandatory, if the ‘Expiry Based’ check box is selected.

  3. Expiry Period - Enter the period in numeric format of six digits maximum length. This field is mandatory if the ‘Expiry Based’ check box is selected.
