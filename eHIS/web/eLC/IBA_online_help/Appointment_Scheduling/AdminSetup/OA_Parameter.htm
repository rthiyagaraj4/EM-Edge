OA Parameter

From this function you can Create a rule builder to define certain controls on various transactions within the module at an enterprise level.

OA parameter allows you to define rules to control allocation criteria and limit scheduling based on either visits or patients. Since this is at the enterprise level, all the clinics across the facilities are governed by rules defined in this parameter. Only one parameter record is available. It is mandatory to set up the parameter before any operations are performed as these records store important details about Appointment Scheduling module.

Defining OA Parameter

From the OA Admin Setup, select the OA Parameter function.

The OA parameter screen gets displayed with General Tab enabled.

  1. Enter the details in the mandatory (*) fields. <Related Field Description>

  2. Click [Apply] to define OA Parameter or [Reset] to clear the entered data. If OA Parameter is already defined, on clicking [Reset], the screen reverts to the previously defined values.
