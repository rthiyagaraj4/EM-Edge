Enter the following details to create a Timetable for Scheduling record:

 

Location/Resource Details

  1. Location Type – Select the Location Type as Clinic, Daycare Unit or Procedure Unit.

  2. Location -  Select the Location based on the selected Location Type. This field is mandatory. On selecting the required Location Type, the Locations listed are based on the following conditions:

  3. All the Clinics/Daycare/Procedure units defined under the selected location types that allow appointments (based on the ‘Allow Appointment’ value set in ‘Clinic’, ‘Daycare Unit’ and ‘Procedure Unit’ setup function of Outpatient Management module).

  4. All the defined Clinics/Daycare/Procedure units must be associated to the operator station assigned to the logged in user.

  1. Resource Type – This field is enabled on selecting a location. Select the Resource Type as Practitioner, Equipment, Room or Other. This field is mandatory.

  2. Resource – Select the Resource based on the selected Resource Type using the [?] function. This field becomes mandatory based on the following condition:

  3. If ‘Check in at identification’ is set to Resource in ‘Clinic’, ‘Day Care’ and ‘Procedure Unit’ setup (OP module), the resource field becomes mandatory while defining the timetable or if ‘Check in at identification’ is set to Non Identifiable. This field is not mandatory.

 

Day/Time Details

  1. Roster Required? - The check box indicates whether the Location/Resource/Day Timetable combination requires a Resource attending on a roster basis. This field is enabled on selecting a Resource. The check box is cleared and disabled, by default. If the check box is selected, rosters for attending Resource must be entered through the ‘Duty Roster for Resources’ function before generating a schedule for the Location.

  2. Days - Select the Day of the Week for which the timetable is to be defined from the list of days displayed. If Resource identification is mandatory, this field gets enabled only on selection of a resource. All the working days of the ‘Clinic’, ‘Day Care’ or ‘Procedure Unit’ as set in the ‘Outpatient Management’ days of the week gets displayed in the list. You can select ‘*All’ from the list of values to select all the displayed days of the week. This field is mandatory.

  3. Time Table Type - Select the Timetable type, which can be one of the three types, mentioned below, from the valid list of Time Table Types. This field is mandatory.

  4. Slot - A Slot is the time required by the Practitioner to consult each patient.

  5. Slab – A Slab defines the time ranges within a day, the Practitioner can see a fixed number of patients. The time taken per patient need not be constant.

  6. Free Format - Free format is a single time frame for a given day, a Practitioner can see a fixed number of patients. The time taken per patient need not be constant.

  1. Start Time - The operating start time of the selected location as set in ‘Clinic’, ‘Day Care’ or ‘Procedure Unit’ function of Ambulatory Care Management module gets displayed on selecting the required timetable type, by default. Enter the Timetable Start Time for the selected Location/Resource/Day combination in hh:mm format. The entered start time must not be less than the location start time and greater than the location end time as set in Outpatient Management module, or the system alerts you with a message. This field is mandatory.

  2. End Time - The operating end time of the selected location as set in ‘Clinic’, ‘Day Care’ or ‘Procedure Unit’ function of Outpatient Management module gets displayed on selecting the required time table type, by default. Enter the Timetable End Time for the selected Location/Resource/Day combination in hh:mm format. The entered end time must not be greater than the location end time as set in Outpatient Management module, or the system alerts you with a message. The end time must be greater than the Start Time entered. This field is mandatory.

 

Slot Details/Slab Details/Free Format Details

  1. Time Per Slot - The base Slot/Slab time as set in ‘Clinic’, ‘Day Care’ or ‘Procedure Unit’ function of Outpatient Management module gets displayed on selecting the required Time Table Type, by default. Enter the required time to be allocated to each Patient in hh:mi format. The entered time per slot must be in multiples of the base time set for slot/slab in Outpatient Management module, or you are alerted with a message. (For example if the ‘Base slot/slab time in Outpatient Management module is set to 00:30 min, you can either retain time per patient/slot as 00:30 minutes or enter new time per slot in multiples of 00:30 min like 00:60 min, 00:90 min, 02:00 hr, etc.). This field is mandatory on selection of the Time Table Type as ‘Slot’.

  2. Maximum Slots per Day - Total number of slots planned for the day for the entered Location/Resource/Day combination is calculated after the time per Slot/Patient is specified. This value is computed and displayed based upon the Time per Patient (for Slot) values.

  3. Max Patients per Day - Total number of appointments planned for the day for the entered Location/Resource/Day combination is calculated after the time per Patient is specified. This value is computed and displayed based on the Time per Patient (for Slot) values. This field is mandatory.

 

Visit/Over Booking Details

  1. No. of First Visits (Patients/Slots) - Enter the maximum number of appointments to the Visit Type category as ‘First Visit’, which is allowed for the entered Location/Resource/Day combination, in numeric format of three digits maximum length. The entered value is based on the OA Parameter setup. The number of first visit is based on ‘Patients’ or ‘Slots’ as set in OA Parameter.

  2. No. of Other Visits (Patients/Slots) - Enter the maximum number of appointments to the Visit Type category as ‘Others’, which is allowed for the entered Location/Resource/Day combination, in numeric format of three digits maximum length. The entered value is based on the OA Parameter setup. The number of other visits is based on ‘Patients’ or ‘Slots’ as set in OA Parameter.

  3. Max New Patients - This field is available only if the 'Schedule Rule Applicable' check box is selected in the OA Parameter. Enter the maximum number of new patients planned for visit. This is a mandatory field.

  4. Max Old Patients - This field is available only if the 'Schedule Rule Applicable' check box is selected in the OA Parameter. Enter the maximum number of old patients planned for visit. This is a mandatory field.

  5. Maximum Over Bookings (Patients/Slots) - Enter the maximum number of over bookings, which is allowed for the entered Location/Resource/Day combination, in numeric format of three digits maximum length. The number of over bookings is based on ‘Patients’ or ‘Slots’ as set in OA Parameter.

The value entered facilitates to accommodate additional patients who need to be scheduled due to unforeseen medical reasons. If the value is not entered, over bookings cannot be made.

 

Other Details

  1. Forced in Bookings allowed – Select the check box to allow forced bookings for the entered Location/Resource/Day combination or clear the check box. This check box is enabled if the Time Table Type is selected. The check box is cleared, by default.

  2. No. of Forced in Booking/day – This field is applicable for Time Table of type ‘Slot’ and ‘Slab’ and is enabled on selecting the ‘Forced in Bookings Allowed’ check box. You can enter one forced in booking per slot in numeric format, or the system alerts you with a message ‘No. of Forced Booking cannot exceed Maximum Patients per Day’.

  3. No. of Global booking per day – Enter the number of bookings that is accepted as global for the Location/Resource/Day combination in numeric format of three digits maximum length. Global booking is specified to bring a control on the number of appointments that is booked to the Location/Resource/Day combination by other resources while referring a patient from Ambulatory Care Management module or while discharging a patient from Inpatient and Day Care Management module.

 

Slab

 

Enter the following details on selection of ‘Slab’:

 

  1. Start Time - The operating start time of the selected location as set in ‘Clinic’, ‘Day Care’ or ‘Procedure Unit’ function of Ambulatory Care Management module is displayed on selecting the required Time Table Type, by default. Enter the Timetable Start Time for the selected Location/Resource/Day combination in hh:mm format. The entered start time must not be less than the location start time and greater than the location end time as set in Outpatient Management module or the system alerts you with a message. This field is mandatory.

  2. End Time - The operating end time of the selected location as set in ‘Clinic’, ‘Day Care’ or ‘Procedure Unit’ function of Outpatient Management module is displayed on selecting the required time table type, by default. Enter the Timetable End Time for the selected Location/Resource/Day combination in hh:mm format. The entered end time must not be greater than the location end time as set in Outpatient Management module or the system alerts you with a message. The end time must be greater than the Start Time entered. This field is mandatory.

 

Slab Details

  1. Time Per Slab - The base slot/slab time as set in ‘Clinic’, ‘Day Care’ or ‘Procedure Unit’ function of Outpatient Management module is displayed on selecting the required Time Table Type, by default. Enter the required time to be allocated to each Patient in hh:mi format. The entered time per slab must be in multiples of the base time set for slot/slab in Outpatient Management module, or you are alerted with a message ‘Time per Slab must be multiples of given duration hh:mm’. This field is mandatory on selection of the Time Table Type as ‘Slab’.

  2. Maximum Patients per Slab – Enter the total number of appointments planned for a Slab for the entered Location/Resource/Day combination in numeric format of three digits maximum length. This field is mandatory, if the Time Table Type is selected as ‘Slab’.

  3. Max Patients per Day - Total number of appointments planned for the day for the entered Location/Resource/Day combination is calculated after the time per slab and maximum patients per slab is specified.

 

Visit/Over Booking Details

  1. No. of First Visits - Enter the maximum number of appointments to the Visit Type category as ‘First Visit’, which is allowed for the entered Location/Resource/Day combination, in numeric format of three digits maximum length. The entered value is based on Rule for limiting first/other visits.

  2. No. of Maximum Other Visits - Enter the maximum number of appointments to the Visit Type category as ‘Others’, which is allowed for the entered Location/Resource/Day combination, in numeric format of three digits maximum length. The entered value is based on Rule for limiting first/other visits.

  3. Max New Patients - This field is available only if the 'Schedule Rule Applicable' check box is selected in the OA Parameter. Enter the maximum number of new patients planned for visit. This is a mandatory field.

  4. Max Old Patients - This field is available only if the 'Schedule Rule Applicable' check box is selected in the OA Parameter. Enter the maximum number of old patients planned for visit. This is a mandatory field.

  5. Maximum Over Bookings - Enter the maximum number of over bookings which is allowed for the entered Location/Resource/Day combination, in numeric format of three digits maximum length. The entered value is based on the OA Parameter setup.

The value entered facilitates to accommodate additional patients who need to be scheduled due to unforeseen medical reasons. If the value is not entered, over bookings cannot be made.

 

Other Details

  1. Forced in Bookings allowed – Select the check box to allow forced bookings for the entered Location/Resource/Day combination, or clear the check box.

  2. No. of Forced in Booking/day – This field is applicable for Time Table of type ‘Slot’ and ‘Slab’ and is enabled on selecting the ‘Forced in Bookings allowed’ check box. You can enter only one forced in booking per slab in numeric format, or the system alerts with a message ‘No. of Forced Booking cannot exceed Maximum Slabs per Day’.

  3. No. of Global Booking for Referral/day – Enter the number of bookings that is accepted as global for the Location/Resource/Day combination in numeric format of three digits maximum length. Global booking is specified to bring a control on the number of appointments that is booked to the Location/Resource/Day combination by other resources while referring a patient from Outpatient Management module or while discharging a patient from Inpatient and Day Care Management module.

  4. Remarks - Enter the Remarks for the timetable in alphanumeric format of 40 characters maximum length.

 

Free Format (Applicable for all Resource Class)

 

Enter the following details on selection of ‘Free Format’:

 

  1. Start Time - The operating start time of the selected location as set in ‘Clinic’, ‘Day Care’ or ‘Procedure Unit’ function of Ambulatory Care Management module is displayed on selecting the required time table type, by default. Enter the Timetable Start Time for the selected Location/Resource/Day combination in hh:mm format. The entered start time must not be less than the location start time and greater than the location end time as set in Outpatient Management module; or the system alerts you with a message. This field is mandatory.

  2. End Time - The operating end time of the selected location as set in ‘Clinic’, ‘Day Care’ or ‘Procedure Unit’ function of Outpatient Management module is displayed on selecting the required time table type, by default. Enter the Timetable End Time for the selected Location/Resource/Day combination in hh:mm format. The entered end time must not be greater than the location end time as set in Outpatient Management module, or the system alerts you with a message. The end time must be greater than the Start Time entered. This field is mandatory.

  3. Maximum Patients per Day – Enter the total number of appointments planned for a day for the entered Location/Resource/Day combination in numeric format of three digits maximum length. This field is mandatory on selection of the Time Table Type as ‘Free Format’.

 

Visit/Over Booking Details

  1. No. of Global booking for Referral/day – Enter the number of bookings that is accepted as global for the Location/Resource/Day combination in numeric format of three digits maximum length. Break Timings
