The following are displayed based on the query criteria:

 

  1. Location Type - The Location Type is displayed.

  2. Location – The Location is displayed.

  3. Resource Class – The Resource Class is displayed.

  4. Resource Name – The Resource Name is displayed.

  5. Date – The date is displayed.

  6. Day – The day is displayed.

  7. No of Appts – The total number of appointments booked for a practitioner on a day. Click 'No of Appts' hyperlink to view the list of appointments for a day.

  8. Time per Slot/Slab – The time allotted to the patient for consultation is displayed.

  9. Slots to be blocked – Total number of slots that are available for blocking within the selected time and date range is displayed.

  10. No. of First Visits – This field gets enabled for entering if the scheduling is based on slots. Enter the Number of First Visits that has to be blocked in numeric format of three digits maximum length.

If the number of slots to be blocked is entered as three and the maximum first visit that can be blocked is one as per visit rule, the number of first visits cannot be more than one. If a greater value is entered in number of first visits field, an alert message 'Total number of block slots must match Slots to be blocked' gets displayed and you cannot proceed.

  1. No of Other Visits – This field gets enabled for entering, if the scheduling is based on slots. Enter the Number of Other Visits that has to be blocked in numeric format of three digits maximum length.

If the number of slots to be blocked is entered as three and the maximum other visit that can be blocked is two as per visit rule, the number of other visits cannot be more than two. If a greater value is entered in number of other visits field, an alert message 'Total number of block slots must match Slots to be blocked' gets displayed and you cannot proceed.

• If the sum of first visit and the other visit is not equal to the total number of slots to be blocked, an alert message 'Total number of block slots must match Slots to be blocked' gets displayed and you cannot proceed.

• If the Visit Limit rule is defined as ‘None’ in the OP Parameter, there are no first visits, only other visits gets displayed and disabled with the number of slots to be blocked. If the Visit Limit Rule is defined as ‘Maximum’ or ‘Exact’, you can enter both first visits and other visits for slot based scheduling.

• If the scheduling is slab based, the slots to be blocked is defaulted and disabled in the ‘Number of Other Visits’ field.
