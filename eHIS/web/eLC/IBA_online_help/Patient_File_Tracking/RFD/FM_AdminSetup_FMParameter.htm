4. Default Settings

Permanent FS Location

Select the Permanent File Storage Location from the list. The listed values are Permanent File storage locations defined in File Storage Location function. The selected permanent file storage location is displayed as default value while generating the patient file in Outpatient module, Inpatient module and Master Patient Index module. This field is mandatory.

Inactive FS Location

Select the File Storage Location from the list. The listed File Storage Locations are defined in File Storage Location function. The File Storage Locations belonging to the File Storage identity Department are listed.

The value specified is displayed while changing the file status using the function Change File Status. The inactive or archived files are stored in this specified File Storage Location. You cannot change the location in the Change File Status transaction function. This field is mandatory.

Lost FS Location

Select the File Storage Location from the list. The listed File Storage Locations are defined in File Storage Location function. The File Storage Locations belonging to the FS identity Department are listed.

The value specified is displayed while changing the file status using the function Change File Status. The lost files are assigned to the specified File Storage Location. You cannot change the location from the Change File Status Transaction function. This field is mandatory.

Incomplete FS Location

Select the File Storage Location from the list. The listed File Storage Locations are permanent File Storage Locations defined in File Storage Location function.

Files of discharged inpatients are routed to the selected File Storage Location. Monitor File Deficiency is performed from the specified File Storage Location.

Narration for Auto Track Out to OPD

The value specified is used in the Transaction Track Out to OPD. This is the default narration to auto track files to the Clinics/Day Care/Procedure unit. This field is mandatory.

Narration for returning files to MRD.

The value specified is displayed as default in the transaction function Receive File at Medical Records Department. This is the default narration for returning files by File Storage Locations (both permanent and non-department) to the Medical Record Department. You can change the default narration to any other narrations while using Receive File at Medical Records Department Transaction function.

5. Other Rules.

No. of Days up to which to check for appointments in File Inquiry.

The value specified is displayed in the query function File Current FS Location. The number of days (specified) indicates the days prior to which you can check for the patient’s appointments with the Outpatient Clinic for auto tracking.

No. of Days up to which to check for bookings in File Inquiry.

The value specified is displayed in the query function File Current FS Location. The number of days (specified) indicates the days prior to which you can check for the patient’s booking for admission with the Nursing Unit for auto tracking.

Control Pulling List Generation By Gender

Select the check box, to specify whether generation of File pulling list is to be gender based. If the control is not to be gender based, clear the check box. Once the value is set, it cannot be changed.

Link Between File and Encounter Required

To link file and encounter select the check box. If the check box is selected the transaction function Link/De-Link/Change Encounter-File Volume can be invoked.

Accept Outside & In Transit Files Directly in Receive File at MRD.

In the function ‘Receive File at MRD’ the field ‘File Type’ becomes mandatory only if the check box is selected. The check box is selected by default. If the check box is selected, you can receive files in ‘Receive File at MRD’ function for the status ‘Returned’, ‘Outside’ and ‘In transit’. If you clear the check box, the File status is changed to ‘Returned’ automatically and displays only the returned files.  

Auto Track Out Upon Confirm Pulling List

If the check box is selected, the Auto Track out happens on Conform Pulling List, otherwise ‘Auto Track out has to be performed manually using function Auto Track out to OPD or Auto Track out to IPW. If the check box is selected the functions ‘Auto Track out to OPD’ and ‘Auto Track out to IPW’ are not applicable.

Enable Monitor File Deficiency when Incomplete FS Location not defined

The parameter enables you to monitor the file deficiency, when the Incomplete FS Location in FM Parameter is not defined. If the check box is selected, the FS Location is not displayed in Monitor File Deficiency function. If the check box is cleared, the File status is displayed as ‘Returned’ and [Show all Files] is enabled in ‘Receive File at MRD’ function. If the check box is selected, the ‘Incomplete FS Location’ field is disabled and vice versa. Select the check box to restrict the selection of any incomplete file storage location in FM Parameter.  The check box is selected, by default.

Auto Check Print Patient File Request Slip in Request File

The parameter enables you to print the patient file request slip. If this check box is selected, the Print Patient File Request Slip is selected by default for every request and enabled in the Request file. If the check box is cleared, Print Patient File Request Slip is unchecked and enabled in the Request file. You can select or deselect when required. The check box is unchecked, by default.
