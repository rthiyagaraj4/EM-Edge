Request

From this function you can Request for Trays from an Owner Store and Query for un-authorized requests. You can also Delete the un-authorized requests.

On raising the request, the transaction is authorized. The following conditions are validated by the system.

  • The logged in user should have rights to request from the requesting store

  • The requesting and owner stores must have access to the requesting group and alternate group.

The following trays cannot be requested:

  • The Trays which are currently not with the owner location

  • If the Physical Inventory process is in progress

Requesting Trays from Owner Store

From SS Transactions > Issue, select the Request function.

The Request screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the details in the mandatory fields. <Related Field Description>

  3. Click [Add] to add the request details or [Cancel] to clear the entered details.

  4. If requested quantity of the sterilized tray is currently not available with the owner, on adding the tray details, the ‘Available Sterilized Quantity’ will display the value as ‘0’. The number of sterilized trays and un-sterilized trays available are displayed.

  5. On adding the request, the requested tray details are displayed in a table format in the right hand side of the screen. The system checks if the requested trays are available or not. If it is not available, the Available Sterilized Quantity will be ‘0’.

  6. Click GroupType hyperlink to modify the request details.

    • You can modify the details like the Document Type, Requested By, Requested On, Auth Group, Req Quantity, Surgery Type and Surgery Name . You cannot modify the Group Type and Req Group as both these fields are disabled.

    • Click [Modify] to save  the modified changes or [Cancel] to clear the enter data.

  7. Click ‘Del’ check box to delete the added request.

The request will be deleted once you click Apply.

  1. Click [Apply] to save the Request details or [Reset] to clear the entered data.

  2. On clicking [Apply] and if ‘Entry Completed’ check box is selected; the ‘Confirm Authorize’ dialog is displayed. Click [Yes] to authorize the requested details or [No] to authorize the requested details in the query mode.

  3. Click [Query] to display all the un-authorized requests.

  4. The un-authorized requests are displayed based on the default criteria set in the SS Parameter for Facility function or enter the search criteria and click [Search] to select a specific record.

    Enter the following search criteria to request the trays:

    1. Doc Type – Select the request document type from the list. The values listed are defined in the function ‘SS Document Type’ for the Transaction Type request group and the default value displayed is specified in the ‘SS Parameter for Facility’ function for request group.

    2. Doc No. – Enter the request document number in numeric format of eight digits maximum length.

    3. Doc Date - Enter the request document date in date format of dd/mm/yyyy or select the date using the Calendar function.

    4. Requested By – Select the requested by store from the list. The stores are listed based on the access rights of the user and applicable transaction.

    5. Requested On - Select the owner store from the list. The stores are listed based on the access rights of the user and applicable transaction.

    6. Request Type - Select the request type from the list. The listed values are Normal, Urgent and Elective.

    7. Entry – Select the value from the list. The values displayed are ‘All’, ‘Completed’ and ‘Not Completed’. The value ‘Completed’ is selected, by default.

  5. Click ‘Doc Type’ hyperlink to authorize the request details. You can modify the request details.

  6. Select the ‘Entry Completed’ check box if the entry is not completed for the requested group type.

  7. Click [Apply] to authorize the requested tray or select the ‘Del’ check box and click [Delete] to delete the requested tray.

  8. On clicking [Apply], the ‘Confirm Authorize’ dialog is displayed. Click [Yes] to authorize the request or [No] cancel the authorization.

  9. On clicking [Delete], the ‘Confirm Delete’ dialog is displayed. Click [Yes] to delete the record or [No] to cancel the action.

An owner store can also request for trays from other owner stores. In the Query mode all the un-authorized requests with entry completed and not completed details are displayed if the sterilized quantity is available in the owner store.
