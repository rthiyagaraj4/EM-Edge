Acknowledgement

From this function you can Acknowledge and Confirm the items after Issue and Issue Return.

This function allows you to record the acknowledgment details of the tray sets received by the receiving store after issue and return transactions. You can acknowledge, reject or keep on hold the items received by the store. Thereby, supports acknowledgement of all the trays/sets that are dispatched to the owner store after sterilization process. You can Query for the existing records.

Acknowledging/Confirming Items

From the SS Transactions > Acknowledgment (Iss/Ret) > select the Acknowledgement function.

The Acknowledgement screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the details in the mandatory (*) fields. <Related Field Description>

  3. Click [Search] to display the specific records based on the entered criteria.

  4. On searching, if the Action Type is 'Acknowledge' the document details are displayed for the selected Store.

  5. To confirm the Issue Group, you must select ‘Confirm’ as Action Type. But you cannot confirm a Return Group.

  6. Click Document Type hyperlink, the Doc type details such as Group Description, Tray No, Expiry Date, Bin Location are displayed at the bottom of the screen.

  7. Click Acknowledge check box for the Document type that you want to acknowledge. The constituent trays are displayed in the bottom of the screen for marking as ‘Acknowledged’, ‘Rejected’ or ‘Pending’.

You must select the Acknowledge check box at least for one document type and click Apply. Else, an error message, ‘Please select at least one document to acknowledge’ will be displayed. From the Acknowledge column, select one of the following options:

  1. Select the Acknowledge value from the list. The listed values are ‘Acknowledge’, ‘Reject’ and ‘Pending’.

    • Acknowledge - Acknowledges the items received by the store. This option is displayed by default.

    • ‘Reject’  - Indicates that the items are rejected by the receiving store. If you select this option, you must enter a reason for rejection in the ‘Remarks’ field.

    • ‘Pending’ - Defers the decision to acknowledge the tray.

  2. Click [Apply];the selected documents in the Document details section as well as the corresponding Sterile Groups and their trays are removed from the display after the changes are committed. However, the rest of the document(s) that are pending acknowledgement remain on the screen even after clicking Apply.

  3. If you have selected ‘Reject’ in the Acknowledge column, click the Ack.Remarks hyperlink to view the acknowledgment remarks.

  4. For Issue Group, click the Tray Discrepancy Remarks hyperlink in the Group details section to view the tray discrepancy remarks recorded through the Tray Discrepancy function in Tray Maintenance; the Tray Discrepancy Remarks is displayed in read-only state in a popup. Click Close to return to the Acknowledge screen.

If there are no remarks available for a document-tray, the corresponding Tray Discrepancy Remarks hyperlink remains disabled. For Return Group, two additional columns Return Remarks and Return Discrepancy Remarks are added to the lower frame when the TRN Type is selected as ‘Return Group’.

  1. Click the Return remarks hyperlink to view the Reason in the Return Remarks popup as read-only text. Click Close to return to the Acknowledge screen.

  2. Click the Return Discrepancy Remarks hyperlink to view the return discrepancy remarks in a read-only popup. If no Return Discrepancy remarks are available for the a document-tray, the Return Discrepancy Remarks hyperlink remains disabled. Click Close to return to the Acknowledge screen.

  3. Click [Apply] to update the tray details in the issuing and receiving stores respectively. An alert message ‘Do you wish to re-issue the rejected item(s)?’ is displayed for rejected items.

  4. Click [Yes] in the alert message, the ‘ Issue’ screen gets displayed to re-issue a new item against the rejected one. The original Issue document detail is displayed and allows selection of new item. No change is allowed for the existing items.

  5. Click [No], the alert message gets closed and the ‘Acknowledge’ screen gets refreshed.

  6. Click [Query] to search for the existing records.
