Allocate/Xmatch/Issue - Blood Units

From this function you can Allocate, Crossmatch, Issue, Return or Revert the blood units and Print unit details report. You can Query to view the existing records.

The Allocate/Xmatch/Issue - Blood Units function is used to allocate, crossmatch or issue the blood units for a patient. You can a return a blood unit after issue if the transfusion is not done. An alert message is displayed, if Blood Typing is not available. You can also revert a blood unit after it is issued and before the transfusion is done. You can view the cross-match results, multi allocation, unit details and can print the unit statistics report. You can allocate the number of units requested through the ‘Request Blood Units’ function or from the Clinical Care Management module. If the specimen blood typing is not confirmed, you cannot allocate the blood unit to a patient. The color code defined at the bottom of the screen describes the status of the specimen selected for the processing. The color codes displayed at the bottom of the screen denotes the following.

– Units issued in emergency

– Autologous Unit

– Stat (to be issued immediately)

– Urgent

– Directed Unit

– Routine

– Multi allocation Unit

Allocate/Xmatch/Issue – Blood Units

From the Transactions > BT Operations, double-click Allocate/Xmatch/Issue – Blood Units function.

The Allocate/Xmatch/Issue – Blood Units screen gets displayed.

  1. Click the [List of Values] to select the number of the blood unit for allocation. Only the  specimens with the blood type of the patient is confirmed are displayed here.

  2. Right click on the link Right Click for User Option to select the user. The link has two pop-ups, namely, Patient Chart and Patient Specimen History. Click Patient Chart option to open the Patient Chart Screen from Clinician Access and Patient Specimen History to open the Lab screen from the Laboratory module.

  3. Click [Execute Query] or press [F8] to view the patient details and requested product details. <Related Field Description>

  4. Click Allocate/ Cross Match/ Issue/ Revert Issue or Return the blood unit. Cross-match tag and Issue tags are printed with the details.  

    Returning Blood Units

    1. Click [Return] to return issued, allocated or reserved blood unit. If the current unit status is allocated, reserved or issued, [Return] is enabled.

    2. Enter the details in the mandatory (*) fields. <Related Field Description>

    3. Click [Save & Close] to return blood unit or [Cancel] to quit the returning process.

    Reverting Blood Units

    1. You can revert the issue of the blood units issued for a patient and not transfused due to any reason. The reverted blood units can be re-reserved for the same patient and not returned to the common stock. The [Revert Issue] gets displayed on clicking the unit number of the issued blood unit

    2. Click [Revert Issue] to revert the issue of the blood units to a patient. <Related Field Description>

    3. Click [Save & close] to revert the issued blood unit the transaction or [Cancel] to cancel the transaction.

    Issuing Blood Units

    1. Click [Issue] to deliver the blood units, by invoking Direct Issue of Compatible Unit. If the current unit status is reserved, [Issue] is enabled.

    2. Enter the details in the mandatory (*) fields. <Related Field Description>

    3. Click [Save & Close] to issue selected blood unit or [Cancel] to quit the process of issuing blood unit.

    Cross Matching Blood Units

    1. Click [Xmatch] to enter crossmatch test results and validate the result by invoking Cross Match Tests. If the current unit status is allocated and the product requires cross match, the [Xmatch] is enabled.

    2. Enter the details in the mandatory (*) fields. <Related Field Description>

    3. Click [Validate] to validate the cross match test results.

    4. Click [OK] to accept the results and return to the Allocate/Xmatch/Issue Blood Units screen or [Cancel] to quit the cross match process.

    Allocating Blood Units

    1. Click [List of Values] to search for the blood unit to be allocated. Double-click the unit number to select the unit for allocation. The system checks the compatibility of the unit based on the setup made in Permissible Blood Type Substitution function. The System displays the error message, if you select an incompatible unit.

    2. Click [Unit Antibodies] to view the antibody details of the unit (if any).

    3. On selecting the unit, press ‘Tab’ to invoke Allocation screen.

    4. Enter the details in the mandatory (*) fields. <Related Field Description>

    5. Click [Save & Close] to allocate the selected unit or [Cancel] to quit the allocation process.

    6. If the selected Blood Unit is reserved for other patient as ‘Autologous/Directed’, you cannot allocate it to another patient. If the required date while requesting the blood unit in Request Blood Units function is crossed, you cannot allocate the blood unit.

  5. Click [Details] to view the patient details.

  6. Click [Antibodies] to view the recorded antibody details of the patient. This button gets enabled if antibodies are associated to the patient.

  7. Click [Transfusion History] to view the transfusion history of the patient (if any).

  8. Click [Requirements] to view the transfusion requirements recorded for the patient. Click [Close] to close the screen.

  9. Click [BB Comments] to view or add the blood bank comments for the patient. On clicking [BB Comments], the ‘Blood Bank Comments’ dialog gets displayed. Select the BB Comment code from the [List of Values] and click [Save]. The Blood Bank Comment codes defined in the Result Notify/BB Comments master appear in the List of Values.

  10. Click [Order Comments] to view the comments entered by the physician against the Blood Transfusion Order in Order Catalog function of Results and Requests module. On clicking [Order Comments] the Order Comments screen is displayed.

    The following details are displayed:

    1. Product Code – The product code of the order is displayed.

    2. Description – The description on the order is displayed.

  11. Click [Financial Details] to view the financial details of the patient. The [Financial Details] gets displayed in Allocate/Xmatch/Issue - Blood Units function if the Billing Module Interface check box is selected in Setup Parameter for Facility function in Blood Transfusion module.

  12. Click [View Xmatch Results] to view cross match test results. Click [OK] to close the Cross Match Tests screen.

  13. Click [View Multi Allocation] to view the units allocated for multiple specimens. Click [Close] to close the screen.

  14. Click [Product/Category Unit Details] to view category and product details of the unit. Click [Close] to close the screen.

  15. Click [Print] to print the report of the unit statistics.

  16. Click [Unit Attributes] to view the Attributes of the patients.

  17. Click [Unit Antigens] to view the Antigens of the patients.

An error message ‘Blood Typing is not Confirmed’ will invoke if try to allocate red blood cell in ‘Allocate/Xmatch/Issue – Blood Units’ function before resulting and confirming red blood cell.

 

The following warning messages gets displayed in Allocate/Xmatch/Issue - Blood Units function.

  • If the entered process does not match the requested process then 'BT0528 Entered Process does not match the Requested Process' warning message gets displayed.

  • If the requested process is not entered then 'BT0529 Requested Process has not been Entered' warning messages gets displayed.

  • If the process done for the selected unit does not match the requested process then 'BT0530 Process done for the Unit does not match the requested Process' warning messages gets displayed.

  • If the Batch has more than one units has process requested then 'BT0531 Cannot Issue by Batch as one or more Units has Process Requested' warning message gets displayed.
