Confirm Blood Typing

From this function you can Confirm the blood group details of a patient. You can Query to view the existing records.

The Confirm Blood Typing function enables you to confirm a patient’s blood group by testing the sample of blood in the laboratory. Blood typing process determines a patient’s Blood Group. The blood unit has to be confirmed before it is provided to the patient against a request. After confirming the blood group, the specimen is available for allocation, crossmatch, issue or transfusion. An additional field called ‘Issued By’ is added in the Component Tag that is printed after confirming blood typing.  You cannot process records, which are cancelled. The priority of specimen process is highlighted using the following colors.

- Stat

- Urgent

- Routine

The Blood Typing, which is confirmed already, cannot be configured. The number inside the color box indicates the number of available specimens registered for transfusion.

Blood Typing Confirmation

From the Transactions > BT Operations, double-click Confirm Blood Typing function.

The Confirm Blood Typing screen gets displayed.

  1. Click [List of Values] to select the specimen to authenticate the blood typing of a patient. Only the specimens requested are listed in the list.  The color code associated to the specimen number defines the priority. The color codes are described at the bottom of the screen.

  2. Click [Execute Query] to view the patient details and the product details. <Related Field Description>  

  3. Right click on the link Right Click for User Option to select the user. The link has two pop-ups, namely, Patient Chart and Patient Specimen History. Click Patient Chart option to open the Patient Chart Screen from Clinician Access and Patient Specimen History to open the Lab screen from the Laboratory module.

  4. Click [Details] to view the patient details by invoking the Patient Details screen. Click [Close] to return to the Request Blood Units screen.

  5. Click [Antibodies] to view the antibodies associated to the patient. Click [Close] to return to the Request Blood Units screen. The [Antibodies] is enabled, only if the antibodies are recorded for the selected specimen.

  6. Click [Transfusion History] to view the previous transfusion details of the patient, if any.

  7. Click [Requirements] to view the existing requirements and associate additional transfusion requirements for the patient. Click [Close] to return to the Request Blood Units screen.

    Enter the following details:

    1. Code – Select the requirement Code from the list of values. The list displays the values defined in the Transfusion Requirements function.

    2. Description – The description of the Requirement Code is displayed.

    The system verifies the selected requirements during allocation/issue of blood units in the Allocate/Xmatch/Issue - Blood Units function.

  8. Click [BB Comments] to view or add the blood bank comments for the patient. On clicking [BB Comments], the ‘Blood Bank Comments’ dialog gets displayed. Select the BB Comment code from the [List of Values] and click [Save]. The Blood Bank Comment codes defined in the Result Notify/BB Comments master appear in the List of Values.

  9. Click [Confirm] to validate the specimen and patient blood type details or [Clear Records] to cancel the process. A component tag is printed with the details.
