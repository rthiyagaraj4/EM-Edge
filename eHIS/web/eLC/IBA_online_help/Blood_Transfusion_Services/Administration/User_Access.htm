User Access

From this function you can Assign rights to the user to confirm the post transfusion remarks.

This function allows you to assign access rights to the user to confirm the post transfusion remarks in BT module.

The User Access function does not allow you to delete any records. In addition, duplicate entries are not allowed. The disabled users are highlighted in grey background. You can only query or view their details but cannot update the data.

Defining User Access

From the Administration menu, double-click the User Access function.

The User Access screen gets displayed.

  1. Select the required Facility name from the list. The logged in user Facility name gets displayed by default. This field is mandatory.

  2. Enter the required User name using [?] function. The existing users name belonging to the selected facility gets listed. This field is mandatory. The disabled user name is highlighted in gray.

  3. Select the ‘Post Transfusion Remarks Confirmation Access Y/N’ check box against the user to give access rights to the user confirm the post transfusion remarks.

  4. Click [Commit] or press [F10] to save the defined access rights or click [Clear Record] to clear the record.

  5. Click [Execute Query] or press [F8] to search for the existing records.   

 

Once User IDs are entered in the function and saved in the database, they cannot be updated or deleted. Only the Post Transfusion Remarks Confirmation Access Y/N check box can be selected or cleared if required.
