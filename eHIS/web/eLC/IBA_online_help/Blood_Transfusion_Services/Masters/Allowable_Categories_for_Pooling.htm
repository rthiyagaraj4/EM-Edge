Allowable Categories for Pooling

From this function you can Associate the allowable Product Categories that can be pooled to create a new Product and Query for the existing records.

The Allowable Categories for pooling function enables you to associate the product(s) that can be pooled to create a new product. Pooling is the process of combining the blood components of the same Blood Group and Rhesus to create a larger volume of blood unit, after performing the necessary tests on a unit. The values defined in this function are used in the ‘Pooled Units’ transactions function. You cannot create duplicate entries while creating Allowable Categories for pooling.

Allowable Categories for Pooling

From the Masters, double-click the Allowable Categories for Pooling function.

The Allowable Categories for Pooling screen gets displayed.

  1. Click [List of Values] to select a Product Category. The list displays the values defined through the ‘Product Categories’ function. The description of the selected Product Category is displayed in the adjacent field.

  2. Click [List of Values] to select a Product Category allowed for pooling for the selected category. The products selected must be of the same category, otherwise pooling cannot be done successfully. For example, you cannot select an RBC product and Platelet product for pooling. The list displays the values defined through the ‘Product Categories’ function. The description of the selected Product Category is displayed in the adjacent field.

  3. Click [Commit] or press [F10] to save the records or click [Clear Record] to clear the entered details.

  4. Click [Execute Query] or press [F8] to search for the existing records.
