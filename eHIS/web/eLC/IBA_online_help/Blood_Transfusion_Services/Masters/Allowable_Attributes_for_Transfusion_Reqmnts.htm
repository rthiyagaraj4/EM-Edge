Allowable Attributes for Transfusion Requirements

From this function you can Associate the acceptable attributes for a selected transfusion requirement and Query for the existing records.

The Allowable Attributes for transfusion requirements function is used to associate the acceptable attribute(s) for a selected tra nsfusion requirement.  The Transfusion Requirements are defined in the Allocate/Xmatch/Issue - Blood Units function. The values created in Transfusion Requirements function is associated with the attributes created in the Blood Unit Attributes function. The attributes are instruments required during transfusion of blood units to a patient.

Associating Allowable Attributes to Transfusion Requirements

From the Masters, double-click Allowable Attributes for Transfusion Requirements function.

The Allowable Attributes for Transfusion Requirements screen gets displayed.

  1. Click [List of Values] to select a Transfusion Requirement Code. The list displays the values defined in the Transfusion Requirements function. The description of the selected Requirement Transfusion is displayed in the adjacent field.

  2. Click [List of Values] to select the attributes. The list displays the values defined in the ‘Blood Unit Attributes’ function. The selected attributes (instruments) are required during transfusion of blood units to a patient.

  3. Click [Commit] or press [F10] to save the created records or click [Clear Record] to clear the entered details.

  4. Click [Execute Query] or press [F8] to search for the existing records.
