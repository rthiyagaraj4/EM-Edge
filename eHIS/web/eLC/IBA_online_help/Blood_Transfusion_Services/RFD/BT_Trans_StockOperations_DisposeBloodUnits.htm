Enter the following details:

 

  1. Product Code – Click [List of Values] to select the Product Code. The list displays the values defined in Product Codes function.

  2. Unit Number – Enter the Unit Number in alphanumeric format of 20 characters maximum length.

  3. Blood Group – Click [List of Values] to select the Blood Group. The list displays the values defined in the Blood Groups function in the Medical Records module.

  4. Rhesus – Click [List of Values] to select the Rhesus factor. The list displays the values defined in Rhesus factor function of the Medical Records module.

  5. Donation Date – The Donation Date of the selected unit is displayed. The Donation Date is recorded in the Record Donation function in Blood Donor function.

  6. Date Of Expiry – The calculated expiry period of the unit based on the expiry period of the product defined in the Product Codes function is displayed.

  7. Volume – Enter the Volume in numeric format of three digits maximum length to search for the unit details.

  8. Collection/Supplier – Click [List of Values] to select the Collection/Supplier code from the list of values. The list displays the values defined in the Collection /Supplier function.

  9. Antibodies – Click [List of Values] to select the Antibody code from the list of values to search for the unit details. The list displays the values defined in Antibodies function.

  10. Donation Type – Select the Donation Type from the list. The valid values are ‘Autologous’, ‘Allogenic’ and ‘Allogenic-Directed’. The values are recorded in Pre Donation function in Blood Donor function.

  11. Patient ID – The Patient ID associated with the unit is displayed. The patient ID is associated to the blood unit while requesting the blood units in the Request Blood Units function.

  12. Storage Location – Click [List of Values] to select the storage location of the associated unit. The storage location is created in the Storage Location function.

  13. Unit Status – The Unit Status is displayed. The status notifies whether the unit is disposed, outstanding, used in any transaction, etc. If the unit is used in any of the transactions, you cannot dispose of the unit. The status of the blood unit is based on the transactions done in the Allocate/Xmatch/Issue - Blood Units, Transfusion of Blood Units functions.

  14. Last Reserved Date – The last reserved date, that is the last time the blood unit was crossmatched for a particular patient is displayed.

 

Reserved for

  1. Patient ID – The Patient ID associated with the unit, if the unit is used in any of the transactions is displayed. If multiple patients associated to the same unit the details of the last association is displayed.

  2. Specimen No – The Specimen number of the unit is displayed, if the unit has been used in transactions such as Allocate/Xmatch/Issue - Blood Units, Transfusion of Blood Units, etc.
