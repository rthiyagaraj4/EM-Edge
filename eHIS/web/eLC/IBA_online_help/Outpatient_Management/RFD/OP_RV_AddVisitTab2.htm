Enter the following details:

 

  1. Visit Date Time - The visit date time of the patient is displayed.

  2. Location - Select the Location from the list. This field represents the Outpatient Location to which the Patient has to be registered. All the locations of type 'Clinic' with level of care set to ambulatory and attached to the operator station is displayed. In case of an Appointment Patient, the Location is displayed from the Appointment details and cannot be modified. If an age group is associated with the location, the validity of the location is verified based on the age and gender of the patient. In case of Invited waitlist patients, the Location Type and Location is displayed from the waitlist details. The Location Type cannot be modified. You can select another location using the [?] function, if required. The locations of the Specialty specified while creating the waitlist is listed. This field is mandatory.

  3. Room - Select the room against which the visit has to be registered from the list. All the rooms associated to the selected location in 'Resource for Location' function is listed.

  4. Service - Select the service against which the visit is to be registered from the list. All the services associated to the selected location is displayed. The Service associated to the Location and Location Type is displayed by default, when a single service is associated to the Location and Location Type combination. This is a mandatory field.

  5. Sub-Service - Select the Sub-Service against which the visit is to be registered from the list. The list gets populated if any sub services are attached to the selected service. This field is optional.  

  6. Visit Type - Select the Visit Type from the list of valid values. This field represents the Visit Type against which the patient is registered. The displayed list is limited to the Visit Types associated to the selected Location. In case of an Appointment Patient, the Visit Type is made defaulted from the Appointment details and cannot be modified. The Visit Type associated to the Location and Location Type is displayed by default, when a single visit type is associated to the Location and Location Type combination. The Visit Type values for a follow up visit type are based on the 'Override Visit Type Control on Episode' setup in OP Parameter. This is a mandatory field.

  7. Practitioner - Enter the practitioner name in alphanumeric format of 30 characters maximum length or click [?] function to search for the required practitioner. This field represents the Practitioner who provides consultation to the patient during the visit. The displayed list is based on various parameters set up already. If the selected location requires a practitioner as a primary resource and is open to all Resources, the list of practitioners who are associated to the logged in facility and to the specialty of the selected clinic form part of the displayed list. If the selected location is not open to all Resources, the list of Practitioners who are associated to the selected location in Practitioner for clinic form part of the displayed list. In case of an Appointment Patient, the Practitioner is made defaulted from the Appointment details and it cannot be modified. If a Practitioner has to be identified while registering a visit to the selected Location, based on the parameter set up of the Location, this field is mandatory, or this field is optional.

 

Note: When you select an appointment number and register the visit is registered, an icon  will be displayed here, if a secondary resource of type “Practitioner” had been captured while booking the appointment.

When you re-assign practitioner for this visit, the practitioner selected during appointment booking will be displayed by default, disabled and you will not be able to change it.

 

  1. This function will come in handy when you want to assign multiple practitioners for the same appointment (group consultations.)

  2. Order Catalog - Enter the order catalog name in alphanumeric format of 40 characters maximum length or click [?] function to search for the required catalog and select orders. This field represents the order that is required to be placed upon registration of the patient's visit. The displayed list is based on order catalogs associated with the selected location.

Note: If orders have already been selected while the appointment was booked, the Order Catalog lookup will be disabled. You can only view the details of the orders already selected. The selected orders are displayed with their code and description.

  1. Other Resource Type - Select the other resource type from the list. According to the value selected, the corresponding values get populated in the [?] function. These values are populated from the Application Masters module.

  2. Reason - Select the Reason for Contact from the list. All the reason codes defined with applicability for Visit registration checked in Application Masters module are listed. In case of an Appointment Patient, the Reason for Contact is made defaulted from the Appointment details, entered when booking an appointment.
