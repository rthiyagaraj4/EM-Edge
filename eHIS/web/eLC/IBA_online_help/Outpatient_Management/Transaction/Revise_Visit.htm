Revise Visit

From this function you can Revise the Visit Registration details.

Visit registration against a clinic may be amended due to various reasons such as non-availability of resources. This function allows revising a visit within a specified time limit. Revision of visit details includes amending location, practitioner or visit type. Reallocating resources to manage patients on the same day is possible using this function. Revise Visit for a selected record fails, if there is no visit status selected at start consultation, if there are active orders or any prescriptions/dispensing documents for visits and entries into the Medical Records for the visit.

 

Hospital and Professional Charges are displayed on recording the financial details of a patient. The patient payable amount towards Hospital charge as well as Doctor’s charge is displayed. Professional Charge is displayed along with the Hospital Charge, only if billing details and visit registration rules are defined for the Clinic. The doctor’s charge has to be a part of the panel, if Professional charge has to be set as zero. If the Professional Charge is set as a Service, you cannot set the doctor’s fee as zero. You can instead associate the Hospital Charges to make up the Professional fees, when the doctor does not charge the patient for the services.

 

The ‘Consultation Started’ queue status of the patient is displayed in the search criteria. The visit can be revised until the consultation is closed. If the consultation is in progress for the patient on click of Encounter ID, you are alerted by a message ‘Consultation in Progress Only Referral Details are allowed to change’.

Revising a Visit

From the OP Transactions, select the Revise Visit function.

The Revise Visit screen gets displayed.

  1. Click [Search] to display all the patients with visits registered or enter the search criteria and click [Search] to select specific patients. Click [Clear] to reset the records displayed in the result area.

  2. On searching, the Visit details of the patient are displayed based on the entered criteria.

    1. Encounter ID – The Encounter Identification number generated at the time of registration is displayed.

    2. Regn Date/Time – Registered date and time of the visit is displayed with a hyperlink.

    3. Patient Details – The details of the patient such as Patient Name, Patient ID and date of birth are displayed.

    4. Location – The Location of the Visit is displayed.

    5. Practitioner Name – The Name of the Practitioner associated with the Visit is displayed.

    6. Queue Status - The Queue Status of the visit is displayed.

  3. Click Encounter ID hyperlink to display the Revise Visit screen. The screen displays the Queue Status and Current Visit Details.

  4. Enter the revise visit details and reason for revision to revise a particular visit.

    Encounter Details

    1. Location - The Location to which the Patient is registered for this visit and the corresponding Care Location Type is displayed.

    2. Room – Select the room from the list of valid values.

    3. Visit Type – The Visit Type and its indicator are displayed.

    4. Service – The Service associated to the Visit is displayed.

    5. Sub Service – The Sub service for the selected Service associated to the visit is displayed. This field is left blank if no Sub-service is selected.

    6. Resource – If the Primary Resource has to be reassigned, enter/select the resource from the list of valid values using [?] function.

    7. Resource Type – The Practitioner Type associated to the Practitioner against whom the visit is revised is displayed.

    8. Referral ID – Enter the Referral ID as generated during referral registration or click [?] function to perform Referral Patient Search to locate the referral patient details. The referrals are listed based on the selected location and Patient ID (if any). You can register new referrals only for the selected Location and Patient ID and associate to the selected visit.

    9. Reason for Contact - This field displays the Reason for Contact that was recorded during visit registration.

    10. Reason for revise visit – Select the reason for revising the visit from the list of valid values. The reasons defined with revise visit check box selected in ‘Other Master Setup (Reason)’ of Application Master module are displayed.

  5. Click [Apply] to revise a visit and display the Visit Charge Details screen or [Reset] to clear the entered data.

 

Visits are revised only if Current date is within the allowed number of days relative to the visit date as per the OP Parameter setup. Visit status has to be ‘Checked In’ or ‘At Nursing Station’. No entries should be there in Medical Records for the visit. If Order Entry and Result Reporting module is operational, there cannot be any active orders for the visit and similarly If Pharmacy Management module is operational, there should not be any prescriptions/dispensing documents for the visit.

You cannot revise visit for order catalog related packages.

You have to cancel the package that is associated to the visit in order to revise the visit.
