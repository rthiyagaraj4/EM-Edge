Enter the following details to create a Database Resource Group:

 

  1. ID – Enter the Resource Group ID in alphanumeric format of 20 characters maximum length. This is a mandatory field.

The starting character of a Resource Group ID has to be a character and cannot be a numeric value.

  1. Name – Enter the Resource Group Name in alphanumeric format of 30 characters maximum length.

 

Password Control

  1. Expiration Period – Select the option for Expiration Period as ‘Days’ or ‘None’. On selection of an option as ‘None’, the ‘Days’ and ‘Grace Time’ legends are disabled.

  2. Days - Enter the number of days in numeric format of three digits maximum length when the ‘Days’ option is selected.

  3. Grace Time – Enter the number of days in numeric format of three digits maximum length as a grace time above an expiration period.

 

Reuse Option

  1. Password Expiry Alert – This text box will be blank and disabled when the ‘Expiration Period’ option is selected as ‘None’.  When the ‘Expiration Period’ option is selected as ‘Days’, this field becomes mandatory and displays the value ‘14’ by default. You can change the value if required. You can enter value from 1 to 999. But you cannot enter the value as ‘zero’ or blank.

  2. Reuse Option – Select a Reuse Option from the list - ‘Prohibit Reuse’ or ‘Reuse’. Select the option ‘Prohibit Reuse’ to prevent the reuse of the same password. On selecting ‘Reuse’, the password can be re-used. The 'Reuse After' field is enabled, if the 'Reuse Option' is selected as 'Reuse'.

  3. Reuse After – On selecting ‘Reuse’ in Reuse Option, ‘Reuse After’ field is enabled. Enter the number of days or the number of changes in numeric format of three digits maximum length. The number of days indicates the reuse time above an expiration period. The number of changes indicates the number of modifications after which the same password can be re-used.

  4. Failed Login Attempts – Enter the Number of Failed Login attempts in numeric format of three digits maximum length. This value indicates the number of failed attempts to login with incorrect password, before the system is locked.

  5. Password Lock Period – Enter the number of days in numeric format of three digits maximum length for which the password is to be locked, if the Login attempts exceed the ‘Failed Login Attempts’.
