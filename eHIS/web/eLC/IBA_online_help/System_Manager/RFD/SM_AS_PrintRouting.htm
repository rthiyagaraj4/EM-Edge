Enter the following details to create a Print Routing record:

 

  1. Module - Select a Module from the list of installed modules. When the Module is selected, all the reports of the selected module are listed in ‘Report’. This is a mandatory field.

  2. Report - Select the report for which the print routing rule has to be defined. If the selected Report is facility based, facilities of the enterprise get listed in ‘Facility’. If the selected report is transaction based, the destination types are listed in ‘Destination’ depending on the report. For example, if the report is transaction based and belongs to Outpatient Management module, the destination is populated as ‘Clinic’, ‘Location’, ‘Procedure Unit’ and ‘Daycare Unit’ . The Default Report Mode of the print routing rule differs based on the Report selected. The list of printers to which the output has to be routed gets populated. This is a mandatory field.

  3. Facility - This field is enabled only if the report is defined as facility based. Select a facility from the list to define the Print routing rule. The facility is displayed only if it is attached to a Report in Facility Based Report Parameters function.  This is a mandatory field.

  4. Destination - This is a mandatory field. Based on the report for which the print routing rule is being defined, the first part of the destination differs based on the Report selected:

  5. If the Report is Transaction and Facility based, ‘Clinic’, ‘Nursing Unit’, ‘Pharmacy’, ‘Store’, ‘Procedure Unit’ and ‘Daycare Unit’ gets listed for Destination.

  6. If the Report is not Transaction Based and only Facility based, ‘Location’ gets listed for Destination.

  7. If the Report is not Transaction Based and Facility based, ‘Location’ gets listed for Destination.

  8. If the Report is Transaction based and not Facility based, ‘Clinic’, ‘Nursing Unit’, ‘Pharmacy’, ‘Store’, ‘Procedure Unit’ and ‘Daycare Unit’ gets listed for Destination.

  9. Select ‘Permanent FS Location’ from the list. The Permanent FS Locations defined in FM Code Setup is displayed in the adjacent field. The report for the selected function is routed based on this setup.

The Facility Based Report Parameters has to be defined, when the report is Facility based. This is a mandatory field.

a. When the destination is selected as ‘Location’, the list of Locations gets listed. Select a Location from the list or select ‘*All’ to define Print Routing rule for all the Locations.

b. When the destination is selected as ‘Clinic’, the list of clinics (limited to the selected facility, for facility based reports) gets listed adjacent to the Destination. Select a Clinic from the list or select ‘*All’ to define Print Routing rule for all the Clinics.

c. When the destination is selected as ‘Nursing Unit’, the list of nursing units (limited to the selected facility, for facility based reports) gets listed. Select a Nursing Unit from the list or select ‘*All’ to define Print Routing for all the Nursing Units.

d. When destination is selected as ‘Pharmacy’, the list of Pharmacies (limited to the selected facility, for facility based reports) gets listed. Select a Pharmacy location from the list or select ‘*All’ to define Print Routing for all the Pharmacy locations.

e. When the destination is selected as ‘Store’, the list of stores (limited to the selected facility, for facility based reports) gets listed.  Select a Store from the list or select ‘*All’ to define Print Routing for all the Stores.

f. When the destination is selected as ‘Daycare Unit’, the list of daycare units (limited to the selected facility, for facility based reports) gets listed. Select a Daycare Unit from the list or select ‘*All’ to define Print Routing for all the Daycare Units associated to the Facility.

g. When the destination is selected as ‘Procedure Unit’, the valid list of Procedure Units (limited to the selected facility, for facility based reports) gets listed. Select a Procedure Unit from the list or select ‘*All’ to define Print Routing for all the Procedure Units associated to the Facility.

  1. Workstation - Enter the IP address of the client as the workstation identity. The Workstation identity is necessary, if the print routing rule has to be defined based on the workstation (the IP address of the client’s machine from which the application is being used).

  2. Default Report Mode – This field is enabled and the values get listed based on the Report selected. The Default Report mode is displayed based on the Report selection,  which can be changed to the other mode (Bitmap to Character or Character to Bitmap) depending on the printer to which it has to be routed. The current version of the product supports only the Bitmapped reports. This is a mandatory field .

  3. Number of Copies - On selection of the report, this field is made default to one, which can be changed to any required number. When the report is generated based on this rule, number of copies specified is generated. This is a mandatory field.

Printer Details

  1. Printer 1, 2, 3 - Select the printer from the list to which the report is to be routed. Printer1 field is mandatory.

  2. From Time - Enter the time in hh:mm format to route the report to the printer from the time specified. The time specified has to be lesser than 24:00 hours.
