Enter the following details to maintain a Site:

 

  1. Site ID – The Site ID is created and fed through the installation scripts of Enterprise Management. This ID cannot be modified. This is a mandatory field.

  2. GSTIN – Enter the GSTIN value in alpha numeric format with maximum 20 characters in length. Special characters are not allowed. The value must be greater than zero. This field is mandatory. This field is applicable only to customers in India.

  3. Site Name - Enter a new site name if required in alphanumeric format of 30 characters maximum length. This is a mandatory field.

  4. Line 1,2,3,4 - Enter the values in alphanumeric format of 15 characters maximum length. The legends to be displayed for Address lines in all other modules wherever required are specified. Examples of prompt are street, locality, road, etc.

  5. Town - Enter the value used as a prompt to capture the Town details of a patient in alphanumeric format of 15 characters maximum length. The field name specified gets displayed in the transactions of external modules while capturing the Town details of the patient.  

  6. Area - Enter the value used as a prompt to capture the Area details of a patient in alphanumeric format of 15 characters maximum length. The field name specified gets displayed in the transactions of external modules while capturing the Area details of the patient.  

  7. Region - Enter the value used as a prompt to capture the Region details of a patient in alphanumeric format of 15 characters maximum length. The field name specified gets displayed in the transactions of external modules while capturing the Region details of the patient.

  8. Postal Code - Enter the value used as a prompt to capture the Postal Code details of a patient in alphanumeric format of 15 characters maximum length. The field name specified gets displayed in the transactions of external modules while capturing the Postal Code details of the patient.

  9. Applicable -  Select the check box to enable the record in the transaction functions. Clear the check box, if the selected field is not to be displayed in the transaction functions. The check box is selected, by default.

  10. Site Logo (Login Screen) - Click [Browse] to select the Site Logo. The Site Logo is displayed in the Login screen.

  11. Postal Code Link Required - Select the check box to link postal code to the area, town and region to automatically display the area, town and region while entering the postal code in the transaction functions. The area, town, region and postal code is  defined in 'MP Code Setup' in Master Patient Index module. If 'Postal Code Link Required' check box is selected, while creating an Area in 'MP Code Setup', associating a Region becomes mandatory. If 'Postal Code Link Required' check box is selected, while creating a Town in 'MP Code Setup', associating an Area becomes mandatory. Clear the check box to de-link the area, town and region.

The Postal Code Link has impact in the following modules:

2) Change Patient Details

2) Revise Attendance Details

Terminology & Coding Management

1) Requestor

  1. Display User Name in Application Header - Select the check box to display the User Name in the Application Header. Clear the check box to display the User ID in the Application Header.

  2. Change Password during First Login – By default the check box is cleared and enabled. If you select the check box, the system prompts you to change the password if you are logging in for the first time or if you are logging in for the first time after the password is reset using the ‘User’ function in System Manager module.

  3. Change Password Mandatory – This check box becomes enabled only when the ‘Change Password during First Login’ check box is selected. If you select the check box, it is mandatory to the change the password. In case, if you select the check box and click [Cancel] in the ‘Change Password’ pop up, the system will not allow you to login.  

  4. Case Sensitive Password – By default, this check box is cleared and disabled. You can enable it only through back end. Select this check box to enforce case sensitive functionality for the password. For example, if the entered password is (mahesh11$), the data gets saved in the DB as ‘mahesh11$’. If there is any mismatch, an error message ‘Invalid User ID/Password is displayed. When this check box is cleared, the entered password always gets saved as Uppercase in the DB, For example, if entered password is (mahesh11$), the data gets saved in the DB as ‘MAHESH11$’. The system will check for password irrespective of case sensitive.

  5. Enforce Password Policy – Select this check box to enforce the Password Policy. When this check box is selected, a new text box ‘Allowed Special Characters in Password’ and the ‘Password Policy’ legend gets displayed. In addition, the Password Policy legend gets displayed in the ‘User’ screen in both Create and Query mode as well as in the ‘Change Password’ screen under Personalize Options.

  6. Allowed Special Characters in Password – When this ‘Enforce Password Policy’ check box is selected, this text box gets defaulted with special characters and disabled.

  7. Multiple Login Allowed – Select the check box to allow multiple user login at the same time i.e., same user credentials can be used by multiple users. By default, this check box is selected. If you clear the check box a new text field ‘Unlock User Session After X Hours’ gets displayed.

If Multiple Login Allowed check box is cleared in Site level and if the user ‘A’ is logged in the application and if another user ‘B’ tries to login with A credentials, the system prompts an error message ‘User already logged in the system. Cannot login again’.

  1. Unlock User Session After X Hours – – Enter the number of hours after which the logged in user get listed in the Release Lock User session. You can enter up to two numeric digits. The value ‘0’ is allowed. This field is not mandatory.
