Database Roles

From this function you can Define the Database Roles at the enterprise level.

Database Roles function allows you to change the password of a specific Database Role. The database roles are created through the installation scripts of Enterprise Management with the ID, Role Name and Password. The three database roles defined are ‘Medstartup’, ‘Medadmin’ and ‘Medusers’. The pre-defined roles are associated to the responsibility.

Defining Database Roles

From the SM Admin Setup > Application Security, select the Database Roles function.

The Database Roles screen gets displayed.

  1. Select the check box to change password for a selected role. <Related Field Description>

  2. On selecting the check box against a role, the field to enter the new password gets displayed. Enter the details in the mandatory (*) fields. <Related Field Description>

  3. Click [OK] to save the new password or [Cancel] to clear the entered data.

 

The minimum length of password has to be 5 characters. Password has to begin with a character and not a numeric value.
