Construct Menu

From this function you can Construct a list of submenus for a Main Menu at the facility level.

As a part of setting up the menu, the submenus and root menus are constructed using this function. Attaching of functions is termed as construction. A submenu is constructed by selecting a set of functions from the standard set of functions supplied as part of the product. It is possible to assemble a set of functions from different modules of the product. While forming the submenus, ensure to assemble the selected functions in the required order of appearance for final usage.

 

Other submenus also can be assembled as part of a submenu. A combination of submenus and functions from different modules can form part of a submenu. The final submenu gets displayed in the assembled order of included submenus and functions. When other submenus are included, the whole structure of the submenu is automatically inherited at the time of using the menu. The construction of root menus is exactly same as the submenu construction except that other root menus cannot be included as part of a root menu. Root menus only can be associated with a responsibility.

Constructing a Submenu for a Main Menu

From the SM Admin Setup > Application Security, select the Construct Menu function.

The Construct Menu screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Select a Menu from the list. This is a mandatory field. <Related Field Description>

  3. Select a Module to list the function names and to construct a submenu.

  4. Click [Apply] to save a constructed submenu or [Reset] to clear the entered data.

 

Provision is available in this function to preview the constructed menu.
