Release Locked User Session

From this function you can Release the locked user session i.e., users for whom session got terminated unexpectedly.

This function lists the users who has logged in to the application after crossing the hours defined in the ‘Unlock User Session After X Hours’ parameter in Site screen. It allows you to release the locked users i.e., the users for whom session got terminated unexpectedly.

 

If the user close/logoff the application in proper way, then system would not display those user IDs in the ‘Release Locked User Session’ screen. After close/logoff the application, the system allows the user to re-login the application without displaying any error message.

 

If the application logged in by the user, unfortunately exceptional shutdown/forced log off happen for a user ID then this user ID gets displayed in ‘Release Locked User Session’ screen. After this if the user tries to re-login the application with the same credential then system displays an error message ‘User already logged in the system. Cannot login again’. You need to release the locked user.

 

On clicking the Release Locked User Session menu, the system displays an alert message ‘Multi User login allowed at site level. Cannot access this function’ if the ‘Multiple Login Allowed’ check box is selected in Site level.

Releasing Locked User Session

From the SM Admin Setup > Application Security, select the Release Locked User Session.

The Releasing Locked User Session screen gets displayed.

  1. By default, the list of all the users logged in the application after crossing the hours defined in the ‘Unlock User Session After X Hours’ parameter is displayed. If hour is not defined in the parameter, the system lists of all the logged in users based on the system date/time.

  2. Select the required ‘User ID’ from the lookup. The system displays the selected User details such as User ID, User Name, Login Date Time, WorkStation along with a Delete check box against the record.

  3. To delete the record, select the ‘Delete’ check box against the record and click [Apply]. The deleted user record gets removed from the list.

  4. The system allows the user to re-log into the application using the same user ID.

If the user is logged in 2T, the same user would not be allowed to login in 3T application. Similarly, if the user is logged in 3T, then same user would not be allowed to login 2T.
