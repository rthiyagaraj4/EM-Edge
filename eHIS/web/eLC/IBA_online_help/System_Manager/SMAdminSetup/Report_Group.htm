Report Group

From this function you can Create a new Report Group of Enterprise Management and Query for a desired Report Group.

Report Group serves the purpose of determining the printer to which the report has to be routed and the number of copies of the report to be printed, in case detailed routing records of a report have not been set up. Report Group is an entirely optional setup. All the reports of Enterprise Management can be classified into one of the report groups that are set up through this function.

Report Group is associated to a report directly in case the report is not facility based and the Report Group can be associated with a report/facility, in case the report is facility based. The system administrator follows the steps given below while setting up the routing mechanism of the reports:

• If for a particular report, rule based print routing is to be followed, all the detailed print routing records are set up.

• If either the rule based print routing is not to be followed or it is not possible to set up all the print routing rules in a detailed way, the Report Group has to be associated to the report.

Depending on the type of the report, the default printer on which the report has to be printed is decided. For example, if the report is ‘Visit Label’, the default printer is decided as the Zebra label printer placed in the registration desk.

The report group for which the required Zebra label printer is set up as the default printer is selected. The report group could be ‘Registration Labels’. If the required Report Group is not present, a new report group is created and the desired printer is set up as the default printer. The default number of copies of the print out has to be set as a common number, say, 1.

The Report Group, say, ‘Registration Labels’ is associated to the report (say, Visit Label) directly, in case, the report is not facility based. Or, the Report Group is associated to the combination of the report/facility, if the report is facility based.

Depending on the requirements, the different Report Groups required to be defined are to be derived. It is possible to define report groups at a higher level like, ‘Online labels’, ‘Online Reports – A4’, ‘Online Reports – A3’, ‘Batch Labels’, ‘Batch Reports – A4’, ‘Batch Reports – A3’, etc. It is also possible to define report groups at a detailed level, like, ‘Registration Labels’, ‘Registration Stats’, ‘Appointment Slips’, etc.

Based on the routing definitions, the system derives the printer to which an output is to be routed and the number of copies to be generated as follows:

• The presence of detailed routing records is selected with the required input parameters.

• If a record is found, the printer and number of copies are derived from that record.

• If a record is not found, Report Group of the report is picked up, in case of non-facility based report or the report group is picked from the facility based parameters record of the report/facility, in case of facility based report.

• In this case, if the report group is not defined at the level of the report (non facility based), or at the level of report/facility (facility based), an error is raised.

• If the report group is derived, the default printer and the number of copies to be printed are derived from the report group record.

Creating a Report Group

From the SM Admin Setup > Print Routing, select the Report Group function.

The Report Group screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the details in the mandatory (*) fields. <Related Field Description>

  3. Click [Apply] to save a Report Group or [Reset] to clear the entered data.

  4. Click [Query] to search for an existing Report Group.
