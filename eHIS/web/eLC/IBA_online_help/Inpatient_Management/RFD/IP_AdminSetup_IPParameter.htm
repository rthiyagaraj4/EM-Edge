Enter the following details under the ADT tab:

 

Booking

  1. Allow Booking For Unregistered Patients - Select the check box to allow booking for admission with/without bed for unregistered patients, that is, create a booking without specifying a Patient ID. If the check box is cleared, the booking can be made only for registered patients. The check box is cleared, by default.

  2. Grace Period For Revision Of Booking () Days – Enter the number of days allowed to revise the booking in numeric format of three digits maximum length. The facility can revise the booking for admission within the number of days specified after the preferred admission date has elapsed.

  3. Max Allowable Period For Advance Bed Booking () Days - Enter the number of days allowed to perform advance booking in numeric format of three digits maximum length. It specifies the number of days the booking can be performed in advance. While  booking a admission, if you specify a preferred admission date greater than the date calculated based on this parameter setting, the system alerts you with an appropriate message. This field is mandatory.

  4. Check Booking Limit Based on – Select one of the values from the list. The listed values are ‘Nursing Unit – Bed Class’ and ‘Age Group – Bed Class’. This field is used to indicate the basis for validating booking limits for bed bookings.

  5. Allowed Booking – Select an option from the list, as either ‘Direct’ or ‘Referral’. Selecting ‘Direct’, indicates that booking is allowed for patients without Referral and selecting ‘Referral’ indicates that booking is allowed only for Referral patients.

  6. Block Bed based on – Select one of the values from the list. This field is used to indicate the basis for validating time period for bed blockings. The listed values are ‘Preferred Admission Date/Time’ and ‘Expected Discharge Date Time’.

  7. Multiple Booking for Patients - Select the check box if multiple bookings are to be performed with the same Patient ID in ‘Create booking’ function.

  8. Capture Financial Details - Select the check box to capture the Financial Details in ‘Create Booking’ function. The check box is visible only if Patient Billing Manager module is installed.

  9. Display No Show Status in Confirm booking after – Enter the numeric value of maximum three characters length, it can be blank or greater than zero. If the patient has not turned after the minutes entered here, the booking status of the patient is displayed as ‘No Show’ in Confirm/Cancel/Revise Booking screen.

  10. Released Booked Bed if Admitted into Different Bed – Select the check box to release the booked bed if the confirmed booking patient gets admitted in a different Bed number. The released bed gets displayed in the Bed availability chart.

 

Admission/Discharge/Transfer

  1. Max Allowable Backdated Admission – Enter the Hours/Days in numeric format of three digits maximum length to indicate the time that is allowed for backdating an admission. This field is mandatory.

  2. Allowed Admission – Select an option from the list to indicate the types of admission allowed. The listed values are ‘Referral’, ‘Booking’, ‘Direct’ and ‘Booking or Referral’. Selecting ‘Referral’ indicates admission is allowed only for referral patients. On selecting ‘Booking’, admission is allowed for patients who have made prior booking for admission. The selection of ‘Direct’ indicates that the admission is allowed for patients without booking or those who are not referral patients and selecting ‘Booking or Referral’ indicates admission is allowed only for patients with prior booking or those who are referral patients.

  3. Invoke Admission from New Born Registration - Select the check box, if admission is allowed for the patient from New Born Registration function.

  4. Max Discharge Period for Day Care () Hours – It is used to indicate the maximum period of day care stay allowed (in hours only), before the patient is discharged from day care unit. This field is applicable for Day Care patient management. This field is mandatory.

  5. Max Allowable Backdated Discharge – Enter the required value in Hours/Days in the numeric format of three digits maximum length. This is used to indicate the time limit set for backdating a discharge. This field is mandatory.

  6. Invoke Admission from New Born Registration - Select the check box if admission can be performed for the patient from ‘Newborn Registration’ function.

  7. Grace period for Assign Bed/Mark arrival - Enter the ‘Grace period for Assign Bed/Mark arrival’ in numeric format of maximum one digit. This field allows you to enter the number of hours after admission which is identified as the grace period and the details of the  patient who has arrived after the mentioned period is highlighted using different color in the function 'Assign Bed/Mark patient arrival'. This is a mandatory field.

  8. Priority Applicable – Select the check box to indicate the applicability of transfer priorities for the facility. This field is used to indicate that the transfers are to be performed based on priorities assigned. The check box is cleared, by default.

  9. Priority – This is used to select a default priority that can be assigned by the system for routine transfers. This field is enabled , if the ‘Priority Applicable’ field is selected.

  1. Allow Patient Check in – Select the check box if the Mark Patient Arrival function has to be a mandatory step for the confirmation of a patient admission, even if the bed and attending practitioner are specified at the ‘Admit Patient’ function. If the check box is selected, the bed is confirmed for a patient only after Patient arrival is marked. If the check box is cleared, the Mark Patient Arrival becomes an optional function to assign bed and/or attending practitioner. The check box is cleared, by default.

  2. Deactivate Pseudo Bed – Select the check box to indicate that all pseudo beds set up for the facility are available for use only once. After they have been associated to a user, they are deactivated and not available for use in future. The check box is cleared, by default.

  3. Request for Bed Class Change – Select the check box to indicate that a request for bed class change is applicable as a part of the transfer process. This enables transferring a patient from one bed class/type to other bed class/type. If this is not selected, ‘Request for bed class change’ check box does not get displayed in any of the transfer functions.

  4. Vital Signs Battery - Select the appropriate value from the list. This field is used to indicate that the default vital sign battery is applicable to all Nursing Units. These values are defined in the Application Masters module.

  5. Discharge Check List Applicable – Select the check box, if ‘Discharge Check List’ has to be a mandatory option before discharging a patient. The check box is cleared, by default.

  6. Discharge Checklist Expiry Period - Enter the required value in Hours/Days in numeric format of three digits maximum length to indicate the expiry period of a discharge checklist.

  7. Display Discharge Patient List – Select one of the values from the list. The listed values are ‘All Patients’ and ‘Only Patient with Discharge Advice’. Selecting ‘All Patients’ indicates the Discharge Patient function will list inpatients, with and without discharge advice. On selecting ‘Only Patient with Discharge Advice’ indicates that the Discharge Patient function will list inpatients with discharge advice.

  8. Alert Re-admission of the Patient Within ( ) Hours  – Enter the time in numeric format of three digits maximum length. You can also leave the box blank or enter zero. Only for values greater than zero validation takes place. If the patient is re-admitted within the defined time, an alert gets displayed.

  9. Check for Pending Orders before Discharge Advice - Select this check box to search for any pending orders at the time of preparing discharge advice.  By default this check box is cleared.

  10. If Orders are Pending - This option appears only when the 'Check for Pending Orders before Discharge Advice' check box is selected.

  11. Do not Allow Discharge Advice - Select this option to reject the preparation of discharge advice if there are pending orders. The system prompts an error message about the pending orders. Click [OK] to view the pending orders.

  12. Allow Discharge Advice - Select this option to allow the discharge advice preparation even if there are pending orders. On clicking the Encounter ID, the View Pending Orders dialog is displayed. Closing the View Pending Order dialog prompts a warning message.

  1. Restrict Discharge Advice if Return Mediation is pending – Select the check box to restrict discharge advice preparation if return medication is pending for a patient. By default, this check box is cleared.

  2. If you select this check box, the system checks whether any Return Medication is pending for the current encounter ID of a patient in Prepare Discharge Advice screen. If ‘Return Medication’ is not performed against any of the ‘Ward Returns’, the system restricts the preparation of discharge advice.

  1. Check for Pending Orders before Discharge - Select this check box to search for any pending orders before discharging a patient. By default, this check box is cleared.

  2. If Orders are Pending - This option appears only when the 'Check for Pending Orders before Discharge' check box is selected.

  1. Do not Allow Discharge - Select this option to reject the discharge of patient if there are pending orders. The system prompts an error message about the pending orders. Click [OK] to view the pending orders.

  2. Allow Discharge - Select this option to allow discharge even if there are pending orders. On clicking Encounter ID, the View Pending Orders dialog is displayed. Closing the View Pending Orders dialog prompts a warning message.  

  1. Reserve Bed while Transfer - If the check box is selected, you can reserve bed transaction in ‘Intra-Ward Patient Transfer’ and ‘Transfer Patient In’ functions. It is mandatory for reservation of beds.

  2. Allow Multiple Beds For Reserving - Select the check box to allow multiple beds for reserving.

  3. Restrict Reserving for Intra-Ward Transfer - Select the check box if you want to restrict the reservation of bed while doing intra-ward transfer. When you select this check box, the 'Reserve Bed' and 'Reason for Reserve Bed' fields under the Reserve Bed Details section in Intra-ward Patient Transfer function get disabled.  This check box is enabled or disabled based on the ‘Reserve Bed While Transfer’ check box.

  4. When the ‘Reserve Bed While Transfer’ check box is unselected, the ‘Restrict Reserving for Intra-Ward Transfer’ check box remains disabled and unselected.

  5. When the ‘Reserve Bed while Transfer’ check box is selected, the ‘Restrict Reserving for Intra-ward Transfer’ check box gets enabled automatically. But, you need to select the ‘Restrict Reserving for Intra-ward Transfer’ check box manually if you want to restrict the reservation of bed while doing intra-ward transfer.

  1. Change Admission Details - Enter the number of hours in numeric format of two digits maximum length. This is used to indicate the hours within which you can change the admission details of a patient after admitting the patient. This is a mandatory field.

  2. Allow invoking Maintain Death Register in Prepare Discharge Advice – Select the check box to invoke Maintain Death Register function in Prepare Discharge Advice function, if the discharge type is deceased.

• Mandatory – If you select this option with the check box selected, the system automatically launches the ‘Maintain Death Register’ screen after the Prepare Discharge Advice transaction.

• Non Mandatory - If you select this option with the check box selected, the system prompts the message ‘Do you want to invoke Maintain Death Register’ after the ‘Prepare Discharge Advice’ transaction. By default, this option is selected.

  1. Admission Consent Text – Select the check box to invoke a text box to add the comments. You can enter up to 3000 characters.

  2. Enable Reason For Late Discharge  – Select the check box to enable ‘Reason For Late Discharge’ field to record the reasons for late discharge in the ‘Patient Discharge’ screen. By default, this check box is not selected.

  3. Allow transfer without baby delink – You can use this check box to define the automatic transfer of a baby bed with the mother’s bed, during an intra-ward or inter-ward transfer. The functionality of this parameter is described as follows:

 

Allow transfer without baby delink (selected):

• The child bed (s) is also automatically transferred, provided there is enough child bed (s) in the destination ward.

• Once the mother and baby are transferred to the destination successfully then a popup message is displayed “Operation completed successfully” with OK command.

• If there is no sufficient baby bed(s) in the destination ward, the following message is displayed:

“Sufficient Baby beds are not available at destination ward. The child patient will be de-linked from mother. Do you want to continue?”

 - Click OK to de-link the baby and transfer the mother patient alone. Transfer the baby manually and then re-link through the Link baby to mother transaction.

 - Click Cancel if you want to cancel the transfer action.

 

Allow transfer without baby delink (cleared):

• The message "Do you want to de link baby bed from mother bed” is always displayed.

• Click OK; the child bed is de-linked from the mother patient and the mother is transferred to the destination bed.

• Re-link the baby to the mother patient through the Link baby to mother transaction.

By default, the check box remains cleared.

 

  1. Enable Initiate Pre-Discharge – Select this check box if you want practitioners to perform the Initiate Pre-discharge activity before nurses can perform the Prepare Discharge Advice activity. By default, this check box is unchecked.

If this check box is cleared, and you select IP Transactions > Initiate Pre-discharge, an error message “This function is not applicable since Enable Pre-Discharge is unchecked in IP Parameter” will be displayed.

 

  1. Reason for Admission Mandatory – Select this check box if you want to capture the admission reason as mandatory. By default, this check box is cleared. When you select this check box, the ‘Reason for Admission’ field becomes mandatory in Admit Patient screen.

  2. Weight on Admission Mandatory – Select one of the values from the list. The listed values are ‘Assign Bed / Mark Patient Arrival’, ‘Prepare Discharge Advice’ and ‘Discharge Patient’. By default, the value ‘Select’ will be displayed. This is not a mandatory field. Based on the selected value, the 'Weight on Admission' field becomes mandatory in the respective function.  

  3. Auto Confirmation request for transfer – Select the check box to confirm the encounter automatically during Request for Transfer and this encounter will not be available in ‘Confirm Request for Transfer’ screen. By default, this check box is not selected. Once you place the request for transfer, you could proceed with ‘Transfer Patient Out’ without the ‘Confirm Request for Transfer’ step. In addition, the Bed No field gets enabled and becomes mandatory in Transfer Patient Out screen.

  4. Display Bed Class change Authorization details in Current Encounter Details and Admission History –  Select the check box to view the confirmed bed class change records in Current Encounter Details and Admission History windows invoked in Request for Transfer, Intraward Transfer, Emergency Transfer, Confirm Bed Class Change transactions.  By default, the check box is cleared.

 

Bed Blocking

  1. Default Period for Normal () Hrs - Enter the default bed blocking period in hours for normal blockings in the numeric format of three digits maximum length. This is used in the ‘Block Bed’ function and ‘Book Bed for Patient Transfer’ while blocking the bed (to default the ‘Blocked until Date/Time’). The default ‘Blocked From Date/Time’ specified in the transaction is calculated based on the value specified in this field. This field is mandatory.

  2. Max Allowable Period for Normal – Enter the maximum allowable period for bed blocking period in numeric format of three digits maximum length. This is specified in terms of hours or days and is used to indicate the maximum period a bed can be blocked manually for any reason. This field is mandatory.

  3. Max Allowable Period for Discharge () Hrs – Enter the maximum allowable time for bed blocking period in numeric format of three digits maximum length. This is specified in terms of hours and is used to indicate the maximum period a bed can be blocked immediately after discharging a patient. This is validated in the Discharge Patient function. This field is mandatory.

  4. Max Allowable Period for Transfer Out () Hrs – Enter the maximum allowable period for bed blocking period in numeric format of three digits maximum length. This is specified in terms of hours and is used to indicate the maximum period a bed can be blocked immediately after transferring a patient out.

  5. Default Bed Blocking Period on Transfer/Discharge of Infectious Disease Patients () Hrs - Enter the maximum allowable period for bed blocking period in numeric format of three digits maximum length. This is specified in terms of hours and is used to indicate the maximum period a bed can be blocked immediately after transferring or discharging a patient with a infectious disease.

 

Enter the following details under the Additional tab:

 

Lodger/Accompanying Person Reference

  1. Next Serial No - Enter the Next Serial Number in numeric format of eight digits maximum length. The system generates the Next Serial Number for each Lodger or person accompanying an inpatient (who is selected by using ‘Check in Lodger’ function). This is used to indicate the reference number that is generated for lodgers. This number is incremented by one against every subsequent transaction. The value entered must be greater than zero. This field is mandatory.

  2. Max Serial No – Enter the Maximum Serial Number in numeric format of eight digits maximum length. The number specified is the Maximum Serial number that can be generated by the system for the reference Serial Number. The entered value cannot be less than the Next Number. When the Maximum Serial Number is reached maximum, no further numbers are generated and the System Administrator has to reset the Max Serial Number. This field is mandatory.

  3. Max Allowed Period of Stay for Lodger/Accompanying Person () Hrs – Enter the time limit of maximum stay for lodger/accompanying person in the facility. This is specified in terms of hours and is used to indicate the maximum period for which a lodger/accompanying person is allowed to stay within the facility.

  4. Allowable Period for Issue/Renewal of Gate Pass – Enter the Maximum allowable period for issue or renewal of a gate pass in numeric format of three digits maximum length. This is specified in terms of hours or days and is used to indicate the maximum period for which a gate pass can be issued or renewed for a person accompanying the patient, who has been selected in.

  5. Max Allowed Accompanying Persons – Enter the maximum number of people that can be selected in as accompanying person for an inpatient. This restricts the number of people that can accompany a patient during the patient’s stay as an inpatient.

  6. Maximum Period after Discharge for Check in Lodger () Hrs - Enter the time limit of maximum period of stay for Check in lodgers after the facility has discharged the patient. This is specified in terms of hours and is used to indicate the maximum period for which a discharged patient can be allowed to stay as a lodger within the facility.

Leave

  1. Release Bed On Leave – Select the check box to release the bed when the patient is sent on leave. If the check box is cleared, the bed is retained for the patient even when patient is sent on leave.

  2. Max Allowable Leave Period () Hrs – Specify in hours the maximum allowable leave period in numeric format of three digits maximum length. This is used to indicate the maximum period for which a patient can avail leave during an inpatient stay.

  3. Allow Leave Cancellation Within () Mins – Specify in minutes the maximum allowable period within which the leave can be cancelled, from the time of granting the leave, in numeric format of three digits maximum length. This is used to indicate the maximum period within which a leave that is granted can be cancelled.

  4. Grace Period for Abscond Patient () Hrs – Enter a value to specify the period that is considered, after the leave period has expired, for the patient status to be set as ‘Absconding’. This period is taken into consideration after the grace period set for ‘No Show’ has elapsed. The value is specified in terms of hours.

  5. Grace Period for No Show () Hrs - Enter a value to specify the period that is considered, after the leave period has expired, for the patient status to be set as ‘No Show’. The value is specified in terms of hours.

  6. Action On Exceeding Max Leave – Select either ‘Warning’ or ‘Error’ as action on exceeding the maximum leave period. If ‘Warning’ is selected, a warning message gets displayed in the ‘Record Inpatient Leave’ function, when the maximum leave is exceeded. If ‘Error’ is selected, in the ‘Record Inpatient Leave’ function an error message gets displayed on exceeding the specified value. The system prevents you from proceeding with the transaction.

 

Billing

  1. Interface to Billing Module – Select the check box, if this module has to be interfaced to the Patient Billing Manager module for billing purposes.

 

Bed Availability Chart Setup

  1. Allow Booking for Current occupied beds without EDD - Select the check box to allow booking the current occupied beds for which the EDD is not recorded.

  2. Allow booking on the current occupied bed considering EDD - Select the check box to allow booking the current occupied bed for which the EDD is recorded.

  3. Type of Booking - The Booking Types 'Booked Overrideable Confirmed', 'Booked Overrideable Unconfirmed', 'Booked Non-Overrideable Confirmed' and 'Booked Non-Overrideable Unconfirmed' are displayed with unique color code.

  4. Indicate future booking - Select the check box if future booking is applicable for the booking type.

  5. Number of days before - Enter the number of days before in numeric format of three digits maximum length if future booking is applicable.

  6. Consider EDD Before Stop – Select the check box if the EDD entered during admission should be considered before it is stopped. If the check box is selected, the system checks if the EDD entered during admission is less than the Booked Date and alerts or stops the admission.

  7. Action on Booking/admission - Select the Action from the list. The listed values are 'Stop' and 'Warn'.

 

General Details

  1. Default Referring Practitioner in Bedside Referral – Select from the drop- down list, the default referring practitioner to be displayed in Inpatient Bedside Referral screen. You can select one of the two values:

a. Attending Practitioner – If you select this value, the attending practitioner for that particular patient will be displayed as the default referring practitioner in the Register Inpatient Referral screen.

b. Login Practitioner – If you select this value, the logged-in user’s name will be displayed as the referring practitioner in the Bedside Referral screen, provided the logged-in user is a practitioner. If a non-practitioner logs in, the referring practitioner field in the Register Inpatient Referral screen will display a blank value and you can select a practitioner’s name.

  1. Allow Multiple Bedside Referral Review - Select this check box if you want to review the bedside referral requests multiple times in the Confirm Bedside Referral screen. By default this check is not selected. If you select this check, you will be able to view a ‘Status’ drop-down list and ‘Audit Trail’ hyperlink in the Confirm Bedside Referral screen.

  2. Enable Recording of Referral Letter Note with Referral Request – Select this check box if you want to enable recording of referral letter note along with the referral request. On selecting this check box, the system displays the ‘Record Referral Letter Note’ button in the Register Outgoing Referral, Register Inpatient Referral and Bed Side Referral screens. By default this check box is selected.
