Confirm Bed Class Change

From this function you can Confirm all unconfirmed Bed Class Change request raised from the Intra-ward Patient Transfer or from Request Patient Transfer function.

The function is accessible to the logged in user only if the flag 'Bed Class Change' is enabled for the Operator Station associated to the logged in user. All records from 'Intra-ward Patient Transfer' and 'Request Patient Transfer' is displayed if the request for Bed Class Change has not been confirmed. Only users who have the privilege to confirm Bed Class Change, which is defined by the flag 'Bed Class Change', are able to confirm Bed Class Change. The request for Bed Class Change can either be Accepted or Rejected. If the request is accepted, the Bed Class change is confirmed and the transactions Intra-ward Patient Transfer/Request Patient Transfer can be further processed. If the request is rejected, select a reason for rejection and the Intra-ward Patient Transfer/Request Patient Transfer function cannot be processed by changing the Bed Class and this process has to be performed again.

The Confirm Bed Class Change function is accessible to the logged in user only if the flag ‘Bed Class Change’ is enabled for the Operator Station associated to the logged in user.

Confirming Bed Class Change

From the IP Transactions > Transfer, select the Confirm Bed Class Change function.

The Confirm Bed Class Change screen gets displayed.

  1. Click [Search] to display the list of unconfirmed Bed Class Change requests or enter the search criteria and click [Search] to display specific unconfirmed Bed Class Change request. Click [Clear] to reset the records displayed in the result area.

    Enter the following query criteria to select the required patient record:

    1. Nursing Unit – Enter the Nursing Unit in alphanumeric format as specified or select the Nursing Unit using the [?] function. The Nursing Units associated to the Operator Station of the logged in user are listed.

    2. Admission Date From-To – Enter the Admission Date range from and to in the format of dd/mm/yyyy or select the date using the calendar function.

    3. Specialty – Enter the Specialty in alphanumeric format or select the Specialty using the [?] function. The Specialties created in the Specialty function of Application Masters is displayed.

    4. Practitioner – Enter the Practitioner in alphanumeric format or select the Practitioner using the [?] function. The Practitioners attached to the logged in facility in Practitioner for Facility function of Application Masters module are displayed.

    5. Bed Class - Select the Bed Class from the list.

    6. Bed Type – Select the Bed Type from the list. The Bed Types listed are based on the Bed Class selected.

    7. Encounter ID – Enter the Encounter ID in numeric format of 12 digits maximum length.

    8. Patient ID - Enter the Patient ID in alphanumeric format of maximum length as specified in MP Parameter or select the Patient ID using the [?] function.

  2. Click ‘+’ hyperlink to view the Current Encounter Details.

  3. On searching, Inpatient Details are displayed based on the entered criteria.

  4. Click the ‘Encounter ID’ hyperlink to display the Confirm Bed Class Change screen. The screen displays the Transfer Details, Current Occupancy Details and Transfer To details.

  5. Select the Request Status from the list. The listed values are ‘Confirm’ and ‘Cancel’.

  6. Click [Record] to confirm the Bed Class Change or [Close] to exit from the Confirm Bed Class Change screen.
