Transfer Patient In

From this function you can Accept or Reject Inpatients transferred from the source Nursing Units at the destination Nursing Unit.

The destination Nursing Unit receives the belongings of the patient and maintains a record, while accepting a patient under its care. This function allows changing of Nursing Unit when the required resources are not available on transfer of the patient. For regular transfers, beds are allocated by the destination Nursing Unit and Other Details recorded. Due to various reasons, the destination Nursing Unit can reject patients transferring into its care. In such cases, the reason for rejection is recorded and patient is sent back to the source Nursing Unit till acceptance.

The logged in user must have necessary rights defined in the Operator Station assigned to invoke this function.

Transferring Patient In

From the IP Transactions > Transfer, select the Transfer Patient In function.

The Transfer Patient In screen gets displayed.

  1. Click [Search] to display the list of Inpatients to be transferred to the destination Nursing Unit or enter the search criteria and click [Search] to select a specific patient transferred to the destination Nursing Unit. Click [Clear] to reset the records displayed in the result area.

    Enter the following query criteria to select the required patient record for transferring into the Nursing Unit:

    From Details

    1. Nursing Unit – Enter the Nursing Unit from where the patient has to be transferred in alphanumeric format of 15 characters maximum length or select the Nursing Unit using the [?] function.

    2. Specialty – Enter the Specialty from where the patient has to be transferred in alphanumeric format of 15 characters maximum length or select the Specialty using the [?] function.

    3. Practitioner – Enter the Practitioner name in alphanumeric format of 30 characters maximum length or select the Practitioner using the [?] function.

    To Details

    1. Nursing Unit - Enter the Nursing Unit to where the patient has to be transferred in alphanumeric format of 15 characters maximum length or select the Nursing Unit using the [?] function.

    2. Specialty - Enter the Specialty to where the patient has to be transferred in alphanumeric format of 15 characters maximum length or select the Specialty using the [?] function.

    3. Practitioner - Enter the Practitioner name in alphanumeric format of 30 characters maximum length or select the Practitioner using the [?] function.

    4. Encounter ID - Enter the Encounter ID in numeric format of 12 digits maximum length.

    5. Admission Date [From/To] - Enter the Admission Date Range in the format of dd/mm/yyyy or select the date using the Calendar function.

    6. Patient ID - Enter the Patient ID in alphanumeric format of maximum length as specified in MP Parameter or select the Patient ID using the [?] function.

  2. On searching, the details of the transferred patients are displayed based on the entered criteria.

  3. Click ‘+’ hyperlink to view the Current Encounter Details.

  4. Click the ‘Encounter ID’ hyperlink to display the Transfer Patient In screen. The screen displays the Admission Details, Occupancy Details, Other Details, Porterage Details and Reserve Bed Details.

  5. Select the Transfer In Status as ‘Accepted’ or ‘Rejected’ and enter the Transfer To Details and Reserve Bed Details in the mandatory (*) fields.

    Reserve Bed Details

    1. Reserve Bed – Select the check box to reserve bed for patient.

    2. Reason for Reserve Bed – Select the reason for reserving the bed from the list. The records defined in Blocking Type function in the IP Code Setup are displayed for selection.

    The ‘From’ bed can be reserved while transferring a patient. Select the check box ‘Reason for Reserve Bed’, if a bed needs to be reserved. You can Reserve the Bed in Intra-Ward Patient Transfer and Transfer Patient In functions. A patient can reserve only one bed at a time and reserved bed is marked with unique border. The reserved beds which are shaded with unique border can be selected. A patient cannot select the beds that are reserved by other patients.

    On selecting the ‘Transfer in Status’ as ‘Accepted’ most of the fields under ‘Transfer To’ heading is enabled. If the destination Nursing Unit rejects the patient, Nursing Unit details are displayed and disabled.

    On marking the Status as Accepted, the destination Nursing Unit details available while transferring out the patients are displayed which can be changed.

    For regular transfers, destination Nursing Unit details are displayed as recorded while transferring out except for the bed details. In such cases, beds are allocated on transferring into the destination Nursing Unit.

    Enter the following details:

    1. Nursing Unit – The destination Nursing Unit to which the patient is to be transferred is displayed which can be changed. Enter the Nursing Unit in alphanumeric format of 15 characters maximum length or select the Nursing Unit to which the patient has to be transferred using the [?] function. The search result displays the Nursing Unit associated to the Operator Station assigned to the logged in user. This is a mandatory field. If the selected Nursing Unit mandates prior request for transfer, the user is informed.

    2. Service – The Service to be rendered to the patient by the selected Nursing Unit is displayed, which can be changed. Select the Service from the list. The listed Services correspond to the selected Nursing Unit. This field is mandatory.

    3. Sub-Service – The Sub-service selected is displayed. Select the required Sub-service from the list. The listed values correspond to the selected Service.

    4. Bed Class – The Bed Class recorded during ‘Confirm Request for Transfer’ function is displayed. To allocate a different Bed Class, select the Bed Class from the list. The Bed Class listed depends on the Nursing Unit selected. This field is mandatory.

    The Bed Class can be changed if the logged in user has the privilege to change Bed Class as defined in the Operator Station associated to the user.

    • This is allowed only for unconfirmed Bed Class change requests, which is raised either from the ‘Transfer Patient Out’ or the ‘Request for Transfer’ functions.

    • If the request for Bed Class change is not confirmed at these stages and if it is not confirmed in the ‘Confirm Bed Class Change’ function, the user can change the Bed Class in this function.

    • If the user does not have the privilege, the request for Bed Class change has to be confirmed in ‘Confirm Bed Class Change’ function in which case the bed class cannot be changed.

    1. Bed Type – The Bed Type to where the patient has to be transferred is displayed.

    2. Bed No – The Bed Number allocated (if any) while confirming request for transfer is displayed. You can change the allocated bed. Enter the Bed Number in alphanumeric format of eight characters maximum length in case of ‘Regular’ transfers or select the Bed Number using the [?] function, which invokes the ‘Bed Availability Chart’. The lookup function list beds belonging to the selected Nursing Unit and Bed Class. This field is mandatory.

    One of the following indicators is displayed based on the condition set while confirming request for transfer with reference to the allocated bed.

    • Overrideable – If the bed has been blocked for the patient while confirming request for transfer and ‘Override’ check box is selected, ‘Overrideable’ is displayed. On moving the mouse over the text, the bed details are displayed. You can change the bed details.

    • Non-overrideable – If the bed has been blocked for the patient while confirming request for transfer and ‘override’ check box is cleared, ‘Non-Overrideable’ is displayed. On moving the mouse over the text, the bed details are displayed. If the bed is allocated and set as Non-overrideable, the bed details cannot be changed.

    • Occupied – If the Bed that was assigned to the patient while confirming request for transfer is occupied by the destination Nursing Unit, Occupied is displayed. On moving the mouse over the text, the Bed details are displayed. Another Bed has to be assigned to transfer patient.

    1. Room No – The Room Number is displayed based on the selected bed.

    2. Specialty – The Specialty of the destination Nursing Unit specified while transferring the patient out is displayed. To change the Specialty of the Nursing Unit, enter the Specialty in alphanumeric format of 15 characters maximum length or select the specialty using the [?] function. This field is mandatory.

    3. Practitioner – The Practitioner assigned to treat the patient in the destination Nursing Unit on transferring the patient in is displayed, by default. To change the Practitioner, enter the Practitioner in alphanumeric format of 30 characters or select the Practitioner using the [?] function. This field is mandatory. The Practitioner selected belongs to the specified Specialty.

    4. Practitioner Designation – The designation of the Practitioner associated to the patient is displayed on selecting the Practitioner.

    5. Medical Team – The Medical Team to which the Practitioner is associated is displayed.

  6. If ‘Accepted’ is selected, the ‘Occupancy Details’ tab is enabled and changes can be made (if allowed). If ‘Rejected’ is selected, only the ‘Reason for Rejection’ has to be entered in the ‘Occupancy Details’ tab.

  7. Click [Record] to transfer the patient in or [Close] to exit from the screen.
