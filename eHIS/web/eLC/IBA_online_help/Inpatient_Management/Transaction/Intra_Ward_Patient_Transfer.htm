Intra-Ward Patient Transfer

From this function you can Transfer a patient from one bed to another within the same Nursing Unit.

The patient is transferred to a bed belonging to the same bed class that is associated with the current bed that the patient holds.

In case the flag ‘Request for Bed class Change’ is selected in the IP Parameter, you have the provision to request for a different bed belonging to a different bed class. If the user performing the intra-ward transfer has the privilege of confirming such requests, as defined by selecting the flag ‘Bed Class Change’ against the Operator Station to which the logged in user is attached, the change for bed class request is confirmed in this transaction itself and the patient transfer process is completed. If the logged in user does not have the privilege, all such requests are routed to the ‘Confirm Bed Class Change’ function, where an authorized user confirms the bed class change after which the intra-ward patient transfer is completed.

Transferring Inpatients within the Ward

From the IP Transactions > Transfer, select the Intra-Ward Patient Transfer function.

The Intra-Ward Patient Transfer screen gets displayed.

  1. Click [Search] to display the list of all Inpatients or enter the search criteria and click [Search] to select a specific Inpatient. Click [Clear] to reset the records displayed in the result area.

    Enter the following query criteria:

    1. Nursing Unit – Enter the Nursing Unit description or select the Nursing Unit using [?] function.

    2. Admission Date From-To – Enter the Admission Date range in date format of dd/mm/yyyy or select the date using the calendar function.

    3. Specialty – Enter the Specialty or select the specialty using [?] function.

    4. Practitioner – Enter the Practitioner name or select the practitioner using [?] function.

    5. Encounter ID – Enter the Encounter ID in numeric format of 12 digits maximum length.

    6. Patient ID - Enter the Patient ID in alphanumeric format of maximum length as specified in MP Parameter or select the Patient ID using [?] function.

  2. On searching, Inpatient details are displayed based on the entered criteria.

  3. Click ‘+’ hyperlink to view the Current Encounter Details.

  4. Click the ‘Encounter ID’ hyperlink to display the Intra Ward Patient Transfer screen.

  5. Enter the details in the mandatory (*) fields. <Related Field Description>

  6. Click [Transfer] to complete the Intra-Ward Patient Transfer or [Close] to exit from the Intra-Ward Patient Transfer screen.

  7. On clicking [Transfer], Patient Charging Basis on Transfer screen gets displayed to select the charging basis for the bed type. The screen gets displayed only if the check box ‘ Allow Allocation of Higher Bed Type on Hospital Request ’ is selected in the ‘ BL Parameters ’ function in the Patient Billing Manager module.

    1. Patient's Request - Select Patient's Request, if the request for a higher bed type was made by the patient.

    2. Hospital's Request - Select Hospital's Request, if the request for a higher bed type was made by the hospital when the requested bed type was not available. An Authorization screen gets displayed on selecting Hospital's Request.

    Enter the User ID and the PIN for the logged in user. Click [OK] to login or [Cancel] to cancel the process. On clicking [OK], the Bed Type valid for the Hospital gets listed. Select a Bed Type based on which the bed charge has to be computed.

  8. If internal reports are defined, the Internal Reports screen gets displayed. Click [OK] to generate Internal Reports or [Cancel] to exit from the screen.

 

If the selected encounter has a request already placed for a transfer to some other Nursing Unit, the record is marked in pink indicating that a request has already been placed for this encounter. Intra-ward transfer is not allowed for such cases. The system prompts you with the appropriate message and does not allow continuing with the transaction.

If the patient for whom the transfer needs to be done has a baby registered, the system prompts you to de-link the mother and baby link (which is established when the baby is given a bed associated with the mother’s bed.) Only on de-linking the link, you are able to proceed with the transfer.
