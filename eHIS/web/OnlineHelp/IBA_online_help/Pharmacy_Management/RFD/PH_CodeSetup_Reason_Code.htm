Enter the following details to define a Reason Code record:

  1. Applicable Transaction - Select the required transaction from the list. The transaction ‘Medication Reissue’ is displayed, by default. This is a mandatory field. . The available transactions are Medication Return, Medication Hold, Allocation, Filling, Delivery, Amend Order, Administration, ADR Reason, TDM Reason, Release Batch(Patient), Release Batch(Others), Complete Order, Re-route Prescription, Override BMS, Balance Medication Remarks, Dosage Change in MAR, PRN Remarks, Cancel NPB Request, ADR Cancel Reason,  ADR  In-Error Reason, Cancel Medication Administration and Drug Disease Interaction Override Reason.

  2. Reason Code - Enter the Reason Code in alphanumeric format of four characters maximum length. This is a mandatory field. The codes created against these transactions will be shown in the Status drop down of Adverse Drug Reaction – Query screen.

  3. Reason Description – Enter the reason description in alphanumeric format of 15 characters maximum length. This is a mandatory field.

  4. Remarks – Enter the Remarks in alphanumeric format.

  5. Applicable for Quality Statistics Report – Select the check box to make the reason code applicable for quality statistics report. If this check box is selected, the QAP Reports are generated based on the reason.

  6. Enabled – Select the check box to enable the reason. The check box is selected, by default.
