Outpatient Tab

The parameter values applicable to outpatients are specified in the tab. Prescription rules for Refill of drugs, default dispensing locations for routine and stat orders, dispensing rules for advance and past orders, Dispense Location controls are entered for the logged in facility.

 

Stock Rules

  1. Consider Stock – Select the check box to verify stock status before ordering.

  2. Allow Without Stock – Select the check box to allow ordering without stock. The check box is enabled only when ‘Consider Stock’ is selected.

  3. Skip Duplicate Alert – Select this checkbox to enable the field Overlap Period. The Skip Duplicate Alert is enabled in the PH Parameter for Facility screen only if the Allow Duplicate Orders check box is selected.

  4. Overlap Period - Enter Period in Days with maximum of 3 character limit. Based on defined overlapping duration, the system skips the duplicate alert during Duplicate Pharmacy Order for all drugs. The defined Overlap duration is common for all drugs. The Skip Duplicate Alert field disables the duplicate alert during any pharmacy prescription for the defined overlap duration for outpatients.

  5. Reconciliation - Select the check box to perform reconciliation for medication orders in the Pharmacy Order screen. When the check box is selected, the medication reconciliation alert is displayed in Pharmacy Place order screen for inpatients. By default, the check box is not selected.  In addition, a [Reconciliation] button appears in the Pharmacy Order screen. The Medication Reconciliation functionality is mandatory during new admission, when patient is transferred from one unit to another unit, and during discharge.

 

Prescription Rules

  1. Allow Refill – If ‘Allow Refill’ is applicable at enterprise level (that is ‘Allow Refill’ selected in PH Parameter) the check box is selected, by default. This indicates that prescription refill is allowed in the logged in facility. Clear the check box, to indicate that Prescription Refill is not allowed in the logged in facility. If refill is not applicable at the enterprise level, then the check box is cleared and disabled.

  2. Maximum Number of Refills allowed – If ‘Allow Refill’ check box is selected, this field gets enabled. The value specified at the enterprise level is made default. Change the displayed value, but cannot exceed the enterprise level setting. Enter the maximum number of refill allowed in numeric format of two digits maximum length. The defined value cannot be exceeded during ordering. This is a mandatory field.

  3. Maximum Duration for Refill Prescription () Days - If ‘Allow Refill’ check box is selected, this field gets enabled. The value specified at the enterprise level is made default. Change the displayed value, but cannot exceed the enterprise level setting. Enter the maximum days for refill prescription in numeric format of three digits maximum length. The defined value cannot be exceeded during ordering. This is a mandatory field.

  4. Print Refill Prescriptions along with Main – Select the check box to print refill prescriptions along with the Main prescription. If the check box is cleared, then only the main prescription is printed. This field gets enabled only when ‘Allow Refill’ is selected.

  5. Auto-transfer A&E Medication Orders to IP Admission - If you select the check box, the Medication Orders associated with a patient’s A&E encounter are automatically transferred to his/her IP encounter when a patient is transferred from A&E to IP. It is unchecked and enabled by default.  Auto-transfer A&E Medication Orders to IP is done if IP admission is made within the defined hours after creating the A&E encounter.

  6. Emergency Encounter Duration within - This field is enabled and becomes mandatory only when Auto-transfer A&E Medication Orders to IP Admission check box is selected. This is applicable for users in MMS.

  7. Split Orders By - Priority - If this check box is selected, ‘Priority’ type is considered for splitting while releasing the prescription.

  8. Split Orders By - Drug Class - If this check box is selected, ‘Drug Class’ is considered for splitting while releasing the prescription.

  9. Amend based on order wise in Dispense Medication - Select the check box to allow the users to amend all drugs under an order ID even if a single drug from the order is selected for amend in Dispense Medication. This is applicable in Verification, Allocate and All Stages of Dispense Medication.  

 

Order Routing Defaults

  1. Routine Orders – Select the default dispensing location to route routine orders of Outpatients. The locations are defined in ‘Dispense Location’ function. The dispensing locations of type ‘Pharmacy’ and catering to both Outpatient/Inpatients and exclusively to Outpatients are listed.

  2. Stat Orders - Select the default dispensing location to route stat orders of Outpatients. The locations are defined in ‘Dispense Location’ function. The dispensing locations of type ‘Pharmacy’ and catering to both Outpatient/Inpatients and exclusively to Outpatients are listed.

 

Dispensing Rules

  1. Allow Dispensing in Advance – Select the check box to allow dispensing of drugs in advance of prescription start date.

  2. No. of Days Ahead – Specify the number of days in advance the drugs can be dispensed from the prescription start date. Enter the period in numeric format of three digits maximum length. This field is enabled only when ‘Allow Dispensing in Advance’ is selected. This is a mandatory field.

  3. Allow Dispensing past Orders - Select the check box to allow dispensing of past orders (prescriptions) that have exceeded the medication end date and time.

  4. Up to No. of Days - Specify the number of days in numeric format of two digits maximum length up to which an expired prescription order can be dispensed. The system validates expired prescriptions for dispensing based on the end date and time of the prescription. This field is enabled only when ‘Allow Dispensing Past Orders’ is selected. This is a mandatory field.

  5. Review/Approve Required for Verification Status – Select the check box to allow the pharmacist to review/approve the alerts available for the drug in the Verification Stage in Dispense Medication screen for outpatients. By default, the check box is not selected.

When you select the check box, the button [Review/Approve] appear in the alert dialogs invoked on clicking the verification icon in Dispense Medication screen for Verification stage only. Click the [Review/Approve] button to invoke the Review/Approve dialog to record the remarks. This parameter is applicable only for users in MMS.

 

Dispense Location Controls

Stages

The prompts specified for various dispensing stages in ‘PH Parameter’ are displayed. You have to specify applicability of these stages for Outpatient Dispensing Location.

  1. Registration - Select the check box to indicate prescription registration is required. If the check box is selected, ‘Registration’ is applicable for outpatients Dispense Locations.

  2. Fill – Select the check box to indicate drug ‘Fill’ is required. If selected, it is applicable for Outpatient-Dispense Location in the function ‘Dispense Location’.

  3. Stages – Select the fill stage from the list. The values listed are ‘Before Allocate’ and ‘After Allocate’. The set value is defaulted in ‘Dispense Location’ function. If ‘filling’ is not applicable, ‘Not Applicable’ is displayed.

  4. Before Allocate – The Drugs are filled and batches are allocated.

  5. After Allocate – Allocate batches and fill the drugs.

  1. Verification – Select the value from the list. The values listed are ‘Separate before Allocate’ and ‘Combined with Allocate’. The set values are made default in ‘Dispense Location’ function.

  2. Separate Before Allocate – The drugs are verified before allocation.

  3. Combine with Allocate – Drug verification and allocation performed at the same time.

  1. Delivery - Select the check box to indicate delivery stage is applicable. If the check box is selected, ‘Delivery’ is applicable for outpatient-Dispense Location in the function ‘Dispense Location’.

  1. Required Approval No in Dispense Medication - Select the check box to make the line level Approval Number field as mandatory in the Dispense Medication, Sales / Patient Issue and Direct Dispensing screens. The Approval No field in the header level will be non-mandatory. By default, the check box is not selected. This is applicable for users in AAKH.

The ‘Stages’ values are set only once and cannot be changed later on saving the record.

 

Outputs

 

  1. Print Bal. Medication Sheet on Partial Disp - Select the check box to indicate printing of balance medication sheet for partial dispensed orders.

  1. Label Format - Select the value from the list. The values listed are Drug wise and Combined.

  • Drug Wise – When the drug is dispensed, the Label formats are printed for each drug.

  • Combined –When the drug is dispensed, the Label formats are printed for the entire prescription.

  1. Default Language – Select the default language from the drop-down list.

  1. The selected language is displayed, as default, in the Print / Edit Label dialogue window and the Dispense label is  printed in the same language. However, if the language ‘Arabic’ is selected, the Dispense label will be printed in both Arabic and English language. The Patient Name, Dosage Details and Cautionary Instructions will be printed in Arabic and the rest of the details will be printed in English.

  2. If ‘None’ is selected , the Dispense label will be printed according to the default language in the Print / Edit Label dialogue window or based on the logged-in user’s language.

  3. Note: By default, ‘None’ is selected as the default Language

 

Drug Return

  1. Allow Medication Return – Select the check box to allow returning of dispensed medication.

  1. Within [] days – Enter the period within which the medications should be returned from the date of dispensing in numeric format of four  digits. This field is enabled only if you select Allow Medication Return. This is a mandatory field.

 

Undelivered Drugs

  1. Max Retention Period before Return to Stock [] days – Specify the number of days to return undelivered drugs held by the dispensing location to the Stores. Enter the period in numeric format of two digits maximum length.
