Adverse Drug Reaction

From this function you can Record Adverse Reactions noticed on administration of Drugs on a patient.  

The function allows recoding the details of unexpected reactions that help in tracking drugs, which may require inspection before further use on patients. Adverse reactions are recorded for drugs prescribed within the period allowed for recording ADR (Adverse Drug Reaction) as set in PH Parameter function.

The following conditions are evaluated before recording ADR:

• Active Drugs (not discontinued, placed on hold or cancelled).

• End date of medication less than system date and time, but within the period set in parameter.

Recording Adverse Drug Reaction

From the PH Transactions, select the Adverse Drug Reaction function.

The Adverse Drug Reaction screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the Patient ID in the format as specified in Master Patient Index module or select the required Patient ID using [?] function. This is a mandatory field.

  3. On selecting the Patient ID, the Patient Name, Age and Gender is displayed in the Patient Name field.

  4. Select the Suspected Drug using [?] function to select for the required drug. On clicking [?], the Drug Search screen displays the drugs prescribed to the selected patient based on the period specified in PH Parameter. This is a mandatory field.

  5. Click ‘Drug’ hyperlink from the Drug Search screen to select the suspected drug.

  6. The Drug details are displayed in the Adverse Drug Reaction screen.

    • Drug Detail

      The Drug Details tab consists of three sections, namely, ‘Drug Detail’, ‘Prescription Detail’ and ‘Diagnosis Detail’.

       

      • Drug Detail section – The selected drug details are displayed as read-only text.

      The following details are displayed.

        1. Drug - Drug name gets displayed. If an external  drug is selected to record the adverse drug reaction, the drug name is displayed with the term ‘external drug’ in parenthesis.

        2. Generic - Generic name of the drug gets displayed.

        3. Trade Name – The trade name associated with the selected drug gets displayed, if the drug is linked to an Item. If multiple Items are associated, select the Trade Name from the list. On selecting the Trade Name, the manufacturer associated to the Trade Name gets displayed.

        4. Strength – The Strength of the Drug gets displayed.

        5. Route - The Administration Route of the drug gets displayed.

        6. Form – The Form of the drug gets displayed.

          • Prescription Detail section - The prescription details displayed are prescribed dosage, frequency, duration, medication start date/time and end date/time.

          • Diagnosis Detail section - If diagnosis is recorded while ordering, the details are displayed. You can edit the displayed details. Enter the diagnosis details in alphanumeric format. . Click [?] to search and select the diagnosis details from the Problem List dialog.

    • Reaction Detail

      The ‘Reaction Detail’ tab consists of two sections, Administration Detail and Adverse Reaction.

       

      Administration Details section

      Administration details are displayed as read-only text for the selected patient, if the Administration details recorded in the function ‘Medication Administration’, enter the following details.

      1. Admin Date and Time – Enter the administration date and time in the format of dd/mm/yyyy hh:mm or select for the required date using Calendar function. The entered date must not be less than medication start date and time or greater than current system date and time. This is a mandatory field.

        • If the selected patient is currently an Inpatient and medication prescribed for administration, the schedule administration date and time values are listed. The schedules from medication start date and time till current system date and time are listed. Select the required value.

        • If the selected patient is currently an Inpatient and ADR recorded on medication administration, the administration date and time gets displayed as read-only text.

      2. Drug/dose – The Prescribed dose of the selected drug gets displayed. Specify the administered dose in numeric format of six digits maximum length. The applicable unit of measurement gets displayed. This is a mandatory field.

      3. Administered By - If ADR is recorded on medication administration for the selected patient, the name of the person recording the administration details get displayed as read-only text or enter the name of the person administering the Drug in alphanumeric format of 30 characters maximum length.

      4. Batch ID - Batch ID selected during medication administration gets displayed as read-only text. To record the Batch ID of the drug, enter the Batch ID in alphanumeric format of 10 characters maximum length.

      5. Expiry Date - Enter the expiry date of the selected Batch ID in the format of dd/mm/yyyy or select the required date using Calendar function. If ADR is recorded on medication administration for the selected patient in ‘Medication Administration’ function, the expiry date of the batch gets displayed as read-only text.

      6. Manufacturer - Select the drug manufacturer name from the list, if the drug is linked to multiple manufacturers. If ADR is recorded on medication administration for the selected patient in ‘Medication Administration’ function, the manufacturer name gets displayed.

      7. Supplier - Enter the supplier name in alphanumeric format of 30 characters maximum length.

       

      Adverse Reaction Description section

      The adverse drug reaction details are specified in this section.

      1. Hypersensitivity – Select the hypersensitivity to the drug such as Allergy, Type II, Type III or Type IV.

      2. Onset Date and Time - Enter the onset date and time in date format of dd/mm/yyyy hh:mm or select the required date using the Calendar function. The specified date and time must be between administration date and time and current system date and time. The ‘Time to Onset of Reaction’ is calculated on entering the ‘Onset Date and Time’.

      3. Time to Onset of Reaction – Enter the period in numeric format of two digits maximum length. The onset time of reaction after drug administration is specified. The onset period is specified in terms of Days, Hours or Minutes. The ‘Onset Date and Time’ is calculated based on the value specified for ‘Time to Onset of Reaction’. This is a mandatory field.

      4. ADR - Select ADR Type from the list. The listed values are defined in ‘ADR Type’ function. This is a mandatory field.

      5. Severity - Select one of the severity values of the drug reaction. The listed values are Unknown, Mild, Moderate and Severe. By default, the value 'Unknown' is displayed. This is a mandatory field.

      6. Certanity – Select one of the the certainity options such as Definitive or Probable. By Default, the ‘Definitive’ is selected.

      7. Source of Info – Select one of the source of information radio button such as Patient, Investigation or Staff. By default, the ‘Patient’ is selected

      8. Drug Reaction Relationship - Select the value from the list. The listed values are Unlikely, Certain, Probable, Possible and unclassifiable. This is a mandatory field.

      9. Questionnaire – Select the Questionnaire from the list. The Questionnaire defined in the ADR Question function is displayed.

      10. System Identified Relationship – Displays the system defined relationship.

      11. Reaction Stopped after stopping/reducing medication - Select the value from the list. The listed values are Unknown, Yes and No.

      12. Reaction reappeared after re-introducing medication - Select the value from the list. The listed values are Unknown, Yes and No.

      13. Outcome – Select the outcome from the listed values. The listed values are ‘Recovered’, ‘Not Yet Recovered’, ‘Fatal’ and ‘Unknown’.

      14. Status – Select one of the statuses such as Active  and Resolved. By default, the ‘Active’ status is selected.

    • Other Medications

      • If adverse reaction is caused after administration of other prescribed drug, the other Drug(s) is selected in ‘Other Medications’ tab. All the drugs prescribed to the selected patient (as per parameter setup) except the selected ‘Suspected Drug’ are listed.

      • Select the check box against the Drug Name to select the Drug for ADR Recording.

    • Remarks

      Enter the following remarks:

      1. Additional Medication – Enter the additional medication details in alphanumeric format. If adverse reaction is Observed/Reported due to drugs other than the prescribed drugs, specify the medication details.

      2. Adverse Reaction Description – Enter the Adverse Reaction Description in alphanumeric format.

      3. Treatment of adverse reaction – Enter the Description for Treatment of Adverse Reaction in alphanumeric format.

      4. Relevant Investigations - Enter the relevant investigations required on reporting of adverse drug reaction in alphanumeric format.

      5. Relevant History - Enter the relevant history details in alphanumeric format.

      6. Reported By – The logged in user name gets displayed, by default. Enter the person reporting the ADR in the required format. This is a mandatory field.

      7. Date/Time – The ADR recording date/time displayed is the current system date/time. The value cannot be edited. This is a mandatory field.

  1. The Weight of the selected patient gets displayed (if recorded) or record the weight of the patient.

  2. Select the Ethnic Group from the listed values.

  3. Select the required Status. The available statuses are ‘Unverified’, ‘Verified’, ‘All’, ‘Cancelled’, ‘In-Error’.   

  4. Click [Apply] to record the Adverse Drug Reaction or [Reset] to clear the entered data.

  5. Click [Query] to invoke the Query screen.

  6. Enter the details in the mandatory (*) fields. <Related Field Description>

  7. Click [Search] to display recorded Adverse Drug Reaction records.

  8. Select Patient ID against the ADRs with the status ‘Un-verified’/’Verified’. You will be redirected to the Adverse Drug Reaction screen. The selected ADR details are displayed.

    • If the selected record is signed, the modification is not allowed and the recorded details are displayed as read-only text.

    • If the record is not signed, except Patient ID, suspected drug and weight, other details can be modified.

  9. Select the value ‘Cancel’ for unverified ADR from the Status drop-down list or select the value ‘In-Error’ for verified ADR from the Status drop-down list. The Remarks hyperlink is displayed once status is changed.

If you click the Un-Verified ADR record in the query screen, only the value ‘Cancel’ will be available in the Status drop-down list in the Adverse Drug Reaction screen. If you click the Verified ADR record in the query screen, only the value ‘In-Error’ is available in the Status drop-down list in the Adverse Drug Reaction screen

  1. Click the Remarks hyperlink to display the Cancel Remarks dialog to enter the remarks about the cancellation/marking In-Error of the ADR. You cannot modify remarks comments and the status for Cancelled and In-Error ADR.
