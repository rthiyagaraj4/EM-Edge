Administering Inbound/Outbound Events

From this function you can View all the Inbound and Outbound transactions.

Administering the Inbound or Outbound events function is used to view all the Inbound and Outbound transactions (non-query based) that are generated. You can also replay, rebuild, view the messages, audit log and debug the log. It also includes viewing the server error and communication client errors.

Administering Inbound/Outbound Events

From Administration > select Administer Inbound/Outbound Events Function.

The Administer Inbound/Outbound Events screen gets displayed.

  1. The screen displays three tabs, namely

General

Enter the following:

  1. Date From–To - Enter the required ‘From’ and ‘To’ dates.

  2. Message ID From–To – Enter the ‘From’ and ‘To’ message ID's for which the interface message has to be populated.

  3. Application - Select the required application from the list. The listed values are defined in the Application function of EM Interface Engine module.

  4. Event Type - Select the required Event Type from the list. The listed values are defined in the Event Type function of EM Interface Engine module.

  5. Communication Mode - Select the Communication Mode from the list. The options are ‘Out Bound’, and ‘Inbound

  6. Facility - Select the required facility form the list. The listed values are defined in the Facility function of System Manager module.

  7. Event Status – Select the suitable status from the list. The options available are – ‘All’, ‘About to split’, Successfully Loaded, Rejected, Successfully Processed, ‘Unprocessed’, ‘Not required’ and ‘Error’.

  8. All – All the events irrespective of their current status.

  9. About to Split – Events that are going to be split based on the control log information depending on the multiple applications.

  10. Successfully Loaded – To view successfully loaded messages.

  11. Rejected – Messages rejected by external application

  12. Successfully Processed – To view messages that are successfully built.

  13. Unprocessed – To view messages that are unprocessed.

  14. Not required – Some events need not be generated based on the application.

  15. Error – Events with error during the event execution.

  1. Protocol ID - Enter or Select the Protocol ID using the [?] function. The listed values are defined in the Protocol function of EM Interface Engine module.

  2. Domain – Select the Domain from the list. The values are listed from the System Manager module.

  3. Sub Domain – Select the Sub Domain from the list. The Sub domains are created in the Interface Domain function.

  4. External Acc No - Enter the External Account Number.

  5. Purge Status - Select the check box to view Purge Status events.

Additional

Click Additional Tab and enter the following query criteria:

  1. Message Status - Select the appropriate Message Status from the list of values. The options available are All, Successful, Accepted, Rejected, Failed, In Error and Pending.

  2. Message Date - Enter the appropriate Message Date and Time.

  3. Patient ID - Enter the Patient ID, which is a part of the message.

  4. Merged Patient ID - Enter the Merged Patient ID, which is a part of the message.

  5. Episode Type - Enter the appropriate Episode Type.

  6. Episode ID - Enter the corresponding Episode ID.

  7. Visit ID - Enter the Visit ID, which is a part of the message type.

  8. Action Type - Enter the required Action Type. The options available are, All, Update and Insert. If the record already exists, ‘Update’ is used and if a new record is to be created, ‘insert’ is used.

  9. Last Processed Date – Enter the Last Processed Date for the event.

  10. Not Required Reason – Enter the Reason Not Required for not communicating the particular event to the application.

Audit

Click Audit Tab and enter the following query criteria:

  1. Added ID – Enter the ID of the user who has the latest HL7 transaction in the Enterprise Management application.

  2. Added Date – Enter the date of the transaction.

  3. Added WS No – Enter the workstation number in valid format from where the transaction is done.

  4. Modified ID – Enter the ID of the user who has loaded the process.

  5. Modified Date – Enter the date on which the record is loaded for transaction.

  6. Modified WS No – Enter the workstation number from where the process is loaded.

  1. Click [Search] to display the Administer Events screen with Event status and Message Status.

  2. Click [Apply] to save the changes and Click [Reset] to reset the selected the values.
