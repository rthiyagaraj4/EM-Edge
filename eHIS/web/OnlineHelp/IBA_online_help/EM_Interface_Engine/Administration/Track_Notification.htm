Track Notifications

From this function you can Track the status of SMS Notifications sent to the practitioner for critical lab results.  

This function allows to track the status of SMS Notification sent to the practitioner for critical lab results. Allows to view, print or save the notifications in pdf format.

Users can select or re-submit the failed and delivered SMS only if 'Notification' check box is selected in the 'Authorised Users' function.

In Notifications function, user must map the 'Application ID (laboratory)' and 'Event type' (Critical Lab Result) with result columns to be displayed in 'Track Notification' screen.

Tracking Notifications

From Administration menu, select Tracking Notifications Function.

The Tracking Notifications screen gets displayed.

  1. Enter the details in the mandatory fields. <Related Field Description>

  2. Click [Search], system displays the results with columns configured in Notification function. If user does not click on Search button, then system will display the records based on the auto refresh timings for the selected criteria.

  3. ‘Select’ check box against the patient records will be enabled only for failed and delivered (sent) records. Select the check box against the required records and click [Export]. The selected records will be displayed in spreadsheet or PDF format. User can view, print, or save the record.

  4. If user clicks, the ‘Select All’ check box, system will select only all failed records, whereas to select already delivered records user need to select one by one.  

  5. To re-submit records, select the check box against the records and click [Re-submit] button. The system will display a prompt message ‘The selected record(s) has been re-submitted’ with OK button. Click OK to close the message.
