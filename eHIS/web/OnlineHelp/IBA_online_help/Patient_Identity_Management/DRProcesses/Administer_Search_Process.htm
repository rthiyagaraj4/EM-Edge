Administer Search Process

From this function you can Review and change the status of the existing DR Process.

The access to this function is determined by the User Access Rights set for the logged-in user. You cannot access this function, if rights for the logged in user are not defined or the rights for the DR Process ‘Administer search process’ is set as ‘None’. The functions ‘Search Duplicates’, ‘Identify Main/Duplicates’ and ‘Merge Duplicates’ together form the Process Cycle and must be performed sequentially to close the DR Process cycle. When the Search Process is submitted, a new Process Cycle begins automatically. On successful or unsuccessful completion of search process, you can abort the process before starting the Merge process. On aborting the process, you can submit the search process again. On identification of Main/Duplicates (after search process), you can perform the merge process. On completion of the merge process (successfully or unsuccessfully), you have to close the process explicitly. When ‘Merge Duplicates for a Patient’ is performed, the Merge Process Cycle begins automatically. After the completion of the process, you must explicitly close the cycle using this function. The current process cycle must be closed before starting another DR process cycle.

Administering Search Process

From the DR Processes, select the Administer Search Process function.

The Administer Search Process screen gets displayed.

  1. The Search Process Details and the Merge Process Details are displayed on invoking the Administer Search Process function.

  2. Click [Refresh] to refresh the screen and view the current status of the Search/Merge Process.

  3. Select the ‘Order By for Cycle’ from the list. The listed values are ‘Patient ID’ and ‘Regn Date’.

  4. Select the Action Type from the list. The listed values are ‘None’ and ‘Abort Process’. You can select ‘Abort Process’ to terminate the process before Merge Process commences. You can close the process cycle by selecting ‘Close Process’, where groups have been identified as main/duplicates and merged with the current, that is, on successful or unsuccessful merge process.

  5. Click [Apply] on selecting the required Action type to abort the Search/Merge Process or to close the Search/Merge Process. Click [Reset] to clear the Entered/Selected data.

 

You can start the process manually using Administer Search Process function in System Manager Module.
