Terms and Conditions

From this function you can Define the terms and conditions to be printed in the Purchase Orders. You can Query about the existing terms and conditions.

This function allows maintaining the terms and conditions to be used in the purchase activities. The terms and conditions are available for selection across the Purchase Order module.

Defining Terms and Conditions

From the PO Code Setup, select the Terms and Conditions function.

The Terms and Conditions screen gets displayed.

  1. Click [Create] to define the terms and conditions.

  2. Enter the Terms and Conditions Code in alphanumeric format of 10 characters maximum length. Once the code is defined and saved, it cannot be changed in Query function. This is a mandatory field.

  3. Enter the Short Description about the terms and conditions type in alphanumeric format of 30 characters maximum length. This is a mandatory field.

  4. Enter the Long Description of the selected terms and conditions type in alphanumeric format of 4000 characters maximum length. This is a mandatory field.

  5. Select the Enabled check box to enable the defined terms and conditions type. By default, this check box is selected. The enabled terms and conditions are available for selection across the Purchase Order module. Clear the Enabled check box to disable the defined terms and conditions type.

  6. Click [Apply] to save the defined terms and conditions or [Reset] to clear the entered data.

  7. Click [Query] to invoke the Query screen.
