Master Type

From this function you can Define different Master types and Query for the existing Master Types.

This function allows defining the different Master types. They are:

  • Purchase Order – It is used to identify the nature of a purchase. For example, the purchase is through a tender, quotation, special permission, Direct Purchase, Purchase under $2000, Purchase Agreement.

  • Purchase Request – It is used to identify the nature of requests made by the stores.

  • Mode of Delivery – It is used to define the mode of delivery of goods from the suppliers. For example, By Air, By Sea or By Road.

  • Terms of Payment – It is used to define the terms of payment. For example, Trade Credit (Net 10, Net 15, Net 30 and Net 60), forward dating.

  • Commitment Code – Define the commitment code, allocated budget amount, additional budget amount, account number, account number description, department and effective from and to dates.

  • Tender – Define the type of tender as a setup and link it during processing of tender. Choosing a Tender Type is mandatory.

Defining Master Type

From the PO Code Setup, select the Master Type function.

The Master Type screen gets displayed.

  1. Click [Create] to define the Master Type.

  2. Select the Master Type to be defined from the list.  The listed values are ‘Mode of Delivery’,  ‘Purchase Order’, ‘Purchase Request’, ‘Terms of Payment’, ‘Selection Criteria’, ‘Evaluation Factor’, ‘Quotations’, ‘Commitment Code’ and 'Tender'.

  3. Enter the details in the respective fields. Related Field Description

  4. [>>] is enabled only when the Master Type is selected as Commitment Code. Click [>>] to invoke the Commitment Code Additional Details screen.

    Enter the following details:

    1. Department – Select the Department from the list.

    2. Supplier Account No - Enter the Supplier Account Number in alphanumeric format of 20 characters maximum length.

    3. Account No Description - Enter the Account No Description in alphanumeric format of 30 characters maximum length.

    4. Allocated Budget – Enter the Allocated Budget in numeric format of 10 digits maximum length.

    5. Additional Budget – Enter the Additional Budget in alphanumeric format of 10 characters maximum length.

    6. Balance Budget – Displays the value of the Allocated Budget and Additional Budget, and the updated balance of the budget remaining after purchase order transactions.

    7. Effective From and Effective To Date - Enter the From and To date in dd/mm/yyyy format.

    8. Override Budget Control – Select the Override Budget Control value from the list. The listed values are ‘Error’, ‘Warning’ and ‘Not Applicable’.

      • Error – When the Balance Budget is zero, the purchase order transaction is not allowed.

      • Warning – When the Balance Budget is zero, an alert message is displayed during purchase order transactions. You can proceed with the transaction if required.

      • Not Applicable – The purchase order transaction is processed without any budgetary control check.

    9. Click [Close] to save the commitment code changes.

  5. Click [Apply] to save the master type record or [Reset] to clear the entered data.

  6. Click [Query] to invoke the Query screen.
