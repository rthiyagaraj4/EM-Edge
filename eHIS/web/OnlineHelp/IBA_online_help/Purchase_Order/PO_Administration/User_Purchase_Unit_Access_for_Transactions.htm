PO User Purchase Unit Access for Transactions

From this function you can Provide rights for a user belonging to a Purchase Unit for transactions applicable for the user. You can Query for the existing records.

You can provide rights such as Create, Finalize/Authorize, Amend, Cancel and Delete for a user belonging to a purchase unit for Purchase Request, Purchase Order and Tender transactions.

Defining PO User Purchase Unit Access For Transactions

From the PO Administration, select the PO User Purchase Unit Access For Transactions function.

The PO User Purchase Unit Access For Transactions screen gets displayed.

  1. Click [Create] to provide the rights of the user belonging to a Purchase Unit.

  2. Select the User using the [?] function. This is a mandatory field.  

  3. Select the Purchase Unit from the drop-down list. This is a mandatory field.

  4. Click [Search].  The transactions applicable for the user are displayed as search results.

  5. Select the Default Store for the user from the drop-down list for Purchase Request transaction.

  6. For Purchase Order and Tender, select the Create, Finalize/Authorize, Amend, Cancel and Delete check boxes, as required. Refer <Related Field Description>

  7. Click [Apply] to save the rights provide to the user or [Reset] to clear the entered data.

  8. Click [Query] to invoke the Query screen.

Amend is applicable for only Purchase Order transaction.
