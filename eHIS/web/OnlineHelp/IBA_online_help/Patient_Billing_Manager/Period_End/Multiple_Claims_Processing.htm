Multiple Claims Processing

From this function you can Process multiple claims of the patient.

This function allows to process multiple claims for the patient having customer group defined in the Multiple Claim setup.   This function lists out patients who were discharged for the current month / previous month.

 

This function is applicable only if the ‘Multi Claims Processing’ check box is selected in BL Parameters screen.  If the check box is cleared, the message ‘Function not applicable for the facility’ gets displayed.

• If the patient is having customer group which is not defined in Claims Setup, the system displays the validation message 'Not applicable for customer group'.  On tab out of the Patient ID, no data will be populated for the same patient.

• If the patient is having customer group defined in claims but discharge is not initiated or bill is not generated, the system prompts the validation message 'Patient details not found,' on tab out of the Patient ID. No data will be populated for the same patient.

• If the patient is having 4 episodes with discharge status as 'Still admitted', with discharge is not initiated or discharge bill not generated for the 5th episode, on tab out of the Patient ID, the system shows the corresponding four episodes for the patient and the fifth episode details will not be displayed since discharge is not initiated.

Multiple Claims Processing

From the Period End, select the Multiple Claims Processing function.

The Multiple Claims Processing screen gets displayed.

  1. Select the Patient ID from the lookup. On tab out of the Patient ID, the system defaults the values in the Patient Name, Customer Group Code, Customer Group Description, Customer Code, Customer Code Description fields. All fields are non-editable.

  2. There are two tabs: Current and History

Current

This tab shows the episodes which are yet to be discharged and their claim status. Only the active records (with the value Still admitted in the Discharge field) are displayed.

 

I. Select the ‘Admission Date/Time’ for the episode by clicking the calendar icon. You cannot select current or future date. It is editable field. This field is mandatory.

Note: Admission date and time can be modified before generation of approval number. Post   generation of approval number, Approval date and time for the episode cannot be changed.

• Select the Discharge Status from the list. The listed values include Select, Still Admitted, and Discharged. By default, the value 'Select' gets displayed.

• If the patient is still admitted with a new episode ID, select 'Still Admitted' value from the Discharge Status.  

• If the patient is discharged, select the value 'Discharged' from the drop-down list. The Discharge date is mandatory field when discharge is selected from drop-down list.

II. The Discharge Date values can be changed at any time before generation of Approval Doc number in ‘Approval Bill’ function. The Discharge Date values cannot be changed after posting.

III. Select the ‘Discharge Date’ by clicking the calendar icon. It is mandatory to select when the patient is marked as discharged. If the patient is not discharged, this field is non-editable.

IV. Select the Status if the corresponding episode bills are pending in Approval Bill Function. The system shows the status color as Grey.

o If the episode bill is approved in Approval Bill function, the system shows the status color as yellow.

o If the episode bill is approved and posted, the system shows the status color as 'Green'.

o The color Indicators are shown at the bottom of the function.

 

History

This tab shows only the patient which are physically discharged and the corresponding claim details. All the fields shown in the History tab are non-editable. The Discharge date updated with physical discharge date time records processed for review, approval and posting to be made available in the history tab.

 

1. The columns available are: Episode ID, From Date & To Date, Bill Doc No, Bill Doc Date, Net Bill Amount, Admission Date/Time, Discharge Date/Time, Approval No, Posted Month, Posted By.

2. Approval No - If the corresponding episode bill is approved, the system shows the Approval No.

3. Posted Month - If the corresponding episode bill is approved and posted, the system displays the Posted month.

4. Posted by -  The system displays the ID of the user who posted it.
