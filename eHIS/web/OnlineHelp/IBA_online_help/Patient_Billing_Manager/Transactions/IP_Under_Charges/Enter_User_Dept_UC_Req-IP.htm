Enter User Dept Under Charge Request – IP

From this function you can Associate the billing services to an inpatient after the discharge advice is processed based on the departments. An authorized User can associate the billing level services to a patient ID.  

Enter User Dept Under Charge Request – IP function is used to associate a list of billing services to an Inpatient or a Daycare patient. A reference document number is generated for the services associated to the patient using the Patient Billing Manager module. This function performs the same transactions as the ‘Enter Request – IP Under Charge Items’, but an authorized User can define the billing location to bill the services. The Accounting Department associated to the Billing Services during definition are superseded by the Accounting department associated using this function. The Accounting Department personnel have to confirm the services. A reference ID is generated for each service associated to the patient. The services with the ‘Direct Entry Allowed’ check box selected are listed. Authorized User has to choose a service from the list. During discharge, a bill is generated for the services utilized based on the quantity of the service used. The service price is based on the Multipricing factor, if the service is rate based. If the service is charge based, the charge defined is calculated. The charges for the services associated are calculated automatically and displayed. You cannot update the charges for the rate based services. The total charges are computed and displayed. The Services with the ‘Autoconfirm’ check box selected during definition are confirmed automatically. If the check box is cleared during definition, you have to confirm the services manually using the function ‘Confirm Inpatient Service Requests’.

Enter User Dept Under Charge Request – IP

From the Transactions > IP Under Charges, double-click Enter User Dept Under Charge Request – IP Function.

The Enter User Dept Under Charge Request – IP screen gets displayed.

  1. 1. Select an Episode Type from the list. The system defined values are Inpatient and Daycare. Select Inpatient to associate services to an Inpatient admitted to the facility and Daycare patient to associate services to a Daycare patient admitted to the facility.

  2. 2. Enter the details in the mandatory (*) fields. <Related Field Description>

OR

  1. Right click on Right Click for User Options to search for the relevant patient record. You can search for a patient record based on the Patients Lookup, Patient/Admission/Transfer lookup or Admission/Transfer lookup.

  2. Click [List of Values] to select an Accounting Department. The services are billed at the selected Accounting Department.

  3. Select the check box View Billing to view the billing details.

    The financial details of the patient for the selected episode get displayed. You can also view the unconfirmed services associated to the patient by clicking [Unconfirmed Services]. The Settlement Details, Billing Group details, service charges, gross amount for the services utilized, adjustments or refunds issued, unbilled and outstanding amount get displayed. The name of the User, who has recorded or modified the financial details gets displayed.

  4. Click [Future Order] to view any orders placed for future visits. If any orders are placed for future visits using the external modules, you can view the orders and the details by clicking [Future Order].

  5. Associate the Billing Services or a Billing Panel to the patient. The services or the panels are charged based on the price defined for the Billing Service. <Related Field Description>

  6. Click [Commit Form] to associate the service or [Clear Record] to clear the entered data.

  7. A Transaction Reference ID is generated on successfully associating a service or a panel.
