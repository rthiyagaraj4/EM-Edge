Enter the following details:

 

  1. Trx Type – The Transaction Type is displayed as Invoice, by default.

  2. Patient ID - Click [List of Values] to select a Patient ID. An invoice is generated for the outstanding bills of the selected patient who is linked to the selected Customer. Age, gender, name and National ID of the patient gets displayed in the respective text boxes on selecting the Patient ID.

  3. Visit Date - Click the Calendar function to select the Visit Date.

  4. Episode Type – Select an Episode Type to search for the respective Episode Types with the outstanding bill details. You can select from the list of outstanding bills of an Episode Type associated to the Customer to generate invoices for the selected Episode Type.

  5. Outpatient – Select ‘Outpatient’ to search for the Outpatient records with the outstanding bills that are associated to the selected Customer.

  6. External – Select ‘External’ to search for the External patient records with the outstanding bills that are associated to the selected Customer.

  7. Emergency – Select ‘Emergency’ to search for the Emergency patient records with the outstanding bills that are associated to the selected Customer.

  8. [Refresh] - Click [Refresh] to view SS Invoice details.

 

Siriraj Invoice Details

The invoice details associated to the patient gets displayed. SS Invoice Type, Description, Invoice Document Type, Invoice Document Number, Invoice Document Date, Policy details and Invoice Amount get displayed.

 

Service Charges

  1. Customer - Click [List of Values] to select a Customer. The values defined using 'Customer' function with the 'Customer Group' as Social Security get listed. This is a mandatory field.

 

Encounter Details for Pending Invoice

  1. Patient ID – The Patient ID selected gets displayed, by default.

  2. Episode Type – The Episode Type of the patient gets displayed, by default.

  3. Episode ID – The Episode ID of the patient gets listed, by default.

  4. Visit ID – The Visit ID of the patient gets displayed, by default.

  5. Acct Seq No – The Account Sequence Number gets displayed. For each patient encounter, when the Policy details are modified, the account sequence number gets updated.

  6. Priority – The Priority assigned to the respective Episode while recording the financial details for the patient gets displayed.

  7. Cust Code – The Customer Code associated to the selected Episode while recording the financial details for the patient gets displayed.

  8. Policy Type Code – The Policy associated to the patient while recording the financial details gets displayed.

  9. Admission  Date – The date when the patient was admitted to the facility gets displayed.

  10. Discharge Date - The date when the patient was discharged from the facility gets displayed.

  11. Additional Dtls Reqd – The check box is selected based on the settings defined for the Policy Type in ‘Setup invoice Additional Info’ function.

  12. Captured – The check box is selected, if additional information is recorded for the Policy Type and the selected Customer Code. The check box is cleared, if the details are not captured for the selected record.

  13. [C] – Click [C] to capture additional details for the selected record. You cannot generate the invoice for the selected records, if the additional details are not captured.

    Enter the following details:

    1. [Populate Additional Info] – Click [Populate Additional Info] to populate the additional details setup for the Payer Group, Payer and the Policy Type combination defined using the ‘Setup Additional Info by Payer/Policy’ tab in the ‘Setup Invoice Additional Info’ function. You can associate additional data which is not associated to the Payer Group, Payer and the Policy Type combination by selecting the values from the list.

    2. SL No – Enter the Serial Number in numeric format in four digits of maximum length. This identifies the order in which the additional information has to appear.

    3. Info ID - Click [List of Values] to select an Information ID. The Info ID’s created using the ‘Setup Data for Additional Info Identifiers’ tab using the ‘Setup Invoice Additional Info’ function gets listed.

    4. Code – Click [List of Values] to select a Code. The Codes defined for the selected Info ID gets listed.

    5. Description – The description for the selected Code gets displayed.

    6. Data – The data set while defining the Info ID using the ‘Additional Info Identifiers’ tab in the ‘Setup Invoice Additional Info’ with the ‘Data Set Available’ check box as cleared gets displayed.

    7. Mandatory – Select the check box, if the additional invoice detail is mandatory while generating the invoice. The additional details are not mandatory, if the check box is cleared.

    The button [C] is replaced by [V] adjacent to the record for which the additional details are captured successfully. [V] indicates Victory, which specifies that the additional details have been successfully captured.  

 

Outstanding Bills

  1. Reference Original Invoice – Click [List of Values] to select the original invoice number. The Original Invoice Number indicates that the invoice generated is in reference to the invoice number, which is already generated and dispatched to the Customer. The invoice reference numbers generated using the function ‘Generate Invoice – OP/EM’ gets listed.

  2. [Service Dtls] – Click [Service Dtls] to view the service details for the selected record.

  3. Select All – Select the check box to generate invoice for all the outstanding bills of the patient. Clear the check box, if the invoice has to be generated for selected bills only.

  4. Select – Select the outstanding bills against which the invoices are to be generated. You can select to print a consolidated invoice by selecting a group of outstanding bills or select individual bills to generate separate invoice for each outstanding bill.

  5. Doc Type – The Document Type selected while generating the bill for an Outpatient or Emergency or an External patient gets displayed.

  6. Doc Number - The Document Number generated for the outstanding bill gets displayed.

  7. Date – The date when the bill was generated for the services utilized by the patient gets displayed.

  8. Policy Type – The Policy Type associated to the patient gets displayed.

  9. Pending Amount – The pending amount, which has to be settled, gets displayed.

  10. Patient ID – The ID of the selected patient gets displayed.

  11. Episode Type – The Episode Type of the patient gets displayed.

  12. Episode ID – The Episode ID for the respective patient in the facility gets displayed.

  13. Visit ID – The Visit ID during which the bill was not settled, gets displayed.

  14. Total – The total outstanding bill amount gets displayed.

  15. [Generate] – Click [Generate] to generate the invoice for the selected outstanding bills. The Document Type has to be defined for the Invoice records using the function 'Document Types by Payers'. A reference number is generated on clicking [Generate]. If the diagnosis is recorded for the selected Policy Type and the diagnosis details are not recorded, the indication is displayed with color. You cannot generate the bills, if the diagnosis is not recorded.
