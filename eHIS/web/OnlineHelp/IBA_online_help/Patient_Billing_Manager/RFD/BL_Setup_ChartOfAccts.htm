Enter the following to define Chart of Accounts:

 

  1. Chart of Account Code – Click [List of Values] to select an Account Code. The values defined using the function ‘Chart of Accounts for Approval’ get listed. This is a mandatory field.

  2. Long Description – The Long Description for the relevant Account Code appears. This is a mandatory field.

  3. Short Description – A Short Description for the respective Account Code is populated from the General Ledger module. This is a mandatory field.

  4. Summary Account? – Select Summary Account to set the Chart of Account as a Summary Account. If selected, no transactions can be entered against this account in the entire Financial Accounting System.

  5. Pseudo Account Code – Click [List of Values] to select a Pseudo Account Code.  

  6. Long Description – The Long Description for the selected Pseudo Account Code appears in the relevant text box.

  7. Short Description – The Short Description for the selected Pseudo Account Code appears in the relevant text box.

 

Account Nature

The Account Nature can be “Balance Sheet” and “P & L Account”.

 

  1. Balance Sheet – Select Balance Sheet to create the selected Account Code as ‘Balance Sheet’.

  2. P & L Account – Select P & L Account to set the Account Code as P & L Account.

 

Balance Type  

The Balance Type can be either set to Debit or Credit.

  1. Debit – Select Debit to set the Account Code as a Debit type.

  2. Credit – Select Credit to set the Account Code as Credit.

 

Module Validity Flag

The values are under this option are – “GL”, “BL”, “AR”, “AP”, “FA”, “PY”, “PO”, “ST”, and “SS”. Select the check boxes to associate the modules with the Account Code.

 

Account Code Element   

The values under this option are – “Department Entry”, “Product Entry” and “Sub-ledger Entry”. This is a mandatory field.

  1. Department Entry - If Department Entry flag is "N" for this Account Code, then the default Department Code and Description specified in GL parameters are displayed in Department Code and Description. Only if the Department Entry is associated with the Posting Account, the check box should be selected. Else, the Department Code combination will not be used towards generating the Posting Account.

  2. Product Entry - If Product Entry flag is "N" for this Account Code, the default Product Group Code, Product Line Code and Description specified in GL parameters are displayed in Product Code and Description. If selected, the Product Entry is associated with the Posting Account, or the Product Entry is not considered towards the generation of the Posting Account.

  3. Sub-ledger Entry - If Sub-ledger Entry flag is "N" for this Account Code, the default Sub-ledger group code, Sub-ledger Line Code and the Description specified in GL parameters are displayed in Sub-ledger Code and the Description. Select the check box if the Sub-ledger Entry is associated with the Posting Account, or the Sub-ledger Entry is not considered towards the generation of the Posting Account.

 

Bank Account Element  

  1. Bank A/C – Select the check box when the defined Account Code is a Bank Account.

  2. Bank Reconciliation – Select Bank Reconciliation if the Bank Reconciliation option is applicable to the selected Bank Account. This option can be selected only when the Bank Account is selected.

  3. Foreign Currency A/C – Select Foreign Currency A/C if transactions related to Foreign Currencies can be processed. This option can be selected only when the Bank Account is selected.

  4. Currency Code – Click [List of Values] to select a Currency Code from the list. The ‘Foreign Currencies’ defined in the Financials Base Administration module appears in the list.
