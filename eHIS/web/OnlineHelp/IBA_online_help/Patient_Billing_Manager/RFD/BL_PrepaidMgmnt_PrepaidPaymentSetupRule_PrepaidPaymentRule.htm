You can define the prepaid payment rule for a Payer Group, Payer and Policy combination through this tab. You can specify the amount and the number of installments in which the specified amount has to be settled.

 

Enter the following details:

 

  1. Prepaid Code - Click [List of Values] to select a Prepaid Code. The description of the selected Prepaid Code is displayed. The values that are defined through the 'Prepaid Code Setup' tab are listed. This is a mandatory field.

  2. Prepaid Group/Protocols By Period - Click [List of Values] to select a Protocol Code. The description of the selected Protocol Code is displayed. The values that are defined through the 'Prepaid Group/Protocols By Prepaid' tab are listed. This is a mandatory field.

  3. Payer Group - Click [List of Values] to select a Payer Group. The description of the selected Payer Group is displayed. The values that are defined through the 'Payer Group' function are listed. This is a mandatory field.

  4. Payer - Click [List of Values] to select a Payer. The description of the selected Payer is displayed. The values that are defined through the 'Payer' function are listed. This is a mandatory field.

  5. Policy - Click [List of Values] to select a Policy. The description of the selected Policy is displayed. The values that are defined through the 'Policy Type' function in Insurance Management module are listed. This is a mandatory field.

  6. Effective [From - To] - Click the Calendar function to select a effective date range. The selected date has to be greater than the system date. The payment rule is valid during the selected period. The 'Effective From' is a mandatory field.

  7. Prepaid Amt - Enter the prepaid amount in numeric format. This indicates the prepaid amount given by a Payer for the respective rule. This is a mandatory field.

  8. Num of Payments - Enter the number of payments through which the prepaid amount has to be settled. This is a mandatory field.

  9. Reminder Option for Payments - Select a value from the list. The system-defined values are 'After Registration' and 'After Last Payment'. Select 'After Registration' to send reminder letter for payments after the registration. Select 'After Last Payment' to send reminder letter for payments after last payment.

  10. Pmnt Num - The Payment Number is displayed automatically with the first sequence number being 1. This indicates the number of installments in which the amount is settled. Click the 'Pmnt Num' field to record the installment payment details.

  11. Pmnt Ind - Select a value from the list. The system-defined values are 'Amount' and '%age'. Select '%age' to specify the settlement amount in terms of percentage. Select 'Amount' to specify a constant settlement amount.

  12. Pmnt(%) - Enter the percentage of the prepaid amount that has to be settled in the respective installment. You can enter the amount in numeric format only if the 'Pmnt Ind' is selected as '%age'. You cannot specify the percentage amount when the 'Pmnt Ind' is selected as 'Amount'.

  13. Pmnt Amount - Enter the payment amount in numeric format. The total of the payment amount has to be equal to the 'Prepaid Amount'. The amount is calculated automatically and displayed, if the 'Pmnt Ind' is selected as '%age' and the 'Pmnt(%)' value is specifed. You cannot update the value when the amount is displayed based on the percentage value specified. You can enter the value only when the 'Pmnt Ind' is selected as 'Amount'.

  14. No of Days after Resg./Last Due Date - Enter the number of days in numeric format of three digits maximum length to specify the number of days after which the reminder letter has to be sent to the Payer for settlement. The number of days has to be greater than or equal to the number of days specified for the previous record. The reminder letters are sent to the payer based on the number of days after registration or after the last payment date. This is a mandatory field.

  15. Total Payment Amount - The total payment amount is displayed based on the payment amount specified for each installment.

 

Click [Commit Form] or press [F10] to save the records or [Clear Record] to delete the data.
