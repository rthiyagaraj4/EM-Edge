Enter the following details to record or process the invoice payments:

 

  1. Payment Reference Doc – Enter the payment reference document number in alphanumeric format of 30 characters maximum length. This indicates the document number using which the details can be viewed.

  2. Pmnt Ref Date – Click Calendar function to select the date when the payment details were recorded for the selected document reference number.

  3. Invoice Dispatch Mode – Select a dispatch mode from the list. The values are system defined – ‘Manual’ and ‘Electronic’. The dispatch mode indicates the means by which the invoice was dispatched to the Customer.

  4. Manual – Select Manual to view the invoices, which were dispatched manually.

  5. Electronic – Select Electronic to view the invoices, which were dispatched through electronic transfer.

  1. Receipt Amount – Enter the Receipt Amount in numeric format. The Receipt Amount indicates the amount dispatched by the Customer to settle the outstanding bills against which the invoice was generated.

  2. Allocated Amount - The Allocated Amount in displayed in numeric format indicating the agreed amount for the Policy and the Payer combination. You cannot update the Allocated Amount.

  3. Balance Amount - The Balance Amount is displayed after deducting the allocated amount from the receipt amount. You cannot update the Balance Amount.

  4. Payer Name – Enter the name of the Payer in alphanumeric format of 100 characters maximum length. The invoices dispatched to the respective Customer for the outstanding bills are listed. You can search and associate the invoices using the ‘Invoice/Debit Note/Credit Note Details’ tab.

  5. Receipt Option – Select a Receipt Option from the list. The values are system defined – ‘Invoice-wise’ and ‘Consolidated by Customer’.

  6. Invoice-wise – Select Invoice-wise to list and to settle the payment against the individual invoices generated for the outstanding bills.

  7. Consolidated by Customer - Select Consolidated by Customer to list and to settle the payment against the consolidated invoices generated for the outstanding bills.

  2. [A] - Click [A] to record adjustment details.

Invoices for Settlement

Using the Invoices for Settlement tab, you can select the invoices from the list against which the receipt amount has to be applied. The invoice details generated using the functions ‘Generate Invoice – OP/EM’ or ‘ Generate Invoice – IP/DC’ functions get listed. The transactions for the selected invoice gets listed. You can edit the following details:

  1. Prepaid - The check box is selected to indicate that the invoice is generated for a prepaid transaction. The check box is cleared, if the invoice is generated for a non-prepaid transaction.

  2. Prepaid Recevd Amt - The prepaid amount received from a payer for the respective invoice is displayed.

 

Bill Details

The bills against which the respective invoice is generated are also listed. Click [D] to view the service details. You can view the Service Details against which the amount was not settled.

 

Invoice/Debit Note/Credit Note Details

The Invoice/Debit Note/Credit Note Details tab enables you to view and to search for the invoices raised. The records are listed based on the search criteria.

 

Enter the following search criteria:

  1. Payer – Click [List of Values] to select the Payer. The Payer defined using the ‘Customer’ function in the AR Setup menu gets listed.

  2. Dispatch Ref – Click [List of Values] to select the Dispatch Reference Number. The invoices dispatched with the selected Dispatch Reference Number get listed. The dispatch document reference number generated while finalizing the dispatch document for the selected Customer gets listed.

  3. Invoice No - Click [List of Values] to select the Invoice Number. The Invoice Document Type along with the system-generated number gets listed.

  4. Dispatch Mode – Select the Dispatch Mode as either ‘Electronic’ or ‘Manual’. The invoices dispatched with the selected Dispatch Mode get listed.

  5. Encounter Type – Select an Encounter Type from the list. The invoices generated for the selected Encounter Type gets listed.

  6. Encounter ID - Click [List of Values] to select the Encounter ID. The Encounter ID for the selected Encounter Type is listed.

  7. Patient ID - Click [List of Values] to select a Patient ID. The invoice generated for the outstanding bills of the selected patient gets listed.

  8. [Search] - Click [Search] to view the records based on the search criteria specified. The invoice generated gets listed.

  9. Last Pmnt – The check box has to be selected, if the payment is the final payment received from the payer and the amount is lesser than the expected less payment for the payer and policy combination defined in the Insurance Management module.

  10. Select – Select the check box against the records to allocate the records under the selected Payment Document Reference indicating that the amount is settled for the selected invoice.

 

Lookup for Search and Invoice Selection

You can search and select the invoice to record or process payments.

Search Criteria

You have to specify either the Payer, patient ID or the encounter ID.

  1. Payer - Click [List of Values] to select the Payer who has made settled the bill amount.

  2. Dispatch Mode - Select a dispatch mode from the list. The system-defined values are 'Manual' and 'Electronic'. Select Manual to dispatch the invoice manually or Electronic to dispatch the invoice automatically.

  3. Dispatch Ref - Click [List of Values] to select the dispatch reference number. The dispatch reference number generated through 'Generate Dispatch Number' are listed.

  4. Patient ID - Click [List of Values] to select the patient ID. The patient records with pending payments and associated to the selected Payer are listed.

  5. Encounter Type - Select an Encounter Type from the list. The system-defined values are 'Inpatient', 'Outpatient', 'Daycare' and 'Emergency'.

  6. Encounter ID - Enter the Encounter ID of the respective patient in the facility. The values are displayed based on the details recorded.

  7. Invoice Date [From - To] - Click the Calendar function to select the invoice date range. The invoices generated during the selected date range are displayed. The Invoice Date selected for 'To' field has to be greater than or equal to the value selected for 'From' field.

  8. Confirmation No - Click  [List of Values] to select the confirmation number. The dispatch document number generated through the 'Generate Dispatch Document' are listed.

  9. Confirmation Date - Click the Calendar function to select the confirmation date. The confirmation date indicates the date when the dispatch document number is generated.

  10. Invoice No [From - To] - Click  [List of Values] to select a range of invoice numbers. The invoice numbers are generated through the 'Generate Invoice' function. The values are listed based on the invoice numbers selected. The Invoice Number selected for 'To' field has to be greater than or equal to the value selected for 'From' field.

  11. [Search] - Click [Search] to query the records based on the search criteria.  The invoice details, dispatch details and the payable amount matching the search criteria are displayed on the search criteria.

  12. Printed - The check box is selected, if the records are printed. The check box is cleared, if the record is not printed. You cannot select or clear the check box.

  13. Direct Inv/Dbn/Crn - The check box is selected, if the record is a direct invoice, debit or credit note.

  14. Select - Select the check box to allocate amount for the respective outstanding payment.

  15. [Allocate] - Click [Allocate] to allocate the amount for respective records. The control is transferred to 'Invoices for Settlement' tab. You can record the adjustments against invoice, debit or credit note.

 

Settlement Details

The amount posted by the Customer to settle the payment for the invoices dispatched is recorded using this tab.

  1. Cash Counter Code - Click [List of Values] to select the Cash Counter Code. The Cash Counter indicates the counter where the settlement is made.

  2. Narration - Enter the narration in alphanumeric format.

  3. Payment Through Mail - Select the check box, if the payment is received through mail for the invoices posted.

  4. SL No - Enter the serial number in numeric format.

  5. Amount Settled - Enter the amount settled in numeric format.

  6. Settlement Type - Click [List of Values] to select the Settlement Type. The settlement type indicates the mode of payment.

  7. Cash Given - Enter the Cash amount in numeric format.  The cash amount indicates the payment received against the invoices posted.

  8. Returned - Enter the cash amount returned in numeric format. This indicates the cash amount returned to the Customer after deducting the outstanding amount.

  9. Cheque/Card No - Enter the Cheque number or card number in alphanumeric format of 20 characters maximum length. If settlement type is non Cash, then these are mandatory. Otherwise, this should be left blanks. Enter the details of the Cheque or a Credit Card. For example, if the settlement is by Credit Card, enter the Credit Card identity and expiry date of the Credit Card. If the settlement type is by Cheque, enter the Cheque details and the date.

  10. Date – Click Calendar function to set a valid date. Enter the date of the Cheque, if the settlement type is Cheque or if the settlement type is Credit Card, enter the date of the expiry of the Credit Card.

  11. Bank - Click [List of Values] to select the Bank name.

  12. Bank/Card Co. Name/Branch – Enter the Bank or Credit Card company name in alphanumeric format of 30 characters maximum length. If settlement type is non Cash, then this is mandatory. Otherwise, this should be left blanks. For example, if the settlement is by Credit Card, enter the Credit Card company name. If the settlement is by Cheque enter the banks name or banks’ branch name. Click [List of Values] to view and select an approved Credit Card company name or an approved bank or branch.

  13. Online Approval – If the check box is selected, Credit Card entered receipt will be forwarded to the Credit Card Company immediately. If the check box is cleared, Credit Card receipts will be forwarded as batch. In case of Online Approval, Batch No. information to be filled manually.

  14. Batch No – Enter the batch number if settlement type is of Credit Card.

  15. Approval Code – Enter the Approval Code with a maximum of 30 characters. If the settlement is made through Credit Card, Approval code of the Credit Card to be entered here.

  16. Sale Draft No – Enter the Sale Draft Number with a maximum of 20 characters. When settlement is made by Credit Card, sale draft number details captured at the time of swiping to be recorded here.

  17. Total - The total amount collected from the Customer is displayed.

  18. Settlement Type - The description for the selected Settlement Type gets displayed.

  19. Payer’s Name - Enter the payer name with a maximum of 40 characters. This field is to be entered if the receipt nature code is ‘MI’.

  20. Payer’s Ref ID – Enter the Payer’s Reference ID with a maximum of 30 characters. When Payer name is captured, is having any reference ID, can be entered as input for this field.

  21. Nominee - Enter the name of the nominee in alphanumeric format.

  22. Finalized – Select the check box to finalize the payment against the invoices selected. Clear the check box, if the payment is not valid for the selected invoices.

  23. [Finalize] – Click [Finalize] to finalize and to settle the received amount on the selected invoices. The Receipt Amount has to match with the allocated amount and the settlement amount to finalize the document.

  24. [Un-Finalize] - The legend changes to [Un-Finalize] only if the invoice payment records are finalized. An authorization screen pops-up on clicking [Un-Finalize]. Enter the Username and PIN number to update the invoice payment details. Click [OK] to login and update or [CANCEL] to cancel the process. A new field 'Record Adjustments' with a button [A] is seen under the 'Invoices for Settlement' tab. Click [A] to modify the details. Record Adjustments screen is displayed with the patient details and pending document details. You can update the Narration, Credit and the Document Amount. Users with 'Un-Finalize Invoice Payment' check box as selected in 'Users for Services' function in Administration menu are permitted to un-finalize invoice payments.

  25. [Generate Receipt] - Click [Generate Receipt] to generate the receipt for the payment received to settle the outstanding amount. You can generate the receipts only after recording the settlement details and finalizing the payment.
