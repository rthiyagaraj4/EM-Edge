Enter the following details to enter inpatient receipts or refund details:

  1. Downtime Entry – If the receipt is a manual receipt entered when the system was down, this flag should be made as ‘Y’. By default his flag is ‘N’.

  2. Document Type/No – If Downtime Entry flag is ‘Y’, then the document type is defaulted from billing parameter setup. The document number is to be entered and cannot be system generated. The document number should be unique.

  3. Date – If Downtime Entry flag is ‘Y’, then the date is mandatory.

  4. Reason – If Downtime Entry flag is ‘Y’, then the reason is mandatory. The reason description is displayed.

  5. Patient No - Enter a valid patient number. If the receipt nature code is “CU” or “MI” then entering a Patient no is optional. For all other receipt nature codes, patient no is mandatory.

If the patient entered is valid, the patient details are displayed are displayed for user verification.

  1. Bed Number - Enter a valid Bed Number. This field is skipped if an valid Patient No is entered. If an invalid Bed Number is entered an error message is displayed.

  2. Type - System displays the episode type as “I”. If a valid patient no is entered this can be changed to “O” for Outpatients and “R” for Referral Patients.

  3. Nature - Enter a valid nature code. Valid nature codes for receipts are as follows:

  4. "AD" - Adjustable Deposits

  5.  "PR" - Prepayments

  6. "RD" - Refundable Deposits

  7. “PD” – Patient Deposit Against Package

  8. "BI" - Against Bills

  9. "CU" - Customer Receipts

  10.  "DC" - Receipts from Deferred Cash Patients

  11.  "MI" - Miscellaneous

 

Valid nature codes for refunds are as follows:

  1. "RD" - Refundable Deposit Refund

  2. "BI" - Refund against a bill

  3. "MI" - Miscellaneous

  4. Package Seq. No - The Package Sequence number is displayed automatically based on the package definition and the package subscription for the respective patient.

  5. Receipt or Refund - Click on “Receipts” (“R”) or “Refunds” (“F”). If "R", RECEIPTS is displayed for user verification. If "F", then REFUNDS is displayed for user verification. Process details

During [Commit Form], the following checks are carried out on-line. The updates are discussed below based on receipts or refunds.

 

Receipts Update  

  • If receipt nature code is "AD", then a record is written into the Patients Ledger with this document details. The Financial Details of the patient are updated for the adjustable deposit according to the episode type. If Outpatients, Visit, Episode and Patient Financial Details are updated. If Inpatients, Episode and Patient Financial Details are updated. If Referrals, Patient Financial Details are updated.

  • If receipt nature code is "PR", then a record is written into the Patients Ledger with this document details. The Financial Details of the patient are updated for the prepayment amount according to the episode type. If Outpatients, Visit, Episode and Patient Financial Details are updated. If Inpatients, Episode and Patient Financial Details are updated. If Referrals, Patient Financial Details are updated.

  • If receipt nature code is "RD", then a record is written into the Patients Ledger with this document details. The Financial Details of the patient are updated for the refundable deposit amount according to the episode type. If Outpatients, Visit, Episode and Patient Financial Details are updated. If Inpatients, Episode and Patient Financial Details are updated. If Referrals, Patient Financial Details are updated.

  • If receipt nature code is "MI", then it has no relevance with Patients or Customers or Deferred Cash Patients. General Ledger distributions are created when the receipts are posted.

  • If receipt nature code is "BI", then the bill's outstanding amount is reduced by the document amount. As a result, if the amount's sign changes after the update, then it is an error.

  • If receipt nature code is "DC", then the deferred Cash patient's bill outstanding amount is reduced by the document amount. As a result, if the amount's sign changes after the update, then it is an error.

  • If receipt nature code is "CU", then there is no update at all. When the receipt document is posted, the receipt document is written onto the customer's account in Accounts Receivable module.

 

Refunds Update

  • If receipt nature code is "RD", then the original document is updated. Before updating, system checks if the document amount and the outstanding amounts are the same for the original document. If not the same, it is considered to be an error. If no error, then the outstanding amount is set as zero for the original document.

  • If receipt nature code is "MI", then it has no relevance with Patients or Customers or Deferred Cash Patients. General Ledger distributions are created when the refunds are posted.

  • If receipt nature code is "BI", then the bill's outstanding amount is increased by the document amount. As a result, if the amount's sign changes after the update, then it is an error.

  • After the [Commit Form] operation is successful, press Return to print the document. The document, depending on whether it is receipt or refund, is printed on the preprinted stationery. Users must ensure that the preprinted stationery are always loaded on the printer so that the document can be printed.

 

Once printed, the document cannot be recalled for correction.
