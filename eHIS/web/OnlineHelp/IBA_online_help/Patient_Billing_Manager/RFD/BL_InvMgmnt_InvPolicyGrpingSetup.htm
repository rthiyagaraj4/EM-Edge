Enter the following details:

 

  1. Facility ID - The name of the logged-in Facility is displayed, by default. You cannot change the Facility ID.

  2. Payer Group Code - Click [List of Values] to select a Payer Group Code. The description of the selected Payer Group is displayed.

  3. Payer - Click [List of Values] to select a Payer Code. The description of the selected Payer is displayed.

 

Policy Group By Payer

  1. Policy Group Code - Enter the Policy Group Code in alphanumeric format of eight characters maximum length. This is a mandatory field.

  2. Policy Group Desc - Enter the Policy Group Description in alphanumeric format of 100 characters maximum length. This is a mandatory field.

  3. Effective Date [From - To] - Click the Calendar function to select the date range. The 'Effective From' date has to be greater than the current system date and lesser than the 'To' date. The 'To' date is mandatory. The Policy Group created is valid for the selected period.

  4. OP/AE - Select the check box, if the Policy Group is valid for Outpatients' or Emergency patients' in the facility. The check box is selected, by default. Clear the check box, if the Policy Group is not valid for Outpatients' or Emergency patients' in the facility.

  5. IP/DC - Select the check box, if the Policy Group is valid for Inpatients' or Daycare patients' in the facility. The check box is selected, by default. Clear the check box, if the Policy Group is not valid for Inpatients' or Daycare patients' in the facility.

  6. [Policies by Policy Group] - Click [Policies by Policy Group] to navigate to the 'Policies by Policy Group' tab and associate Policy Types to the Policy Group.

 

Policies by Policy Group

  1. Policy Grp Code - The code and description of the selected Policy Group code in 'Policy group By Payer' tab is displayed as read-only text. You cannot edit the value.

  2. Policy Type Code - Click [List of Values] to select a Policy Type Code. The selected Policy Type is associated to the Policy Group.

  3. Policy Description - The description of the selected Policy Type is displayed.

  4. Primary Policy - Select the check box to indicate that the Policy Type is a primary policy. You can select only a single record as a Primary Policy. The Invoice Format,  Signatory and agreed amount are displayed based on the Primary Policy selected. The values are populated from the Invoice Process Parameters.
