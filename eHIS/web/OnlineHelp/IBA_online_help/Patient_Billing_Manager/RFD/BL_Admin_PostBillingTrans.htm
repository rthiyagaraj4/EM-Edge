Existing functionalities of this function is subject to the processing/validations mentioned below:

 

  • While posting the patient folio transactions for income and discounts, system checks whether the transactions are updated with the miscellaneous document reference. The validation takes place when Financial Accounting cashier generates miscellaneous receipts in acknowledgement of cash collection from the Cash Counters.

  • The transactions are updated with the miscellaneous document reference when receipts are successfully generated using the function 'Cash Control Receipts'. The transactions at the logged in Cash Counter, relevant shift and date are updated with miscellaneous receipt Document Type and Number.

  • For each transaction, bill document reference must have the miscellaneous document type and number. The posting fails, if the miscellaneous receipt document type and number is null.

  • If Miscellaneous Document Type/Number is null in the bill document for the posting date, the folio records are not posted. If the Miscellaneous Document Type/Number is not null, the posting takes place and miscellaneous document type and number stored against each record is posted in to the GL Distribution along with the other transaction details. Later on while querying/reporting the GL Distribution, miscellaneous document reference for each record written into distribution table are also displayed.

  • The above said validation is in addition to the existing validations done by system while posting the patient charges folio for income and discounts.

  • Patient charges folio has one more flag ‘Confirmation’. This has been provided to mark the billing service requests of both OP and IP as being provided to the patient. The billing services requests marked as confirmed can also be cancelled under any circumstances. Please refer to Cancel and Adjust Direct Billing Items for IP and OP for more details.

  • In addition to the above, system checks for the existence of Billing Group – Price Class combination against each record to be posted.

  • When a bill or receipt/refund is being posted, system checks for the date, Cash Counter and shift. System generates a miscellaneous receipt document. The cashier transactions are to be updated with the Miscellaneous Document Type and Number. This takes place when the Financial Accounting cashier generates a miscellaneous receipt in acknowledgement of cash collection from the Cash Counters/Cashiers.

  • Whenever receipts are successfully generated using the new function 'Cash Control Receipts', the update process is triggered. The transactions processed associated to the Cash Counter, Shift and date are updated with receipt Document Type and Number, since all the bills and receipts contain the Cash Counter Code, shift number and date. In the case of bills, when the receipt document reference is updated, the patient charges folio against each source document reference also gets updated.  

  • Each receipt/refund transaction Document Type and Number includes the. If Miscellaneous Document Type and Number for a respective date, against any of the records to be posted to the GL Distribution, the posting fails. If Miscellaneous Document Type/Number is null for any of the receipt and refund documents selected for the posting date, the cash transaction records are not posted.

  • The Miscellaneous receipt Document Type and Number are stored as a part of the billing and receipt transactions that are transferred to the GL Distributions. The Miscellaneous Document Type and Number stored against each record are also posted to the GL Distribution along with the other transaction details. While querying/reporting the GL Distribution, miscellaneous document reference for each record written into distribution table gets displayed.

  • Posting month and year for each record written into GL Distribution is as per the ‘Post Month/Year’ assigned at the time of generation of Miscellaneous Document Type Code and Number by the financial accounting cashier using the function 'Cash Control Receipt'.

  • In the case of exempted patient where zero bills are generated for the transactions, Miscellaneous Document Reference for the record is updated. This facilitates posting zero bills, as the transactions are not a part of the Cash Counter transactions.

  • A cash control receipt for zero bills is generated for every month. The amount for the cash control receipt is zero since it is a sum of all zero bills for the month.

 

GL Interface from Billing Module

  • Bills, receipts and refund transactions are posted during this process. A bill, receipt and refund document is selected for posting only if the bill, receipts and documents are included in a Cash Control Receipt.

  • When a bill is posted, the billing receivable account defined against the Bill Type is debited with the bill amount; Income Accounts defined for a Billing Service is credited with the gross charge amount and Discount Accounts defined for the Billing Group are debited with the discount amount.

  • When a receipt raised against a bill is posted, the billing receivable account defined against the Bill Type is credited and the account defined for the Cash Counter and Settlement Type are debited.
