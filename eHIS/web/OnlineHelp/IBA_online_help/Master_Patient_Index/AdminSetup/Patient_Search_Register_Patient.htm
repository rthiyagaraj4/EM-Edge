Patient Search (Register Patient)

From this function you can Search records of patients by specifying the search criteria and Register Patients if a record do not exist for the specified criteria.

Patient Search is a common function used by all the modules of Enterprise Management to retrieve records of patients by specifying the search criteria. Based on the specified search conditions, records are displayed for selection. When the search function retrieves a single record, the control automatically returns back to the called function along with the retrieved data. If the search function retrieves more than one record, these records are displayed in the result area. You can select the appropriate record. On selecting the required record, the search screen closes and control shifts back to the called function.  

In MP Parameter, if 'Accept Patient Name in Local Language' is selected, the patient name is entered in local language as one of the search criteria. If 'Maintain Family Link' is set in MP parameter, the Head Patient Identifier is specified for searching the required patient. Based on the result formats specified in the function 'Patient Search Result Format', the retrieved details are displayed in the result area.  

Searching Patient Records

From the MP Transactions > select the Query function for any transactions.

The Patient Search screen gets displayed.

  1. The screen displays two tabs 'Basic' and 'Advanced'.

  2. Enter details in the 'Basic' and 'Advanced' tabs .

  3. Click [Search] to retrieve the patient records based on the specified criteria.

  4. The Patient Records are listed for the specified criteria.

  5. Click [+] to view the Patient Details. The patient name is displayed in local language also.

  6. Click 'PC' hyperlink to view the Encounter Details.

  7. Click Multiple MRNs hyperlink to view the Patient File No’s associated to the Patient ID.

  8. [Register Patient] is enabled if the specified criteria do not retrieve any records.

  9. Click [Register Patient] to register patient.

 

Searching Patient Records using Emirates ID

Scenario 1 - When the patient comes to the hospital for the first time:

1. Launch the “Patient Search Screen” and swipe the Emirates ID card; the temporary file “Emirates_ID.txt” will be created in the (C:\SmartIC) folder

2. Clicks the “M” button; the system will populate the details from the temp file into the relevant fields and deletes the temporary file in the system.

Scenario 2 - When the Patient is already registered with the hospital:

1. Launch the “Patient Search Screen” and swipe the emirates ID card; the temporary file “Emirates _ID.txt” will be created in (C:\SmartIC) folder

2. Click the “M” button; the system will check for the Patient details in EM and populate the details which will be based on the NRIC Number.
