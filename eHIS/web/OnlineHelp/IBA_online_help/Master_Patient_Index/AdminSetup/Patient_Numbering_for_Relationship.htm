Patient Numbering for Relationship

From this function you can Define rules for generation of patient identifier for a Relationship Level.

This function is used to define the rules for generation of patient identifier for a Relationship Level. Patient Numbering for Relationship depends on the characteristics of the selected Patient Series. The range specified for a Patient Series is used to generate Patient ID in the defined format for the selected Relationship during Patient Registration.  You cannot access this function if ‘Generate Patient ID Using Alternate ID1 Rule’ check box is cleared in MP Parameter function. A message ‘This function is not applicable because Alternate ID1 Rule is not applicable’ is displayed on selecting this function if the ‘Generate Patient ID Using Alternate ID1 Rule’ check box is cleared in MP Parameter.

Defining Patient Numbering Rules for Relationship

From the MP Admin Setup > MP Admin Setup, select the Patient Numbering for Relationship function.

The Patient Numbering for Relationship screen displays.

  1. Select the desired Patient Series from the list. This field is mandatory.

  2. On selecting the Patient Series, the Separator Value and the Separator Position set for the selected Patient Series and the Relationships with all the levels are displayed.

  3. Enter the details in the mandatory fields. <Related Field Description>

  4. Click [Apply] to save the Patient Numbering for Relationship record or [Reset] to clear the entered data.

 

The prefix values recorded in this function for a relationship are generated along with Separator Value which is suffixed or prefixed based on the settings in the Patient Series function while registering a patient for the relationship.
