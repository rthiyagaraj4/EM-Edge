Patient Series - If 'Details Unknown' is selected, then the patient series is defaulted to Emergency series value set in MP Parameter.  If the details are known, then all the patient series belonging to emergency series type and 'Unrestricted Series' are available in the list . Select the required series. On selecting the Patient Series, you can capture the mandatory fields which is controlled internally.

 

Patient ID generation is based on the selected patient series. If the system automatically generates Patient ID for the selected series, you are not allowed to enter the Patient ID. The Patient field becomes disabled/dimmed.

 

Users in UAE, click the button 'M' available next to the field Emirates ID. The screen gets populated with the patient’s information.

 

  1. Citizen ID - Enter the Citizen ID in the numeric format as specified in MP Parameter.  The displayed field legend corresponds to the 'Prompt' set in MP Parameter.  

  2. Membership - Select the Membership from the list. The listed values are Organization Member, Related Member - Family, Related Member - Non Family, Non-Organization Member. It is mandatory to select the Alternate ID as set in the field 'Alternate ID Type for Membership ID' in MP Parameter function. This field is displayed if 'Maintain Organization Membership' check box is selected in the MP Parameter function.

  3. Organization - Select the Organization using [?] function. The Organization Sub Unit is displayed on selecting the Organization. If more than one Sub Unit is associated to the Organization, select the Sub Unit from the list.

  4. Head of Family Patient ID - This field is displayed based on the ‘Maintain Family Link’ set in MP Parameter. If family link is to be maintained, enter the Head Patient ID in the required format as specified in the MP Parameter, or click on the [?] to select the required Head Patient ID. The Head patient name is displayed next to the field.

  5. Relationship - The Relationship is displayed as set in the 'Relationship for Organization Member' field in the MP Parameter function on selecting the Membership as 'Organization Member'. Select the Relationship to the displayed Head of Family. This field is mandatory.

  6. Relationship Level - This field is displayed on selecting the Relationship with a level defined in the Relationship function. Select the Relationship Level using [?] function. The Relationship Levels defined in the Relationship Level function is displayed. This field is mandatory.

 

Patient Names

 

The multi parts of Patient Name are the Prefix, First Name, Second Name, Third Name, Family Name and Suffix. Based on the setup in MP Parameter (Legal Names), the multi parts of patient name are displayed. The displayed titles and order of appearance are specified in the parameter function.  If ‘Accept Patient Name in Local Language’ is set in the MP Parameter, the patient name can be entered in local language. If a '*' sign appears next to the name part, then it is mandatory to enter data.

 

  1. Title - Select the Title from the list. The listed values are defined in the function 'Name Prefix'.

  2. First Name - Enter the First Name in alphanumeric format as specified in MP Parameter.

  3. Second Name - Enter the Second Name in alphanumeric format as specified in MP Parameter.

  4. Third Name - Enter the Third Name in alphanumeric format as specified in MP Parameter.

  5. Family Name - Enter the Family Name in alphanumeric format as specified in in ‘Legal Names’ tab in MP Parameter. This is applicable irrespective of whether the 'Accept Patient Name in Multi parts' check box, Suffix or Prefix check box is selected or not. But if any other Name component check box is selected other than the ‘Family Name’ check box  in MP Parameter, the Family Name field size will be changed to 20 characters.  This is a mandatory field.

  6. Suffix - Select the Suffix from the list.   

 

On entering the patient’s name, the system builds the patient name based on the derivation logic defined in MP Parameter and displays them in the text area after Patient ID value.  If the details of the patient are not known, then 'Unknown (Gender of the patient)' appears.

 

  1. Gender - Select the Gender of the Patient from the  list. The listed values are Male, Female and Unknown. This is a mandatory field. Based on the routine defined for the National ID in MP Parameter, the gender is displayed automatically on entering the national ID.

  2. Age - Select the check box to enter the Age of the Patient in years, months and days. If age is entered, the system calculates the date of birth and displays it automatically in the ‘Birth Date’ field. Enter either Age or Date of Birth. On entering one of them, the other field value is disabled. If National ID is entered, the age is displayed automatically based on the age validation logic defined in the routine associated with the National ID.

  3. Y/M/D - Enter the Age of the patient in years, months and days in the respective fields. If National ID is entered, the age is calculated and displayed automatically based on the age validation logic defined in the routine associated with the National ID.

  4. Birth Date – Enter the Birth Date in dd/mm/yyyy format. On entering the Birth Date, the system calculates the Patient’s age and displays it in the Age (Y/M/D) fields. This field is mandatory. If National ID is entered, the Date of Birth is displayed automatically based on the age validation logic defined in the routine associated with the National ID. Optionally, select the date of birth using the Calendar function.

  5. Birth Place - Enter the Birth Place of the Patient in alphanumeric format of 30 characters maximum length.

  6. Blood Group - Select the Blood Group from the list. The listed values are defined in the Medical Records module.

  7. Nationality - The default Nationality set in MP Parameter is displayed. Select the required value using the [?] function. This field is mandatory. If National ID is entered, the default Nationality set in MP Parameter is displayed and disabled.

  8. Patient Category - Select the Category of the Patient from the list. The listed values are defined in the function ‘Patient Category’ and the patient category must be associated to the Membership using ‘Pat Cat by Org. Membership’ master function.

  9. Residency – The residency of the patient is automatically validated by the system on selecting the nationality of the patient (other than the default nationality) at registration. The system automatically validates the Nationality on selecting the Residency and the ‘Nationality’ of the patient is changed from default value to other value. Select either ‘Citizen’ or ‘Non-citizen’.

  10. Status - Select either ‘Legal’ or ‘Illegal’ as citizen status. By default, ‘Legal’ is selected. If residency is set as ‘Citizen’, ‘Legal’ is selected and disabled. If the residency is set as ‘Non-Citizen’, status can be either ‘Legal’ or ‘Illegal’.

  11. Race – Select the Race from the list. This field is mandatory. The listed values are defined in 'Race' function.

  12. Ethnicity - Select the Ethnicity from the list. The listed values correspond to the selected race.

  13. Religion - Select the Religion of the Patient from the list. The listed values are defined in the function 'Religion'.

  14. Language - Enter the Language in alphanumeric format of 20 characters maximum length or select the Language using [?] function.

The Other Alternate ID appears as a separate field for each Alternate ID, if they are defined in MP Parameter under ‘Alt ID Checks’. The field legends depend on the short description of the ‘Alternate ID Type’.
