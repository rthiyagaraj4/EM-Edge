Enter the following details to define Package Parameters:

 

  1. Package Effective from Back Dated (Days) - Enter the number of days in numeric format of three digits maximum length to record the number of days allowed for package creation from the current system date. You can specify the number of days if the check box 'Package Applicable' is selected.

  2. GL Posting by Summary - Select the check box to post summarized General Ledger details.

  3. Validity for Availing Package Service Orders (Days) - Enter the number of days to avail the package services in numeric format of three digits maximum length. You can specify the number of days if the check box 'Package Applicable' is selected. It indicates the grace period to utilize the package services.

  4. Variance Service Code - Enter the Billing Service Code for variance. Variance is computed as [Package Price - Actual Service Price].

  5. Package Refund Service Code - Enter the package service code to be used for refund in the text field. This is applicable only for users in ME region.

  6. Auto Refund Applicable - Select the check box if automatic refund is applicable. The check box is selected in 'Package Definition' function enabling you to issue a refund, if you select the check box. You cannot issue a refund if the check box is cleared.

  7. Validity for Package Service Refund (Days) - Enter the number of days in numeric format of three digits maximum length. It indicates the grace number of days to issue a refund for the unavailed services in the package.

  8. Patient Deposit for Package - Click [?] function to select a patient deposit receipt type for package transactions. The Receipt Types that are defined through the 'Receipt Types' function with the 'Receipt Nature' as 'Patient Deposit' is listed. A receipt number is generated with the selected receipt type when patient deposit is recorded through the 'Enter OP Receipts/Refunds' or 'Enter IP Receipts/Refunds' function.

  9. Apply customer package discount – New packages – Select the check box to apply customer package discount to new packages. It is one- time setup.

When the check box is selected, the Package Discount tab is disabled while creating a new package in the Package Definition screen.  The system automatically applies all the active / future discounts to the package. Upon querying the package, ‘Package discount’ tab gets enabled and you can view the discounts applied in package discount tab. This is applicable for users in MMS.

  1. Apply Customer Discount Policy – Select the check box to manage the policy level discount. By default, the check box is not selected.

When the check box is selected, the ‘Policy’ gets enabled with the value ‘All policies’ in addition to the policies filtered for the selected Customer group and customer combination in Package Definition screen. This is applicable for users in MMS.

  1. Package Deposit during subscription – Select the check box to invoke the Package Deposit screen while subscribing the package.

  2. Auto Distribute Package Variance to Services - OP - Select the check box to define distribution of package variance amount for outpatients. Distribution is to be identified for Services / Pharmacy/Medical items or both. By default, both the Services and Pharmacy/Medical Items check box are cleared. Select the ‘Service’ or ‘Pharmacy/Medical Items’ check box or both as required.

  3. Auto Distribute Package Variance to Services – IP - Select the check box to define distribution of package variance amount for inpatients. Distribution is to be identified for Services / Pharmacy/Medical items or both. By default, both the Services and Pharmacy/Medical Items check box are cleared. Select the ‘Service’ or ‘Pharmacy/Medical Items’ check box or both as required.

• When the Services checkbox is selected and Pharmacy / Medical Items check box is not selected, the distribution of the package variance is done considering only the services and ignoring the PH / Medical Items.

• When the Pharmacy / Medical Items checkbox is selected and Services checkbox is not selected, the distribution of the package variance is done considering only the PH / Medical Items and ignoring the services.

• If both the Services and Pharmacy / Medical Items check boxes are selected, the distribution of the package variance is done considering both the service and PH / Medical Items.

• If both the Services and Pharmacy / Medical Items check boxes are not selected, the system will not distribute the package variance amount.
