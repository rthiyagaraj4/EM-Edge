Package Price Revision

From this function you can Revise single and bulk update of the package and discount price revision of standard and customer price

The Package Price Revision is used to perform single and bulk update the package and discount price revision of standard and customer price.

Updating Single/Bulk Package Price details

From the Setup > Package, select Package Price Revision Function.

The Package Price Revision screen gets displayed.

  1. Enter the details in the mandatory (*) fields. <Related Field Description>

  2. Click [Search]. The search results appear based on the entered criteria.

  • If Individual Package is selected as search criteria, two tabs Package Price Revision and Discount Price Revision appear in the search result.

    The following details are displayed in the Discount Price Revision tab:

     

    Package Code, Package Desc, Billing Group, Billing Class Current Discount Value, Current Discount Type, Revised Revision Value Revised Revision Type, Effective [From – To]

     

    On click of Package Code, the following details are displayed:

     

    1. Click {Search] to find the relevant record.

    2. Package Code – The Package Code is displayed.

    3. Package Desc – The Package Description is displayed.

    4. Billing Group – Select the Billing Group using lookup. The Billing groups defined in the Billing masters are displayed.

    5. Billing Class - Select the Billing classes using lookup. The Billing classes defined in the Billing masters are displayed.

    6. Current Discount Value – The current discount value of the package is displayed. You cannot edit the value.

    7. Current Discount Type – The current discount type of the package is displayed. You cannot edit the value.

    8. Revised Revision Value – Enter the discount value of the package. Based on this value, the

    9. Revised Revision Type – Select the Price or Percentage for the revision type.

    10. Effective [From – To] – Select the package date range using calendar function.

    11. Click [Add] to add the new row.

    12. To remove the record, select the check box against the record and click [Remove].

    ‘Effective From Date’ should not be greater than existing ‘Effective From Date’. ‘Effective To Date’ should not be lesser than existing ‘Effective To Date’.

    The following details are displayed in Package Price Revision tab:

    Package Code, Package Desc , Package Description, Billing Group, Billing Class, Current Discount Value, Current Discount Type , Revised Revision Value, Revised Revision Type, Effective [From – To]

     

    ‘Effective From Date’ should not be greater than existing ‘Effective From Date’. ‘Effective To Date’ should not be lesser than existing ‘Effective To Date’.

    Click on Package Code, the following details are displayed:

     

    1. Package Code - The package code is displayed.

    2. Package Desc - The description of the package is displayed. You cannot edit the data.

    3. Customer Group - The customer group is displayed. You cannot edit data. This field is displayed only when you select a value 'Customer Price' in Std/Customer Price drop-down.

    4. Customer - The customer Name is displayed. You cannot edit the data. If it is bulk package, ** is displayed.  This field is displayed only when you select a value 'Customer Price' in Std/Customer Price drop-down.

    5. Billing Class - The Billing Class of the selected package is displayed.

    6. Current Price - The current Price of the package is displayed.

    7. Revised Price - Enter the revision amount in the numeric format.

    8. Type - Select Amount of Percentage based on which the package amount is to be revised.

    9. Effective [From-To] - Select the revised amount effective date using calendar function based on which the package is updated.

    10. Click [Add] to add the new row.

    11. To remove the record, select the check box against the record and click [Remove].

  • If Bulk Update is selected as search criteria, two sections Package Price Bulk Update and Package Discount Bulk Update appear in the search result.

    The following details are displayed in Package Discount Bulk Update section

     

    1. Discount Type - Select the discount type 'Amount' or 'Percentage' from the drop -down list.

    2. Percentage / Amt - Enter the value for percentage or amount in numeric format.

    3. Mark Up/Down - Select Up or Down from drop-down list. Based on the selection, the Package price is updated.

    4. Round Off - Enter the round off value in numeric value.

    5. Round Up/Down - Select the value 'Up' or 'Down' from drop-down list. The rounded amount is updated based on the selection.

    6. Effective [From-To] - Select the effective date using calendar. The package bulk update is done based on the effective date.

    7. Select - Select the check box to delete the selected entry.

    8. Click [Add] to add the new row.

    9. To remove the record, select the check box against the record and click [Remove].

    The following details are displayed in Package Price Bulk Update section

     

    1. Package Price Type - The package price type is displayed.

    2. Percentage /Amt - The Package amount is displayed.

    3. Mark Up/Down - Select Up or Down from drop-down list. Based on the selection, the Package price is updated.

    4. Round Off - Enter the round off value in numeric value.

    5. Round Up/Down - Select the value 'Up' or 'Down' from drop-down list. The rounded amount is updated based on the selection.

    6. Effective [From-To] - Select the effective date using calendar. The package bulk update is done based on the effective date.

    7. Select - Select the check box to delete the selected entry.

    8. Click [Add] to add the new row.

    9. To remove the record, select the check box against the record and click [Remove].

  1. Click [Apply] to save the details or [Reset] to clear the details.

If you are the privileged user, the authentication screen is displayed to enter your user name and password on click of Apply button. The privileged user setup is done in Function Privileges Setup for Policy.
