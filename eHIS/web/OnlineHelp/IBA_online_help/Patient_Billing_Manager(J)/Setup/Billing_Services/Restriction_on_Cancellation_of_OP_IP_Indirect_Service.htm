Restriction on Cancellation of OP/IP Indirect Service

From this function you can Define setup to restrict cancellation of OP/IP Indirect services.

This function allows to define the restriction to cancel OP/IP indirect services.  Based on the setup defined against the order category and order status, the user will be able to cancel the billing services in Indirect services in ‘Cancel OP/IP Indirect’ function.  This functionality is applicable to IP, OP, AE, EX, DC patient classes.

 

For each order category there are multiple order status associated, user need to configure the respective order statuses for the order category. It is a one-time setup. User can only view the order statuses and no modification will be allowed.  

Defining restriction for cancelling OP/IP Indirect Services

From the Setup > Billing Services, select Restriction on Cancellation of OP/IP Indirect Services Function.

The Restriction on Cancellation of OP/IP Indirect Services screen gets displayed.

  1. Select the required Order Category from the ‘Order Category’ drop-down list. The default value is ‘Select’.

  2. Based on the Order Category selected, the Order Status gets displayed. By default, all the check boxes against the Order Status will be cleared.

  3. Select the ‘Restrict cancellation of OP/AE/EX indirect service’ check box against the required Order Status to restrict the cancellation of indirect Services in the transactions.

  4. Similarly, select the ‘Restrict cancellation of IP/DC indirect service’ check box against the required Order Status to restrict the cancellation of indirect Services in the transactions.

  5. If the Order status check boxes are cleared, the system will not restrict the cancellation of OP/IP Indirect Services.

  6. Click [Apply] to save the setup done or click [Reset] to clear the changes done.

 

Without selecting any "Order Category", if user clicks Apply button, the message “Apply is not a valid operation here” gets displayed.
