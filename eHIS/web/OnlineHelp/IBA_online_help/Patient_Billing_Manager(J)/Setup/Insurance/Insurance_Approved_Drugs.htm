Insurance Approved Drugs

From this function you can Define the agreement for drugs between hospital and insurance        

This function defines the Agreement for drugs between the hospital and insurance. On loading the screen, the system displays the list of active drugs available in Drug masters. User must select the drugs and map it against the Billing group / Payer group / Payer / policy combination and the patient class.

 

Based on the Patient Class selected against each individual drug, drug will be available for selection in Pharmacy/ Inventory screens. For example, if drug ‘Acarbose’ is marked against OP episode type only, then drug ‘Acarbose’ will be available for OP patient class only. Similarly, if 'All' patient class is selected, then the drug will be available for selection against all patient classes.

Defining Insurance Approved Drugs

From the Setup > Insurance, select Insurance Approve Drugs function.

The Insurance Approved Drugs screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the details in the mandatory (*) fields. <Related Field Description>

  3. Against the selected Billing Group / Payer group / Payer / Policy / Episode type combination, select the  IP/DC/OP/EM/All check box against the required patient class.

  4. After entering the required details, click [Apply] to save the changes done or [Reset] to clear the entered details.

  5. While placing drug, agreement drugs must be allocated or displayed for the encounter based on this setup, provided the encounter financial detail will match with the payer group / payer / policy combination.

  6. Click [Query] to search for existing records.
