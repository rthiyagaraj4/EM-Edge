Revenue Recognition Stage

From this function you can Define Revenue recognition stage.

This function is used to define the status for each order category for posting revenue to interface table. This setup is done based on Order category and Order type wise.

This is for patients who have cash receipts irrespective of billing group. The system considers cash patients and insurance patients who have co-pay components / exclusion services for which cash receipt is generated.

Even after the bill and receipts are generated, the respective services will be considered as revenue based on the setup.

Defining Revenue Recognition Stage

From the Setup > Insurance, select Revenue Recognition Stage function.

The Revenue Recognition Stage screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the details in the (*) mandatory fields. <Related Field Description>

  3. Click [Apply] to save the settings.

  4. Click [Query] to search for existing revenue recognition records.

  5. Once the records are created, by default the records get displayed on clicking the Revenue Recognition Stage hyperlink in the menu. Only active and future records are displayed in descending order of the date sequence.

 

You can change the Revenue stage at any time.  If you change the revenue stage for an order category / order type, you need to define the Valid From as system date + 1 / future dates. The system automatically updates the Valid To date as ‘(New valid from date) - 1’ to the previous record.
