Pre-authorization/E-claims Status Tracking

From this function you can View the status of insurance approval pending for the patient.                   

This function allows to check the status of insurance approval pending for the patient. It provides a dashboard view with auto refresh of records, which will help the hospital Insurance Team and Admin team to track more effective of the approval status. Based on the privileges defined in the ‘Function Privileges Setup for Policy’ function, user can view the dashboard and do the transaction.

E-Claims Tracking

From the Transactions > Insurance, select the Pre-authorization / E-claims Status Tracking  function.

The Pre-authorization / E-claims Status Tracking  screen gets displayed.

There are two tabs:

Pre-Authorization

E-claims

 

Pre-Authorization tab

 

  1. Enter the details in the mandatory (*) fields. <Related Field Description>

  2. Click [Search] to view records matching the search criteria. The  Dashboard displays count of records at encounter level and grouped by payer.

  3. On clicking a particular Payer tile, system will display the encounter details in a dialog based on selected status.

  4. Click the 'Encounter ID', the Record Approval function gets invoked.

  5. Click [Apply] in the Record Approval function. The orders will be sent for approval and the status will be updated from 'Pending' to 'Send to Insurance', only when the request is sent successfully from Billing > Interface > NPHIES. Then the status will be changed to “Send to Insurance” in record approval function without manual intervention.

 

E-Claims tab

  1. Enter the details in the mandatory (*) fields. <Related Field Description>

  2. Click [Search] to view records matching the search criteria in a dashboard.

  3. Under the header ‘Claims by Payer at Encounter Level’, tiles will be displayed each corresponding to a payer. The displayed payers belong to the payer group selected in the search criteria.

  4. The dashboard displays all the Payers belonging to the selected payer group and the number in each tile is the total number of Encounters for each payer for OP encounters (For which bills have been approved through the “Approve Bill-new” Function).

  5. The Number displayed in each tile is the total number of Encounters for the respective payer within the search criteria chosen (Encounter type, dates, payer, etc.).

  6. Clicking on a Payer’s tile, system will show the count of claims/bills or the count of Billing services related to the Encounters in the payer’s tile under the header ‘Insurance Status for – Payer Group’.

  7. If the user selects ‘Claims’ in the search criteria. The tiles will show the total count for claims/bills in each insurance status.

  8. If the user selects Services, then the tiles will show the total count for Billing Services in each insurance status.

  9. If user keys in the “Approval Do Ref No”, system will retrieve records corresponding to that No and other mandatory fields will not apply.

  10. If the user keys in the Patient ID, system will retrieve records corresponding to that No and other mandatory fields will not apply.

  11. If the user keys in “Claim ID”, system will retrieve records corresponding to that No and other mandatory fields will not apply.

  12. If user did not key in a “Patient ID” or “Approval Doc Ref No” or “Claim ID” then, Payer group and the Statement From and To dates become mandatory.

  13. On clicking the insurance status tile, the 'RCM Transactions' screen gets displayed. For example, if user clicks on Pending tile, the pending records will be displayed and filtered by the search criteria.  The buttons Send to insurance and Export to Excel are available.

  14. Click [Export to Excel] button to download the data in the screen in excel sheet.

  15. Click [Send to Insurance] button to send documents to insurance team.  The selected records will be picked by interface API and submitted to NPHIES. A request ID will be generated. It will be used by interface to poll for updates regarding the insurance status, this request ID will be referred to in EM as 'Claim ID'.

    • The ‘Send to Insurance’ button will be greyed out since the 'Insurance status' is selected as 'Sent to Insurance'.

    • Records that were resubmitted to NPHIES following a clarification and upload of documents will also be displayed as "Sent to insurance".

  16. On closing the 'RCM Transctions' popup, the dialog gets closed and system focus is back to E-Claims Dashboard screen. Records get refreshed and update.
