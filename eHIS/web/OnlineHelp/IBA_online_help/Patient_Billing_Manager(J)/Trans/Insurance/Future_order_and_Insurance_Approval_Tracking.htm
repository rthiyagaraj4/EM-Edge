Future Order and Insurance Approval Tracking

From this function you can Track the status of future orders and insurance approvals.                    

This function allows to track the future orders and insurance approvals. This function is accessible only if the ‘Future order and Insurance Approval Tracking’ check box is selected in the Function Privileges Setup for Policy function. If user is not authorized, then a validation message ‘Privilege is not provided for login user, cannot proceed’ will be populated.    

 

From this function, user can invoke the Order Entry screen from CIS to place new Orders specific to respective patient and encounter. Also, amend already placed orders. Users can place lab, radiology, OT and Pharmacy orders. Users can identify future orders and transfer only when the orders are in “ordered” status. Future orders can also be linked to the encounter.

 

This function is applicable to Insurance billing group.

Tracking Future Orders and Insurance approvals

From the Transactions > Insurance > select the Future Order and Insurance Approval Tracking function.

The Future Order and Insurance Approval Tracking screen gets displayed.

  1. Enter the details in the mandatory (*) fields. <Related Field Description>

  2. Click [Search] to view records matching the search criteria.

  3. The search results include:

  • Patient Name – Displays the patient name as a hyperlink. Click the Patient Name hyperlink to view the patient demographics in a pop-up window.

  • Encounter ID – Displays the encounter ID of the patient as a hyperlink. Click the Encounter ID hyperlink to invoke the Order Entry function from CIS. The Order Entry window will have New Order and Existing Order tabs that allow users to place/amend/cancel order.

-  New Orders tab – Use this tab to place new orders specific to respective patient and encounter.

-  Existing Orders tab – Use this tab to amend already placed orders. Also, allows to link previous encounter orders to current encounter. Using this Tab, user have the option to identify future orders with colour legend.

-  Once the invoked window is closed, the ‘search result gets refreshed with whatever orders placed or amended.

  • Payer, Policy Number, Visit/Admission Date and Time, , Specialty Name

  • Practitioner Name - For OP system will display ‘Attending Practitioner’ and for IP it will be ‘Admitting Practitioner’

  • D button – Click the [D] button next to Encounter ID hyperlink, system displays the order details under the ‘Details’ section.

o Encounter ID, Order Catalog, Order Date Time, Signed Date, Ordered By, Order Status, Future Order Linked Date & Time, Coder Name, Remarks, Category, Coding Status, Insurance Status, Remarks, Authorised By

o Approval Informed to Patient - Select the value Yes/No from the drop-down list.

o Approval Informed By - It is a free text field. Enter the approval informed by name.

o Approval Informed Date & Time - Enter the approval informed date and time using the calendar icon.

o Approval Start date, Approval Expire date. The Approval start and expire date will be defaulted based on Record Approval function.

o Service Performance Date - Click the calendar icon to select the date when the service is performed.

o Service Status - Select the service status from the drop-down list. The list values include Service Performed, Called patient but no response, Appointment Booked, Followed-up with Patient. The default value is 'Select'.

o Remarks

o Follow up Visit/Book Appointment - Select the value 'Yes' and 'No' value from the Drop-down list.

o Appointment Date and Time - Select the appointment date and time by clicking the calendar icon.

o Follow-up Visit/Admission - Select the value 'Visit' or 'Admission' from the drop-down list.

  1. After making the required selections, click [Save] to save the transaction.  

  2. Click [Save & Export] to save and updated information will be populated in excel.

  3. Click [Export to Excel] to export the data in the screen in an excel sheet.
