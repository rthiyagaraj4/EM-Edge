RCM Transactions

From this function you can Sent records approved in Approved Bill New function to Insurance for claim processing.

The records approved through Approved Bill-New function and successfully invoiced to ERP via “Invoice posting” function be included in this function. The records will be grouped together with a single “Approval Ref No”.

 

Only users who are defined access rights in 'Function Privileges setup for Policy' function can access this function.

Tracking RCM Transactions

From the Transactions > Insurance, select the RCM Transactions function.

The RCM Transactions screen gets displayed.

  1. Enter the details in the mandatory (*) fields. <Related Field Description>

  2. After entering the required criteria and click [Search]. The records approved through Approved Bill-New function and successfully invoiced to ERP via “Invoice posting” function appear. The records are grouped by 'Approval Ref No'

  3. The search result columns include: Claim ID, Payer Group/ Payer, Encounter Type/ID, Visit Date/Admit date, Check out/Dis. Date, Patient ID, Patient Name, Bill No, Net Bill Amount Gross, Approved Bill Amount, Receipt Reference No,  Remarks, Claim Status.

  4. On selecting a Claim record from the search results, the details of that record appear in the lower section. The details include: Service Code/Description, Service Amount, Approved Amount, QTY, Approved QTY, Date/Time, Remarks, and Insurance Status.

  5. Upload Document - Click the [Browse] to upload documents to support the billing service in case NPHIES requested clarification. By default, this button will be greyed out except in the insurance status of 'Clarification'.

    • Users can upload PDF or JPEG files to the respective bill (maximum 2MB). If file size is greater than or equal to 2MB, system will restrict the by displaying message 'Uploading File size must be less than 2MB'.

    • If multiple documents are requested by NPHIES against a particular Bill, user manually need to scan and convert the multiple documents into single PDF and should upload it.

    • Other than table level document inserts, the uploaded documents will not be displayed at the front end.

  6. Click [Export to Excel] button to download the data in the screen in excel sheet.

  7. Click [Send to Insurance] button to send documents to insurance team.  The selected records will be picked by interface API and submitted to NPHIES. A request ID will be generated. It will be used by interface to poll for updates regarding the insurance status, this request ID will be referred to in EM as 'Claim ID'.

    • The ‘Send to Insurance’ button will be greyed out since the 'Insurance status' is selected as 'Sent to Insurance'.

    • Records that were resubmitted to NPHIES following a clarification and upload of documents will also be displayed as "Sent to insurance".

  8. Following successful submission, Insurance status of records is updated to "Sent to Insurance". However, if the submission fails due to a network issue, then records will persist in the "pending" Status until the user manually resubmits these records.

  9. In case submission fails due to any reason, an error message 'Submission failed due to "place holder for reason"' will be displayed. On clicking OK in the message, system focus is back to the Claim transaction screen, but the records will be refreshed to show the submission error under the remarks section Bill/Claim and service level.

  10. If submission is successful, the message 'Records Successfully Submitted' gets displayed. Click [OK], the focus is back on the Claims screen. Now gets refreshed but displays only records deselected by user before submission since the submitted records are now in the "Sent to insurance" Status.
