Ambulance Chart

The ‘Ambulance Chart’ section appears next to the ‘Requested Queue’ section. This section displays the ambulance details in a bubble format. By default, the Assigned, Maintenance, and Available status check boxes are selected. All the three status are differentiated by hardcoded colour indicators. The ambulance status is as follows: Grey for Maintenance, Green for Available, and Pink for Assigned.

 

In collapsed mode, the ambulance will be displayed in bubble format.  The ambulance bubble details include the following:

• First row displays the Ambulance ID.

• Second row displays the Registration number of the Ambulance.

• Third row displays the Current location defined for Ambulance.

• Fourth row displays the ‘Description of Staff,’ if Display Contact is selected else it will not be displayed.

 

In Expanded mode, the ambulance details get displayed as row. When more than four records are available, the Next icon gets enabled. The Ambulance Chart section details include Status, Ambulance ID, Description, Registration No, Make, Contact, and Last Mileage. The Status shows the Ambulance Type and status colour change.

 

This section allows you to do the following:

Updating Ambulance Status

You can change the status of the Ambulance in Assigned status (pink). Double-click on the Ambulance Status, the ‘Update Ambulance Status’ section gets displayed.

To update the Ambulance status, enter the following details:

  1. Current Location - Enter the current location of the ambulance.

  2. Remarks – Enter the remarks if any. You cannot enter more than 2000 characters.

  3. Distance Covered – Enter the distance covered, the request status gets changed to completed while the ambulance status gets changed from ‘Assigned’ to ‘Available’.

  4. Start Time – This field displays the start time of the ambulance.

  5. End Time –This field t will be blank and data entered here will not be saved

  6. Starting Meter – If Starting Meter value is entered in the request details, it will appear here else it will be blank.

  7. End Meter -  This field will be blank and data entered here will not be saved.

  8. Click [Record]. The ambulance update gets saved and status gets changed from ‘Assigned’ to ‘Available’.

Recording Ambulance under Maintenance Details

When the ambulance is in available status, you can record the maintenance details if required. Double-click Ambulance bubble; the ‘Record Under Maintenance Details’ section gets displayed.

To record the ambulance maintenance details, enter the following details:

  1. Current Location - Enter the current location in alphanumeric format of 60 characters maximum length.

  2. Under Repair – Select this check box if you want to record the repair details.  When this check box is selected, the new fields such as Period From, Period, and Reason get displayed.

  3. Period From – Enter the date from when the ambulance is under repair in DD/MM/YYYY HH:MM format.

  4. Period – Enter the period from which the ambulance is under repair in days or hours. You can enter in alphanumeric format of 3 characters maximum length.

  5. Reason - Enter the reason if any.

  6. Click [Record]. Now, the ambulance status gets changed from ‘Available’ to ‘Maintenance’.

Revoking Ambulance Maintenance Details

Once the maintenance of the ambulance is done, you can revoke the ambulance is in Maintenance status.

Double-click Ambulance bubble; the ‘Revoke Under Maintenance Details’ section gets displayed. The previously recorded details such as Current Location, Period From, Period, and Remarks appear as read-only field. You cannot modify the field.

If the ambulance maintenance gets completed, select the ‘Revoke Maintenance’ check box and click [Record]. The ambulance status gets changed to from ‘Maintenance’ to ‘Available’.

You can drag and drop the ambulance in Available and Assigned status to the Request Details section. The ambulance detail gets populated in the Request Details section. You need to enter the Request details to proceed further.

• You cannot drag and drop the Ambulance in ‘Maintenance’ status to the Request Details section
