Visit Detail Tab

 

Patient Details

  1. My Kad/Kid – The compulsory identity document number of the registered patient is displayed.

  2. My PR – The Permanent Resident status and the expiry date of the registered patient is displayed.

  3. Citizen ID – The default Citizen ID in numeric format is displayed while registration. The ‘Prompt’ specified for the Citizen ID in MP Parameter appears as the field legend. The length and format as specified in MP Parameter has to be adhered to.

  4. Nationality – The default Nationality set in MP Parameter is displayed. Select the required value using the [?] function. This field is mandatory. If National ID is entered, the default Nationality set in MP Parameter is displayed and disabled.   

  5. Residency – The selected residency in Emergency Registration is displayed.

  6. Status – The selected status in Emergency Registration is displayed.

  7. Blood Group – The blood group of the patient registered through emergency registration is displayed.

The above field names are displayed based on the legends for Alternate ID defined in the MP Parameter.

Visit Detail

  1. Visit Date Time – Enter the Visit Date and Time. The visit date has to be entered in dd/mm/yyyy format for the visit. This field is mandatory. Based on the number of hours set in ‘Allow backdating of Visit Date/Time’ in AE Parameter the visit time and date can be calculated or, select the Calendar icon to select the appropriate date.

  2. Location - Select the Location from the valid list of values. This field represents the Location to which the Patient has to be registered. All the locations associated to the logged in users operator station with Level of Care as ‘Emergency’ are displayed. This field is mandatory.

  3. Registration Type - Select the Registration Type from the valid list of values. This field represents the Visit Type of Patient for the visit. The displayed list is limited to the Visit Types based on the selected Location. This field is mandatory.

  4. Service - The Service associated to a Location is displayed. Select the appropriate service that is to be rendered to the patient. This field is mandatory.

  5. Sub-Service - The Sub-Service associated to the selected service is displayed, if any.

  6. Mode of Arrival – This function is used to record the Mode of Arrival of the patient, which is provided in two list values. The first list value is used to select the Transport Mode and the second is used to select the Mode of Arrival as set in Application Masters module, which is filtered based on the Transport Mode selected.

  7. Brought Dead – Select the check box to indicate that the patient being registered is deceased and has been brought dead to the facility.

  8. MLC – Select the check box to indicate that the patient is registered under Medico Legal Case (MLC), or clear the check box.

  9. Motor Vehicle Accident – Select the check box to indicate if the patient registered had been subjected to motor vehicle accident, or clear the check box. By default, this check box is cleared.

  10. Date/Time of Accident – Select the Date and Time of the Accident using calendar function.

  11. Place of Accident – Enter the Place of Accident in alphanumeric format of 30 characters maximum length.

  12. Vehicles Involved – This is used to record the vehicles involved (if any) for the case of trauma. The list box displays all the values set in ‘Transport Mode’ of Application Masters module.

  13. Position During Incident – Select the Patient Position during the Incident from the list.

 

MLC Detials

The 'MLC Details' section is visible only for users in Medicity.

  1. MLC check box - By default, the check box is enabled. Select the check box to enable the “MLC Death”, “Police Report no and Police station details”, “Outside MLC no./Details”,” Date of MLC capture”,” Remarks” fields.

  2. MLC Death check box – By default, the check box is disabled. This check box gets enabled only if the MLC check box is selected. User can select the check box if required.

• If ‘Brought Dead’ check box is selected and MLC checkbox is also selected, then automatically MLC Death checkbox gets selected. User can clear the check box if required.

  1. Police Report No – Enter the police report number in alphanumeric format up to 100 characters. Special characters are allowed. By default, it will be disabled.  Only when the MLC check box is selected, this field gets enabled.

  2. Police Station Details – Enter the police station details in alphanumeric format up to 100 characters. Special characters are allowed. By default, it will be disabled.  Only when the MLC check box is selected, this field gets enabled.

  3. Outside MLC No./Details – Enter the details in alphanumeric format up to 100 characters in length. Special characters are allowed. By default, it will be disabled. Only when the MLC check box is selected, this field gets enabled.

  4. Date of MLC Capture - By default, this field is disabled. When the MLC check box is selected, this field gets enabled. By default, it will be system date and time. User can edit the date only be between the visit date time and system date time.

  5. Remarks- Enter the remarks in alphanumeric format up to 500 characters in length. By default, it will be disabled. Only when the MLC check box is selected, this field gets enabled.

Note: If MLC check Box is selected and all the details are entered in MLC Details section, on clearing the MLC check box then system will clear the entered details and disable all the fields. Again, if user selects the MLC check box, then fields will get enabled with blank value.

If MLC check box is selected in AE Register Attendance, and all details are entered in MLC Details section then the entered details get auto populated in ‘Secondary Triage’ and ‘Marking of MLC/OSCC’. User can edit the MLC Details data if required.

 

Primary Triage Details

  1. Priority Zone – Select any one of the Priority Zone by clicking on the required check box. The colors indicate the following conditions:

  2. Red - Critical

  3. Yellow – Semi Critical

  4. Green – Non- Critical

  1. Priority – Select the Priority from the valid list of values. The values corresponding to the selected ‘Priority Zone’ are displayed.

  2. Practitioner - This field is enabled and mandatory only if the Priority Zone and Treatment Area are selected. The Practitioner lookup lists the practitioners that are associated with the Priority Zone and Treatment Area. Select the required practitioner using [?] function.

  3. Treatment Area – Select the required Treatment Area from the valid list of values. All Treatment Areas corresponding to the selected ‘Priority Zone’ are displayed.

  4. Bed/Bay No – This field is enabled on selecting a Treatment Area. Enter the required Bed/Bay Number in alphanumeric format of eight characters or click [?] to search for the required Bed/Bay Number. All the bed/bay associated to the selected Treatment Area in the master function ‘Assign bed/bays for Treatment Area’ gets displayed.  

  5. Disaster – Select the check box to indicate the patient as a victim of disaster, or clear the check box.

  6. Disaster Area – This field is enabled on selecting the disaster check box. Enter the area of occurrence of the disaster in alphanumeric format or click [?] to search for the required area. The areas defined in the function ‘Area’ of Master Patient Index module are displayed.

  7. Disaster Type – Select the Disaster Type from the valid list of values. This field is enabled when the disaster check box is selected. The values defined in the ‘Disaster Type’ in AE code setup are displayed.

  8. Disaster Registration Reference - This field is enabled on selecting the Disaster check box. Enter the Disaster Registration Reference Number in alphanumeric format of 15 characters maximum length. This is a mandatory field. The Disaster Registration Reference Number entered is displayed in 'Current Emergency Patients' function when disaster patients are listed.

  9. Presenting Problem - Enter the Presenting Problem of the patient or select using [?] function. The Presenting problems are defined in 'Reason for Admission' function in Application Masters module.

 

Arrival Tab

 

Escort Detail

  1. Accompanied By - This field is used to record the person accompanying the patient who is being registered. The list displays all the values set in ‘Escort Type’ under Other Master Setup of Application Masters module.

  2. Relationship – Select a valid Relationship of the person accompanying the registered patient.

  3. Other Alt ID – Select the required Alternate ID from the list. The listed values are defined in the function ‘Alternate ID Type’. On selecting the required Alternate ID type, enter the ID in alphanumeric format of 20 characters maximum length. The field ‘Other Alt ID’ is displayed provided ‘Accept Other Alt ID’ is set in MP Parameter (Alt ID checks tab).

The above field names are displayed based on the legends for Alternate ID defined in MP Parameter.

  1. Address Line (1 to 4) – Enter the address details in alphanumeric format of 30 characters maximum length in each line. The legends are set in ‘System Manager’ module.

  2. Town - Enter the name of the Town in alphanumeric format of 30 characters maximum length. The entered name is listed under the selected area. If the selected area has more towns attached to it, select the required town using the [?] function. The field legend is set in System Manager module.

The Area and Region associated to the selected Town is displayed by default.

  1. Area - Enter the Area in alphanumeric format of 30 characters maximum length or select it using the [?] function. The field legend is set in System Manager module.

  2. Region – Enter the name of the Region in alphanumeric format of 30 characters maximum length or select it using the [?] function. The area must correspond to the entered Region. The field legend is set in System Manager module.

  3. Postal Code - Enter the Postal Code in alphanumeric format of eight characters maximum length. The entered value must correspond to the selected Town. Select the required code using the [?] function.

  4. Country - The Country name gets automatically populated when the Region is selected.

  5. Remarks – Enter the Remarks, if any, for the arrival details in alphanumeric format.

 

Ambulance Details

  1. Name of Escort/Ambulance Service - Enter the Name of Escort/Ambulance Service in alphanumeric format of maximum 30 characters. This is used to record the name of the escort service or ambulance service if any.

  2. Escort/Ambulance Contact No. - Enter the Contact Number of the escort service in alphanumeric format of 10 characters maximum length.

  3. Motor Vehicle Registration No. - Enter the Registration Number of the vehicle that takes the patient to the facility in alphanumeric format of 10 characters maximum length.

Employee Details

  1. Employment ID - Enter the Employment ID of the Patient in alphanumeric format of 15 characters maximum length.

  2. Occupation Class – Select the Occupation Class from the valid list of values. The listed values are defined in the function 'Occupation Class’ of Master Patient Index  Code Setup.

  3. Occupation of the Person - Select the Occupation from the valid list of values. The listed values correspond to the selected occupation class. If the required occupation is not available, enter the same in alphanumeric format of 15 characters maximum length in the text box provided.

  4. Status - Select the Employment Status of the Patient from the valid list of values. The listed values are pre-defined.  

 

Financial Details Tab

 

  1. Billing Group – Select the Billing Group from the lookup provided. The Billing Group listed is based on all Billing Groups applicable for the selected Location and Visit Type in the Patient Billing Manager module. The legend is a hyperlink, which shows the explanatory note for the selected billing group, if defined.

  2. No. of Dependants – Enter the number of dependants for the selected patient. This field is enabled based on the type of billing group selected.

  3. Responsible for Payment – Enter the name of the person who is responsible for payment of charges for the current encounter. This field is enabled based on the type of Billing Group selected.

  4. Patient Reln with Responsible Person – Select the relationship that the patient shares with the responsible person. This field is enabled and is mandatory based on the type of Billing Group selected.

  5. Update Patient Financial Details - Select the check box, if the patient financial details are to be updated each time the patient visits the location, or clear the check box.

 

Settlement Details / Document References

  1. Settlement Type – The fields that appear under this section are dependant on the type of billing group selected. For cash billing group, the field Cash Settlement Type appears from which you can select the appropriate settlement type.

  2. Document Reference – Enter the Document Reference Details in alphanumeric format. This field is mandatory.

  3. Document Start Date – Click the Calendar icon to select a date. The documents are considered valid from the date specified.

  4. Document End Date – Click the Calendar icon to select a date. The documents are considered valid until the date specified.
