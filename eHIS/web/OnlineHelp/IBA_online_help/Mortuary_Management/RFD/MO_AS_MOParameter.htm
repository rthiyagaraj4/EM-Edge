Enter the following details:

 

  1. Billing Interface – Select the check box to interface the mortuary management module to billing module.

  2. Department – Select the department to be associated to the mortuary from the list. The values listed are the departments defined in the Application Masters module. This is a mandatory field.

  3. Default Service – When you select a Department, all associated services will be populated in this drop-down list and the alphabetically first value will be displayed by default. You can select the required default service from the list.

The value selected in Default Service will be displayed as the default service in the Deceased Registration screen.

If there is only one service associated with a Department, it will be displayed by default in the Deceased Registration even if it is not selected in the ‘Default Service’ field in MO Parameter screen.

 

  1. Number of days to preserve unknown body – Enter the number of days in numeric format of two digits maximum length to preserve unknown bodies before the body is registered in mortuary and the deceased registration number is generated. This is a mandatory field.

  2. Enable RFID Tag No -  Select the check box to display the RFID Tag No field across MO module. By default, the check box is selected.  If you clear the ‘Enable RFID Tag No’ checkbox, the system will not display ‘RFID Tag No’ legend along with respective control type across the Mortuary Management module.

  3. Police  Details Mandatory – Select the check box to make fields in the Police Details dialog as mandatory or non-mandatory while performing Deceased Registration and Body Part Registration. By default, the check box is not selected.

  4. Allow user to record PM Finding details - Select the check box to display both Finding and Specimen tabs next to Registration tab in Record Postmortem screen. By default, the check box is selected. If you clear the check box, Finding tab will not appear next to Registration tab in Record Postmortem screen. This filed is applicable for users in MOHML sites.

  5. Date Time of Claiming Mandatory - The drop-down list values include Select, Deceased Registration, and Release Deceased/Body Part’. The default value is ‘Select’. This field is mandatory. This is applicable for users in MOHML sites.

  • If user selects the value ‘Deceased registration’ from the drop-down list, then ‘Date Time of claiming’ field will be visible and mandatory in Claimant tab in Deceased Registration function.

  • If user selects the value ‘Release Deceased/Body part’ from the drop-down list, then ‘Date Time of Claiming’ field will be visible and mandatory in Claimant tab in Release Deceased/Body Part function.

 

Cut-off period to look for deceased patients from MPI

Once a patient is declared dead, the dead body can be registered in the mortuary based on the cut-off period set in the parameters.

  1. More than (in hrs) – Enter the hours in numeric format of two digits maximum length. This is a mandatory field. The deceased registration for a deceased patient in the transactions can be done only after the specified hours of declaring the patient as dead in the other modules.

  1. Less than (in hrs) - Enter the hours in numeric format of two digits maximum length. This is a mandatory field. The body has to be registered within the specified hours of declaring the patient as deceased.

The ‘More than’ time should be less than the ‘Less than’ time.

 

Order Catalog

  1. Clinical Postmortem – Select the type of clinical postmortem from the list. The listed values are populated from the OR module. This is a mandatory field. Clinical postmortem is carried out on normal postmortem request based on the order catalog selected.

  1. Medico legal Postmortem – Select the type of medico legal postmortem from the list. The listed values are populated from the OR module. This is a mandatory field. Medico legal postmortem is carried out for deceased/body parts registered as medico legal case based on the order catalog selected.

 

Burial Permit

  1. Burial Permit Mandatory – Select the check box to make burial permit as mandatory. If you select this check box, Burial Permit Number field is made mandatory in the Burial Permit, Body Part Registration and Deceased Registration screens.

  1. Next Burial Permit No – Enter the next number for burial permit in numeric format of 16 characters maximum length. This is a mandatory field. Once a burial permit is generated for a deceased patient, this number gets updated automatically.

  2. Maximum Number & characters allowed in Burial Permit Number – Enter the maximum no of characters/numbers allowed in the Burial Permit Number. User can enter two numeric characters (Ex: 00 - 99) in the text field. The value updated in this field gets reflected for the ‘Build Permit Number’ field in the transactions, queries and reports. This is applicable for users in MOHML.

 

Patient Series

  1. External Bid Registration – Select the patient series to be used while registering external BID patients. By default, the value ‘Select’ is displayed.

  1. External Body Part Registration – Select the patient series to be used while registering External Body Part. By default, the value ‘Select’ is displayed.

 

Queue Management

  1. Refresh Interval (Min) – Enter the time interval for the application to be automatically refreshed in numeric format of 4 characters maximum length.
