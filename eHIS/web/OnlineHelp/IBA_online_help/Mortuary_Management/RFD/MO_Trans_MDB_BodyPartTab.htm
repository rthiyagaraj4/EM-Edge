Body Parts Tab

  1. Enter the search criteria and click [Search] to select a specific body part. Body Part details are displayed based on the entered criteria. The status of the patient is indicated by a unique color code.

    1. Registration No - The mortuary registration number generated on registering the deceased /body part is displayed.

    2. Registration Date/Time - The date and time of registration is displayed.

    3. Deceased Name/Patient Name - The name of the deceased is displayed and hyperlinked in 'Deceased Patient' Tab and the name of the patient whose body part is registered in the mortuary is displayed and hyperlinked in the 'Body Parts' Tab.

    4. Patient ID - The Patient ID generated when the patient was registered in the Master Patient Index module is displayed.

    5. Age - The age of the patient recorded when the patient was registered to the facility is displayed.

    6. Sex - The gender of the patient is displayed.

    7. PM Y/N - If postmortem is required for the deceased/body part, then "Yes" is displayed, else "No" is displayed. The values are displayed based on the Postmortem Required field in Deceased Registration/Body Part Registration function.

    8. Postmortem Status - The postmortem status of the deceased /body part is displayed. The various statuses are 'Not Applicable', 'Not Requested', 'Requested', 'In Progress' and 'Completed'.

    Statuses

    Descriptions

    Not Applicable

    If the ‘Postmortem Required’ check box is cleared in the “Deceased Registration” function and “Body Part Registration” function, then the status ‘Not Applicable’ is displayed.

    Not Requested

    If the ‘Postmortem Required’ check box is selected in the “Deceased Registration” function and “Body Part Registration” function but the request for postmortem has not been yet placed in “Manage Deceased/Body Part” function, then this status is displayed.

    Requested

    If the ‘Postmortem Required’ check box is selected in the “Deceased Registration” function and “Body Part Registration” function and the request for postmortem has been placed in “Manage Deceased/Body Part” function, then this status is displayed.

    In Progress

    If the ‘Postmortem Required’ check box is selected in the “Deceased Registration” function and “Body Part Registration” function and the postmortem is being recorded in “Manage Deceased/Body Part” function, then this status is displayed.

    Completed

    If the ‘Postmortem Required’ check box is selected in the “Deceased Registration” function and “Body Part Registration” function and the completion of postmortem is recorded in “Manage Deceased/Body Part” function, then this status is displayed.

    1. Duration in Mortuary - The duration of the deceased/body part in the mortuary is displayed.

    2. Religion - The religion of the deceased/body part is displayed.

    3. Encounter ID - The last encounter ID of the patient is displayed.

    4. Location - The location assigned to the deceased /body part in the area during registration is displayed.

    5. Body Part Type - This is displayed only in the 'Body Parts' Tab. The type of body part registered for the patient is displayed as 'Amputated Body Part/Dead Fetus/Unidentified Body Part'.

    6. Burial Permit No - The Burial Permit Number is displayed if the burial permit was generated for the patient.

    1. Registration number - Enter the registration number or select the registration number using [?] function.

    2. Patient ID - Enter the patient ID in alphanumeric format of 10 characters maximum length or select the deceased patient ID using [?] function.

    3. Area Type - Select the area type from the list. The listed values are Postmortem Area, Release Area, Storage Area and Others.

    4. Area - Select the area from the list. The listed values are defined in the Area function. The list also displays 'Unassigned' and 'Unregistered'. 'Unassigned' Area applies to those deceased/body parts that are not yet assigned to any area. 'Unregistered' refers to deceased/body parts that are not yet registered to the mortuary but have placed an order from the "Clinician Access" module.

    5. Postmortem Status - Select the postmortem status from the list. The listed values are 'Not Applicable', 'Not Requested', 'Requested', 'In Progress' and 'Completed'.

    6. Release Status - The value is displayed as "Not Released" and is disabled, by default.

    7. Date From/To - Enter the date range in dd/mm/yyyy format to manage deceased patients in the mortuary within the specified date range. The current date and time is displayed, by default. This is a mandatory field.

    8. Gender - Select the gender of the patient from the list.

    9. Burial Permit No - Select the burial permit number using the [?] function.

    10. Body Part Type - This field is displayed as 'Not Applicable' and is disabled for deceased patients. Select a value from the list for body parts. The listed values are Amputated Body Part, Dead Fetus and Unidentified Body Part.

    11. Refresh Interval - The Refresh Interval is displayed as recorded in the MO Parameter function.

  2. The records displayed are grouped by the areas where the body part has been registered. If no area was assigned during registration, all such body part records are grouped under "Unassigned".

  3. Click the Patient Name hyperlink to manage body parts in the mortuary.
