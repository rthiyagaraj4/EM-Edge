Enter the following details to record financial details:

 

  1. Billing Group – Click [?] to display Billing Group dialog. The selected Patient Billing Group defines if the patient is Cash Patient or Credit Patient or Insurance Patient. The Billing Groups defined in the Patient Billing Manager module gets displayed. The Billing Group legend is displayed as a hyperlink. Click on the link to view explanatory note of the respective Billing Group recorded during Billing Group definition. Click [Close] to exit the screen

    Enter the following details:

    1. Billing Group – Click [?] to select the Billing Group. The Billing Groups defined in the billing master are listed.

    2. Payer group - Click [?] to select the Payer Group. The Payer Groups defined in the billing master are listed.

    3. Billing group – Select this radio button to search the financial details based on the billing group. By default, this radio button is selected.

    4. Payer Group– Select this radio button to search the financial details based on the Payer group. By default, this radio button is selected.

    5. Policy - Click [?] to select the Policy. The Policies defined in the billing master are listed. The dtls hyperlink is displayed on selection of the policy. You can view the policy details on click of the dtls hyperlink.

    6. Payer - Click [?] to select the Payer. The Payers defined in the billing master are listed.

    7. Click [Search] to display the Billing Group, Payer Group, Payer and Policy. The Billing Group and Payer Group are displayed in the hyperlink format.

    8. Click on the hyperlink; the selected record is displayed in the Selected Record section.

    9. You can select the check box against the record and click Remove Payer button to remove the record.

  2. No. of Dependants – Enter the number of dependant family members in numeric format with a maximum of two digits.

  3. Responsible for Payment – Enter the name of the person in alphanumeric format of maximum 30 characters. This indicates the name of the person who is settling the bill amount on behalf of the patient. This is a mandatory field.

  4. Patient Reln with Responsible Person – Click [?] function to select a Relationship. The Relationship selected indicates the association of the person to the patient.

  5. Primary Referral Source - Click [?] function to select a Primary Referral Source. The values defined for Primary Referral Source in the Application Masters with the Visit check box selected gets listed. This field is set as mandatory only if the check box ‘Ref Source Main Mandatory’ is selected for the Billing Group in the Patient Billing Manager module. A button adjacent to the referral source enables the user to capture the referral source to a maximum of 100 characters. Place the mouse on the icon to display the long name of the External or Referral Source recorded in ‘External/Referral Source’ function of System Configuration module. The  icon appears on the screen if the long name for the External Referral Source is recorded in ‘External/Referral Source’ function of Application Masters module.

  6. Secondary Referral Source - Click [?] function to select a Secondary Referral Source. The values defined for Primary Referral Source in the Application Masters with the Visit check box selected gets listed. This field is set as mandatory only if the check box ‘Ref Source Sub Mandatory’ is selected for the Billing Group in the Revenue Management module.  A button adjacent to the referral source enables the user to capture the referral source to a maximum of 100 characters.

  7. [Select From Previous Encounters] - Click [Select From Previous Encounters] to copy financial details from previous encounters.

  8. GL Holder Name – The GL holder name associated to the patient while registration gets displayed, if any.

  9. Relation with GL Holder – The relationship of the patient with the GL Holder gets displayed.

  10. Update Patient Financial Details – Select this check box to update the financial details of the selected patient. This check box is selected by default.

  11. Special Clinical Related Encounter – Select this check box to view the financial details of the patient with special clinical related encounter.

 

Settlement Details/Document References

The Settlement Details varies based on the Billing Group selected.

 

When the Billing Group is selected as Cash, enter the following settlement details:

  1. Settlement Type – Select a Settlement Type from the list. This indicates the mode of settlement of the bill amount. This is a mandatory field.

  2. Document Reference – Enter the Document Reference Details in alphanumeric format. This is a mandatory field.

  3. Document Start Date – Click the Calendar function to indicate the date from when the document reference is considered valid. This is a mandatory field.

  4. Document End Date – Click the Calendar function to indicate the date until when the document reference is considered valid. This is a mandatory field.

 

When the Billing Group is selected as Sponsor, enter the following settlement details:

  1. Customer Group - Click [?] function to select a Customer Group. The values get listed from the Patient Billing Manager module.

  2. Customer - Click [?] function to select a Customer. This indicates the payer settling a part of the bill amount. The values get listed from the Patient Billing Manager module. This legend is displayed as a hyperlink. Click the hyperlink to view the Customer details.

  3. Credit Document Reference – Enter the Document Reference Details in alphanumeric format. This is a mandatory field.

  4. Document Start Date – Click the Calendar function to indicate the date from when the document reference is considered valid. This is a mandatory field.

  5. Document End Date – Click the Calendar function to indicate the date until when the document reference is considered valid. This is a mandatory field.

 

When the Billing Group is selected as Insurance, enter the following settlement details:

If the Payer/Policy details are associated to the patient during patient registration, the values get displayed. If the Payer/Policy Details are not associated, you can associate a Payer using [Add More Secondary Billing Groups] or [Add More Payers] or [Add More Payers] to invoke Policy Definition screen to change policy level either encounter wise or episode wiseSelect the check box against a record and click [Modify Payer] to invoke Authorization Screen. Enter User ID, PIN and Remarks for authorization. This button is enabled only if you do not have modification rights to update Payer details or [Remove Payer] to dissociate Payer. Click [Upload Image] to display Insurance Image Upload screen to upload a scanned image with payer details.

On click of the Payer Group hyperlink, the following details are displayed:

  1. Payer Group – The Payer Group is displayed. You cannot Modify the data.

  2. Payer – The Payer name is displayed. You cannot Modify the data.

  3. Policy – The Policy name is displayed. You cannot Modify the data.

  4. Policy No - The Policy No is displayed. You cannot Modify the data.

  5. Policy Start Date – The Policy Start Date is displayed. You cannot modify the data.

  6. Policy Expiry Date – The Policy Expiry Date is displayed. You cannot modify the data.

  7. Cancel – Select this check box to cancel the image upload. This check box is not selected by default.

Click [Browse] to upload the scanned image. Click [Apply] to save the payer group details with uploaded image. Once the image is uploaded, the alert message ‘Image is uploaded’ is displayed.

You can view the uploaded image on click of the ‘View Image’ hyperlink.
