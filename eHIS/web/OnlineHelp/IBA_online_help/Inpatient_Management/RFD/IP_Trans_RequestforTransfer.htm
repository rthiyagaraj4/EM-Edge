Enter the following details:

 

  1. Reason for Transfer – Select the Reason for Transfer from the list. This field is mandatory. The values in the Reason for Transfer list are defined in the ‘Transfer Type’ function of IP Code Setup.

  2. Preferred Date/Time – The system date/time is displayed as the preferred date/time. This field can be changed. Enter the preferred date and time in dd/mm/yyyy hh: mm format or select the date and time from the calendar function. This field is mandatory. The entered date and time must be greater than or equal to current system date and time.

  3. Priority – Select the Priority from the list.

 

Nursing Unit Details

 

Transfer To

  1. Nursing Unit – Select the Nursing Unit to transfer the patient from [?] function. Transfer to the same nursing unit is not allowed. This field is mandatory.

  2. Service - Select a Service from list of valid values. This field is mandatory.

  3. Sub-Service – Select a Sub Service that is associated to the service selected from the list.

If the user performing the request for transfer has the privilege of confirming such requests, as defined by the selecting the flag ‘Bed Class Change’ against the operator station to which the logged in user is attached to, the change for Bed Class request is confirmed in this transaction itself and the transfer of the patient is completed. If the logged in user does not have the privilege, all such requests are routed to the ‘Confirm Bed Class Change’ function where an authorized user confirms the bed class change after which the request for patient transfer is completed.

  1. Bed Class - The Bed Class associated to the patient is displayed. The value can be changed if the ‘Request for Bed Class Change’ check box is selected. This field is mandatory.

  2. Bed Type - The Bed Type associated to the patient is displayed. The value can be changed if the ‘Request for Bed Class Change’ check box is selected. The values are displayed based on the bed class selected. This field is mandatory.

  3. Bed No – The Bed cannot be specified at this stage.

  4. Room/Cubicle No – The Room/Cubicle number is not displayed as the bed is not selected.

  5. Specialty – If the Nursing Unit caters to multiple Specialties, a Specialty can be associated by entering the Specialty or using [?] function. This is a mandatory field.

  6. Attending Practitioner – The Name of the Practitioner currently attending the patient is displayed. The name of the Practitioner can be changed if required. Enter the name of the Practitioner or select the Practitioner from [?] function for the selected Specialty. This is a mandatory field.

  7. Medical Team – The Medical Team (if any) associated to the selected practitioner is displayed. The Medical Team can be changed if required. Select the Medical Team from among the values in the list. The values populated correspond to the selected practitioner.

 

Reserve Bed Details

  1. Reserve Bed - Select the check box to Reserve the Bed

  2. Reason for Reserve Bed - Select the Reason for Reservation from the list.
