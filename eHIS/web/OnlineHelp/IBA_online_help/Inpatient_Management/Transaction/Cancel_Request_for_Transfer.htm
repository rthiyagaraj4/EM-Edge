Cancel Request for Transfer

From this function you can Cancel request for transfers placed by the source Nursing Unit.

The source Nursing Units utilize this function to cancel requests placed by the patient or the facility. Requests not confirmed by the destination Nursing Unit can be canceled.

The logged in user must have rights to cancel the request for transfer. The rights are defined in the Operator Station assigned to the logged in user.

Cancelling Request for Transfer

From the IP Transactions > Transfer, select the Cancel Request for Transfer function.

The Cancel Request for Transfer screen gets displayed.

  1. Click [Search] to display the list of all Inpatients with open transfer requests or enter the search criteria and click [Search] to select a specific Inpatient with an open transfer request. Click [Clear] to reset the records displayed in the result area.

    Enter the following query criteria to select the patient record:

    From Details

    1. Nursing Unit – Enter the Nursing Unit from where the patient has to be transferred in alphanumeric format of 15 characters maximum length or select the Nursing Unit using the [?] function.

    2. Specialty – Enter the Specialty from where the patient has to be transferred in alphanumeric format of 15 characters maximum length or select the specialty using the [?] function.

    3. Practitioner – Enter the Practitioner name in alphanumeric format of 30 characters maximum length or select the Practitioner using the  [?] function.

    To Details

    1. Nursing Unit - Enter the Nursing Unit to where the patient has to be transferred in alphanumeric format of 15 characters maximum length or select the Nursing Unit using the [?] function.

    2. Specialty - Enter the Specialty to where the patient has to be transferred in alphanumeric format of 15 characters maximum length or select the Specialty using [?] function.

    3. Practitioner - Enter the Practitioner name in alphanumeric format of 30 characters maximum length or select the Practitioner using the [?] function.

    4. Admission Date From-To – Enter the Admission Date range in date format of dd/mm/yyyy or select the date using the calendar function.

    5. Encounter ID – Enter the Encounter ID in numeric format of 12 digits maximum length.

    6. Transfer Type – Select the Transfer Type from the list. The listed values are the values from Transfer Type function of the IP Code Setup.

    7. Patient ID - Enter the Patient ID in alphanumeric format of maximum length as specified in MP Parameter or select the Patient ID using [?] function.

    8. Gender – Select the Gender of the patient from the list. The values listed are Male, Female and Unknown.

    9. Patient Name Components – The query components and the legends pertaining to the patient name depends upon the set up done in MP Parameter under the ‘Legal Names’ Tab. The available components for the patient name are Prefix, First Name, Second Name, Third Name, Family Name and suffix. The components that are made applicable for the facility get displayed as query criteria. The appropriate patient name can be searched by using the search options Starts With, Contains, and Ends With.

    10. Search Name With Local Lang – Select the check box to search the name with the local language.

  2. On searching, Transfer Request details are displayed based on the entered criteria.

  3. Click ‘+’ hyperlink to view the Current Encounter Details.

  4. Click the ‘Encounter ID’ hyperlink to display the Cancel Request for Transfer. The screen displays Admission Details, Current Occupancy and Transfer To details.

  5. Enter the Cancellation Details in the mandatory (*) fields.

    1. Reason for Cancellation – Select the Reason for Cancellation from the list. This field is mandatory. The values in the list are populated from the Reason master where ‘Cancel Booking’ is selected.

    2. Authorizing Practitioner – Enter the name of the Authorizing Practitioner in alphanumeric format of 30 characters maximum length or select the Practitioner using the [?] function. This field is mandatory.

  6. Click [Confirm] to cancel the transfer request or [Close] to exit from the screen.
