Check-in Lodger

From this function you can Check-in a lodger.

A lodger is an Inpatient who seeks a short stay in the facility on discharge due to specific reasons. You have to select patients who are recently discharged and wants to extend their stay by a short period for specific reasons.

Checking-in a Lodger

From the IP Transactions > Lodger/Accompanying Person, select the Check in Lodger function.

The Check-in Lodger screen gets displayed.

  1. Click [Search] to display the list of recently discharged patients or enter the search criteria and click [Search] to select specific discharged patients. Click [Clear] to reset the records displayed in the result area.

    Enter the following criteria to select the required patient:

    1. Nursing Unit – Select the Nursing Unit using the [?] function. The values listed belong to the Operator Station associated to the logged in user.

    2. Discharge Date [From/To] - Enter the Discharge Date range in dd/mm/yyyy format or select the date range using the calendar function.

    3. Specialty – Select the Specialty of the Nursing Unit by clicking on the [?] function.

    4. Practitioner – Select the Practitioner by clicking on [?] function.

    5. Encounter ID – Enter the Encounter ID in numeric format of 12 digits maximum length.

    6. Patient ID - Enter the Patient ID in alphanumeric format of maximum length as specified in MP Parameter or select the ID using [?] function.

  2. On searching, Discharged Patient details are displayed based on the entered criteria.

  3. Click ‘+’ hyperlink to view the Current Encounter Details.

  4. Click the ‘Encounter ID’ hyperlink to display the Check-In Lodger screen. The screen displays the Lodger Details and Stay Details.

  5. Click [Contact Details] to record the contact details of the lodger.

  6. Enter the Stay Details in the mandatory (*) fields.

    1. Reason For Stay – Select a reason from the list. The values listed are defined in the function ‘Reason for Lodger Stay’. This field is mandatory.

    2. Period From – Enter the date and time from when the lodger is going to stay in the facility or use the calendar function to select the period. This field is mandatory.

    3. Period To – Enter the date and time till when the lodger is going to stay in the facility or use the calendar function to select the period. This field is mandatory. The value specified must be within the period set in IP Parameter (Maximum Stay for Lodgers after Discharge [] Hrs).

    4. Nursing Unit – Select the Nursing Unit to which the lodger has checked in using [?] function.

    5. Authorized By – Enter the person who authorized the lodger to check-in in alphanumeric format of 200 characters maximum length. This field is mandatory.

    6. Remarks – Enter the Remarks (if any) regarding the check-in of the lodger in alphanumeric format of 200 characters maximum length.

  7. Click [Record] to check-in the lodger or [Close] to exit from the Check-In Lodger screen.
