The details recorded in the Ordered Examinations/Appointments function are displayed for the selected patient. The displayed details are Registration Date, Section/Wing, Appt Ref No, Order Number, Priority, Ordering Facility ID, Episode Number, Patient ID, Name, Gender, Age, Patient Type, Referral Source, Doctor and Queue number. You cannot modify the displayed details. The ‘Patient Type’ field is mandatory if the value ‘Patient Type’ is selected in the ‘Assign Queue By’ drop-down list in Sections for Operating Unit/Wing screen.

 

  1. Click [Request Comments] to view the Clinical Comments, Examination comments and the Item Narration. Click [Other Appt.] to view other appointment details for the selected patient. Click [Examination History] to view the examination history of the selected patient.

  2. Enter the details in the Examination and Views tab.

Examinations Tab

Enter the following details:

  1. Select – Select the check box against the required Examination to register the examination.

  2. Exam Code – Enter the Exam code in alphanumeric format of 10 characters maximum length.

  3. Description – On entering the Exam code, the description of the exam is displayed.

  4. Laterality – Select the laterality applicable for the examination from the list. The listed system-defined values are ‘Bilateral’, ‘Left’ and ‘Right’.

  5. Mobile Request – If a mobile request is placed for the exam violet color indicator is displayed. The mobile request status cannot be changed. Requests for a mobile Study Location are placed at the time of ordering the examination.

  6. Study Loc – The Study Location as entered at the time of appointment is displayed. Enter the desired Study Location code or click [List of Values] to select the Study Location.

  7. Radiologist ID – Enter the Radiologist ID or click [List of Values] to select the attending Radiologist. The Radiologist IDs are created in ‘Practitioners’ function in the Application Masters module.

  8. Name – On entering the Radiologist ID, the name of the selected Radiologist is displayed.

  9. Exam Date – The system displays the current date and time. Enter the required date in the format of dd/mm/yyyy or select the date using the Calendar function.

  10. Amount – The bill amount is displayed. This column is displayed if  Revenue Management module is interfaced with the Radiology Information Systems module and Requests and Results module for the system to display the bill amount as soon as an order is placed.

  11. [Edit Charge] – Click [Edit Charge] to invoke ‘Edit/Enter Charge Details’ screen.

  12. Exam Narration – Enter remarks for the examination.

Views Tab

The following details are displayed:

  1. Exam Code – The Examination Code selected in the Examination tab is displayed.

  2. View Code – The View Code applicable for the examination is displayed. This field is displayed if ‘view’ is applicable to the examination.

  3. Description – The description for the view code is displayed.

  1. Click [Save] to register the examination and a Print Outputs dialog gets displayed. The Required and Print check boxes against the Documents for example, Registration Label are selected based on Administration setup.

  2. Click [OK] to close the dialog and the documents for example, registration label gets printed.  If the duplicate examination is registered, the Duplicate Found screen is invoked.

    The Item Name, Start Date/Time, Ordered by, End Date/Time and Status are displayed.

     

    Click [Proceed] to invoke the Reason For Proceeding screen to enter the reason for proceeding the duplicate order or click [Stop] to close the Reason For Proceeding screen.

On clicking [Save], a pop-up message ‘Queue number is generated’ is displayed if Queue Management System (QMS) application is interfaced with Enterprise Management. The queue number can be entered only if the ‘Manual Allocation For Queue number’ check box is selected in the ‘RD Parameters for Facility’ screen.

If the  external QMS system is down, a warning message is displayed with OK and Cancel buttons. Click [OK] to enter queue number manually or [Cancel] to proceed with the transaction without entering any queue number.
