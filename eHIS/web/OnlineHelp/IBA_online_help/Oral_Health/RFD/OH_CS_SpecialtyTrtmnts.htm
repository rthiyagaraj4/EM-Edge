Enter the following details to create Specialty Treatments:

 

  1. Treatment Code – Enter the Treatment Code in alphanumeric format of 20 characters maximum length. A Treatment Code is defined to identify the Specialty Treatments/Procedures. The Specialty Treatments/Procedures have to be unique and cannot be changed once defined. This is a mandatory field.

  2. Treatment Description – Enter the Treatment Description for the Specialty Treatments/Procedures in alphanumeric format of 60 characters maximum length. This is a mandatory field.

  3. Treatment Category – Select a Treatment Category from the list to associate a Treatment Category to the Specialty Treatment/Procedures. All the Treatment Categories defined initially are displayed in the list. This is a mandatory field.

  4. Link Catalog – Select the check box to associate an Order Set to the Treatment Code. The check box is enabled, by default.

  5. Catalog Cross Reference – Click [?] function to select and associate an Order Set to the Specialty Treatment Code. The Order Set defined in the Order Entry & Result Reporting module gets listed. This is a mandatory field.

  6. Treatment has multiple tasks? – Select the check box, if multiple tasks are to be associated to the Treatment Code. Clear the check box and multiple tasks cannot be associated to the Treatment Code. In treatment plans such as, Complete Denture, there are multiple tasks to be performed like impression, border molding, articulation, first try in, etc.  These subsidiary activities are created as secondary orders and can be tracked individually including scheduling appointment, monitoring the status, recording the completion, resulting, etc.

  7. Crown Applicable - Select the check box, if the specialty treatment is applicable for crown. You can record treatment for all surface of a tooth in the Restorative Chart, if the check box is selected. If the check box is selected, the 'All Surface' option is enabled for recording. If the check box is cleared, you can record treatment on multiple surfaces of a tooth, but the 'All Surface' option is disabled. When the respective Specialty Treatment is selected, the surface is defaulted to 'All Surface' and disabled. The selected crown image is replaced with a new image and multiple cusp tip recording is displayed for selection. You cannot record any treatment on the surface when the image is replaced, but you can record treatments recorded to Arch, Quadrant and a tooth. You cannot record a condition or select Bridge Treatment when the image is replaced.

  8. Progress Note Group – Click [?] function to select and associate a Progress Note Group to the Specialty Treatment. The values that are defined through the ‘Note Type’ function in the Clinical Care Management module with the ‘Note Group’ as ‘Progress Notes’ are listed. The Progress Notes gets displayed in the Treatment function when the treatment is registered. If you do not associate ‘Progress Note’ to a Specialty Treatment, an error message is displayed when you click the ‘Progress Note’ hyperlink indicating that the Progress Note is not associated.

  9. Enabled – Select the check box to enable the record in the transaction functions. The check box is selected, by default.
