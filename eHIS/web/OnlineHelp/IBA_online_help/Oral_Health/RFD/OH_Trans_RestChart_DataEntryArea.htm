Selecting appropriate information for root, surface and entire tooth is explained:

 

  1. Category – Select a value from the list. The system-defined values are 'Advanced Periodontal Treatment', 'Bridge', 'Crown', 'Denture', 'Extraction', 'Impaction', 'Implant', 'Maxilla Facial Prosthesis', 'Oral Cavity', 'Orthodontic Appliance', 'Others' 'Root Treatment' and 'Tooth Treatment'. This is a mandatory field. You can record Pontic and Abutment details when a tooth is extracted and the Bridge treatment is recorded.

    You can record Bridge treatment on a Pontic and Abutment teeth. The teeth image is replaced by the respective images when you record Abutment and Pontic. The Pontic Tooth Details are displayed, if you record Extraction treatment on the Abutment.

    Enter the following details:

    1. Tooth No – The Pontic tooth associated to the selected Abutment is displayed.

    2. Select – Select the check box to record Pontic tooth details. Clear the check box, if the Pontic tooth details are not mandatory. The check box is selected, by default.

    3. [Record] – Click [Record]  to record Pontic tooth details.

    4. [Close] – Click [Close] to cancel the process.

    To record Pontic Tooth Details, you have to follow the steps mentioned below:

    1. Record Exfoliation Condition. The exfoliated tooth is considered to be the Pontic.

    2. Record Bridge Treatment. The tooth images are replaced by Pontic and Abutment images.

    3. Record Extraction on Abutment.

    4. Pontic Tooth Details screen appears.

    5. Record the details as mentioned above.  

    6. Extraction image is displayed on Abutment.

    7. Pontic tooth image is displayed as blank.

    8. When you try to extract another Abutment, Pontic tooth details screen is displayed with the check box already selected and disabled.

    1. The Pontic details window pops-up when you extract an Abutment tooth. The first check box is selected, by default.

    2. The Pontic details window pops-up for a treatment only if the extraction treatment is complete.

    3. The Pontic details window consists of tooth numbers and the corresponding check boxes. The Pontic teeth appear in a sequence in the Pontic details window. An error message is displayed, if the Pontic teeth are not selected in a sequence.

    4. The Pontic details screen pops-up when you mark successful extraction treatment as a failure.

    5. The icon representing the Pontic tooth is removed, if the Pontic tooth is cut. The menu options for treatment on the Pontic changes to Denture and Implant when the Pontic tooth is cut.

    6. The Pontics that are excised reappear when the Extraction treatment is marked as an error.

    7. You can extract a tooth without recording Pontics.

    You can record ‘Pontic’ and ‘Abutment’ for a Bridge through an external treatment or a treatments tab. The missing tooth is considered to be the Pontic and the adjacent teeth that supports the bridge treatment is called the Abutment. Pontic is represented using icon and Abutment is represented through icon in the Restorative chart. You can also cancel the treatment that is recorded.

    1. The bridge related treatment has to be marked as failure in order to record any crown related treatment on the abutment.

    2. When you record a Bridge treatment the missing tooth is replaced by icon and the existing tooth is replaced with icon.

    3. The Pontic icon replaces the crown and the root is absent for a Pontic.

    4. The Abutment icon replaces the crown and the root image is present.

    5. The root related conditions are valid for Abutment.

    6. External Treatment/Treatment and Tooth Treatment are valid for an Abutment irrespective of the success or failure of Bridge Treatment.

    7. Extraction is valid for an Abutment and Pontic, if the Bridge treatment is a success.

    8. You can excise either of the Abutment tooth, if the Bridge treatment is a failure.

    9. The menu options for treatment on the Pontic changes to Denture and Implant when the Bridge Treatment fails.

    10. The limitation for the Bridge treatment is restricted to five tooth and is valid for permanent dentition.

    11. The Pontic tooth that is excised is shown against the Bridge recordings as a tool tip when you click the history area of the respective Pontic tooth.

  2. Treatment – Select a Treatment from the list. The treatments associated to the selected Treatment Categories are displayed. It identifies the treatment for the orthodontic condition recorded using the Condition Type tab initially. This is a mandatory field.

When the Treatment Category is selected as ‘Maxilla Facial Prosthesis’, treatments associated to the selected Treatment Category are listed. The system-defined values for ‘Site’ and ‘Area’ differ based on the Treatment Category selected. When you select the ‘Area’ as ‘Intra Oral’ or ‘Extra Oral’, the values in the list for ‘Parts’ change accordingly.

=> If the Condition is recorded as 'Supernumerary' for a tooth, respective image is displayed to indicate the condition recorded against the tooth and the tooth number is displayed with a suffix as 's'.  The Base tooth number is highlighted using a color indicator. Click on the tooth number displayed as a hyperlink to record a treatment or view the treatment. Click on the image to view the treatments that are recorded.

=> You can record crown related treatment only if the external treatment is recorded as failure.

=> Implant treatment is not applicable for supernumerary and exfoliate supernumerary tooth.

=> You cannot record any external treatment or treatment, if the external treatment or a treatment recorded for a crown or root is marked as successful.

=> External treatment or a treatment performed on a root or crown that is marked as a failure is indicated using brown color.

=> You can record a Denture treatment category multiple times, if it is recorded as ‘Removable’. If the treatment category is ‘Fixed’, you cannot record it multiple times unless the status is marked as failure.

=> You can record denture on a tooth, tooth range, arch or quadrant.

=> Denture is applicable for permanent dentition and not valid for mixed dentition or deciduous tooth.

=> The removable denture has to be registered to add another removable denture on the same location.

=> Fixed denture has to be marked as failure to add another fixed or removable denture.

=> You cannot record denture for a set of tooth, if a tooth within the selected range is not marked as failure.

=> You can record removable denture only on denture of different categories.

=> You can record a fixed denture only on a fixed/removable denture if it is marked as failure.

=> You cannot record denture on a set of tooth, if the outcome is set as Success and Failure for each tooth.

=> Denture can be recorded if the tooth is missing. Reasons for missing tooth can be impaction, exfoliation or extraction.

=> Denture is applicable for 3rd molars.

=> You can record Denture on an extracted tooth irrespective of the outcome.

=> You cannot record Implant for a successful Denture.

=> You can record fixed denture on a removable denture, if it is marked as failure.

=> There should be only one Removable denture category at a time.

  1. Site – Based on the Category selected, the Site details are displayed. The values are system defined and differ based on the Category selected. This is a mandatory field. The Site values can be:

  2. Arch – If selected, the Treatment Category is applicable for the Arch.

  3. Quadrant – If selected, the Treatment Category is applicable for the Quadrant.

  4. Tooth – If selected, the Treatment Category is applicable for a tooth.

  5. Tooth Range – If selected, the Treatment Category is applicable for a tooth range. You can select the tooth numbers for which the Treatment Category is applicable.

  1. Area – The Area differs based on the Site selected. This is a mandatory field. The values are:

  2. If the Site selected is ‘Arch’, the Area is set to ‘Arch’. You can select either the Upper Arch or the Lower Arch from the list. The values are system defined.

  3. If the Site selected is ‘Quadrant’, the Area is set to ‘Quadrant’. You can select either the Upper Right, Upper Left, Lower Right or the Lower Left quadrants from the list. The values are system defined.

  4. If the Site selected is ‘Tooth’, the Area is set to ‘Tooth No’. You can select a tooth from the list. The selected Treatment Category is applicable for the tooth selected. The values are system defined.

  5. If the Site selected is ‘Tooth Range’, the Area is set to ‘Tooth Nos’. You can select a range of tooth from the list using [Ctrl] key. The selected Treatment Category is applicable for the selected tooth range. A minimum of two teeth has to be selected for the tooth range. The values are system defined.

  1. Surface – The value is set as Select, by default and the field is disabled. You can record treatment for all surface of a tooth in a Restorative Chart, if the check box 'Crown Applicable' is  selected for the treatment in 'Specialty Treatment' function. If the check box is selected, the 'All Surface' option is enabled for recording. If the check box is cleared, you can record treatment on multiple surfaces of a tooth, but the 'All Surface' option is disabled. When the respective Specialty Treatment is selected, the surface is defaulted to 'All Surface' and disabled. The selected crown image is replaced with a new image and multiple cusp tip recording is displayed for selection. You cannot record any treatment on the surface when the image is replaced, but you can record treatments recorded to Arch, Quadrant and a tooth. You cannot record a condition or select Bridge Treatment when the image is replaced.

  2. Status – The Status is set to ‘Ordered’, by default. The value is system defined. You can record the treatment against the selected area on selecting a system defined status. This is a mandatory field.

Click [Record] to save the details or [Reset] to clear the data and to enter new external treatment. Multiple entries can be stored against the current encounter, as patient might have undergone multiple treatments for teeth elsewhere prior to the current visit. Click [Record] to record the treatment. On entering the treatment details and clicking [Record], the Order Entry screen gets displayed with the Orderable, Order Type, Priority, Quantity, Frequency and Duration. You can update the Start Date, Priority and the Charges. The recorded treatments get listed under the ‘Treatments To Do’  menu.
