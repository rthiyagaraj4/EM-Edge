Sign Notes

From this function you can Sign clinical documents that are pending for signature.

Signing is a step which finalizes a clinical document after which a document is only allowed to be modified or marked as error. Practitioners use this function to sign notes prepared by them and those that require verification prepared by transcribers or junior staff. Using this function a practitioner can view the note content, update, mark the note as error and sign it for verification.

Signing Notes

From the CA Transactions > Clinician Access > Desktop Menu > Pending Activities > Verifications, select the Sign Notes function.

The Signing the Notes screen gets displayed.

  1. Notes pending for Signature is displayed for the default date range. Enter the query criteria and click [Search] to display specific orders for Signing. Click [Clear] to clear the displayed data.

    1. Requested By – Select the Requested By using [?] function. ‘All’ is displayed, by default. This is applicable when the note is prepared by a person other than the logged in practitioner and has requested for verification.

    2. Patient ID – Select the Patient ID using [?] function.

    3. Period - The date range is displayed, by default. Enter a period in date format of dd/mm/yyyy.

    1. Note Type – The various types of notes requiring sign are displayed on the left hand side of the screen along with the count of number of notes pending for signature. The displayed Note Types are hyperlinked. The option “All” provides viewing of all notes requiring signature. You can click on a Note Type to view the notes requiring consent.

    2. Sign – Select the check box to select Notes for signing. All the check boxes are cleared, by default. To sign all the displayed notes, select the check box provided before ‘Sign’ heading.

    3. Subject – Subject for the note is displayed, if recorded.

    4. Requested At – The date and time of the request is displayed.

    5. Request By – The Practitioner requesting sign is displayed. If the logged in practitioner has performed recording and is signing the document, ‘Self’ is displayed.

    6. Patient Details – The Patient details are displayed. The details are Patient Name, Age, Gender, Patient ID, Attending Practitioner, Care Location and Encounter ID.

    7. Forward To – This field is applicable when logged in practitioner has been assigned the role of ‘Author without Authorization’ against a note. You can select the author to whom the note has to be forwarded after signing.

  2. Click View hyperlink to view the details of the Note to be signed.

    Click 'View' hyperlink to view the Notes recorded for a particular patient. The screen displays options for marking error, updating and signing based on the rights assigned to the practitioner requesting signature. Click [Close] to close the screen.

  3. Select the check box against the required patient details and click [Sign] to the confirm signing the selected notes.
