Privilege Group

From this function you can Define Privilege Groups used to restrict or control Practitioner privileges for various tasks.

The Privilege Groups are used in the Order Entry and Results Reporting module and Pharmacy Management module. A Privilege Group is defined at the level of an Order Category or Order Type or at the level of an individual Orderable.

Defining Privilege Group

From the OR Admin Setup, select the Privilege Group function.

The Privilege Group screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the details in the mandatory (*) fields. <Related Field Description>

  3. The Order Categories are displayed in the Selected Order Category frame.

Select Tab

  1. The tabs Ordering and Authorizing are displayed in the Select Tab. You can associate rules to these Order Categories.

  2. Select the Copy From value from the list. Select the required tab and assign rules to the required Order Category. To copy the same rules assigned to the Order Categories of one tab to another tab, select the required tab name from the list in which the rules are assigned.

  3. Select the Rule against the required Order Category to associate the Order Category to the Rule.

  4. Click Order Category hyperlink to view the Order Types associated to the selected Order Category.

  5. Select the Rule against the required Order Category to associate the Order Type to the Rule.

  6. Limit To – Select the values from the list. The listed values are ‘All’ and ‘Authorization Applicable’. Authorization Applicable is selected, by default.

Associate Tab

  1. All Order Categories for which rules are already defined are listed under this tab.

  2. The Order Categories are displayed in the tree structure at the left hand side of the screen.

  3. The Rules associated to the Order Categories and Order Types are displayed in the right hand side. You can edit the Rules associated to the Order Category and Order Type.

  1. Click [Apply] to save the Privilege Group or [Reset] to clear the entered data.

  2. Click [Query] to search for existing Privilege Group records.
