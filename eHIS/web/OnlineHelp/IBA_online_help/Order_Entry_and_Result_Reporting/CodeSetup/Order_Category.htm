Order Category

From this function you can Update an Order Category.

The Order Category function enables you to change parameters for the selected Order Categories. The Order Categories are system defined. The Order Categories listed are Departmental Diagnostics, Dietary, Laboratory, Miscellaneous, Monitoring, OT Surgery, Oral Health, Order Set, Patient Care, Pharmacy, Radiology and Trmt/Procedures.

Updating an Order Category

From the OR Code Setup, select the Order Category function.

The Order Category screen gets displayed.

  1. Enter the long description for the selected Order Category in alphanumeric format of 30 characters maximum length. This is a mandatory field.

  2. Enter the short description for the selected Order Category in alphanumeric format of 15 characters maximum length. This is a mandatory field.

  3. Select the check box ‘Gen External Application Accession Num’, if the order category has to be interfaced to an external application. In such a case, a number is generated (in the Enterprise Management application) and passed on to the external application for identification.

  4. Select the check box 'Clinical Comment Mandatory' if recording clinical comments for the Order Category is mandatory.

  5. Select the check box 'Allow Multi Patient Resulting' to populate the order category in the Multi Patient Result Reporting function. The Allow Multi patient Recording is not applicable for Consult Order and Order Set Order Categories. If Allow Result reporting is not set for any of the order categories, in Multi Patient Result Reporting function an alert message 'Order Category not setup for multi patient result reporting' is prompted on clicking the Order Category list item.

  6. Select the 'Result Type Displayed Control' check box to display the selected Order Category in the Result Type list box in Result function of Clinician Access module.

  7. Select 'Completion Remarks Mandatory' check box against the appropriate Order Categories to indicate that entering Remarks for completed orders is mandatory.

  8. Select 'Prompt for Book Appointment on Order' check box to prompt for OT booking while placing an OT order.

  9. Select 'Reason code for Hold/Discontinue is Mandatory’ check box against the appropriate Order Categories to indicate that entering reason code for holding or discontinuing orders is mandatory.

  10. Select 'Reason code for Not Done is Mandatory’ check box against the appropriate Order Categories to indicate that entering reason code for Not Done orders is mandatory. This check box is enabled only for Treatment Procedures and Patient Care order categories.

  11. Select the value for Pin Number for placing orders   from the list.

    1. Logged in Practitioner’s PIN - The system prompts for Logged in Practitioner’s PIN, for every Order category configured, when this value is selected.

    2. Ordering Practitioner’s PIN - The system prompts for Ordering Practitioner’s PIN, for every Order category configured, when this value is selected.

    3. Logged in Practitioner’s PIN Prompt once for all Categories - The system prompts only once for all the orders belonging to different categories. When this value is selected , all the other categories having different values are over ridden.

    4. Ordering Practitioner’s PIN Prompt once for all Categories - The system prompts only once for all the orders belonging to different categories. When this value is selected, all the other categories having different values are over ridden.

    5. Not Applicable - By default, this value is selected for all order categories. The system does not prompt if this value is selected.

  12. Click [Apply] to save the updated details or [Reset] to clear the entered data.

The ‘General External Application Accession Number’ is used in cases where external applications are interfaced to the Enterprise Management application. For example, if the Order Category is ‘Radiology’, a number is generated (in the Enterprise Management application), passed on to the external application for identification, and linking of the image sent from our application to the external application.
