Existing

Click Existing Orders tab to view existing orders of the selected patient.

The clinician can view order details and perform activities like cancel, amend, discontinue, hold or resume orders using the existing orders tab. The practitioner can enable an order having a future date. To enable a future order, the Encounter ID at the time of ordering and Encounter ID at the time of activating future order has to be different. Viewing of orders can be performed at two levels, Header or Line level. Header level contains multiple Order Catalogs grouped under a single Order ID (based on Order Type). The line level contains individual Order Catalogs. System defined or user defined status is displayed against each order. Depending on the tasks specified in the function ‘Tasks for Resp/Reln’ function in ‘Clinician Access’ module; the clinician can perform the activities.

  1. Existing Orders are displayed for the default date range. Enter the query criteria and click [Search] to display and authorize specific orders. <Related Field Description>

  2. Click [Clear] to clear the displayed data.

  3. To View, Cancel, Amend, Preps, Comments, Instructions and Link to Current Encounter; click the Order Catalog hyperlink and select the required option from the list. The accession number of the Order Catalog is displayed on mouse over.

View Order

Click ‘View’  hyperlink to view the order details. The displayed values cannot be edited.  

If any modifications are done in the clinical comments of an existing order, the Clinical Comments hyperlink is available. Click the Clinical Comments hyperlink to open the Clinical Comments dialog.

  • In the Clinical Comments dialog, the recently recorded note is displayed in the text box along with the  Amended By name, Amended Date and Amended Reason. These are read only fields.

  • To view the previously modified clinical comment, click the respective Number hyperlink and the modified comment appears. For e.g., on clicking ‘1’ hyperlink, the first clinical comment entry gets displayed.

Click [View  Amend Reason] to view the reasons for amending the order with the amended date/time, ordered ID in a new dialog. This button is visible only if the order is already amended, else it will not appear.

Click [View Audit Trail] to view the audit trail details of the order in a new dialog. A hyperlink ‘R’ appears in the Remarks column  when there is a duplicate order for the patient. Click the ‘R’ hyperlink to view the reason for duplicate order.

On viewing the required details, click [Close] to close the View Order screen.

 

In addition, you can see the both Header and Line level Order Format appearing as hyperlink. On clicking that hyperlink, the Order Line Format dialog gets displayed.  In that dialog, a ‘View Audit Trail’ button is available to allow you view the modifications done.

Cancel Order

Click ‘Cancel’ hyperlink to cancel the Order. The Cancel Reason dialog gets displayed.  

  1. Select the appropriate cancel reason from the list provided. The values displayed in the list are defined in the function ‘Reason code’ in Order Entry and Result Reporting (OR) module.

  2. On selecting the reason, description of the reason is displayed in the text box provided. You can modify the displayed text if required.

  3. Click [Record] to confirm the entry of the reason. The cancel reason window closes and control shifts back to the Existing Orders screen. The cancelled orders are removed from the list.

  4. Click [Close] to close the Cancel Order screen.  

Discontinue Order

The practitioner can discontinue an order using the ‘Discontinue’ function. You can discontinue order at both ‘Header’ and ‘Line Level'.

The Discontinue option is not available for radiology orders with status ‘Performed’.

Click ‘Discontinue’ hyperlink to discontinue the Order. The Discontinue Reason dialog gets displayed.  

  1. Select the Discontinue reason from the list provided. These values are defined in the function ‘Reason Code’. Only discontinued Reason codes belonging to the order category of the order catalog are displayed in the check box. On selecting the reason code, description of the code is displayed in the text box provided. Edit the content

  2. Click [Record] to record the reason for discontinuing.

  3. Click [Close] to close the window

Amend Order

Click ‘Amend’ hyperlink to amend the order details.

  1. Select the required amending reason from the drop-down list. The amending reason description gets displayed in the text box. The ‘Amend’ reason type gets populated from the Reason Code screen in OR module.

  2. Enter the reason for amending the order in the text box if required.

  3. Click [OK] to close the Amend dialog.

  4. The Order to be amended gets displayed in the Amend Order screen. Make the required changes to the order and click [Record].

  1. The Status column is displayed with button if the’Record Consent Form With Order’ check box is selected in the OR Parameter screen. Click [RC] button to record consent for that order ID. A Notify icon appears for Resulted Complete and Rejected statuses for lab orders in Header level if notification details are available for that order. Click icon to view the notification details in a dialog. The Notify icon appears only if the ‘Notify Details in Clinician Access’ check box is selected in Departmental Parameter for Facility screen.

  2. The Medication Administration status is displayed with unique colour indicators. Click on the colour indicator to view the Medication Administration Details in the Medication Administration Detail screen. The Medication Administration Detail column is displayed if the ‘View Order By’ is selected as ‘Line’. When the doctor selects the medication order and any dose of medication is administered in ‘Medication Administration’ function, the medication status is displayed with    icon, which indicates the medication administration is in progress. When the last dose or a single dose is administered to the patient, the Medication Administration status is displayed in icon, which indicates the medication administration as completed. On placing the cursor on the icon, it displays a tool tip ‘Progress’. On placing the cursor on the icon, it displays a tool tip ‘Completed’.

  3. Select the check box against the required orders catalog and click [Print] to print orders. An Authorize Pin Form dialog gets displayed if ‘Confidential Pin Required for Printing’ check box is selected in CA Parameter screen. Enter the pin you have defined in the User screen in System Manager module and click [OK] to print the clinical details of the patient.

  4. Select the check box and click [Copy Previous Orders] to copy the selected order.
