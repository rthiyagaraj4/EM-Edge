Enter the Drug details:

  1. Drug – Enter the Drug name in alphanumeric format of 60 characters maximum length or select the required Drug from the Drug Search dialog using [?] function. The Drug selection can be limited to a single Order type or across Order types based on OR Parameter Setup. Click to view the Essential Classification details and the information regarding Pregnancy Effects. Once the drug is selected, the other details are populated in the respective fields. You cannot modify the drug that is tapered.

While ordering the drugs for the pregnant patient, the alert displays the adverse effect of the prescribed drug defined in the Drug screen. The adverse effect captured within the Drug screen is displayed with the Pregnancy Category. The drug can be prescribed after entering the override reason. Without entering the override reason, the system invokes an alert message ‘Please capture the override reason’ on clicking the OK button. On clicking Close button the alert message is closed.

A new icon is introduced to display the alert message. Click the icon to view the alert message.

 

The details of the first drug are displayed in the ‘Drug details’ area of the screen, by default. The Practitioner can select the required Drug from prescription pad onto the drug details section for viewing/modifying the details. The following details are displayed.

On selecting the Drug, the relevant templates of the sliding scale associated to the Drug are displayed. The default template is selected, by default. You select any template which is associated to the Drug. The sliding scale template holds good for the current prescription only. If the clinician copies the previous order or the current medication, the customized sliding scale template will be available. You can release the order without recording the scheduled dose.

 

  1. Priority – Routine is displayed, by default. Select the value from the list. The other priority listed is Urgent.

  2. Route – The default route specified for the selected drug is displayed, by default. Select the required route for administration from the list. The listed values correspond to the form of drug. Form of a drug is associated to various routes in the function ‘Route for Form’. The selected route color depends on the setup done in Admin Route.

  3. Form - Form of the selected drug is displayed based on the drug setup as read only text.

  4. Strength – Base strength of the selected drug is displayed as read only text.

  5. Build MAR – Medication Administration is applicable for all patient classes depending on the Parameter settings. For inpatients, Build MAR is default checked and enabled. For remaining patient classes based upon parameter settings it will be checked and enabled.

If the check box Print Label in PH Parameter for Facility and the Build MAR check box in the Pharmacy Orders screen are selected, the MAR Label can be printed.

  1. Trade Name – The trade name of the drug is displayed, by default. Select the drug name from the list.

  2. Authorize/Cosign/Special Approval - The ‘Authorize’ icon displays based on the setting in Drug catalog. It indicates the selected drug requires authorization for prescribing. These drugs are to be authorized by users with authorizing rights. If necessary rights are not provided to the logged in user, ‘Requires Authorization’ displays, or ‘Authorize’ hyperlink gets displayed.

 

Dosage Details

  1. Dosage By – By default, the Dosage By value is selected based on the ‘Default Dosage By For Normal Rx’ in the Drug master. The listed values are Quantity and Strength.  

 

Dosage By: Strength

This value is displayed provided base strength is defined for the selected drug. The ‘Strength’ field displays the set base strength value and unit of measurement. The practitioner can change only the strength value.

  1. Recom. Dosage – The recommended drug dosage is displayed. The standard dosage set for the drug in the function ‘Drug Dosage Limit’ is displayed. Click on the ‘Recom. Dosage’ hyperlink. The recommended dosage is displayed in the ‘Strength’ field, which can be changed.

 

Duration Details

The following duration details are displayed.

  1. Duration – Duration specified for the selected frequency while setting up the selected drug is displayed by default. The duration can be changed only in certain cases. If the selected frequency is of the nature ‘One Time’, then the duration is made default as ‘One’ as read only text. For frequency of nature ‘PRN’, the duration defaults as one day. This value can be changed to the required number of days. The practitioner can enter a value up to 999 days. For frequency of type ‘Frequency’, the drug setting value is displayed. This value can be changed to the required period. If the entered duration exceeds the duration limit defined in Order catalog setup, then the system displays message specified by you and depending on the setting of the order catalog appropriate action to warn or reject or authorization required is performed.

  2. Start Date –The current system date and time of medication is displayed as start date of medication. The displayed date and time can be pre-dated or post-dated based on the setting in OR Parameter function. Enter the required date in date format of dd/mm/yyyy hh:mm or use the calendar function to select the required date.

  3. End Date -The end date and time is calculated based on the duration and start date. It indicates the end date for the prescribed medication. The calculated date cannot be changed.

The prescription Start Date and End Date is defaulted with home leave date based on the home leave applied within the PAS module for the inpatient. If the doctor modifies the Start Date and End Date or selects the weekly or monthly frequency, which is beyond the home leave date, the following validation message is displayed:

“The Start Date / End Date is beyond the home leave date.”

  1. Order Quantity -Enter the Quantity which a  Practitioner can advice a suitable Order Quantity.

  2. Refill – Displays the Refill dialog which allows you to modify the Refill quantity and displays the Start date and End date.

  3. Dispense Location - Based on the order routing specified for the selected drug, the dispensing location is displayed. Practitioner cannot change the location. The system verifies routing in the following order.

  1. Taper Level Qty – Displays the quantity of the each taper level. You can enter any value including ‘0’. You can modify the taper level qty even after the order is confirmed but you cannot modify it once the order is released.

  2. Total Taper Qty – Displays the previous taper level quantity. You can modify this quantity and the system displays the adjusted quantity as dispense quantity. The Total Taper Qty field will be recalculated based on the changes made in Taper Level Qty field.

  3. Patient Counseling Required – Select the check box to indicate patient counseling is required on dispensing. This field is applicable if ‘Patient Counseling Required’ is checked for the drug in ‘Drug’ master.

  4. Patient Brought Medication –Select the check box to indicate the prescribed drug has already been brought by the patient. It is an indication to the facility pharmacy not to dispense the prescribed drug. This field is available only for inpatients and will not appear for outpatient/emergency patients, if the selected drug class belongs to controlled/narcotics and for discharged patients.

The font and background color of the Route value is based on the setup done in Admin Route. The Total Taper Qty Prescribed/Dispensed are displayed.

Charge Details
