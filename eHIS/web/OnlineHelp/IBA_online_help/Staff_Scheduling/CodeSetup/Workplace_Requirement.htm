Workplace Requirement

From this function you can Create Workplace Requirement and Query for an existing Workplace Requirement.

To provide a comprehensive roster functionality to cater to the needs of any facility in the enterprise, it is necessary to identify the allocation criteria and the staff requirement for all the workplaces in the facility. The staff requirement indicates manpower that is required to efficiently handle the operations at a given workplace. It also defines the various positions of staffs to be allocated to the workplace and the number of staffs in a particular position for a given shift. The workplace requirement can be varying or constant. Constant requirements are set without specifying a date range, as it is applicable to the workplace as long as the workplace exists. Varying requirements are set for a specific date range and expire after the period is over.

The allocation criteria is also set up for staff resource to patient ratio at workplaces where the number of patients may vary over a shift period. This helps in relieving staff in such locations and relocates them to other locations where there may be a requirement for staff of the same position.

This can be extended to cater for the staff to bed ratio if, at any given point of time the number of persons at a workplace is to be dependent on the number of beds, irrespective of the occupancy level.

Finally, requirement is also defined by full time equivalent of staff required for a workplace for a shift.

Access to specific workplace(s) for setting up the requirement is controlled by a parameter at user level for workplace(s) in the function 'Access Rights'.

Creating Workplace Requirement

From the RS Code Setup, select Workplace Requirement function.

The Workplace Requirement screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the details in the mandatory (*) fields.  <Related Field Description>

  3. Click [Apply] to save the Workplace Requirement or [Reset] to clear the entered data.

  4. Click [Query] to search for existing 'Workplace Requirement' records.
