Requested Blood Units

From this function you can Query blood unit details that are requested from a storage location and transfused to the patients.

The Requested Blood Units function enables you to view the details of blood units that have been requested from a storage location and transfused to the patients. You can query the details based on specimen number, date range, priority and blood product. You can also transfuse the blood units and process a request for the allocation. The mode of request of the blood units along with the number of requests are indicated with the following color codes.

- Stat

- Urgent

- Routine

Querying Requested Blood Unit details

From the Queries, double-click Requested Blood Units function.

The Requested Blood Units screen gets displayed.

  1. Enter the query criteria such as Priority, Specimen Number Range, Product Range, Category Range, Source Type, Start and End date, and the Expected Transfusion date range to view the blood unit details.

  2. Click [Specimen List] to view the blood unit status. The Specimen List screen gets displayed with the details of all the specimens. You can view the following details of a selected specimen from the Specimen List screen.

  3. Click [Patient’s Antibodies] to view the Antibodies of the selected blood unit.

  4. Click [Patient’s Requirements] to view the Requirements of the selected patient.

  5. Click [Transfusion History] to view the Transfusion History and print the details.

  6. Click [Process Request] to request for a new process of the blood unit. The Allocate/Xmatch/Issue – Blood Units screen gets displayed.

  7. Click [Transfuse] to transfuse the selected blood unit. The Transfusion of Blood Units function screen gets displayed to view the transfusion details.  

  8. Click [Blood Unit Details] to view the specimen unit details and print the details.
