Enter the following details:

 

  1. Source – Click [List of Values] to select the default source. The list displays the values defined in ‘External Referral Source’ function in the Application Masters module.

  2. Consultant – Click [List of Values] to select a consultant. The list displays the names of the users, who are associated to the selected source through the ‘External Practitioner for Referral Source’ function in Application Masters module. This value is used as default value.

  3. Patient ID – Enter the Patient ID that is used as default Patient ID, when a unit from Collection Centre is sent to the Laboratory for investigation. This is a one-time setup.

  4. Specimen Validity period (in Hrs) – Enter the Validity period in hours for the specimen after which it has been collected.

  5. Time Period Unit for Blood Unit Reservation (In Hrs) – Enter the validity period in hours for the Blood Unit reservation.

  6. Default Volume – Enter the default volume of the product. This volume is to be used as the default value of the product.

  7. Volume Units – Click [List of Values] to select the Volume Units. The list displays the values defined in the ‘Unit of Measurements’ function in Application Masters module.

  8. Time Period to Modify Blood Unit Reserved Date (In Hrs) - Enter the time period to allow modifications of blood unit reserve date. The sum of time period specified and Time Period for Blood Unit Reservation must not exceed time period specified in the 'Specimen Validity Period' field.

  9. Entry of Units into Stock from External Source Allowed - Select the check box if the blood units from External sources are allowed in the stock. If the check box is cleared, blood units from External sources are not allowed in the stock. Blood units are added to the stock using the 'Blood Units Entry' function.

  10. Simultaneous Allocation of Blood Units – Select the check box to allocate single blood unit against different specimen numbers.

  11. Automatic Deletion of Simultaneous Units On Issue – Select the check box to automatically delete units allocated against different specimens, once the unit is issued for transfusion. Clear the check box to delete the units manually after transfusion.

  12. Print Blood Tags – Select the check box if blood unit tags are to be printed while allocating and issuing blood units. Clear the check box, if blood unit tags are not to be printed during blood allocation.

  13. Issue without Collection Slip – Select the check box if Blood units can be issued without issuing a blood collection slip. Clear the check box if Blood units cannot be issued without issuing a blood collection slip.

  14. Cross match result before transfusion – Select the check box to complete the crossmatch tests before transfusing the unit. Clear the check box, if crossmatch test is not mandatory before blood transfusion.

  15. Emergency comment code – Click [List of Values] to select an Emergency Code. The values created through the ‘Crossmatch Comments’ function are listed. The selected value is  displayed as the default value while issuing blood units in emergency.

  16. Restrict regrossing after sample is consumed - Select the checkbox to restrict regrossing function  after the sample is consumed. By default this checkbox is selected.

  17. Collection Centre (Pooled Unit) – Click [List of Values] to select a default collection centre. The values created through the ‘Collection Center/Supplier’ function are listed. The selected value is displayed as the default value for Collection Centre in Pooled Units function.

  18. Fate (Pooled unit) – Click [List of Values] to select the default fate code from the list of values. The list displays the values defined in the ‘Fate Codes’ function. The selected value is displayed as the default fate code value while pooling the blood unit.

  19. Fate (Transferred Unit) – Click [List of Values] to select a Fate Code. The list displays the values defined in the ‘Fate Codes’ function. The selected value is displayed as the default value while transferring blood units.

  20. Fate (Partially Transfused Unit) – Click [List of Values] to select a Fate Code. The list displays the values defined in the ‘Fate Codes’ function. The selected value is displayed as the default value If the blood transfusion is discontinued in the middle of the process and the clinician wants to dispose the blood.

  21. Fate (External Issued Unit) – Click [List of Values] to select the Fate Code to be used while issuing the blood units to the External Sources. The list displays the values defined in the ‘Fate Codes’ function. The selected value is displayed as the default value while issuing the blood units to an external source.

  22. Use Neonatal Permissible Blood Type Substitution? – Select the check box to enable Neonatal Permissible Blood Type Substitution. Neonatal Permissible Blood Types are used for issuing blood units to newborn babies. This is used in 'Allocate/Xmatch/Issue - Blood Units' function.

  23. Confirm Blood Typing By Operator – Select the check box to reconfirm blood typing when you receive a blood unit from Blood Donor, External Source in the Confirm Blood Typing function.  

  24. Validate Antibody Presence – Select the check box to enable [Validate] in 'Allocate/Xmatch/Issue – Blood Units' function.

  25. Print Compatibility Report - Select the check box to print the Compatibility X-Match Report and Compatibility Report.

  26. Test Method - Enter the test method name to be printed in the Compatibility X-Match Report and Compatibility Report.

  27. Enable Charging from second cross-match (OP/EM) – Select this checkbox to enable to collect charges for further cross – matches.

  28. Allow Refund If blood bag returned within (OP/EM) – Enter numeric value as minutes. based on this time duration the charges collected for blood bag are refund.

  29. Stop Issue if cross-match charges are unbilled / outstanding – Select this checkbox to stop the issue if cross-match charges are unbilled / outstanding.
