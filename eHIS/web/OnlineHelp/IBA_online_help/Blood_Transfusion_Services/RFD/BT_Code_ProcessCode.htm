Enter the following details to create Process Codes:

 

  1. Code – Enter the Process Code in alphanumeric format of four characters maximum length. This is a mandatory field.

  2. Long Description - Enter the Long Description of the Process code in alphanumeric format of 40 characters maximum length. This is a mandatory field.

  3. Short Description - Enter the Short Description of the Process code in alphanumeric format of 15 characters maximum length. This is a mandatory field.

 

Expiry Period

  1. Period – Enter the Expiry Period for the defined process in numerical format of three digits maximum length.

  2. Unit – Select appropriate Unit for expiry period from the list. The system-defined values are ‘Years’, ‘Months’, ‘Weeks’, ‘Days’ and ‘Hours’.

  3. Compute Expiry Based On - Select 'Process Date' or 'Donation Date' to compute the expiry date. If 'Process Code' is selected, the Expiry date is calculated based on the process date and if 'Donation Date' is selected, the Expiry date is calculated based on the donation date.

  4. Refund Bill on Return - Select the check box to refund the billed amount charged while issuing the blood unit on returning the blood unit back to the blood bank. If the check box is cleared the billed amount is charged for the blood unit, even if it is returned due to any reasons.

  5. Action Indicator – Select the action indicator values from the list. The system defined values listed are ‘Error’, ‘Warning’ and ‘No Action’. ‘Error’ is selected, by default. This is a mandatory field.

  6. Error – Select the action indicator 'Error' from the list of values. On selecting the action indicator as 'Error', the system displays an error message if the order process and the issue time of the unit is different.

  7. Warning – Select the action indicator 'Warning' from the list of values. On selecting the action indicator as 'Warning', the system displays a warning message if the order process and the issue time of the unit is different.

  8. No Action - Select the action indicator 'No action' from the list of values. If the action indicator is selected as 'No Action', the system does not display any message if the order process and the issue time of the unit is different.
