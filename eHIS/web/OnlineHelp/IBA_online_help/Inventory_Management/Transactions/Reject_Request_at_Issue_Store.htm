Reject Request at Issue Store

From this function you can Query and Reject requests which are not authorized or issued (not even partially).

This function is used to reject request which are not authorized / issued (not even partially). User can reject either the entire document or specific items inside the document.

For example, if there are five items in a request and three items have been authorized / issued (partially / fully) only remaining two items in the request can be rejected by selecting individual item. User cannot reject the entire request document in this case. But if all 5 items in the request have not been authorized / issued, the entire document can be rejected.

Rejecting Request at Issue Store

From the ST Transactions > Issue, select the Reject Request at Issue Store function.

The Reject Request at Issue Store screen gets displayed.

  1. Click [Query] to invoke the Query screen.

  2. Enter the  search criteria  and click [Search] to search for Issue Requests matching search criteria.

    Enter the following search criteria:

    1. Requests Of – Select the required value from the list. The listed values are ‘All’, ‘Internal Stores’ and ‘External Stores’. The value ‘All’ is displayed, by default.

    2. Doc Type – Select the required Document Type from the list. The documents associated to the transaction ‘Requisition Issues’ are displayed in the list. The Document Type, as set in ‘ST Facility Parameter’ is displayed, by default.

    3. Doc No. - Enter the Document Number in numeric format of eight digits maximum length. If the system generates the number, you are not allowed to enter the field value.  

    4. Req By Store - Select the Requested by Store from the list. Only those Stores for which you have access are displayed. Store access is provided by selecting ‘Reject Request  at Issue Store’ check box in  ‘ST User Access for Store’ function.

    5. Req On Store - Select the Requested on Store from the list. Only those Stores are displayed for which you have access. Store access is provided by selecting ‘Reject Request  at Issue Store’ check box in  ‘ST User Access for Store’ function.

    6. Item Class – Select the required Item class from the list. The Item Class associated to the selected Requested By store and Requested On store are displayed in the list.

    7. Doc Date [From – To] - The system date is displayed, by default. You can enter the date in date format of dd/mm/yyyy or select the required date using the Calendar function. This is a mandatory field.

    8. Req Status – Select the required Request Status from the list. The listed values are ‘All’, ‘Authorized at Req by Store’ and ‘Partially Issued’. The value ‘All’ is displayed, by default.

    9. Item– Enter the stock item name or click [?] to select the item from the Item search dialog.

  3. For rejecting requests in scenarios, refer the below steps:

    •  To reject all the Issue Requests at a time, select the check box at the column header level.    

If the Reject Check box at the column header level is checked, all the checkboxes at document level (i.e. Issue Requests level) as well as the Line Item level (i.e Stock item level) are also checked.

    • To reject single/multiple Issue Requests, select the respective check box(es) of the Issue Requests at the Request Issue level.

    • To reject individual stock items within a single/multiple Issue Requests, click + to expand and select the check box of the stock item from multiple Issue Requests.

  1. After selecting the ‘Reject’ check box, the Remarks field becomes mandatory. Click ‘Remarks’ hyperlink to invoke a dialog where user can select the remarks for rejection from look up or enter the text manually in free text box with maximum 100 characters.

  2. Once it is saved, the request will be automatically closed and will not appear in Authorize at issue Store and in ‘Reject Request at Issue Store’ function.

User cannot reject the Requested items in document if they are authorized or issued.
