Sales/Patient Issues

From this function you can Record the details of the Items sold to a Patient and Query for the unauthorized Item Issue details.

Sales transactions are performed with respect to Items to be sold to a Patient. The Patient can be an ‘Inpatient’, ‘Daycare’, ‘Outpatient’, ‘Emergency’, ‘Referral Patient’ or a ‘Direct Sales Patient’. You can choose the store from which the sale/issue is performed and then search for the patient against whom the sale/issue needs to be done.  

Recording the details of Item sold to a Patient

From the ST Transactions > Sales, select the Sales/Patient Issues function.

The Sales/Patient Issues screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Modify Document Details screen is displayed by default or click Document hyperlink to modify the document details.

  3. On entering the document details, click [OK] to record the Item details sold to a Patient or [Reset] to clear the entered data.

  4. Enter the details in the mandatory (*) fields. <Related Field Description>

  5. Enter the Item to record the patient issue details in alphanumeric format of 60 characters maximum length or select the Item using [?] function. Displays the Item Search dialog. This is a mandatory field. If the item barcode is scanned, then the Item is automatically populated.

  6. Enter the search criteria and search for the item in a specific store from the Item Search dialog; displays the Item search results. Refer Item Search dialog.

Select the item only if the required quantity is available in the store. If the required quantity is unavailable or if the Sale quantity is greater than the available quantity, the application displays the available quantity of the chosen item and displays an error message. You can pick an item from the item search results if the Available Qty is greater than 0. The Sale/Issue will not proceed if adequate Available Qty is not available in the chosen store.

  1. On selecting the Item, the Unit of Measure, the available stock quantity and the Insurance billing details are displayed against the Item field.

  2. Enter the quantity of the item, which require to be sold to the selected patient in numeric format of six digits maximum length. The entered quantity must not be greater than the available quantity of the Item or an error message will be displayed. This is a mandatory field.

  3. On entering the Quantity, the Charge Amount and the Patient Payable Amount is displayed.

  4. Enter the Remarks in alphanumeric format of 200 characters maximum length.

  5. Click [Edit/View] to issue the first expiring batch in the slot. The Item Batch details are displayed with the Sale Quantity. The Quantity entered is displayed as the Sale Quantity.

  6. Click [Batch Search] to view the Batch details. The selected Item Batch details are displayed. Select the check box to select the Batch and click [Close] to exit from the batch search screen. If batch details are selected using [Edit/View], a message ‘Batch Data will be refreshed’ is displayed. Click [OK] to overwrite the existing batch details or [Cancel] to retain the selected batch.

  7. On selecting the batch, the batch details are displayed in the Batch Entry area.

    Enter the following details of the batch:

     

    1. Expiry Date – The Expiry Date of the batch is displayed. You can modify the date by entering the date in dd/mm/yyyy format or select the date using the Calendar function.

    2. Batch ID – The selected Batch ID is displayed. You can enter new Batch ID in numeric format as set in the ‘Batch ID’ label in Item function for expiry and non-expiry based Items.

    3. Trade Name – The selected Trade Name is displayed. You can select the required Trade Name from the list. The Trade Names associated to the selected Item in Materials Base module are displayed in the list.

    4. Bin Location – The selected Bin Location is displayed. You can select the required Bin Location from the list. The Bin Locations associated to the selected Item in ‘Item for Store’ function are displayed in the list.

    5. Available Qty – The Available Quantity of the Item is displayed.

    6. Sale Quantity - Enter the Sale Quantity in numeric format of five digits maximum length. The entered Sale quantity must be equal to Item quantity or an error message ‘Transaction quantity cannot be greater than available quantity’ is displayed.

  8. Click [Add] to add the Item for sale or [Cancel] to cancel the selected Item details.

  9. The selected Item details are displayed in the Item Details frame on the left hand side. Click ‘Item Details’ hyperlink to modify the Item issue details. Click [Modify] to modify the entered details or [Cancel] to cancel the modified changes.

  10. You can select the ‘Delete’ check box to delete the selected record. The Delete check box is selected or cleared by default based on the setup done in Kit Template screen.

  11. Click [Apply] to finalize the Item issue details or [Reset] to clear the entered data.

  12. The ‘Confirm Finalize’ dialog is displayed. Click [Yes] to finalize the Item issue details or [No] to finalize the Item issue details in the query mode.

  13. The ‘Confirm Printing’ dialog is displayed based on the attribute set for the Document Type. Click [Yes] to print the Item issue details or [No] to cancel the action.

  14. Click [Query] to invoke the Query screen.

  15. The unauthorized Item issue details are displayed based on the default criteria or enter the search criteria and click [Search] to select a specific record.

    Enter the following criteria to query unauthorized Stock Adjustment details:

     

    1. Doc Type – Select the Document Type from the list. The values listed are defined in the function ‘ST Document Type’ and the default value displayed is specified in the ‘ST Facility Parameter’ function.

    2. Doc No – Enter the Document number in numeric format of eight digits maximum length.

    3. Doc Date [From-To] - Enter the date in date format of dd/mm/yyyy or select the required date using the Calendar function.

    4. Store – Select the Store from the list. The Stores with access for ‘Sales’ transaction function is displayed.

    5. Item Class – Select the required Item Class value from the list.

    6. Priority - Select the required Priority from the list. The system-defined values are 'Routine', 'Urgent' and 'Both'. The value is selected as 'Both', by default. The sale documents are filtered based on the Priority selected.

    7. Patient ID – Enter the Patient ID in alphanumeric format as set in Master Patient Index module or select the Patient ID using [?] function.

    8. Nursing Unit/Clinic - Select a Nursing Unit or Clinic from the list. The Nursing Units/Clinics with unauthorized sale documents are listed.

  16. On searching, the unauthorized Item issue details are displayed.

  17. Click ‘Doc Type’ hyperlink to authorize the Item issue details.

  18. Click [Apply] to finalize the Item issue details or [Delete] to delete the selected record.

  19. On clicking [Apply], the ‘Confirm Finalize’ dialog is displayed. Click [Yes] to finalize the Item issue details or [No] to finalize the Item issue details in the query mode.

  20. On selecting the ‘Delete’ check box, ‘Confirm Delete’ dialog is displayed. Click [Yes] to delete the record or [No] to cancel the action.

 

The Minimum Stock Level or Reorder Level of the selected item is displayed next to the Item field while performing the transaction. Based on selection of Display Min Level & ROL check box in User Access for Store, this functionality will be enabled.
