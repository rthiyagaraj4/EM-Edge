Facility Parameter

From this function you can Define business logics or rules that influence the functioning of transactions in the Inventory Management module at the Facility Level.

The behavior of Code Setup function depend on the Facility Parameter values. The parameter values, if defined at the Facility Level overrides the values set at the entity level.

Only one record can be created for the facility. It is mandatory to set up the Facility Parameter before any operations are performed as this record stores important details about Inventory Management module. If Facility Parameter record exists, existing Parameter details are displayed. You can update these details.

Defining Facility Parameter

From the ST Administration, select the Facility Parameter function.

The Facility Parameter screen gets displayed.

  1. Enter the details in mandatory (*) fields. <Related Field Description>

  2. Click [Apply] to save the defined Facility Parameter rules or [Reset] to clear the entered data.

 

The ‘Current Processing Month/Year’ option is based on the Period End Scope defined in MM Parameter. If the Period End Scope is defined as Facility Level, enter the Current Processing Month/Year. This is a one-time setup. The Current Processing Year and Month gets updated in the ST Parameter whenever you run the Period End Process.
