User Access for Store

From this function you can Create and restrict the user’s access to the transactions of the defined Stores and Query for the desired ‘User Access for Store’ records.

User Access for Store function helps you in managing the Store and the applicable transactions that you can operate on within the Store. Users associated to a Store can perform the tasks that are defined while creating the Store. The access control fixes the boundary of operations for a User. The restrictions on the functions applicable for the store can be explicitly selected and cleared as required.

Defining User Access for Store

From the ST Administration, select the User Access for Store function.

The User Access for Store screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the User in alphanumeric format or select the User using [?] function. This is a mandatory field.

  3. Select the Store from the list. The stores defined in the Store function are displayed. This is a mandatory field.

  4. Click [Search] to display the applicable transaction functions of Inventory Management and select the check box against the applicable transactions. The check boxes are selected, by default based on the transaction functions applicable while defining the Store.

  5. Select the default item search criteria from the ‘Default Item Search By’ drop-down list. The listed values are Item Code, Item Name, Machine Code, Part Number, Trade ID, Trade Name and Alpha Code. If the default search criteria is selected as ‘Trade Name’, then the ‘Trade Name’ check box is selected by default in the Item Search dialog.

  6. Select the Default Sale Type value from the drop-down list. The listed values are Inpatient, Daycare, Emergency, Outpatient, Referral and Direct Sales. The Sale Type values are displayed based on the ST Store setup. The selected Sale Type is displayed in the Sales Return and Sales/Patient Issues screens.

  7. Select the Display Current Encounter Details check box to assign the rights to the user to view the Patient’s current encounter financial details.

Only if you select this check box, you can view the new $ symbol along with tool tip in the Register Order screen, Dispense Medication screen – (Verification, Allocation, Fill, Deliver, All Stages), Direct Dispensing, Sales/Patient Issues and Sales Return screen.

  1. Select the  applicable Transactions. The transactions displayed are Opening Stock, Vendor Return, Requisition issues, Issues Returns, Stock Adjustment, Product Transfer, Split Batches, Sales, Authorize Special Request, Process Expired Batch, Cancel Request, Goods Receipt, Requisition, Urgent Issue, Manufacturing Items, Stock Transfer, Variance, Change Bin, Sales Returns, Stock Item Consumption, Process Suspended Batch, Condemn Expired Batches, Display Min Level & ROL, Reject Request  at Issue Store, Cancel Authorization at Issue Store etc.

If the configuration ‘Process Suspended Batches’ is not set in the User Access for Store screen, these Suspended batches of items are not displayed in the batch searches of the transaction screens.

  1. Select Finalize/Authorize check box against the applicable transactions to authorize or finalize the transactions. You can set Finalize/Authorize rights for the transaction functions Vendor Return, Requisition Issues, Issue Returns, Stock Adjustment, Sales Goods Receipt, Requisition, Urgent Issue, Manufacturing Items, Stock Transfer and Sales Returns.

  2. Select Prompt for finalize check box against the applicable transactions if you want to explicitly finalize the transaction. The Confirm Finalize popup is displayed. Click Yes to finalize the transaction. If you clear the Prompt for finalize check box, the corresponding transaction is auto finalized without presenting the Confirm Finalize popup.

If finalization/authorization is not required for the transaction, then the Finalize/Authorize checkbox will not be available in the screen. Thereby, the Prompt for finalize checkbox will also not appear. The Prompt for finalize checkbox is enabled and selected, by default, if the Finalize/Authorize checkbox is selected for the transaction. Else, it is in cleared and disabled state .

  1. Select Acknowledge check box against the applicable transactions to Acknowledge the transactions. You can set acknowledgement rights for the transaction functions Requisition Issues, Issue Returns, Urgent Issue and Stock Transfer by selecting the check box.

  2. Select Confirm check box against the applicable transactions to confirm the transactions. You can set confirm rights for the transaction functions Requisition Issues, Issue Returns, Urgent Issue and Stock Transfer by selecting the check box.

  3. Select Delete check box against the applicable transactions to delete the transactions. You can set delete rights for the transaction functions Vendor Returns, Requisition Issues, Issue Returns, Stock Adjustment, Product Transfer, Sales, Goods Receipt, Requisition, Urgent Issue, Manufacturing Items, Stock Transfer and Sales Returns by selecting the check box.

  4. Select Cost Price and Sales Price for Stock Transfer transaction.

The check boxes are selected, enabled, or disabled based on the transaction functions applicable while defining the Store in the ST Code Setup. The restrictions on the functions applicable for the Store is explicitly selected or cleared as required.

  1. Click [Apply] to save the User Access rights created or [Reset] to clear the entered data.

  2. Click [Query] to search for existing User Access for Store records. All user privileges are grouped with the corresponding transactions.

The check boxes are selected, enabled or disabled based on the transaction functions applicable while defining the Store. The restrictions on the functions applicable for the Store is explicitly selected or cleared as required.
