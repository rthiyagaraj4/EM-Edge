Enter the following details to create a Store record:

  1. Store – Enter the required Store Code in alphanumeric format or click [?] function to select the required store. The enabled ‘Stores’ defined in Materials Base module are displayed. This is a mandatory field. When you select the store, the Store type and Store Group are populated automatically and the Store type field is disabled.

  2. Bin Location – Click [?] function to select the Bin Location associated to the selected owner store. The bin locations available in the Materials Base module are displayed.

  3. Department  - The department associated to the respective store gets displayed.

  4. External – Displays a Tick mark if it is an external resource.

  1. Store Type – Select the required Store Type from the list. The listed values are ‘Main’, ‘Sub-store’, ‘Pharmacy’ and ‘Non-Stock store’. The value ‘Main’ is displayed, by default. This is a mandatory field. When you select the store, this is populated automatically and is disabled.

  2. Store Group – Displays the group under which the store is classified. When you select the store, the Store Group is populated automatically.

  3. Parent Store - Select the required Parent Store from the list. The Stock Stores are displayed in the list.

  4. Reorder Days – Specify the frequency in which the Items are to be refilled from the Parent Store to the non-stock store in numeric format of three digits maximum length.

  5. Medical Item – Select the check box, if the selected store can access Medical Items.

  6. Non-Medical Item – Select the check box, if the selected store can access non-medical items.

  7. Consider Stock position for Reorder – Select the check box, if the stock position must be considered for reordering Items in the Store. These Stores are termed as re-orderable Stores.

  8. Auto calculate ROL, ROQ, Min & Max Stock Levels – Select the check box to automatically calculate the Minimum Stock Level, Maximum Stock, Reorder Quantity, and Reorder Level  in Item for Store screen. By default, the check box is not selected. This is applicable only to users in Medicity.

  9. Consumption Period (in days) – This field gets enabled only if the ‘Auto calculate ROL, ROQ, Min & Max Stock Levels’ check is selected in Store screen. You can enter upto two digits. Maximum Consumption period is 90 days.  Zero is not allowed. This is applicable only to users in Medicity.

  10. Alert Pickslip Print – Select the check box to display an alert before printing the pickslip in the issuing store or clear the check box to directly proceed with the printing without an alert.

  11. Inpatient Sale Applicable - Select the check box, if the sale is applicable for Inpatients. The check box is selected, by default.

  12. Outpatient Sale Applicable – Select the check box, if the sale is applicable for Outpatients. The check box is selected, by default.

  13. Referral Sale Applicable – Select the check box, if the sale is applicable for Referral patients. The check box is selected, by default.

  14. Day Care Sale Applicable – Select the check box, if sale is applicable for Day Care patients.

  15. Emergency Sale Applicable – Select the check box, if sale is applicable for Emergency patients.

  16. Direct Sale Applicable – Select the check box, if sale is applicable for Direct patients.

  17. Automatically associate new interface Items to Store – Select the check to automatically associate new interface items with the required Stores. By default, the checkbox is cleared. This is applicable only for users in WCH.

  18. Transactions – List of applicable transactions of Inventory Management module are displayed. The applicability of transactions is limited to the Store Type value selected as shown in table.

  19. Applicable – Select the check box to make the transaction applicable for the defined Store.

  20. Acknowledge - Check or uncheck the check box against the transaction in the Acknowledge column to enable or disable the acknowledgement at the Store level. This check box is selected by default
