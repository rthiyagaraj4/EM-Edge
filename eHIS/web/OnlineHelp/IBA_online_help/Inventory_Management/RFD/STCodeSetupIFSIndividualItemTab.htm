Individual Item

  1. Store – Select the required Store from the list. This is a mandatory field.

  2. Item - Enter the required Item in alphanumeric format of 60 characters maximum length or select the required Item using [?] function. The Items are displayed based on the Store setup. If the selected store allows both ‘Medical’ and ‘Non-Medical’ Item, the entire list of Items are displayed. If the selected store allows ‘Medical’ or ‘Non-Medical’ Item; Medical/Non Medical Items are displayed as set in the ‘Store’ function. This is a mandatory field.

  3. Stock Item – Select the check box, if the Item is a Stock Item in the Store or clear the check box to indicate the Item is a Non-stock Item. When the Items are defined as Stock item, the Store is expected to maintain the stock positions. When the Items are defined as Non-stock Items, the Items are treated as consumption Item. The corresponding department will be charged for the issue and the stock is not maintained for this Item.

  4. Min Stock – Enter the minimum stock of the Item that requires to be Stored in the Store in numeric format of 12 digits maximum length.

  5. Reorder Quantity - Enter the reorder quantity of the Item when the Item required to be reordered in the store in numeric format of 12 digits maximum length.

  6. RO Level - Enter the reorder level of the Item when the Item required to be reordered in the store in numeric format of 15 digits maximum length. The reorder level cannot be lesser than minimum stock level or an error message will be displayed.

  7. Max Stock – Enter the maximum stock of the Item that requires to be stored in the Store in numeric format of 12 digits maximum length.

  8. Bin Location – Select the required Bin Location from the list to specify the Master Bin Location. This field is enabled on selecting the ‘Stock Item’ check box. The selected bin Location is made default in the stock inflow transactions. The Bin Location defined for the selected Store in Materials Base module are displayed in the list.

  9. Lead Days – Enter the lead days for refilling the stock from the Parent Store in numeric format of four digits maximum length. This field is enabled on selecting the ‘Stock Item’ check box.

  10. Min Stock Days - Enter the minimum stock days within which the Items have to be stored in numeric format of four digits maximum length. This field is enabled on selecting the ‘Stock Item’ check box.

  11. Buffer Percentage - Enter the Buffer Percentage in numeric format. This field is enabled on selecting the ‘Stock Item’ check box.

  12. Req UOM – Select the default UOM for request from the list. The UOM associated to the selected Item and its equivalent UOM, as set in Application Masters module are displayed.

  13. Issue UOM - The UOM associated to the selected Item is displayed and disabled for Non-stock Items as the default UOM for issue.

  14. Reorder Freq Days – Enter the reorder frequency days in numeric format of three digits maximum length for reordering the Items in the Store.

  15. Req Generation Base – Select the basis for request generation from the list. The values listed are ‘By Stock Level’, ‘By Order’, ‘By Dispense’, ‘By Administration’ and ‘By Manual’. The system displays ‘By Stock Level’, by default. Items having Request Generation Base set to ‘By Manual’ are considered in the ‘Generate Request’ function.

  16. Sales/Patient Issues – Select the checkbox to allow item for Sales/Patient Issues transaction. By default, the check box is cleared and in disabled state until an item is selected.

  17. Internal Consumption - Select the checkbox to allow item for internal transaction. By default, the check box is cleared and in disabled state until an item is selected.

  18. Pre Allocation of Drugs Based on – This field is enabled only if ‘Pre Allocation of Drugs Based on’ checkbox is selected in PH Parameter for Facility screen. The definition done in ‘PH Parameter for Facility’ gets displayed in Item for Store screen. If you want to override the definition in PH Parameter for Facility for an item, use the field available in Item for Store screen. You can define different period for ‘Near Expiry’ or ‘Stock Level’ for the item as required.

  19. Prevent new Request if the Item is received in the past X Days - Enter the number of days to be considered to prevent new request if the item is received in the past X days. By default, the value will be blank. You can enter value from 1 to 999 days.  Based on the entered days, the system prompts an error message ‘New Request cannot be raised for the item now as per configuration. Stock has been received recently’ in Create/Review Request screen.
