Unposted Transactions

From this function you can Query on Transactions that have not been finalized, allowing you to identify blocked batches.  

Unposted Transactions are filtered based on transaction or document type, Store, Item or Period Range. This function is used to monitor un-posted transactions based on a time frame and use this query to avoid delays in posting transactions.

Querying Unposted Transactions

From the ST Queries, select the Unposted Transactions function.

The Unposted Transactions screen gets displayed.

  1. Click [Search] to display all the un-finalized transaction details or enter the search criteria and click [Search] to view the required un-finalized transaction details. Click [Reset] to clear the entered data.

    Enter the following search criteria to view un-posted transactions:

     

    1. Transaction - Select the required transaction for which a report is to be generated from the list. All the transactions of this module are listed in the list.

    2. Doc Type – Select the required document type from the list. All the documents associated to the selected transaction type are displayed in the list.

    3. Doc No – Enter the document number for which for which the unfinalized transaction details require to be viewed in numeric format of eight digits maximum length.

    4. Store - Enter the store code for which the unfinalized transaction required to be viewed in alphanumeric format of 15 characters maximum length or select the required Store using [?] function.

    5. Item - Enter the required Item in alphanumeric format of 60 characters maximum length or select the Item using [?] function.

    6. Batch ID – Enter the required Batch ID for which the query result has to be generated in alphanumeric format.

    7. Doc Date [From-To] – Enter the document ‘From’ and ‘To’ date for which the request status is to viewed.

  2. On searching, the un-finalized transaction details are displayed based on the entered criteria.
