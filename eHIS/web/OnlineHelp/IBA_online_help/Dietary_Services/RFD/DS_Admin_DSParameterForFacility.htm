Enter the following details to define the parameters:

 

  1. General Default Diet for Patients Without Orders – Select the check box, if default diet orders are applicable for patients for whom the diet orders are not placed. Clear the check box, if the default diet order is not applicable and order has to be placed for each patient using the Place Diet Order function.    

  2. Label Message – Enter the Label message to be displayed for default diet orders while printing Meal Tickets. Clear the check box, if the label message is not to be printed on the meal tickets.

  3. Multi Kitchen - Select the check box to indicate that the facility has multiple kitchens. Clear the check box to indicate that the facility has only one kitchen. If the check box is selected, the Kitchen Code field in this parameter setup is enabled and you can select a Kitchen from the list. This indicates that the selected kitchen has multiple kitchen areas associated to it. If the check box is cleared, the Kitchen Code field gets disabled. This indicates that the facility has only kitchen and no kitchen areas associated to the facility.

  4. Kitchen Code - Select the Kitchen Code from the list. The selected kitchen code is displayed during Report generation, by default. The values are defined using the  ‘Kitchen/Kitchen Area’ function. This field gets enabled, if the check box ‘Multi Kitchen’ is selected.

  5. Max Meal Plan Allowed Days – Enter the maximum number of days for which the Meal Plan is allowed in numeric format of two digits maximum length. This controls the number of days allowed for meal plan generation. You can generate Meal Plans using ‘Generate Meal Plan’ function for the number of days specified beginning from the current system date. You can generate a meal plan using the 'Generate Meal Plan' function for the number of days specified from the current date. An error message 'Date is outside the limit specified in Parameter' is displayed, if the number of days specified in 'Generate Meal Plan' function exceeds the days entered.

  6. Next Meal Plan No. – Enter the next meal plan number in numeric format of three digits maximum length. The values in this field are setup only once. The entered value is validated during Report generation.

  7. Accept diet type for each meal type - Select the check box to assign different diet types for each meal type for a patient while placing the diet orders in ‘Place Diet Order’ function in Transactions. Clear the check box to disable the selection of diet types for each meal type.

  8. Interfaced to Patient Billing – Select the check box to interface Dietary Services module to Patient Billing Manager module. Clear the check box, if the interface to Patient Billing Manager module is not required.  

  9. Maximum Days Allowed (Including Current Day) - Enter a single numeric character ranging from 1 to 9. By default, the value 9 is displayed in the text field.

  10. Attendant Orders Applicable – Select the check box to place diet orders for attendants accompanying the patients. You can place the diet order for attendants using the ‘Place Diet Order’ function, if the check box is selected. Clear the check box, if the attendants are not served in the facility.

  11. Applicability For Grace Period –Select the check box to display the Grace Period text field in the DS Parameter For Facility screen.

  12. Grace Period – Enter the grace period to place a diet order even if the patient is discharged or consultation for the patient is closed and the patient is deceased.

If you enters the grace period greater than 24 hrs, the system invokes an error message ‘Time should not exceed 24 Hours’.

 

  1. Cut-off Time to Acknowledge Frequency Based Items – Enter the cut-off time to acknowledge Frequency Based Special Food Items in numeric format of two digits maximum length. The cut-off time indicates the maximum time limit from the scheduled time to acknowledge the frequency based special food items. The scheduled time is recorded while placing an order for the patient in ‘Place Diet Order’ function.

  2. Patient Care Taker Label – Enter the label name in alphanumeric format upto 30 characters. Based on label name the care taker terminology is used across the Inpatient functions in the system.  

  3. Restrict Diet Order Date Range for Inpatients – Select the checkbox to allow configuring the date range within which Inpatient diet orders will be allowed in the system in Place Diet Order screen. By default this checkbox is not selected.

  4. Generate Meal Plan for patients with Nil By Mouth – Select the checkbox to configure whether or not patients with Nil By Mouth (NBM) will be considered while generating the meal plan, Diet Order Sheet and Special Food Meal Ticket reports. It is a one-time setup.

  5. Notification Message for Place Diet Order - Enter message to be displayed while placing irregular diet order in the system. The text box allows maximum 150 characters. By default, this text box will be blank. If no message is configured in this field, the system will allow you to continue placing diet order without displaying any notification message.

  6. Instruction in Label (Formula) – Enter the instruction in alphanumeric format up to 50 characters in length. The dietary orders ordered from Order Entry screen will be authorized by the dietitian or any privileged users. The dietary orders for which authorization is complete, a label will be printed by the kitchen staff. This is applicable only for users in ML MMOH sites.

  7. Maximum Diet Order Date Range Allowed for Inpatients (Including Current Day) – Enter the maximum number of days within which Inpatient diet orders will be allowed in the system. The text box will be enabled and mandatory only if the checkbox Restrict Diet Order Date Range for Inpatients is selected, otherwise this text box will be disabled and greyed. By default this textbox is blank. However, the previously entered value will be retained in the text box.

  8. Enable Multi Facility Meal Order for Outpatients / Emergency Patients – Select the check box to place a meal order for outpatients and Emergency Patients across facilities. If the check box is selected, you can place the meal order for outpatients across the facilities from the Place Meal Order for Outpatients screen.

  9. Enable Drug Food Interaction Message For IP Patient - Select the check box to prompt drug-food interaction alert in the ‘Place Diet Order’ and ‘Modify Meal Plan for a Patient’ screens for inpatients across facilities.  This is applicable only for normal Rx with Inpatient and Daycare patient class.

Serving Time and Late/Irregular Diet Orders Applicable for

  1. Inpatient – Select the checkbox to enable the functionality of placing Irregular diet orders for Inpatients in the system. This checkbox is enabled only if the checkbox ‘Accept diet type for each meal type’ is selected; otherwise it will be disabled and greyed out.

  2. Outpatients – Select the checkbox to enable the functionality of placing Irregular diet orders for Outpatients in the system. This checkbox is enabled only if the checkbox ‘Accept diet type for each meal type’ is selected; otherwise it will be disabled and greyed out.

  3. Emergency Patients – Select the checkbox to enable the functionality of placing Irregular diet orders for Emergency Patients in the system. This checkbox is enabled only if the checkbox ‘Accept diet type for each meal type’ is selected; otherwise it will be disabled and greyed out.

  4. Staff Orders – Select the checkbox to enable the functionality of placing Irregular diet orders for Staff in the system. This checkbox is enabled only if the checkbox ‘Accept diet type for each meal type’ is selected; otherwise it will be disabled and greyed out.

  5. Supplementary Order - Select the checkbox to enable the functionality of placing Irregular Supplementary diet orders in the system. This checkbox is enabled only if the checkbox ‘Accept diet type for each meal type’ is selected; otherwise it will be disabled and greyed out.
