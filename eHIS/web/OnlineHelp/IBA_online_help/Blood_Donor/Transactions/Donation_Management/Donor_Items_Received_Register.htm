Donor Items Received Register

From this function you can Record the receipt of the mandatory items used and the blood units received from the mobile camp.

The Donor Items Received Register function allows you to record the receipt of the details of the mandatory items and the blood units received from the mobile camp. The drive date and mobile code recorded using this function is displayed automatically in ‘Pre Donation’ function. If blood donation is made in a mobile camp, the items used and which are marked as mandatory in ‘Devices’ function are to be returned back along with the donated blood units. The system alerts you, if any mandatory items are missing.

Registering the received donor items

From the Transactions > Donation Management, double-click the Donor Items Received Register function.

The Donor Items Received Register screen gets displayed.

  1. Click [List of Values] to select the Mobile Code. The values defined using the Mobile Units function get listed. This is a mandatory field.

  2. Click the Calendar function to select the drive date. The drive date indicates when the mobile drive camp is conducted.

  3. Click the Calendar function to select the date when the receipt is generated on receiving the items from the mobile drive camp.

  4. Click [List of Values] to select the Batch Number from the list of values. The listed values are created in Create Batch function.

  5. Enter the details in the mandatory fields in the following tabs:

Drive Items

Units by Drive Item

  1. Click [Confirm] to confirm that the specified mandatory items are received from the mobile unit. Click [Clear Record] to clear the entered data.
