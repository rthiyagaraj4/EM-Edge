Enter the following details:

  1. Product Code/Description – Enter a unique Product code in alphanumeric format of 10 characters maximum length. Enter the Description for the Product code in alphanumeric format of 40 characters maximum length in the adjacent field. This is a mandatory field.

  2. Short Description – Enter the short description of the Product code in alphanumeric format of 15 characters maximum length. This is a mandatory field.

  3. Category – Click [List of Values] to select the Product category code. The values defined using the 'Product Categories' function get listed. This is a mandatory field.

  4. Blood Group Confirmation Required? – Select the check box, if the Product requires blood screening. Clear the check box, if blood screening is not mandatory.

  5. Pooled Product? – Select the check box, if the Product requires pooling. Clear the check box, if the Product does not require pooling.

  6. Expiry Period – Enter the Expiry Period of the Product in numeric format of three digits maximum length. The entered value has to be a valid non-zero positive integer. This is a mandatory field.

  7. Units – Select the unit from the list to indicate the expiry period of the Product. This is a mandatory field.

  8. Max Vol – Enter the Maximum Volume of the particular product that can be collected at a time from a donor in alphanumeric format of five characters maximum length. This is a mandatory field.

  9. Units – Enter the corresponding Units for the maximum volume specified. This is a mandatory field.

  10. Stock Units – Click [List of Values] to select the required Stock Units of the Product or enter a valid Stock Unit. The values defined using the Unit of Measurements function get listed. This is a mandatory field.

  11. Generate Unit Number – Select the check box to generate the Unit Number automatically based on the product code. Clear the check box, if the unit number has to be entered manually for each Product Code.

Note: The system prompts you to enter the Unit Number, if setup is not defined to generate unit number automatically in ‘BD Parameter’, ‘Collection Centre/Supplier’ or ‘Products’ functions. This rule takes precedence in checking the setup. First, the system checks if the set up is defined in Collection Centre/Supplier, then in Products and finally in BD Parameters.

  1. Length – Enter the Length of the unit number that you want to generate in the numerical format of two digits maximum length. The specified length of the unit number must not exceed 20. If you use the ISBT, standards, the maximum number you can use is 13. You have to define it in the BD Parameters.

  2. Prefix – Enter the Prefix of the unit number in alphanumeric format of six characters maximum length.

  3. Next Number – The Next Number field is disabled. On creation of the new unit number, the next number is created automatically.

  4. Number of Labels – Enter the Number of Labels required for the Product in numeric format of two digits maximum length.

  5. Specimen Type – Enter a valid specimen type or click [List of Values] to select the required Specimen Type. The values defined using the ‘Specimen Type’ function get displayed.

  6. Cross Match Required? – Select the check box to specify whether Crossmatch is required. The value for this field depends on the specimen type selected. If ‘Cross Match Required’ check box is cleared for the selected specimen, you cannot modify it. If ‘Cross Match Required’ check box is selected for the selected specimen, you can modify it and specify other specimen type under ‘Spec Type’ tab. The ‘Spec. Type’ tab is enabled only if the check box is selected.

  7. Max Time out of Blood Bank – Enter the maximum time allowed for the Product to be out of the blood bank before transfusion. It must not exceed the specified expiry period. This is a mandatory field.

  8. Units – Select the required units for the maximum time allowed out of the blood bank before the transfusion from the list. The valid values are ‘Minutes’, ‘Hours’ and ‘Day’. This is a mandatory field.

  9. Processing of Units at issue time? – Select the check box to process the Product during issue. If this check box is selected, while transfusing the blood, you have to enter the process code. If you clear the check box, you need not enter the process codes during the transfusion of blood units. The process codes are created in Process Codes function in Blood Transfusion module.

  10. Refund Bill on Return? – Select the check box to enable refund of the bill on returning the product. Clear the check box, if the bill amount cannot be refunded on returning the product.

Donation

  1. Product Storage Location – Enter a valid Product Storage Location code or click [List of Values] to select the required product storage location code and corresponding description. The values defined using the function ‘Product Storage Location’ gets displayed. This is a mandatory field.

  2. Reject Unit Storage Location – Enter a valid Reject Unit Storage Location or click [List of Values] to select the required reject unit storage location code and the description. This is a mandatory field.

  3. Donated Product? – Select ‘Yes’ from the list if the respective product is a donated product. Select ‘No’ from the list if the product is not a donated product. The value is set to ‘No’, by default.

  4. Apheresis Product? - Select ‘Yes’ from the list if the respective product is an apheresis product or ‘No’, if the product is not an apheresis product. The value is set to ‘No’, by default.

  5. Storage Instruction Code – Enter a valid Storage Instruction Code or click [List of Values] to select the required storage instruction code. The description for the respective Storage Instruction code gets displayed. The values defined using the function ‘Storage Instructions’ get listed.

Blood Type/Stock Levels

  1. Blood Group – Enter the Blood Group for the Product or click [List of Values] to select the required Blood Group.

  2. Rhesus Code - Enter the Rhesus Code for the selected blood group or click [List of Values] to select the required Rhesus Code.

  3. Minimum Level – Enter the value for minimum stock level of the Product in numerical format of five digits maximum length. This indicates the minimum level of stock for the particular Product and the stock level cannot be lesser than the number specified.

  4. Maximum Level - Enter the value for Maximum Stock Level of the Product in numerical format of five digits maximum length. This indicates the maximum level of stock for the particular Product and the stock level cannot exceed the number specified.

  5. Reorder Level - Enter the value for Reorder Stock Level of the Product in numerical format of five digits maximum length. When the stock reaches the entered value, an alert message gets popped up, alerting the user to reorder the particular product. You have to recall the donors or select new donors to donate the blood again.

  6. Current Level – The Current Stock Level of the product is displayed and you update the value.

Order

The transactions in this tab are done in the Lab centre.

Spec Type

  1. Type – Click [List of Values] to select the desired Specimen Type. The values defined using the ‘Specimen Type’ function get displayed.

  2. Description – The corresponding Description of the selected Specimen Type is displayed.

Spec Type tab is enabled only if the ‘Cross Match Required?’ check box is selected.

Barcode

  1. Product Barcode Value – Enter the Product Barcode Value in alphanumeric format of 20 characters maximum length.

  2. Barcode Message1 – Enter the Barcode Message that needs to be associated with the code.

  3. Barcode Message2 – Enter the additional Barcode Messages, if any.
