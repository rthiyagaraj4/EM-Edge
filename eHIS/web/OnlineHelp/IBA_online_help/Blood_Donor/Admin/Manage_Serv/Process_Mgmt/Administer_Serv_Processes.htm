Administer Server Processes

From this function you can Start and Stop the predefined Server processes. You can Maintain/Regenerate the report server error logs and View all the errors from the report server.

The Administer Server Processes function starts/stops the predefined Server processes. Each process is started as a single instance depending on the design of the process. It maintains the Report Server Error Logs and shows all the errors from the report server. The details displayed in the function cannot be modified. You can regenerate the report request and purge the report errors. You have to start Print Service first and the other services after that. While stopping you have to stop all the services before you stop the Print Service.

Administering Server Processes

From the Administration > Manage Services > Process Management, double-click Administer Server Processes function.

The Administer Server Processes screen gets displayed with the Process and the Administer tabs enabled.

  1. The Processes, which are to be executed gets displayed. <Related Field Description>

  2. Click [Start] to start the process. If you select the value of Priority as ‘*ALL’ you cannot start the process for an individual facility.

  3. Click [Stop] to stop the process. You can stop the services only in the workstation where the services started.

  4. Select the Active tab from the screen to view the details about the process execution using the Administer tab <Related Field Description>

  5. Select the Logs tab from the screen to view the details. <Related Field Description>

  6. Select the report generation request record and click [View Error Log]. The system displays a pop-up message window with all the error messages information for the selected record.

  7. Select a report generation request record and click [Regenerate] to delete the corresponding error messages and make the report generation request as active request for regeneration.

  8. Select a report generation request record and click [Purge] to delete the error log after the confirmation message gets acknowledged.

  9. Click [Purge All] to delete all the queried report generation requests after the confirmation message is acknowledged.
