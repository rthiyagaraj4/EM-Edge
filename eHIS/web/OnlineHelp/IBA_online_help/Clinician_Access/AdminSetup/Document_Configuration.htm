Document Configuration

From this function you can Define whether to store the clinical note related files in Server Folder or EM Database.

The Document Configuration function allows the Administrators to choose either the Server Folder or EM Database to store the clinical note related files. The documents are uploaded using the Upload button functionality in the Clinical notes. The shared server path must be ‘Read-Write’ accessible to the EM Server.

This setup allows to store the below clinical note related files in the Database or in the External File Server.

• Files uploaded to the CA Clinical Note

You can switch the configuration for the Storage Location of any of the above four options at any time from Database to the External File Server and Vice-Versa.

Defining Document Configuration

From the CA Admin Setup > CA Parameter Setup > select the Document Configuration function.  

The Document Configuration screen gets displayed.

  1. Click [Query] to invoke the Query screen.

  2. Select the facility for which the configuration must be created from the Facility drop-down list. This is a mandatory field.

  3. Click [Search]. The Note Content Option and Storage Option Type gets displayed.

  4. Note Content Option – The option available - Uploaded Files  

  1. Storage Option Type - Select the required Storage Option Type where you want to store the uploaded files. The options are 'Database (Existing)' and 'Shared Path'. By default, the Database (Existing) option is selected.

• Database (Existing) option - By default, the Database (Existing) option is selected. The system continues to store the uploaded documents into the EM Database.  

• Shared Path option -  Select this option if you want to store the data in shared path. Once the Shared Path option is chosen, it is mandatory to enter the shared path in the text field.

The entered External File Server shared path must follow the below properties:  

• The configured shared server path must be 'Read-Write' accessible to the EM Server. Else an error message 'Shared path is either incorrect or is not Read-Write accessible' is displayed. The same error gets displayed even if the path is incorrect.

• Valid Shared Paths must always begin with a "\\" and contain only \, _ and - special characters besides alphabets and numbers.

  1. Click [Apply] to save the changes made or click [Reset] to clear the entered data.

The system will read a file from the same folder into which the file is uploaded based on the document configuration. The files must be readable from the respective source where they are stored even if the storage option is changed.
