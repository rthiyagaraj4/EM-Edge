Restricting Clinical Event History Data

From this function you can Define the setup to restrict Clinical Event History data.

Using this function, you can define the setup to restrict the clinical event history data. The restriction is applicable to print, comment and manage the clinical event history data.

Defining Clinical Event History Data Restriction

From the OA Transactions, select the Restricting Clinical Event History Data function.

The Restricting Clinical Event History Data screen gets displayed.

  1. By default, the ‘History Type’ option is selected against the Restriction Applicable to Print/Comments/Manage field.

  2. Select the required history record types check box to apply restriction for the clinical event history data.

  3. You can change the Restriction Applicable to Print/Comments/Manage field option as ‘Event Class Type’ if required.

  4. On selecting the ‘Event Class Type’ option, the History Type drop-down list appears.

  5. Select the required history type from the drop-down list and the event class type associated with the particular history type gets displayed.

  6. Select the required Event Class Type check box.

  7. Click [Apply] to save the settings done or [Reset] to clear the entered data.
