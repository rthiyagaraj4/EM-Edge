Coder Function

From this function you can Perform medical / billing coding activities for Insurance Billing Group patients.

This function allows you to perform medical / billing coding activities. This function is applicable only for Insurance Billing Group patients who are discharged or checked out. You can do the following:.

• Add / Modify the diagnosis recorded by the physician

• Add / Remove charges for the patient

• Generate Insurance bill

• The system will not consider the diagnosis support codes in the Coding function.

 

When you try to access the function for a Non-insurance patient or a patient with active encounter (not discharged/checked out), then the message 'This function is applicable only for Insurance Patients who are Discharged/Checked out' gets displayed.

Perform Coding Process

From the CA Transactions > Clinician Access > Desktop Menu > select required patient > Patient Chart Menu > Coder Function.

The Coder Function gets displayed.

  1. Enter the details in the mandatory (*) fields. <Related Field Description>

  2. Click [Search], the record matching the search criteria appear.

  3. Click the Patient ID, the Coder Function window gets displayed.

  4. The Patient Demographics and Encounter Details of the patient are displayed.

Diagnosis Details

  1. In the Diagnosis Classification section, the recorded diagnosis ICD codes from the Record Diagnosis function is defaulted in the Diagnosis Set, Diagnosis Classification, Code/Description, POA Indicator, Onset Date fields. You can change the values if required.

  2. Add/Delete - You can add new code in the respective fields and it will be checked by default. While saving the diagnosis, the system will consider all the checked codes and then it will save. You can add up to 25 diagnosis classification in addition to defaulted diagnosis.

  3. If you know the ICD code to be added, you can type the same and on tab out description gets defaulted. If you type the code partially and on tab out, the look up gets opened and the you can select the necessary ICD codes.

  4. There can be only one primary diagnosis. If you select one or more primary, an alert will be displayed.  

  5. When you modify the defaulted ICD code, the system will update the modified code and save. The system will not consider it as a new code, and it will not add in a new row and save.

  6. After entering the details, click [Save].

  7. Once records are saved in the Coder function, the system will not default the record diagnosis code again even if it is changed or updated by any user in the Record Diagnosis function.

  8. Similarly, whatever code has been added / modified / deleted in the Coder screen, will not be reflected in Record Diagnosis function for the respective patient encounter.

Services

  1. Displays maximum 40 records from billing folio, and the records will be non-editable.

  2. CPT Code – Enter the required CPT code to search for a particular record and it will be highlighted.

  3. Click [Details /Edit] button to invoke a separate window to display all the services available for the patient and editable based on the setup. You can Add / Modify / Cancel the services and click [Save].

  4. Click [Generate Bill] to generate the bill.

Financial Status

  1. Displays the Patient Share, Patient Paid, Outstanding, Gross Amount, Net Claim Amount.

  2. Patient Share - Displays the sum of total patient price amount (Net amount), which patient should pay. This is non-editable field.

  3. Patient Paid- Displays the amount already paid by patient against this bill. This is non-editable field

  4. Outstanding - Displays the is the outstanding amount; patient should pay against the bill. This is non-editable field.

  5. Gross Amount: Total sum of all services (Patient price + Insurance Price) in the bill. This is non-editable field.

  6. Net Claim Amount: The amount user was claiming to insurance company against patient bill. This amount is calculated after all deductions and Co-payments.

  7. Click the [Encounter-Generated bills] button to generate the bills for the encounter. The Authorisation dialog gets displayed where you need to enter pin, user name and remarks..

  1. Completion Status – By default, the Status of the record is ‘New’. Based on the status selected in the Coder Function, the coding status is updated.

  2. Remarks – Enter the remarks in the text field.  You can enter maximum of 4000 characters. There is no audit provided for the remarks and can be overridden.

  3. Doctor Query – Select the Doctor Query check box if the coder wants the Attending Practitioner of the patient to review and save the record. If the Doctor Query check box is selected, then the record will be available under ‘Pending Doctor Documentation’ link in Summary menu.

  4. Supervisor Query – Select the check box if the coder wants the Supervisor to review and save the record. Once the record is saved with Supervisor Query selected, then the record is available under ‘Supervisor Pending Documentation’ link in Summary menu.

  5. Hold – Select the checkbox if you want to put the record on hold. The Coder must review the Coder screen again. You cannot perform any action in ‘Diagnosis’ or ‘Services’ section until the Hold checkbox is selected.

  6. Partially Completed – Select the check box if the Coder function is partially completed.

  7. Complete – Select the check box to complete the coder function. If the ‘Completion’ checkbox is selected and saved, the coder will not be allowed to make further changes. The Patient will be available in Search Screen under Completed Status and only Preview option will be enabled.

If you select the 2nd check box, the first check box gets cleared.

  • If any of the Doctor query / Supervisor query / Hold checkbox is selected, the system should not allow to generate the Partial bill / Final bill.

  • If Doctor query / Supervisor query / Hold checkbox is selected, when you click on generate bill, system displays an alert message ‘Bill cannot be generated. Either Supervisor Query / Doctor Query / Hold check box is ticked’.

  1. Supervisor Clearance Completed – Select the check box to generate the clearance. Only users for whom Supervisor check box is selected in Practitioner setup (AM module) will be allowed to provide the clearance. Once after the clearance is provided, the check box must be selected.

• If the bill is not generated and user provides clearance by mistake before generation of bill, then user can revoke the clearance by clearing the ‘Supervisor Clearance Completed’ check box.

• If bill has been generated and supervisor clearance is provided for incorrect patient, then system will allow the user to cancel the clearance only when the bill is cancelled. Only after bill is cancelled, then the ‘Supervisor Clearance’ check box will get enabled so that user can clear the check box to cancel the clearance provided.

• If the bill is generated and if Coding Supervisor forgets to select the ‘Supervisor Clearance completed’ check box, it will be in disabled mode. If user want to select the ‘Supervisor Clearance Completed’ check box, then patient bill must be cancelled in order to select the ‘Supervisor Clearance Completed’ check  box.

Cancel Supervisor Clearance can only be done by the supervisor.

  1. Click [Preview] to preview the coder details.

  2. Click [Save] to save the remarks and check box selection.

  3. Author Name: Displays the user name who selected the ‘Completed check box which is the indication of the insurance coder who has finished this visit coding and corresponding Date and Time of completion.
