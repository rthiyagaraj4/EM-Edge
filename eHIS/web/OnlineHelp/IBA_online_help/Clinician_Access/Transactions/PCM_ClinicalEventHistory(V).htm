Clinical Event History (V)

From this function you can View the clinical event history recorded for a patient.

The Clinical Event consists of past clinical data across encounters recorded for the patient and provides the facility to search by previous documents recorded. This function allows a practitioner to access past clinical documents and print them for reference. This function allows the facility to search down by event or date and across a period of time.

You can  restrict access to print, manage and comment clinical history data in Flowsheet and Tree view mode based on the parameter setup done in ‘Restricting Clinical Event History Data’ function.

Viewing Clinical Event History

From the CA Transactions > Clinician Access > Desktop Menu > select required patient > Patient Chart Menu > Chart History, select Clinical Event History (V).

The View Clinical Event History screen gets displayed.

  1. By default, the system displays clinical events within the displayed date range.

  2. Enter the search criteria and click [Search] to view specific clinical events.

  3. The Clinical Events recorded for the patient are displayed on the left hand side of the screen on searching based on the entered criteria without selecting the Flow Sheet View check box. The Symbolic Legends or Colour Indicators as set in the 'Clinical Event Parameter' function of CA Admin Setup is displayed on the screen. Click  to expand the Clinical Notes and to collapse the Clinical Notes.

  4. Click Patient Encounter ID hyperlink under History section in the left hand side. The Visit Details are displayed for outpatients while the Admission and Stay Details are displayed for inpatients.

  5. Select the required Clinical Event to display the details of the Event under the Event Group on the right hand side. The various dates of recording the Event are displayed. On placing the cursor over the date of any Event, details of the Event such as Performed By, Encounter Details, Visit/Admission Date/Time and Attending Physician are displayed at the bottom of the screen.

  6. Place the cursor over the Symbolic Legend or Colour Indicator to display the order results by providing view of Normal Range, Critical Range and Performed By details.

  7. Click icon to view the event details in an Event Details dialog.

• You can print the event details by clicking [Print].

• If the images are linked to the note, an hyperlink 'View' is displayed. Click 'View' to view the images that are associated to the note. Click other icons to display the details of the selected Clinical Event.  You can print the event details by clicking [Print]. An Authorize Pin Form dialog gets displayed if ‘Confidential Pin Required for Printing’ check box is selected in CA Parameter screen. Enter the pin you have defined in the User screen in System Manager module and click [OK] to print the event details.

• You can view if clinical notes in the Event History has been forwarded or modified by another practitioner even if the encounter is closed by clicking Audit Trail button/hyperlink.

  1. Click the icon  to view the Neonatal Mother specimen and antibody test details for the neonatal newborn related orders if 2T LIS module is used. This icon is visible only for the neonatal Newborns for laboratory Order category with Order type as Blood bank related order catalogs. The neonatal maximum and minimum age group to be considered while displaying the icon is defined in ‘Age Group’ setup in Application masters, This icon appears only for users in ML-MMOH sites.

  1. Click [Zoom] to zoom the view of details of the Clinical Event. Click X at the top right corner of the screen to exit from the screen.

  2. To copy the clinical event details, select the text and copy it by clicking [Copy Text] or using ‘Ctrl+C’ option. Paste the copied text any where in EM using ‘Ctrl+V’ option.

  3. Click [Print] to print the Clinical Event History recordings. An Authorize Pin Form dialog gets displayed if ‘Confidential Pin Required for Printing’ check box is selected in CA Parameter screen. Enter the pin you have defined and click [OK] to print the clinical details of the patient

  4. Select the event and click [Email] to email the Clinical Event History recordings.

  5. Select the event and click [Manage] to manage the clinical events.

  6. Click [Comments] to enter comments and view existing comments recorded either by the performing practitioner or any other reviewing practitioner. The comments screen is displayed to view or record comments. Click [Record] to save the comments entered, [Clear] to clear the entered data or [Cancel] to exit from the comments screen. On recording comments, an icon is displayed, which is clicked to view or record more comments.

    The details of the existing comments are displayed like Event Date, Event name, Comment Created By, Comment Created Date and Remarks.

    Add New Comments

    1. Event – The name of the event is displayed.

    2. Value – This field is valid only for the Events having numeric values like lab results. The numeric value of the Event is displayed.

    3. Remarks – Enter the Remarks. This field is mandatory.

  7. Click [View] to invoke the ‘View Clinical Documents’ screen.

    Enter the following search criteria to view the consolidated list of notes for an encounter:

    1. Period [From-To] - Enter the date range in the format dd/mm/yyyy to search for clinical events within the displayed date range or select the period using the Calendar function. The first encounter date of the selected patient is displayed, by default.

    2. Include No of Previous Notes - Enter the number of previous notes to view in the range 1-9.

    3. Recorded Practitioner Type - Select the Practitioner Type from the list. All the system defined practitioner types are displayed.

    4. Recorded Practitioner – Select the practitioner who has recorded the clinical notes using [?] function.

    5. Encounters – Based on the selected period, the encounters are listed. Select the encounter from the list. The recent encounter is displayed, by default if there is no active encounter.

    Applicable Task

    1. Select the Applicable Task check box to search the consolidated notes based on the Applicable Task. The Applicable Tasks are: Record Clinical Notes, Record Discharge Summary and Referral Letter. The tasks Record Clinical Notes and Record Discharge Summary are selected, by default.

    Click [Show] to display the system generated consolidated notes for an encounter or [Clear] to clear the data.

     

    • The details displayed are entered during recording the clinical notes in an encounter. The Patient ID, Patient Name, Gender, Age, Encounter ID, Encounter Type, Admission Date, Discharge Date, Location, Specialty and Attending Practitioner.

    • Click  icon to go to the next page,  icon to go to the previous page,  icon to go to the first page of the consolidated clinical notes page,  icon to go to the last page.

    At-least one applicable task should be selected.

  8. Select Flow Sheet View check box and search to view Clinical Events in the form of flow sheet.

    The flow sheet gives the chronological view of events that the patient had undergone in a particular encounter or historically all the events the patient had in the facility. The flow sheet in Clinician Access module is not restricted to a specific event. The flow sheet gives a view of the documents, notes and events that were recorded based on date and time.

    The following details are displayed in the flow sheet view.

    1. Event – The name of the Event is displayed. Events are broadly grouped under Event Types.

    2. Unit – This field is valid for the Numeric values like Vital Signs, Lab Results, etc.

    3. Graph – Graph is valid for the Numeric values like Vital Signs, Lab Results, etc. The values in the flow sheet, which have to be analyzed, can be represented in the form of a graph. Select the check box displayed adjacent to the numerical value to be represented graphically and click on [View Graph] represents the values chosen graphically. Select the check box against the required components and click [New Group] to create a new group.

    4. The values entered for the particular parameter across the dates can be compared in the graph. Choose more than one parameter at a time to be shown in the form of a graph.

    5. Date and Time – The dates and time for which Clinical Events are recorded, is displayed. The values are displayed with Symbolic Legends or Colour Indicators as defined in the Clinical Event Parameter function.  Place the mouse over the Symbolic Legends or Colour Indictors to view Normal Range, Critical Range and Performed By details. Click the required icons to view details of the recorded Clinical Event.

Lab results received from external labs are uploaded using attach image function in the result entry function in labs. You can view the uploaded image by clicking the View Image hyperlink.
