Enter the following details to define CA Parameter:

 

  1. Confidential Pin Required for Printing – Select the check box if you want to confirm PIN to print the clinical details of the patient even if they have access rights for the selected patient. By default, the check box is cleared. If you select the check box, an ‘Authorize Pin Form’ dialog gets invoked on clicking Print button while trying to print the clinical details of the patient from Existing Orders, Order Tracking and View Clinical Event History screen.  This dialog appears even though you have confidential Rights.

  2. Auto Refresh for summary menu (time stamp) – Enter the required time for auto refresh of summary menu as per preference of the facility. The default time is 10 minutes. You can define the time limit from 10 to 99 minutes.

  3. Pending Discharge Summary Grace Period [X] Days -  Enter the grace period i.e., the number of days after the patient is discharged wherein the doctor has not recorded the discharge summary and signed. The patient record gets displayed under the ‘Pending Discharge Summary Patient List’ with count under the Summary menu in Practitioner’s Desktop. Enter numeric digits from 1 to 9. This field is mandatory.

  4. Pop up required to view patient financial details in viewing chart summary of selected patient – Select the check box to display the Financial Details dialog on clicking View Patient Chart Summary or refreshing that page. This dialog displays the outstanding or unbilled amount of the patient across encounters. This field is available only for users in MOHML sites.

  5. Clinical Event History Default View -  Select either the ‘Flow Sheet’ or ‘Tree View’ option. By default, the Tree View is selected. Based on the option selected, the Default View of the Clinical Event History screen gets displayed.

  6. Auto Refresh Time for Self-Notes in Summary Menu - Enter the required time for auto refresh of ‘Self Note’ screen under Summary menu as per preference of the facility. You can define the time limit from 1 to 99 minutes. The field can be left blank and it is not mandatory. This field is applicable only for users in MMS.

  7. Default period to be considered while fetching patient encounter details in CDR – Enter the default number of days to be considered to fetch the patient encounter details in CDR link. The default number of days is 30. This field is applicable only for users in MMS and Medicity.

  8. Third party vitals data required to acknowledge – By default, the check box is not selected. Select the check box so that the interfaced vital sign data will be pushed to temporary table of CA wherein you can access the data from Task list to acknowledge and delete it accordingly. This is applicable only for users in MMS, AAKH, and KAUH.

If the check box is not selected and when the interfaced vital sign data is pushed from Interface, the data will be moved to permanent tables of CA where you can view this data directly in View Chart Summary and View Clinical Event History screens.

  1. Patient Class - All Patient Class check boxes (IP, OP, Emergency, Day care) get selected by default when the ‘Third party Vitals Required to Acknowledge’ check box is selected. You can select or clear whichever patient class type is applicable while acknowledging third-party vitals. The 'Patient Class' check box is disabled when the ‘Third party Vitals Required to Acknowledge’ check box is not selected.

  2. Third Party Vitals Data Representation – This field gets displayed only when the ‘Third party vitals data required to acknowledge’ check box is selected. Select either Table View or Graph View to view the third party vitals data in table format or graphical view in the Task List screen. By default, the Table View option is selected.

Date/Time wise View – Select this option to sort the vital signs in descending order of data/time. Sequence of components to be displayed as – SBP, DBP, Temp, SPO2, Heart rate as this sequence will be system-defined and not user-defined.

  1. Mapping Existing Chart ID Applicable to Third Party Vitals Data - Select the existing chart ID from the lookup to map with third party vitals data. Thereby the third party interfaced vitals data can be viewed accordingly. This field is applicable only for users in MMS, AAKH and KAUH.

  1. Hide Bed No Search Option for Inpatients – Select the check box if you want to hide the Bed No field in Patient List By Location screen.

  2. Chief Complaint Fields to be mandatory –

• Complaint Details check box – Select the check box to make Complaint Details field appear as mandatory in Record Chief Complaints function. By default, both the check box will be cleared.

• Remarks – Select the check box to make Remarks field appear as mandatory in Record Chief Complaints function. By default, the check boxes will be cleared.

  1. Control to view and Print lab result reports from New Clinical Event History - Select either Specimen wise or individual option to view and print lab result reports in the New Clinical Event History screen.

• If you select Specimen wise option, the lab results get printed specimen wise.

• If you select the Individual Test option, the lab results get printed for individual test that is part of Specimen number.
