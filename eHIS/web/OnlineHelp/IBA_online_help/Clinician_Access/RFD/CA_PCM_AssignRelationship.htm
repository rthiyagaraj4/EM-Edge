Enter the following details:

 

  1. Limit Practitioner List to Access Location – Select the check box to indicate that the list of practitioner names listed in the ‘Practitioner’ field is restricted to the practitioners that have access to the location of the patient. The location access is defined in ‘Location for Practitioner’ master.

  2. Practitioner Type – Select the Practitioner Type that needs to be assigned the relationship. The list has values as defined in the ‘Practitioner Type’ master in Application Masters module.

  3. Practitioner – Select the Practitioner Name for which the relationship has to be assigned. The practitioner names correspond to the Practitioner Type chosen. Further the practitioner name is listed based on the location access, if the ‘Limit Practitioner List to Access Location’ check box is selected. In such a case, the names of all the practitioners that have access to the location in which the patient is currently hosted are displayed. If the ‘Limit Practitioner List to Access Location’ check box is not selected, the names of all the practitioners belonging to the selected Practitioner Type are listed.

  4. Relationship – Select the Relationship from the list. The relationships are defined in the ‘Relationship’ function but the values listed in the list are filtered based on the relationships associated with the logged in responsibility. This is defined in the ‘Responsibility for Relationship’ function. This field is mandatory.

The relationships that a logged in user can assign are based on the responsibility using which the practitioner has logged in to Clinician’s Access. This is defined in the master ‘Relationship for Responsibility’. The practitioner can assign only those relationships that have the ‘Can Assign’ flag selected against the responsibility that the practitioner uses to log in to Clinician’s Access. The system also validates for the practitioner to whom the relationship is being assigned. If the practitioner to whom the relationship is being assigned has the ‘Can Hold’ flag selected against any of the responsibilities the practitioner has been associated with, all such relationships can be assigned to the practitioner.

  1. Expiry Based – This is a display only check box. This check box displays selected or cleared based on the relationship selected. This is as defined in the ‘Relationship’ function.

  2. Rule - This field displays the expiry rule as defined in the ‘Relationship’ function for the chosen relationship. This field is disabled and the value cannot be changed. If the Relationship is not expiry based, this field is blank and disabled.

  3. Reason – Select the required reason from the drop-down list. The reason codes defined in the Reason Code master will be populated in the list. This field is mandatory.

  4. Reason Text - On selecting the reason, the reason text defined in the Reason master gets displayed in the ‘Reason Text’ field. This field cannot be edited.

The 'Reason' and 'Reason Text' fields gets displayed only if the ‘Remarks Mandatory to Establish Without Encounter Relationships' check box is selected in Encounter Parameter screen. This is applicable only if 'Establish Relationship' check box is selected in Encounter Parameter.

  1. Start Date & Time - The system date and time is displayed by default as the start date/time for the relationship. This field is disabled and cannot be changed.

  2. End Date & Time - The end date/time is displayed based on the expiry rule specified.

  3. Performing Location – The Facility selected in the External Facility Booking dialog is displayed by default. You cannot edit the data.
