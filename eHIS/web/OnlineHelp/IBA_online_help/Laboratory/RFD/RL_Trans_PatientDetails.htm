The following details are displayed:

  1. Patient ID – Patient ID will be displayed.

  2. Sex – Patient's Gender is displayed.

  3. Age – Patient's age is displayed.

  4. Source – The source from where the request originated. The valid available values are 'Ward', 'Clinic' and 'External' depending on the episode type. The description for the source code is displayed.

  5. Category – Category number is displayed which is created in Application Masters.

  6. Location – Location of the patient is displayed.

  7. Assign To – The name of the technician who has assigned to the selected test in Acknowledge/Verification function is displayed.

  8. Priority – The specimen's priority indicator is displayed. The valid values are 'Routine', 'Urgent' and 'Stat'.

  9. Recd. Date – Registration date of specimen is displayed.

  10. Collection date – Specimen collection date is displayed.

  11. Doctor – The Name of the doctor who has placed the order is displayed.

  12. Clinician Comments – Clinician comments which has given while registration is displayed.
