Enter the following details:

  1. Analyzer Code - Enter the Online analyzer code in alphanumeric format of eight characters maximum length. This is a mandatory field.

  2. Department - The logged in Department Code is displayed, by default. Select required department from the list, if you have access to more than one department. This is a mandatory field.

  3. Long Description - Enter the Long Description for the Analyzers in alphanumeric format of 40 characters of maximum length. This is a mandatory field.

  4. Short Description - Enter the Short Description for the Analyzers in alphanumeric format of 15 characters maximum length. This is a mandatory field.

  5. Download - Select the check box, if the analyzer has to be programmed for processing the specimens based on the requests entered in the laboratory information system.

  6. Mode – This field is enabled on selecting the ‘Download’ check box. Select the method used to download the specimen details to the analyzer. The system-defined values are 'Manual', 'Automatic' and 'Host Query'. To have various analyzers linked to the system, the relevant programs for the analyzers must be installed. The value is set to ‘Manual’, if the check box ‘Download’ is cleared.

  7. Perform Site - Click [List of Values] to select the site. The list displays the values defined in Perform Site function.

  8. Analyzer Test - Enter a valid test code defined for the analyzer and supplied by the equipment manufacturer in alphanumeric format of 15 characters maximum length. This is a mandatory field.

Analyzer Test is not defined and maintained in the Laboratory module. This information is supplied by the equipment manufacturer.

  1. Test Code - Click [List of Values] to select a test code. The values are defined through the ‘Test Codes’ function. Test code has to represent an individual test. The same test code can be associated to different analyzers, but you cannot associate it to the same analyzer. Long name of the test code is displayed for user verification on entry of a valid test code. Numeric result has to be applicable for the test code entered. This is a mandatory field.

  2. Description - The corresponding Description of the selected Test code is displayed, by default.

  3. Result Type - Select a Result Type from the list. The system-defined values are ‘Numeric’, ‘Comment’ and ‘Both’. The list is set to 'Both', by default.

  4. Correction Factor - Enter a Correction Factor associated to the numeric result type of Analyzer equipment (if required). The correction factor has to be in numeric format not exceeding five digits.

  5. Interpret - Select the Interpret check box to invoke the ‘Interpretation of Analyzer Comments with Result Comments’ screen. The Non Numeric Result tab is displayed by default. The Interpret check box is enabled if selected Result Type is ‘Both’ or ‘Comment’. The check box is disabled, if the ‘Result Type’ is selected as ‘Numeric’.

    Enter the following details:

    1. Test Code – The selected test code in Analyzer is displayed, by default.

    2. Numeric Range [From-To] – Enter the numeric range [From-To] for cross reference.

    3. Interpret Numeric Result – Enter a valid value to interpret the Numeric Results in alphanumeric format of two characters maximum length.  

    4. Result Comment Code – Click [List of Values] to select the result comment. The list displays the values defined in ‘Result Comment’ function. This is a mandatory field.

    5. Description - The corresponding Description of the result comment is displayed, by default.

    Enter the following details:

    1. Test Code – The selected test code in Analyzer is displayed, by default.

    2. Analyzer Comment – Click [List of Values] to select Analyzer Comments. The listed values are created on clicking [Analyzer Comment]. This is a mandatory field.

    3. Result Comment Code – Click [List of Values] to select the result comments. The list displays the values defined in ‘Result Comment’ function. This is a mandatory field.

    4. Description - The corresponding Description of the result comment is displayed, by default.

  6. Auto Release ? – Select this check box to invoke Analyzer Auto Release screen to define auto validation for normal test results.

    Enter the following details:

    Normal Values

    1. Low – Enter the low value of the normal range of numeric result.

    2. High - Enter the high value of the normal range of numeric result.

    3. Delta Check – Select this check box to enable delta check for the normal test results.

     

    Delta Values

    1. Validity Period – Enter the Validity Period for the delta values.

    2. Validity Period Units – Enter ‘Days’ or ‘Months’ or ‘Year’ as the Validity Period Units.

    3. Check Type – Select the Check Type to enable Delta Numeric Result tab if the test result is of numeric type. Select the Delta Numeric Result check box, if the test requires delta check during validation. If the previous result for the test is within the time and the variation between the current numeric result and the previous result is more than the specified absolute or percentage value, the validation fails and the specimen is directed to technical review.

    4. Percentage – Enter a valid value to perform Delta Check is based on percentage.

    5. Path Review - Select the Path Review check box, if the test requires pathologist review for the delta result check.

    6. Facility Name – Select the Facility from the drop-down list. The Auto Validation is applicable for the selected facility.

    7. Rules –  Click [Rules] button to invoke the Rules dialog. Select the rules which has the “Analyzer auto release” selected. If a Rule is created in Rules masters but no configuration is made in ‘Review’, ‘Reflex test’ and ‘Auto Release’ tab, still rule can be linked with Analyzer master, but the rule will not be considered as active for any validation.
