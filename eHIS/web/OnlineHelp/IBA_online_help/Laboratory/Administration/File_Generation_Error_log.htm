File Generation Error Log

From this function you can View the error log of specimen records whose finalized specimen result report is not autosaved in the configured share path.

This function allows to view the error log of specimen records whose finalized specimen result report is not autosaved in the configured share path.

Also, allows user to retry for saving the Specimen result report pdf into the configured folder.

The shared path must be configured in the ‘Save Specimen Result Report pdf’ text box in Parameter for Facility screen.

Note:

• Only the finalized Specimen result report will be saved in the shared folder.

• The Specimen Result Report with prelim/draft status will not be saved into folder.

• If any modification is made to the finalized specimen results, system will replace (overwrite) the latest specimen result report file in the same folder.

Generating Error Log File

From Administration > Print Services, double-click the File Generation Error Log.

The File Generation Error Log screen gets displayed.

  1. By default, the specimen records that failed to save the pdf in the shared path for the current released week gets displayed.

  2. User can modify the default search criteria. <Related Field Description>

  3. Click [Search]. The specimen records whose result report is not saved in the shared folder appear as per search criteria.

  4. The search results include Facility, Department, Patient ID, Reference No, Released Date, Error Text, Last Run. The Last Run column displays the date and time of recent attempt to save pdf file into the shared folder, either from Result transaction screens by system (or) manual run.

  5. Select the required specimen record and click [Run] button. System will save the Specimen Result Report into the configured folder path.

  6. If save was unsuccessful again then 'Error text' will be updated with latest failure reason for the record.
