SS User Access for Store

From this function you can Create and restrict the user’s access to the transactions of the defined Stores and Query for the desired ‘User Access for Store’ records.

User Access for Store function helps you in managing the Store and perform applicable transactions that you can operate within the Store. All the stores defined in ‘CSSD Store’ are listed in the function. Users associated to a Store can perform the tasks that are defined while creating the Store. The access control fixes the boundary of operations for a User. The restrictions on the functions applicable for the store are explicitly selected and cleared as required.

Defining User Access for Store

From SS Administration, select the SS User Access for Store function.

The SS User Access for Store screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the User in alphanumeric format or select the User using [?] function. The User’s defined in the User function of System Manager module are displayed for selection. This is a mandatory field.

  3. On selecting the User, all the Stores defined in the CSSD Store function are displayed.

  4. Select the ‘Select’ check box to restrict the user’s access to the Transactions. On selecting the ‘Select’ check box, all the transactions applicable to the store are enabled and selected. You can select the relevant check box under Issue, Acknowledge and Confirm Acknowledge options under Issue Against Loan and Return of Loan.

  5. Click [Clear] to clear the entered data.

  6. Click [Apply] to save the User Access rights created or [Reset] to clear the entered data.

  7. Click [Query] to search for existing User Access for Store records. Transactions which are applicable to the store are displayed with Tick mark  and those transactions which are not applicable are marked with X mark.

Based on Store details defined in the CSSD Store function, the applicable transactions are enabled on selecting the ‘Select’ check box for any Stores.

All the SS Transaction functions Prepare Group, Request Group, Issue Group, Return Group,  Request for Loan, Issue against Loan, Return of Loan, Acknowledge Return of Loan, Acknowledge Issue against Loan Request, Confirm Acknowledgement of Issue against Loan Request, Return Instruments, Tray Discrepancy, Write Off, Physical Variance and Sterilization Request Processing are displayed with check boxes below each transaction. On selecting [Select] check box, all the transactions applicable to the store are enabled and checked, and those transactions not applicable to the store are disabled.  You can also individually selecting the required transaction against the store and then click Apply.

 

The administrator can grant or revoke the following new rights to the users for specific stores:

o Acknowledge Return of Loan

o Cancel Request for Loan (Rights to ‘Create’ Loan Request imply rights to ‘Cancel’ Loan Request)

o Acknowledge Issue against Loan

o Confirm Acknowledgement of Issue against Loan

User who has access to ‘Create’ a Request will imply access to ‘Cancel the Request’ also.
