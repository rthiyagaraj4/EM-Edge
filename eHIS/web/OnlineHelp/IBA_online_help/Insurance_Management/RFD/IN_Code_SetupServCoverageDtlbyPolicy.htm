Enter the following to set up Service and material item Coverage Detail by Policy:

 

  1. Facility – The Facilities associated to the logged in user are listed. Select a Facility from the list. This is a mandatory field.

  2. Payer Group – Click [List of Values] to view and select a Payer Group. The description for the selected Payer Group code gets displayed. The values defined using the function Customer Groups in the Patient Billing Manager or Accounts Receivable module get listed. The Coverage percentage is applicable for the selected Payer Group. The patients associated to the selected Payer Group avail the defined percentage of coverage amount. This is a mandatory field.

  3. Payer – Click [List of Values] to select a Payer code. The description for the selected Payer gets displayed. The values defined using the function Customer in the Patient Billing Manager or Accounts Receivable module get listed. The Payers associated to the selected Payer Group get listed. This is a mandatory field.

  4. Policy Type – Click [List of Values] to select a Policy Type. The values defined using the ‘Policy Type’ function get listed. The coverage percentage is valid for the selected Policy Type, the associated Payer Group and the Payer in the Facility. This is a mandatory field.

  5. Coverage % - Enter the Coverage Percentage in numeric format of three digits maximum length. The coverage percentage indicates the percentage of Insured amount on the total bill amount which is covered by the Insurance Company for the selected Policy Type. The percentage value cannot be greater than 100.

  6. Default Policy Start Date – Select the check box to default the ‘Policy Start Date’ and ‘Policy Expiry Date’ for a Payer Group, Payer and Policy Type combination. The ‘Policy Start Date’ is defaulted to the system date and the ‘Policy Expiry Date’ is calculated based on the number of days mentioned in ‘Default Policy Expiry Days’. The ‘Policy Start Date’ and ‘Policy Expiry Date’ fields are defaulted when the corresponding Payer Group, Payer and Policy Type is selected while recording financial details in the following functions in respective modules. The ‘Policy Effective From’ and ‘Policy Effective To’ date is defaulted to ‘Policy Start Date’ and ‘Policy Expiry Date’ respectively.

  7. Default Policy Expiry Days – Enter the expiry days in numeric format of four digits maximum length. The default expiry days for a Payer Group, Payer and Policy Type combination is based on the number of days recorded from the ‘Policy Start Date’. This field is mandatory, if the check box ‘Default Policy Start Date’ is selected. You can record the number of days only if the check box ‘Default Policy Start Date’ is selected. The ‘Policy Expiry Date’ is calculated and displayed by default in the following functions when you record the Payer details. You can click the Calendar function to update the Policy Expiry Date. The ‘Policy Effective To’ date is also defaulted to the ‘Policy Expiry Date’.
