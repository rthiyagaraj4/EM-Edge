Inpatient/Daycare

  1. Credit Auth Required – Select the check box, if the credit authorization is mandatory for the selected Policy Type and the associated Payer. The system prompts you to enter the credit authorization details in the transaction functions while modifying the patient billing details in the external modules and in the Patient Billing Manager module. The prompt is displayed only on selecting the relevant Policy Type, Payer and Payer Group.  

  2. Credit Auth Mandatory - Select the check box if credit authorization details are mandatory. If the check box is selected Credit Authorization details are mandatory during inpatient and daycare patient admission.

  3. Default Credit Auth Ref as Policy No. – Select the check box to default a part of the Policy Number as the Credit Authorization Reference number. The Policy Number can be a maximum of 30 alphanumeric characters. The first 20 characters out of the 30 alphanumeric characters are considered the Credit Authorization Reference number. This is applicable for the transaction functions in the Patient Billing Manager module, where you are able to update the financial details of an inpatient using the Modify Financial Details – Patient and Modify Financial Details – Admission in IP Direct Charges menu.

  4. Inpatient/Daycare Covered – Select the check box, if the selected Service Policy Coverage for the selected Policy Type is applicable to Inpatients or Daycare patients. If the check box is cleared, Policy Coverage details defined for the selected Policy Type is not valid for Inpatients or Daycare patients in the facility.

  5. Inpatient/Daycare Coverage Limit Type – Select an Inpatient or Daycare Coverage Limit Type from the list. The values are system defined – ‘Restricted’ and ‘Un-Restricted’. ‘Restricted’ is displayed, by default. You can select ‘Restricted’ as the value, only if the check box Inpatient/Daycare Covered is selected. If the check box is not selected, ‘Un-Restricted’ is displayed in the relevant fields.

  6. Restricted – Select Restricted to restrict the Inpatient or Daycare Coverage Limit for the Policy Type.

  7. Un-Restricted - Select Un-Restricted and the Inpatient or Daycare Coverage Limit is not restricted for the Inpatients or Daycare patients associated to the selected Policy Type, Payer and the Payer Group.

  1. Approved Days – If ‘Restricted’ is selected in the Coverage Limit Type, the Approved Days is also displayed as ‘Restricted’. If ‘Un-Restricted’ is selected in the Coverage Limit Type, the Approved Days is also displayed as ‘Un-Restricted’. Select a value from the list. The values are system defined – ‘Restricted’ and ‘Un-Restricted’. ‘Restricted’ is displayed, by default. You can select ‘Restricted’ as the value, only if the check box Inpatient/Daycare Covered is selected.

  2. Restricted – Select Restricted to restrict the Inpatient or Daycare Approved Days for the Policy Type.

  3. Un-Restricted - Select Un-Restricted and the Approved Days is not restricted for the Inpatients or Daycare associated to the selected Policy Type, Payer and the Payer Group.

  1. Approved Amt - If ‘Restricted’ is selected in the Coverage Limit Type, the Approved Days is also displayed as ‘Restricted’. If ‘Un-Restricted’ is selected in the Coverage Limit Type, the Approved Days is also displayed as ‘Un-Restricted’. Select a value from the list. The values are system defined – ‘Restricted’ and ‘Un-Restricted’. ‘Restricted’ is displayed, by default.

  2. Restricted – Select Restricted to restrict the Approved amount for the Inpatients/Daycare patients associated to the Policy Type, Payer and the Payer Group. The Inpatients/Daycare patients cannot avail an amount more than the amount specified for a specific admission to the facility.

  3. Un-Restricted - Select Un-Restricted and the approved amount is not restricted for the Inpatients/Daycare patients associated to the selected Policy Type, Payer and the Payer Group. On selecting Un-restricted, the Inpatients/Daycare patients admitted to the facility can avail any amount from the Insurance Company as the coverage amount.

  1. Mandatory Check for Bill Gen – Select a value from the list. The system defined values are ‘Not Required’, ‘Diagnosis by MR’ and ‘Diagnosis by Practitioner’. ‘Not Required’ is displayed, by default.

  2. Not Required – Select Not Required and the bill is generated without any mandatory checks.

  3. Diagnosis by MR – Select Diagnosis by MR and diagnosis by Medical Record is mandatory while generating the bill for an Inpatient or Daycare patient with the selected Policy Type associated to the selected Payer and Payer Group.

  4. Diagnosis by Practitioner - Select Diagnosis by Practitioner and Diagnosis by Practitioner is mandatory while generating the bill for an Inpatient or Daycare patient with the selected Policy Type associated to the selected Payer and Payer Group.

  1. Mandatory Check For Invoice Gen - Select a value from the list. The values are system defined – ‘Not Required’, ‘Diagnosis by MR’ and ‘Diagnosis by Practitioner’.

  2. Not Required – Select Not Required and the invoice is generated without any mandatory checks.

  3. Diagnosis by MR – Select Diagnosis by MR and diagnosis by Medical Record is mandatory while generating an invoice for an Inpatient with the selected Policy Type associated to the selected Payer and Payer Group.

  4. Diagnosis by Practitioner - Select Diagnosis by Practitioner and Diagnosis by Practitioner is mandatory while generating an invoice for an Inpatient or Daycare patient with the selected Policy Type associated to the selected Payer and Payer Group.
