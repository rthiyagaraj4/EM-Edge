Enter the following details to associate Payers to a Billing Group:

 

  1. Facility – The logged-in facility is displayed, by default. You can select a Facility from the list to set up the payers by Billing Group. The Facilities associated to the logged-in user get listed. This is a mandatory field.

  2. Billing Group - Select a Billing Group from the list. The listed Billing Groups are populated only if the Settlement Indicator is set as ‘Sponsored Patient’ and the Payer Settlement Type is set to ‘Insurance Company’ in the Patient Billing Manager module. This is a mandatory field.

  3. Patient Class – The Patient Class values are system defined. Click on the relevant Patient Class to associate patient class to the Billing Group and the Payers.

  4. Description – The Description for the Patient Class is displayed in the adjacent text field.

  5. Customer Group - Click [List of Values] to select a Customer Group. The listed Customer Groups are populated only if it is defined in the Patient Billing Manager or Accounts Receivable module. The selected Billing Group is valid to the Customer Group and the Patient Class selected. This is a mandatory field.

  6. Description – The Description for the relevant Customer Group is displayed.

  7. Customer - Click [List of Values] to select a valid Customer. The value is set to ‘**’, by default. The Billing Group selected is applicable to all the Payers associated to the Customer Group on selecting **. This is a mandatory field.

  8. Payer Name – The Payer Name is set to All Payers, by default.

  9. Policy Code - Click [List of Values] to select a Policy Code. The respective Policy is valid for all the Payers under the Customer Group associated to the Billing Group. The Description for the selected Policy Code gets displayed. This is a mandatory field .

  10. Priority – Enter the Priority for the Customer Group and the associated Policy Code in numeric format of two digits maximum length. This is a mandatory field .

  11. Adj Rule – Select an Adjustment Rule from the list. The system defined values are ‘Automatic’ and ‘Not Applicable’. ‘Automatic’ is displayed, by default. This indicates if the adjustment is applicable for the policy coverage amount or not. You can update the details associated to the Policy in the billing transactions.

  12. %age/Amt Ind - Select a value from the list. The values are system defined – ‘Percentage’ and ‘Amount’. ‘Percentage’ is displayed, by default. This indicates that the adjustment value is calculated on the basis of percentage or the specific amount set in Adj Value field.

  13. Adj Value – Enter the Adjustment Value in numeric format. If ‘Percentage’ is selected, the Adjustment Value has to be in a range between 1 to 100. The adjustment value is calculated based on the amount defined for the Policy Code, Customer, Customer Group and the associated Billing Group.

  14. Adj Interface Code - Click [List of Values] to select a valid Adjustment Interface Code. The Description for the selected Adjustment Interface Code is displayed. The interface codes defined in the Patient Billing Manager module get listed to associate the payer details to the selected GL Interface Codes.

  15. DRG Adj Interface Code - Click [List of Values] to select an Adjustment Interface Code for the Diagnosis Related Group. The Description for the selected DRG Adjustment Interface Code is displayed. The interface codes defined in the Patient Billing Manager module get listed .

  16. Spl Service Adj Interface Code - Click [List of Values] to select an Adjustment Interface Code for the Special Services. The Description for the selected Special Service Adjustment Interface Code is displayed to associate the payer details for special services to the GL Interface Codes.
