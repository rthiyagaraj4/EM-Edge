Enter the following details to define OA Parameter:

 

Appointments

  1. Control By - During first time setup, a list is displayed and subsequently the selected values are defaulted on clicking ‘Apply’. The list of valid values is ‘Patients’ and ‘Slots’. ‘Patients’ is displayed, by default. This setup is applicable for Timetable of type ‘Slot’. On selection of ‘Patients’ from the list of values the ‘No. of First Visit’, ‘No. of Other visits’ and ‘Maximum Over bookings’ are patients based; or on selection of ‘Slots’ from the list of values, the ‘No. of first visit’, No. of other visits’ and  ‘Maximum Over bookings’ are slots based for a patient.

  2. Overriding No. of Slots - Select the check box if Maximum Appt slots as set in ‘Visit Type for Location’ in Outpatient Management module can be skipped, and on selecting the check box a list gets displayed next to the ‘Visit Type’ while booking the appointment or if the check box is cleared, Maximum Appointment slot value as set in ‘Visit Type for Location’ in Outpatient Management module is set as default in the check box next to the ‘Visit Type’ while booking the appointment. The check box is cleared, by default.

  3. Visit Limit Rule - During first time setup a list gets displayed and subsequently the selected values are set as default on clicking ‘Apply’. The list of valid values includes Max No. of Visits, Exact No. of Visits and None. ‘Max No. of Visits’ gets displayed, by default. On selection of ‘Max No. of Visits’ from the list of values, the ‘No of First Visits’ and ‘No of other visits’ can exceed the ‘Maximum Patients per day’ while defining the timetable for a location. If you select ‘Exact No. of Visits’ from the list of values, the ‘No of First Visits’ and ‘No of other visits’ cannot exceed the ‘Maximum Patients per day’, while defining the timetable for a location. If the  No of First Visits’ and ‘No of other visits’ exceed the ‘Maximum Patients per day, the system displays an error message ‘Sum of Maximum first visits and Maximum other visits must be equal to Maximum patients per day’. On selection of ‘None’, you cannot enter the ‘No of First Visits’ and ‘No of other visits’ while defining the timetable for a location.

  4. Max Recurring Days – Enter the maximum recurring days for booking appointments of a patient with follow up visit in numeric format of three digits maximum length. This is applicable only for ‘Series’ visit type.

  5. Minimum elapsed period for scheduling No show - Enter the Minimum elapsed period for scheduling No show in numeric format of 3 characters maximum length. You are alerted for the No show appointments based on the number of days defined after which the alert message is shown. You cannot reschedule the No show appointments within the number of days specified.

  6. By Specialty/Location - Select either Specialty or Location to restrict rescheduling of No show appointments within the minimum elapsed duration based on the number of days entered in the 'Minimum elapsed period for scheduling No show days'. This is a one time setup.

  7. Capture Financial details during Appointment - Select the check box to enable the financial details while booking an appointment or clear the check box to disable the financial details. If the check box is selected in OA Parameter and if Patient Billing Manager module is installed, Financial Details Tab is displayed while booking an appointment. Once the check box is selected, Financial Details Tab is displayed and capturing financial details during Appointment is mandatory.

  8. Contact Number required during Appointment - Select the check box to display the 'Primary Residence Number' field in the Contact Details tab as mandatory. On selecting the check box, enter the Primary Residence Number of the patient while capturing the contact details during appointment scheduling. If the check box is cleared, entering the Primary Residence Number of the patient is optional.

  9. Integration of Radiology Appointments Applicable – Select the Integration of Radiology Appointments Applicable check box to integrate Appointment Scheduling with the Radiology module. It is a one-time setup and you cannot make any changes to it later. The check box is cleared by default.

  10. Number of No show Appts For Alert - Enter the number of No show appointments to display an alert in numeric format of two digits maximum length.

  11. Period Check For No Show Appts For Alert – Enter the number of days in numeric format of three digits maximum length for displaying the alert for No Show Appointments.

  12. Max Patient Cancelled Appointments for Alert – Enter the maximum appointment cancellation count for a patient in numeric format of three digits maximum length. Zero cannot be entered. If entered, the field resets to blank.

  13. Patient Cancellation Reason for Alert – Select the Reason for cancelling the appointment from the list. The list value is populated from the Reason masters in Application Masters. This field gets enabled only if value is entered in the ‘Max Patient Cancelled Appointments for Alert’ text box.

  14. Allow Group Appointment -  Select this check box if you want to enable the group appointments function in the system. By default, this is selected.

  15. Block Schedule even if Appointments exist - Select this check box if you want to block an appointment schedule, even if appointments are already booked within it.

  16. Synchronise contact details to Master Patient – Select this check box to synchronise the changes done in contact address in Appointment scheduling with MPI. The change reflects in all current and future appointments.

  17.  If this check box is not selected, the changes done in contact address in Appointment scheduling will not be synchronised to MPI. By default, this check box is not selected.

  18. When an appointment is booked without Patient ID, the contact details in MPI will not get updated since there is no patient ID. At a later stage, if the appointment without Patient ID is merged with a Patient ID, all the contact details available in the MPI will take precedence and get updated in Appointment.

 

Waitlist

  1. Waitlist Category Applicable - Select the check box to allow creating wait list category using the function 'Waitlist Category' or clear the check box.

  2. Waitlist Category Required - Select the check box if the wait list category is required or clear the check box. On selecting the check box, the 'Waitlist Category' field becomes mandatory in the function 'Create Waitlist'.

  3. Invitation Applicable - Select the check box to access the function Invite Waiting List Patients or clear it.

  4. Grace Period To Confirm Elapsed Waitlist - Enter the Grace period to confirm that the wait list is elapsed in numeric format of three digits maximum length. This field is mandatory.

  5. Cutoff Period For Closing Elapsed Waitlist - Enter the Cutoff period for closing the elapsed wait list in numeric format of three digits maximum length. This field is mandatory.

  6. Highlight Patients if not invited within - Enter the number of days after which the wait list patients are highlighted, if they are not invited.

  7. Restrict only One Waitlist for a Patient Against Category and Specialty - Select the check box to restrict one wait list for a patient against category and specialty or clear it.

 

Email

  1. Email Applicable - Select the check box to allow sending reminders or notifications to the patients through email. If the 'Email Notification for Appointment Applicable' check box is selected in Patient Series function of Master Patient Index module, sending notifications through Email is applicable to patients belonging to the specific Patient Series.. The 'Email Applicable' check box is enabled if email applicability is set up at the Site internally in System Manager installation. Clear the check box and sending reminders or notifications to the patients through email is not allowed.

  2. Email Applicable for Appointment with out Patient ID - This field gets enabled only if the 'Email Applicable' check box is selected. Select the check box to allow sending reminders or notifications through email to patients without Patient ID. Clear the check box and sending reminders or notifications through email to patients without Patient ID is not allowed.

  3. Send Reminder for Appointment Before - This field gets enabled only if the 'Email Applicable' check box is selected. Enter the number of days in numeric format of two digits maximum length for sending reminders through email to patients prior to the appointment date. If the appointments are transferred or cancelled the reminders are send to patients through email within 24 hours.

 

Rule Based Scheduling

  1. Schedule Rule Applicable - Select the Schedule Rule Applicable check box to decide whether the site has provision to do scheduling with and without rule. This is a onetime setup.

The below mentioned fields gets enabled only when the ‘Schedule Rule Applicable’ check box is selected.

  1. Mobile Number required during Appointment - Select the check box to record mobile number as mandatory while booking appointments.

  2. Patient Language required during Appointment - Select the check box to record patient language as mandatory while booking appointments.

  3. Number of Reschedule Appointments for Alert - Enter the number of appointments that can be rescheduled, after which an alert is displayed.

  4. Number of Cancelled Appointments for Alert - Enter the number of cancelled appointments, after which an alert is displayed.  

  5. Number of Future Appointments for Alert - Enter the number of future appointments, after which an alert is displayed.

  6. Period Check for Rescheduled Appointments for Alert (Past/Future) - Enter the number of days to check the appointments rescheduled in the past and future, after which an alert is displayed.

  7. Period Check for Cancelled Appointments for Alert (Past/Future) - Enter the number of days to check the appointments cancelled in the past and future, after which an alert is displayed.

  8. Period Check for Future Appointments for Alert - Enter the number of days before which booking can be made, after which an alert is displayed.

 

 

Appointment Slip Remarks

Enter the remarks that must be displayed while printing the appointment slip in alphanumeric format of 90 characters maximum length.

 

Report Tab

 

Reports Details

  1. Date - Enter the legend for the ‘Date’ that gets displayed while printing the report in English/local language of 11 characters maximum length.

  2. Reference - Enter the legend for the ‘Reference’ that gets displayed while printing the report in English/local language of 15 characters maximum length.

  3. Name - Enter the legend for the ‘Name’ that gets displayed while printing the report in English/local language of 11 characters maximum length.

  4. Address - Enter the legend for the ‘Address’ that gets displayed while printing the report in English/local language of 11 characters maximum length.

  5. National ID - Enter the legend for the ‘National ID’ that gets displayed while printing the report in English/local language of 11 characters maximum length.

  6. Salutation - Enter the legend for the ‘Salutation’ that gets displayed while printing the report in English/local language of 15 characters maximum length.

  7. Specialist/Medical Officer - Enter the legend for the ‘Specialist/Medical Officer’ that gets displayed while printing the report in English/local language of 15 characters maximum length.

  8. Clinic - Enter the legend for the ‘Clinic’ that gets displayed while printing the report in English/local language of 10 characters maximum length.

  9. Thanks - Enter the legend for the ‘Thanks’ that gets displayed while printing the report in English/local language of 11 characters maximum length.

 

No Show/Reschedule/Cancellation

  1. Title - Enter the Title that gets displayed while printing the report in English/local language of 15 characters maximum length for No Show, Reschedule and Cancellation.

  2. Report Body - Enter the content for the ‘Report Body’ that gets displayed while printing the report in English/local language for No Show, Reschedule and Cancellation.
