Reschedule Appointment

Appointments are rescheduled mainly due to patient’s request, changes in resource’s schedule or for any other reasons. An appointment is rescheduled either for the same day or for any other requested/available date and time for the particular Location/Resource combination.

The Reschedule Appointment screen displays a calendar and the day schedule of the clinic/resource combination.

  1. Select the available free time for the same day or future date (using the calendar) for the same practitioner to which the existing appointment has to be revised by clicking on the ‘From Time’ hyperlink from the Day Schedule screen.

  2. To reschedule the patient to the care of another resource, select the name of the resource from the list. Available schedules of the selected resource appears in green on the calendar that is displayed. Select the required date hyperlink in the calendar and the Day Schedule for that particular date is displayed. Proceed with selecting the required slot/slab from the Day Schedule for transferring the appointment to that date.

  3. Reschedule Appointment screen displays with appointment details of the patient.

  4. Select the reason for revision by clicking the [?] function. All the reasons applicable for revision of appointment as set in Reason master of Application Masters module is displayed for selection.

  5. Select the reason for overriding by clicking [?] function. The 'Reason for Override' field is available only if the ‘Schedule Rule Applicable’ check box is selected in OA Parameter.

  6. Click [Edit Instructions] to modify instructions if Patient Instructions exist.

  7. Click [Contact Details] to view or modify the contact details of the patient.

  8. Click [Secondary Resources] if the secondary resources booked against an appointment has to be revised to a new time. Select the new time against which the secondary resources are to be blocked by clicking on check box against the required time slot/slab provided. Click [OK] to accept the change and the control reverts back to the ‘Reschedule Appointment’ screen; else click [Cancel] to retain the same time.

    The Secondary Resources page displays the following details:

    The header displays the Patient details like Name, Gender, Age, Patient Id, Location/Resource, Appointment Ref No, and Appointment Date.

    1. Resource Class – The resource class of the secondary resource is displayed.

    2. Resource Type – The resource type of the secondary resource is displayed.

    3. Resource Name – The resource name of the secondary resource is displayed.

    4. From Time – The time from when the secondary resource is required for the appointment is displayed.

    5. To Time – The time until which the secondary resource is required for the appointment is displayed.

  9. Click [Confirm Revision] to confirm reschedule appointment to the new date/time/resource selected. A confirmation message appears. Click [OK] and control reverts to the main screen.

  10. Internal Reports screen appears if defined in print routing setup, click [OK] to generate Internal Reports or click [Cancel] to exit from the screen.

 

  • Appointments can be rescheduled to a particular time/date/resource, if the schedules are generated and available for a particular time/date/resource.

  • If the selected patient has other appointments then the [Other Appts] button is enabled and if previous encounter exist for the patient then the [Previous Encounter] button is enabled.

    The Previous Encounter button will be enabled only when the Patient has made preceding visits to other locations. When [Previous Encounter] hyperlink is clicked, all the previous visits of the patient are displayed. The previous visit details are only displayed and cannot be changed.

    The Previous Encounter page displays the following details:

    1. Encounter Date/Time – The Date and Time of past visits are displayed.

    2. Encounter Type – The type of encounter made by the patient is displayed.

    3. Encounter Id – The ID generated for the encounter is displayed.

    4. Location Type – The location type against which the encounter was made is displayed.

    5. Location – The location against which the encounter was made is displayed.

    6. Service - The service that was applicable during that encounter is displayed.

    7. Sub-Service - The sub service that was applicable during that encounter is displayed.

    8. Room No – The room number against which the encounter was registered is displayed.

    9. Bed No – The bed number assigned for an ‘Inpatient’ encounter is displayed.

    10. Resource - The resource against which the encounter was recorded is displayed.

    11. Disp Date/Time – The disposition date and time in dd/mm/yyyy hh24:mm format is displayed.

    12. Disp Type – The disposition type recorded at the close of the encounter is displayed.

    Other Appointment button appears only for patients with current/future day appointments or bookings and having a valid patient ID. When [Other Appts] button is clicked, all other appointment of the Patient is displayed. The appointment details are only displayed and cannot be changed.

  • Forced Appointments cannot be transferred.

  • Bulk transfer cannot be performed when 'Schedule Rule Applicable' check box is selected in the OA Parameter.

  • An alert message ‘Patient has cancelled appointments X times previously. Click OK to notify the patient else Cancel and Proceed’ is displayed during appointment scheduling when the number of appointments cancelled by the patient reaches the ‘Max Patient Cancelled Appointments for Alert’ count defined in OA Parameter while trying to book, reschedule, cancel or transfer appointment. Click [OK], the cancellation count resets to zero provided the transaction is complete.  The cancellation count is not impacted by Bulk Cancellation or Bulk Transfer and no alert message is displayed.
