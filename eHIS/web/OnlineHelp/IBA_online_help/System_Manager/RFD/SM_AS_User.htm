Enter the following details to create a User record:

 

  1. ID – Enter the User ID in alphanumeric format of 30 characters maximum length. Numeric value cannot be the starting character of the ID. This is used to log-in to Enterprise Management application. The User ID is maintained in upper case and displayed on the desktop of the application. The User ID has to be unique and cannot be changed after creation. ID – Special character ‘$' and' #’ are allowed.  This is a mandatory field.

  2. Name - Enter the Name of the User in alphanumeric format of 50 characters maximum length. This is a mandatory field.

  3. Long Name – Enter the long name of the user in this field. You can enter upto 300 characters. This field is optional.

  4. Preferred Language - Select the Preferred Language from the list of values. The displayed values are ‘Thai’ and ‘English’. This is a mandatory field.

  5. Effective [From/To] - Enter the Effective date range in dd/mm/yyyy format. The date range defines the period until when the user is allowed to access the application.

  6. Link Active Directory User – Select the check box to link the EM user as Active Directory user. By default, the check box is cleared.

  7. Active Directory ID – This field gets enabled and mandatory when the ‘Link Active Directory User’ check box us selected. Select the required Active Directory ID from the LOV. Once the active ID is selected, the Password and Confirm Password gets auto filled and disabled.

  8. Enabled – Select the check box to enable the record in the transaction functions. The check box is selected, by default. The enabled users only are allowed to access the system.

 

Contact Details

 

  1. Reporting Facility - Select the facility from the list of values. The selected facility appears as the default facility in the Responsibility screen during login. In case, if you do not select any value, the available Facility is displayed in alphabetical order in the Responsibility screen.  The Reporting Facility identifies the facility where the user normally reports.

  1. Telephone - Enter the Telephone number of the User in alphanumeric format of 20 characters maximum length.

  2. Fax - Enter the Fax number of the User in alphanumeric format of 20 characters maximum length.

  3. Email - Enter the Email ID of the User in alphanumeric format of 50 characters maximum length.

 

Functional Role Details

 

  1. Role Type - Select the functional Role Type of the User, if applicable. Current version supports the role type 'Practitioner' and 'Other Staff'.

 The legends below change based on the 'Role Type' selected.

  1. Practitioner/Other Staff Type - Select a value from the list. The values in the list differ based on the 'Role Type' selected.

  1. Practitioner/Other Staff Name - Select the Practitioner or Other Staff Name by using [?] function. The values in the list differ based on the 'Role Type' selected.

 

Other Details

 

  1. Resource Group – Select a Resource Group from the list.

  1. Employee No - Enter the Employee Number in alphanumeric format of 10 characters maximum length.

  2. Responsibility Group – Select the Responsibility Group from the list. The Responsibility Group defined gets listed.

  3. Report Output Directory – Enter the Location, where the Report file is to be stored. This has to be entered for the particular user in reports section. If the location is not specified, the reports are saved in default directory.

 

Password

 

  1. Password - Enter the Password of the user in alphanumeric format of 15 characters maximum length. The minimum length of characters entered has to be of five characters. This is a mandatory field.

  1. Confirm password - Re-enter the password of the user in alphanumeric format of 15 characters maximum length. This is a mandatory field and has to be the same as the value in Password field.

  2. PIN No – Enter the Personal Identification Number in alphanumeric characters of length 20 characters maximum length.

When the ‘Enforce Password Policy’ check box is selected in Site screen, the Password Policy legend ‘Password should start with Alphabet and followed either by Numeric values or these Special characters _~$^*`?-[]’ appears at the bottom of the screen.
