Responsibility for User

From this function you can Associate a Responsibility to a User for a specified date range and Query for a desired Responsibility associated to a User.

The Responsibility for User function is used to associate multiple responsibilities to a user. For example, the User ‘Allen’ can be assigned the responsibilities ‘Appointment Scheduling’ and ‘Blood Transfusion’. User is able to select from a list of associated Responsibilities while logging-in to the system and access the functions within the selected responsibility. While using the system, a user can switch between responsibilities using [Switch Responsibility]. On successful switching of responsibility, a user gets revised access to all the functions of the menu associated to the new responsibility.

Associating Responsibility to a User

From the SM Admin Setup > Application Security, select the Responsibility for User function.

The Responsibility for User screen gets displayed.

  1. Click [Create] to invoke the Create screen.

The Responsibility for User is set under two Tabs, namely:

By User

By Responsibility

  1. Enter the details and click [Apply] to save the record or [Reset] to clear the entered data.

  2. Click [Query] to search for an existing record.

 

If a user is not assigned to any responsibility, the user cannot log-in to the system.
