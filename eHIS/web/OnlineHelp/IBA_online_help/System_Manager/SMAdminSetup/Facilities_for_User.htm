Facilities for User

From this function you can Associate Facilities to a User for a specified date range and Query for a desired Facility associated to a User.

A User can be assigned access to one or more facilities using this function. For example, the user ‘Allen’ can be given access to the facilities ‘Get Well Clinic’ and ‘Good Health Clinic’.

A user with multiple responsibilities can select one of the responsibilities while logging-in to the system. The list of applicable facilities the User has access to and the corresponding responsibility associated to it are listed. On selection of a valid facility, the User gets access to all the functions of the menu associated to the selected responsibility and limited to data pertaining to the login facility. While using the system, the user has an option to switch the responsibility and facility at any point of time using [Switch Responsibility]. On successful switching of responsibility/facility, the user gets the access to all the functions of the menu associated to the new responsibility and limited to data pertaining to the new log-in facility.

Associating Facilities to a User

From the SM Admin Setup > Application Security, select the Facilities for User function.

The Facilities for User screen gets displayed.

  1. Click [Create] to invoke the Create screen.

The Facilities for User is set under two Tabs, namely:

By User

By Facility

  1. Click [Apply] to save the record or [Reset] to clear the entered data.

  2. Click [Query] to search for an existing record.

 

It is mandatory to associate a Facility to the user in order to log-in and access the application. User is unable to log-in to the system, if a Facility is not assigned to the User. If the selected responsibility of the user is not associated to the facility with which the user is associated, such a facility cannot be selected during log-in.
