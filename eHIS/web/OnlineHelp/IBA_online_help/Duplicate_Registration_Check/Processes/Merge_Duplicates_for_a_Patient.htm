Merge Duplicates for a Patient

From this function you can Merge duplicates of a patient by searching for the possible duplicate.

You can Search, View Possible Duplicates of a Patient using the user-specified Search Criteria, Identify main and duplicates and merge records using this function. You can perform all the DR Processes in this function. This function cannot be invoked if DR cycle is in progress. The access to this function is determined by the User Access Rights set for the logged-in User. If the rights are not defined or the right for the DR Process is set as ‘None’ for the logged in user, the user is denied access. The user is allowed to access this function only if the access rights to both the functions ‘Identify Main/Duplicates’ and ‘Merge Duplicates’ are defined.

If the privilege level is set as Logged-in facility, you can perform the DR Process only for the Logged-in facility. If the privilege level is set as All Facilities, you are allowed to access the function and can perform the DR Process for all facilities.

On invoking this function from the menu, if the DR Process is active or the cycle is still not closed, a message ‘The Cycle is already in progress’ is displayed. The DR Process has to be closed in the ‘Administer Search Process’ function.

Merging Duplicate Records for a Patient

From the DR Processes, select the Merge Duplicates for a Patient function.

The Merge Duplicates for a Patient screen gets displayed.

  1. Enter the Patient ID in alphanumeric format of maximum length as specified in the MP Parameter or select the Patient ID using the [?] function. This field is mandatory.

  2. Click [Execute] to invoke ‘Duplicate Search Criteria’ frame.

  3. Select the desired (minimum three) search criteria. The buttons [Accept], [Identify Duplicate], [Start Now] and [Start Later] are disabled while specifying the search criteria.

    The details of the patient are displayed in the text field.

    1. Name in Local Language - Select the check box to include Patient Name in local language as search criteria. This field is displayed, if ‘Accept Patient Name in Local Language’ check box is selected in Legal Names tab of MP Parameter in the Master Patient Index module.

    2. Nationality - Select the Nationality check box to consider Nationality as a criterion while searching for possible duplicates.

    3. Name - The name components (First/Second/Third/Family Name) set in Master Patient Index module are displayed. Select the check box against each component to include it as search condition.

    4. Gender - Select the Gender check box to consider Gender of patient as a criterion while searching for possible duplicates.

    5. Search in all Facilities - Select the check box to search for duplicate records across facilities or clear the check box to perform the search within the logged in facility.

    6. Date of Birth - The Date of Birth of the patient entered during patient registration is displayed

    7. By - Select the value from the list of valid values. The values displayed are Year, Year and Months and Days. The Year is selected, by default.

    8. Primary Resi No. - Select the Primary Residence Telephone Number check box to consider Residence Telephone Number as a criterion while searching for possible duplicates.

    9. Other Resi No. - Select the Other Residence Number check box to consider Other Contact Number as a criterion while searching for possible duplicates.

    10. Area – Select the Area check box to consider the Area as a criterion while searching for possible duplicates.

    11. Other Alt ID - The Alternate ID(s) set in Master Patient Index are listed. Select the Alternate ID from the list.

    12. Soundex - The Soundex can be selected as either 'None' or 'Generic'. This field is disabled if the ‘Name in Local Language’ is selected.

    13. Date &Time - The Date and Time field is enabled only after clicking [Accept].

    A check box is provided against each field in the ‘Duplicate Search Criteria’. On selecting the check boxes, you can specify the search criteria for searching duplicate records for the selected patient. The user has to select a minimum of three fields as search criteria.

  1. Click [Search] to view the possible duplicate records. On searching, the possible duplicate patient details are displayed and [Accept] option is enabled.

  2. Click ‘+’ hyperlink to view the patient details recorded during registration.

  3. Click [Accept] to accept the searched records as duplicates. On accepting, this option is disabled and [Identify Duplicate], [Start Now] and [Start Later] options are enabled.

  4. Click [Identify Duplicate] to identify Main/Duplicate patient records. The ‘Identify Main/Duplicate Patients’ function screen is invoked.

    1. Click the Patient ID hyperlink from the Duplicate Groups frame to display the duplicate details in the Group Details frame. Click the + hyperlink from the Duplicate Groups frame to view the Patient Details recorded during registration.

      1. Patient ID - The Main Patient ID and the Duplicate Patient ID's followed by the Name of the patient, Age and Gender are displayed.

      2. Main - A tick mark is displayed for the main patient record.

      3. Duplicate - A tick mark is displayed for the duplicate records.

      4. Do not consider - A tick mark is displayed for records not considered.

       

    Or

    1. Enter the desired Patient ID or select the Patient ID using the [?] function. Click [Search] to view the duplicate patient in the Group Details frame.

    2. Click [Search Criteria] to view the defined search criteria and [Close] to exit from the Search Criteria screen.

    3. Select at least two patients under the view heading and click [Duplicate Details] to view Possible Duplicate Details. Click Patient ID hyperlink to view the patient details or click [Close] to exit from the Possible Duplicate Details screen.

    4. Click 'E' hyperlink to view previous encounter details of the patient. Click Encounter ID hyperlink to view the encounter details of the selected patient.

    5. Click Patient ID hyperlink to view Patient details. The details entered while registering a patient is displayed.

    6. Identify the patients as Main, Duplicate, Do not consider or Exclude. Select the desired option to identity the records.

      Exclude – Select the radio button to exclude the record from merge process.

      Do Not Consider – Select the radio button not to consider the record as main/duplicate.

      Duplicate – Select the radio button to select the record as duplicate record.

      Main - Select the radio button to select the record as Main record. Only one main record can be selected.

    7. Click [Apply] to save the transaction or [Close] to exit from the Identify Main/Duplicate Patient screen.

  5. On identifying the main and duplicate patients, click [Start Now] to merge the duplicate records identified or enter the date and time and click [Start Later] to start the merge process later.

  6. Click [Reset] to clear the entered data.

 

To view the merge process details, invoke the ‘Administer Search Process’ function and view the merge details.

The Merge process of the main and duplicate patient records begin when the user starts the duplicate merge Process in ‘Administer Background Process’ function in System Manager Module.
