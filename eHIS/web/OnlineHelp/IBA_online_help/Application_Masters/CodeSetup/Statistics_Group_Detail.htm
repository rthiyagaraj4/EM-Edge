Statistics Group Detail

From this function you can Associate the Statistics Group to system-defined categories such as Nationality, Specialty and Service. You can Query to view the existing records.

Statistics Group Detail function is used to associate the Statistics Group to system-defined categories such as Nationality, Specialty and Service. Based on the required Statistics Group, different components associated to the selected Category can be grouped to generate a detailed report. For example, a Statistics Group is defined based on the nationality. This function is used to include applicable nationalities to the defined Statistics Group. Nationalities are defined in the Master Patient Index module.

Associating Statistics Group to a Category

From the Statistics Setup, select the Statistics Group Detail function.

The Statistics Group Detail screen gets displayed.

  1. Click [Create] to invoke the create screen.

  2. Select a Category from the list. The system-defined values are ‘Nationality’, ‘Service’ and ‘Specialty’. This is a mandatory field.

  3. Select the Statistics Group from the list. The Statistics Group created through the ‘Statistics Group’ function with the ‘Enabled’ check box as selected gets listed. This is a mandatory field.

  4. Click [Search Member] to select the values from the list displayed in alphabetical order based on the Category selected. The screen displays alphabets from ‘A to Z’ and ‘Others’. These alphabets are hyperlinked. Each alphabet contains data relating to Nationality, Service or Specialty. The ‘Others’ hyperlink contains data relating to Nationality, Service or Specialty that begins with characters other than the alphabets. Click on the required alphabet and the values beginning with the selected alphabet for the respective Category are listed.  

  5. Select the check box against the required value to associate it to the selected Statistics Group. Clear the check box to dissociate the values from the Statistics Group.

  6. Click [Apply] to save the records or [Reset] to clear the entered data.

  7. Click [Query] to search for an existing record.
