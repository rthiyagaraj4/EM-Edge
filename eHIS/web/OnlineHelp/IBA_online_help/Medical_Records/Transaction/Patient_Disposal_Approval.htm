Patient Disposal Approval

From this function you can Approve the patient IDs for disposal.

This function allows you to generate the patient IDs or lists out the patient ID’s based on the defined criteria, who are not having any active encounter or inactive or no visit, since parameter defined years (like 1, 2 or 5 YRS) against a specialty. Thereby, allows to mark the Patient ID’s as Disposal.

 

The system considers the following hierarchy to mark patient as disposal:

  • The system will not list the patient in the transaction list, if below criteria is met:

    • If patient is having a case of MLC, Post Mortem, OSCC.

    • If the patient has undergone Blood Transfusion and Blood Donor.

    • If the patient is Psychiatric and Alive.

  • The system will list the patients only if medical report status is in ‘Cancelled’ and ‘Report Delivered’.

  • The patient marked for Blood Donor will not be listed.

  • The system will exclude the patient to appear in the transaction list, if it is initiated for merge process.

 

The system considers following criteria before marking the Patient ID’s as Disposal.

  • If encounter is open, the system will not allow to mark EMR status as disposed.

  • The date Patient is checked out/discharged, will be considered to mark EMR status as disposed.

  • The encounter Discharge /Checked out date will be considered to mark EMR status as disposed.

  • The Past and Future Appointment/Referral/ Booking will not be checked while listing these patients in the transaction screen and marking as disposal.

Approving Patient ID for Disposal

From MR Transactions, select the Patient Disposal Approval function.

The Patient Disposal Criteria screen gets displayed.

  1. Select the Patient ID from the lookup.

  2. Select the required Date From by clicking the calendar icon. This field is mandatory.

  3. Click [Search] to view the list of patient IDs to be considered for disposal.

  4. Once the patient records get displayed, the Disposal Approval button gets enabled.

  5. Select the check box against the patient IDs to be disposed and click [Disposal Approval] button to send for approval. The message 'Operation completed successfully' gets displayed with OK button.  

A batch ID gets generated and stored internally. The batch ID consists of 'User ID- System Name-Date & Time'. The disposal is yet not done. It means one group of patients pending for disposal approval.

  1. Click [OK], the ‘Internal Report’ dialog gets displayed to print patients selected for disposal.

  2. If you search again, the list will exclude the selected (for disposal approval) patient in the newly generated list. The system will not list those patient records who have been sent for disposal approval.
