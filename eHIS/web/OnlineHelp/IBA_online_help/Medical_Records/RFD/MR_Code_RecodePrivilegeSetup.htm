Enter the following to create a Recode Privileges Setup:

  1. Stage No - Select the Stage Number from the list. You can create users with three levels of privileges such as Level 1, Level 2 and Level 3. The transactions done by a lower level user can be approved or rejected by a user with a higher level. This is a mandatory field.

  2. Stage Description - Enter the Stage Description to define the stage in alphanumeric format of 30 characters maximum length. This is a mandatory field.

  3. Specific Indicator - Select the specific indicator from the list. The listed values are 'Specific users' and 'All Users'. If you select 'Specific users', the level you are creating is applicable to any required user. If you select 'All users', the level you are creating is applicable to all the users in the facility.

  4. Cut off Period for Recoding - Enter the cut off period for recoding in days from the date of visit completion or discharge date. If the 'Applicable Patient class' in the MR Parameter is set as Inpatient/Day care, the cut-off period on visit completion is disabled. If the 'Applicable Patient class' in the MR Parameter is set as Outpatient/Emergency, the cut-off period on discharge is disabled.

  5. Allow Authorization - Select the check box to allow authorization for the defined stage. This check box is enabled based on the stage set in 'Stage at which authorisation is applicable' field in MR Parameter function.

  6. Action If Not Done - Select the value from the list. The listed values are Alert and Stop. If the value Alert is selected and the higher level user tries to perform recoding beyond the days set in 'Cut off Period for Recoding' field, the system alerts the user based on this setup. If the value is selected as Stop, the higher level user cannot view the details to perform recoding beyond the days set in 'Cut off Period for Recoding' field.

 

Applicable Recoding Component (Diagnosis)

Select the check box against 'Diagnosis Classification', 'Accuracy', 'Severity', 'Priority', 'Onset Type', 'Onset Date', 'Applicable Side', 'Status', 'As on Date', 'Diagnosis', 'Remarks', Encounter Stages’, ‘Chief Complaint’ and ‘Work Related’ if applicable for Diagnosis Recoding to the user.

Applicable Recoding Component (Procedure)

Select the check box against 'Laterality', 'Code Extension' and 'Remarks' if applicable for Procedure Recoding to the user.

 

  1. User ID - Enter the User ID in alphanumeric format of six characters maximum length or select the User using [?] function. The Users with Operator Station privilege for Diagnosis/Procedure Recoding for the logged in facility are displayed for selection. This is a mandatory field. This field is disabled if 'Specific Indicator' field is selected as 'All Users'.

  2. User Name - The name of the User is displayed on selecting the User ID.

  3. Authorize - This field is enabled, if 'Allow Authorization’ check box is selected. Select the check box to indicate that the selected User has privilege to authorize the recoding completed. The check box is enabled based on the stage set in 'Stage at which authorization is applicable' field in MR Parameter function.
