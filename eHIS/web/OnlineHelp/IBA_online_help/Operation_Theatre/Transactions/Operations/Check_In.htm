Check In

Check In function is used to record checking in of the patient.

  1. The Order ID, Date and Time of booking, Surgeon, Specialty, Theatre, Booking Number, Date and Time of the surgery are displayed.

  2. The following details are displayed as recorded while booking – Waitlist No, Surgery Type, Admitting Specialty, Anesthetist, Anesthesia Type, Operating Specialty, Theatre, Surgeon, Assistant Surgeon 1, Assistant Surgeon 2 and Check-In Date/Time.

You can edit the Check-In Date/Time.

 

  1. Select the ‘Anaesthesia Review’ check box to perform pre-anaesthesia, intra-anaesthesia and post-anaesthesia evaluations in Booking Verification function. If the check box is cleared, the system does not allow pre-anaesthesia, intra-anaesthesia and post-anaesthesia evaluations in Booking Verification function. This check box is cleared, by default.

  2. Patient details are displayed when the Patient ID is selected.

    The Name of the Patient and Gender are displayed. Encounter ID is displayed by default only if the selected patient has a single encounter ID. Select the Encounter ID of the patient using the [?] function if there are multiple encounters.

  3. Click Pre-operative Diagnosis hyperlink to view the details recorded while booking.

  4. Click Pre-Operative Diagnosis hyperlink to record the record the pre-operative diagnosis.

  5. Click Verify Checklist hyperlink to verify the checklist.

 

Operations/Procedures

  1. The Operation Code, Description, Side Applicable, Remarks recorded for the Operation/Procedure and Charges are displayed. Click  to edit or enter charge details. The status ‘Included?’ and ‘Approval Reqd?’ of Insurance Billing Manager are displayed if the billing charges are applicable in Operation Theatre module.

  2. Click [Apply] to save checking in. If checklist is not marked as completed and if it is required for the Check In function, the system does not allow saving. Mark the checklist as completed by clicking the Verify Checklist hyperlink. Click [Apply] to save checking in.

 

The system  displays a warning message about the existing scheduled operation when a patient already has an existing scheduled and/or unscheduled operation for a particular day and you try to check-in or book the patient for the same day. The warning message is ‘Patient has existing operations for this day, Do you want to continue?’i
