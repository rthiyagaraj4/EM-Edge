Post Credit Notes

From this function you can Post error free Credit Notes to the General Ledger interface.

This function is used to post the error free credit notes. You cannot update the records after the credit notes are posted. System carries certain validations while posting the credit notes. The documents are posted to the General Ledger interface. The Document Type in the General Ledger interface has to be of type credit.

• Validation of each selected document is done and all the errors encountered are recorded. Error free documents are posted. If AR to GL interface is "Online", the parallel updating is done in General Ledger module. When AR to GL interface is in the status of "None" or "Batch", the General Ledger module is not updated.

• The erroneous documents (those which could not be posted) are printed together with the list of errors, if any. Such documents can be recalled through the entry

option and corrected.

• If the document requires printing before posting, such documents are not posted. Print the document first using the function "Print Credit Note" and post the document. Otherwise, Automatic Print can be disabled in Document Type Details for this document type in the Systems Manager module and then posted.

Posting a Credit Note

From the Transactions > Credit Notes, double-click Post Credit Notes function.

The Post Credit Notes screen gets displayed.

  1. Enter the range for Document Type or click [List of Values] to select the required Document Type. Both the values are optional. If ‘From’ value is not entered, the system assumes the lowest value for the ‘From’ and if ‘To’ value is not entered, the system assumes the highest value for the ‘To’ value. When both values are entered, ‘To’ value must be greater than or equal to the ‘From’ value.

  2. Enter the range for Document Number or click [List of Values] to select the required Document Number. Both the values are optional. If ‘From’ value is not entered, the system assumes the lowest value for the ‘From’ and if ‘To’ value is not entered, the system assumes the highest value for the ‘To’ value. When both values are entered, ‘To’ value must be greater than or equal to the ‘From’ value.

  3. Enter the range for Document Date or select the required date using the Calendar function. Both the values are optional. If ‘From’ value is not entered, the system assumes the lowest value for the ‘From’ and if ‘To’ value is not entered, the system assumes the highest value for the ‘To’ value. When both values are entered, ‘To’ value must be greater than or equal to the ‘From’ value.

  4. Click [Run Process] to view, print or save the adjustment details in a file in the local machine.  

  5. A log file gets generated. Click [OK] to view the log file or [Cancel] to exit from the screen.
