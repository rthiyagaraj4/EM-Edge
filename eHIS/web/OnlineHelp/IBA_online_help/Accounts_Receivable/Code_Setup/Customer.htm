Customer

From this function you can Create a Customer and Query for a desired Customer record.

The function Customer is used to enter the details of the Customers. Sometimes, a patient can be defined as a Customer. A person paying on behalf of the patient too can be considered as a Customer. All transactions processes are made against Customer codes. You also have an option to set the Customer validity for a patient class in the facility.  

Customers can be broadly classified as follows:

• Patient related Customers

• Non-Patient related Customers

Patient related Customers are classified into two types:

• Credit patients (patients themselves have to settle the bills)

• Organizations to which credit facility is offered (sponsored patients)

Organizations can be insurance companies, patient sponsoring companies, etc. that settles the organization’s invoices on behalf of the patients.

Non-Patient related Customers are those Customers who purchase scrap or other items from the organization. Such Customers can be classified as:

• Normal Customers

• Miscellaneous Customers

Miscellaneous Customers are one-time Customers and repeat business with such Customers is not envisaged. The name and address of such Customers is accepted during the processing of transaction.

For each Customer, the mutually agreed frequency for sending the statement of accounts also is specified. The statement of accounts are posted on a weekly, monthly, quarterly or yearly basis.  You can define the credit limits for credit control purposes, which can be reviewed and updated. The validity period for each Customer within the facility is also defined.

Creating a Customer

From the Masters, double-click Customer function.

The Customer screen gets displayed.

  1. Enter the details in the mandatory (*) fields. <Related Field Description>

  2. Enter the parameters for the selected Customer under the following tabs:

Reference Codes

Enter the following details:

  1. Customer Type – Click [List of Values] to select a Customer Type code. The list displays Customer Types as created in the ‘Customer Type’ function under Masters. If the code selected is valid, the short description of the respective code is displayed for user verification. The description cannot be edited. This is a mandatory field.

  2. Customer Group - Click [List of Values] to select a Customer Group code. The list displays Customer Groups as created in the ‘Customer Group’ function under Masters. If the code selected is valid, the short description of the respective code is displayed for user verification. The description cannot be edited.  This is a mandatory field.

  3. Payment Terms - Payment Terms is a display field. On entering the value for Customer Group value, the associated Payment Terms gets displayed.   This is a mandatory field.

  4. Control Account – Click [List of Values] to select a Control Account code. The list displays the control account as created in the ‘Chart of Accounts’ function under GL Setup. The description for the selected Chart of Account gets displayed.  This is a mandatory field.

  5. Reminder Letter - Click [List of Values] to select a Reminder Letter. The list displays the values created using the ‘Reminder Letter’ function under Masters. The code cannot be edited. The selected Reminder Letter is dispatched when the payment is due for the selected Customer. This is a mandatory field.

Customer Valid Flags

Select or clear the following check boxes:

  1. Valid for Inpatients – Select the check box, if the Customer is valid for Inpatients. The check box is cleared, by default. If the check box is selected, the Customer code gets displayed while recording the financial details for an admission in Inpatient and Day Care Management module.

  2. Valid for Outpatients – Select the check box, if the Customer is valid for Outpatients. The check box is cleared, by default. If the check box is selected, the Customer code gets displayed while recording the financial details for an Outpatient visit in Outpatient Management module.

  3. Valid for External – Select the check box, if the Customer is valid for external or referral  patients . The check box is cleared, by default. If the check box is selected, the Customer code gets displayed while recording the financial details for a referral or external patient registration.

  4. Covered by Insurance – Select the check box, if the Customer is applicable for Insurance coverage.  The check box is cleared, by default. If the check box is selected, the Customer code gets displayed while recording the financial details for an insured patient.

Analysis

Enter the following details:

Analysis

  1. Statement Frequency – Select the appropriate Statement Frequency from the list. The listed values are Weekly, Monthly, Quarterly and Yearly. “Weekly” is displayed on the screen, by default.

Analysis Codes

  1. Billing Group – Click [List of Values] to select a Billing Group . The entries with code and its description get populated only when valid Billing Groups are defined in the Patient Billing Manager Module.

  2. Code 2 - Enter the analysis code in alphanumeric format of eight characters maximum length. The analysis codes are used for reporting purpose.

  3. Code 3 - Enter the analysis code in alphanumeric format of eight characters maximum length.  The analysis codes are used for reporting purpose.

  4. Credit Limit - Enter the credit limit in numeric format of 15 digit maximum length. The credit limit recorded indicates the maximum credit amount for the selected Customer in the facility.

Local Address

Enter the following details:

 

Name & Address in Local Language

This tab page is enabled for entry, if the Local Language Reporting Required check box is selected.

  1. Long Name – Enter the long name of the Customer in local language in alphanumeric format of 60 characters maximum length. This name is printed in Statement of Accounts and Reminder Letters. This is a mandatory field.

  2. Short Name – Enter the short name of the Customer in local language in alphanumeric format of 15 characters maximum length.

  3. Address [Line 1 – 5] – Enter the address details of the Customer in local language in alphanumeric format of 60 characters maximum length for each line.  

  4. State – Click [List of Values] to select a State. The entries with code and its description get populated only when Regions are defined in the Master Patient Index Module.

Other Details

Enter the following details:

Other details

  1. Bank Name - Enter the Bank Name in alphanumeric format of 30 characters maximum length. This indicates the name of the Bank where the transactions take place between the respective Customer and the facility.

  2. Bank Acc No. - Enter the bank account number of the Customer in alphanumeric format of 20 characters maximum length.

  3. Email ID - Enter the bank E-mail ID in alphanumeric format of 30 characters maximum length.

  4. Alternate Address 1 - Enter the contact address of the Customer in five lines in alphanumeric format of 60 characters maximum length each. This address is an alternate  to the main address.  

  5. Alternate Address 2 - Enter the contact address of the Customer in five lines in alphanumeric format of 60 characters maximum length each, This address is an alternate  to the main address and Alternate Address 1.  

  1. Click [Commit Form] or press [F10] to save a created Customer or [Clear Record] to clear the entered or queried data.

  2. Click [Execute Query] or press [F8] to search for an existing Customer record within the facility.
