Enter the following to create Refund request:

 

  1. Doc Type - Select a valid Document Type from [List of Values]. This is a mandatory field.  

  2. Doc No. - Document Number is generated by the system, if it is defined for the entered document type.

  3. Date – Enter the date for document. This can be selected from the calendar list. Document date has to be less than or equal to system date.

  4. Customer - Select a valid Customer from [List of Values]. The list displays customer codes as created in the ‘Customer’ function under Masters. If the code selected is valid, the short description of that code is displayed for user verification. The description cannot be edited. This is a mandatory field.  

  5. Amount – Enter the Document Amount in numeric format of nine digits maximum length. This is a mandatory field.

  6. Reference – Enter the Reference in alphanumeric format of 15 characters maximum length. The default value is populated by the system if it is of allocated entry.

  7. Cheque No, Dated, Of Bank – These are non-enterable field in case the check box for the parameter ‘Cheque details as part of Refund entry’ field is not selected in the AR Parameters under Admin. The cheque detail is updated through ‘Customer Refund Query’ field. If the check box in parameter ‘Cheque details as part of Refund entry’ field is selected, the Cheque information can be entered directly.

  8. Bank Acct - Select the Bank Account Code from [List of Values]. This is a mandatory field.

  9. Desc – This is a display field. On entering Bank Account, Bank Account description is displayed.

  10. Month/Year – Enter the Month/Year in mm/yyyy format. This is a mandatory field. System displays the current processing month and year from AR parameters as default value.

  11. Narration – Enter the Narration in alphanumeric format of 40 characters maximum length.

 

Refund Lines

  1. Line Type - Select a valid Line Type from [List of Values]. The values listed are Pending, Direct and Advance. This is a mandatory field.  

  2. Doc Type – Enter the Document Type in alphanumeric format of six characters maximum length. This can be selected from [List of Values].

  3. Doc Num – Enter the Document Number in numeric format of eight digits maximum length. This cannot be entered if line type is "D" (Direct). If line type is "A" (Advance), the receipt document type and receipt document number is displayed. If line type is "P" (Pending), the outstanding document type and the document number must be entered. Click [List of Values] to view and select a valid outstanding document reference for that customer. If document type & Number to be allocated is a Consolidated Invoice document, bills need to be allocated from ‘Bill wise Allocation (Consolidated Invoice)’.

    If the Original document type/number to be allocated is a consolidated Invoice, the Patient bill details pertaining are displayed. Value of the ‘Select’ is selected, by default and the Outstanding Amount is defaulted as Allocated Amount. You need to clear the ‘Select’ bills, which is not required for allocation. You are allowed to change the allocated Amount at the bill level. Allocated Amount cannot be greater than the outstanding amount.

    Click [Commit Form] to save the entries.

  4. Amount - Enter the Amount in numeric format of nine digits maximum length. This can be selected from [List of Values]. If line type is "P" (Pending), the amount entered must not be greater than the pending amount of the outstanding document being allocated. If the amount entered is positive, it is shown under Credit Column. If the amount entered is negative, it is shown under Debit Column.

  5. Debit – This is a display field. When the value for amount entered is greater than zero, it is displayed as Debit.

  6. Credit – This is a display field. When the value for amount entered is less than zero, it is displayed as Credit.

  7. Posting Account – Select the Posting Account Code from [List of Values]. This is a mandatory field.

  8. Narration - Enter the Narration in alphanumeric format of 50 characters maximum length. The description for Refund can be entered as narration.

  9. Bill Doc. Date – You cannot modify the values in this field.

  10. Bill Type – You cannot modify the values in this field.

  11. Bill Number – You cannot modify the values in this field.

  12. Patient No - You cannot modify the values in this field.

  13. Outstanding Amt – Outstanding document Amount for the Bill. This is a display field.

  14. Allocated Amt – Value is Outstanding Amount. You are allowed to modify the data, by default.

  15. Select – To select or clear the Bill document for allocation.
