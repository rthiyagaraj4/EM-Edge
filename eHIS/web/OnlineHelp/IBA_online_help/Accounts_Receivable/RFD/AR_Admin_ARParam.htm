Enter the following details to define AR Parameters:

 

  1. Current Month/Year – Enter the Current Month/Year in numeric format of mm/yyyy. The Month and Year is associated with the month end processing. You cannot modify the month and year once the values are recorded. This period denotes the period when the Accounts Receivable module is implemented. The values set for the Current Month and Year is updated automatically to the next period during the month end processing. This period must always denote the fiscal month and year and not the calendar month and year. This is a mandatory field.

  2. AR to GL Interface – Select the value from the list. The listed values are ‘Batch’, ‘Online’ and ‘No Integration’. If ‘Batch’ is selected, transaction records are posted to General Ledger module periodically. If ‘Online’ is selected, individual transaction records are posted to General Ledger module whenever it is saved. If ‘No Integration’ is selected, individual transaction records are not posted to General Ledger module.

  3. Purge Doc After Months – Enter the value in numeric format of two digits maximum length. This function allows you to remove all fully settled documents from the database after the specified period. For Example, if the value entered is six, the fully settled records are available in the system for six months and deleted after six months.

 

Customer Code Parameters

  1. Customer Code Length – Enter the Customer Code length in numeric format of one digit maximum length. You cannot modify the recorded value. When you specify the Customer Code value, the system checks for the Code Length specified.

  1. Customer Code Scheme Type – Select the ‘Customer Code Scheme Type’ option. The options are ‘Numeric’ and ‘Alphanumeric’. If ‘Numeric’ is selected, you must enter the Customer Code in numeric format. If ‘Alphanumeric’ is selected, you can enter the Customer Code in combination of alphabets and numerals. You cannot modify the recorded value.

  2. Validity Period – Enter the Validity Period in numeric format of three digits maximum length. The Customer Code is considered valid for usage until the recorded period.

  3. Back Dated Validity Allowed – Enter the number of days in numeric format of three digits maximum length. The Customer Code is considered valid for usage preceding the current day.

  4. Reminder Frequency – Enter the Reminder Frequency period in numeric format of two digits maximum length. The reminder letters are delivered to the Customer based on the period recorded.

 

Ageing Analysis Slots in Days

  1. Ageing Analysis Slots in Days –Enter the Ageing Analysis period in the Slot1, Slot2, Slot3, Slot4 and Slot5 in numeric format of three digits maximum length. Subsequent slot values must be in incremental order.

 

Default Posting Accounts

  1. Bank A/c – Select the required Bank Account in Posting Account Code format from the [List of Values]. The displayed values are defined in the  Chart of Accounts function with Bank Code check box enabled.

The Bank Account Code is displayed while entering the receipt transaction in Receipts entry option in Cashier’s menu, by default.

  1. Suspense A/c – Select the required  Suspense Account in Posting Account Code format from the [List of Values]. The displayed values are defined in the  Chart of Accounts function.

The Suspense Account is displayed while entering receipt transaction in Receipts entry option in Cashier’s menu, by default.

  1. Mark Up A/c - Bills - Select the required Mark Up Account - Bills in Posting Account Code format from the [List of Values]. The displayed values are defined in the  Chart of Accounts function.

  2. Mark Up A/c - Inventory - Select the required Mark Up Account - Inventory in Posting Account Code format from the [List of Values]. The displayed values are defined in the  Chart of Accounts function.

  3. Mark Up A/c - CSSD - Select the required Mark Up Account – CSSD (Sterile Stock) in Posting Account Code format from the [List of Values].  The displayed values are defined in the  Chart of Accounts function

 

Default Transaction Document Types

  1. Invoices – Enter the required Invoices or select the required Invoice Code from the [List of Values]. The Document Types defined in the Financials Base module are displayed. The Invoice code set is considered as the default value, if no data is specified for Invoice in transactions.

  2. Debit Notes – Select the required Debit Note Code from the [List of Values]. The Document Types defined in the Financials Base module are displayed. The Debit Note code set is considered as the default value, if no data is specified for Debit Notes in transactions.

  3. Credit Notes – Select the required Credit Note Code from the [List of Values]. The Document Types defined in the Financials Base module are displayed. The Credit Note code set is considered as the default value, if no data is specified for Credit Notes in transactions.

  4. Adjustments – Select the required Adjustment Code from the [List of Values]. The Document Types defined in the Financials Base module are displayed. The Adjustment Code set is considered as the default value, if no data is specified for Adjustments in transactions.

  5. Receipts – Select the required Receipt Code from the [List of Values]. The Document Types defined in the Financials Base module are displayed. The Receipt code set is considered as the default value, if no data is specified for Receipts in transactions.

  6. Refunds – Select the required Refund Code from the [List of Values]. The Document Types defined in the Financials Base module are displayed. The Refund code set is considered as the default value, if no data is specified for Receipts in transactions.

  7. Ext. Sales Dispatch – Select the required Extended Sales Dispatch from the [List of Values]. The Document Types defined in the Financials Base module are displayed. The Extended Sales Dispatch code set is considered as the default value, if no data is specified for Extended Sales Dispatch.

 

Document Types

  1. Credit Note settlement Discounts – Select the required Credit Note Settlement Discounts from the [List of Values]. The Document Types defined in the Financials Base module are displayed. The Credit Note Settlement Discounts code set is considered as the default value, if no data is specified for Credit Note Settlement Discounts in transactions.

The Adjustment Unmark Bad Debts code set here is considered as the default value, if no data is specified for Adjustment Unmark Bad Debts in transactions.

 

Functional Parameters

  1. Online Posting of Official Receipts? – Select the check box to indicate that the Official Receipts can be posted online, instead of batch processing.  

  2. Cheque Details entry as part of Refund? – Select the check box to indicate that the details of cheque entry are part of Refund details.

  3. Allow Marking/Unmarking of Bad Debts? – Select the check box to allow you to mark or unmark the bad debts.
