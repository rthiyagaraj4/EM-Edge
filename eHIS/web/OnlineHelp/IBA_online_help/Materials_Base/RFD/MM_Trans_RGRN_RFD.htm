Enter the following additional information in the header section:

 

  1. PO No – Enter the Purchase Order Number against which the goods have been received in numeric format of 10 digits maximum length. If the Purchase Management module of Enterprise Management is operational, this number is validated against Purchase Order Number available in the system.

  2. Supplier – Select the Supplier from whom the goods are received from the list. This is a mandatory field.

  3. Currency – Select the foreign currency applicable in which the goods are received from the list. Presently this option is disabled.

  4. Receipt Date – The current system date is displayed, by default. Enter the Receipt Date in dd/mm/yyyy format or select the date using Calendar function. This is a mandatory field.

  5. Remarks - Click 'Remarks' hyperlink to record the replaced goods remarks. Select the pre-defined remarks using [?] function. The recorded remarks is printed in the GRN report and Reprint Document Report.

 

Enter the following details in the Item Entry Area:

 

  1. Item - Enter the required Item in alphanumeric format of 60 characters maximum length or select the Item using [?] function. The Items displayed are limited to the selected Store. This is a mandatory field.

On selection of an Item, the UOM associated to the Item displayed by default and the Batch Entry section is displayed, which allows entry of batch details.

  1. Normal - Enter the quantity received (against the Purchase Order) in numeric format of 12 digits maximum length. On saving the record, the quantity entered is in terms of Purchase Unit of Measure and the quantity in terms of stock Unit of Measure gets added to the Item's Quantity on Hand. When the Purchase Management module of Enterprise Management is operational, the Pending Purchase order quantity is reduced by the normal quantity received and the system ensures that normal quantity is not greater than the pending Purchase Order quantity. This is a mandatory field.

  2. Sample - Enter the number of Items in numeric format of 12 digits maximum length. This field is enabled if ‘Allow Sample Entry’ check box is selected in MM Parameter or this field is disabled.

  3. Unit Cost – Enter the cost of the Item for one unit, if known or once the Item is selected the Cost is displayed automatically, which is entered while defining the Item.

  4. Value – Enter the Value or the Value is displayed based on the Unit Cost. The value is the total value on the Items received. The Value calculated is: Unit Cost x Normal.

  5. Label – The check box is made default based on the value selected in ‘Item Master’ in Inventory Management module. On selecting the check box, bar code label is generated for the Item on applying.

  6. No of Label – The Label value is displayed based on the entered Normal value. You can enter the number of labels required in numeric format of 12 digits maximum length. If the quantity entered is in decimals, the Number of Labels will round off to the next number. This check box is enabled, if the ‘label’ printing is applicable.

  7. Replace on Expiry – Select the check box, if the Supplier agrees to replace the Item on expiry.

  8. Purchase – Select the Purchase UOM from the list when the goods are received other than the stock Unit of Measurement.

  9. Conv Factor/Rule – The Conversion Factor for Purchase UOM is displayed.

 

Batch Entry Area

This area gets displayed on selection of an Item and allows entry of batch details.

 

Enter the following Batch Details:

 

  1. Expiry Date – Enter the Expiry Date for expiry related Items in dd/mm/yyyy format or select the date using the Calendar function. For non-expiry related Items the current date is displayed as the receipt date and it is disabled.
