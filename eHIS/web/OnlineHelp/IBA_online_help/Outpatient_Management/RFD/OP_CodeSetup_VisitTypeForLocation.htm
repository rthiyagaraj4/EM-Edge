Enter the following details to create a Visit Type for Location record:

 

  1. Location – The Location has two components where you have to select a value from list and [?] function to identify the care location. The list is the Location Type and [?] function is used to select the location that come under the selected Location Type. The Location Type values listed are 'Clinic', 'Procedure Unit' and 'Daycare Unit'.  On selecting the required Location Type, the Locations listed are based on the following condition:

  2. All locations that are created for the logged in facility under the selected Location Type and have effective status as enabled.

  3. All the defined Locations must be associated to the Operator Station assigned to the logged in User.

  1. Applicable Visit Type Indicator - Select the Visit Type Indicator from the valid list of values. This field represents the Visit Type Indicator of the Visit Type, to be associated to the Location. The values displayed are based on ‘Applicable Visit Type’ set up at the time of creating the selected Location. This field is mandatory.

  2. Visit Type – Select the Visit Type from the list. This field represents the Visit Type, which has to be associated to the Location. The values displayed are based on Visit Types of the logged in Facility, pertaining to the selected ‘Visit Type Indicator’ as set in ‘Visit Type’ function. This field is mandatory.

When you select the Location as ‘Procedure Unit’ created as specific to Radiology then the Visit Type field displays only the Visit Type created as Applicable to Radiology.

  1. Vital Signs Battery – Select the Vital Signs Battery from the list. All the Discrete Measure of type ‘Vital Signs Battery’ set up in Application Masters module is displayed in the list. The associated Vital Sign Battery is used to record the Vital Signs of the patient during the outpatient encounter against the Location. This field is optional and if Vital Sign Battery is not defined at the level of Visit Type, the Vital Signs Battery set at either Location Level or OP Parameter Level is taken into consideration.

  2. Max Appt Duration (in minutes) - Enter the maximum appointment duration allowed for the selected Location and Visit Type in numeric format of three digits maximum length. The value set here indicates the maximum appointment time that the patient can be given for the defined Visit Type, when an Appointment Booking is made in the Appointments Scheduling module for time table type ‘Free Format’. This field is not applicable for visit type indicator ‘Emergency’. The system defaults values if the same has been set at Visit Type level in ‘Visit Type’ function.

  3. Max Appt Slots - Enter the Maximum Appointment Slots allowed for the selected Location and Visit Type in numeric format of three digits maximum length. The value set indicates the maximum appointment slots that the patient can be given when an appointment booking is made for a specific Visit Type in Appointment Scheduling module for time table of type ‘Slot’. This field is not applicable for Visit Type Indicator ‘Emergency’. The system defaults values if the same has been set at Visit Type level in ‘Visit Type’ function.

  4. Enabled - Select the check box to enable the record and clear the check box to disable the record.
