Enter the following details to create a Daycare Unit record:

 

  1. Code - Enter the Location code in alphanumeric format of four characters maximum length. A Location Code is to identify the Location and has to be unique and cannot be changed after creation. This field is mandatory.

  2. Type - Select the required Location Type from the list of valid values. All the system defined and user defined Care Location Types associated to Indicator ‘Daycare Unit’ in Application Masters module are listed. This field is mandatory.

  3. Long Description - Enter the long description of the Location in alphanumeric format of 30 characters maximum length. This field is mandatory.

  4. Short Description - Enter the short description of the Location in alphanumeric format of 15 characters maximum length. This field is mandatory.

  5. Physical Location - Enter the Physical Location of the Daycare Unit in alphanumeric format of 40 characters maximum length. A Physical Location is to identify the exact Location of the Daycare Location.

  6. Enabled - Select the check box to enable the record and clear the check box to disable the record.

 

Daycare Unit characteristics can be entered under two tabs, namely, ‘Characteristics’ and ‘Others’ tab.

 

Characteristics Tab

 

Working Days

  1. Working Days - A Location can function on all or specific days of the week. Select the appropriate check boxes to indicate the location working days. The Visit Registration and appointment processes can be used for a Location only on its working days. The default working days are selected based on the day of week defined in System Manager module.

 

Operating Time

  1. Start – Enter the operating start time of the location in time format of hh: mm. This field is optional.

  2. End - Enter the operating end time of the location in time format of hh:mm. This field is optional. Operating End time has to be greater than Operating Start Time.

  3. Base Slot/Slab Time – Base slot/slab time is used to identify the minimum unit of time frame to be used to book an appointment where multiple resources are involved by using multiples of the base time defined. Enter the base slot/slab time in time format of hh:mm This field is optional.

 

Controls

  1. Allow Visit Registration – Select the check box to Allow Visit Registration for the defined Location.

  2. Allow Walk-in – This field is enabled on selecting the ‘Allow Visit Registration’ check box. Select the check box, to allow walk-in patients to the defined Location.

  3. Allow Appointment - This field is enabled on selecting the ‘Allow Visit Registration’ check box for Daycare Unit with Level of Care set to ‘Ambulatory’. For Locations with Level of Care set to ‘Emergency’ this field is disabled. Select the check box to allow appointments for the defined Location. If the check box is cleared, the time table records cannot be set up for this Location in Appointment Scheduling module and appointments cannot be booked.

  4. Allow Referral - Select the check box to indicate if referrals are allowed in this location. This field is enabled on selecting the ‘Allow Visit Registration’ check box. Most Locations allow visit registration only on producing a referral identification number. If the check box is cleared, the Location does not support registration of visits for referral patients.

  5. Allow Non-Referral - This field is enabled on selecting the ‘Allow Visit Registration’ check box. Select the check box to indicate if non-referrals are allowed in this Location.

  6. Age Group - Select the Age Group of patients who are eligible to receive the services provided from the list of valid values. If this value is set up, the Visit Registration process restricts the patients based on their age and gender depending on the corresponding Age Group parameters set in ‘Other Master Setup’ of Application Masters module.

  7. Level Of Care - Select the Level of Care from the list of valid values. This defines the Level of Care provided by the Daycare Unit. Valid values are ‘Ambulatory’ and ‘Emergency’.  

If the Level of Care is set to ‘Ambulatory’ the applicable Visit Type Indicator check box, ‘First’, ‘Follow up’, ‘Routine’, ‘Series’ and ‘Consult’ are selected by default. Clear the required field if necessary.

If the Level of Care is set to ‘Emergency’ the ‘Emergency’ check box in applicable Visit Type Indicator is checked by default.

 

Others Tab

 

Acct Department/Service

  1. Specialty - Select the Specialty from the list of valid values. All the Specialties created in Application Masters module with enabled check box selected are listed. This field represents the Specialty, which is applicable for the Daycare Unit. This field is mandatory.

  2. Vital Signs Battery – Select the Vital Signs Battery from the list of valid values. All the Discrete Measures of type ‘Vital Signs Battery’ set up in Application Masters module are displayed in the list. The Vital Signs Battery associated with the Daycare Unit is used to record the Vital Signs of the patients arriving into the Daycare Unit.

Vital Signs Battery in Outpatient Management module is set at three different levels, namely, 'Daycare Unit Setup', 'Visit Type for Location Setup' and 'OP Parameter Setup'. Vital Signs Battery set up is not mandatory while defining the 'Daycare Unit' and 'Visit Type for Location,' but is mandatory while defining 'OP Parameter'. The system first checks for Vital Signs in the 'Daycare Unit setup', if it is not defined, the system defaults the battery setup in 'Visit Type for Location Setup'. If Vital Signs Battery is not defined in Visit Type, the system defaults the battery setup in 'OP Parameter'.

  1. Account Department - Select the department for the location from the list of valid values. The list of departments displayed is limited to the departments of the logged in facility as set in ‘Department in facility’ of Application Masters module. This field is mandatory.

  2. Section – Select the Section for the location from the list of valid values. All the Sections associated to the selected Accounting Department in Application Masters module are listed in the list.

  3. Service - Select the Service from the list of valid values. This field represents the Medical/Ancillary Service that is applicable to the Location. The list of Services displayed is limited to the services offered by the selected department of the logged in facility. This field is mandatory.

  4. MR Section Type – Select the MR Section Type from the values displayed, namely, ‘Central’ or ‘Departmental’. This field is enabled only if File Maintenance is set to ‘Multiple’ in MP parameter (Logged in facility) of Master Patient Index module or if File Maintenance is set to ‘Single’ in MP Parameter of Master Patient Index module, this field is disabled. The value ‘Departmental’ has to be selected when the Location maintains an exclusive MR Section for the patient records and the value ‘Central’ has to be selected when the Location uses a common centralized MR Section. The MR Section value is displayed based on what is set at the facility level first and if these values are not set at the facility level, the values set in MP Parameter are made default.

  5. MR Section – Select the MR Section from the list of valid values to be associated to the Location. This gets enabled only if MR Section Type is selected as ‘Departmental’. This field represents the Medical Records Section of the facility associated with the Daycare Unit. The list of values displayed is limited to the MR Sections of the logged in facility.

  6. FS Location – Select the File Storage Location to request for files from the list of valid values. These values are displayed only if the Patient File Tracking module is installed. On selection of MR Section Type as ‘Central’ and all those File Storage Locations with identity Daycare unit and belongs to the logged in facility are displayed. On selection of MR Section Type as ‘Central’ and all those File Storage Locations with identity department and belongs to the logged in facility are displayed.

  7. Default File Type – Select the applicable file type from the value displayed. The list of values is populated from the File Type function defined in Master Patient Index module and is applicable when File Maintenance is opted for single file maintenance in Master Patient Index.

  8. Auto File Request – Select the check box to automatically request for files from permanent MRD on Visit Registration to the defined Daycare Unit or the files has to be manually requested.

 

Applicable Visit Type Indicator

  1. Emergency/First/Follow-up/Routine/Series/Consult – Select these check boxes to indicate whether each of these Visit Types are applicable for this Location. If ‘Level of Care’ is selected as ‘Emergency’, the ‘Emergency’ Visit Type check box is selected by default.  Select the other Visit Types applicable. If the ‘Level of Care’ is set as ‘Ambulatory’, all Visit Types except the ‘Emergency’ Visit Type are selected by default. Select the ‘Emergency’ Visit Type if applicable.

 

Applicable Patient Class

  1. Outpatient/Inpatient/Daycare/External – The Location created is a Daycare Unit so the Patient Class applicable is selected to Day Care and disabled. All other Patient Classes are disabled.

 

Resource Privileges

  1. Primary Resource Class/Type – The Primary Resource Class is displayed as ‘Room’ and disabled.

  2. Open to All Resources – The system displays the check box as selected and disables the field to indicate the Daycare Unit is open to all primary resources of the facility.

  3. Check in at Identification – This field is disabled and displays Not Identifiable.

  4. Select Unassigned Patients - This field is disabled.

  5. Allow Consultation On – This field is disabled and displays Checked-In.

  6. Contact No – Enter the contact number of the day care unit in alpha numeric format of maximum 15 characters. Special characters +(Plus) and - (Hyphen) are only allowed.

 

Patient Instructions tab

 

  1. Serial No - Enter the Serial Number in numeric format of two digits maximum length. You can record a maximum of 10 Serial Numbers. This is a mandatory field.

  2. Instruction Description - Enter the Instruction Description in alphanumeric format of 200 characters maximum length or select the Instruction Description using [?] function. This is a mandatory field.

  3. Instruction ID -The corresponding Instruction ID is displayed on selecting the Instruction Description.

Click [Add] to display the selected Patient Instruction to the list at the bottom of the screen.

  1. Default - Select the check box to default the selected patient instructions while booking an appointment for the selected clinic.

Click on Serial Number hyperlink to delete the record. Click [Clear] to clear the selected details. Click [Delete] to delete the record from the list.
