Departed

Departed tab is used to record the patient departure from a care location. The list of patients is populated in Departed tab if the patients have an open and active encounter. If the Encounter Type is Outpatient, the patient has to be marked as Arrived using Manage Patient Queue function.

  1. Select a Location Type from the list. Based on the selected Location Type, select the required Location using [?] function.

  2. Click [Search] to display the departed patient details or enter the search criteria and click [Search] to select specific departed patients. Click [Clear] to reset the records displayed in the result area.

    Enter the following details:

    1. Location – Select the location to which patient arrival has to be marked from the list  provided. The list lists all location types and based on selection of location type, and  the lookup [?] lists all valid care locations that are derived based on the operator station applicable to the logged in user.

    2. Encounter - Select the Encounter type from list provided. The valid lists of values are ‘Outpatient’, ‘Inpatient’, ‘Daycare’ and ‘Emergency’. Based on the selection, enter the valid Encounter ID to filter the patient listing based on encounter ID.

    3. Patient ID - Enter the Patient ID in alphanumeric format of maximum length as specified in the MP Parameter. You can also query for the patient details by invoking the [?] function. On entering the relevant details in the lookup screen, the result screen will be populated with patient details that satisfy the entered query details. You can then select the patient by clicking on the hyperlink of the patient. The control now shifts back to the ‘Departed’ screen with the patient details populated.

    4. Encounter Date – Enter the date of encounter or select it using the calendar function to list all patients who have had encounters on the selected date. System displays current date by default and you can change it, if required.

    5. Order By – Select the order by from the list provided. This is to group the listing based on either Encounter date or Encounter Type.

  3. On searching, the encounter details are displayed based on the condition.

    • Patient should have an open and active encounter

    • If Encounter type is Outpatient, then patient should have been marked as Arrived using Manage Patient Queue.

    1. Encounter Date Time – The date and time of generation of the encounter is displayed.

    2. Encounter Type – The Type of Encounter is displayed.

    3. Encounter ID – The Encounter Identification Number is displayed. This field is a hyperlink, which allows selection of the patient for recording departed details.

    4. Patient ID – The Patient Identification Number is displayed.

    5. Patient Name – The Patient Name is displayed.

  4. Click Encounter ID hyperlink of a particular patient to display the ‘Departed’ screen.

  5. Enter the details in the mandatory (*) fields. <Related Field Description>

  6. Click [Apply] to confirm the departure of the patient or [Close] to exit from the screen. The control shifts back to Record Patient Movement screen.
