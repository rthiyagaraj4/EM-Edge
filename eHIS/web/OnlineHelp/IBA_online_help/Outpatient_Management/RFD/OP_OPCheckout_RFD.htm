Enter the following details to perform the Check out operation:

 

  1. Visit Type - Visit Type of the Visit is displayed and cannot be changed.

  2. Practitioner – If a Practitioner is assigned, the Practitioner Name and Practitioner type are displayed and cannot be changed. In case the Practitioner is not assigned, you can assign a practitioner and the displayed list is limited based on a similar logic adopted while performing visit registration. This field is mandatory if Practitioner is not yet assigned.

 

Disposition

  1. Date/Time – The current date/time is displayed as the disposition date and time. The date/time cannot be modified.

  2. Type - The Disposition Type defined in the OP Parameter gets displayed, by default. If no value is defined in the OP Parameter, the field displays as 'Select' for you to select the Disposition Type from the list of valid values. The displayed list is limited to a set of active Disposition Types valid for Visit Conclusion.

 

Case Definitions

  1. Medical - Select the check box to indicate whether the case requires only medical attention.

  2. Surgical - Select the check box to indicate whether the case requires surgical attention.

  3. Circumstance of Injury - Select the required value for Circumstance of Injury from the list. The list displays all the enabled records created in 'Circumstance of Injury' master of Application Masters module.

 

In the Event of Death

  1. Brought Dead – Select the check box to indicate whether the patient is deceased and has been brought dead to the facility. If the patient is already registered as a brought dead case in 'Register Attendance’ function, this field is disabled.

  2. MLC – Select the check box to indicate the patient is registered under Medico Legal Case, or clear the check box.

  3. Postmortem Required – Select the check box to indicate postmortem is required for the deceased body, or if post mortem is not required, clear the check box. This field is enabled only if the ‘Type’ selected is deceased.

Note: If the deceased / body part is going to be released to other facility, then performing postmortem will become optional, even if this check box has been selected.

  1. Deceased Date/Time – Enter the deceased date and time of the patient. If the ‘Type’ is selected as deceased, this field becomes mandatory.

  2. High Risk Body – Select the check box to indicate the deceased body is a high risk body. If it is not of high risk, clear the check box.

  3. Body Release Date/Time – If the ‘Type’ is selected as deceased, this field becomes mandatory and the current system date/time gets populated automatically.

 

Recall Details

If the Patient is required to come back to the clinic, then the following details are entered:

  1. Come back After - Click on the check box to indicate whether the Patient has to come back again on a future date. If it is selected, enter the number of days within which the patient has to come back in numeric format of three digits maximum length in ‘Day(s)’, ‘Week(s)’ or ‘Month(s)’. This field is mandatory on checking the ‘Come Back’ check box.

  2. Reason - Enter the Reason for recalling the Patient in alphanumeric format of 200 characters maximum length.

  3. Instructions - Enter the Instructions of Recall to the Patient in alphanumeric format of 200 characters maximum length.

  4. Book Appointment – This button is enabled on checking the ‘Come Back’ check box. Appointment is booked in advance for patients who are asked to come back on a future date using this button. On clicking the [Book Appointment], diary of the currently registered Location/Practitioner is invoked if schedule has been generated in ‘Appointment Scheduling’ module. The system also highlights the earliest available slot based on the come back period defined.
