Register Visit

From this function you can Register a visit for an Outpatient visiting a Location.

Register Visit function is used when a patient arrives at a registration desk as an Outpatient in a Location. Visits are registered for walk-in patients, referral patients, patients with appointment number and invited patients from waiting list. Patients visiting Locations are identified by two unique identifiers. The first is to identify the patient using Patient Identification Number (Patient ID) and the other is to identify the visit using an Encounter Identification Number (Encounter ID). An Encounter ID is generated on completing the process of Visit Registration. This function allows you to perform a Visit Registration and generate outputs required for managing the patient after registration. This function is used to Register Visits to Clinics and Procedure Units. The Register Visit screen has three tabs, namely, Invitation List, Appointment List and Add Visit. A patient file is created with appropriate file storage location. A relevant patient file label is also printed for the respective outpatient file.

 

Hospital and Professional Charges are displayed on recording the financial details of a patient. The patient payable amount towards Hospital charge as well as Doctor’s charge is displayed. Professional Charge is displayed along with the Hospital Charge, only if the billing details and visit registration rules are defined for the Clinic. The doctor’s charge has to be a part of the panel, if the Professional charge has to be set as zero. If the Professional Charge is set as a Service, you cannot set the doctor’s fee as zero. You can instead associate the Hospital Charges to make up the Professional fees, when the doctor does not charge the patient for the services.

Registering a Visit

From the OP Transactions, select the Register Visit function.

The Register Visit screen gets displayed.

 

Select patients for Visit Registration from the following tabs:

Invitation List

Appointment List

Add Visit

 

When a patient is selected for registering a visit, the system prompts with the following:

• If patient has any active encounter as an Inpatient, the system prompts a message ‘Patient is currently an Inpatient’.

• If patient has any outstanding amount, the system pops-up window for Credit Authorization which displays the outstanding amount.

• If in OP Parameter ‘External Interface for Privilege’ is selected, you can use the external interface for fetching details of the patient during registration and validate the privilege conditions while registering a visit for a patient. If cleared, you can proceed with normal registration. If the patient is eligible for privileges, you can register visit, or the system alerts you with a message.

• The patient must have a National ID or the system alerts the user with a message that ‘National ID cannot be Null’ even if the patient is eligible for coverage.

•An alert message is displayed listing the outstanding, unbilled or any installment due amount for patient's previous visits to the facility. This message is displayed only if the Credit Authorization parameters are setup using the 'BL Parameters' function in Patient Billing Manager module.

The system prompts an alert message on close of the Bill Settlement Details screen when the Settlement Amount is greater than 0.
