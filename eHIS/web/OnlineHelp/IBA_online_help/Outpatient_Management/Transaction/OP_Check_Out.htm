OP Checkout

From this function you can Close an active encounter of a patient and record the outcome details of the encounter.

Check out is a process of concluding an encounter against a Clinic at the end of the consultation. OP Check out function is used when the patient’s stay is complete and sent home after the treatment or is referred to another facility for further treatment. OP Check out function is used to close active encounter and allows concluding an active episode by issuing a Medical Certificate, recalling a patient back to the same Clinic by Booking an Appointment, Creating and Modifying Wait list and referring the patient to another provider for further treatment.

Concluding Outpatient Encounters

From the OP Transactions, select the OP Check out function.

The OP Check out screen gets displayed.

  1. Select the Specialty and click [Search] to display all the patients with commencement of consultation marked or enter the search criteria and click [Search] to select specific patients. Click [Clear] to clear the records displayed in the result area.

    1. Specialty  - Select the Specialty from the list.

    2. View (Single/Multiple) - Click on the required option to select the view required. If you select Single View, additional query criteria can be specified to restrict the Patient records. Single is selected, by default.

    3. Location Type - Select the Location Type and Location from the list of valid values. The Locations that are associated to the Operator Station are displayed.

    4. Practitioner - Select the Practitioner using [?] function. The Practitioners associated to the displayed Locations are displayed.

    5. Patient ID - Enter the Patient ID in alphanumeric format of maximum length as specified in MP Parameter.

    6. Gender - Select the required value for gender from the list. The valid values are 'Male', 'Female' and 'Unknown'.

    7. Encounter ID - Enter the Encounter ID in numeric format of 12 digits maximum length.

    8. Queue No - Enter the Queue Number in alphanumeric format of six characters maximum length.

    9. Date – The current date is displayed. It cannot be modified.

    10. Start Time – The Start Time is displayed as '00:00'. The patients registered from '00:00' hours onwards for the current day are displayed. It cannot be modified.

    11. Display – Select the display type as ‘By Visit Type’, ‘Previous Day Visits Only’ and ‘Include Check-out patients’ from the list. Based on the selection the records are displayed.

    12. Appointment Status - Select the required value for appointment status from the list. The valid values are 'Yet to come', 'On time', 'Late' or 'No show'.

  2. Click the ‘+’ hyperlink of the required patient from the search result displayed to view the Patient Movement details.

  3. Click ‘Patient Details’ hyperlink of a particular patient to display the Check out screen.

  4. Enter the details in the Checkout screen to perform the Checkout. <Related Field Description>

  5. Click [Certificate] in the Check Out screen to issue a medical certificate. Click [OK] to save the certificate, [Clear] to clear the details entered or [Cancel] to exit from the screen.

  6. Click [Create Waitlist] to create a wait list for the patient if schedule is not available. This button is enabled if the ‘Type’ in the Check Out screen is selected as a value with the ‘Use at Visit Conclusion’ check box selected in the Disposition Type function of the Application Masters module.

  7. Click [Modify Waitlist] to modify the details of Waitlist created.

    Wait List is modified on Patient’s request or due to other reasons. This function is used to view details and modify certain details of the wait list. This function allows for reallocating the location for which the wait listing has been assigned as well as reallocates the resource for the wait listing. Further, this function is used to postpone the preferred date and reassign a priority for the waitlist.

    To modify wait list, from the already created wait list record(s) click on ‘Modify Wait List’ hyperlink for the selected patient.

    Enter the Reason for Modification or select the reason using [?] function.

  8. Click [Conclude Episode] to conclude episode.

    If the Episode of the selected visit has to be concluded, click [Conclude Episode]. Episode cannot be concluded for patients who are asked to come back on a future date, in such cases the button is disabled. Conclude Episode screen displays the Service Start and End date of the current episode. You can enter the Notes pertaining to the Episode.

  9. Click [Record] to record the Checkout details or click [Cancel] to return to patient queue screen.  

  10. On clicking [Record], the system prompts a pop-up message if patient requires a referral registration. Click [OK] in the message dialog to register a referral and the Register Referral Details screen is displayed. Click [Cancel] to return to patient queue screen. The prompt message is populated only if the ‘Prompt for Register Referral on close consultation’ check box is selected in the OP Parameter function. If the ‘Prompt for Register Referral on close consultation’ check box is cleared in the OP Parameter, the control goes back to the OP Check out screen.  

  11. Enter the details in the mandatory fields in Register Referral Details screen. Click [Clear] to clear the entered details if required.

  12. On entering the details, click [Record] to add the patient as referral and Referral Details are displayed. Click [Clear] to clear the displayed details. Click [Refresh] to refresh the screen and on completion of registration of referrals, click [Close] to return to the patient queue screen.

    1. Referral ID - The Referral Identification Number generated for referral is displayed and hyperlinked. You can view the referral details on clicking hyperlink.

    2. Preferred On – The preferred date of referral is displayed.

    3. Referred To – The destination facility of referral is displayed.

    4. Specialty – The Specialty name to which referral has been recorded is displayed.

    5. Location - The Location to which referral has been recorded is displayed.

    6. Practitioner - The name of the practitioner against whom referral has been recorded is displayed.

    7. Appt. No – If appointment has been confirmed using the referral identification number, the Appointment Number is displayed. You can proceed with booking an appointment using the [Book Appt] hyperlink.

 

1. This function is invoked from ‘Manage Patient Queue’ or ‘OP Check out’ function from the menu. This option is provided for the users who do not have access to ‘Manage Patient Queue’ function to record the Check out details.

Check out or Close Consultation is valid only for the mother encounter of a package patient with single encounter or multiple consultations.

2. When you click the [Record] button, the system will check for the minimum data sets (MDS) for the encounter based on the configuration. If the MDS criteria are not met, the system will display the message “Criteria not met” with Check hyperlink. The [Record] Button will be disabled. You can click the Check hyperlink and see the MDS values which have not been met in the Minimum data set compliance dialog. Once you complete the required MDS, the [Record] Button will be enabled.
