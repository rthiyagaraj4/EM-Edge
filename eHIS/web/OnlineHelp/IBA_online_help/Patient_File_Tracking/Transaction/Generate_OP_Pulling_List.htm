Generate OP Pulling List

From this function you can Generate the main and supplementary lists of patients expected to visit the facility.

The permanent File Storage Location identifies files of the patients prior to the appointment date and issues them to the concerned locations (Clinics/Day Care Units/Procedure Unit) based on the parameter set up. The list is generated irrespective of the file status. The File Status are - 'I' - Inside, 'O' - Outside, 'L' - Lost, ' A' - Archived, 'T' - Transit, 'E' - in Other facility and 'R' - 'Returned'.  When the list is generated for the first time, it is considered as the Main list. Subsequent list(s) generated is categorized as Supplementary list. The list can be generated until the files are tracked to respective locations. Once the files are auto-tracked, this function cannot be invoked.

The list is generated based on Location, Resource Class and Type. You must have appropriate rights to invoke this function.

Generating OP Pulling List

From the FM Transactions > Track Out to OPD, select the Generate OP Pulling List function.

The Generate OP Pulling List function screen gets displayed.

  1. Select the FS Location from the list. The listed locations are populated based on the user access rights. This field is mandatory.

  2. The name of the person generating the list is displayed. In the FM Parameter, if Security Access to be Enforced is applicable, the logged in User Name is displayed and disabled. In the FM Parameter, if Security Access to be Enforced is not applicable, the field is enabled and you can select the User Name using [?] function.

  3. On selecting the FS location the query criteria screen gets displayed.

  4. Enter the details in the mandatory (*) fields. <Related Field Description>

  5. Click [Generate] to include the files in the pulling list or [Reset] to clear the entered data.

The user is informed about the number of files included in the pulling list.

The generated pulling list provides the files pulled (first time pulling) as Main and subsequent files pulled as Supplementary. You cannot select Main for the Pulling List Type after first pulling list is generated. The system pops-up an alert message.

Click [OK] and select the Pulling List Type as Supplementary to generate the pulling list. You can generate the pulling list any number of times and all the files pulled after generation of first pulling list is displayed with type as Supplementary.

The system identifies only the active volumes of the File Types.
