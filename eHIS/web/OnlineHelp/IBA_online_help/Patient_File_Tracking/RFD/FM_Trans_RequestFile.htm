Enter the following details:

 

Request From

  1. FS Location Identity - Select the FS Location identity from the list. The values displayed are Clinic, Nursing Unit, Day Care Unit, Procedure Unit, Practitioner and Department.

  2. FS Location - Select the required FS Location from the list of valid values. The list displays all the FS Locations (Permanent, Non-permanent and Temporary locations) for the selected Identity and user rights defined. This field is mandatory.

  3. If the 'Security Access to be Enforced' check box in FM Parameter function is selected, only those File Storage Locations of the selected Identity and to which the logged in user has access are displayed in the list.

  4. If the 'Security Access to be Enforced' check box in FM parameter function is cleared, all the File Storage Locations associated with the logged in facility of the selected identity are displayed in the list.

  1. Requesting - Based on the selected identity, the field label is displayed. Enter the required location name in alphanumeric format of 30 characters or use [?] function to search for the requesting location. The [?] function displays locations corresponding to the selected identity and associated to the selected FS Location. Select the required location. This field is mandatory.

  2. Reason for Borrowing - On selecting the File Storage Location, the narration specified for the selected FS Location is displayed. Select the applicable reason for borrowing from the list. This field is mandatory.

  3. Name - The logged in user name is displayed, by default. The person requesting for the files on behalf of the selected FS Location is specified in this field.

  4. If the 'Security Access to be Enforced' check box in FM parameter function is selected, the user name cannot be changed and [?] function is disabled.

  5. If the 'Security Access to be Enforce' check box in FM parameter is cleared, the user name is defaulted to the logged in user but the other names can be selected using [?] function.

  1. Remarks – This field appears only when ‘Others’ value is selected in the ‘Reason for Borrowing’ field. Enter the remarks in alphanumeric format up to 100 characters except these four " '~` special characters. But you can enter these four special characters in the text box by copy and paste function. This is a mandatory field.

 

Request To

  1. Facility - The field is displayed based on the requesting FS Location.

  2. If the request for file(s) is made by non-permanent/temporary FS Location, the user can select only the Logged in facility. The non-permanent FS Locations can request for files only from the Permanent Locations of the logged in facility.

  3. If request for file(s) is/are made by the permanent FS locations, the user can select either Logged in facility or Enterprise facility from the list. The permanent FS locations are allowed to request for files from other facilities of the enterprise, provided, they have the access rights to request across facilities of an enterprise. If access rights are not defined, they cannot request for files across the facilities of an enterprise.

  4. If 'Request To' is selected as 'Enterprise Facility', select the required facility name from the Facility list displayed. This field is mandatory. The list contains facility names of the enterprise.

On entering the Request From and Request To details, the required patient file details are to be entered. Enter the following details for requesting file(s) of a patient.

  1. Patient ID - Enter a valid Patient ID in alphanumeric format of length as specified in Master Patient Index module or use [?] function to select the required patient. This field is mandatory.

  2. File Type - On selecting the Patient ID, the File Type(s) associated to the patient is displayed. If more than one File Type is associated, select the required Type from the list of values. On selecting the Patient ID, the default File Type associated in the File Type For FS Location is displayed. This field is mandatory.

  3. Volume No - The active Volume no is displayed, by default. Select required Volume Number from the list.

  4. Print Patient File Request Slip – Select the ‘Print Patient File Request Slip’ check box to print the Patient file request slip for Walk in patients. Clear the check box, not to print the patient file request slip for the selected criteria. This check box is selected or unselected by default according to setup made in FM Parameter.

If File Maintenance is set as Multiple and Numbering Control is set to Keep File Number same as Patient ID in Master Patient Index module, MP Parameter function, the File Type field is displayed. If Numbering control is Allow separate MRN and File Maintenance is set as Multiple, the File Number is displayed.
