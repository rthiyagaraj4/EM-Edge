Issue File

Files selected for issue are displayed as a consolidated summary list under each requesting FS Locations on click of Issue File Tab. The summary list includes files requested electronically and manually. Issue of files against manual request is displayed with the (#) indicator.

On clicking Issue File tab, the complete list of files that are selected for issue is displayed. Respective color codes are assigned for files Issued within facility and across facilities.

The (#) against the file selected for issue indicates that the file was requested manually by the FS Location.

You can clear the Issue check box for the required files. These files are removed from the issue list and re-populated under the requested file details.

  1. Click Issue File tab to view the selected files for issue (includes files requested both electronically and manually) for each requesting FS Location.

  2. The Return Date of the file is displayed in date format of dd/mm/yyyy. The return date is calculated based on the return criteria set for the FS Location in the function File Storage Location. Enter the Return Date in date format of dd/mm/yyyy if Return Date is not set for the FS Location.

  3. The ‘Issue?’ check box is selected, by default. On clearing the ‘Issue?’ check box, the file is removed from the list.

  4. Click [Mode of Transport] or 'Mode of Transport' hyperlink against the patient file to display the ‘Mode of Transport’ screen. Select the Mode of Transport from the list. If ‘Mode of Transport’ is manual, enter the name of the person carrying the file. This field is mandatory. Files listed for issue against the manual request is indicated by #.
