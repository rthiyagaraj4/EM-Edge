Enter the following details:

  1. Unit Number – The Unit Number of the blood unit transfused to the patient is displayed and disabled.

  2. Check By Practitioner – Click [?] to select the practitioner who observed the transfusion process. This is a mandatory field.

  3. Transfuse By Practitioner – Click [?] to select the practitioner who transfused the blood unit to the patient. This is a mandatory field.

  4. Practitioner ID – Click [?] to select the Practitioner ID of the practitioner who transfused the blood unit. The list displays the values defined in the ‘Practitioner’ function of Application Masters module. The Practitioner ID must be the logged in Practitioner’s ID.

  5. Start Date Time – Enter the Start Date and Time or click the Calendar function to select the transfusion Start Date and Time. This is a mandatory field.

  6. End Date Time – Enter the End Date and Time or click the Calendar function to select the transfusion End Date and Time. This is a mandatory field.

  7. Volume – The transfused volume of the blood unit is displayed. You can modify the details of this field.

  8. Storage Location – Click [?] to select the Storage Location of the blood unit that is transfused. The displayed values are defined in the Storage Location function of Blood Transfusion Services module.

  9. Description – Click [?] to select the reaction code. The list displays the values defined in the Transfusion Reactions function in Blood Transfusion Services module. On selecting the reaction code, the description of the reaction code is displayed. If selected reaction is an adverse reaction, a warning message ‘Please Order Transfusion Reaction Investigation’ is displayed. Click [OK] and sample is sent to the Laboratory for investigation.

  10. Reaction Code – On selecting the reaction code from the description code, the actual code is displayed.

  11. Adverse Reaction – This check box is selected and disabled, if the reaction is an adverse reaction. The reaction is specified as an adverse reaction in Transfusion Reactions function of Blood Transfusion Services module.

  12. Transfusion Remarks – Enter the Transfusion Remarks in free text format.

  13. Discontinue? – Select the check box, to discontinue the blood transfusion. The transfusion is discontinued, if the patient has serious reactions while consuming the blood units.

  14. Partial Transfusion Reason & Description – Click [?] to select the Partial Transfusion Reason. The list displays the values defined in Reason function of Blood Transfusion Services module. The description for the reason code selected is displayed.

Click [Apply] to save the blood consumption details or [Cancel] to cancel the transaction. On recording the blood consumption details, the status is changed to ‘Transfused’ and the ‘Sel’ check box is selected and disabled in the Blood Unit Consumption screen.

Blood Unit Consumption

From this function you can Record the consumption details of blood units after the blood is transfused to the patient.

You can search for the issued blood units for a selected patient and record the transfusion details of the unit. You can also view the Antibody details, Transfusion History of the patient and Transfusion requirements recorded in Allocate/Xmatch/Issue - Blood Units function in Blood Transfusion Services module.

Recording Blood Unit Consumption details

From the CA Transactions > Clinician Access > Desktop Menu > Pending Activities > Tasks, select the Blood Unit Consumption function.  

The Blood Unit Consumption screen gets displayed.

  1. The Issued Blood Units details such as Specimen Number, Unit Number, Product Code, Blood Group, Rhesus factor, Reserved date, and Expected Transfusion date are displayed.

  2. Select the check box to record the consumption details of the blood unit. On selecting the check box, the Transfusion Dialog screen is displayed.

  3. Click [Antibodies] to view the Antibody details recorded for the patient. The Antibody details are recorded in ‘Blood Unit Entry’ or ‘Rectify Blood Units’ functions in Blood Transfusion Services module.

  4. Click [Transfusion History] to view the previous transfusion details of the patient, if any.

  5. Click [Requirement] to view the transfusion requirements recorded for the selected patient. The transfusion requirements are recorded in Allocate/Xmatch/Issue - Blood Units function in Blood Transfusion Services module.

The Specimen Number of the issued blood unit is displayed with the following colour indicators.

- Autologous Units

- Directed Units

- Units Issued in Emergency

No Color – Normal Issues
