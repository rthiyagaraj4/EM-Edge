Enter the following details to define Product Storage Locations:

 

  1. Product Code - Click [List of Values] to select a Product Code. The list displays the values defined in the ‘Product Codes’ function. The description of the selected Product is displayed in the adjacent field. This is a mandatory field.

  2. Source Type – Select the Source Type from the list. The system-defined values are ‘BTS-Department’, ‘Ward’ and ‘Clinic’. The field ‘BTS-Department’ is displayed, by default. This is a mandatory field.

  3. Source Code – Click [List of Values] to select the Source Code. Enter the ward name if ‘Source Type’ is ‘Ward’ or ‘Clinic’ if the ‘Source Type’ is ‘Clinic’. ‘BTS’ gets displayed, by default, if ‘Source Type’ is ‘BTS’. The list of values are available for selection for ‘Ward’ and ‘Clinic’. The values created through ‘Source Types’ function in the Laboratory module gets listed. The description of the selected Source is displayed in the adjacent field. This is a mandatory field.

  4. Blood Group – Click [List of Values] to select a Blood Group. The values created through ‘Blood Group’ function in the Medical Records module get listed. This is a mandatory field.

  5. Rhesus – Click [List of Values] to select a Rhesus code. The values created through ‘Rhesus Codes’ function in the Medical Records module get listed. This is a mandatory field.

  6. Event – Select the Unit Event for which storage location is being identified. The system-defined values are Unit Awaiting Screening, Unit Available, Unit Allocated, Unit Disposed (Temporary), Unit Disposed (Permanent), Unit Xmatched and Unit Issued, Unit Returned and Unit Transfused. This is a mandatory field.

  7. Storage Location Code – Click [List of Values] to select a Storage Location Code. The values defined in the ‘Storage Location Code’ function get listed. If the above combination of values are already defined, you cannot create a duplicate record. This is a mandatory field.

  8. Description – The Description of the selected Storage Location is displayed.

  9. Default Location Y/N – Select the check box, if the Storage Location is to be identified as the default storage location for the above combination. You can select only one Storage Location as the default Storage Location. If no default location is selected, you have to select the storage location during the event.
