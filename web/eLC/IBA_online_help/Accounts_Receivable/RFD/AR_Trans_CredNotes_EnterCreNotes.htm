Enter the following details to generate a Credit Note:

 

  1. Doc Type/No – Click [List of Values] to select the required Document Type Code. The Document Type Code set for Credit Notes in AR parameter is displayed, by default. If the specification for the document type states that the document number is system generated, the document number is generated when transaction is saved. When system generates document number, field is protected from entering. This is a mandatory field.

  2. Date – Enter the date in the format of dd/mm/yyyy or select the required date using the Calendar function. The system date is displayed as Invoice Date, by default. Document date has to be less than or equal to system date. This is a mandatory field.

  3. Customer – Enter the Customer Code in alphanumeric format of eight characters maximum length or select the required Customer Code from [List of Values]. The Customers defined in the Customer function are displayed for selection. If the code selected is valid, the Customer Name is displayed as read-only text for user verification. When the customer is of miscellaneous type, click on Name and Address button and a screen gets displayed for entering details. This is a mandatory field.

  4. Doc Amt – Enter the Document Amount in numeric format of nine digits maximum length. This is a mandatory field.

  5. Month/Year – Enter the Month/Year in the mm/yyyy. The current processing month value mentioned in AR parameters is displayed, by default. This value can be updated. This is a mandatory field.

  6. Balance Amt – On entering the value for Document Amount, Balance Amount value is displayed. This field cannot be updated.

  7. [Name/Addr] - Click [Name/Addr] to record the miscellaneous Customer details. The button is enabled only if the Customer is a miscellaneous Customer.

 

Credit Note Details

  1. Orig Doc Type/No - Enter the Document Type in alphanumeric format of six characters maximum length or select the required Document Type from [List of Values]. If Original document & Number to be allocated is a consolidated Invoice document, the bills need to be allocated from the pop-up ‘Bill wise allocation (Consolidated Invoice)’. This is a mandatory field.

  2. Reference – Enter the relevant Reference in alphanumeric format of 15 characters maximum length. The default value is populated by the system if it is of is allocated entry.

  3. Payment Terms - On selecting the Customer, payment terms details set for the selected Customer in Customer Master is displayed. This can be changed. You can select the Payment Terms from [List of Values]. This is a mandatory field.

  4. Base Date – Click the Calendar function to enter a valid Base date. Base date is used to calculate the due date and the settlement date. After calculation, both are displayed. This cannot be changed, if entry is allocated. This is a mandatory field.

  5. Due Date – On selecting the Base Date, the Due Date is calculated automatically based on the value for Due Date entered in “Payment Terms” screen, by default. You can set another date as Due Date.

  6. Settlement Date – On selecting the Base Date, the Settlement Date is calculated automatically based on the value for Settlement Date entered in “Payment Terms” screen, by default.

  7. Settlement Percentage – On selecting the Settlement Date, the Settlement Percentage is displayed, by default. You can enter another Settlement Percentage in numeric format of three digits maximum length. It is mandatory to enter the Settlement Percentage if Settlement Date is entered.

  8. Gross Amount – Enter the Gross Amount in numeric format of nine digits maximum length. This is a mandatory field.

  9. Disc Amt – Enter the Discount Amount in numeric format of nine digits maximum length.

  10. Mkup Amt – This is a display field. The value for Markup Amount is derived from the Markup percentage entered in ‘Payment Terms’ screen. The Markup value is calculated on the amount derived from Gross Amount minus Discount Amount and displayed as read-only text.

  11. Pay Amount – This is a display field. The value for Payment Amount is derived from Gross Amount less of Discount amount and addition of markup amount, that is,

Payment Amount = Gross Amount – Discount Amount + Markup Amount

    3. Billing Group – Click [List of Values] to select a valid Billing Group. The entries with code and its short description get populated only when valid Billing Groups are set initially in the Patient Billing Manager module. This is a mandatory field.  

    4. Patient No – Click [List of Values] to select a valid Patient Number. The entries get populated from the Master Patient Index module. On specifying the Patient Number, Patient Name is displayed. This is a mandatory field.

    5. Bill Type - Click [List of Values] to select a valid Bill Type. The entries with code and its short description get populated only when valid Billing Types are set initially in the Patient Billing Manager module. This is a mandatory field.

Credit Note Lines

  1. Posting Account – Select the Posting Account Code from [List of Values]. This is a mandatory field.

  2. Amount – Enter the Amount in numeric format of nine digits maximum length. This is a mandatory field.

  3. Debit – This is a display field. When the value for amount entered is greater than zero, it is displayed as Debit.

  4. Credit – This is a display field. When the value for amount entered is less than zero, it is displayed as Credit.

  5. Narration - Enter the Narration in alphanumeric format of 50 characters maximum length. The description for invoice line can be entered as narration.

  6. Total Debit/Credit - System calculates the total debit or credit amount for the respective Customer.

You are not allowed to modify/delete any values for the Credit Notes generated from other modules.
