Enter the following details to generate an Invoice:

 

  1. Doc Type - Select the required Document Type Code from [List of Values]. The Document Type Code set for Invoices in AR Parameter is displayed, by default. This is a mandatory field.

  2. Doc No – Enter the Document Number in numeric format of eight digits maximum length. If the specification for the Document Type states that the document number is system generated, the document number is generated when transaction is saved. This field is disabled if the system generates document number.

  3. Date – Enter the date in the format of dd/mm/yyyy or select the required date using the Calendar function. The system date is displayed as Invoice Date, by default. This is a mandatory field.

  4. Customer – Enter the Customer code in alphanumeric format of eight characters maximum length or select the required Customer Code from [List of Values]. The Customers defined in the Customer function are displayed for selection. If the code selected is valid, the Customer Name is displayed as read-only text for user verification. This is a mandatory field. If ‘Misc Customer’ check box is selected for the selected Customer, the [Name/Address] gets enabled. Click [Name/Address] to invoke Miscellaneous Customer’s Details screen.

  5. Reference – Enter the relevant Reference in alphanumeric format of 15 characters maximum length.

  6. Payment Terms – On selecting the Customer, payment terms details set for the selected Customer in Customer Master is displayed. This can be changed. You can also select the Payment Terms from [List of Values]. This is a mandatory field.

  7. Base Date – On selecting the Payment Terms, the current system date is displayed as Base Date, by default. You can set another date as Base Date. This field is considered as base to calculate due date. The Document Date cannot be greater than Base Date. This is a mandatory field.

  8. Due Date – On selecting the Base Date, the Due Date is calculated automatically based on the value for Due Date entered in ‘Payment Terms’ screen, by default. You can set another date as Due Date. This is a mandatory field.  

  9. Settlement Date – On selecting the Base Date, the Settlement Date is calculated automatically based on the value for Settlement Date entered in ‘Payment Terms’ screen, by default.

  10. Settlement Percentage – On selecting the Settlement Date, the Settlement Percentage is displayed, by default. You can enter Settlement Percentage in numeric format of three digits maximum length. It is mandatory to enter the Settlement Percentage if Settlement Date is entered.

  11. Consolidated Invoice – Select the check box, if the Invoice is to be displayed as Consolidated Invoice. In case Invoice is Consolidated, on clicking [Patient Bills], the system displays ‘Patient Related Bill Details (Consolidated Invoice)’ screen. You cannot modify the data. If the Invoice is a Single invoice, ‘Patient related Bill details (Single Invoice)’ is displayed.

    When the Invoice entry is unallocated type and if the customer is patient related, the following details are entered.

    1. Doc Type/No – Enter the Document Type in alphanumeric format of six characters maximum length or select the required Document Type from [List of Values]. This is a mandatory field. Enter the Document Number in numeric format of eight digits maximum length.

    2. Date – Enter the date in the format of dd/mm/yyyy or click the Calendar function to enter a valid date. The date has to be less than or equal to the system date. This is a mandatory field.

    3. Billing Group – Select the required Billing Group from [List of values]. The entries with code and its short description get populated when valid Billing Groups are set initially in the Patient Billing Manager module. This is a mandatory field.  

    4. Patient No – Select the required Patient Number from [List of Values]. The entries get populated from Master Patient Index module. On specifying the Patient Number, Patient Name is displayed. This is a mandatory field.

    5. Bill Type - Select the required Bill Type from [List of Values]. The entries with code and its short description get populated only when valid Billing Types are set initially in the Patient Billing Manager module. This is a mandatory field.

    In case of consolidated Invoice generation, from the Patient Billing Manager module, the individual bill information such as Billing Document Type/Number, Document Date, Billing Group, Patient ID, Amount and Bill Type Code are displayed. You are not allowed to change the values.

    Invoice Lines

    1. Posting Account – Select the Posting Account Code from [List of Values]. This is a mandatory field.

    2. Amount – Enter the Amount in numeric format of nine digits maximum length. This is a mandatory field.

    3. Debit –When the value for amount entered is greater than zero, it is displayed as Debit.

    4. Credit –When the value for amount entered is less than zero, it is displayed as Credit.

    5. Narration - Enter the Narration in alphanumeric format of 50 characters maximum length. The description for Invoice line is entered as narration.

    If the Original Document Type/Number to be allocated is a consolidated Invoice, the Patient Bill Details pertaining are displayed. If the ‘Select’ check box is selected the Outstanding Amount is defaulted as Allocated Amount, by default. You need to clear the ‘Select’ bills, which is not required for allocation. You are allowed to change the Allocated Amount at the bill level. The Allocated Amount can be greater than the outstanding Amount.

    After selecting the bill document, click [Accept] and the Total Allocated Amount is copied to Allocated Amount in the header block. Click [Cancel] to cancel all the changes and to allocate new document for allocation. The Allocated Amount, which is copied, is not allowed to modify.

    1. Bill Doc Type – You cannot update the data.

    2. Bill Doc Number – You cannot update the data.

    3. Bill Doc Date – You cannot update the data.

    4. Billing Group – You cannot update the data.

    5. Patient No - You cannot update the data.

    6. Amount – You cannot update the data.

    7. Bill Type Code - You cannot update the data.

  12. Gross Amount – Enter the Gross Amount in numeric format of nine digits maximum length. This is a mandatory field.

  13. Disc Amt – Enter the Discount Amount in numeric format of nine digits maximum length.  

  14. Markup Amt – The value for Markup Amount is derived from the Markup percentage entered in ‘Payment Terms’ screen. The Markup value is calculated on the amount derived from Gross Amount minus Discount Amount and displayed as read-only text.

  15. Rec Amt – The value for Received Amount is derived from Gross Amount less of Discount amount and addition of markup amount, that is,

Received Amount = Gross Amount – Discount Amount + Markup Amount.

The Received Amount is calculated and displayed as read-only text.

  1. Post Account – Select the Posting Account Code from [List of Values]. This is a mandatory field.

  2. Description - On entering valid Posting Account, Description is displayed as read-only text.

  3. Month/Year - Enter the Month/Year in mm/yyyy format. The current month value mentioned in AR Parameter is displayed, by default. The value can be updated. This is a mandatory field.

  4. Narration – Enter the Narration in alphanumeric format of 50 characters maximum length.

You are not allowed to modify/delete any values for the Invoices generated from other modules.
