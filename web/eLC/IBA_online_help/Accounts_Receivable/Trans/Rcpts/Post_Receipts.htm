Post Receipts

From this function you can Post error free Receipts to the General Ledger interface.

This function is used to post error free Receipt entries. System carries certain validations while posting the debit notes. The documents are posted to the General Ledger interface. The Document Type in the General Ledger interface has to be of type receipt. You cannot update the record after posting.

• Validation of each selected document is done and all errors encountered are recorded. Error free documents are posted. If Accounts Receivable to General Ledger interface is "Online", the parallel updating is done in General Ledger module also. When Accounts Receivable to General Ledger interface is in the status of "None" or "Batch", the General Ledger module is not updated.

• The erroneous documents (those which could not be posted) are printed together with the list of errors, if any. Such documents can be recalled through the entry option and corrected.

• If the document required printing before posting, such documents are not posted.

Posting a Receipt

From the Transactions > Receipts, double-click the Post Receipts function.

The Post Receipts screen gets displayed.

  1. Select a valid Document Type from [List of Values]. Both the values are optional. When both values are entered, ‘TO’ value must be greater than or equal to the ‘FROM’ value.

  2. Enter the range for Document Number. Both the values are optional. When both values are entered, ‘TO’ value must be greater than or equal to the ‘FROM’ value.

  3. Click Calendar function to select a valid date range. Both the values are optional. When both values are entered, ‘TO’ value must be greater than or equal to the ‘FROM’ value.

  4. Click [Run Process] to view, print or save the adjustment details in a file in the local machine.   

  5. A log file gets generated. Click [OK] to view the log file or [Cancel] to exit from the screen.
