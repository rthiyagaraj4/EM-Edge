Enter the following details to create a new Note Type record:

 

  1. Code - Enter the code in alphanumeric format of four characters maximum length. This code is to identify the Note Type. The entered code must be unique and cannot be changed after creation. This field is mandatory.

  2. Description - Enter the description for the note type in alphanumeric format of 30 characters maximum length. This field is mandatory.

  3. Note Group - Select a note group from [?] function. This field is mandatory.

  4. Enabled – Select the check box to enable the record and clear the check box to disable the record.

  5. Class - Select an event class from the list of values provided. This field is mandatory. The event class values are 'Physician', 'Nursing' and 'Clinician'. This field is mandatory.

  6. Applicability - Select the appropriate check boxes to indicate the applicability of the Note Type.

  7. IP - Select the check box to make the Note Type applicable to Inpatient encounters.

  8. OP - Select the check box to make the Note Type applicable to Outpatient encounters.

  9. DC – Select the check box to make the Note Type applicable to Daycare encounters.

  10. EM – Check the check box to make the Note Type applicable to Emergency encounters.

  11. XT - Check the check box to make the Note Type applicable to External encounters.

  1. Service/Facility Specific - Select the check box to associate the Note Type with one or more service or facility.

  2. Document Linked Note – Select the check box to link any document to the Note during the transaction.

  3. Print Logo Header - Select the check box to print the Logo Header in Note print outs. Clear the check box and the Logo Header is not included in the Note print outs. By default, this field is cleared.

  4. Print Notes on Sign - Select the required radio button to print the notes on signing. If Print on Confirm is selected, the Note Type is printed only on confirming the Print. If Print Directly is selected, the Note Type is directly printed without confirming. If None is selected, select the records from the Search option to select the Note.

  5. Publish Level - Select either Authorized or Unauthorized from the list. This determines the status based on which the care providers can view the notes. If the user selects the publish level as 'Unauthorized', the notes generated for the selected note type can be viewed by other users even if the note status is unauthorized. If the publish level status is marked as 'Authorized', the notes generated for the selected note type has to compulsorily be authorized to be viewed by the other care providers. This field is mandatory.

  6. Amendment Type - Select one of the values 'Amend Original Section', 'Create New Addendum' or ‘Not Allowed’ from the list of valid values. This determines the amendment type that is possible to the specific note type. This field is mandatory. This is used while modifying a note type. If the user selects 'Amend Original Section' then changes are made in the existing document and saved and if 'Create new addendum' is selected, the modifications made to the note type is saved as an addendum to the existing document with changes incorporated. If ‘Not Allowed’ is selected, the changes cannot be made to the document.

  7. Disclaimer – Select the Note Disclaimer from the list. The disclaimers are defined in the Note Disclaimer function of CA Code Setup.

  8. Report Header –Select the Report Header from the list. The field displays Template based Sections created with Template Type as Report Header in the Section function of CA Code Setup.

  9. Note Header - Select the Note Header from the list. The field displays Template based Sections created with Template Type as Note Header in the Section function of CA Code Setup.

  10. Note Footer - Select the Note Footer from the list. The field displays Template based Sections created with Template Type as Note Footer in the Section function of CA Code Setup.

  11. Confidential – Select the confidential check box to restrict the users from viewing the clinical notes in the history.

 

Event Captions

The event captions appear as legends on selection of a note type for which they are set.

  1. Type - Enter the Note Type Caption in alphanumeric format of 30 characters maximum length.

  2. Date - Enter the Note Date & Time Caption in alphanumeric format of 20 characters maximum length.

  3. Status - Enter the Status Caption in alphanumeric format of 20 characters maximum length.

  4. Title - Enter the Event Title Caption in alphanumeric format of 20 characters maximum length.

  5. Performed By - Enter the Event Performed By Caption in alphanumeric format of 20 characters maximum length.

  6. Authorized By - Enter the Authorized by Caption in alphanumeric format of 20 characters maximum length.
