DS Parameter For Facility

From this function you can Define rules that influence the functioning of Dietary Service Module within the facility.

The DS Parameter for Facility function helps in defining and maintaining various DS Parameter setups specific to a Facility. Using this function, you can identify the default diet for patients without orders, label messages for meals and multi-kitchens. You can choose the maximum number of days to repeat a specific meal cycle in the facility. You can specify whether the diet type is applicable for each meal type. You can also define if the attendant orders are accepted in the facility. The Dietary Services module can be linked to the  Revenue Management module to compute the charges for the services availed by a patient.

Each facility can have a single parametric record. It is mandatory to set up the parameter before any operations are performed as these records store important details about Dietary Services module specific to a facility.

Defining DS Parameter for Facility

From the Admin Setup, select the DS Parameter For Facility function.

The DS Parameter for Facility screen gets displayed.

  1. Enter the details in the mandatory (*) fields. <Related Field Description>

  2. Click [Apply] to save the parameter defined or [Reset] to clear the entered details.

 

You can update the existing details, if parameter is already defined.
