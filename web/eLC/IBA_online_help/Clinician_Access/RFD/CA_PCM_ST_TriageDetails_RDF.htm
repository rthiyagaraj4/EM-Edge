Condition/Case Tab

 

Patient Condition

  1. Priority Zone – Select the required priority from ‘Red’, ‘Yellow’ and ‘Green’ by selecting the appropriate check box. The Priority Zone recorded for the patient while emergency registration is displayed.

  2. Priority – Select the required priority from the list box. The listed priority corresponds to the selected priority zone. The Priority of the patient as recorded is displayed.

  3. Treatment Area – The treatment area assigned to the patient during registration is displayed. The treatment area listed corresponds to the Priority Zone selected.

  4. Bed/Bay No. - This field is enabled on selecting a Treatment Area. Enter the required bed/bay no. in alphanumeric format of eight characters or click [?] function to search for the required bad/bay no. All the bed/bay associated to the selected Treatment Area in the master function ‘Assign bed/bays for Treatment Area’ is displayed. The Bed/Bay No assigned to the patient is displayed, by default.

  5. Disaster – Select the check box if disaster is the cause for the patient’s medical condition. Clear the check box if disaster is not the cause for the patient’s medical condition. The check box is cleared by default. If the Disaster check box is selected during Register Attendance, it is selected by default in Secondary Triage function.

  6. Dis. Regn. Reference – The Disaster Registration Reference Number captured during registering attendance gets displayed. You can edit the Disaster Registration Reference Number, if it was captured wrongly while registering the attendance. If the Disaster check box is not selected during register attendance,  select the Disaster Check box in this function to enable the Dis. Regn. Reference field. Enter the Disaster Registration Reference number of the patient in alphanumeric format of 15 characters maximum length. Enter the reference number allocated by the hospital, in case of disaster, for easy retrieval of patient’s records pertaining to a disaster. This number is common for all the patients pertaining to one disaster and helps the practitioner to know about the patient’s background.

  7. Disaster Area – This field is enabled on selecting the Disaster check box. Enter the required Disaster Area in alphanumeric format or select using [?] function. All the areas defined in Master Patient Index module are listed.

  8. Disaster Type – Select the Disaster Type from the list.

  9. Ambulatory Status - Select the Ambulatory Status from the list.

  10. Referral ID – Select the Referral ID using [?] function.

  11. Presenting Problem - Select a presenting problem using [?] function. The listed presenting problems are defined in 'Reason for Admission' function in Application Masters module. The Presenting Problem recorded during primary triage is displayed.

  12. Remarks – Enter remarks, if any.

 

Case Details

  1. Case of Trauma – Select the check box if the accident is a case of trauma.

  2. OSCC – Select the check box if the patient is bought to the facility relating to an OSCC case.

  3. Medical – Select the check box to indicate the case requires medical attention.

  4. Surgical - Select the check box to indicate the case requires surgical attention.

  5. Date/Time of Accident – Select the Date and Time of the Accident using calendar function.

  6. Place of Accident – Enter the Place of Accident in alphanumeric format of 30 characters maximum length.

  7. Mechanism of Injury – This is used to record the mechanism of injury of the patient, which is provided in two lists. The first list is used to select the Injury Category and the second is used to select the Injury Subcategory as set in ‘AE Code Setup’.

  8. Vehicles Involved - This is used to record the vehicles involved (if any) for the case of trauma. The list box displays all the values set in ‘Transport Mode’ of Application Masters module.

  9. Protective Device Used - Select the Protective Device Used from the list.

  10. Position during Incident - Select the Patient Position during the Incident from the list.

 

MLC Details

  1. MLC – Select the check box if the patient is registered under an MLC case. This check box is selected and disabled on selecting the OSCC check box.

  2. Police Report No - This field is enabled on selecting the OSCC/MLC check box. Enter the police report number for the case in alphanumeric format of 20 characters maximum length.

  3. Police Station Details - This field is enabled on selecting the OSCC/MLC check box. Enter the police station details in alphanumeric format of 20 characters maximum length.

 

Others Tab

 

Initial Management

  1. Cap Blood Sugar, Spo2, FBC, ECG and Dressing check boxes are disabled.

  2. Immobilization – Select the Immobilization from the list. The values listed are defined in the Immobilization function of AE Code Setup.

 

If the patient registered in an AE location is a female patient then the Menstrual History for the patient is entered based on the ‘Min. Age of the patient for capturing pregnancy details’ as set in AE Parameter.

 

Menstrual History

  1. Pregnant at Present -Select the check box if the female patient brought into the location under emergency situation is pregnant. Clear the check box, if the female patient brought into the location under emergency situation is not pregnant.

  2. LMP Date - Enter the Last Menstrual Period (LMP) Date of the female patient in dd/mm/yyyy or select it using the Calendar function. The LMP is captured even though the patient says that she is not pregnant to find out whether there is any delay in the Menstrual period and to verify that the patient is not pregnant. There are certain drugs that are not to be prescribed for pregnant patients. The LMP is captured, so that such drugs are not prescribed for patients who are not aware that they are pregnant. The LMP is captured for non-pregnant patients based on the AE Parameter Setup. The gender of the patient is validated internally by the system while capturing the LMP date for non-pregnant patients. You can enter the LMP date, if the AE parameter is set-up. You can proceed even without entering LMP details. This is an optional setup.

  3. POA – POA indicates the Period of Amenorrhea and is displayed based on the LMP date entered in weeks and days. The POA is computed, if the patient is pregnant.

  4. EDD – This indicates the Expected Date of Delivery of the patient based on the LMP date entered. The EDD is computed, if the patient is pregnant.

 

If the patient registered in an AE location is a child under the age as set in AE Parameter for ‘Max Age of the patient for capturing pediatric details’ then the Pediatric Details of the patient is entered.

 

Pediatric Detail

  1. Growth/Development Milestone Age – Select the check box if the Growth/Development Milestone for Age details is applicable.

  2. Immunization Up to Date – Select the check box to indicate if the immunization is up to date.

  3. Menarche – This check box is disabled for ‘Male’ pediatric patients and is enabled for ‘Female’ pediatric patients. On selecting the check box for female patients the Menstrual History details section is displayed and enabled based on the AE Parameter setting for ‘Min Age of the patient for capturing pregnancy details’ and ‘Max Age of the patient for capturing pediatric details’.  

If the Growth/Development Milestone for Age check box is selected, the Height, Weight and Circumference fields are enabled.

  1. Height – Enter the height in Centimeters in the height text box.

  2. Weight - Enter the weight in Kilograms in the weight text box.

  3. Circumference - Enter the circumference in Centimeters in the circumference text box.
