Update Diagnosis Profile

  1. Diagnosis Set - Select the Diagnosis Set from the list. The list displays the Terminology Sets as defined in the Terminology Set function that is associated for a practitioner, facility and specialty in Medical Record module. On selecting the Diagnosis Set, the screen as shown is displayed.

 

Enter the following details:

  1. Diagnosis – Enter the Diagnosis code  in alphanumeric format of 100 characters maximum length, if the 'Free Text Applicable' check box for the selected Terminology Set is selected in the Terminology Set function or click [?] function to search and select the desired diagnosis. This field is mandatory. On selecting [?] function,  the Custom List Tab of the screen is displayed, by default.

Select the Search By and Search Criteria fields, enter Search Text and click [Search] to display diagnosis codes. You can search using diagnosis long description if the ‘Diagnosis Search by Long Description’ check box is selected in MR Parameter.

Based on the nature of the diagnosis selected, the system displays a hyperlink to select from the associated supporting diagnoses. Based on the nature of the diagnosis selected (Dagger Code or Consequence) the system enables you to associate the Manifestation/External Cause.

Custom List tab – Click this tab; custom defined specialties are listed as hyperlinks. Click the required hyperlink and enter the search criteria to search for recorded diagnosis information.

Click the Level Tab to select the required diagnosis based on the levels as defined in the Terminology Code function in Medical Records module. Click the required hyperlink under Level and click [Select] to select the diagnosis.

Click the Select Tab to select diagnosis from the classifications defined in the Terminology Code function in Terminology & Coding Management  module.

The codes as defined for the terminology sets in the function Terminology Codes are displayed. Select the Code hyperlink. Based on the nature of the diagnosis selected, the system displays a hyperlink for the user to select from the associated supporting diagnoses. Based on the nature of the diagnosis selected (Dagger Code or Consequence) the system enables associating the Manifestation/External Cause.

[+] is enabled, if the chosen diagnosis code has an indicator set as either ‘D’ or ‘C’ which indicate ‘Dagger’ and ‘Consequence’ respectively. This setup is done in the ‘ICD Code’ master function in the Terminology and Coding Management module. Select the supporting diagnosis to be associated with the main diagnosis and click [OK].

The selected code is displayed in the Diagnosis Field.

Click [!] corresponding to the diagnosis field to view the description of the diagnosis as set in MR Terminology Code setup. On viewing the description, click [Close].

On mouse-over of this icon, the associated High Risk Condition gets displayed. This icon gets displayed when a practitioner records a sensitive diagnosis and if there is any high risk condition associated with the sensitive diagnosis in Terminology Code function in Medical Records module.

If a disease is a Notifiable disease [+] is enabled, Click [+] to record Notifiable Form. Click [Copy From Previous Note] to copy previous clinical note details to the current encounter. Once Notifiable Form is signed by the practitioner, it is saved in PDF format and mailed to public health authorities.

  1. Encounter Stages – Encounter Stage field is displayed, if set in the 'Term Set Criteria for Patient Class' function in Medical Records module. Select the desired encounter stage from the list. The valid values are ‘Admission’, ’Intermediate’ and ‘Discharge’.

  2. Diagnosis Classification - This field is displayed if 'Terminology Classification Applicable' check box is selected for the Term Set in the Terminology Set function. Select the required Diagnosis Classification from the list. The listed values are defined in the Diagnosis Classification function in Medical Records module.

  3. Notifiable – If you want to notify the recorded diagnosis to specific authorities, select a notifiable clinical note type from the drop-down list. The already created notifiable codes are available for selection in the drop-down list.

The default notifiable code as configured in Notifiable Setup  will be displayed as by the default note type.

After you select the note type, the + icon will be enabled. Click the + icon and the Notifiable Form dialog  is displayed. You can enter the details for the clinical note.  The note group will display the value Notifiable form by default. You can also click [?] and select another note group, if required. This field is mandatory if ‘Notifiable Form Mandatory’ check box is selected in MR Parameter.

  1. Chief Complaints - Click [?] function to select the problems and link it to the diagnosis. You can link the problem to the diagnosis either when you record the diagnosis or by clicking the diagnosis hyperlink after recording the diagnosis. If any Chief complaints are recorded for the patient in Chief Complaints function, the values are displayed in the 'Select Problem' screen to link to the diagnosis. Select the check box to link the problem to the diagnosis. On selecting the Problems whose onset dates are beyond the selected diagnosis, the message 'Onset Date of the problem is beyond the onset date of the diagnosis' is displayed and the check box of the selected Problem is cleared. If a problem is already associated to the diagnosis, the Select check box is disabled. Click [OK] to link the problem to the diagnosis, or [Cancel] to close the Select Problem screen.

  2. Nature - Select the nature of diagnosis as Actual, Potential or Wellness. The Nature is displayed if set in the 'Term Set Criteria for Patient Class' function in Medical Records module. This field is mandatory.

  3. Accuracy – Select Accuracy from the list. The Accuracies are defined in the Accuracy function in the MR Terminology Set. On selecting the Accuracy, Differential Group hyperlink is displayed. Select the hyperlink to view a screen. Select the Group 1 radio button or if the diagnosis is to be recorded under a new group, select New Group radio button. On selecting New Group radio button, Group 2 is displayed, which can be selected. Click [Close] to exit from the screen.

  4. Severity – Select Moderate, Mild, Severe or Extreme the severity from the list. This field is mandatory.

  5. Priority – Select priority as Normal, High or Critical. The Priority is displayed if set in the 'Term Set Criteria for Patient Class' function in MR module. This field is mandatory.

Select ‘As Evidenced By’ hyperlink.

The Defining Characteristics are displayed, which can be selected. The characteristics are defined as displayed in ‘Factors for Terminology Code’ function. On selecting the Defining Characteristics, click [OK] or click [Close] to exit from the screen.

  1. Onset Type – Select the Onset Type as Acute or Chronic from the list. This field is mandatory. Not Applicable is displayed, by default.

  2. Onset Date – This field denotes the date from which the Patient has the problem. The current system date is displayed by default, which can be changed to an earlier date. To change the date, choose the date form the Calendar function. This field is mandatory.

  3. Since – This field is updated automatically, on entering the Onset Date.

  4. Status – The Status displayed is Active by default. You  cannot modify the status while recording the current diagnosis. Other available statuses are 'Resolved' and 'In Error'.

  5. As On – The current date is displayed for the status.

  6. Include in Current Encounter – The check box is selected and disabled, which indicates that the diagnosis is included in the current encounter.

  7. Anatomical Site - Select the Anatomical Site types from the list. The listed values are Oral Cavity and Others. Select Others to list Anatomical Sites created in the Anatomical Site function in Application Masters module with the Oral Cavity check box cleared. Select Oral Cavity to list the Anatomical Sites created in the Anatomical Site function in Application Masters module with the Oral Cavity check box selected.

  8. Applicable Side - Select the applicable side from the list. The listed values are Left, Right, Both and Not Applicable. This field is displayed only on selecting anatomical type 'Others' in Anatomical Site field.

  9. Remarks – Enter the remarks, if any.

On recording the diagnosis, the data entry part of the screen is cleared and the screen is refreshed to include the new problem assessment by the practitioner as part of the ‘Current Problems’ list.
