Allergies

From this function you can Record allergies for a patient and view allergies already recorded for the patient.

The allergy recording helps care providers in managing patients from known adverse reactions during the course of a treatment. The allergies recorded are validated in some transactions. For example, when a drug allergy is recorded for a patient, the system validates when the patient is being prescribed the same drug in the system. The allergies can remain active across encounters. If a patient is cured of an allergy, there is provision to close the active allergy. There is also a provision to mark the allergy recorded as an erroneous entry. An audit trail is maintained for all such erroneous entries.

Recording Allergies

From the CA Transactions > Clinician Access > Desktop Menu > select required patient > Patient Chart Menu > Other Clinical Event Recording, select Allergies.

The Allergies screen gets displayed.

The screen displays an option “Do Patient Have New Allergy” at the top, if the ‘Allergy confirmation Required’ parameter is selected in the MR parameter screen in the Medical Records module. By default, No, will be selected for this option.  

 

  • If you select Yes, the check box ‘No Known Allergy’ in the Current Assessment grid will be cleared and disabled. You can select an event type and record an allergy.

  • If you select No, the check box ‘No Known Allergy’ in the Current Assessment tab will be selected and disabled. All the fields in Current Assessment grid will be disabled except “Record” and “Clear”.

 

  1. Any active Adverse Events already recorded for the patient is displayed on the top part of the screen.

    1. Click the ‘+’ hyperlink to display ‘–’ hyperlink and view the details of the Adverse Event recorded. On clicking the + hyperlink, details like Reaction Date, Reaction Description, Severity and Site of Reaction are displayed as recorded.

    2. Event Type – The Event Type recorded for the patient is displayed.

    3. Allergen – The Allergen recorded for the patient is displayed.

    4. Onset Date – The Onset Date recorded for the Adverse Event is displayed. The Onset Date is displayed is the Active Since Date recorded in the 'Event Type Details' screen.

    5. Status – The Status of the Adverse Event is displayed.

  2. Enter the details in the mandatory (*) fields. <Related Field Description>

  3. Click [Record] to save the Adverse Events recorded or [Clear] to clear the entered data.

  4. The details recorded are displayed on the top part of the screen. Click Allergen hyperlink to modify recorded details of the Adverse Event.

  5. The status field displays Resolved and In Error options in addition to Active.

  6. On selecting In Error, a reason field is displayed for recording the reason to mark the Adverse Event as error. Click [Record] to save the record marked as error.

  7. Select Resolved option to record the Adverse Event as a resolved event. Click [Record] to save the record marked as resolved.
