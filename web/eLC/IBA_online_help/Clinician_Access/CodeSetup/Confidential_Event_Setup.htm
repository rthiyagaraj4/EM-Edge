Confidential Event Setup

From this function you can Define privileges to view patient events that are marked as confidential.

This function allows you to set privileges to view patient events marked as confidential. For any particular clinical event, the confidential privileges can be given either to a set of practitioners or practitioners based on their specialty or practitioners based on their responsibility.

 

Based on the setup done, the practitioners with privileges alone can view the confidential events. For other practitioners, the events will not be displayed. The privileges setup done in this function is applicable in Events Occurred in last 24 Hrs, Patient care flow sheet, View Clinical Notes, View Nurse Notes, Clinical Notes, Existing Order, Order tracking, Clinical Event History and Review Result functions.

Defining Confidential Event Setup

From the CA Code Setup > CA Others Setup, select Confidential Event Setup.

The Confidential Event Setup screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Select the History type from the drop-down list. It is mandatory to select the history type.

  3. Select the Event Type from the drop-down list.  The Event Type list is populated based on the selected history type.

  4. Select the required value from the Filter Based On drop-down list. The list values are ‘Event’, ‘Practitioner’, ‘Specialty’ and ‘Responsibility’. You can set the event privileges with respect to Specialty, Practitioner and Responsibility.

Setting Privileges with respect to Events

  1. Select the value ‘Event’ from the Filter Based On drop-down list. This is a mandatory field.

  2. Select the required event from the Event drop-down list. On event selection, Practitioner, Specialty and Responsibility tabs are displayed. This is a mandatory field.

  3. To set confidential privileges with respect to events for a practitioner, click [Practitioner] tab.

i. The practitioner search criteria is displayed. Enter the search text and click [Search]. The search results grid includes Practitioner Name, View List drop down and Select check box.

ii. In the View List, select ‘Self Related’ or ‘All Patients’ against the required practitioner.

• If ‘Self Related’ is selected, practitioners can view the selected confidential event history for their entire self-related patients.

• If ‘All Patients’ is selected, practitioners can view the selected confidential event history for all patients.

iii. Select the check box against the required practitioner and click [Apply]. The practitioner can view selected event history for all patients/self related patients.

  1. To set confidential privileges with respect to events for a specialty, click [Specialty] tab.

i. The specialty search criteria is displayed. Enter the search text and click [Search].

ii. The search results grid includes Specialty name, Based On, View By and Select check box.

iii. Select the Based On value from the drop down list. The list values are ‘Encounter’, ‘Authorized By/Ordered By’ and ‘Both’.

• Encounter – The encounter related to the Specialty.

• Authorized By/Ordered By – The user who placed the order or the user who signed the clinical notes.

• Both – Filter specialty based on  both encounter and authorized by/ordered by.

iv. Select the View By from the drop-down list. The listed values are ‘Primary Specialty’, ‘Secondary Specialty’ and ‘Both’.

v. Select the check box against the required specialty and click [Apply]. The practitioners belonging to the selected specialty can view the selected event history.

  1.  To set confidential privileges with respect to events for a responsibility, click [Responsibility] tab.

i. The Responsibility search criteria is displayed. Enter the search text and click [Search].  The search results grid includes Responsibility name, View List, Select check box.

ii. Select the ‘Self Related’ or ‘All Patients’ from the drop-down list.

iii. Select the check box against the required responsibility and click [Apply]. The practitioners belonging to that responsibility can view the selected event.

 

Setting Privileges with respect to Specialty

  1. Select the value ‘Specialty’ from the ‘Filter Based On’ drop-down list.

  2. Select the required specialty from the lookup. This is a mandatory field.

  3. Select the ‘Based On’ from the drop-down list. The list values are ‘Encounter’, ‘Authorized/Ordered By’, and ‘Both’.

  4. Encounter – The encounter related to the specialty.

  5. Authorized By/Ordered By – The user who placed the order or the user who signed the clinical notes.

  6. Both – Filter specialty based on both encounter and authorized by/ordered by.

  1. Select the ‘View By’ from the drop-down list. The list includes ‘Primary Specialty’, ‘Secondary Specialty’ and .Both’.

  2. Enter the Event search criteria and click [Search]. The events are displayed with their code and description.

  3. Select the check box against the required event and click [Apply]. The practitioners belonging to that specialty can view the selected event history.

 

Setting Privileges with respect to Responsibility

  1. Select the value ‘Responsibility’ from the ‘Filter Based On’ drop-down list.

  2. Select the required responsibility from the lookup. This is a mandatory field.

  3. Select the View List from the drop-down list. The drop-down list values include ‘All’, ‘Self Related’.

  4. If ‘Self Related’ is selected, practitioners can view the selected confidential event history for their entire self-related patient.

  5. If ‘All’ is selected, practitioners can view the selected confidential event history for all patients.

  1. Enter the Event search criteria and click [Search]. The events are displayed with their code and description.

  2. Select the check box against the required events and click [Apply]. The practitioner belonging to respective responsibility can view selected confidential event history of all/self-related patients.

 

Setting Privileges with respect to Practitioner

  1. Select the practitioner value from the ‘Filter Based On’ drop-down list. The ‘Practitioner’ field gets displayed.

  2. Select the practitioner name from the lookup. It is mandatory to select the practitioner name.

  3. Select the View List from the drop-down list. The list includes ‘All’ and ‘Self Related’.

  4. If ‘Self Related’ is selected, practitioners can view the selected confidential event history of their entire self-related patients.

  5. If ‘All Patients’ is selected, practitioners can view the selected confidential event history for all patients.

  1. Search for the required event by entering the search criteria and click [Search].

  2. The event search results grid includes Event name, Event Description, Select check box.

  3. Select the check box against the required event and click [Apply]. The practitioner can view the selected confidential event history of the all /self-related patients.  

 

  1. Click [Apply] to save the defined privileges.
