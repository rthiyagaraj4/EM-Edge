Enter the following details to create a Visit Details Template record:

 

  1. Code – Enter the Visit Details Template code in alphanumeric format of four characters maximum length.

  2. Description – Enter the description of the Visit Details Template in alphanumeric format of 40 characters maximum length. This is a mandatory field.

  3. Location  – Select the Location Type for the Visit Details Template from the list. The listed system-defined values are Clinic and Procedure Unit. Select the Location using [?] function. Locations are displayed based on the selected Location Type. If Location Type is selected as ‘Clinic’, all locations created using ‘Clinic’ function under OP Code Setup is listed. If Location Type is selected as ‘Procedure Unit’, all locations created using ‘Procedure Unit’ function under OP Code Setup is listed. The selected location is defaulted when the Visit Details Template is selected during Visit Registration.

  4. Room – Select the Room from the list. Rooms are displayed based on the selected Location. Rooms are associated to the Location in the ‘Resource for Location’ function selecting ‘Resource Class’ as ‘Room’ under OP Code Setup. The selected room is defaulted when the Visit Details Template is selected during Visit Registration.

  5. Service – Select the Service from the list. Services are displayed based on the selected Location. Services are associated to the Location in the ‘Service for Location’ function under OP Code Setup. The selected service is defaulted when the Visit Details Template is selected during Visit Registration.

  6. Sub-service – Select the Sub-service from the list. Sub-services are displayed based on the selected service. Sub-services are created using ‘Sub-Services’ function under Service Setup in Application Masters module. The selected Sub-service is defaulted when the Visit Details Template is selected during Visit Registration.

  7. Visit Type – Select the Visit Type from the list. Visit Types are displayed based on the selected location. Visit Types are associated to the Location in the ‘Visit Type for Location’ function under OP Code Setup. The selected Visit Type is defaulted when the Visit Details Template is selected during Visit Registration.

  8. Practitioner – Select the Practitioner using [?] function. Practitioners are displayed based on the selected Location. Practitioners are associated to the Location in the ‘Resource for Location’ function selecting Resource Class as ‘Practitioner’ under OP Code Setup. The selected Practitioner is defaulted when the Visit Details Template is selected during Visit Registration.

  9. Other Resource – Select the Other Resource Type from the list. The listed values are ‘Equipment’ and ‘Others’. The selected Other Resource Type is defaulted when the Visit Details Template is selected during Visit Registration. Select the Other Resource using [?] function. Other Resources are displayed based on the selected Other Resource Type. Other Resources are associated to the Location in the ‘Resource for Location’ function selecting Resource Class as ‘Others’ under OP Code Setup. The selected Other Resource is defaulted when the Visit Details Template is selected during Visit Registration.

  10. Order Catalog – Select the Order Catalog using [?] function. Order Catalogs are created using ‘Order Catalog’ function under ‘OR Order Setup’ in Order Entry & Result Reporting module. The selected Order Catalog is defaulted when the Visit Details Template is selected during Visit Registration.

  11. Reason – Select the Reason using [?] function. Reasons are created using ‘Reason’ function under ‘AM Other Setup’ in Application Masters module. Reasons associated to ‘Register Visit’ function under Outpatient Management are listed. The selected Reason is defaulted when the Visit Details Template is selected during Visit Registration.
