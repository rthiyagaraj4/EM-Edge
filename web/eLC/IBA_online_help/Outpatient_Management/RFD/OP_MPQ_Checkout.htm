Check Out

Check out function is used to record the Check Out details of the Patient at the end of consultation. The Practitioner has to be assigned to the patient to perform Check Out. The Check out function can be performed on the patients with queue status as ‘Consultation In Progress’ or ‘Undergoing Treatment/Investigation’.

  1. Click Check out option from the queue status to display the Check out screen.

  2. Enter the details in the Checkout screen to perform the Checkout. <Related Field Description>

  3. Click [Certificate] in the Check Out screen to issue a medical certificate. Click [OK] to save the certificate, [Clear] to clear the details entered or [Cancel] to exit from the screen.

  4. Click [Create Waitlist] to create a wait list for the patient if schedule is not available. This button is enabled if the ‘Type’ in the Check Out screen is selected as a value with the ‘Use at Visit Conclusion’ check box selected in the Disposition Type function of the Application Masters module.

  5. Click [Modify Waitlist] to modify the details of Waitlist created.

    Wait List is modified on Patient’s request or due to other reasons. This function is used to view details and modify certain details of the wait list. This function allows for reallocating the location for which the wait listing has been assigned as well as reallocates the resource for the wait listing. Further, this function is used to postpone the preferred date and reassign a priority for the waitlist.

    To modify wait list, from the already created wait list record(s) click on ‘Modify Wait List’ hyperlink for the selected patient.

    Enter the Reason for Modification or select the reason using [?] function.

  6. Click [Conclude Episode] to conclude episode.

    If the Episode of the selected visit has to be concluded, click [Conclude Episode]. Episode cannot be concluded for patients who are asked to come back on a future date, in such cases the button is disabled. Conclude Episode screen displays the Service Start and End date of the current episode. You can enter the Notes pertaining to the Episode.

  7. Click [Record] to record the Checkout details or click [Cancel] to return to patient queue screen.  

  8. On clicking [Record], the system prompts a pop-up message if patient requires a referral registration. Click [OK] in the message dialog to register a referral and the Register Referral Details screen is displayed. Click [Cancel] to return to manage patient queue screen. The prompt message is populated only if the ‘Prompt for Register Referral on close consultation’ check box is selected in the OP Parameter function. If the ‘Prompt for Register Referral on close consultation’ check box is cleared in the OP Parameter, the control goes back to the OP Check out screen.  

  9. Enter the details in the mandatory fields in Register Referral Details screen. Click [Clear] to clear the entered details if required.

  10. On entering the details, click [Record] to add the patient as referral and Referral Details are displayed. Click [Clear] to clear the displayed details. Click [Refresh] to refresh the screen and on completion of registration of referrals, click [Close] to return to the patient queue screen.

    1. Referral ID - The Referral Identification Number generated for referral is displayed and hyperlinked. You can view the referral details on clicking hyperlink.

    2. Preferred On – The preferred date of referral is displayed.

    3. Referred To – The destination facility of referral is displayed.

    4. Specialty – The Specialty name to which referral has been recorded is displayed.

    5. Location - The Location to which referral has been recorded is displayed.

    6. Practitioner - The name of the practitioner against whom referral has been recorded is displayed.

    7. Appt. No – If appointment has been confirmed using the referral identification number, the Appointment Number is displayed. You can proceed with booking an appointment using the [Book Appt] hyperlink.
