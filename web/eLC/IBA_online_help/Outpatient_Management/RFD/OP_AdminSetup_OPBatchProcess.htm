Enter the following details to perform the required batch process:

 

  1. Process - Select the process from the list. The listed values are ‘Auto- Complete Visit’ and ‘Auto-Close Episode’. This is a mandatory field.

 

If ‘Auto-Complete Visit’ is selected, the system displays the cut-off date for Auto-Complete Visits:

  1. Cut-off Date – The cut-off date is displayed in the format dd/mm/yyyy. The field displays the computed value based on the defined value in OP Parameter, by default. This is a mandatory field.

 

If Auto-Close Episode’ is selected, the system displays the cut-off date for Auto-Close Episodes:

  1. Service – Select the required service for which the process is to be defined form the list.

  2. Cut-off Date with Follow-up – The cut-off date with follow-up is displayed in the format dd/mm/yyyy. The field displays the computed value based on the defined value in OP Parameter, by default. This is a mandatory field.

  3. Cut-off Date without Follow-up – The cut-off date without follow up is displayed in the format dd/mm/yyyy. The field displays the computed value based on the defined value in OP Parameter, by default.

 

The configuration in “Parameter for Facility” for “No Show” will be defined as default and disabled in “OP Batch Process” function. You can only perform the services of your logged-in facility as batch process.

 

If the Episode closure by value is selected as “No Shows” in the OP Parameter screen, the following fields will be displayed here:

 

  1. Number of Consecutive No shows   - Define the number of no shows to be elapsed after which you  can consider a patient’s episode for closure.

  2. No Visits in Last X Months – Define the minimum number of past months within which if a patient has not made a visit, their episode can be considered for closure.

  3. No Appointments for next X Months - Define the maximum number of future months within which if a patient’s next appointment is not scheduled, their episode can be considered for closure.

Note: Only when all the above mentioned three conditions are met, the system will consider the patient record for batch process to conclude the episode; else the record will not be considered.
