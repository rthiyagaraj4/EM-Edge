Enter the following details to create a Clinic record:

 

  1. Code - Enter the Clinic code in alphanumeric format of four characters maximum length. A Clinic code is to identify the Clinic. The Clinic code has to be unique and cannot be changed after creation. This field is mandatory.

  2. Type - Select the required Location Type from the list of valid values. All the system defined and user defined Care Location Types associated to Indicator ‘Clinic’ in Application Masters module are listed. This field is mandatory.

  3. Long Description - Enter the long description of the Clinic in alphanumeric format of 30 characters maximum length. This field is mandatory.

  4. Short Description - Enter the short description of the Clinic in alphanumeric format of 15 characters maximum length. This field is mandatory.

  5. Physical Location - Enter the Physical Location of the Clinic in alphanumeric format of 40 characters maximum length. A Physical Location is to identify the exact location of the Clinic.

  6. Enabled - Select the check box to enable the record and clear the check box to disable the record.

  7. Specialist Clinic – Select the check box to indicate that the Clinic is a Specialist Clinic.

  8. Visiting Practitioner – Select the check box to indicate that the Practitioner is a Visiting Practitioner. The check box is enabled if Specialist Clinic check box is selected.

 

Clinic characteristics are entered under two tabs, namely, ‘Characteristics’ and ‘Others’ tab.

 

Characteristics Tab

 

Working Days

  1. Working Days - A clinic can function on all or specific days of the week. Select the appropriate check boxes to indicate the Clinic working days. The visit registration and appointment processes are used for a Clinic only on its working days. The default working days are selected based on the Day of Week defined in System Manager module.

  2. Max Walk-In Per Day - This field is enabled if Allow Walk-in check box is selected. Enter the number of walk-in patients per day for all the days in numeric format of 3 digits maximum length.

 

Operating Time

  1. Start – Enter the operating start time of the Clinic in time format of hh:mm. This field is optional.

  2. End - Enter the operating end time of the Clinic in time format of hh:mm. This field is optional. Operating End time should be greater than Operating Start Time.

  3. Base Slot/Slab Time – Base slot/slab time is used to identify the minimum unit of time frame used to book an appointment where multiple resources are involved by using multiples of the base time defined. Enter the base slot/slab time in time format of hh:mm. This field is optional.

 

Controls

  1. Allow Visit Registration – Select the check box to Allow Visit Registration for the defined Clinic. For some of the Clinics like Immunization, Visit Registration is not required and patient can get the treatment directly.

  2. Allow Walk-in – This field is enabled on selecting the ‘Allow Visit Registration’ check box. Select the check box to allow walk-in’s to be entertained to the defined Clinic.

  3. Allow Appointment - This field is enabled on selecting the ‘Allow Visit Registration’ check box for clinics with level of care set to ‘Ambulatory’. For Clinics with level of care set to ‘Emergency’, this field is disabled. Select the check box to allow appointments for the defined Clinic. If this check box is set as cleared, the time table records cannot be setup for this Clinic in Appointment Scheduling module and appointments cannot be booked.

  4. Allow Referral - This field is enabled on selecting the ‘Allow Visit Registration’ check box for Clinics with level of care set to ‘Ambulatory’. Select the check box to indicate if referrals are allowed in this Clinic. Mainly, Clinics allow Visit Registration only on producing a referral identification number. If the check box is cleared, the Clinic does not permit referral patients.

  5. Allow Non-Referral - This field is enabled on selecting the ‘Allow Visit Registration’ check box for Clinics with level of care set to ‘Ambulatory’. Select the check box to indicate if non-referrals are allowed in the Clinic.

  6. Age Group - Select the Age Group of patients who are eligible to receive the services provided by the Clinic from the list of valid values. If this value is set up, the Visit Registration process restricts the patients based on their age and gender depending on the corresponding Age Group parameters set in ‘Other Master Setup’ function of Application Masters  module.

  7. Level Of Care - Select the Level of Care from the list. This defines the level of care provided by the Clinic. The listed values are ‘Ambulatory’ and ‘Emergency’.

If the Level of Care is set to ‘Ambulatory’, the applicable Visit Type Indicator check box, namely, ‘First’, ‘Follow up’, ‘Routine’, ‘Series’ and ‘Consult’ are checked by default under the ‘Others’ tab. Clear the required field if necessary.

If the level of care is set to ‘Emergency’ then the ‘Emergency’ check box in Applicable Visit Type Indicator is selected by default under the ‘Others’ tab.

 

Others Tab

 

Acct Department/Service

  1. Specialty - Select the Specialty from the list. All enabled Specialties created in Application Masters module are listed. This field represents the specialty applicable for the Clinic. This field is mandatory.

Note: If you have assigned a multi specialty to the clinic, you cannot modify it.

  1. Vital Signs Battery – Select the Vital Signs Battery from the list. All the Discrete Measures of type ‘Vital Signs Battery’ set up in Application Master module are displayed in the list. The Vital Signs Battery associated to the Clinic is used to record the Vital Signs of the patients arriving to the Clinic. This field is disabled, by default.

Vital Signs battery in Outpatient Management module is set at three different levels, namely, ‘Clinic setup’, ‘Visit type for location setup’ and ‘OP Parameter Setup’. Vital Signs Battery set up is not mandatory while defining the ‘Clinic’ and ‘Visit type for Location’ but is mandatory while defining ‘OP Parameter’. The system first checks for Vital Signs Battery in the ‘Clinic Setup’.  If it is not defined, then Vital Signs Battery set in ‘Visit Type for Location’ is set as default. If Vital Signs Battery is not defined in Visit type, the Vital Signs Battery set in ‘OP Parameter’ is displayed by default.

  1. Account Department - Select the Account Department for the Clinic from the list. The list of Accounting Departments displayed is limited to the departments of the logged in facility as set in ‘Department in facility’ of Application Masters module. This field is mandatory.

  2. Section – Select the Section for the Clinic from the list. All the sections associated to the selected Accounting Department in Application Masters module are listed in the list.

  3. Service - Select the Service from the list. This field represents the Medical/Ancillary Service that is applicable to the Clinic. The list of services displayed is limited to the services offered by the selected department of the logged in facility. This field is mandatory.

  4. MR Section Type – Select the MR Section Type from the values displayed namely, ‘Central’ or ‘Departmental’. This field is enabled only if File Maintenance is set to ‘Multiple’ in MP Parameter (Logged in facility) of Master Patient Index module or if File Maintenance is set to ‘Single’ in MP Parameter of Master Patient Index module, this field is disabled. The value ‘Departmental’ has to be selected when the Clinic maintains an exclusive MR Section for the patient records and the value ‘Central’ has to be selected when the Clinic uses a common centralized MR Section. The MR Section value is displayed based on the values set at the facility level and if these values are not set at the facility level, the values set in MP Parameter are made default.

  5. MR Section – Select the MR Section from the list of valid values to be associated to the Clinic. This is enabled only if MR Section Type is selected as ‘Departmental’. This field represents the Medical Records Section of the facility associated to the Clinic. The list of values displayed is limited to the MR Sections of the logged in facility.

  6. File Storage Location – Select the File Storage Location to request for files from the list. These values are displayed only if the Medical File Management module is installed. On selection of MR Section Type as ‘Central’ and all those File Storage  Locations of identity ‘Clinic’ which belongs to the logged in facility are displayed. On selection of MR Section Type as ‘Central’ and all those File Storage Locations of identity is department which belongs to the logged in facility are displayed.

  7. Default File Type – Select the applicable File Type from the list. The list of values is populated from the File Type function defined in Patient Identity Management module and is applicable when File Maintenance is opted for single file maintenance in Master Patient Index module.

  8. Auto File Request - Select the check box to automatically request for files from Permanent MRD on Visit Registration to the defined Clinic, or the files has to be manually requested.

 

Applicable Visit Type Indicators

  1. Emergency/First/Follow-up/Routine/Series/Consult - Select the check boxes to indicate whether each of these Visit Types are applicable for the Clinic. If ‘Level of Care’ is selected as ‘Emergency’. The ‘Emergency’ Visit Type check box is selected by default. Select the Other Visit Types applicable for the Clinic. If the ‘Level of Care’ is set as ‘Ambulatory’, all Visit Types except the ‘Emergency’ Visit Type are selected by default or select the ‘Emergency’ Visit Type if applicable for the Clinic.

 

Practitioner/Room Privileges

  1. Primary Resource Class/Type - Select the Resource Class and Resource Type from the list of valid values. The values populated for Resource Class are ‘Practitioner’ and ‘Room’ for Clinics which is based on Ambulatory Level of Care and value is ‘Practitioner’ for Clinics which is based on Emergency Level of Care. Based on the value selected in Resource Class, the values in Resource Type are filtered and populated from Application Masters module.

  2. Open to All Resources - Select the check box to indicate the Clinic is open to all primary resources of the facility. If this check box is not selected, the specific resources have to be associated to the Clinic using appropriate function. This field is applicable when Resource Class is set to ‘Practitioner’. If Level of Care is set to ‘Emergency’, the check box is selected and disabled.

  3. Check in at Identification – Select the value from the list. If Primary Resource Class is ‘Practitioner’, the values are ‘Non-identifiable’ and ‘Resource’. If Primary Resource Class is ‘Room’, the value is set to ‘Resource’. If value is set to ‘Resource’, the same value has to be identified at the time of Visit Registration. Otherwise, the resource identification becomes optional at the time of Visit Registration and the resources are identified before the Consultation process is started. If Level of Care is set to ‘Emergency’, the value Non-identifiable is defaulted and disabled. This field is mandatory.

  4. Select Unassigned Patients - Select the check box to indicate whether unassigned patients can be selected from the Patient Queue for consultation process. Based on the value selected, the attending Practitioner for the encounter is automatically assigned based on who selects the patient. If Level of Care is set to ‘Emergency’ then the check box is selected and disabled.

  5. Allow Consultation On – Select the required value from the list. The values displayed are ‘Checked-in’ and ‘Arrived’. Based on the value selected, the patient can be selected from the queue for consultation by the Practitioner either on checking in or on arriving at the Clinic. The patient name is listed in the Clinician Access queue based on the value selected.

 

Queue Numbering Controls

  1. Numbering Application – Select the check box to assign Numbering for the Clinic. Numbering is set according to the set up made in Queue Numbering Control function.

  2. QMS Interface Applicable – Select the check box to enable Queue Management System for the particular clinic.

  3. Calculate Wait Time based on Status – Select the required value from the drop-down list. The list values include Checked-In and Arrived At Nursing Station. Based on the selected value, the patient Waiting Time is calculated in the Manage Patient Queue. By default, the value ‘Select’ is displayed. The ‘Calculate Wait Time based on Status’ configuration done in OP Parameter will apply for all Clinics. If the setup is done in Clinic screen, then Clinic setup will take precedence. At any point of time the wait time will be calculated based on the latest configuration.

 

Patient Instructions tab

  1. Serial No - Enter the Serial Number in numeric format of two digits maximum length. You can record a maximum of 10 Serial Numbers. This is a mandatory field.

  2. Instruction Description - Enter the Instruction Description in alphanumeric format of 200 characters maximum length or select the Instruction Description using [?] function. This is a mandatory field.

  3. Instruction ID -The corresponding Instruction ID is displayed on selecting the Instruction Description.

Click [Add] to display the selected Patient Instruction to the list at the bottom of the screen.

  1. Default - Select the check box to default the selected patient instructions while booking an appointment for the selected clinic.

Click on Serial Number hyperlink to delete the record. Click [Clear] to clear the selected details. Click [Delete] to delete the record from the list.

 

The instructions you add and select as default in this screen, will be available for printing in the Appointment Slip report when you schedule appointments.
