Service for Location

From this function you can Create an association of Services to a Location and Query for a desired Service for Location.

A Care Location (Clinic, Procedure Unit or Daycare Unit) offers multiple services. Service for Location function allows associating one or more Services to a care location. Services that are associated to a Location are limited to those that belong to the Accounting Department of the Location. The association of Service to a Location is used at the time of registering a visit and optionally used to build multiple encounters as a single episode based on a common service.

Creating an association of Services to a Location

From the OP Code Setup, select the Service for Location function.

The Service for Location screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Select the Location Type as Clinic, Daycare Unit or Procedure Unit. This field is mandatory.

  3. Select the Location using [?] function based on the selected Location Type. Locations must be associated to the Operator Station assigned to the logged in User. This field is mandatory.

  4. On selecting the Location, the Age Group, Account Department, Service and Specialty of the Location is displayed. Select a different Account Department, if required.

  5. Click [Search] to display the Services associated to the Account Department.

  6. Select the check box under Enabled against the required Services to associate them to the selected Location.

  7. Click [Apply] to associate Services to a Location or [Reset] to clear the entered data.

  8. Click [Query] to search for an existing Service for Location record.
