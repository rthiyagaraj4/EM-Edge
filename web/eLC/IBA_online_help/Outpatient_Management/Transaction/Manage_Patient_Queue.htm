Manage Patient Queue

From this function you can Manage patients by viewing the patient queue status and recording the Arrival of Patient, Vital Signs, Assigning or Reassigning Practitioner and Patient Check Out.

Queue management for Clinic helps in streamlining the flow of patients by Assigning Queue Number to patients and managing them based on the queue numbering. Patient queues comprise of patients at various stages of their interaction with the facility. This includes patients already in the facility and patients due to arrive, that is, patients holding appointments for the day. Each specific location such as a Clinic, Procedure Unit and so on separately or in conjunction with a Practitioner, constitutes an individual queue. These queues are managed at the Operator Station, which are control points for a set of Clinics/Departments (Outpatient Locations).

A Patient’s interaction goes through a series of stages represented as the Patient queue status:

• Checked in

• Arrived at Nursing Station

• Vital Recording

• Consultation in Progress

• Undergoing Treatment/Investigations

• Returned from Investigations

• Visit Completed/Checked out

A Patient’s current Location status are of any of these pre-defined values:

• At Assigned Location

• In Transit

The two possible ways in which the Patient Queues viewed are Single View and Multiple View.

The Multiple View presents a comprehensive view of the operator station as multiple queues, where the queues are segregated based on Location/Practitioner/Room combination. The Multi-view is more useful when performing the queue management (monitoring and load balancing).

The Single View presents a user-selectable view as a single queue of Patients that is based on user-specified selection criteria of Patient/Location/Practitioner. The Single view is more useful in locating a specific Patient record. Single view shows all Queue records of the locations controlled by the Operator Station, which is derived based on the login user, by default.

Managing Patient Queue

From the OP Transactions, select the Manage Patient Queue Function.

The Manage Patient Queue screen gets displayed.

  1. Click [Search] the system displays a message to enter any one of the criteria: Location, Specialty, Patient ID, Encounter ID and Queue No.  

  2. Enter the search criteria and click [Search] to display all the patients with visit registered. Click [Clear] to reset the records displayed in the result area.

    1. Specialty - Select the required Specialty from the list.

    2. View (Single/Multiple) - Click on the required option to select the view required. If you select Single View, additional query criteria can be specified to restrict the Patient records. Single is selected, by default.

    3. Location Type - Select the Location Type and Location from the list. The Locations that are associated to the Operator Station are displayed.

    4. Location – Select the Location using [?] function. This is a mandatory field.

    5. Practitioner - Select the Practitioner using [?] function. The Practitioners associated to the displayed Locations are displayed.

    6. Patient ID - Enter the Patient ID in alphanumeric format of maximum length as specified in MP Parameter.

    7. Gender - Select the required value for gender from the list. The valid values are 'Male', 'Female' and 'Unknown'.

    8. Encounter ID - Enter the Encounter ID in numeric format of 12 digits maximum length.

    9. Queue No - Enter the Queue Number in alphanumeric format of six characters maximum length.

    10. Date – The current date is displayed. It cannot be modified.

    11. Start Time – The Start Time is displayed as '00:00'. The patients registered from '00:00' hours onwards for the current day are displayed. It cannot be modified.

    12. Display – Select the display type as ‘By Visit Type’, ‘Previous Day Visits Only’ and ‘Include Check-out patients’ from the list. Based on the selection the records are displayed.

    13. Appointment Status - Select the required value for appointment status from the list. The valid values are 'Yet to come', 'On time', 'Late' or 'No show'.

  3. Click [Expand] to modify the query criteria and to expand the query criteria section.

  4. Click [Clear] to reset the records displayed in the result area.

  1. On searching, the displayed patient records are segregated based on the Location/Practitioner/Room combination.

  2. Click the ‘+’ hyperlink to view the Patient Movement details.

  3. Click the ‘Patient Details’ hyperlink to view the set of options that can be performed on the patient record. If you mouse over on any other patient details hyperlink, the options pop-up window will disappear.

  1. Click the required option from the queue status to display the corresponding screen. The options are:

Arrived

Assign Practitioner

Reassign Practitioner

Record Allergy

Record Vital Signs

Revert to Check-In

Start Consultation

Check Out

Patient Chart

Multi Consult Orders

 

 

When you click the Arrived or Start options, and the patient has any unsettled or unbilled due, the following actions are expected based on the OP Parameter setting.

• No Action – The system will not display any warning or error, and you can proceed with the Mark Arrival and Start Consultation activities for the patient without interruption.

• Warning – The system will display a warning message both when marking arrival and starting consultation for the patient, stating that the patient has an unbilled/unsettled amount due. You can click OK and still proceed with the activities.

• Stop – The system will display an error message notifying you about the unsettled/unbilled amount for the patient. You cannot mark arrival or start consultation for the patient.

Queue Management System is interfaced and queue numbers are generated. You can click  to call patient,   to call next patient. Once the patient is called icon changes to  and the queue number is set to zero. When you click Recall the count gets added by one.

Patient Status and Location updations

The patient status and location for each encounter will be updated according to the following conditions:

  • When a patient visit is registered, the status will be “Checked-In” and the respective location will be displayed against each encounter.

  • When an action is performed in a Clinic, the status in other locations will be updated as Status (At Different Location) and action performed at location will be updated in all other encounters Current location field

  • When  a patient is checked-out from a location, in all other encounters the current location will be updated to their respective locations and the actual status of the encounter will be displayed.

  • When Cancel checkout is performed, the current status and Current location of respective encounter will be displayed and there will be no change in Status and Current Location.

  • When Revise visit is completed with a change in location, the status of the respective encounter will remain same, but the current location will be updated to the changed location.

  • When Record Vital signs is performed before Start consultation, the status will be updated as “Record Vital Signs” and the location will be updated accordingly.

  • When an action is performed in “Procedure unit” other than “Check out” the status, in other locations will be "Gone for investigation" and the procedure unit location will be updated as the current location for all other encounters.

 

 

  • This change will be applicable when a patient has got multiple encounters in the same day in different location either Clinic or Procedure unit.

  • These changes will be reflected in the Manage Patient Queue screen when you perform the following transactions: Arrived, Assign practitioner, Re-assign Practitioner, Start Consultation, Patient Chart, Record Allergy, Record Vital Sign, and Revise Visit, Revert Check in, OP Checkout and Cancel OP Checkout.

  • This will be applicable for Single View only and It will not be applicable for Previous day Visits.
