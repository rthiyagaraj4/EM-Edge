Register Visit to Procedure Unit

From this function you can Register a visit for an Outpatient visiting a Procedure Unit.

Register Visit to Procedure Unit function is used when a patient arrives at a registration desk as an Outpatient in a Location. Visits are registered for walk-in patients, referral patients, patients with appointment number and invited patients from waiting list. Patients visiting Locations are identified by two unique identifiers. The first is to identify the patient using Patient Identification Number (Patient ID) and the other is to identify the visit using an Encounter Identification Number (Encounter ID). An Encounter ID is generated on completing the process of Visit Registration. This function allows you to perform a Visit Registration and generate outputs required for managing the patient after registration. The Register Visit to Procedure Unit screen has three tabs, namely, Invitation List, Appointment List and Add Visit.

 

'Register Visit to Procedure Unit' function is similar to 'Register Visit' function. 'Register Visit' function allows registering Visits to Clinics and 'Register Visit to Procedure Unit' function allows registering Visits to Procedure Units.

Registering a Visit to Procedure Unit

From the OP Transactions, select the Register Visit to Procedure Unit function.

The Register Visit to Procedure Unit screen gets displayed.

 

Select patients for Visit Registration from the following tabs:

Invitation List

Appointment List

Add Visit

 

When a patient is selected for registering a visit, the system prompts with the following:

• If patient has any active encounter as an Inpatient, the system prompts a message ‘Patient is currently an Inpatient’.

• If patient has any outstanding amount, the system pops-up window for Credit Authorization which displays the outstanding amount.

• If in OP Parameter ‘External Interface for Privilege’ is selected, you can use the external interface for fetching details of the patient during registration and validate the privilege conditions while registering a visit for a patient. If cleared, you can proceed with normal registration. If the patient is eligible for privileges, you can register visit, or the system alerts you with a message.

• The patient must have a National ID or the system alerts the user with a message that ‘National ID cannot be Null’ even if the patient is eligible for coverage.
