Dispense Medication

From this function you can Verify/Fill/Allocate/Deliver the prescribed Medication Orders.

You can also dispense the prescription using single function All Stages. The Pharmacist performs all the activities pertaining to dispensing in Dispense Medication function. The various dispensing stages and category applicable to the selected Dispense Location are listed for selection. The Pharmacist selects the stage as per the setup and proceeds with dispensing activity.

Dispensing of medication involves Four major stages, namely;

• Order Verification

• Fill

• Allocate

• Delivery

Depending on the Dispense Location setup, Fill and allocate are interchangeable, i.e. Filling is performed before or after Allocation.

 

In the Dispense Medication screen, you have to login and select the required stage i.e. Allocate and Fill stages. Click Refresh and select the required patient from the search results. When the drug details of the patient is displayed, enter the details as required and click the <stage> button. In the <stage> window, when you select the Select checkbox, the Remarks window is displayed.

  3. Enter the required details in the Dispense Medication screen.

    • Queue Date – Select the date of queue from the calendar control. This is mandatory.

    • Token Series – Select the Token series. This default value will be based on the value selected in the Default Token Series By drop-down list in the Dispensing Rights screen.

    • Token No – Enter the token number in alphanumeric format of maximum eight characters and click Next Token. The Token Number is generated for all the selected orders and the generated Token Number is unique for all the selected orders. On generating the token for the Dispense Location, the registration process completes and following activities are performed:

    • If all the prescribed drugs of an order are registered, the Order ID is not displayed in the patient order list.

    • If all the orders (including their constituents) of a patient are registered, the Patient ID is not displayed in the Patient list.

    • Next Token – Click this button to view the next token no.

    • Order By – Select the order based on which the token numbers should be arranged. The default value of this field will be based on the value selected in the new Order by drop-down list within the Dispensing Rights screen. But you can select the desired value. Based on the value selected, the column value is displayed as given below.

    • If you select the value ‘Order Date and Time’, the order date and time is displayed under the Release Date/Time column when you click Refresh button.

    • If the value ‘Register Date and Time’ is selected, the column name will be displayed as ‘Register Date/Time’.

    • If the value ‘Patient Arrived Time’ is selected, the column name will be displayed as ‘Patient Arrived Time’.

    The functionality is applicable for Allocate, Fill, Deliver and All Stages.

    • Patient ID – Select the patient identification and the patients list will be sorted based on the value selected.

Click the ‘Remarks’ hyperlink in the Drug/Dosage details column in the Dispense Medication – Allocate stage and Allocate / Reallocate dialog window, if ‘Applicable with Optional’ or ‘Applicable with Mandatory’ option is selected in the PH Parameter for Facility screen.

  1. Click [Hide criteria]  to view only the ‘Ordering Facility’ and ‘Order ID’ criteria. All the other criteria will be hidden.

  2. Click [Show Criteria] to view all the hidden criteria. This is done to show more number of records within each stages of Dispense Medication.

The Hide Criteria and Show Criteria buttons will not appear for IP Against Fill List – Deliver stage.

  1. Click [Switch Login] to display Dispense Medication Login screen. Switch Login aids in switching to different Dispense Locations and/or Category from the current login. The Dispense Medication screen closes and control shifts back to the login screen.

Enter the practitioner name or select the Practitioner from the Practitioner search dialog using function. Enter the nationality or select the Nationality from the Nationality search dialog using function. Select the Gender from the drop-down list.

  4. Click on the patient ID and complete the selected stage; the Print/Edit Label screen is displayed.

  5. Enter the required details and click [Ok] to print the dispense sheet/dispense label. Else click [Cancel].

The stages (process names) are displayed based on the legends set in PH Parameter.

  • In Dispensing Stage, you can view and print the split dose details in the Dispense Label. You can print the scheduled dose for up to three times a day.

  • The extra selected search criteria (other than the default criteria) based on which the records are fetched, are displayed below the header level.

  • The Drug with Route Color and Dosage details are displayed in the Drug/Dosage details dialog in the Verify, Fill, Allocate, Deliver, All Stages respectively .

  • For all the home leave prescription, an indicator with green background will be displayed so that the pharmacists are aware that the prescription is a home leave prescription. This indicator will appear in Verification, Fill and Deliver Stages.

  • A new red colour-blinking indicator has been introduced in the Dispense Medication screen to indicate that the patient is a ‘Paying Patient’ and not a patient who have availed medical insurance. This indicator will appear in Verification, Allocation, Fill, Deliver and All stages. This is applicable only for OP patient class.

  • The Minimum Stock Level or Reorder Level of the selected item is displayed while performing the transaction. If the stock is less than the Minimum Stock Level or Reorder Level those stock levels are yellow highlighted. When the available quantity of a selected item is less than the Minimum Stock Level or Reorder Level in Dispense Medication and Sale/Patient Issue, a link to ‘Create/Review Request’ is displayed to request new stocks. Based on selection of Display Min Level & ROL check box in User Access for Store, this functionality will be enabled.
