Verify Order

From this function you can Verify stock status of the prescribed drugs of Inpatients and of the registered Outpatient Orders and orders.

The Pharmacist can view the Drug Details like Drug Information, prescribed frequency and schedule details. In this process, the Pharmacist decides whether to allow order for dispensing or put it on hold. To aid the Pharmacist in decision-making, information like Patient History, Allergy Details, Active Problems, Drug Dosage Over Limit and additional information are provided as input. The Pharmacist can check the stock status of the prescribed drug before completing verification process.

Verifying an Order

From the PH Transactions, select the Verify Order function.

The Verify Prescriptions Login screen gets displayed.

    1. User Name – The logged in User Name gets displayed as read-only text.

    3. Dispense Location – Select the Dispense Location from the list. The locations listed (Pharmacy Type Only) are assigned to the logged in user with rights to perform various dispensing functions. This is a mandatory field.

    4. Category – Select the required value. Based on the Dispense Location the category value gets displayed. The table provides the value(s) listed. This is a mandatory field.

Verifying registered Outpatient Orders for which Token is issued

  • Select the Token Series from the list. The Token Series associated to the selected Dispense Location are available for selection. This is a mandatory field.

  • On selecting the Token Series, the Token generated for the selected Token Series is displayed. The displayed Token Series is in the order the token was generated. You can click [Next Token] to selected the new token or enter the desired Token number. The system increments Token Number by 1 and displays the Token Number.

  • Click [ Show Criteria] to specify additional search criteria and [Hide Criteria] to hide the additional search criteria. Click [ Additional Criteria] to specify additional search criteria.

    Enter the following details:

    1. Practitioner – Enter the Practitioner name in alphanumeric format or use [?] function to search for the required practitioner.

    2. Nationality – Select Nationality of the patient from the list.

    3. Gender - Select Gender of the patient from the list.

    4. Click [OK] to search the orders with the specified criteria or [Cancel] to clear the entered data.

The orders can be registered before the patient arrives to the pharmacy to support processing the prescription before patient arrival. Once the patient arrives after order registration, the patient arrival is acknowledged. The patient ‘Arrival Time’ and ‘Token Number’ are displayed in green colour in the Patient ID column. The arrival time is the system time at the time of acknowledging.

In case, if the patient has not arrived to the pharmacy counter, the Patient ‘Not Arrived’ status is displayed in red colour in the Patient ID column.

Verifying registered Outpatient Orders for which Issue Token is not applicable

  • Click [ Additional Criteria] to specify additional search criteria.

    Enter the following details:

    1. Practitioner – Enter the Practitioner name in alphanumeric format or use [?] function to search for the required practitioner.

    2. Nationality – Select Nationality of the patient from the list.

    3. Gender - Select Gender of the patient from the list.

    Click [OK] to search the orders with the specified criteria or [Cancel] to clear the entered data.
