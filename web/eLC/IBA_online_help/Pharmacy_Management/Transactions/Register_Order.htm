Register Order

From this function you can Query and Register the prescribed Medication Order.

Order Registration is an optional process. If the Dispense Location mandates Order Registration, all the orders placed on the Dispense Location are to be registered. The unregistered orders cannot be dispensed. If registration is not applicable to a Dispense Location, orders placed on the Location are available for dispensing without registration. The following conditions are to be satisfied before invoking the function.

• The logged in user must be a Pharmacist.

• Order Entry and Resulting Reporting module must be installed and interfaced with Pharmacy Management module.

• Workstation of the logged in user must be mapped to Dispense Location.

• Logged in user must have rights to Register Orders for the Dispense Location.

Registering an Order

From the PH Transactions, select the Register Order function.

The Register Order screen gets displayed.

  1. Click [Refresh] to display all the unregistered Outpatient Medication Orders or enter the search criteria to search for particular outpatient orders.

    Enter the following details to view and Register Orders:

    1. Dispense Location – Select the Dispense Location from the list. The listed locations are assigned to the logged in user with rights to perform the Order Registration. The Dispense Locations of type ‘Pharmacy’ are listed.

    2. Patient ID – Enter the Patient ID in alphanumeric format of length as specified in MP Parameter or select the required Patient ID using [?] function.

    3. NRIC - Enter the National Registration Identity Card (NRIC) number of the patient.

    4. Citizen ID – On entering the Patient ID, the Citizen ID gets displayed and disabled. The field legend depends on the MP Parameter Setup.

    5. Previous HN. – On entering the Patient ID, the Previous HN gets displayed and disabled if Previous HN is recorded while registering a patient or enter the Previous HN. The field legend depends on the MP parameter ‘Alternate ID1’ selected under ‘Alt ID Checks’ tab.

    6. Order No - Enter the Order Number in alphanumeric format of 20 characters maximum length.

    7. Priority – Select the Priority from the list. The listed values are ‘All’, ‘Urgent’ and ‘Routine’. The value ‘All’ gets displayed, by default.

    8. Scope - Select the Order Scope from the list. The listed values are New Prescriptions, Expired Prescriptions, External Prescriptions, BMS Prescriptions, Refill Prescriptions and Discharge Medication. The table defines each scope.

    Scope

    Description

    New Prescriptions

    Medication Orders placed that are yet to be dispensed by Pharmacists are dispensed.

    BMS Prescriptions

    Medication Orders partially dispensed by Pharmacist to the patients.

    Refill Prescriptions

    Refill Orders placed by clinicians. A refill prescription is a main order placed repetitively.

    Discharge Medication

    Medication Orders placed on discharging inpatients. Orders for which ‘Discharge Medication’ check box is selected, while prescribing are listed.

    Expired Prescriptions

    Medication Orders with end date less than current system date and time.

    External Prescriptions

    Medication Orders placed using the function ‘Transcribe External Prescription’ are listed.

    1. eriod From – Enter the period in date format of dd/mm/yyyy or select the required date using the Calendar function. The search is performed  based on the Prescription Start Date. The orders placed within the specified range that require registration are listed.

    2. Order Date [From – To] – Enter the Order Date range in date format of dd/mm/yyyy or select the required date using the Calendar function. The orders placed within the specified range that require registration are listed.

    3. Order Location – Select the Order Location from the list. The listed values are ‘Clinic’ and ‘Nursing Unit’. The value ‘Clinic’ is selected, by default. On selecting the Order Location, the Location associated to the Order Location gets displayed.

  2. The most recent order details are displayed, by default. The Route Color is displayed within brackets next to the Drug Name, Dosage Detail, Charge Amount and Patient Payable Amount are displayed in the Prescription Details section.

If the patient is a Paying patient, an indicator [Paying Patient] red in colour keeps blinking on the patient details header to identify that the patient is a paying patient.

  1. If the Current Encounter Financial Details check box is selected in the Dispensing Rights screen, the $ symbol is displayed along with a tool tip as ‘Current Encounter Financial Details’.

  2. Click the $ symbol to view the patient’s current encounter Financial Details, Settlement Details / Document References, Charge Details and Unadjusted Deposit / Pre-payment in a separate Patient Encounter Financial Details dialog.

If there are drugs from multiple encounters for registration, only the latest encounter details will be displayed when you click the $ symbol. If the patient policy is expired, a text ‘Policy Expired’ with red colour will blink as an indicator to the user.

  1. Click Uploaded Insurance Document hyperlink to view the Patient’s policy details, if available.

Billing check box  must be selected in Entity Parameter screen in the Inventory Module. This is not applicable for non-encounter patients .

  1. Click Re-route Prescription hyperlink to open the Re-route prescription screen to enable re-routing of prescriptions across facilities.

  2. Click Patient ID hyperlink on the left hand side of the screen to view required patient order details.

  3. Click Pending Orders hyperlink to view the details of pending orders. The Refill hyperlink is displayed if Refill is applicable for the ordered drug.

    The following details are displayed:

    1. Order ID - The Order ID of the Pending Refill Order gets displayed.

    2. Start Date - The Start Date and Time of the Pending Refill Order gets displayed.

    3. End Date - The End Date and Time of the Pending Refill Order gets displayed.

  4. Select the ‘Select’ check box to select the Order for Registration, ‘Print’ check box to print the prescription. Print is enabled, if ‘Print Prescription’ is applicable for the selected Dispensing Location.

  1. Click [Register] to register the selected order.

If ‘Issue Token’ is applicable for the selected Dispense Location, enter the Token details. If Token Series is not applicable to a Dispense Location, an alert gets displayed.

  1. Enter the details in mandatory (*) fields. <Related Field Description>

  2. Click [Generate Token] to generate the token number.

    The Token Number is generated for all the selected orders and the generated Token Number is unique for all the selected orders. On generating the token for the Dispense Location, the registration process completes and following activities are performed:

    • If all the prescribed drugs of an order are registered, the Order ID is not displayed in the patient order list.

    • If all the orders (including their constituents) of a patient are registered, the Patient ID is not displayed in the Patient list.

 

The orders can be registered before the patient arrives to the pharmacy to support processing the prescription before patient arrival. Once the patient arrives after order registration, the patient arrival has to be acknowledged.

Marking Patient’s Arrival  

  1. Enter the Patient ID and click [Refresh] in the Register Order screen.

  2. A message is prompted to register patient arrival. Click [Yes]. The Mark Patient Arrival screen gets displayed.

  3. Enter the following details:

  4. Patient ID – Select the patient ID using [?] function. The patient name gets displayed. This is a mandatory field.

  5. Order Date From and To – Enter the Order From and To dates by clicking the calendar function. This is a mandatory field.

  6. Token No – Enter the token number in alphanumeric format of maximum eight characters.

  1. Click [Search]. The drug details matching search criteria is displayed.

  2. Select the required drugs by selecting the ‘Select’ check box. This check box is selected, by default.

  3. Click [Patient Arrived] to acknowledge the arrival of patient to the pharmacy counter. The patient arrived status is updated in the Dispense Medication screen.

When the order is registered before the patient arrival, the token number will not be generated.

 

Only Outpatient Orders are registered.

If the message ‘Acknowledge is not required at Dispense Rights Level’ gets displayed, you need to have registration rights for selected Dispense Location. Based on the selected scope and order date range, the result values are displayed. The system evaluates the following rules before displaying orders.

• In ‘PH Parameter for Facility’, the system checks whether dispensing in advance is applicable. If applicable, it checks the number of days ahead orders can be dispensed. Orders with start date within the specified ahead period from the current system date and time are listed for registration.

• In ‘PH Parameter for Facility’, the system checks whether dispensing past orders is allowed. If it is applicable, it checks the number of days up to which an expired order can be dispensed from the current system date and time. Orders within this period are only listed for registration.

•If another user is accessing the same order, an icon is displayed in the 'User Locked' column. On clicking the Patient ID, a alert message 'This Patient Record is opened by another user. Access Denied' is displayed. Click icon to unlock and proceed.
