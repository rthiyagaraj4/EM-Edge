Package Subscription

From this function you can Subscribe a non-Order catalog related Package to a patient.

The Package Subscription function is used to subscribe a pre-defined package to a patient. Based on Package definition the Package end date is validated on selection of Package start Date and Price based on the Billing class selected. The Package Deposit window gets displayed when package is subscribed. You cannot subscribe the package without a deposit amount, if the deposit is recorded as mandatory during package definition.

 

 A package is a set of services defined by the facility at a discounted price. You can associate the packages that are non-order catalog related. The packages with the order catalog as associated can be subscribed only during encounter registration. The packages that are subscribed through this function are populated during encounter registration. During encounter registration, a hyperlink 'Associate Subscribed Packages' is displayed in the 'Financial Details' tab. You can view and associate the packages that are subscribed through this function. The package deposit accepted for non-package services that are taken along with the package has to be refunded before availing a new package.

Subscribing a Package

From the Transactions > Package, select Package Subscription Function.

The Package Subscription screen gets displayed.

  1. Click [?] function to select the patient ID and subscribe the package. This is a mandatory field.

  2. Press [Tab] to subscribe a package. The name of the patient, gender, age and ID is displayed.

  3. Click '+' displayed as a hyperlink to view the package details. You can view the details only when you select a package for subscription.

  4. Enter the details to subscribe a package. <Related Field Description>

  5. Click [Apply] to save the changes or [Reset] to clear the details.

  6. Package Deposit screen is displayed on clicking [Apply] to record the advance amount received against the package. Deposit amount is mandatory, if it is recorded as mandatory during package definition.

  7. A receipt number is generated on recording the deposit details.
