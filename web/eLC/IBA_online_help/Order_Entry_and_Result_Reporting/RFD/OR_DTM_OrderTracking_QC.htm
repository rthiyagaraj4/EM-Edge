Enter the following details:

 

  1. Status – Select the Status from the list. The value Active is displayed, by default. The other values are All, Pending Reviews, Outstanding, Completed, Suspended, Discontinued/Cancelled, Future Orders, Not Done, and Appointment Booked. This is used to list orders based on the specific status. The orders with the status as allocated, verified, appointment scheduled, registered, ordered and pending for review are listed when you select the status as 'Active'.

  2. Elapsed Order - Select the check box to view the elapsed order which are still active.

  3. Ordering Practitioner - Select the Ordering Practitioner from the list. The list value contains all the practitioners defined for the facility. This is used to filter orders based on the Ordering Practitioner.

  4. Order Category - Select the Order Category from the list. The values displayed are All, Laboratory, Miscellaneous, Monitoring, Patient Care, Pharmacy, Radiology, Radiology (Department) and Treatment. The value 'All' is selected, by default. This is used to filter order listing based on a specific Category.

  5. Order Type - Select from the list of valid values. The values listed are filtered based on order category selected. The value 'All' is selected, by default.

  6. Activity Type - Select the Activity Type using [?] function. The values listed are filtered based on Order Type selected. The value 'All' is selected, by default.

  7. Order Catalog - Enter the Order Catalog Description in Alphanumeric format of 20 characters maximum length.

  8. If the remark for the alternate drug exists, a new icon  appears in the Order Catalog column for all the orders . Click and the system will launch the existing Dispensed Locations screen. Click the Remarks hyperlink in the Remarks column to view the existing remarks for the alternate drug in the Remarks dialog window.

  1. Period [From-To] - Enter the date range in the format dd/mm/yyyy. 'To Date' field is displayed, by default as the current system date and the period date range is displayed as eight days. The date entered is checked against the Start Date of the Order.

  2. Priority - Select the Order Priority from the list. The values displayed are All, Stat, Urgent and Routine. 'All' is displayed, by default.

  3. Patient ID - Enter the Patient ID in the format of maximum length as specified in MP Parameter or click [?] function to search the Patient ID.

  4. Encounter ID - Enter the Encounter ID of the patient in alphanumeric format of 12 characters maximum length.

  5. Location Type - Select the Location Type from the list. The value 'All' is selected, by default. The other values are Clinic, Nursing Unit and Referral.

  6. Location - The values displayed in the list depends on the Location Type selected. If 'All' is selected, Clinics, Nursing Units and Referral locations of the logged in facility are displayed. If clinic is selected, all the clinics of the logged in facility are displayed and if nursing unit is the value, all the nursing units of the facility are displayed. Practitioner access to location is selected.

  7. Current Inpatient List – If you select the Location as ‘Nursing Unit’, this check box gets enabled. You can search and print the list of all inpatients admitted in the hospital for the selected time period. If you select the Location field value as ‘ALL’ or ‘CLINICS’, then you cannot select the Current IP List filter. This check box is disabled and cleared by default.   

It is mandatory to select either one location or one patient to generate the report. Enabling the Current IP List filter will remove the mandatory selection of patient or specific Location for report generation excluding ‘All’ status in Order Status drop-down list. The mandatory selection of Patient ID or other fields is applicable only for ‘All’ status in Order Status Filter.

  1. Group By - Select Group By from the list. The value entered is for the grouping of the Orders in the display list. The list values are Patient ID, Order Category, Order Type, Location, Date/Time and Priority.
