Enter the following details to record the Orderable Information:

 

  1. Pregnancy Details – Displays the pregnancy details in red colour based on the set up done for the pregnancy category against each trimester within the Drug screen and based on the selection of ‘Display of no pregnancy record details in Pharmacy Orders’ parameter in the PH Parameter screen.

  2. If the patient is not pregnant, the text ‘Pregnant = NO’ will be displayed.

  3. If the patient is pregnant, the pregnancy details will be displayed as shown below:

  • If the Practitioner has not recorded the pregnancy details, ‘No Record of Pregnancy’ message is  displayed based on the selection of ‘Display of no pregnancy record details in Pharmacy Orders’ parameter in the PH Parameter screen. This message will be displayed only for the patients who fall in the range of age 12-65 yrs set in the Master Patient Index (MPI) Parameter.

  1. Height – The latest height value recorded of the patient using the function ‘Vital Sign’ of Clinician Access module is displayed. If height of the patient is not recorded, then enter the value in numeric value of three digits maximum length. The entered value is validated with the range defined for the order format item that is associated to the pharmacy category at header level in ‘Order Entry Format Item’ master function.

  2. Weight - The latest weight value recorded of the patient using the function ‘Vital Sign’ of Clinician Access module is displayed. If weight of the patient is not recorded, then enter the value in numeric value of three digits maximum length. The entered value is validated with the range defined for the order format item that is associated to the pharmacy category at header level in ‘Order Entry Format Item’ master function.

  3. BSA (Body Surface Area) – The body surface area is calculated automatically and displayed by the system on specifying the height and weight values.

  4. BMI (Body Mass Index) - The body mass index is calculated automatically and displayed by the system on specifying the height and weight values.

  5. Priority - Select the priority of prescription from the list The listed values are ‘Routine’ and ‘Urgent’. The specified priority is set at header level.

  6. Date/Time - The current system date/time is displayed by default. It indicates the prescription date and time. The displayed period can be pre-dated or post-dated based on the rules specified for pre-dating and post-dating orders. These rules are specified in the function ‘OR Parameter’ (Time period for Orders/Results). Specify the required period in date format of dd/mm/yyyy hh:mm. Alternatively, use the calendar function to search for the required period.

  7. Discharge Medication - This check box is enabled based on the following conditions:

The check box is checked and enabled by default. The practitioner can release the prescription as Take Home Medication. If the check box is cleared then, the prescription is treated as a regular inpatient prescription.

 

  1. Home Leave Medication – This check box is visible only if the patient has recorded home leave using Inpatient Management module. This check box is cleared by default.

  1. If the patient has recorded for a home leave and if the practitioner places order and the order date falls within the recorded current leave, then a prompt message is displayed to indicate the doctor that the patient has applied for home leave. On clicking OK, the Home Leave Medication check box gets selected. The ordering drugs are considered as Home Leave Medication and as an inpatient order.

  2. If the recorded leave date is of future date, then a prompt message is displayed to indicate the doctor that the patient is planned to go on home leave. The starting date of the future leave schedule is defaulted for the order. The order is considered as future home leave medication order.

  3. If there are multiple leaves recorded for the patient and if the order date falls within one of the leaves schedule, the starting date of that leave is defaulted for the order.

  4. If the doctor changes the date manually after the Home Leave Medication check box is selected and that date does not fall within the home leave schedule, then a prompt message is displayed. On clicking OK, the Home Leave Medication check box gets cleared and the order is treated as normal order.

  5. This functionality is applicable only for normal Rx.

  1. Known Case of Allergy – If the patient is allergic to stimulants, then select ‘Yes’. Else, select ‘No’. For patients with drug allergies, you need to record allergy before prescribing drug(s). Only on recording allergy(s), drugs can be prescribed.  If patient is not allergic to any stimulant, then select ‘No’. For non-allergic cases, the practitioner can prescribe drugs without allergy(s) recording. Once you record an Adverse Event, the system displays a pop-up message: “Patient ever had  a drug’s allergy”. If you click ‘No’, the system records and displays the Known case of Allergy in red colour in the Pharmacy Orders screen. If you click ‘Yes’, the system displays the Adverse Event function where you can enter the allergy details.

  1. Clinical Comments - On clicking the ‘Clinical command’, the Clinical Comments screen as shown below gets displayed.

  2. ADR – Click ‘ADR’ hyperlink to view the adverse drug reaction recorded for the external drug.

  3. Pregnancy Remarks - Click Pregnancy Remarks to enter the pregnancy details.

    Enter the following details:

    1. LMP - Enter the LMP in the format of dd/mm/yyyy or select the LMP using the [?] function.

    2. Gestation Period - Enter the Gestation period of the patient.

    3. EDD - Enter the EDD of the patient.

     

  4. Click on 'Diagnosis' hyperlink to view the associated problems of the patient for the prescription. The ‘Diagnosis Description’ screen gets displayed.

If the existing parameter “Allow Medication Order Without Diagnosis” is unchecked in the PH Parameter for Facility screen and patient diagnosis is not already recorded, the following error message is displayed when the user tries to place a new Pharmacy Order for a patient from this screen:

Patient Diagnosis is not recorded. Do you want to record Diagnosis now?

 

You can acknowledge the message by clicking “OK” and capture the patient diagnosis details in the Record Diagnosis  dialog that is displayed.
