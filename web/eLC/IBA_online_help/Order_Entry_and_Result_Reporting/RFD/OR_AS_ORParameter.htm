Enter the following details to create an OR Parameter record:

 

New Orders

  1. Pre-select: Order Category - Select the check box to indicate that the selection of Order Category is mandatory for new orders in Clinician Access module. If the check box is cleared, the selection of Order Category is not mandatory or new orders.

  2. Order Type by Class - The check box is enabled if 'Pre-select: Order Category' is selected. Select the check box to make the selection of Order Type based on Patient Class mandatory. If the check box is selected, in the function 'Ordering Rule by Patient Class', ' Pre-select Order Type' is selected, by default.

  3. Order Search Results Display in Vertical – Select this check box to display the order catalogs and tick sheets vertically in Clinician Access. This check box is cleared by default.

  4. Record Consent Form With Order - Select the check box to record consent form after placing orders as well as from Existing Orders screen. By default this check box is cleared.

 

Time Period for Orders/Results

  1. Max Period to Back - Date Orders (Hrs) - Enter the time in numeric format of three digits maximum length to backdate orders from the system date. You can enter the previous date as ordering date while placing the order.

  2. Max Period to Future - Date Orders (Hrs) - Enter the time in numeric format of three digits maximum length to future date orders from the system date. You can enter the future date as ordering date while placing the order.

  3. Max Period to Back - Date Results (Hrs) - Enter the time in numeric format of three digits maximum length to backdate results from the system date. You can enter the previous date as the day for entering the result.

  4. Maximum Period to Modify Completed Results - Enter the maximum number of days allowed to modify completed results. Select the 'By 24 hrs' check box to consider the period of modification based on the 'Resulted-Complete' date and time.

 

Update Orders

  1. Default Value: User defined 'In-Process' Status - Select the required 'In process' status from the list of valid values. The user defined status created in 'Order Status' functions are displayed in the list. These user-defined statuses are used after registering an order till order completion after which the System defined statuses are displayed. The selected status gets displayed by default in the function 'Review Order Status' and 'Review Line Status' function.

  2. Audit Applicable for Amending Order – Select this check to enable amending orders and audit trail links of Order Entry Formats and Instructions. By default this check box is cleared.

  3. Order Amending Reason Required – This check box gets enabled only when the ‘Audit Applicable for Amending Order’ check box is selected. Select this check box to allow the users to enter the reason for amending orders. The system pop ups the ‘Order Amend Reason’ dialog while amending the Existing Order and will display the recorded reason in all the Audit Links of Order Entry Formats, Comments and Instructions. You can enter the reason for amending or can close the popup without entering the reason.

  4. Order Amending Reason Mandatory – This check box gets enabled only when the ‘Audit Applicable for Amending Order’ check box is selected. Select this check box to make it mandatory to enter the reason for amending orders. You have to select a reason from the predefined list and enter the reason as a free text in the Existing Order screen. A Reason Type ‘Amend’ is available in the Reason Code screen in OR module

 

Future Order

  1. Link future order to the current encounter during visit registration/admission applicable – Select the check box to link future orders to the current encounter during Visit Registration and Admission of the patient. If the check box is selected, the link future order window is populated in the Admission and Register Visit functions.

 

Control by Privilege Group Setup

  1. Ordering – Select the check box to control ordering by privilege group. Clear the check box, if you do not want to control ordering by privilege group. This is one time setup.

  2. Order Authorization - Select the ‘Order Authorization’ check box to control order authorization by privilege group. Clear the check box, if you do not want to control order authorization by privilege group. This is one time setup.

On selecting these check boxes a prompt messages will appear stating ‘Do you want to control ordering or authorization by privilege group setup’ click [OK] to control ordering or authorization by privilege group setup or click [Cancel] to control ordering or authorization by privilege group setup.

 

  1. Apply Order Privilege Rules only – This check box is disabled and not selected by default. This check box is enabled only if the ‘Order Authorization’ check box is selected. If you select select the ‘Apply Order Privilege Rules only’ check box, the system allows practitioner to authorize placed orders based on Privilege definition only and it will not check for Location and Specialty logic to authorize the selected Order in 'Order Authorization' and 'Pending Authorization' screens. The ‘Specialty’ and ‘Location’ filters are disabled in ‘Order Authorization’ screen.

 

Order Clinical Note Result Type

  1. Restrict auto complete of result status to "Resulted-Complete" upon sign for clinical note result type - Select the check box to manually record the result status as ‘Resulted-Complete’ in Order completion function or the status shows ‘Resulted-Complete’ automatically.

 

OT Surgery Order

  1. Preferred Surgery Date ahead - Enter the number of days required to default the date ahead of preferred surgery date. Whenever an OT order is placed, the system defaults that many days ahead as the preferred surgery date."1" day is displayed by default. If no value/zero is entered, the current date is taken as the preferred surgery date. The option 'Prompt for Book Appointment on Order' in the Order Category must be selected in order to prompt for the OT booking while placing an OT order.

 

Specimen Order

  1. Allow Dispatch Slip Barcode Search - Select this check box to display the order records with the Select check box as cleared in ‘Manage Specimen’ screen for Reprint Dispatch Slip. So, that when you scan the specimen number with barcode reader, the selected order gets selected.  You can record and print the Dispatch slip for the selected records. By default this check box is cleared.

 

Order Entry Format

  1. Multiselect List Item Is Mandatory – Select the check box to enable validation for all orders in the Order Entry Format for both header and line level functions. By default, this check box is cleared.

 

Replaceable Orderable

  1. Replace Order through Term – Select the check box to allow order catalog mapping based on Term Set. On selecting the check box, Term Set ID and Term Code fields appear in the ‘Replace’ dialog so that you can select the required Term Set ID and respective term code while replacing the order. By default, this check box is cleared.

 

Cancel option for Daycare and Inpatient

  1. Applicable to Inactive Orders – Select the check box to enable Cancel option for inactive orders with status ‘Ordered’ for all order categories for Day care and Inpatients.
