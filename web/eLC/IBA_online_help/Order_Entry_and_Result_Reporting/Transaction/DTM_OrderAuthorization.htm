Order Authorization  

From this function you can View, Cancel, Amend and Authorize orders pending for verification.

The Order Authorization function is used by senior practitioners to verify and authorize orders placed by other logged in practitioners not privileged to authorize (such as junior practitioners). Using this function practitioners are allowed to amend or cancel orders if the order is not eligible for verification.

Authorizing Orders

From the CA Transactions > Clinician Access > Desktop Menu > Pending Activities > Verifications, select the Order Authorization function. The Order Authorization screen gets displayed.

  1. Orders pending for authorization are displayed for the default date range. Enter the query criteria and click [Search] to display and authorize specific orders. Click [Clear] to clear the displayed data.

    1. Location Type – ‘All’ is displayed, by default. Select the required Location Type from the list. The values displayed are All, Clinic, Day care, Emergency, Nursing Unit and Referral.

    2. Location – ‘All’ is displayed, by default. Depending on the value selected for Location Type, the [?] function displays Locations. If Location Type is selected as ’Clinic’, [?] function displays all the clinics of the logged in facility to which the logged in user has access. If the Location Type is ‘Nursing Unit’, [?] function displays all the Nursing Units of the logged in facility to which the logged in user has access.

    3. Period [From-To] - The date range is displayed, by default. Enter a period in date format of dd/mm/yyyy. This is a mandatory field.

    4. Include Force Authorization - Select the check box to list the orders which requires forced authorization. This check box is selected, by default. On selecting the 'Include Forced Authorization' check box a new list will displayed. Select  the values from the list. The listed values are 'All', 'My Patient', 'My Related Patient' and 'My Location Patient'. If you select 'My Patient' the order displays for which the logged user is the attending practitioner.

    5. Filter - Select the values from the list to filter the orders. The listed item are All, Self, My Speciality and Specialty Specific. All is selected, by default.

    6. All - Select 'All' from the list to display all orders including 'Self' and 'Specialty      Specific' orders.

    7. Self - Select 'Self' from the list to display orders that are ordered on behalf of the logged in practitioner.

    8. Specialty Specific - Select 'Specialty Specific' from the list to display specialty specific orders. If the 'Specialty Specific' check box is selected in Order Catalog function then orders are called Specialty Specific Orders.

    1. Patient ID – Enter the Patient ID in alphanumeric format of maximum length as specified in the MP Parameter.

    1. Status by Priority - The number of orders requiring authorization against each priority is displayed. The displayed priorities are hyperlinked. The total number of orders requiring authorization of all the displayed priorities is shown against 'Al' hyperlink. Click on a priority to view the orders requiring authorization.

    2. Status by Category – The number of orders requiring authorization against each category is displayed. The displayed categories are hyperlinked. The total number of orders requiring authorization of all the displayed categories is shown against 'All' hyperlink. Click on a category to view the orders requiring authorization.

    3. Authorize – Select the check box to select orders for authorization. All the check boxes are cleared, by default. To authorize all the displayed orders, select the check box provided before ‘Authorize’ heading.

    4. Priority - The priority of the order is displayed. The priorities displayed are Routine, Stat and Urgent.

    5. Ordered At - The date and time of ordering is displayed.

    6. Order Catalogs - The Order Catalog is displayed and hyperlinked. Click on the hyperlink to view/cancel/amend the order (based on the user access rights in CA Admin Setup, tasks are displayed).

    7. Order Details – The Order details displayed are the patient and encounter details. The patient details displayed are name of the Patient, Gender, Age and Patient ID. The encounter details displayed are Encounter ID and Name of the Daycare Unit/Emergency Unit/Nursing Unit/Clinic/Referral Location. Click Patient History hyperlink to view the patient history.

    8. Charge Dtls – Click View/Edit hyperlink to view and record charge details.

  2. Click the Order Catalogs hyperlink and select the required option from the list to View, Cancel and Amend orders.

    On clicking Amend option, search for Amend Practitioner screen gets displayed. Search for a practitioner to amend the order by clicking any one of the following options.

    Attending

    1. On clicking the Attending radio button the name of the Attending Practitioner of the patient.

    2. Select the Instruction Mode from the list. The values for Instruction Mode populates from ‘Reason for Code’ function in OR Code Setup. On selecting the value from the Instruction Mode list, the corresponding text is displayed in the text box.

    3. Click [OK] to confirm the selection or click [Cancel] to exit from the screen.

    Medical Team

    1. Click Medical Team radio button to select the Practitioner.

    2. Click any one radio button, either ‘Teams of Attending Practitioner’ or ‘Other Team’ to search for the Practitioner from a Medical Team.

    3. Click [?] function to select a Practitioner. This field is mandatory.

    4. Select the Instruction Mode from the list. The values for Instruction Mode populates from ‘Reason for Code’ in OR Code Setup. On selecting the value from the Instruction Mode list, the corresponding text is displayed in the text box.

    5. Click [OK] to confirm the selection or click [Cancel] to exit from the screen.

    Location

    1. Click Location radio button to select the Practitioner from a location.

    2. Select the Location from [?] function. The values for Location populates from ‘IP Code Setup’ in Inpatient and Day Care Management module.

    3. Click [?] function to select practitioner associated to the selected Location. This field is mandatory.

    4. Select the Instruction Mode from the list. The values for Instruction Mode populates from ‘Reason for Code’ in OR Code Setup. On selecting the value from the Instruction Mode list, the corresponding text is displayed in the text box.

    5. Click [OK] to confirm the selection or [Cancel] to exit from the screen.

    Specialty

    1. Click Specialty radio button to select the Practitioner from a Specialty.

    2. Select the Specialty from [?] function. The values for Specialty populates from ‘Practitioner Setup’ in Application Master Setup.

    3. Select the practitioner from [?] function. Practitioners associated to the Specialty are listed. This field is mandatory.

    4. Select the Instruction Mode from the list. The values for Instruction Mode populates from ‘Reason for Code’ in OR Code Setup. On selecting the value from the Instruction Mode list, the corresponding text is displayed in the text box.

    5. Click [OK] to confirm the selection or click [Cancel] to exit from the screen.

    Self

    1. Click 'Self' radio button to select the user as the Practitioner.

    2. On clicking, Self radio button, the logged in user name is displayed by default in the Practitioner text box.

    3. Click [OK] to confirm the selection or [Cancel] to exit from the screen.

    All

    1. Click All radio button to select a Practitioner from all the practitioners of Enterprise Management.

    2. On clicking the All radio button, search for the Practitioner by clicking [?] function. This field is mandatory. The list of Practitioners is displayed from ‘Practitioner Setup’ in Application Master module.

    3. Select the Instruction Mode from the list. The values for Instruction Mode populates from ‘Reason for Code’ in OR Code Setup. On selecting the value from the Instruction Mode list, the corresponding text is displayed in the text box.

    4. Click [OK] to confirm the selection or click [Cancel] to exit from the screen.

    After selecting the required criteria Place Orders screen displays to amend order details.

    On clicking the Cancel option, the ‘Cancel Order’ screen is displayed.

    1. The screen displays Order ID, Order Category, Order Type, Priority, Patient, Encounter ID, Order Date Time, Order Status, Ordering Location, Ordering Practitioner, Ordering Facility, Performing Facility, Order Format, Clinical Comments and Order Placed By. The screen also displays the Order Catalog, Priority, Frequency, Start Date/Time, End Date/Time, Quality, Status, Format and Reason.

    2. Click Format hyperlink to view the details in the Order Line Format screen. Click [Close] to close the Order Line Format screen.

    3. Click [Schd] hyperlink to view the Schedule Frequency of the order and click [Close] to close the Schedule Frequency screen.

    4. Click Preps, Dept Instrn, Pat Instrn hyperlink to view details, if recorded while creating the Order Catalog.

    5. Select the check box to enter the Reason for cancellation of the order and Cancel Reason screen is displayed.

    6. Select the required Cancel Reason from the list and the description of the selected reason is displayed which can be edited.

    7. Click [OK] to confirm the Reason. The [R] icon below the Reason check box becomes hyperlinked on confirming the reason. Click [R] hyperlink to select another reason for cancelling.

    8. To allot the Reason for all the orders, select the check box provided before ‘Reason’ label.

    9. Click [Record Cancellation] to cancel the order or [Close] to close the Cancel Order Screen.

    On clicking the View option, the ‘View Order’ screen is displayed.

    1. The screen displays Order ID, Order Category, Order Type, Priority, and Patient details, Encounter ID, Order Date Time, Order Status, Ordering Location, Ordering Practitioner, Ordering Facility, Performing Facility, Order Format, Clinician Comments and Order Placed By. The screen also displays the Order Catalog, Priority, Frequency, Start Date/Time, End Date/Time, Quantity, Status, Formats and Comments. The Order Status is shown as ‘Pend Auth’, indicating that order is pending for authorization.

    2. Click Format hyperlink to view details in the Order Line Format screen. Click [Close] to close the Order Line Format screen.

    3. Click [Schd] hyperlink to view the Schedule Frequency of the order and [Close] to close the Schedule Frequency screen.

    4. Click Preps, Dept Instrn, Pat Instrn hyperlink to view details if recorded while creating the Order Catalog.

    5. Click [View Audit Trail] to display View Audit Trail screen and view audit trail details. Click the Remark (R) hyperlink to display remarks for the order. Click [Close] to close the remark screen. Click [x] to exist from the View Audit Trail screen.

    6. Click [Close] to close View Order screen.

  3. Click Patient History hyperlink under Order Details to view Clinical Events recorded for the patient.

  4. Click View/Edit hyperlink under Charge Details to view charge details of the patient. Charge 'Dtls' screen is displayed. Click [Record] to record Charge Details. Click [Cancel] to exit from the screen.

  5. Select the check box against the required orders for authorizing and click [Authorize] to confirm authorization or click [Clear] to clear the selected orders.
