Result Reporting

From this function you can Perform Result Reporting of an Order that is registered.

This function is used to perform resulting of an order that is registered and has been performed. This function is also used to mark completion of an order before reporting the same. Reporting is done either partially or fully based on status of completion. This function allows users to update the status of the order either at header or line level and assign a user defined status to indicate any delays, if applicable. This function is used to result orders across categories and this is based on access rights of the logged in practitioner.

Event status displays ‘In progress’, by default. Based on the entered result, the normalcy indicator provided next to the event status is displayed. Normalcy indicator displays ‘Not applicable’, by default. Discrete measure with result type as numeric, normalcy indicator is displayed (normal low, critical high, critical low and normal high) on entering values. The system evaluates the entered value with the reference range and displays the appropriate normalcy indicator. User cannot select the values. For result type as List items, check box, etc; only ‘Normal’, ‘Abnormal’ and ‘Critically Abnormal’ are displayed, which can be selected by user depending on the entered result value. The values specified in event status are reflected in field ‘Result Status’. If the result type is text, then ‘Result’ hyperlink gets displayed under ‘Result’ column. For the discrete measure, if text blocks are attached or available to all the discrete measures, then the same are displayed under ‘Text’ column of the result.
