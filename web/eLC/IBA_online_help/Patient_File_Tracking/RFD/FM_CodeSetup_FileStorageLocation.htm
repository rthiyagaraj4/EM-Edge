Enter the following details:

 

  1. Code – Enter the File Storage Location Code in alphanumeric format of eight characters maximum length. The File Storage Location Code has to be unique and cannot be changed after creation. This field is mandatory.

  2. Long Description – Enter the long description of the File Storage Location in alphanumeric format of 30 characters maximum length. This field is mandatory.

  3. Short Description – Enter the short description of the File Storage Location in alphanumeric format of 15 characters maximum length. This field is mandatory.

  4. MR Section – Select the MR section from the list. The listed values are defined in MR Section function of Master Patient Index module for the logged in facility. This field is mandatory.

  5. Enabled - Select the check box if the File Storage Location is to be enabled or clear the check box, if the File Storage Location is to be disabled.

 

Characteristics

  1. Belongs to MRD – Select the check box if the defined File Storage Location belongs to the Medical Record Department, or clear the check box.

  2. Permanent – This field is applicable only when Belongs to MRD check box is selected. Select the check box to indicate that the defined File Storage Location is a permanent area for storing patient files. If the check box is cleared, it indicates that the File Storage Location belongs to Medical Record Department but not a permanent area for file storage. If Belongs to MRD is not cleared, this field is not applicable. It indicates the FS Location is a temporary location.

  3. Identity – Select the Identity from the list provided. An Identity is assigned to the defined File Storage Location. This field is mandatory. Refer to the table below for Identity values listing.

  2. Holder Phone No. – Enter the Holder Phone Number in alphanumeric format of 20 characters maximum length.

  3. Narration - Select the Narration to be associated from the list of narrations displayed in the list.

 

File Return Details

The file return details are specified under this section of FS Location function. The file return details are not applicable for permanent File Storage Locations.

  1. Criteria – The condition to return files received by the defined FS Location to the Permanent FS Locations is specified. Select either Discharge Date or File Issue Date.

  2. Discharge Date –The defined FS location has to return the patient files to MRD on discharge of the patient.

  3. File Issue Date –The file return date is calculated from the file issue date.

  4. Duration Type – Select the option as either Days or Hours.

  5. No of Days/Hours to Return (OP/EM/IP/DC) – Based on the Duration Type selected, the field name is displayed as either No of Days to Return or No of Hours to Return. Enter the required period in numeric format of three digits maximum length. Specify the number of days/hours to return for Outpatient (OP), Emergency Patients (EM), Inpatients (IP) and Day care patients (DC).
