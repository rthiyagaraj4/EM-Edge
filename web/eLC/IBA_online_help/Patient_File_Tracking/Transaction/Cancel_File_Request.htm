Cancel File Request

From this function you can Cancel the patient file(s) request placed on the Permanent Locations for the File Storage Location.

The system checks for user access to invoke the function. If rights are defined for canceling request, then the function can be invoked.

Cancelling File Request

From the FM Transactions, select the Cancel File Request function.

The Cancel File Request function screen gets displayed.

  1. Select the FS Location Identity from the list. The values displayed are Department, Clinic, Nursing Unit, Practitioner, Day Care Unit and Procedure Unit.

  2. Select the File Storage Location from the list. The FS Locations are listed based on the access rights defined for the logged in user and the selected FS Location Identity. This field is mandatory.

• If the 'Security Access to be Enforced' check box in FM Parameter Setup is selected, only those File Storage Locations to which the logged in user has access to is displayed in the list.

• If the 'Security Access to be Enforced' check box in FM Parameter Setup is cleared, all the valid File Storage Locations that are associated to the logged in facility are displayed in the list of values.

  1. On selecting the FS Location, the query criteria screen gets displayed.

  2. Enter the Requested Date in the format of dd/mm/yyyy or use the Calendar function to select the required return date.

  3. The name of the person cancelling the file request is displayed. If the 'Security Access to be Enforced' check box in FM Parameter Setup is selected, the logged in User Name is displayed and disabled. If the 'Security Access to be Enforced' check box in FM Parameter is cleared, the User Name is made default to the logged in user, but the other users can be selected using [?] function.

  4. Enter the Patient ID in alphanumeric format of length as specified in MP Parameter or use [?] function to select the required patient. This field is mandatory.

  5. On selecting the Patient ID, the File Type(s) associated to the patient is displayed. On selecting the Patient ID, the default File Type associated in the File Type For FS Location is displayed. This field is mandatory.  

  6. The active volume number is displayed, by default. Select the required Volume Number from the list.

  7. Click [Add] to view the Outstanding Requests for the patient file, [Show All Files] to view all the Outstanding Requests for the selected FS Location or [Clear] to clear the entered data.

  8. Select the ‘Cancel’ check box against the desired patient file.

  9. Click [Apply] to save the record or [Reset] to clear the entered data.
