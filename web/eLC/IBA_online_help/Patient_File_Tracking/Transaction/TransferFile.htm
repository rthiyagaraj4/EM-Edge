Transfer File

From this function you can Record the transfer details of a patient file while transferring files from the current File Storage Location to destination File Storage Location.

The user with appropriate privileges can transfer the file by invoking this function. The files are transferred without the intervention of owner locations. The transfer of files across the facilities of an enterprise is not allowed.

Transferring File

From the FM Transactions, select the Transfer File function.

The Transfer File screen gets displayed.

Transfer From:

  1. Select the FS Location Identity from the list. The listed values are Clinic, Practitioner, Nursing Unit, Day Care Unit and Procedure Unit.

  2. Click Mode of Transport hyperlink to enter the File transportation details. The mode used to transport files from the current location to destination location is specified.

    Enter the following Mode of Transport details:

     

    1. Mode of Transport - Select the mode of transporting the file.

    2. Carried By - If Mode of Transport is manual, enter the person carrying the file.

    Click [OK] to record the details or [Cancel] to exit from the screen.

  3. Select the FS Location from the list. This is a mandatory field. The values are listed based on the conditions.

    • Corresponding to selected FS Location Identity

    • User Access Rights (access to FS Location and to perform Transfer File function)

  1. Click ‘Remarks’ hyperlink to enter the transfer remarks. The Transfer Remarks screen gets displayed. Enter the transfer remarks and click [OK] to save the remarks and close the screen.

  2. Enter the ‘Transfer From’ Location or select the Location using [?] function or enter the location name. The Locations displayed in the query result correspond to the selected FS Location Identity and the FS Location. This is a mandatory field.

  3. Enter the name of the person transferring the file. If the 'Security access to be Enforced' check box is selected in the FM Parameter, the logged in user name is displayed as read-only text. If the check box is cleared, the logged in user name is displayed, by default. You can select the User Name using [?] function. This field is mandatory.

Transfer To:

  1. Select the FS Location Identity from the list. The listed values are Clinic, Practitioner, Nursing Unit, Day Care Unit and Procedure Unit.

  2. Select the FS Location from the list. This is a mandatory field. The listed values correspond to the selected identity and User access for the selected FS Location.

  1. Enter the ‘Transfer To’ location or select the Location using [?] function. The Locations displayed in the query result correspond to the selected FS Location Identity and the FS Location. This is a mandatory field.

  2. On selecting the FS Location, the Narration associated to the selected FS Location during File storage Location is displayed, by default. You can select the required narration from the list.

  3. On selecting the ‘Transfer To’ location, the fields Patient ID, File Type and Volume Number are displayed.

  4. Enter valid Patient ID in alphanumeric format of length as specified in MP Parameter or use [?] function to select the required patient. This is a mandatory field.

  5. On selecting the Patient ID, the File Type(s) and the Volume Number associated to the patient is displayed. If more than one File Type and Volume Number is associated, select the required value from the list.

  6. Click [Add] to view the existing files of the patient for transferring, [Clear] to reset the records displayed in the result area or [Show All Files] to view all files from the selected FS Location.

  7. Select the ‘Select’ check box against the required files to transfer. The check box is selected, by default.

  8. On selecting, click [Apply] to transfer the required file or [Reset] to clear the entered data.

You can access this function if ‘Transfer of File outside MRD is Allowed’ check box is selected in FM Parameter.
