Enter the following details to view the Problem List:

 

  1. Active Only/All - Choose one of the radio buttons to define the kind of diagnosis that is displayed. If Active Only is chosen, all the active diagnosis for the patient is displayed and if All is chosen, both active and closed diagnosis for the patient are displayed.

  2. Type - Select the diagnosis type from the list. The listed values are 'Provisional', 'Differential' and 'Final'. This again defines the kind of diagnosis that is listed in the view. The list in this case is based on the stage at which the diagnosis has been recorded.

  3. Code - Enter the Diagnosis Code in alphanumeric format of five characters maximum length or select the code using [?] function. This criterion filters the view of the particular diagnosis chosen.

  4. Encounter ID - Select the Encounter ID of the patient using [?] function.

  5. Period [From-To] - Enter the period in the dd/mm/yyyy format or select the date from the Calendar function. This restricts the view to list only those diagnoses that have been recorded within the date range specified.

The details that are displayed in the view are based on the query criteria entered above. The diagnosis is sorted based on the stage at which they have been recorded, onset date of the problem and the Encounter ID selected of the patient. The details that are displayed are as follows:

  1. Type - The diagnosis type, that is, the stage at which the diagnosis has been recorded is displayed.

  2. Onset Date - The onset date as entered while recording the diagnosis is displayed.

  3. Description - The diagnosis description that has been recorded is displayed and hyperlinked. If there is a '*' sign against the description, it means that this has been recorded as the primary diagnosis.

  4. Stage - The Stage of the patient is displayed.

  5. Code - The diagnosis code is displayed.

  6. Scheme - The scheme used to record the diagnosis is displayed. This is based on the scheme chosen in MR Parameter.

  7. Close Date - The date on which the diagnosis was closed is displayed. This field is blank for an 'Active' diagnosis.

  8. Supporting Diag Code - The Supporting Diagnosis code is displayed, if it is recorded in the ‘Diagnosis’ function in Clinician Access module.

On clicking the diagnosis description, further audit details regarding the recording of the diagnosis are displayed.
