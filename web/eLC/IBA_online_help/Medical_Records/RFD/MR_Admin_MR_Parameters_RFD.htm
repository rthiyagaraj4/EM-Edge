Death Register

  1. Single Stage Death Registration Applicable – Select the check box to register the death of a patient in single stage. If the check box is cleared, you have to register the death in the Maintain Death Register function in multiple stages.

While preparing the Discharge advice for a patient, the Maintain Death Registration screen is invoked to register the death in the Inpatient and Day Care Management module. The death is registered in the Maintain Death Register function in a single stage.

If post mortem is requested, the finalization of cause of death is allowed only on completion of the post-mortem and only after recording the cause of death in the Mortuary Management module by the practitioner who has performed it. If post mortem is requested while preparing the discharge advice, Maintain Death Register function cannot be invoked in any mode for the patient till cause of death is finalized in Mortuary Management module, irrespective of the parameter ‘Single stage Death Registration Applicable’ is selected or cleared.

 

Allergy Confirmation

  1. Allergy Confirmation Required – Select this check box if you want to confirm allergy for a patient before recording. Select ‘Yes’ to proceed with placing an order. Select ‘No’ if you do not wish to place an order.

  2. Supporting diagnosis mandatory for "Dagger" - Select the check box to make the Supporting Diagnosis mandatory for Dagger.

These parameters identify whether Supporting Diagnosis is mandatory for Consequence or Dagger codes, during Diagnosis Recording.

  1. Max Primary Diagnosis to indicate cause of Death – Enter the number of primary cause of death to be allowed in the Maintain Death Register function, in numeric format of single digit. This is a mandatory field.

  2. Max other disease or complication to indicate cause of Death – Enter the maximum other diseases or complications to indicate cause of Death in the Maintain Death Register function, in numeric format of single digit.

  4. Diagnosis Classification Mandatory – Select this check box if you want the system to check if any active ‘Prinicipal Diagnosis’ has been recorded for the encounter and alert you accordingly with the message “Principal Diagnosis not recorded”, when you record diagnosis in the Record Diagnosis screen. Recode Setup

  1. Allow Recoding - Select the check box to allow recoding of Diagnoses or Procedures.

  2. Allow Procedure Recoding - Select the check box to allow recoding of OT Procedures or Non OT Procedures.

  3. Applicable Patient Class for Recoding - Select the required Patient Class applicable for recoding. The Patient Classes are OP, IP, EM and DC.

  4. Recoding Allowed beyond Maximum level cut off period - Select the check box if recoding is allowed beyond the maximum level cut off period.

  5. Applicable Patient Class - This Patient Classes are enabled if 'Recoding Allowed beyond Maximum level cut off period' field is selected. The Patient Classes enabled are based on the selected patient clasess in the 'Applicable Patient Class for Recoding' field. Select the required Patient Class applicable for recoding beyond the maximum level cut off period.

  6. Grace Period - Enter the grace period in days to allow recoding beyond the maximum level cut off period for the stages selected in 'Stage at which Authorisation is applicable'.

  7. DRG Calculation Applicable - Select the check box if DRG Calculation is allowed. If this check box is selected, the DRG Calculation is done and stored on authorisation. If the DRG Calculation parameter is selected, the DRG calculation happens .The DRG recalculation can be done by selecting the authorised encounter status. DRG Calculation is applicable for Inpatients.

  8. Stage at which Authorisation is Applicable - The list displays values from 1-3. Select the stage at which authorization for the Diagnosis / Procedure is allowed from the list. The Grace Period is applicable to the Stage set in this field. If stage 1 is selected, authorisation is allowed for level 1 and higher level users. If stage 2 is selected, authorisation is allowed for level 2 and higher level users. If stage 3 is selected authorisation is allowed only for level 3 users. Authorisation for users are set in the Recode Privilege Setup function.

  9. Allow new procedure code added in the procedure screen as - Select the value from the list. The listed values are 'Operational', 'OT Procedure' and 'Non OT Procedure'. This list is enabled if 'OT Procedure' or 'Non OT Procedure' check box is selected in the Allow Procedure Recoding.

  10. If the 'OT Procedure' check box is selected in Allow Procedure Recoding, the value 'OT Procedure' is displayed and disabled.

  11. If the 'Non OT Procedure' check box is selected in Allow Procedure Recoding, the value 'Non OT Procedure' is displayed and disabled.
