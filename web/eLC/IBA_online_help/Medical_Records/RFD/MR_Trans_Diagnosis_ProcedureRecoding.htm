Diagnosis/Procedure Recoding

  1. You can recode the Diagnosis or the Procedure. The selected patient's details are displayed in the left hand side of the screen. The displayed details are Name, Sex and Age of the Patient. Click the [+] hyperlink to display the Encounter ID along with the status of recoding.

  2. Click View Clinical History hyperlink to view the event details recorded for the patient.

  3. Click to view the diagnosis marked as 'Error' in the View Error History screen. The codes marked as error will disappear from the recoding screen.

  4. Click to view the excluded codes. You can exclude the codes only if ' Exclude Original codes from Recoding Applicable’ is selected in the MR Parameter function. The Exclude Codes option is available only if the status of the diagnosis is in review state. You can add the same code which is excluded. You can also exclude the newly added codes using Mark Error tab.

  5. Click to view the error codes marked by the practitioner. The icon is displayed if the diagnosis is recorded in the 'Record Diagnosis' function after the review is complete. If recoding is started, on marking error, Diagnosis status remains the same in 'Recoding' function. When recoding is started, the codes marked as 'Error' by a practitioner is displayed in the lisf of 'Show Mark Error Codes'. If the diagnosis or procedure given to the patient is marked as 'Error' before recoding the diagnosis or procedure on the patient, the code is not displayed in the recoding screen.

  6. If recoding is in progress, on marking error, Diagnosis status remains the same in 'Recoding' function. When recoding is in progress, the codes marked as 'Error' by a practitioner is displayed in the list of 'Show Mark Error Codes'.

  7. If the diagnosis or procedure given to the patient is marked as 'Error' before recoding the diagnosis or procedure on the patient, the code is not displayed in the recoding screen.

  8. The indicator indicates the recoding is pending for review.

  9. Select the Encounter ID to display the diagnosis or procedure recorded for the patient on the right side of the screen.

  10. Show - Select the Level from the list. The listed values are 'Original' and 'Level' of recoding. The actual level of recoding is displayed, by default. If recoding is not done, the value 'Original' is displayed.

  11. Icons - The following icons are displayed.

Click to view the recoding History of Diagnosis and Procedure.

Click to view the remarks recorded while recoding Diagnosis and Procedure.

Click to view the Equivalent Term Set Code details of Diagnosis and Procedure.

Click to view the Supporting Diagnosis of the recorded diagnosis.

Click to view the Order Catalogs recorded for procedures.

  1. Select - Select the check box to Recode, Confirm or Mark Error for the selected Diagnosis or Procedure. You can mark the codes as 'NA' if the codes are not applicable for sequencing while performing the DRG and sequencing. The sequencing is performed excluding the codes marked as 'NA'. The order of sequence numbering is dagger followed by its supporting asterisks. An alert is displayed on entering the same sequence number for two codes within a diagnosis group or a procedure group.

  2. Click [New Diagnosis] to record a new diagnosis. The Problem Assessment section is displayed to record details.

  3. Click [New Procedure] to record a new procedure. The Problem Assessment section is displayed to record details.

    Enter the following details to record new procedure:

    1. Procedure Set - The Procedure Set is displayed in the list. Select the Procedure Set from the list. This is a mandatory field.

    2. Procedure - Enter the Procedure code or select it from the [?] function. Click   to enter more than one Procedure. Two additional lines are displayed to enter the procedure details. You can enter a maximum number of three procedure details. This is a mandatory field.

    3. Procedure Type - Enter the code in alphanumeric format of two digits maximum length. This field is displayed if DRG Applicable check box is selected in the MP Parameter function. This field is displayed only for OT Procedures.

    4. Code Extension - The Code Extension field is enabled if Code Extension check box is selected in the Recode Privilege Setup. Enter the Code Extension value in numeric format of two digits maximum length.

    5. Laterality - Select the Lateralitry value from the list. The listed values are Right, Leth, Both and Not applicable.

    6. Check In Date/Time - If the Procedure Type selected in 'OT', the field Check In Date/Time is displayed. The Operation Check in time is displayed by default if OT Procedure details are captured in Operation Theatre module. You can enter the Check In Date/Time in dd/mm/yyyy hh:mm format or select the date and time using the Calendar function.

    7. Start Date/Time - If the Procedure Type selected in 'Non OT', the field Start Date/Time is displayed. Enter the Start Date/Time in dd/mm/yyyy hh:mm format or select the date and time using the Calendar function.

    8. Check Out Date/Time - If the Procedure Type selected in 'OT', the field Check Out Date/Time is displayed. The Operation Check Out time is displayed by default if OT Procedure details are captured in Operation Theatre module. You can enter the Check Out Date/Time in dd/mm/yyyy hh:mm format or select the date and time using the Calendar function.

    9. End Date/Time - If the Procedure Type selected in 'Non OT', the field End Date/Time is displayed. Enter the End Date/Time in dd/mm/yyyy hh:mm format or select the date and time using the Calendar function.

    10. Remarks -  Enter the Remarks in alphanumeric format of 30 characters maximum length.

    11. Click [Record] to record the new procedure details or [Reset] to clear the entered details.

  4. Click [Recode] to recode the selected Diagnosis or Procedure record. The Problem Assessment section is displayed. Enter the details to modify the Diagnosis and Procedure details.

Diagnosis Recoding

Recoding Diagnosis

  1. Select the check box against the recorded Diagnosis and click  [Recode] to recode the recorded diagnosis. The Problem Assessment screen is displayed.

  2. Selected Items - This field is displayed when you select [Recode]. The recorded diagnosis is displayed.

  3. Diagnosis Set - Select the Diagnosis Set from the list. This is a mandatory field.

  4. Diagnosis - Enter the Diagnosis or select the Diagnosis using the [?] function. This is a mandatory field. You can record more than one diagnosis while recoding.

  5. Encounter Stages – Select the required Encounter Stages from the drop-down list. The valid values are ‘Admission’, ’Intermediate’ and ‘Discharge’. This field is visible for Inpatients only. This is a mandatory field.  

  6. Chief Complaint – Click [?] function to select the problems and link it to the diagnosis.

  7. Work Related – Select the required work related value from the drop-down list. This field is mandatory if the Diagnosis at Encounter Level check box is cleared and “Work Related Form Mandatory” check box is selected in MR Parameter screen.

  8. Diagnosis Classification - Select the Diagnosis Classification from the list. The listed values are defined in the Diagnosis Classification function of Medical Records module.

  9. Status - Select the status as 'Active' or 'Exclude Code'. If 'Exclude Original Codes from Recoding Applicable' is selected in MR Parameter, when the recoder selects the patient for recoding for the first time where the original codes are listed for recoding, the 'Exclude Code' option is listed. On selecting 'Exclude Code' option you can exclude the codes from recoding. Exclude code option is applicable only for codes under 'Review' status.

  10. Accuracy - Select the 'Accuracy' value from the list. The listed values are 'Certain', 'Confirmed', 'Differential' and 'Uncertain'. This is a mandatory field.  

  11. Severity - Select the Severity level from the list. The listed values are 'Moderate', 'Mild', 'Severe' and 'Extreme'. This is a mandatory field.

  12. Priority - Select the Priority from the list. The listed values are 'Normal', 'High' and 'Critical'. This is a mandatory field.

The Accuracy and Priority are displayed as set for the Patient Class in the Term Set Criteria for Patient Class function.

  1. Onset Type - Select the Onset Type from the list. The listed values are 'Not Applicable', 'Acute' and 'Chronic'.

  2. Onset Date - The onset date recorded for the previous diagnosis is displayed. Enter the Onset Date or select the Onset Date using the Calendar function. This is a mandatory field.

  3. As on - This date is displayed automatically on entering the Onset Date.

  4. Applicable side - Select the applicable side of the diagnosis.

  5. Remarks - Enter the remarks, if any. Remarks icon is displayed if recorded.

The fields for recoding diagnosis are displayed based on the setup done in ‘Recode Privilege Setup’ function.

  1. Click [Record] to save the recoded diagnosis.

 

Procedure Recoding

Procedure Recoding

  1. Select the check box against the recorded Procedure and click [Recode]. The Problem Assessment screen is displayed.

  2. Selected Item - The selected Procedure is displayed.

  3. Procedure Set – Select the Procedure Set from the list. This is a mandatory field.

  4. Procedure– Select the Procedure using the [?] function. The diagnosis displayed are based on the selected Diagnosis Set. This is a mandatory field.

  5. Extension - Enter the extension code in alphanumeric format of two digits maximum length. This field is displayed if DRG Applicable check box is selected in the MP Parameter function. This field is displayed only for OT Procedures.

  6. Remarks – Enter the Remarks if any. Remarks icon is displayed if recorded.

  7. Click [Record] to save the procedure recoding.

  1. Once the diagnosis is recoded, the encounter is indicated with showing the recoding as Partially Completed. The record is displayed with a + hyperlink and status as Recoded. Click the +hyperlink to view the original diagnosis/procedure and the recoded diagnosis/procedure.

  2. Select the Term Set/Code hyperlink to update the details of the Diagnosis/Procedure without changing the recorded code.

  3. The Description, Classification, Coded By, Date and Time and Caused Death details are displayed.

If the selected diagnosis is selected as the cause of death, the value Y is displayed, or the value N is displayed. If the Diagnosis recorded is the cause of Death, the value Y is displayed within a red box indicating that the recorded diagnosis is a cause of death. On mouse over the cause of Death field, the text ‘Primary’ is displayed, if the recorded diagnosis is a cause of death with the Sequence No. 1. If the Diagnosis recorded is the ‘Related’ cause of Death (Other diseases or complications which caused Death), it is displayed with ‘Y’ indicating that the recorded diagnosis is a Related cause of death. On mouse over the cause of Death field, the text ‘Related’ is displayed.

 

You can recode the asterisk independent code to any other code. You can recode the asterisk dependent code to another asterisk dependent code of the same dagger if more than one dagger dependent asterisk is linked to the Dagger in the ‘Cross Reference Rule’ function.

The icon is displayed if an asterisk is defined for a dagger and if consequence is defined for an external cause in ‘Cross Reference Rule’ function.

 

  1. Click [Confirm] to Confirm the diagnosis/procedure. An alert message 'You are about to confirm the selected code. You wont be able to make anymore changes. Do you want to continue?' is displayed. Click [OK] to Confirm or [Cancel] to cancel the process. The  status is displayed as Confirmed. If the Diagnosis is entered as free text while recording, the [Confirm] is not enabled. Recode the diagnosis to enable [Confirm].

  2. Click [Mark Error] to mark the recoded Diagnosis or Procedure as error. An alert message 'You are about to mark the selected code as an error. You wont be able to revert back. Do you want to continue?' is displayed. Click [OK] to mark the recoding as error or [Cancel] to cancel the process. The record is displayed with a + hyperlink and status as Error. Click the + hyperlink to view the original diagnosis/procedure and the diagnosis/procedure marked as error. If the diagnosis is marked as 'Error' in the diagnosis recording screen after the recoding is in progress for the patient, an alert message 'Recoding of the Diagnosis for the Patient is In Progress and the same will not be marked as error in the Recoding screen. Do you want to continue? is displayed. Select [OK] to mark the diagnosis as error or [Cancel] to revert back to 'Diagnosis/Procedure Recoding' screen.

  3. You can complete the recoding only after confirming the diagnosis. You have to confirm all the diagnoses and procedures of the selected patient to Complete the process. Select the Confirmed Diagnoses/Procedures and click [Complete]. An alert message 'You are about to complete the recoding of Diagnosis/Procedure. You wont be able to make any more changes. Do you want to continue?' is displayed. Click [OK] to Complete or [Cancel] to cancel the process. The record is displayed with status as Completed. Once the diagnosis is completed, the encounter is indicated with showing the recoding as Completed.

  4. You can Authorise or Reject only the completed diagnoses/procedures. Authorising is based on the MR Parameter and Recode Privilege Setup functions. Click [Authorise] to authorise recoding. If the Primary Diagnosis is not selected for the Inpatient, you cannot authorise. Click [Reject] and Recode a Diagnosis with primary diagnosis classification. Confirm, Complete and Authorise. An alert message 'Do you want to Authorise?' is displayed. Click [OK] to authorise the diagnosis. Once the diagnosis is authorised, the encounter is indicated with showing the recoding as Authorised. At any point, click [Reject] to reject the codes. You have to confirm and complete the codes for authorising again.

  5. The new DRG is calculated based on the recoding done to the existing diagnosis and procedures. When the Diagnosis Coding is Authorized, the DRG is calculated using the .exe provided by the DRG department. The calculated DRG values are stored in the database. This value is used for raising the invoice to CG/UC for Inpatients covered.

    The following details are displayed.

    1. MDC – The Major Diagnostic Category code is displayed. These codes are to be stored and used in Finance Report that is discussed in Finance output discussions in the facility.

    2. WARN – Warning code is displayed, when information is wrong but the system still able to find DRG.

    3. DRG – The Diagnosis Related Group Code is displayed. These codes are to be stored and used in Finance Report that is discussed in Finance output discussions in the facility.

    4. WTLOS – The Weight Length of Stay code is displayed. The information is stored against invoice prepared.

    5. RW – Relative Weight captured in Enterprise Management is displayed. This is used for deciding whether the recoding is high cost or normal cost. If RW = 4, it is treated as high cost; if it is less than 4, it is normal. This is used for identifying the type of payer to which the interface is to be done.

    6. OT – The Over Stay Time is displayed. The information is stored against invoice prepared.

    7. Adj RW – The Adjusted Relative Weight (Adjusted by Length of Stay) is displayed. This is used for calculating the reimbursable amount.

    8. Error - Error Code is displayed when DRG code cannot be found. If DRG can be found, Error is equal to 0.

    In case the system cannot find DRG, Error and Warning may not completely be shown because the system stops processing the transaction when Error is occurred, so the information is not completely processed. The values of Warning Codes are as follows:

    0 = DRG is found without warning

    1 = DRG is found without warning

    2 = DRG is found with warning

    3 = DRG is not found without warning

    The values of Error Codes are as follows:

    1 = No Principal Diagnosis

    2 = Invalid Principal Diagnosis

    3 = Unacceptable Principal Diagnosis

    4 = Principal Diagnosis not valid for age

    5 = Principal Diagnosis not valid for sex

    6 = Ungroupable due to age error

    7 = Ungroupable due to sex error

    8 = Ungroupable due to discharge type error

    9 = Length of stay error 9

    10 = Ungroupable due to admission weight error

    1. Click [Reject DRG] to reject the calculated DRG. After you reject the DRG, you have to recode (if needed), Confirm, Complete and Authorise the records once again.

    2. Click [DRG] to view the calculated DRG. This button is enabled only if the DRG details are not displayed on the screen.

    3. Click [Calculate DRG] to recalculate the DRG. This is done after rearranging the sequence numbers of the diagnosis and procedures.

    A Sequence field is attached to each diagnosis and procedures. You can rearrange them as required. If you rearrange them, you have to calculate the DRG again using the [Calculate DRG].

  6. On completing the DRG calculation, click [Save] to save the calculation or [Reset] to reset the calculation.

You can reject the authorized ICD codes for patient class - Emergency, Outpatient, Inpatient, and Daycare. If the lower level of re-coder authorizes the codes, the user of the same level or that of the different level rejects the authorized codes using the ‘Reject’ option, based on the cut off period.
