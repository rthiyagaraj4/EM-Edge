CSSD Store

From this function you can Define and Maintain Sterile Stock store details. You can also Query for the existing records.

A store is a location in a facility that stocks Sterile Stock items. A Store can be an Owner or Requestor location. Stores are defined in the Materials Base module. The applicable transactions are defined based on the Store status (Owner or Requestor).

Defining and Maintaining Sterile Stock store details

From SS Code Setup, select the CSSD Store function.

The CSSD Store screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Select the Store from the list. The Stores defined in the Materials Base module are displayed in the list. This is a mandatory field.

  3. Select the ‘Owner’ check box if the selected Store is an Owner Store. Clear the check box, if the selected Store is a Requestor Store.

  4. The transactions are pre-defined. Based on the store’s status, relevant transactions are enabled. All the Transactions are applicable to the Owner Store. The following transactions are enabled for the Requestor Store:

    • Request Group

    • Return Group

    • Return Instrument

    • Tray Discrepancy

    • Request for Loan

    • Issue against Loan

    • Return of Loan

    • Request Loan From Vendor

    • Return Vendor Loan

  1. Select the ‘Applicable’ check box, if the transaction is applicable to the selected Store. The selected stores can perform the applicable transactions only. The check box is cleared, by default.

The checkboxes under Applicable column are all enabled and unchecked by default, if the store in context is a non-owner store. If the ‘Owner’ checkbox is checked, all three ‘Applicable’ checkboxes of these new transactions are unchecked and disabled.  

  1. Click [Apply] to save the defined CSSD Store or [Reset] to clear the entered data.

  2. Click [Query] to search for existing CSSD Store records.
