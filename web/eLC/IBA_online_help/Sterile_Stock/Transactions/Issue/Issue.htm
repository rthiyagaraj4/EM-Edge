Issue

From this function you can Issue the trays requested by other stores.

The owner store issues trays requested by other stores in this function. The system validates the following conditions while issuing the requested trays.

  • The logged in user should have rights to Issue from the Owner store

  • Both the requesting and owner stores must have access to the requesting group, alternate group and issue group.

Only the sterilized trays currently available with the owner store are issued.

Issuing Trays

From SS Transactions > Issue, select the Issue function.

The Issue screen gets displayed.

  1. Click [Create] to invoke the Pending Issues screen.

  2. All the requests which are pending for issue are displayed. Click on the ‘Doc No’ hyperlink to select the request for issuing or enter the criteria and click [Search] to select a specific record.

    Enter the following criteria to select the record to issue the requested tray:

    1. Doc No. – Enter the request document number in numeric format of eight digits maximum length.

    2. Doc Type – Select the request document type from the list. The values listed are defined in the ‘SS Document Type’ function for the Transaction Type ‘Request Group’ and the default value displayed is specified in the function ‘SS Parameter for Facility’ for request group.

    3. Doc Date - Enter the request document generation date in the format of dd/mm/yyyy or select the date using the Calendar function. The entered date must not be less than the processing month/year specified in ‘SS Parameter for Facility’ function and not greater than current system date.

    4. Requested By Store – Select the requested by store from the list. The stores are listed based on the access rights of the user and applicable transaction.

    5. Requested On Store - Select the owner store from the list. The stores are listed based on the access rights of the user and applicable transaction.

  3. On selecting the Doc No hyperlink, the request group details such as Req Doc Type, Req Doc No, Issue Store, Receiving Store, Authorized By, Doc Type, Doc No, Doc Date and Doc Refare displayed in the Issue screen.

  4. Select the Issue Group from the drop-down list. All the valid requested Issue Groups and its alternate groups that are associated with both the Requesting and Issuing Stores are displayed in the Issue Group drop-down list. Select the Tray number, Bin Location, Expiry Date and the Issue Cost of the selected Tray are displayed. The associated available tray numbers based on the quantity requested are populated in the Tray No. drop-down list excluding the tray numbers that are already chosen for the same Issue Group earlier.

  5. Enter the Patient ID or select the Patient ID using the [?] function to issue the tray for a patient. On selecting the Patient ID, the Patient name is displayed in the Patient Name field.

  6. Select Del check box for the Request Groups that you wish to delete.

  7. Click ‘Req Group’ hyperlink to issue the trays. The Request Group, Authorized Group and Issue Group details are displayed.

  8. On selecting the request document type, the request details are displayed. The displayed details are Request Document Type, Request Document Number, Issue Store, Receiving Store, Authorized By, Document Type, Document Number, Document Date, Document Reference, Request Group, Issue Group, and Authorized Group.

  9. On selecting the Request Group hyperlink, the Request Group, Authorized Group and Issue Group details are displayed.

  10. Enter the Tray number or select the Tray number using the [?] function. All the Sterilized Trays of the selected group is displayed.

  11. Select the Tray No hyperlink to select the tray.

  12. On selecting the Tray number, the Bin Location, Expiry Date and the Issue Cost of the selected Tray are displayed.

  13. Enter the Patient ID or select the Patient ID using the [?] function to issue the tray for a patient. On selecting the Patient ID, the Patient name is displayed in the Patient Name field.

  14. Click [Modify] to record the selected tray details or [Cancel] to clear the entered data.

  15. On clicking [Modify], the Tray number and the Patient ID is updated in the right hand side of the screen.

  16. Select the ‘Del’ check box to exclude the record.

  17. Click [Apply] to record the tray details and issue the tray to the requesting store or [Reset] to clear the entered data.

  18. A document number is generated on issuing the tray.
