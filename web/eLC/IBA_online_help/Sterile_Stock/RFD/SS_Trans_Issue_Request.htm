Enter the following details to request for tray(s):

  1. Doc Type – Select the request document type from the list. The values listed are defined in the ‘SS Document Type’ function and the default value displayed is specified in the ‘SS Parameter for Facility’ function for Request group. This is a mandatory field.

  2. Doc No. – The request document number generation is based on the selected document type. If the document number has to be manually generated for the selected document type, the user is informed. Enter the document number in numeric format of eight digits maximum length. If the system generates the number, you are not allowed to enter the document number. This is a mandatory field.

  3. Doc Date - The system date is displayed, by default. You can change the document date. Enter the date in format of dd/mm/yyyy or select the Date using the Calendar function. The entered date must not be less than the Processing month and year set in ‘SS Parameter for Facility’ and not greater than current system date. This is a mandatory field.

  4. Requested By – Select the request store from the list. Only those Stores are displayed for which the User has access. This is a mandatory field.

  5. Requested On – Select the owner store from the list. Only those Stores are displayed for which the User has access. This is a mandatory field.

  6. Doc Ref – Enter the document reference in alphanumeric format of 15 characters maximum length.

  7. Entry Completed – Select the ‘Entry Completed’ check box to indicate that entry is completed and request document is available for authorizing.  This check box is selected by default.

  8. Entered By – The logged in user name is displayed and disabled.

  9. Request Type - Select the Request Type from the list. The values displayed are Normal, Urgent and Elective. The value Normal is displayed, by default.

  10. Del – Select this check box to delete.

  11. Group Type – Type of group to which the request belongs. The values displayed are Set, Tray, Soft Pack and Single. The value ‘Set’ is displayed, by default.

  12. Req Group – Group to which the request belongs.

You can search for the sterile groups for raising ‘Issue Request’ based on surgery type/name. The Surgery Type/Name search criterion is non-mandatory. The disabled text field next to the Surgery Type/Name search criterion displays the associated OT Surgery Name.  If you don’t select the Surgery Type/Name, the application considers ‘all’ surgery types while performing the Sterile Group Search.

 

The Surgery Type column is also added to the Sterile Group lists column header. The application displays the Surgery Type information of the Sterile Groups being requested, if the information is available.

  1. Req Group – Click [?] to display Group Search and Select dialog. This is a mandatory field.

    1. Search By – Select the Code or Description radio button. The search results are displayed based on selection.

    2. Search Text – Enter the keyword to search the records.

    3. Search Criteria – Select the criteria from the drop-down. The listed values are ‘Starts With’, ‘Contains’ and ‘Ends With’.

    4. Surgery Type/Name – Click [?] to search and select the CSSD Surgery types from the CSSD Surgery Type – Search and Select dialog.

      1. Search By – Select the search by option. The options available are Code and Description.

      2. Search Text – Enter the search text. If you select the search by option as Code, enter the Code in this text box. Else if you select Description, type the description here.

      3. Search - Click this button to view the search results.

      4. Reset - Clears the values entered in the fields.

      5. Surgery Type(Code) - Displays the code of the surgery type.

      6. Surgery Type(Short Desc.) - Displays the short description of the surgery type.

      7. Surgery Name - Displays the surgery name.

      8. Surgery Name(Short Desc.) - Displays the short description of the surgery name.

      9. Nature - Displays the nature of the surgery type.

    5. Group Type – Select the Group Type from the list. The values displayed are Set, Tray, Soft Pack, Single and All. The value ‘All’ is displayed, by default

    6. Consider Groups in Target Store – Select this check box to exclude the sterile groups not associated with the ‘Requested On’ store from the search results. The system excludes the sterile groups not associated with the ‘Requested By’ Store by default.

    7. Click [Search] to display the Code, Description, Surgery Type and Group Type or click [Reset] to clear all values.

    8. Select – You select the check box against each item. Select the Select check box at the header level to select all the records.

  2. Auth Group – The requested group is displayed, by default. You can select an alternate group. Note: When you select the alternate group, the Available.Sterilized Quantity is displayed. The Entry Completed checkbox is enabled by default.

  3. Req. Quantity – Enter the requesting quantity in numeric format of 12 digits maximum length. This is a mandatory field.
