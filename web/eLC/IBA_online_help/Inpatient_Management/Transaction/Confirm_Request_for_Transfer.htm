Confirm Request for Transfer

From this function you can Confirm request for transfers placed by other Nursing Units at the destination Nursing Unit.

The destination Nursing Unit allocates a bed to the Inpatient during the confirmation process. The bed is not blocked for the patient until the patient is transferred to the destination Nursing Unit.

The logged in user must have rights defined to invoke the function. Rights are defined in the Operator Station assigned to the logged in user.

Confirming Request for Transfer

From the IP Transactions > Transfer, select the Confirm Request for Transfer function.

The Confirm Request for Transfer screen gets displayed.

  1. Click [Search] to display the list of all Inpatients with open transfer requests or enter the search criteria and click [Search] to select a specific Inpatient with an open transfer request. Click [Clear] to reset the records displayed in the result area.

    Enter the following query criteria to select the required patient record:

    From Details

    1. Nursing Unit – Enter the Nursing Unit from where the patient has to be transferred in alphanumeric format of 15 characters maximum length or select the Nursing Unit using [?] function.

    2. Specialty – Enter the Specialty from where the patient has to be transferred in alphanumeric format of 15 characters maximum length or select the Specialty using [?] function.

    3. Practitioner – Enter the Practitioner name in alphanumeric format of 30 characters maximum length or select the Practitioner using [?] function.

    To Details

    1. Nursing Unit - Enter the Nursing Unit to where the patient has to be transferred in alphanumeric format of 15 characters maximum length or select the Nursing Unit using [?] function.

    2. Specialty - Enter the Specialty to where the patient has to be transferred in alphanumeric format of 15 characters maximum length or select the specialty using [?] function.

    3. Practitioner - Enter the Practitioner name in alphanumeric format of 30 characters maximum length or select the practitioner using [?] function.

    1. Admission Date From-To - Enter the Admission Date Range in date format of dd/mm/yyyy, or select the date using the calendar function.

    2. Encounter ID - Enter the Encounter ID in numeric format of 12 digits maximum length.

    3. Transfer Type - Select the Transfer Type from the list. The listed values are the values defined in the ‘Transfer Type’ function of the IP Code Setup.

    4. Patient ID - Enter the Patient ID in alphanumeric format of maximum length as specified in MP Parameter or select the Patient ID using [?] function.

    5. Gender – Select the Gender of the patient from the list. The values listed are Male, Female and Unknown.

    6. Patient Name Components – The query components and the legends pertaining to the patient name depends upon the set up done in MP Parameter under the ‘Legal Names’ Tab. The available components for the patient name are Prefix, First Name, Second Name, Third Name, Family Name and suffix. The components that are made applicable for the facility gets displayed as query criteria. The appropriate patient name can be searched by using the search options Starts With, Contains and Ends With.

    7. Search Name With Local Lang – Select the check box to search for the name with the local language.

  2. On searching, Transfer Request details are displayed based on the entered criteria.

  3. Click ‘+’ hyperlink to view the Current Encounter Details.

  4. Click the ‘Encounter ID’ hyperlink to display the Confirm Request for Transfer screen.

  5. Select the Nursing Unit from the lookup if you want to change the nursing location.

  6. Select the Bed Number to which the patient has to be transferred in the Nursing Unit from the bed availability chart by clicking [?] function. This field is mandatory.

  7. Enter the Bed Block Details if the bed has to be blocked in the destination location while confirming request for transfer.

    1. Block Bed – Select the check box if the bed has to be blocked in the destination location while confirming request for transfer.

    2. Period – Enter the Period in hours to block the bed on confirmation of request for transfer. This field is enabled if the ‘Block Bed’ check box is selected and this becomes mandatory.

    3. Override – Select the check box if the bed which is blocked can be overridden and used by another patient. This field is enabled if the ‘Block Bed’ check box is selected.

  8. Click [Confirm] to confirm the transfer request or [Close] to exit from the Confirm Request for Transfer screen.

  9. If internal reports are defined, the Internal Reports screen gets displayed. Click [OK] to generate Internal Reports or [Cancel] to exit from the screen.

 

If the Bed Class selected is of a higher privilege level, as defined against the billing group of the bed class than the privilege level of the patient, as defined against the billing group of the patient, the system pops-up a message asking you to clarify if the bed class change request  was from the patient or the facility. If the bed class change request is from the patient, all the charges for the patient are re-calculated based on the new billing group associated to the bed class selected. If the bed class change request is from the facility, you must authorize it by providing a Username and Pin as defined in System Manager module. If the privilege level of the bed class is lesser than the privilege level associated to the patient, by virtue of the billing group associated to the patient, this authorization is not required and this message does not pop-up.
