Enter the following details to create a new Nursing Unit record:

 

  1. Code - Enter the Nursing Unit code in alphanumeric format of four characters length. A Nursing Unit code is used to identify the Nursing Unit uniquely and cannot be changed after creation. This field is mandatory.

  2. Location Type – Select the Location Type from the list. The values displayed are defined in the function Care Location Type of Application Masters module. Only those care location types defined with the care location type indicator Nursing Unit gets displayed. This field is mandatory.

  3. Long Description – Enter the long description of the Nursing Unit in alphanumeric format of 30 characters maximum length. This field is mandatory.

  4. Short Description – Enter the short description of the Nursing Unit in alphanumeric format of 15 characters maximum length. This field is mandatory.

  5. Enabled -  Select the check box to enable the record in the transaction functions.  The check box is selected, by default.

  6. Applicable To – Select the appropriate value from the list. The listed values are Daycare and Inpatient. This field is mandatory.

  7. Physical Location – Enter the Physical Location of the Nursing Unit in alphanumeric format of 40 characters length.

 

Details Tab

File Details

  1. MR Section Type – Select the required Medical Records Section Type from the list. The files generated for the particular Nursing Unit are maintained either centrally or departmentally. If the type selected is central, the Inpatient files are stored at central level. If the type selected is departmental, the files are stored at departmental levels.

  2. MR Section – Select the required MR Section from the list. When the MR Section Type is selected, the MR sections defined under the selected MR Section Type gets listed. If the selected type is Central, the MR sections defined under central MR section type are displayed. If the MR Section Type selected is Departmental, the MR sections defined as departmental gets listed.

  3. File Storage Location – Select the File Storage Location from the list. The file storage locations are the physical locations within the facility meant for storing patient medical records. The values displayed are defined in the function File Storage Location of the Patient File Tracking module. This field lists the file storage locations that are associated to the MR section chosen with the identity set as Nursing Unit.

  4. Default File Type – Select the Default File Type associated to the file storage location from the list. Once the file storage location is selected, this field displays the file types associated to the storage location.

  5. Auto File Request – Select the check box to enable auto file request. This indicates that the Nursing Unit need not explicitly request for files from the Permanent File Storage location when an inpatient is admitted to the defined Nursing Unit for treatment. The patient files are automatically requested when admitted to the defined Nursing Unit. Clear the check box to disable the option. The check box is selected, by default. If the check box is cleared, the Nursing Unit has to request for patient files explicitly through the Patient File Tracking module.

  6. Auto File Transfer - The check box signifies that the Nursing Unit need not transfer patient files to another location within the current healthcare facility when the patients are transferred. The files are transferred automatically on transferring the patient to another Nursing Unit. The check box is selected, by default. Clear the check box to disable the option. If the check box is cleared, the Nursing Unit has to transfer patient files using the Patient File Tracking module.

 

Characteristics

  1. Nursing Unit Type - Select the appropriate Nursing Unit Type from the list. The Nursing Unit Types created using the Nursing Unit Type function gets listed (only enabled Nursing Unit Types are displayed). This field is mandatory.

  2. Age Group - Select the Age Group of the patients eligible to receive the services provided by the Nursing Unit from the list. This value restricts the admission/transfer of inpatients based on their age and gender depending on age group parameters setup in Application Masters module.

  3. Level of Care - Select the Level of Care from the list. This defines the level of care provided by the Nursing Unit. The values being populated are from the values set up in the Level of Care function.

  4. Allow Transfer W/O Request – Select the check box to transfer the patient without raising a request. If the check box is selected, you can transfer the patient out and in without request. If the check box is cleared, you can raise a request for transfer, confirm the request and transfer the patient out and transfer in.

  5. Check for Same Sex in Room – Select the check box to indicate whether the Nursing Unit comprises of rooms/cubicles that are specially meant to allow admission for inpatients belonging to a specific gender. When selected, the user is prevented from admitting/transferring patients of different genders to any room under the Nursing Unit. The check box is cleared, by default.

  6. Temporary Location – Select the check box to use the Nursing Unit as a temporary location. Temporary locations are created for emergencies where additional Nursing Units are required to meet sudden demands, if required. The check box is cleared, by default.

  7. Teaching Location - Select the check box to use the Nursing Unit as a teaching location. The check box is cleared, by default.

  8. Release Bed on Leave – Select the check box to indicate whether a bed from the Nursing Unit can be released when a patient is on leave. This field can be cleared even if ‘Release Bed on Leave’ field is selected for the current facility in the IP Parameter.

  9. Include for Hospital Occupancy Calculations - Select the check box to include the beds in the Nursing Unit for calculation of hospital occupancy census. The check box is cleared, by default and indicates that beds belonging to a particular Nursing Unit cannot be considered for calculation of hospital occupancy census calculation.

  10. Allow Newborn Regn – Select the check box to allow registration of newborns in the Nursing Unit. The check box is cleared, by default.

 

Pseudo Beds

  1. Allow Pseudo Beds - Select the check box to allow pseudo bed creation and subsequent assignment to the Nursing Unit. Pseudo beds are used to replace the functional beds when patients are sent on leave or can be used as additional beds to meet bed shortages. The check box is cleared, by default.

 

Acct Dept/Service

  1. Account Department – Select the Department from the list. This represents the Cost Center/Administrative Department to which the Nursing Unit is associated. The values displayed are limited to the departments of the logged in facility, which are set up in Application Masters module. This field is mandatory.

  2. Section – Select the Section from the list. This field represents a section in the accounting department to which the Nursing Unit is associated. The values displayed are limited to the sections in the accounting department of the logged in facility.

  3. Service – Select the required Service from the list. This field represents the medical/ancillary service applicable to the Nursing Unit. The list of services displayed is limited to the services offered by the selected department. This field is mandatory.

 

Patient Instructions Tab

  1. Serial No – Enter the Serial No in numeric format of two characters maximum length.

  2. Instruction Description – Select the instruction available in Application Masters - Patient Instruction using [?] function

  3. Instruction ID – The selected Instruction Description ID gets displayed which cannot be edited.

  4. Add – Click [Add] after entering the Serial No and selection of Instruction Description to add the selected instruction to the grid after validation.

  5. Default Select to Print – Select the check box to print the patient instructions by default in the Create Booking and Confirm/Cancel/Revise Booking screens.
