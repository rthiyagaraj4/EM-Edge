Admission Details

Nursing Unit Details

  1. Specialty – Select the Specialty using [?] function. The list of values displayed correspond to the Nursing Unit selected based on the values set up for each Nursing Unit using the IP Code Setup function ‘Specialty for Nursing Unit’. This is a mandatory field.

  2. Nursing Unit – Enter the Nursing Unit or select using [?] function. This field is mandatory. The locations displayed are based on the value chosen for the ‘Applicable to’ flag. If the applicable to flag is chosen as ‘Inpatient’, the list of Nursing Units set up in the facility is displayed. If the applicable to flag is chosen as ‘Day care’, the list of day care units set up in the facility is displayed. The Nursing Unit/Day care listing is based on the operator station assigned to the logged in user. Specialty has to be selected before selecting the Nursing Unit. Selecting just the ‘Specialty’ does not allow selecting bed, unless ‘Nursing Unit’ is selected. If the mouse is moved over the ‘No. of Beds’, ‘Beds Occupied', ‘Beds Blocked’ and ‘Beds Vacant’ the statistics gets displayed in a small screen. The table displays the ‘Bed Class’, 'Total Beds’, ‘Occupied Beds’, ‘Vacant Beds’ and ‘Blocked Beds’ for each Bed Class. Select the required Nursing Unit/Day Care Unit Code. The control shifts back to the ‘Admit Patient’ screen with the selected value.

  3. Service – On selecting the Nursing Unit, the Primary Service attached to the Nursing Unit is displayed. This list contains other services attached to the Nursing Unit. Select the required service.

  4. Sub Service – Select the appropriate Sub Service from the list. The listed Sub Services correspond to the selected Service.

  5. Bed Class - Select the Bed Class from the list. This field is mandatory. The appropriate value in this field is populated if the required ‘Bed No’ is selected from the ‘Bed Availability Chart’.

  6. Bed Type – Select the Bed Type from the list. The Bed Types listed are based on the bed class selected. The Bed Type is defaulted if the ‘Bed No’ is chosen from the ‘Bed Availability Chart’. This field is mandatory.

  7. Bed No. – Enter Bed Number in alphanumeric format of eight characters maximum length, or select the bed number from [?] function. This field is enabled provided, ‘Admit Patient with Bed’ is applicable for the operator station (‘Operator station’ function in System Configuration module) assigned to the logged in user. The beds populated in the ‘Bed Availability Chart’ are based on the Nursing Unit and bed class selected.

  8. Room/Cubicle No. – The Room/Cubicle Number corresponding to the entered bed number is populated if the appropriate bed is selected from the ‘Bed Availability Chart’. The value displayed cannot be modified.

 

Encounter

  1. Admission Date – The current system date and time is displayed, by default. The date can be modified. Enter the required date and time in dd/mm/yyyy hh:mm format. The entered date must not be greater than system date and time. The system evaluates the entered date with the value set in IP Parameter for the field ‘Max Allowable Backdated Admission’. If the entered value is greater than the specified period, an error message is shown to enter/select the appropriate date, or the date can be selected by clicking the Calendar function.

  2. Admission Type – Select the Admission Type from the list. This field is mandatory. The admission types defined in the function ‘Admission type’ are populated in the list.

  3. Ambulatory Status - Select the Ambulatory Status of the patient from the list. The values displayed are defined in the function ‘Ambulatory Status’ of Application Masters module.

  4. Expected Days Of Stay – Enter the Expected Days of Stay in numeric format of three digits maximum length. If the value was entered during booking, the same is displayed. The displayed value can be modified using this function.

  5. Expected Discharge Date – The Expected Date and Time of Discharge is derived and displayed based on the number of days entered in expected days of stay. The displayed value cannot be changed.

  6. Procedure/Surgery Date Time - Enter the date and time in the Date Format ‘dd/mm/yyyy hh:mm’ if the patient has to undergo a surgery or a procedure. The entered date and time must be between the preferred admission date and expected discharge date and time, or select the Date/Time by clicking the Calendar function. This date is defaulted if the same has been entered during the booking function.

  7. Admitting Practitioner - Select the Admitting Practitioner in charge of the patient for the selected specialty using [?] function. This is a mandatory field.

  8. Attending Practitioner - Select the Attending Practitioner in charge of the patient for the selected Specialty using [?] function. This is a mandatory field only if a bed is also being assigned at the stage.

  9. Medical Team - Select the Medical Team in charge of the patient from the list. The medical teams are created in the Medical Team function of Application Masters module. The values listed are filtered based on the medical teams to which the practitioner belongs.

  10. Reason for Admission – Enter the Reason for Admission in alphanumeric format of 40 character maximum length or select the reason for admission using [?] function. This field becomes mandatory if ‘Reason for Admission Mandatory’ check box is selected in the IP Parameter screen.

  11. Admission Remarks - Enter the admission remarks if required, for the patient in alphanumeric format of 250 characters maximum length. For patients with booking, the Booking Remarks entered while booking the patient in Create Booking function is displayed and can be modified. The recorded remarks are displayed during discharge bill generation.

 

Escort/Ambulance Service

  1. Mode of Arrival - Select the Mode of Arrival of the patient from the list. The values displayed are defined in the function ‘Mode of Arrival’ of Application Masters module.

  2. Name - Enter the Name of the person escorting or the name of the Ambulance Service utilized in alphanumeric format of 30 characters maximum length.

  3. Tel No - Enter the Telephone Number of the escort/ambulance service in alphanumeric format of 18 characters maximum length.

  4. MV Regn No - Enter the Registration Number of the Motor Vehicle used to transport the patient to the healthcare facility in alphanumeric format of 15 characters maximum length.

  5. Emergency Detail - Enter the Emergency Details in alphanumeric format of 500 characters maximum length.
