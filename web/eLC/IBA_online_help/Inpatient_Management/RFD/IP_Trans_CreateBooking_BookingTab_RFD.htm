1. Expected Days of Stay – Enter the Expected Number of Days of patient stay in numeric format of three digits maximum length.

  2. Expected Discharge Date Time - The Expected Date and Time of patient discharge is displayed depending on the Expected Days of Stay entered.

  3. Priority - Select the Booking Priority from the list. The listed values are Non-Urgent and Urgent.

 

Patient Details

  1. Patient ID – Enter the Patient ID in the format as specified in the MP Parameter. The ID entered has to be unique and valid. Patient ID entry is mandatory provided ‘Allow Booking for Unregistered Patients’ in IP parameter is not set. On entering the Patient ID, all the demographics of the patient are displayed automatically in the ‘Create Booking’ screen. The Patient ID field can be left blank, if the Patient is not registered in the Facility. In such cases, minimum demographic details has to be entered while creating booking or click the patient [?] function to search and select the required Patient ID. On selecting the required Patient ID, the search patient screen closes and the control shifts automatically to the booking screen. The patient details like Name, Age, Gender, Citizen ID, Alternate ID, Date of Birth, Nationality and Contact Details are displayed for the selected patient and you cannot change the values displayed for the patient.

The fields and their order displayed in the screen may vary depending on the MP Parameter Setup.

  1. Citizen ID - Enter the Citizen ID and other Alternate ID's in alphanumeric format of maximum length as specified in the MP Parameter. The ‘Prompt’ specified for the Citizen ID in MP Parameter gets displayed as the field legend. The Citizen ID and other alternate IDs gets displayed on entering Patient ID of the registered patient. You have to enter it for an unregistered patient.

  2. Previous HN – Enter the Previous Hospital Number in alphanumeric format of maximum length as specified in the MP Parameter.

  3. Government No - Enter the Government Number in alphanumeric format of maximum length as specified in the MP Parameter. Enter the expiry date of the Government Number.

  4. Driving License No - Enter the Driving License Number in alphanumeric format of maximum length as specified in the MP Parameter. Enter the expiry date of the Driving License number.

  5. Expatriate No - Enter the Expatriate Number in alphanumeric format of maximum length as specified in the MP Parameter. Enter the expiry date of the Expatriate number.

  6. Other Alt Type – Select the Other Alt Type from the list. The listed values are Birth Certificate, Passport, Mother’s PMI, PMI, etc.

  7. Other Alt ID - Enter the Alternate ID in alphanumeric format of maximum length as specified in the MP Parameter.

  8. Title - Select the Title from the list. The listed values are defined in the function Name Prefix.

  9. First Name – Enter the First Name of the patient in alphanumeric format of  maximum length as specified in MP Parameter. If the Patient ID is selected, the patient name is displayed and it cannot be changed. The system allows entry of patient name for only unregistered patients. This field is mandatory.

  10. Second Name – Enter the Second Name in alphanumeric format of  maximum length as specified in MP Parameter.

  11. Third Name – Enter the Third Name in alphanumeric format of  maximum length as specified in MP Parameter.

  12. Family Name – Enter the Family Name in alphanumeric format of  maximum length as specified in MP Parameter.

  13. Suffix - Select the Suffix from the list provided. The values are listed from Name suffix function.

  14. Birth Place – For an unregistered patient enter Birth Place in alphanumeric format of 30 characters maximum length, or use [?] function to select the required birthplace. If Citizen ID is entered, Birth Place is displayed based on the Birth Place logic defined in the routine associated with the Citizen ID number. If the required Birth Place is not available in the search list, enter the Birth Place in alphanumeric format of 30 characters maximum length in the text box provided next to [?] function.

  15. Gender - Select the Gender of the Patient from the list. The listed values are Male, Female and Unknown. This field is mandatory. If Patient ID is entered, gender of the patient gets displayed. For an unregistered patient, this needs to be selected.

  16. Age – For an unregistered patient, enter the Age of the patient in years, months and days if date of birth is not known. If age is entered, the system automatically calculates the date of birth and displays it in the 'Birth Date' field. Either Age or Date of Birth can be entered. On entering any one of them, the other field is displayed. For a registered patient, details in the age fields are displayed (and disabled) automatically on entry of Patient ID.

  17. Birth Date - For an unregistered patient enter the Birth Date in dd/mm/yyyy format. If it is entered, system calculates the Patient's age and displays it in the Age field. Either Age or Date of Birth can be entered. On entering one of them, the other field is displayed. This field is mandatory. For a registered patient, Date of Birth field is displayed (and disabled, based on existing registration information) on entry of Patient ID.

  18. Residency - While creating a booking for an unregistered patient, select either Citizen or Non-citizen. Citizen and Legal is selected and disabled, by default. If Citizen ID is entered for the patient, ‘Citizen’ is selected and disabled or, both the values are available for selection. For a registered patient, these details gets displayed (and disabled) on entry of Patient ID.

  19. Status - Select either ‘Legal’ or ‘Illegal’ as status., ‘Legal’ is selected, by default. If the patient is a ‘Citizen’, ‘Legal’ is selected and disabled. If the patient is a ‘Non-Citizen’, status can be either ‘Legal’ or ‘Illegal’. For a registered patient, these details gets displayed (and disabled) on entry of Patient ID.

  20. Nationality – The default Nationality set in MP Parameter is displayed and disabled if Citizen ID is entered, or select the required value using [?] function. This field is mandatory. For a registered patient, Nationality details gets displayed (and disabled) on entry of Patient ID.

  21. Race - Select the race from the list. This field is mandatory. For a registered patient, race gets displayed (and disabled) on entry of Patient ID.

  22. Ethnicity - Select the Ethnicity from the list. The listed values correspond to the selected race. For a registered patient, value in Ethnicity is displayed (and disabled) if previously captured, on entry of Patient ID.

  23. Contact Details – Click the button to enter the address details of the Patient. If address details are already available for a registered Patient, they are displayed on entry of Patient ID, or enter the contact details. On entering the contact details, click [Close] to close the contact details screen.

Case study for patient instructions while rescheduling appointments

Consider that patient instructions 1, 2, 3, 4, 5, 6, 7, 8, 9, and 10 are defined in the Applications Master function in the following manner:

Clinic

Patient Instructions checked by default

Patient Instructions un-checked by default

Clinic A

1, 2, 3

4, 5

Clinic B

7, 8

9, 10

 

While rescheduling appointments, the following logic will be applied:

Reschedule within same clinic with Patient Instructions

  • For Patient A, book an Appointment in Clinic A with Patient instructions 1, 2, 3 and instruction 10 added from 8 to 8.30.

  • Reschedule within Clinic A from 10 to 10.30

When you click Edit instructions hyperlink, you will be able to view 1, 2, 3, 10 checked and 4, 5 un-checked.

Reschedule within same clinic without Patient Instructions

  • For Patient A, book an appointment in Clinic A without Patient instructions by un-checking 1, 2, and  3 from 8 to 8.30

  • Reschedule within Clinic A from 10 to 10 .30

When you click Edit Instructions hyperlink, you will be able to view 1, 2, 3, 4, 5 un-checked.

Reschedule to another Clinic with Patient Instructions

  • For Patient A, book an Appointment in Clinic A with Patient instructions 1, 2, 3 from 8 to 8.30

  • Reschedule to Clinic B

When you click Edit instructions hyperlink, you will be able to view 7 and 8 checked and 9 and 10 un-checked.

 

Reschedule to another Clinic without Patient Instructions

  • For Patient A, book an Appointment in Clinic A without Patient instructions by un-checking 1, 2, 3 from 8 to 8.30

  • Reschedule to Clinic B from 10 to 10.30

When you click Edit Instructions hyperlink, you will be able to view 7 and 8 checked and 9 and 10 un-checked.

Note

  • If you do not click Edit patient Instructions and transfer appointment within the Clinic, the instructions which are captured during book appointment will retained.

  • If you do not click Edit patient Instructions and transfer appointment to other Clinic, then the Default Patient instructions of the selected Clinic will be considered.

  • In case of Bulk transfer within clinic or to other clinics, there will not be any change in Patient Instructions, irrespective of whether the new clinic supports patient instructions or not.

Nursing Unit Details

The Nursing Unit Details are displayed based on a setting within the Operator Station of the logged in user. If ‘Confirm on Create Booking’ is selected in the ‘Operator Station’ function of Application Masters module, the Nursing Unit Details are displayed, or it is not displayed. Only those Nursing Units, which are associated to the Medical Service Group selected and subsequently attached to the specialty are listed. The Nursing Unit/bed can be assigned at the time of booking or later. If a Bed is assigned during booking, a bed blocking is created against the bed by the system. The Bed Avail Chart provides beds available for allocation. You can select the required bed and details of the bed are displayed, or directly enter bed details.

 

  1. Nursing Unit – Enter the Nursing Unit in the alphanumeric format of 15 characters maximum length or select the Nursing Unit using [?] function. All the nursing units associated to the operator station of the logged in user is listed. From among those all enabled nursing units attached to the selected specialty and the nursing unit type of the selected Medical service group is displayed.

  2. Service – The primary service associated to the ‘Nursing Unit’ as set in the master function ‘Nursing Unit’ of ‘IP Code Setup’ is displayed, by default. You can select additional service associated to the nursing unit in the function ‘Service for Nursing Unit’ from the list. For one booking, only one service can be associated at a time.

  3. Bed Class - Select the Bed class from the list. The listed values are the bed classes created in the ‘Bed Class’ function of ‘Inpatient and Day Care Management Code Setup’.

  4. Bed Type – Select the bed type from the list. The bed types are listed based on the Bed class selected.

  5. Bed No - Enter the Bed Number in alphanumeric format of eight characters maximum length. Bed number is assigned to the patient while booking. All the vacant beds of the selected Bed Class, Nursing Unit and Specialty are displayed. On selecting the required bed, the control returns back to the ‘Create Booking’ screen with the bed number and other details populated. The bed details can be selected from the bed availability chart, which is invoked when you click [?] function against the Bed No field. The vacant beds are displayed in Green color. On selecting the vacant bed, the details are populated onto the nursing unit details. The data can be entered.

When the Bed Status is selected as All, the ‘Preferred Admission Date/Time is defaulted as Bed Availability From Date only from Booking. This is to book the beds that are available from the preferred date. The status bar displayed in Bed Availability Chart is as per current availability of bed. The status does not change when date range is provided.

  1. Room/Cubicle No - The Room Number is displayed on entering the bed number.

  2. Override – Select the check box to allow overriding bed blocking. The check box is cleared, by default. If cleared, the booked  bed is blocked and it cannot be overridden and cannot be assigned to other patients.
