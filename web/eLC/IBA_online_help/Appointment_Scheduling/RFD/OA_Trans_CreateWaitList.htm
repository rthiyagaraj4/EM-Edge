Enter the following details to create an appointment waitlist:

 

  1. Specialty – Select the required specialty against which the appointment waitlist has to be booked from the list. All the enabled specialty records created in ‘Application Masters’ module gets displayed. This field is mandatory.

  2. Waitlist Category - Select the wait list category using [?] function. This field gets displayed on selecting the ‘Waitlist Category Applicable’ check box in the OA Parameter.

This field becomes mandatory on selecting the ‘Waitlist Category Required’ check box in the OA Parameter.

  1. Radiology Wait List – Select the Radiology Wait List check box to create waiting list for radiology appointments. This checkbox gets displayed only on selecting the ‘Integration of Radiology Appointments Applicable’ check box in the OA Parameter screen. If this check box is selected then the Order Catalog field and Location field are mandatory and Location Type is defaulted as ‘Procedure Unit’ and disabled.

  2. Order Catalog – Select the Order Catalog from the [?] function.

  3. Priority – Select the required priority from the list of valid values. The listed values are Urgent, Semi Urgent and Normal. This field is mandatory.

  4. Location Type – Select the Location Type as Clinic, Daycare Unit, Nursing Unit or Procedure Unit. This field is mandatory.

  5. Location – Select the required Location based on the selected Location Type. This field is mandatory. The locations are displayed also based on the selected Specialty.

  6. Resource - This field is enabled on selecting a location. On selecting the location, the resource type is displayed against the resource field. Enter the required resource name in alphanumeric format of 30 characters maximum length or click [?] function to search for the required resource.

  7. Preferred Date – Enter the preferred date for creating a waitlist in date format of dd/mm/yyyy format or click on the calendar function to select the required date. This field is mandatory.

  8. Reason for Create Waitlist - Select the reason to create the waitlist from the [?] function. All the enabled reasons created in ‘Application Master’ module are displayed. This field is mandatory.

  9. Reference No - Enter the Reference Number of the patient referred to or within the hospital in alphanumeric format of 12 characters maximum length. The Reference Number helps in identifying patients referred to the hospital for appointment. The Reference Number is used to identify the patients while creating waitlist for patients.

 

Patient Details

  1. Patient ID - Enter the Patient ID (if it is known) in alphanumeric format of maximum length as specified in the MP Parameter. The ID entered must be unique and valid or on clicking [?] function, the Patient Search window gets displayed which enables you to select the required Patient ID. In both the cases, remaining fields in Patient Details are populated from the Master Patient Index. The Patient ID field can be left blank, if the Patient has not been registered with the Facility. In that case, minimum demographic details must be entered.

  2. National ID - Enter the National ID in alphanumeric format of maximum length as specified in the MP Parameter. The ‘Prompt’ specified for the National ID in MP Parameter gets displayed as the field legend. The National ID is displayed, if it is entered in the Patient demographic details tab while registering the patient in Master Patient Index module.

  3. Other Alt ID – Select the required Alternate ID from the list of valid values. The listed values are defined in the function ‘Alternate ID Type’ of Master Patient Index module. On selecting the required Alternate ID type, enter the ID in alphanumeric format of 20 characters maximum length. The field ‘Other Alt ID’ is displayed provided ‘Accept Other Alt ID’ is set in MP Parameter (Alt ID checks tab).

  4. Alternate ID Number - Enter the Alternate ID number in alphanumeric format of maximum length as specified in the MP Parameter. The field title(s) is displayed based on the Alternate ID type(s) set in MP Parameter. Based on the MP Parameter setup, system displays the Alternate ID Type description(s). The Alternate ID is displayed, if it is entered in the Patient demographic details tab while registering the patient in Master Patient Index module.

  5. Patient Name Details - The multi-parts of Patient Name are the Title, First Name, Second Name, Third Name, Family Name and Suffix. Based on the setup in MP Parameter (Legal Names), the multi-parts of patient name are displayed. The displayed titles and order of appearance are specified in the parameter function. On entering the patient’s name, the system saves the patient name based on the derivation logic defined in MP Parameter and displays them in the text area after Patient ID value. This field is mandatory.

  6. Gender - Select the Gender of the unregistered Patient from the list of valid values. The valid values are Male, Female and Unknown. This field is mandatory. For registered patients, the gender selected while registering the patient in Master Patient Index module is displayed.

  7. Age - Enter the Age of the Patient in years, months and days, if date of birth is not known. If age is entered, the system calculates the date of birth and displays it in the ‘Birth Date’ field. Either Age or Date of Birth is entered. On entering one of them, the other field is made read-only and disabled. For registered patients the age entered while registering the patient in Master Patient Index module is displayed.

  8. Birth Date – Enter the Birth Date in dd/mm/yyyy format. If it is entered, system calculates the Patient’s age and displays it in the Age field. Either Age or Date of Birth is entered. On entering one of them, the other field is made read-only and disabled. This field is mandatory.

  9. Birth Place – Enter the Birth Place of the Patient in alphanumeric format of 15 characters maximum length or use [?] function to select the required birthplace. If National ID is entered, birthplace is displayed based on the birthplace logic defined in the routine associated with the National ID. If the required birthplace is not available in the search list, enter the birthplace in alphanumeric format of 30 characters maximum length in the text box provided next to [?] function.

  10. Residency - Select either ‘Citizen’ or ‘Non-citizen’. ‘Citizen’ is selected, by default. If National ID is entered for the patient, ‘Citizen’ is selected and disabled; or both the values are available for selection.

  11. Status - Select either ‘Legal’ or ‘Illegal’ as citizen status., ‘Legal’ is selected, by default. If the patient is a ‘Citizen’, ‘Legal’ is selected and disabled. If the patient is a ‘Non-Citizen’, status can be either ‘Legal’ or ‘Illegal’.

  12. Nationality - The default Nationality set in MP Parameter is displayed and disabled, if National ID is entered, or select the required value using [?] function.

  13. Race - Select the Race from the list. This field is mandatory on selecting the ‘Residency’ option as ‘Citizen’. The listed values are defined in ‘Race’ function.

  14. Ethnicity – Select the ethnicity from the list of valid values. The listed values correspond to the selected race. This field is mandatory.

 

Contact Details

  1. [Contact Details] – To enter the contact addresses of the patient, click [Contact Details] and the Contact Details screen gets displayed. If address details are already available for the Patient, those details are displayed. If not, enter the address details in the columns provided.
