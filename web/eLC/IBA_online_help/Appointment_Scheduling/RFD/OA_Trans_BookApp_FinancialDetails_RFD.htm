Enter the following details:

 

  1. Billing Group – Click [?] function to select the Patient Billing Group which defines if the patient is Cash Patient or Credit Patient or Insurance Patient. The Billing Groups defined in the Patient Billing Manager module gets displayed.

  2. No. of Dependants – Enter the number of dependant family members in numeric format with a maximum of two digits.

  3. Responsible for Payment – Enter the name of the person in alphanumeric format of maximum 30 characters. This indicates the name of the person who is settling the bill amount on behalf of the patient.

  4. Patient Reln with Responsible Person – Click [?] function to select a Relationship. The Relationship selected indicates the association of the person to the patient.

  5. Primary Referral Source - Click [?] function to select a Primary Referral Source. The values defined for Primary Referral Source in the Application Masters with the Visit check box selected gets listed. This field is set mandatory only if the check box ‘Ref Source Main Mandatory’ is selected for the Billing Group in the Patient Billing Manager module.

  6. Secondary Referral Source - Click [?] function to select a Secondary Referral Source. The values defined for Primary Referral Source in the Application Masters with the Visit check box selected gets listed. This field is set mandatory only if the check box ‘Ref Source Sub Mandatory’ is selected for the Billing Group in the Patient Billing Manager module.

 

Settlement Details/Document References

The Settlement Details varies based on the Billing Group selected.

 

When the Billing Group is selected as Cash, enter the following settlement details:

  1. Settlement Type – Select a Settlement Type from the list. This indicates the mode of settlement of the bill amount. This field is mandatory.

  2. Document Reference – Enter the Document Reference Details in alphanumeric format. This field is mandatory.

  3. Document Start Date – Click the Calendar function to indicate the date from when the document reference is considered valid. This field is mandatory.

  4. Document End Date – Click the Calendar function to indicate the date until when the document reference is considered valid. This field is mandatory.

 

When the Billing Group is selected as Sponsor, enter the following settlement details:

  1. Customer Group - Click [?] function to select a Customer Group. The values get listed from the Patient Billing Manager module.

  2. Customer - Click [?] function to select a Customer. This indicates the payer settling a part of the bill amount. The values get listed from the Patient Billing Manager module.

  3. Document Reference – Enter the Document Reference Details in alphanumeric format. This field is mandatory.

  4. Document Start Date – Click the Calendar function to indicate the date from when the document reference is considered valid. This field is mandatory.

  5. Document End Date – Click the Calendar function to indicate the date until when the document reference is considered valid. This field is mandatory.

 

When the Billing Group is selected as Insurance, enter the following settlement details:

If the Payer/Policy details are associated to the patient during patient registration, the values get displayed. If the Payer/Policy Details are not associated, you can associate a Payer using [Add More Secondary Billing Groups] or [Add More Payers]. Click [Modify Payer] to update the Payer details and [Remove Payer] to disassociate the payer from the patient.

 

Click [Add More Secondary Billing Groups] to add additional Billing Groups. Enter the following details:

 

  1. Billing Group – Click [?] function to select a Billing Group. The Billing Groups defined in the Patient Billing Manager module gets listed. This field is mandatory.

A Payer has to be associated to the Billing Group. Click [Add More Payers] to associate a Payer to the Billing Group. Enter the following details:

  1. Payer - Click [?] function to select a Payer. Customers defined in the Patient Billing Manager module gets listed. This field is mandatory.

  2. Priority – Enter the Priority in numeric format with a maximum of two digits. The Priority indicates the precedence of the Payer in the list. This field is mandatory.

  3. Policy - Click [?] function to select a Policy. Policy Types defined in the Insurance Management module gets listed. This field is mandatory.

  4. Policy No – Enter the Policy Number in numeric format with a maximum of 30 digits. This field is mandatory.

  5. Policy Start Date – Click the Calendar function to select a date. The selected Policy Type is considered valid from the date selected. This field is mandatory.

  6. Policy Expiry Date – Click the Calendar function to select a date. The selected Policy Type is considered valid until the date selected. This field is mandatory.

  7. Credit Authorization – Enter the Credit Authorization amount in numeric format. This field is mandatory. The amount indicates the total bill amount that is being settled by the selected Payer. This field is mandatory.

  8. Credit Auth Date - Click the Calendar function to select a date. This field is mandatory.

  9. Policy Effective From – Click the Calendar function to select the date beginning when the Policy is effective. The Policy Start Date is set, by default.

  10. Policy Effective To – Click the Calendar function to select the date until when the Policy is effective. The Policy Expiry Date is set, by default.

  11. Click [Accept] to associate Payers to the Billing Group or [Cancel] to exit from the screen.

The screen gets shifted to Add More Secondary Billing Groups where you can associate additional payers to the Secondary Billing Groups, modify the payer details and remove a payer from the list.

  1. Payer Based – Select the check box to update or to remove the selected Payer details.

  2. Click [Modify Payer] to update the payer details. Enter the details in the mandatory fields. Click [Accept] to save the changes or [Cancel] to exit the Modify Payer window.

  3. Click [Remove Payer] to exclude the payer details. A pop-up appears requesting confirmation to exclude the selected payer. Click [OK] to exclude or [Cancel] to cancel the transaction.

  4. Exit the window on adding additional payers. The screen returns to Financial Details tab where you can view the payer/policy details and update or remove payers.

  5. Non-Insurance Billing Group – Click [?] function to associate a Non-Insurance Billing Group to the selected Billing Group. This field is mandatory.

  6. The Settlement Details vary based on the Billing Group selected. Click [Close] to save the details.
