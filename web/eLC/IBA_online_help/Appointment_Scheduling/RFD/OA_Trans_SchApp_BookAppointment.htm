Book Appointment

Appointments are booked only for a particular Location/Primary Resource. The appointment details for a patient are captured under three tabs, Patient Details Tab, Secondary Resources Tab and Financial Details Tab.

  1. Select the Visit Type and enter other fields. <Related Field Description>

    In case of schedule generated for a timetable type ‘Slot’, on selection of a visit type from the list, you can select the number of slots from the adjacent list next to Visit Type. You can select the values in the list based on the value set for ‘Maximum Appointment Slots’ in Visit Type for Location of Outpatient Management module. If ‘Overriding Number of slots are permitted while Booking Appointment’ the check box is selected in OA Parameter. If the Overriding Number of slots are permitted while booking Appointment’, the check box is cleared and the values set in ‘Maximum Appointment Slots’ in Visit Type for Clinic of Outpatient Management module is defaulted.

  2. Enter the details in the mandatory (*) fields of the Tab pages.

  1. Click [Previous Encounters] to view the previous encounters of the patient if the selected patient had previously received care from the facility.

  2. Click [Other Appts] to view all other appointments of the patient.

    Other Appointment button appears only for patients with current/future day appointments or bookings and having a valid patient ID. When [Other Appts] is clicked, all other appointment of the Patient is displayed. The appointment details are only displayed and cannot be changed.

  3. Click [Recurring Days] to create Recurring Appointments for the patient.

  4. Click [Confirm Booking] to book an appointment. The system allows Overbooking and Forced Booking of appointments.

    The system allows forced in bookings against a particular Location/Resource thus accommodating special cases within a defined appointment schedule for a day. Forced Appointments for a particular Location/Resource can be performed, only if the defined time table is of type ‘Slot’ or ‘Slab’ and value for  ‘No of Forced in Booking’ is set while defining the timetable. Forced Bookings cannot be performed for a future date else the system would prompt the user with a message – Forced Booking possible for future date. User can perform ‘Forced Appointment’ by clicking on any of the already booked time.

    The system allows overbooking against a particular Location/Resource thus allowing certain amount of flexibility in accommodating additional flow of patients within a defined appointment schedule for a day. Overbooking for a particular Location/Resource can be performed, only if value for ‘Maximum Over Bookings’ is set while defining the timetable.

  5. If internal reports are defined, Internal Reports screen gets displayed, click [OK] to generate Internal Reports or [Cancel] to exit from the screen.

  6. Appointment reference number is generated successfully on booking an appointment.

  7. Click [OK] and the control shifts back to Day Schedule Tab and displays the recently booked appointment entry.

An alert message ‘Patient has cancelled appointments X times previously. Click OK to notify the patient else Cancel and Proceed’ is displayed during appointment scheduling when the number of appointments cancelled by the patient reaches the ‘Max Patient Cancelled Appointments for Alert’ count defined in OA Parameter while trying to book, reschedule, cancel or transfer appointment. Click [OK], the cancellation count resets to zero provided the transaction is complete.

The cancellation count is not impacted by Bulk Cancellation or Bulk Reschedule and no alert message is displayed.

 

You will be able to book appointment for radiology appointments without patient ID, when OA and RIS modules are integrated.
