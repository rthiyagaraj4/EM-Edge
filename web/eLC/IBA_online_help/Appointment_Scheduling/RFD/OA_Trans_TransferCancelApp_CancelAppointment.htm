Cancel Appointment

Appointments are cancelled on patient’s request or for any other reasons. This function allows only future appointments to be cancelled.

The Cancel Appointment screen displays the appointment details of the patient.

  1. Click [Contact Details] to view or modify the contact details of the patient.

  2. Select the reason for cancellation by clicking [?] function. All the reasons applicable for appointment cancellation as set in Reason master of Application Masters module is displayed for selection.

  3. Click [Secondary Resources] to view the secondary resources allocated against the appointment.

    The Secondary Resources displays the following details:

    The header displays the Patient details like Name, Gender, Age, Patient Id, Location/Resource, Appointment Ref No, and Appointment Date.

    1. Resource Class – Displays the resource class of the secondary resource.

    2. Resource Type – Displays the resource type of the secondary resource.

    3. Resource Name – Displays the resource name of the secondary resource.

    4. From Time – Displays the time from when the secondary resource is required for the appointment.

    5. To Time – Displays the time until which the secondary resource is required for the appointment.

  4. Click [Confirm Cancellation] to cancel the booked appointment or [Close] to exit from the screen. On successfully cancelling the appointment, a confirmation message gets displayed. Click [OK] and control reverts to the main screen.

  5. Internal Reports screen gets displayed if defined in print routing setup; click [OK] to generate Internal Reports or [Cancel] to exit from the screen.

An alert message ‘Patient has cancelled appointments X times previously. Click OK to notify the patient else Cancel and Proceed’ is displayed during appointment scheduling when the number of appointments cancelled by the patient reaches the ‘Max Patient Cancelled Appointments for Alert’ count defined in OA Parameter while trying to book, reschedule, cancel or transfer appointment. Click [OK], the cancellation count resets to zero provided the transaction is complete.

The cancellation count is not impacted by Bulk Cancellation or Bulk Transfer and no alert message is displayed.
