Enter the following details to create a Document Type record:

 

  1. Based on - Select the value from the list. The pre-defined values ‘Accounting Entity’ and ‘Facility’ are listed. This is a mandatory field.

  2. Doc Type – Enter the Document Type Code in alphanumeric format of six characters maximum length. The Document Type Code has to be unique and cannot be changed after creation. This is a mandatory field.

  3. Long Description - Enter the long description of the Document Type in alphanumeric format of 100 characters maximum length. This is a mandatory field.

  4. Short Description - Enter the short description of the Document Type in alphanumeric format of 40 characters maximum length. This is a mandatory field.

  5. System Generated Document Number – Select the check box to enable the document number to be system generated. The numbers for the documents are automatically generated during transaction when the check box is selected. Clear the check box to generate the document numbers manually.

  6. Start Doc Number – Enter the starting Document Number of the Document Type in numeric format of eight digits maximum length. This field is enabled only if the ‘System Generated Document Number’ check box is selected. This is a mandatory field.

  7. Next Doc Number – On clicking this field the value entered in the ‘Start Doc Number’ field gets displayed. You can enter the next Document Number of the Document Type in numeric format of eight digits maximum length. This field is enabled only if the ‘System Generated Document Number’ check box is selected. This is a mandatory field.

  8. Automatic Print – Select the check box to print the Document Type automatically.

  9. Entry Mode – Select the entry mode from the list. The pre-defined values are ‘Online’ and ‘Batch’.

  10. Module Name - The list of modules that are enabled during installation gets displayed.

  11. Select - Select the check box to associate Modules to the Document Type by selecting the check box provided against the module name. Associating a module to the Document Type implies that the Document Type is going to be used in the respective modules during transaction.
