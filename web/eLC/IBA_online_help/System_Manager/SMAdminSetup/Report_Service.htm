Report Service

From this function you can Create a new Report Service for every module of Enterprise Management and Query for a desired Report Service.

Report Services are background processes of Oracle, created as services of the operating system of the application service in which Oracle 9i Application Server is installed.

The working mechanism of the report service is given below:

The report services are started as part of the application server startup. A report service can start up with one or more execution engines based on the parameter setup.

When a report is to be executed, the report job is submitted to a report service with the required parameters. The submitted report is queued under the specified report service.

The report service picks up the next job from its queue. If an execution engine already loaded is free, it executes the report using that execution engine.

If no execution engines are free and if an additional engine can be started up based on the parameter setup, an additional execution engine is started and the report is executed.

If no execution engines are free and the maximum limit of number of execution engines reached is based on the parameter setup, the report is not executed. Whenever, an execution engine becomes free, the report is picked up from the queue and executed.

The report can be executed in synchronous or asynchronous mode.

• In synchronous mode, the client, who requested for the report, waits till the report execution is completed by the report service.

• In asynchronous mode, the client, who requested for the report, gains the control back on submission of the report and is free to perform any other activity.

• In Enterprise Management, all the reports to be printed are executed in asynchronous mode and all the reports to be viewed are executed in synchronous mode.

During the setup stage, the required report service is derived based on the following guidelines:

The different printers used for printing various reports of Enterprise Management are identified.

Distinct set of printers are derived at. For example, in an installation, HP 1100A, HP4050N, HP6L are the some models of printers in use.

It is a good idea to classify the printers based on online and batch reports to be printed on them. Minimum of one report service for each of the printer model has to be defined. Separate report services has to be created for online and batch reports, even if these reports are printed on same model of the printers.

Depending on the expected load of printers, more than one report service has to be created for same printer models and the report services has to be associated to the printer. You are able to associate only one report service to the printer. This means that all the reports printed on a printer are serviced through the same report service.

Derived set of report services are to be created as services in the operating system of the application server using appropriate commands documented in the installation manual. It is also possible to distribute the report services to different print servers for load balancing purpose.

The report services that are set-up on the application server(s) are to be created as entries in Enterprise Management. This function is used to create entries of the actual report services that are already created on the server.

Creating a Report Service

From the SM Admin Setup > Print Routing, select the Report Service function.

The Report Service screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the details in the mandatory (*) fields. <Related Field Description>

  3. Click [Apply] to save a record or [Reset] to clear the entered data.

  4. Click [Query] to search for an existing Report Service record.
