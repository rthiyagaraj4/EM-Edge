Menu Restriction for Responsibility

From this function you can Assign and Maintain the accessibility and functional rights of a menu associated to a responsibility.

The Menu Restriction for Responsibility function facilitates in managing the access rights for the menus in a responsibility. It permits the administrator to exclude or include the functions in a responsibility. Menu restrictions set using this function overrides the access rights set for a menu in a responsibility. You can modify only the default access rights of a menu.

Assigning Menu Restrictions for a Responsibility

From the SM Admin Setup > Application Security, select the Menu Restriction for Responsibility function.

The Menu Restriction for Responsibility screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Select a Responsibility from the list. This is a mandatory field.

  3. The name of the menu gets displayed on selecting the Responsibility. The top half of the screen displays the restricted submenu/function details. You can also remove the restrictions on the submenus/functions. The lower half allows you to exclude or remove the submenu/functions from the responsibility. <Related Field Description>

  4. On specifying the criteria, click [Search] to display the functions that satisfy the search criteria. <Related Field Description>

  5. On specifying the exclude/remove access rights for submenu/function process, click [Apply] to save the record.

  6. To modify the access rights or restrictions, choose the same responsibility to view the details of the record saved. <Related Field Description>

  7. Click [Apply] to save the record.
