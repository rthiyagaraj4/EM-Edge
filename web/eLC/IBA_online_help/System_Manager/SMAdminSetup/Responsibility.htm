Responsibility

From this function you can Create a Responsibility, Associate it to a Responsibility Group, Menu and Role for a specified date range and Query for a desired Responsibility.

The Responsibility function is used to establish an user’s privilege while working with the application. An end-user can perform multiple responsibilities in a facility. The necessary responsibilities are set-up using this function. Pre-defined responsibilities are packaged with the software, which represent each of the modules supplied and associated to a menu covering all the functions of the module.

 

A constructed menu is associated to the responsibility. When you log-in to the system with a responsibility, you can access all the functions of the associated menu. It is possible to define and restrict the effectiveness of a responsibility to a specific period. During log-in, you have an option to choose from several responsibilities where the user can perform the transactions. The user can switch between several responsibilities using [Switch Responsibility]. The typical examples of responsibility are System Administrator, Registration Clerk, Outpatient Cashier, Lab Technician, Radiologist, etc.

Creating a Responsibility

From the SM Admin Setup > Application Security, select the Responsibility function.

The Responsibility screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the details in the mandatory (*) fields. <Related Field Description>

  3. Click [Apply] to save a Responsibility or [Reset] to clear the entered data.

  4. Click [Query] to search for an existing Responsibility record.
