Document Configuration   

From this function you can Define whether to store the uploaded documents in Server Folder or EM Database.

The Document Configuration function allows the Administrators to choose either the Server Folder or EM Database to store the documents uploaded in PAS and Billing modules. The documents are uploaded in the Documents & Others tab in the Change Patient Details/Register Patient screens. The shared server path must be ‘Read-Write’ accessible to the EM Server. The files must be readable from the respective source where they are stored even if the storage option is changed.  The file formats supported are .jpg, .gif and .pdf. The supported file size is 2GB.

 

Also, you can configure whether to enable or disable document scanning and scanners native scanning settings UI. You can mark whether the scanned image must be stored in PDF or Image Format.

Defining Document Configuration

From the SM Admin Setup > select the Document Configuration function.  

The Document Configuration screen gets displayed.

  1. Click [Query] to invoke the Query screen.

  2. Select the facility for which the configuration must be created from the Facility drop-down list. This is a mandatory field.

  3. Click [Search]. Two section ‘Scan Document Option’ and ‘Document Option’ appears.

  4. Do the required configuration settings in the Scan Document Option section and Document Option section.

    1. Select the required Document Storage Option where you want to store the uploaded documents. The available options are Database (Existing) and Shared Path. By default, the Database (Existing) option is selected.

    2. If you select the Database (Existing) option, the system continues to store the uploaded documents into the EM Database. This is the default option. The 'Patient Identification Documents' and 'Finance Documents' text fields are in disabled state but continues to display the earlier shared path if configured. Else, the field is blank.  

    3. If you select the Shared Path option, you need to enter the shared path where you want to store the documents in the 'Patient Identification Documents' and 'Finance Documents' text fields. These text fields are mandatory and you can enter up to 128 characters. This option allows the Administrator to switch the storage location of the documents being uploaded into a shared Server Folder.

    The shared path format must be in Windows folder path format. Unix/Linux shared paths are not supported. If Shared Path begins with the System Name, it must begin with ‘\\’. The format is: \\SystemName or SystemIPAddress \Folder1\Folder 2

    This section allows you to enable or disable scanning or scanning UI. You can also choose whether to store the scanned documents as PDF or image format.

    1. Scanning Enabled – By default the check box is enabled and not selected. If you select the check box, the system automatically enables ‘Scanning UI Enabled’ check box and selects ‘Document to be saved as’ option as ‘Image’. If you clear the check box, the system automatically clears and disables the ‘Scan UI Enabled’ option and disables ‘Document to be saved as’ field.

    2. Scan UI Enabled – By default the check box is disabled and not selected. It gets enabled only when the ‘Scanning Enabled’ check box is selected.

    3. Document to be saved as – You have options to save the document as PDF or Image. If you select ‘Scanning Enabled’ check box, the system automatically selects ‘Document to be saved as’ option as ‘Image’.

  5. Click [Apply] to save the changes made or click [Reset] to clear the entered data.

 

The system will read a file from the same folder into which the file is uploaded based on the document configuration. The files must be readable from the respective source where they are stored even if the storage option is changed.
