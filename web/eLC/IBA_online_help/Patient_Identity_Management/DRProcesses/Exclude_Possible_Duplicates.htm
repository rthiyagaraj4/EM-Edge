Exclude Possible Duplicates

From this function you can manually Exclude similar set of patient records, which are not duplicates to prevent such Patient IDs from being identified as possible duplicates and Query the excluded possible duplicate records. You can Delete the queried records.

In case of newborn twins of same gender, you can exclude records of twins from considered as duplicates for future search process. Exclude Possible Duplicate process has to be done before invoking the search process. You can invoke this function even when DR Cycle is in progress.

Excluding Possible Duplicates

From the DR Processes, select the Exclude Possible Duplicates function.

The Exclude Possible Duplicates screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the non-duplicate Patient ID1 and Patient ID2 in alphanumeric format as specified length in the Master Patient Index module or select the non-duplicate Patient ID1 and Patient ID2 using the [?] function On selecting the Patient IDs, the Patient Name, Gender, Age and Patient ID is displayed below the Patient ID field.

  3. Click [Apply] to save the record or [Reset] to clear the entered data.

  4. Click [Query] to invoke the query screen.

  5. Enter the Patient ID or click [Execute] to view the excluded duplicate records in Excluded Possible Duplicates frame.

  6. Click ‘+’ hyperlink to view the patient details.

  7. Select the ‘Delete’ check box against the desired patient. Click [Patient Details] to view possible duplicate details. Click [Close] to exit from the Possible Duplicate Details screen.

  8. Click [Delete] to delete the selected patient record. Click [OK] to confirm deletion or [Cancel] to cancel the deletion.
