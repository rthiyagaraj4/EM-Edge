Billed Trx Cancellation Rule

From this function you can Define the cancellation rule for the selected responsibilities. You can Query to view the individual modules to view the cancellation rule and the user restriction. You can also Associate valid users, who can cancel the billed transactions to the responsibilities.  

Billed Trx Cancellation Rule function is used to permit authorized Users to cancel the billed transactions. You can select the modules for which the cancellation rule is applicable. An authorized User is permitted to cancel the transactions against which the bill is generated.

Defining Cancellation Rule for Billed Transactions

From the Administration, double-click Billed Trx Cancellation Rule function.

The Billed Trx Cancellation Rule screen gets displayed.

  1. Enter the details in the mandatory (*) fields found in the following Tab pages.

Cancellation Rule by Service Module

Cancellation Rule By Service Module

The Cancellation Rule by Service Module tab enables you to cancel the billed transactions from the selected modules.

Enter the following details to define the bill cancellation rule for selected modules:

  1. Module ID – Click [List of Values] to select a Module ID. The responsibilities associated to the User and the logged in Facility in the System Manager module gets listed. This is a mandatory field.

  2. Module Name – The Module Name gets displayed.

  3. Allowed to Cancel Billed Trx – Select the check box to permit the User to cancel the billed transactions. Clear the check box and the user cannot cancel the billed transactions for the selected module.

  4. Restrict By User – Select the check box, if relevant users are to be authorized to cancel the billed transactions. The tab ‘Billed Trx Cancellation User’ is enabled on selecting the check box. Clear the check box and the tab is disabled.

Billed Trx Cancellation Rule

Billed Trx Cancellation User

The Billed Trx Cancellation Rule tab enables you to authorize users cancel the billed transactions from the selected modules.

Enter the following details:

  1. Module – The module name for which the Restrict by User option is selected, gets displayed. You cannot update the Module ID and name.

  2. User ID – Click [List of Values] to select a valid User ID. The selected user is authorized to cancel the billed transactions in the respective module. This is a mandatory field.

  3. User Name – The User Name for the selected User ID gets displayed.

  4. Active – Select Active, if the User is an active user.

  1. Click [Commit Form] or press [F10] to save the records or [Clear Record] to clear the entered data.

  2. Click [Execute Query] or press [F8] to search for existing records within the facility.
