Inactive Requests, Approvals

The Inactive Requests, Approvals tab enables you to view the refund requests, which are either inactive or not approved.

 

Enter the following details to view the inactive requests:

 

  1. Status – Select a value from the list. The values are system defined - either “Bill Cancelled” or “Receipt Adjusted/Cancelled” from the list.

  2. Bill Cancelled – Select Bill Cancelled to view the refund requests, which were placed for cancelled bills.

  3. Receipt Adjusted/Cancelled - Select Receipt Adjusted/Cancelled to view the refund requests, which were placed for receipts, which were cancelled or the amount was adjusted.

  1. Nature – Select a value from the list. The values are system-defined – “All”, “Manual”, “Receipt” and “Bill&DCP”.

  2. All – Select All to view all the records for which the refund request has been approved.

  3. Manual – Select Manual to view all the records for which the refund request has been approved manually.

  4. Receipt - Select Receipt to view all the receipt transactions against which the refund has been approved.

  5. Bill&DCP - Select Bill&DCP to view all the bill and DCP transactions against which the refund has been approved.

  1. Req.Dt [From-To] – Click Calendar function to select the date range. The requests placed for the cancelled or inactive bills during the selected date range get listed.

  2. Patient No – Enter the Patient Number to view the refund request for a specific patient. The refund request in an inactive state gets displayed.

  3. Approved By – Enter the name of the Billing Personnel who has approved the refund request.

  4. Appr.Dt [From–To] – Click Calendar function to select the date range. The refund requests approved during the selected date range gets displayed.

  5. Request No – Enter the Request Number generated during refund request using the function ‘Request for Refund’.

  6. Bank Name – Click [List of Values] to select a Bank Name. The refund requests associated to the selected Bank gets displayed. The information recorded using the function ‘Request for Refund’ gets displayed.
