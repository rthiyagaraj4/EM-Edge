Related Field Description

 

  1. Pat Class – Enter the Patient class or select the Patient class using LOV. The LOV lists All, Daycare, Emergency, External, Inpatient, and Outpatient. This field is mandatory.

  2. Cust Grp Code - Enter the Customer Group Code or select the Customer Group Code using LOV. The LOV lists the Customer Group Code defined in the masters. This field is mandatory.

  3. Cust Code – Enter the Customer Code or select the customer code from the LOV. The LOV lists the customer codes which are mapped to the selected customer group code. This field is mandatory.

  4. Policy Type Code – Enter the Policy Type Code or select the Policy Type Code from LOV. The selected customer code policy is displayed in this LOV. This field is mandatory. .

  5. Item Type – Select ‘Material’ or ‘Drug’ from the drop-down list. This field is mandatory.  By default, the value ‘Material’ is displayed.

  6. Item Code - Enter Item Code or select the Item code from LOV. This LOV lists material/drug items based on the item type selected. This field is mandatory.

  7. Item Description – The Item Description is displayed based on the selected the Item Code.

  8. Batch ID – Enter the Batch ID or select the batch ID from LOV. The LOV lists all the batch IDs (including expired batches). The Batch ID contains the selected item code. This field is mandatory.

  9. Cost Indicator - Select the Cost Indicator from the drop-down list. The listed values are ‘Amount’, ‘Percentage’ (%) and ‘Factor’. This field is mandatory. .

  10. Cost – Enter the numerical value except zero. Only positive values can be entered in this field. This field is mandatory.

  11. Amount: If cost indicator is amount, maximum of 5 digits before decimal and 2 digits after decimal can be entered.

  12. Percentage (%): If cost indicator is percentage (%), any number from 1 to 100 can be entered. 2 digits can be entered in decimal (like example 10.50).

  13. Factor: If cost indicator is factor, the price of the item is considered at the time of allocation/dispense and factored by the value given in this field.

  1. Effective [From-To] – Select the Effective date range using calendar function. Any date greater than current date is entered. This field is mandatory.

The combination of same patient class, customer group code, customer code, policy type code, item type, item code, batch ID, cost indicator, and effective from are not allowed. Similarly combination of same patient class, customer group code, customer code, policy type code, item type, item code, batch id, different cost indicator and same effective from are not allowed.
