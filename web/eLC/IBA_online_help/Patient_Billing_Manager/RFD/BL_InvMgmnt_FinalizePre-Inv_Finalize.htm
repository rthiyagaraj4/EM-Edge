The following details are displayed:

 

  1. Doc Type Code - The document type created while generating pre-invoice is displayed. The document type is defined through 'Document Types by Payers' function.

  2. Doc Num - The document number generated while generating pre-invoice is displayed.

  3. Doc Date - The date when the pre-invoice is generated is displayed.

  4. Amount - The amount that has to be settled by the payer is displayed.

  5. Payer Group/Payer/Policy Type Code - The Payer Group, Payer and the Policy Type associated to the patient is displayed.

  6. Click [Save] to save and finalize the pre-invoice details or click [Cancel] to abort the process. The financial details are rolled-back to the financial details of the patient that is recorded during registration.

The pre-invoice details have to be saved within two minutes. The financial details are rolled back, if the records are not saved within the time limit specified. The remaining time limit to save the details are displayed in seconds in the screen.
