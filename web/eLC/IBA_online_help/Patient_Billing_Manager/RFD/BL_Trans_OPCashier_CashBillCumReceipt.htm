Enter the following details to view and to settle the outstanding bills for an Outpatient, Emergency or an External patient:

 

  1. Episode Type – Select an Episode Type from the list. The values are “Outpatient”, “Emergency” and “External”. The Episode Type is selected as “Outpatient”, by default.

  2. Episode No - Enter the Episode Number for the bill generation.

  3. Visit No – Enter the Visit Number with a maximum of four digits.

  4. Patient No - Based on the visit details entered, the patient number is displayed along with the patient details. You cannot change. You can verify whether the correct patient number has been displayed.

  5. Bill Generation Option – You can select an option to generate the bill. The values defined are “Patient”, “Customer” and “Both”. Select a value from the list.

  6. Customer – You can select a Customer from the list when the Bill Generation Option is set to Customer.

  7. Facility - The facility ID has to be selected for the patient who has undertaken the service in the concerned facility.

  8. Age - The age of the patient is displayed in years, months and the days.

  9. Sex - The Gender of the patient is indicated as Male or Female.

  10. User – The name of the user logged-in to the Cash Counter is displayed.

  11. Cash Counter/Shift – The Cash Counter code along with the Shift details is displayed.

 

Search Patient Visit

  1. Location Type – The values are system defined – “Clinic” and “Ward”. Select one of the values from the list. The details are displayed for the Location Type selected.

  2. Location Code – Select a Location Code from [List of Values]. The values are populated in the list based on the Location Type selected.

  3. Episode Type – The values are system defined. Select a value from the list.

  4. Option – The values selected for the ‘Inpatient Bill Type’ is defaulted.

  5. Encounter ID – Enter the Encounter ID of the respective patient. The visit details for the specific encounter are displayed.

  6. Encounter Date Range – Click Calendar function to set the date. This field is mandatory. The details for the encounter between the specified date range are displayed.

  7. Patient No – Enter the Patient Number in alphanumeric format of 20 characters, to see the admission of particular patient.

  8. NRIC – Enter the NRIC number for the respective Patient ID.

  9. Customer [From-To] – Click [List of Values] to select the customers range. The search visits of the customers will be displayed.

  10. Ward Code – Enter the Ward code in alphanumeric format of four characters maximum length. This can be selected from the list of valid values. Leave blank if all the records need to be displayed.

  11. [Search] – Click this button to display the details in the detail block. The details displayed will be of Admission Date, Patient No, Ward Code and description, episode type, Admission No and Unbilled amount. To select a particular admission number for raising receipt, double-click on the specific record from the details displayed.

 

Generate Bill Cum Receipt

The fields ‘Avail. Pat Dep’ and ‘Avail. Pat Prepay’ indicates the adjusted and remaining patient deposit amount and the prepaid amount received from the payer.

  1. Service Location – Select the required service location. The system-defined values are 'IPD' and 'OPD'. The services utilized by the patient from the respective Service Location are displayed.

  2. InterCo. Encounters - The encounter IDs that are generated in the referral facility when the patient is registered are listed. You can filter encounter details to view the services utilized by the patient in referral facility during respective encounter.

 

Outstanding Amount

The Outstanding amount the patient owes to the facility is displayed in red.

  1. Cash - The Patient’s outstanding Bill amount is displayed for user verification. You cannot change this value. It will display only the bills that are payable by the patient.

  2. Credit - The outstanding credit bill amount is displayed for user verification. You cannot change this value. It will display only the bills that are payable by the other party.

 

Patient Other Details

This frame consists of the following fields:

  1. Episode ID – The Episode type is set to ‘Outpatient’, by default.

  2. Episode – The Episode number for the relevant visit is displayed.

  3. Billing Group – The Billing Group assigned to the patient for the respective Episode is listed.

  4. Clinic/Ward – The Clinic or the Ward to which the patient is assigned gets displayed.

  5. Encounter Date – The Encounter date and time is displayed for the selected Episode.

  6. Unbilled - The Patient’s outstanding Unbilled amount is displayed for user verification. You cannot change this value.

  7. Exp Discharge Date – The expected discharge date recorded for the patient in the Inpatient and Day Care Management module is listed.

  8. Deposits – The Deposit amount given by the patient during admission is displayed. You cannot change this value. On entering the Valid Episode and Patient number, the service availed details are displayed. The fields displayed are Service, Description, Gross Amount, Net Amount, Customer Amount/Special Agency Deposit and Patient Amount.

  9. Adjust – Select the check box to adjust the deposit amount against the unbilled amount.

  10. Prepay Amt – If any amount was collected from the patient as the pre-payment amount, the amount is displayed here.

  11. Adjust - Select the check box to adjust the prepayment amount against the unbilled amount.

  12. Gen Bill – Select the check box to generate a bill for the selected encounter.

The transactions that exist for the selected visit are listed.

  1. Service – The Service used by the patient is listed.

  2. Description – The description for the respective package services of the patient is listed.

  3. Queue No – The Queue Number gets displayed.

  4. Episode ID – The Episode ID of the patient gets displayed.

  5. BG – The Billing Group assigned to the patient for the specific visit gets displayed.

  6. Payer – The Payer associated to the patient for the specific visit gets displayed.

  7. Gross Amount – The Gross Amount for the services utilized in the specific visit gets displayed.

  8. Discount – The Discount rendered amount on the bill for the services utilized in the specific visit gets displayed.

  9. Net Amount – The Net Amount for the services utilized in the specific visit gets displayed.

  10. Sel – Select the check box to generate the bill for the services associated to the selected visit.

  11. Approx. Payer Amt – The approximate amount paid by the payer is displayed.

  12. Approx. Patient Amt – The approximate amount paid by the patient is displayed.

  13. Patient Amt – The amount to be paid by the patient for the service is displayed.

  14. Exempt Req – Enter the amount in numeric format, if the patient has requested an exemption. The exemption amount has to be lesser than or equal to the payable amount.

  15. Reason – Click [List of Values] to select the exemption reason. The reasons defined using the function ‘Exemption Setup’ gets listed.

  16. Settlement Amt – Enter the settlement amount in numeric format. The settlement amount has to be lesser than the allocated amount.

 

Settlement Details

  1. Amount Settled - Enter the document amount for this transaction. The amount should be positive only, the amount should not be greater than the outstanding amount. For this comparison, the absolute value of the amounts are considered.

  2. Settlement Type - Click [List of Values] to view and to select a valid settlement type code. If the code entered is valid, then the short description is displayed for user verification.

If settlement type is Cash settlement, the settlement reference and remarks fields are disabled. If settlement type is not Cash oriented, the settlement reference and remark field is accepted.

  1. Cheque/Card No - If settlement type is non-cash, this field mandatory. Enter the details of the Cheque or a Credit Card. For example, if the settlement is by Credit Card, enter the Credit Card identity, and expiry date of the Credit Card. If the settlement type is by Cheque, enter the Cheque details and the date.

  2. Date - Enter the date of the Cheque, if the settlement type is Cheque. If the settlement type is Credit Card, enter the date of the expiry of the Credit Card.

  3. Bank/Card Co. Name/Branch - If settlement type is non-cash, this field is mandatory. For example, if the settlement is by Credit Card, enter the Credit Card company name. If the settlement is by Cheque enter the banks name or banks’ branch name.

Click [List of Values] to view and to select an approved Credit Card company name or an approved bank or branch.

  1. Online Approval – If the check box is selected, Credit Card entered receipt is forwarded to the Credit Card Company immediately. If the check box is cleared, Credit Card receipts are forwarded as batch. In case of Online Approval, Batch Number has to be filled manually.

  2. Batch No. – If online approval is selected, the Batch Number is mandatory to identify the batch of the credit card.

  3. Approval Code – If the settlement type is Credit Card, enter the approval code of the Credit Card.

  4. Sale Draft No. – When settlement is made by Credit Card, sale draft number details captured at the time of swiping have to be recorded.

  5. Payer’s Name - Enter the payer name. This field is mandatory, if the receipt nature code is miscellaneous (MI).

  6. Payer’s Ref Id – When Payer name is captured, is having any reference ID, can be entered as input for this field.

  7. Through Mail – The receipts received through post, for the settlement of bill or deposit has to be recorded by selecting the check box. If the check box is selected, Medical Report Payment receipt is printed.

  8. Remarks – Enter the remarks in alphanumeric format.

After entering required data, click [Process Run] upon which a confirmation screen is displayed. Click [YES] to confirm processing or [NO] to re-enter input parameters.

  1. Receipt Type - Enter a valid receipt type code. Click [List of Values] to view and to select a receipt type code. If the code entered is valid, the short description is displayed for user verification. The account code and department is displayed.

 

Record Adhoc Discount

You can grant a discount for the individual services availed by the patient. Right-click on a selected record under the Discount Column to grant a discount for the respective service. You can specify the value as either percentage or the actual discount amount. An authorization screen pops-up on clicking Adhoc discount option. The authorised users only can login to update the discount. The reasons defined using the function Reasons for Billing Actions in the Masters with the ‘Service Level Discount’ as selected gets displayed. The Net Amount for the service is calculated as ‘Gross Amount-Discount’. Click [OK] to login or [Cancel] to cancel the transaction.

The Gross Amount, Net Amount, Additional Amount for the services and Discount Amount if any towards the selected service gets displayed.

  1. User ID – Enter the User ID in alphanumeric format. A authorised user can only login to update the discount details. This is a mandatory field.

  2. PIN – Enter the PIN number for the respective user. This is a mandatory field.

  3. Use Prev Disc Reason – Select the check box to issue a discount for the same reason selected during the previous transaction.

  4. Adhoc Discount Amt – Enter the discount for the selected service in numeric format.

  5. Actual Gross Amt – The gross amount for the service gets displayed.

  6. Discount Reason – Click [List of Values] to select the Discount Reason. The reasons defined using the function ‘Reasons for Billing Actions’ in the Masters with the ‘Service Level Discount’ as selected gets displayed.
