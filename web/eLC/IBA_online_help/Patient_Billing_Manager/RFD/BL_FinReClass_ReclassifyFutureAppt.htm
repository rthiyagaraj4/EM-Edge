Enter the following details to re-classify the patient details for future admission/visit:

 

  1. Current Admission – Select the check box, if the re-classification is valid from current admission of the patient. Clear the check box, if the re-classification is not valid for current admission of the patient. The Episode ID of the patient is displayed on selecting the check box. You can select the check box, only if the patient is currently admitted to the hospital or availing services from the hospital as an Outpatient.

  2. Future Admission – Select the check box, if the re-classification is valid from future admission of the patient. Clear the check box, if the re-classification is not valid for future admission of the patient.

  3. Future Visits – Select the check box, if the re-classification is valid for future visits of the patient. Clear the check box, if the re-classification is not valid for future visits of the patient.

You cannot select both Future Admission and Future Visit. An error as shown gets displayed on trying to select both the check boxes.

  1. [Clinic Exclusion] – Click [Clinic Exclusion] to exclude the clinics for which the re-classification is not valid for future visit of the patient. This is enabled only if ‘Future Visits’ check box is selected.

  2. Episode Type – The Episode Type of the patient is displayed.

  3. Reason Code - Click [List of Values] to select a reason. The reasons defined using the ‘Reasons for Billing Actions’ function with the ‘Patient Reclassification’ check box as selected get listed. This is a mandatory field.

  4. Classification Type – Select a Classification Type from the list. This is a mandatory field.

  5. Classification Code - Click [List of Values] to select a Classification Code. The codes defined using the ‘Financial Classification Rule’ function for the respective ‘Classification Type’ get listed. This is a mandatory field.

  6. Billing Group - Click [List of Values] to select a Billing Group. The codes defined using the ‘Financial Classification Rule’ function for the respective Classification Type.

  7. Customer Group/Customer – Click [List of Values] to select a Customer Group and Customer. This field is enabled based on the Classification Type selected.

  8. Credit Doc Ref – Enter the Credit Document Reference number in alphanumeric format. This field is enabled based on the Classification Type selected.

  9. Doc Date [Start-End] – Click the Calendar function to select the document start and end date. This field is enabled based on the Classification Type selected.

  10. Reclassification Doc Number -  Enter the Reclassification Document Number in alphanumeric format.

  11. Reclassification Period – The reclassification period is calculated and displayed based on the parameters defined. The start date has to be equal to or greater than the system date. The end date has to be greater than the start date.

 The period between the start and the end date has to be within the ‘Max. Period’ allowed specified in ‘Financial Reclassification Parameters’ function.

  1. Policy details – Enter the policy details if the Classification Type is selected as ‘Insurance’.

  2. [Finalize] – Click [Finalize] to finalize the re-classification details for future encounters. The Users with the ‘Reclassification Allowed’ check box as selected in the ‘Users for Services’ function under the Administration function are authorized to finalize the details.

    1. User ID – Enter the User ID. The Users with the ‘Reclassification Allowed’ check box as selected in the ‘Users for Services’ function under the Administration function are authorized to finalize the details. This is a mandatory field.

    2. PIN No – Enter the PIN number of the respective User. This is a mandatory field.

    3. Remarks – Enter the Remarks in alphanumeric format. This is a mandatory field.
