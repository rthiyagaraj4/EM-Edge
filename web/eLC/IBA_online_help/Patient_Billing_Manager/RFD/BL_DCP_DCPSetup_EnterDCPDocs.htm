Enter the following to enter the DCP documents:

  1. Doc Typ/No - Enter a Document Type and a unique Document Number for the Document Type. If the Document Type code is valid, the short description of the Document Type code is displayed for user verification along with the transaction type code. The Document Type Code and the Document Number cannot be changed after creation. A transaction type has to be associated to the selected Document Type using the function ‘Document Type’ in the Masters. The entry has to be unique. This is a mandatory field.

  2. Patient No - Enter a Patient Number. If the patient number is valid, the Name of the patient, and the Age in Years, Months and Days gets displayed. This is a mandatory field.

  3. Doc Date – Click Calendar function to select a Document Date. The date selected has to be equal to or less than the current system date. This is a mandatory field.

  4. Due Date – The due date is computed by the system, based on the document date and the ‘Normal Credit Days Offered’ for deferred cash patients defined in the BL Parameters. You can override the due date by selecting a Due date using the Calendar function. Due date has to be greater than the Document Date. This is a mandatory field.

  5. Doc Nature - Enter the Document Nature in alphanumeric format. The document nature indicates the reason.

  6. Doc Amt - Enter the Document Amount in numeric format. You can enter zero, if the document amount is not applicable. Note that the Document Amount is not the Outstanding Amount. If the transaction type is "Receipts" or "Cash Control Receipts", the amount entered has to be negative only. If the transaction type is "Invoice/Bill", "Adjustments", “Payments” or "Miscellaneous", the amount entered has to be positive. If the transaction type is “Payments”, enter the right sign, since both signs are acceptable. This is a mandatory field.

  7. Posting A/C - Click [List of Values] to view and select a valid posting account. The description for the respective posting account code gets displayed.

  8. Post mth/yr - Enter the Post month and year in numeric format. The posting month/year cannot be greater than the Billing Parameters post month and year. This is a mandatory field.

  9. Narration - Enter the Narration for the Outstanding Document in alphanumeric format.

  10. Amt Ent. - This is the sum of the amounts entered in the detail block (associated transactions). System calculates the amount automatically and you cannot change the amount.

  11. Doc Ref - Enter the document reference in alphanumeric format.

  12. Bill Type – Click [List of Values] to view and select a Bill Type. The description for the selected Bill Type code gets displayed in the adjacent text box. This is a mandatory field.

  13. Episode – Select an appropriate Episode Type from the list. The values are system defined – “Outpatients”, “Inpatients” and “Referral patients”.

  14. Enter an episode number, if episode type is entered. For referral patients, the episode number field is not valid.

  15. Enter a visit number in case of Outpatients. For Inpatients and Referral patients, the episode number field is not valid.

 

Details

  1. Doc Type – Click [List of Values] to select a Document Type. The values get listed from the Financials Base Manager module. This is a mandatory field.

  2. Number - Enter a unique Document Number in numeric format with eight digits maximum length. System checks if the document type and document number in the detailed block against the document type and the document number in the header block. The Document Type and Number mentioned must not be the same as the original document. This is a mandatory field.

  3. Doc Date – Click Calendar function to select a Document Date. The date selected has to be equal to or lesser than the current system date.

  4. T - The Transaction Type Code for the selected document gets displayed. You cannot update the Transaction Type.

  5. Nat - Enter the Document Nature in alphanumeric format. The narration can be a reason for the transaction with the respective document.

  6. Reference - Enter a Reference for the selected document.

  7. Amount - Enter the Document Amount in numeric format. Zero is a valid entry. This is a mandatory field.

  8. Narration - Enter the Narration for the associated transaction in alphanumeric format.
