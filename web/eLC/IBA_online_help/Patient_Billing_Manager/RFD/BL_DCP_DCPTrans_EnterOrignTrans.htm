Enter the following to create originating transactions:

 

  1. Doc No – Click [List of Values] to select a Document Number. A description for the selected Document Number gets displayed. The Document Type Code and Document Number combination has to be unique. This is a mandatory field.

  2. Date – The system date is displayed as the document date, by default.  

  3. Trx Type - The Transaction Type is displayed based on the Document Type selected. You cannot update the Transaction Type.

  4. Patient No - Enter a valid Patient Number. Right click on ‘Click for User Option’ to search for a patient record. If the patient number is valid, the patient name, gender, age in years, months and days are displayed. This is a mandatory field.

  5. Due Date – The due date is computed based on the Date and the number of days set as ‘Normal Credit Day offered’ in the BL Parameters. Click Calendar function to update the Due Date for the selected patient transaction. Due date has to be greater than the Document Date. This is a mandatory field.

  6. Episode No - If Episode Type is Referral Patient, the Episode Number is not valid. If Episode Type is Inpatient or Outpatient, enter the Episode Number.

  7. Visit No – Visit number is not valid, if the Episode Number is not specified. If episode type is Referral Patient or Inpatient, the Visit Number is not valid. If episode type is Outpatient, enter the Visit Number.

  8. Doc Ref/Nature - Enter the document reference details or nature in alphanumeric format. The Document Reference/Nature is considered as the remarks.

  9. Bill Type - Click [List of Values] to view and to select a valid Bill Type code. The description for the selected Bill Type gets displayed.

  10. Amount - Enter the Document Amount for this document. Zero is a valid entry. If the transaction type is "3" or "5", the amount entered has to be negative only. If the transaction type is "1", "2" or "6", the amount entered has t5 be positive only. You have to take care to put the correct sign in case of transaction type equal to "4", since both signs are acceptable.

  11. Posting A/C – Click [List of Values] to view and to select an account element code. The description is displayed below the posting account code. When the document is posted, the associated Posting Account is debited or credited with the Amount (whether the amount is positive or negative respectively) with the document amount.

  12. Period - The post month and year is made default from the Billing Parameters. Click Calendar function to update the period. The post month and year has to be greater than the Billing parameters post month and year.

  13. Narration - Enter the narration for the originating document. The narration is considered as the remarks for the selected Document Type.

 

Originating Transactional Lines

  1. Posting A/C – Click [List of Values] to view and to select a posting account. The description for the selected posting account gets displayed. When the document is posted, the selected posting account is credited or debited (whether the amount is positive or negative respectively) with the line amount.

  2. Description – The Description for the respective Posting Account gets displayed.

  3. Narration - Enter the narration for the General Ledger distribution record in alphanumeric format.
