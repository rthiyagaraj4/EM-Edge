Enter the following details to generate invoice:

 

  1. Facility – Select the Facility from the list. The facilities associated to the logged in user gets listed.

  2. Trx Type – The Transaction Type is displayed as Invoice, by default.

  3. Customer – Click [List of Values] to select the Customer. The outstanding bill details associated to the selected Customer gets listed under the ‘Outstanding Bills’. You can select the required bills to generate an invoice for the respective bills.

  4. Policy Type - Click [List of Values] to select a Policy Type. The outstanding bill details associated to the selected Customer gets listed under the ‘Outstanding Bills’. You can select the required bills to generate an invoice for the respective bills.

  5. Patient ID - Click [List of Values] to select a Patient ID. An invoice is generated for the outstanding bills of the selected patient who is linked to the selected Customer. The age, gender, name and National ID of the patient gets displayed in the respective text boxes on selecting the Patient ID.

  6. Encounter ID - Click [List of Values] to select an Encounter ID. An invoice is generated for the outstanding bills of the selected patient for a specific Encounter.

  7. Episode Type – Select an Episode Type to search for the respective Episode Types with the outstanding bill details. You can select from the list of outstanding bills of an Episode Type associated to the Customer to generate the invoices for the selected Episode Type.

  8. Inpatient – Select ‘Inpatient’ to search for the Inpatient records with the outstanding bills that are associated to the selected Customer.

  9. Daycare – Select ‘Daycare’ to search for the Daycare patient records with the outstanding bills that are associated to the selected Customer.

 

Range of Bills

  1. Visit/Adm Date [From – To] – Click Calendar function to select the date range. The patient records with the visit registration or admission date matching the selected date range are listed.

  2. Discharge Adv Date [From – To] – Click Calendar function to select the discharge advice date range. The patient records with the discharge advice generated during the selected date range are listed.

  3. [Refresh] – Click [Refresh] to view the outstanding bill and the encounter details to generate the invoice.

  4. Narration – Enter the narration in alphanumeric format of 200 characters maximum length.

 

Encounter Details for Pending Invoice

  1. Patient ID – The Patient ID selected gets displayed, by default.

  2. Episode Type – The Episode Type of the patient gets displayed, by default.

  3. Episode ID – The Episode ID of the patient gets listed, by default.

  4. Visit ID – The Visit ID of the patient gets displayed, by default.

  5. Spl Clinic - The check box is selected if the encounter is related to special clinic.

  6. Acct Seq No – The Account Sequence Number gets displayed. For each patient encounter, when the Policy details are modified, the account sequence number gets updated.

  7. Payer Group - The Payer Group associated to the respective patient encounter is displayed.

  8. Payer - The Payer associated to the respective Payer Group is displayed.  

  9. Policy/Group Code - The Policies associated to the respective Payer and Payer Group are displayed. The Policy Group code is displayed, if multiple Policy Types are grouped under the corresponding Policy Group.

  10. Acct Seq No – The Account Sequence Number gets displayed. For each patient encounter, when the Policy details are modified, the account sequence number gets updated.

  11. Priority – The Priority assigned to the respective Episode while recording the financial details for the patient gets displayed.

  12. Cust Code – The Customer Code associated to the selected Episode while recording the financial details for the patient gets displayed.

  13. Policy Type Code – The Policy associated to the patient while recording the financial details gets displayed.

  14. Admission  Date – The date when the patient was admitted to the facility gets displayed.

  15. Discharge Date - The date when the patient was discharged from the facility gets displayed.

  16. Additional Dtls Reqd – The check box is selected based on the settings defined for the Policy Type in ‘Setup invoice Additional Info’ function.

  17. Captured – The check box is selected, if additional information is recorded for the Policy Type and the selected Customer Code. The check box is cleared, if the details are not captured for the selected record.

  18. [C] – Click [C] to capture additional details for the selected record. You cannot generate the invoice for the selected records, if the additional details are not captured. The patient details are recorded during patient registration are displayed. The Account Sequence Number, Priority, Customer Details, Policy Type and Number are displayed along with the patient details.

    Enter the following details:

    1. [Populate Additional Info] – Click [Populate Additional Info] to populate the additional details setup for the Payer Group, Payer and the Policy Type combination defined using the ‘Setup Additional Info by Payer/Policy’ tab in the ‘Setup Invoice Additional Info’ function. You can associate additional data which is not associated to the Payer Group, Payer and the Policy Type combination by selecting the values from the list.

    2. SL No – Enter the Serial Number in numeric format in four digits of maximum length. This identifies the order in which the additional information has to appear.

    3. Info ID - Click [List of Values] to select an Information ID. The Info ID’s created using the ‘Setup Data for Additional Info Identifiers’ tab using the ‘Setup Invoice Additional Info’ function gets listed.

    4. Code – Click [List of Values] to select a Code. The Codes defined for the selected Info ID gets listed.

    5. Description – The description for the selected Code gets displayed.

    6. Data – The data set while defining the Info ID using the ‘Additional Info Identifiers’ tab in the ‘Setup Invoice Additional Info’ with the ‘Data Set Available’ check box as cleared gets displayed.

    7. Mandatory – Select the check box, if the additional invoice detail is mandatory while generating the invoice. The additional details are not mandatory, if the check box is cleared.

    The button [C] is replaced by [V] adjacent to the record for which the additional details are captured successfully. [V] indicates Victory, which specifies that the additional details have been successfully captured.  

  19. Name - The name of the patient is displayed.

  20. Customer - The description of the Customer is displayed.

  21. Policy Name - The description of the associated Policy Type is displayed.

 

Outstanding Bills

  1. Reference Original Invoice – Click [List of Values] to select the original invoice number. The Original Invoice Number indicates that the invoice generated is in reference to the invoice number, which is already generated and dispatched to the Customer.

  2. [Service Dtls] – Click [Service Dtls] to view the service details for the selected record.

  3. Select All – Select the check box to generate invoice for all the outstanding bills of the patient. Clear the check box, if the invoice has to be generated for selected bills only.

  4. Select – Select the outstanding bills against which the invoices are to be generated. You can select to print a consolidated invoice by selecting a group of outstanding bills or select individual bills to generate separate invoice for each outstanding bill.

  5. Doc Type – The Document Type selected while generating the bill for an Outpatient or Emergency or an External patient gets displayed.

  6. Doc Number - The Document Number generated for the outstanding bill gets displayed.

  7. Date – The date when the bill was generated for the services utilized by the patient gets displayed.

  8. Policy Type – The Policy Type associated to the patient gets displayed. Separate bill is generated for each Policy Type that is associated to the Payer Group.

  9. Pending Amount – The pending amount, which has to be settled, gets displayed.

  10. Patient ID – The ID of the selected patient gets displayed.

  11. Episode Type – The Episode Type of the patient gets displayed.

  12. Episode ID – The Episode ID for the respective patient in the facility gets displayed.

  13. Visit ID – The Visit ID during which the bill was not settled, gets displayed.

  14. Total – The total outstanding bill amount gets displayed.

  15. [Generate] – Click [Generate] to generate the invoice for the selected outstanding bills. The Document Type has to be defined for the Invoice records. A reference number is generated on clicking [Generate]. If the diagnosis is recorded for the selected Policy Type and the diagnosis details are not recorded, the indication is displayed with  color. You cannot generate the bills, if the diagnosis is not recorded.
