Enter the following details to record the Credit Card Payment details:

 

Create

  1. Credit Card Type – Click [List of Values] to select a Credit Card Type associated to the Credit Card company, from whom the payment is received. The description for the selected Credit Card Type gets displayed. This is a mandatory field.

  2. Receipt Type – Click [List of Values] to associate a Receipt Type. The values defined using the Receipt Type function get listed. This is a mandatory field.

  3. Amount Settled – Enter the payment received from the Credit Card company via Cheque or Demand Draft in numeric format. This is a mandatory field.

  4. Settlement Type – Click [List of Values] to associate a Settlement Type. The values defined using the Settlement Type function in the ‘Accounting Entity Setup’ menu get listed.

  5. Instrument No – Enter the Instrument Number in numeric format of 20 digits maximum length. The Instrument No reflects the Cheque or the Demand Draft number through which the payment is received from the Credit Card company. This is a mandatory field.

  6. Date – Click Calendar function to select the date. The date mentioned in the Cheque or the DD issued by the Credit Card Company is entered. This is a mandatory field.

  7. Bank & Branch – Enter the Bank and the Branch name in alphanumeric format of 30 characters maximum length. This represents the Bank and the Branch name, where the Cheque or the Demand Draft is drawn.

  8. Payer’s Name – Enter the Payer’s Name in alphanumeric format of 40 characters maximum length. The Credit Card company name can be set as the Payer’s Name. This is a mandatory field.

  9. Payer’s No – Enter the Payer’s ID or any other reference number in numeric format of 30 digits maximum length. This is a mandatory field.

  10. Narration – Enter the Narration in alphanumeric format of 50 characters maximum length. A brief description of the payment collected from the Credit Card company can be set as its input.

Press [Tab] or [F8] or click [Execute Query]. The system populates the list of records with the outstanding credit card batch details.

 

Outstanding Credit Card Batches for Payment

  1. Select All – Select the check box to choose all the records populated in the detail block to generate a batch of credit card documents with outstanding payment details. If the check box is cleared, all records marked for selection are reverted to the status of unselected . If the check box is selected, the selected record amount is collected as the batch amount, which is displayed as “Total”.

  2. Select – Select the records to process the payments in batch. The records, which are not selected are not considered towards the batch processing. If the check box is selected, the document amount is accumulated as the batch amount, which is displayed as the “Total”.

  3. Approval No – The Approval Reference Number for each Document Reference is displayed. You cannot update the Approval Number.

  4. Batch Date – The system defaults the date when  the batch is processed. The Credit Card batch is generated using the function Credit Card Batch Entry. You cannot change the Batch Date.

  5. Batch Document No – The Batch Document Number represents a batch of documents processed. The Batch Document Number generated using the function Credit Card Batch Entry gets displayed. The system defaults the value  in this field.

  6. Credit Card Receipt No – The system displays the generated receipt number. The system defaults the value in this field. The  Receipt Number created against batch of credit card.

  7. Patient No - The system displays the Patient ID for whom the receipt was generated against the credit card payment.

Click [Commit Form] to generate individual for the list of payment entries for outstanding records. A new Batch record is generated for the entire list of detail record upon confirmation. The information of Receipt Number generated and Batch number generated is displayed as a Note window pop-up.
