Refund Cash Book Process

From this function you can Generate Replacement Vouchers and Create Allocation Limits. You can Update Replacement Vouchers or Allocation Limits. You can Query to view the Allocation transactions or Cash Book details.

Refund Cash Book Process function is used to view refund transactions, replacement voucher details, replace the existing voucher details and create or increase the allocation limit for a Cash Book. It is used to process the Refund Cash Book. This function permits you to view the Allocation Transactions performed by an Accountant General. You have a provision to preview the Cash Book voucher before it is printed. Authorized users only can access ‘Generate Replacement Voucher’, ‘Maintain Alloc.Trx/Repl.Vrs’ and ‘Create/Increase Allocation Limit’ tabs to perform transactions.

Processing Refund Cash Book

From the External Accounting System > Refund Cash Book, double-click Refund Cash Book Process Function.

The Refund Cash Book Process screen gets displayed.

  1. Click [Authorize] to login and record the details in the mandatory (*) fields in the following tabs:

    Enter the following details:

    1. User ID – Enter the User ID. This is a mandatory field.

    2. Pin No – Enter the Pin Number for the relevant User ID. This is a mandatory field.

    Click [OK] to proceed with the transaction. On clicking [OK], the control is shifted to the main screen. Click [Cancel] to terminate the transaction.

Create/Increase Allocation Limit

An authorized user can create an allocation limit or increase the voting limit for amount received.

Enter the following details to increase or create allocation limit:

Voting Cheque Details

  1. Cheque No – Enter the Cheque Number in alphanumeric format with a maximum of 30 characters. This is a mandatory field.

  2. Cheque Date – Click the Calendar function to set the date the cheque is issued. This is a mandatory field.

  3. Bank – Click [List of Values] to select a Bank from the list. This specifies the name of the Bank where the Cheque is issued.

  4. Cheque Received Date – Click the Calendar function to set the date the cheque is issued. This is a mandatory field.

  5. Cheque Amount – Specify the amount received through Cheque. This is a mandatory field.

  6. Remarks – Enter the Remarks in alphanumeric format. This gives an overview of the Allocation Limit created.

  7. Cash in Hand – Select this check box if the transaction is cash in hand transaction. cash in hand.

Document Details

The document type is displayed based on the document type set for ‘Refund Cash Book Doc’ in the ‘Setup External Acct Parameters’ function under the ‘Set Up’ menu. System automatically generates the document number, if ‘System Generated Document Number’ check box is selected in the function ‘Maintain Document Type Details’ function in the Financials Base Manager module. If the check box is cleared, the Document Number has to be entered manually by the user. The system date is displayed, by default.

Maintain Alloc.Trx/Repl.Vrs

Maintain Allocating Transactions/Replacement Vouchers tab enables you to maintain the allocation and Replacement voucher details. An authorized user is only permitted to login and update the details.

The following details are displayed on entering the details:

Document Details

The document details recorded using the ‘Create/Increase Allocation Limit’ tab are displayed. You cannot update the details.

Cheque Details

If the cheque details are already mentioned under the ‘Create/Increase Allocation Limit’ tab, the details are displayed. The Cheque details get displayed and you cannot update the details. You can record the cheque details, if it is not recorded.

  1. Cheque Received – Select the check box, if a Cheque has been received for relevant transaction. The check box is selected and disabled, if the Cheque details are recorded under the ‘Create/Increase Allocation Limit’ tab.

  2. Cheque No – Enter the number printed on the Cheque. The cheque number is displayed and disabled, if the Cheque details are recorded under the ‘Create/Increase Allocation Limit’ tab. This is a mandatory field.

  3. Cheque Date – Click the Calendar function to select the date when the Cheque is issued.. The cheque date is displayed and disabled, if the Cheque details are recorded under the ‘Create/Increase Allocation Limit’ tab. This is a mandatory field

  4. Bank – Click [List of Values] to select a Bank. The Bank name specifies where the Cheque is issued. The Bank name is displayed and disabled, if the Cheque details are recorded under the ‘Create/Increase Allocation Limit’ tab. This is a mandatory field.

  5. Cheque Received Date - Click the Calendar function to select the date when the Cheque is received. The cheque received date is displayed and disabled, if the Cheque details are recorded under the ‘Create/Increase Allocation Limit’ tab. This is a mandatory field.

Bank Clearance Details

  1. Cheque Cleared – Select the check box, if Cheque is received and processed. Clear the check box, if the transaction is not processed.

  2. Cash in Hand – Select this check box if the transaction is cash in hand transaction. cash in hand.

  3. Cheque Cleared Date – Click the Calendar function to record the cheque clearance date. The date selected has to be greater than or equal to the cheque date. This field is enabled for entry only if the ‘Cheque Cleared’ check box is selected.

Generate Replacement Voucher

The replacement receipts for the existing allocating transactions can be generated using the function, Generate Replacement Voucher. You can specify the date for the voucher generation and the report can be previewed before generation. Authorized users are permitted to login and generate replacement vouchers.

Last Cashbook Period

  1. Date [From – To] – The date indicates the beginning and last date when the cash book was created previously.

The following details are displayed on logging in:

  1. Refund Date – The date when a refund was issued is displayed.

  2. Payer Name – The name of the payer to whom the refund was issued is displayed.

  3. Deposit Doc No – The deposit document number for refund is displayed.

  4. Refund No – The refund document type and number generated during transaction is displayed.

  5. SRNo – The serial number selected for the respective document type and number during transaction is displayed.

  6. Cancelled Refund Amt – The refund amount cancelled is listed.

  7. Account Code – The account code selected is displayed.

Current Period Cashbook Details

  1. From Date – The date gets displayed automatically based on the date upto when the Replacement Voucher was generated previously.  

  2. Upto Date – Click the Calendar to select the date until when the Replacement Voucher has to be generated. A date prior to the current system date gets displayed, by default. You cannot create Replacement Voucher for the current system date. This is a mandatory field.

  3. Total Refund Amt – The Total Refund Amount is displayed.

  4. Voucher No – Enter the Voucher Number in alphanumeric format of 10 characters maximum length. This is a mandatory field.

  5. Voucher Date – The Voucher Date is set automatically based on the ‘Upto Date’. Click the Calendar function to select the Voucher Date. This is a mandatory field.

  6. Voucher Amount – The Total Refund Amount is displayed as the Voucher Amount. You cannot update the Voucher Amount.  

  7. Remarks – Enter the Remarks in alphanumeric format. This is a mandatory field.

Document Details

The document type is displayed based on the document type set for ‘Refund Cash Book Doc’ in the ‘Setup External Acct Parameters’ function under the ‘Set Up’ menu. The system date is displayed, by default.

  1. [Preview Cashbook] – Click [Preview Cashbook] to view the Cashbook details before generating the Cashbook

  2. [Print Edit List] – Click [Print Edit List] to view, print or save the Cashbook details in a file.

  3. [Close Preview] – Click [Close Preview] to exit the preview.

  4. [Generate Cashbook] – Click [Generate Cashbook] to generate replacement voucher for the cash refund transactions for a selected period.

A receipt number is generated for the replacement voucher.

  1. Click [Commit Form] or press [F10] to save or [Clear Record] to clear the entered data.

  2. Click ' View Alloc. Trx by AG' tab to view Allocation Transaction or the Cash Book details.

    Enter the following details to view the Cash Book entries:

    1. OB Date – Click the Calendar function to select the Opening Balance Date. Cashbook transaction details for the specified period are displayed.

    2. CB Date - Click the Calendar function to select the Closing Balance Date. Cashbook transaction details for the specified period are displayed.

    3. Cash-in Hand – The total amount collected from the replacement vouchers are displayed.

    The details recorded using the ‘Generate Replacement Voucher’ tab are displayed.

    Enter the following to view the allocation details:

    1. Option – Select either Alloc.Trx or Cashbook. The transaction details are listed out based on the option selected.

    2. Alloc.Trx – ‘Allocating Trx by AG’ and ‘Replacement Vouchers’ tabs are enabled and the details are listed on selecting the option as ‘Alloc.Trx’.

    3. Cashbook - ‘Cashbook Trx’ tab gets enabled and the details are displayed.

    1. Criteria – Select a value from the list. The system-defined values are Document Date, Cheque Recvd Date, Cheque Cleared Date, Cheque Date and Voucher Date.

    2. Date [From-To] – Click the Calendar function to select the period when the transactions were performed.

    3. [Refresh] – Click [Refresh] to refresh the details in the screen.

    You can view Allocation Transactions or the Cash Book entries using this tab. The tabs 'Allocating Trx by AG' and 'Replacement Vouchers' are enabled on selecting 'Alloc.Trx' as the Option. The tab 'Cashbook' is enabled on selecting 'Cashbook' as the Option.

  3. Click [Refresh] to query. The details are displayed based on the Option selected.

  4. The Allocating Trx by AG, Replacement Vouchers and Cash in hand tabs are enabled on selecting the Option as 'Alloc. Trx'. The Cashbook Trx tab is enabled on selecting the Option as 'Cashbook'.

    The details recorded using the ‘Generate Replacement Voucher’ tab are displayed.

    Cash in hand

    The transactions which have been done with selecting Cash in hand check box are displayed in Cash in hand tab. You cannot edit the data.

     

    Enter the following details:

    1. Chq. No - The Cheque Number entered in transactions is displayed.

    2. Chq. Date - The date on which the cheque is issued is displayed.

    3. Bank – The selected Bank during transaction is displayed. This specifies the name of the Bank where the Cheque is issued.

    4. Doc.Amt –The amount received through Cheque is displayed.

    5. Chq. Recvd. Dt –The date on which the cheque is received is displayed.

    6. Cleared –The value ‘yes’ is selected if the cheque is cleared in the bank, else the value ‘No’ is selecetd.

    7. Cleard On –The cheque clearance date is displayed.

    8. Doc. Type – The selected doc type during transaction is displayed.

    9. Doc. No – The selected doc number during transaction is displayed.

    10. Doc. Date - The selected doc date during transaction is displayed.

    The receipts or the vouchers, which are replaced are listed. The details recorded using the ‘Generate Replacement Voucher’ tab is listed. The voucher details, which are replaced, are displayed. You cannot update the details.

    The cheque details recorded or updated using the ‘Maintain Alloc.Trx/Repl.Vrs’ and ‘Create/Increase Allocation Limit’ tabs are listed based on the date selected.

  5. Click [Execute Query] or press [F8] to search for the existing records within the facility.
