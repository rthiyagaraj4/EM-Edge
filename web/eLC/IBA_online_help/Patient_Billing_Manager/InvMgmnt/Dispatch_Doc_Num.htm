Dispatch Document Numbering

From this function you can Define the format necessary to dispatch the invoices to the Customer. You can Query for the existing details.

The Dispatch Document Numbering function enables you to define the format necessary to dispatch the invoices to the Customer. You also have an option to define a range of Document Series for a specific Customer associated to the facility. The invoices raised against the bills for a Customer are dispatched using the Document format setup.

Defining Dispatch Document Numbering

From the Invoice Management > Setup, double-click on Dispatch Document Numbering function.

The Dispatch Document Numbering screen gets displayed.

  1. Enter the details in the mandatory (*) fields in the following tab pages:

Dispatch Document Series

Enter the following details:

  1. Letter Prefix Series - Enter the Letter Prefix Series in alphanumeric format of 10 characters maximum length. This is a mandatory field.

  2. Use as Default Prefix Series – Select the check box, if the respective Letter Prefix Series has to be the default prefix for the invoices that are to be dispatched to the Customers. A single prefix series can be made as the default series.  

  3. Long Description – Enter the Long Description in alphanumeric format of 30 characters maximum length. This is a mandatory field.

  4. Short Description - Enter the Long Description in alphanumeric format of 15 characters maximum length. This is a mandatory field.

  5. Year Prefix – Select a value from the list based on which the year format gets displayed in the letter. The system defined values are – “Not Applicable”, “YYYY – Format” and “YY – Format”.

  6. Separator Before Year – The Separator Before Year is set as “/”, by default. You can enter the value in the field using which the year has to be separated from any other preceding values.

  7. Month Prefix – Select a value from the list based on which the month format gets displayed in the letter. The field contains system defined values – “Not Applicable”, “Number - Format” and “Character – Format”.

  8. Separator Before Month – The Separator Before Month is set as “/”, by default. You can enter the value in the field using which the month has to be separated from any other preceding values.

  9. Separator for Serial Number – The Separator Before the Serial Number of the letter is set as “/”, by default. You can enter the value in the field using which the serial number has to be separated from any other preceding values.

  10. Starting Number – Enter the Starting Number in numeric format of 10 digits maximum length. The first invoice letter generated begins with the starting number incrementing each time an invoice is generated for a specific Customer in the facility. This is a mandatory field.

  11. Next Number – Enter the Next Number in numeric format of 10 digits maximum length. The next number indicates the following number for the second invoice letter generated for a specific Customer in the facility. This is a mandatory field.

  12. Last Number – Enter the Last Number in numeric format of 10 digits maximum length. The last number indicates the final number, which is applicable for an invoice letter generated in the facility for a specific Customer. This is a mandatory field.

  13. Recycle Every Year with Starting Number – Select the check box, if the Starting Number has to be recycled every year. Clear the check box to continue using the numbers irrespective of the year until the Last Number is reached.

Dispatch Document Series Setup

Enter the following details:

  1. Payer Group - Click [List of Values] to select a Payer Group. The values defined using the ‘Customer Groups’ function under the AR Setup menu gets listed. This is a mandatory field.

  2. Description – The Description for the selected Payer Group gets displayed.

  3. Payer - Click [List of Values] to select the Payer. The Customers associated to the selected Customer Group using the function ‘Customer’ under the AR Setup menu gets listed. This is a mandatory field.

  4. Description – The Description for the selected Payer gets displayed.

  5. Policy Type Code - Click [List of Values] to select a Policy Type. The policies associated to the selected Payer and the Payer Group using the function ‘Setup Service Coverage Detail by Policy’ function in the Insurance Management module get listed. This is a mandatory field.

  6. Description – The Description for the selected Policy Type gets displayed.

  7. Dispatch Document Series - Click [List of Values] to select the Dispatch Document Series. The Document Series defined using the ‘Dispatch Document Series’ tab gets listed.

  8. Description – The Description for the selected Dispatch Document Series gets displayed.

  1. Click [Commit Form] to save the records or [Clear Record] to delete the data entered.

  2. Click [Execute Query] to query for the existing records.
