OT Charge Type and Associated Resource

From this function you can Define charge type for resources used in the Operating Theatre and Accident and Emergency wards of a Facility. You can Define the charging rule for each resource usage. You can Query to view the records defined.

OT Charge Type and Associated Resource function is used to define the charges for the resources used in a Operating Theatre. You can define the charge types based on which the price has to be computed for a Resource associated to a Resource Type. You can also define the charges for a bed occupied in an Accident and Emergency ward.

Define Charge types and Associate Resources

From the Masters > Interfaced Modules Financial Dtls, double-click on OT Charge Type and Associated Resource function.

The OT Charge Type and Associated Resource  screen gets displayed.

  1. Enter the details in the mandatory (*) fields found in the following tabs.

Associate Charge Type by Resources

Enter the following details to associate Charge Type to Resources:

  1. Resource Type - Select a value from the list. The system-defined values are 'OT Room', 'Holding Bay', 'Recovery Room' and 'AE Bed/Bay Type'. Select a Resource Type to associate a Charge Type to the Resources associated to it.

  2. Resource - Click [List of Values] to select a Resource. The Resources get listed based on the Resource Type selected. The description for the respective Resource gets displayed.

  3. Charge Type - Click [List of Values] to select a Charge Type. The values defined under the 'Resource Usage Charging Rule' gets listed.

You cannot create duplicate records.

Resource Usage Charging Rule

Enter the following details to define a Charge Type:

  1. Charge Type - Enter the charge Type in alphanumeric format of six characters maximum length. This is a mandatory field.

  2. Long Description - Enter the long description in alphanumeric format of 80 characters maximum length. This is a mandatory field.

  3. Short Description - Enter the short description in alphanumeric format of 40 characters maximum length. This is a mandatory field.

  4. Resource Type - Select a Resource Type from the list. The system-defined values are - 'Holding Bay', 'OT Room', 'Recovery Room' and 'AE Bed/Bay Type'. The Charge Type defined is valid for the selected Resource Type. This is a mandatory field.

The value ‘AE Bed/Bay Type’ is enabled only if the check box ‘Bed Charge Applicable’ is selected in ‘BL Parameters’ function in the ‘Outpatient/Emergency/external’ tab. The Bed charge is computed and displayed in the ‘Visit Charge Details’ screen when the patient is checked-out through the ‘AE Checkout’ function.

  1. Minimum Chargeable Unit(s) - Enter the minimum chargeable unit in numeric format of three digits maximum length. This indicates the minimum  unit of resource to be considered in the time specified.

  2. Subsequent Period Charging Basis - Select a value from the list. The system-defined values are 'Multiples of Resource Usage Duration' and 'Slab Based Resource Usage Duration'. Select 'Multiples of Resource Usage Duration', if the resource usage should be computed based on the usage duration.  Select 'Slab Based Resource Usage Duration', if the resource usage should be computed based on the slab duration.

  3. Subsequent Duration Per Chargeable Unit - Enter the time in minutes to indicate the subsequent duration for charging. This field is enabled for entry only if the Subsequent Period Charging Basis is selected as 'Multiples of Resource Usage Duration'.

  4. Unit Rounding - Select a value from the list. The system-defined values are 'Next Higher Chargeable Duration', 'Next Lower Chargeable Duration' and 'Not Applicable'. This is used to round up the values to the next higher or the next lower level when the slab used exceeds the minimum level specified. This is a mandatory field.

  5. Linked Billing Service Per Unit - Click [List of Values] to select a Billing Service. The charges are computed based on the service price defined. This is a mandatory field.

  6. Slab Search Type - Select a value from the list. The system-defined values are 'Exact Range Basis', 'Incremental Basis' and 'Not Applicable'. This field is enabled for selection, if 'Subsequent Period Charging Basis' is set to 'Slab Based Resource Usage Duration'. This is a mandatory field.

Slab Based Subsequent Duration

  1. From - Enter the duration in numeric format. Slab Based Subsequent Resource Usage duration has to begin with zero.

  2. To - You cannot enter the details. The details are computed based on the 'From' value entered in the subsequent record.

  3. Chargeable Units - Enter the units to be charged in numeric format of three digits maximum length. This indicates the number of units that are to be charged for the time duration specified.

  1. Click [Commit Form] or press [F10] to save or [Clear Record] to clear the entered data.

  2. Click [Execute Query] or press [F8] to search for the existing records within the facility.
