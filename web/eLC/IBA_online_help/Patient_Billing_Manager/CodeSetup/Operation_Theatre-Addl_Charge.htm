Operation Theatre - Addl Charge

From this function you can Define surgery templates and associate services to the surgery template to define additional charges.  You can Query to view the records created.    

Operation Theatre – Addl Charge function is used to define a Surgery Template and define additional charge for the selected Surgery template in the Operation Theatre. You can also issue a discount on the subsequent surgery when multiple surgeries are performed for the patient in the facility under the same booking.

Defining Additional Charges on Operation Theatre

From the Masters > select Interfaced Modules Financial Dtls, double-click Operation Theatre - Addl Charge  function.

The Operation Theatre – Addl Charge screen gets displayed.

  1. Enter the mandatory (*) details under the following tabs:

Additional Charge Template

The Additional Charge Template tab is used to define additional charges for the surgery templates. You can associate multiple OT Services to a Surgery Template and define the additional charges for these templates. You can also define the discount for the multiple services.

Enter the following details:

  1. Surgery Format – Enter the Surgery Template code in alphanumeric format of 10 characters maximum length. This is a mandatory field.

  2. Long Description - Enter the Long Description in alphanumeric format of 60 characters maximum length. This is a mandatory field.

  3. Short Description - Enter the Short Description in alphanumeric format of 30 characters maximum length. This is a mandatory field.

  4. Billing Class – Click on the Billing Class to associate the services to the Surgery Template valid for the Billing Class. The values defined using the function ‘Billing Class’ in the Master gets displayed. Select ‘**’, if the Surgery Template is valid for all the Billing Class in the facility.

Dependent Services

  1. Srl No – Enter the Serial Number in numeric format. The serial number indicates the precedence of the services associated to the Surgery Template for the respective Billing Class.

  2. Charge Type – Select a value from the list. The system-defined values are ‘As per Setup’ and ‘%age of Other Setup’. The services are charged based on the Charge Type selected. Select ‘As per Setup’ to calculate the price based on the charges defined in the Main Surgery. Select ‘%age of Other Setup’ to calculate the price based on the dependant services.

  3. Billing Service for Additional Charge – Click [List of Values] to select and associate a Billing Service for additional charges. The description for the selected Billing Service gets displayed.

  4. %Age of – Enter the percentage factor in numeric factor. The additional charges are computed as the product of the Billing Service price and the factor recorded.

  5. Dependant Billing Service Code – Click [List of Values] to select and associate a dependant Billing Service to compute the additional charges. The description for the selected Billing Service gets displayed. The value in the list is displayed as **MAIN** for the first row.  The description for the selected Billing Service code gets displayed.

  6. OT Role Type – Click [List of Values] to select an OT Role Type. The values defined in the Operation Theatre module gets displayed.

  7. Description – The description for the selected OT Role Type gets displayed.

Additional Charge Setup by Surgery

Enter the following details:

  1. Facility – The logged in Facility name gets displayed, by default.

  2. Order Catalog for Procedure – Select the Order Catalog to which the main surgery is linked. The description for the selected Order Catalog gets displayed. This is a mandatory field.

  3. Billing Service – The Billing Service associated to the Order Catalog gets displayed. You cannot edit the Billing Service.

Discount for Multiple Surgery

  1. Discount 1% - Enter the discount percentage in numeric format. This defines the discount on the highest prices surgery.

  2. Discount 2% - Enter the discount percentage in numeric format. It indicates the discount for the Billing Service, if the defined Billing service is linked to an Order Catalog, which identifies the second procedure performed for the respective Billing Class.

  3. Discount 3% - Enter the discount percentage in numeric format.  It indicates the discount for the Billing Service, if the defined Billing service is linked to an Order Catalog which identifies the third procedure performed for the respective Billing Class.

  4. Discount 4% - Enter the discount percentage in numeric format.  It indicates the discount for the Billing Service, if the defined Billing service is linked to an Order Catalog which identifies the fourth procedure performed for the respective Billing Class.

  5. Discount 5% - Enter the discount percentage in numeric format.  It indicates the discount for the Billing Service, if the defined Billing service is linked to an Order Catalog which identifies the fifth procedure performed for the respective Billing Class.

  6. Apply on - Select ‘the required Gross amount’ or ‘Net amount’ value from the drop-down list. If the value ‘Gross amount’ is selected, the details such as Gross Amount, Discount Amount, Discount Percentage and Net amount are displayed in Record Additional Charges dialog invoked from Record Surgeon Notes function in OT module. If the value ‘Net amount’ is selected, the details such as Gross amount, Discount Percentage, Discount amount and Net Amount are displayed in Record Additional Charges dialog invoked from Record Surgeon Notes function in OT module. This is applicable only for Indian users

 

  1. Discount percentage – This field displays the percentage as 0.00

  2. Discount amount – This field displays the discount amount as 0.00

  3. Net amount – This field displays the same value as gross amount.

The dependant services defined under the ‘Additional Charge Format’ gets displayed. You can insert new services under this tab.

  1. Click [Commit Form] or press [F10] to save or [Clear Record] to clear the entered data.

  2. Click [Execute Query] or press [F8] to search for the existing records within the facility.
