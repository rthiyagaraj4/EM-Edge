Parameter for Search Process

  1. Search Across Facility - The value is either ‘Yes’ or ‘No’. If the value is ‘Yes’, you can search for patient records across facilities or the patient records can be searched only for the logged in facility.

  2. Criteria for Suggested Main Patient - The value is either ‘By Encounter’ or ‘By Registration Date’. If the displayed value is ‘By Encounter’, while searching for duplicate patient records, the Patient ID with maximum number of encounters is considered as main patient. If the value displayed is ‘By Registration Date’, the patient record with latest Registration Date is considered as main patient.

 

For Merge Process

  1. Rebuild Dictionary Before Merge - The value is either ‘Yes’ or ‘No’. If the value is ‘Yes’, the dictionary is rebuilt before the merging process or the dictionary is not rebuilt.

  2. Merge Patient Only During - Specify the period for merge process in the format of hh:mm. The merge process is performed within the specified duration. For example, if the values specified are 08:00/23:00, it means that the merge process can be performed only between 08:00 to 23:00 hrs.

 

For Unmerge Process

  1. Warning/Error before Unmerge - The value is either ‘Warning’ or ‘Error’. If the value displayed is ‘Warning’, during the process of unmerging the records, a warning message gets displayed. If the value displayed is ‘Error’, during the process of unmerging the records, an error message gets displayed.
