Encounter Specific Policy Definition – Material Items

From this function you can Include or Exclude the material items for a particular patient encounter associated to specific Policy Types and Query for a desired Encounter Specific Policy Type record.                                                  

The function Encounter Specific Policy Definition – Material Items is used to include or exclude the material items for a particular patient encounter in the facility. The coverage charge can be recalculated for specific patient encounter. You can associate additional material items to the patient encounter. The material items associated to the selected Policy Type, Customer Group and the Customer are defined using the function Setup Service Coverage Detail by Policy. You can associate additional material items to the patient encounter other than the material items which are already associated  Policy Type, Customer Group and the Customer. You can also delete the material items from the patient encounter.

Encounter Specific Policy Definition – Material Items

From the Masters, double-click on Encounter Specific Policy Definition – Material Items function.

The Encounter Specific Policy Definition screen gets displayed.

  1. Select an Episode Type from the list. The values are system defined - Inpatient, Daycare, Outpatient and Emergency. The policy definition for services is applicable for the selected Episode Type.

  2. Click [List of Values] to select an Episode ID. The Episode ID applicable for the selected Episode Type gets listed. The patient details get displayed based on the Episode ID selected. This is a mandatory field.

  3. The Policy Type details associated to the patient during the specific encounter registration gets displayed. <Related Field Description>

  4. Click [View] to view the service coverage detail for a specific policy.

  5. You can include additional material items to the specific encounter and define the coverage limit for the specific policy using the tabs:

Service Item Inclusion/Exclusion

Service Inclusion/Exclusion

  1. Excl Incl Criteria – Select a value from the list. The system defined values are ‘Enterprise Material Group’ and ‘Material Item’.

  2. Exclude/Include Code – Click [List of Values] to view and select a Billing Service, Payer Service Group or Payer Service Group/Line.

  3. Code – Click [List of Values] to view and select a material item based Billing Service Code.

  4. Special Service – Select the check box, if special items are to be associated to the specific encounter.

  5. Excl Incl Ind – Select a value from the list. The system defined values are ‘Excluded’ and ‘Included’.

  6. Override Excl/Incl – Select the check box to override the exclude/include indicator selected for the material item based Billing Service associated. The override performs inverse action based on the value selected for ‘Excl Incl Ind’ field.

  7. Inclusion Basis - Select a value from the list. The system defined values are ‘Percentage’, ‘Rate’ and ‘Fixed’. This is applicable for co-pay value calculation. On selecting ‘Percentage’, the symbol % is displayed beside it. On selecting ‘Rate’ or ‘Fixed’, currency name is displayed beside it.

  8. Policy Level Coverage – Select the check box, if the service exclude/include detail is valid only for the selected policy.

  9. Copay Value – Enter the Copay Value in numeric format. The amount to be settled by the Co-payer is calculated based on the Payment Indicator selected and the amount specified for Copay value.

  10. PreApproval – Select the check box, if preapproval is necessary to associate the services and the co-pay value defined.

  11. Effective Date – Click Calendar function to select the date. The material based Billing Services are valid for the selected encounter on the date specified.

  12. Pref Check – Select the check box to associate preferred items.

  13. Pref Basis – Select a value from the list. The system defined values are ‘Specific Drug’ and ‘Generic Drug’. The field is enabled only on selecting ‘Pref Check’.

  14. Pref Item - Click [List of Values] to select a preferred item. The items in the list differ based on the value selected for ‘Pref Basis’.

  15. Approved – Select the check box and the parameter defined is approved for the selected encounter.

  16. Coverage includes other Payers - Select the check box to define an additional payer for the specific Policy, Payer and Payer Group. Clear the check box, if an additional payer is not required for the selected combination.

  17. Last Payer - Select the check box, if the payer is the last payer. Based on whether the coverage amount is selected and the payer is the last payer or not, the policy coverage amount is distributed as follows:

  18. If the check boxes ‘Coverage includes other Payers’ and ‘Last Payer’ are selected, the amount is sent as discount.

  19. If the check box ‘Coverage includes other Payers’ is selected and the check box ‘Last Payer’ is cleared, the remaining amount has to be paid by the patient.

Policy Coverage Limits

Payer Coverage Limits

  1. Coverage Limit Criteria – Select a value from the list. The system defined values are ‘Sale Category’ and ‘Material Item’.

  2. Material Item Code - Click [List of Values] to view and to select a material item based Billing Service code.

  3. Description – The description for the selected Billing Service code gets displayed.

  4. Amt Limit Per Admission – Enter the maximum limit amount covered for the selected material based Billing Service associated to the patient encounter in numeric format.

  5. Amt Limit Per Day – Enter the maximum limit amount per day in numeric format. The amount limit is valid for the selected material based Billing Service and is valid for a day and applicable to the selected patient encounter.

  6. No Of Days – Enter the number of days until when the coverage amount limit is valid for the selected patient encounter.

  7. Override Limit – Select the check box to override the coverage limit. The override performs inverse action based on the value selected for ‘Amt Limit Per Enc’ or ‘Amt Limit Per Day’.

Coverage Limit By Mapping Code

Enter the following details:

  1. Mapping Code – Click [List of Values] to select the Mapping Code. The values that are defined through the ‘Service/Material Item Mapping’ function are listed.

  2. Mapping Description – The description of the selected Mapping Code is displayed.

  3. Term Set ID – Click [List of Values] to select the Term Set ID. You can associate the same term set ID to different Mapping Codes. This is enabled for selection only if the ‘Physician & Diagnosis Applicable for Coverage’ check box is selected for the respective Mapping Code.

  4. Term Set Description – The description of the selected Term Set is displayed.

  5. Term Code – Click [List of Values] to select the Term Code. The DRG codes that are associated to the selected Term Set ID are displayed. This is enabled for selection only if the ‘Physician & Diagnosis Applicable for Coverage’ check box is selected for the respective Mapping Code.

  6. Term Code Description – The Description of the selected Term Code is displayed.

  7. Physician ID - Click [List of Values] to select the Physician ID. You can select a value only if the ‘Physician & Diagnosis Applicable for Coverage’ check box is selected for the respective Mapping Code. The name of the Ordering Physician is displayed.

  8. Physician – The name of the selected Physician is displayed.

  9. Level - Click [List of Values] to select a Level. The coverage amount is based on the level selected. For example, if the Level is 1 and the coverage amount is 5,000, the amount covered is 5,000. If level 2 coverage is defined as 7,000, then the total coverage is the sum of [level1 + level2]. This is valid for Mapping Codes with the ‘Level Applicable for Coverage’ check box as selected.

  10. Include/Exclude – Select a value from the list. The system-defined values are ‘Include’ and ‘Exclude’. Select ‘Include’ if the coverage is valid for the selected diagnosis code and physician. Select ‘Exclude’, if the coverage amount is not valid for the associated diagnosis code and physician. You can specify the Copay Value and the Effective Date only if you select the value as ‘Include’. The value is selected as ‘Exclude’, by default.

  11. Rate – Select ‘Rate’ from the list to define Copay Indicator for the Mapping Code.

  12. Copay Value – Enter the Copay Value in numeric format. It defines the coverage amount for the Mapping Code. This is a mandatory field.

  13. Effective Date – Click the Calendar function to select the effective date. The copay coverage amount is valid from the selected date for the associated Mapping Code. The date selected has to be greater than the system date.

Policy Coverage Limits - History

Using this tab, you can view the payer coverage limits recorded for a patient encounter.

Coverage Limit By Mapping Code History

Using this tab, you can view the coverage limit defined for the Mapping Code.
