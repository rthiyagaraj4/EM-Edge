Request Comment (Clinical)

From this function you can Create clinical comments specific to a Department used for processing the specimen. You can Query for the existing records.

The request comments defined can be linked to different tests through the ‘Test Codes’ function. If test linked to request comment require validations, such tests are held for review on resulting. An authorized user is permitted to access and perform transactions in the assigned department.

Creating Request Comment (Clinical)

From the Masters > Request, double-click Request Comment (Clinical) function.

The Request Comment (Clinical) screen gets displayed.

  1. The Department selected as ‘Default Department’ in the ‘Authorized Users’ function gets displayed. Select desired department from the list if you have access to more than one department. This is a mandatory field.

  2. Enter a unique request comment code in alphanumeric format of four digits maximum length. The code cannot be modified after creation. This is a mandatory field.

  3. Enter the Request Comment in alphanumeric format of 40 characters maximum length. This is a mandatory field.

  4. Click [Commit] or press [F10] to save the records or click [Clear Record] to clear the entered data.

  5. Click [Execute Query] or press [F8] to search for existing records.

‘User is not linked to any Department?’ error messages gets populated on access Request Comment (Clinical) Master, if you have not linked any department to the user in the Administration Setup.
