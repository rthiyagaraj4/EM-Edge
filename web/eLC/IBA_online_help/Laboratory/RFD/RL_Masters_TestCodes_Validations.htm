Validations

This tab allows you to define validations for the test.

 

Enter the following details:

  1. Technical - Technical Review check box is enabled if the test result is of numeric type. Select the Technical Review check box, if the numeric result requires validation. If the result value of the test is out of the specified range, the validation fails and the specimen is directed to the technical review.

  2. Path Review - Select the Path Review check box if the test requires pathologist review for the technical review done.

  3. Range [Low-High] - Enter the acceptable lower limit and higher limit value to validate numeric result while performing technical review.

  4. Comment - Select the Comment check box to view the result in 1st and 2nd level review.

  5. Path Review - Select the Path Review check box, if the test requires pathologist review for the result comments.

  6. Result Cmnt - Select the Result Comment check box, if the test result comment requires validation. You can specify which result comment requires review while defining Result Comments for the test. If the validation fails for a result comment, the specimen is directed to the technical review.

  7. Path Review - Select the Path Review check box, if the test requires pathologist review for the result comments.

  8. Delta Numeric Result - Delta Numeric Result tab is enabled if the test result is of numeric type. Select the Delta Numeric Result check box, if the test requires delta check during validation. If the previous result for the test is within the time and the variation between the current numeric result and the previous result is more than the specified absolute or percentage value, the validation fails and the specimen is directed to technical review.

  9. Path Review - Select the Path Review check box if the test requires pathologist review for the delta numeric check.

  10. Valid Period - Enter a Valid Period for which the previous result is valid to perform delta check.

  11. Units - Select the required unit for the entered Valid Period from the list.

  12. Type - Enter a valid value to indicate whether Delta Check is based on absolute value or on percentage.

  13. Value - Enter the permissible difference between the previous and the current result for the patient based on the type.

  14. Delta Result Comment - Select the Delta Result Comment check box to carry out the delta check for the test during the result comment validation. If the previous result for the test is within the period and the variation between the current result comment and the previous result comment is more than the specified absolute or percentage value, the validation fails and the specimen is directed to technical review.

  15. Path Review - Select the Path Review check box, if the test requires pathologist review for the delta result check.

  16. Valid Period - Enter a Valid Period value for which the previous result is valid to perform delta check.

  17. Units - Select the required unit for the entered Valid Period from the Units list.

  18. Delta Sensitivity - Select the Delta Sensitivity check box to carry out the delta check for the test during test validation. If the previous result for the test is within the period and the variation between the current sensitivity and the previous sensitivity is more than the specified absolute or percentage value, the validation fails and the specimen is directed to technical review.

  19. Path Review - Select the Path Review check box, if the test requires pathologist review for the delta sensitivity check.

  20. Valid Period - Enter a Valid Period value for which the previous result is valid to perform delta check.

  21. Units - Select the required unit for the entered Valid Period from the Units list.

  22. Quality - The Quality tab is enabled if the test result is of numeric type. Select the Quality check box to carry out the quality check for the test during test result validation. The values defined through the 'Quality' function get listed. A quality code can have various quality tests and a valid range of low and high values. You can provide the results of quality check through the worklist. If the quality result is out of range and the elapse period is more than the allowed period, the test is rejected. The specimen is directed to the technical review, if the validation fails.

  23. Path Review - Select the Path Review check box, if the test requires pathologist review for the numeric result value.

  24. Valid Period - Enter a Valid Period value for which the quality check is valid.

  25. Units - Select the required unit for the entered Valid Period from the Units list.

  26. One in N Count - Select the 'One in N Count' check box to calculate the number of times the validations are done.

  27. Path Review - Select the Path Review check box, if the test requires pathologist review for the numeric result value.

  28. Count - Enter the number of count for review.

  29. Path Review - Select required value from the Path Review list to specify whether the test desires 'Normal' or 'Abnormal' Pathologist Review.

  30. Range [Low -High] - Enter the acceptable lower limit and upper limit value to validate numeric result while performing pathological review.

  31. Critical Check - Select the 'Critical Check' check box to check whether the numeric result of the test is out of critical range specified. If the result is out of critical range, a warning message is attached to the result of the test and an sms alert is sent to the concerned personnel while releasing the result. Critical checking is available only for the Laboratory sections that have selected this option while setting up 'Department Details' masters and when a result is processed from the abnormal review.

  32. Range [Low-High] - Enter the acceptable lower limit and upper limit value to validate numeric result while performing critical check.

 

Report Range

  1. Low - Enter the default Low value to be printed on the report.

  2. High - Enter the default High value to be printed on the report.

  3. Report Range Comment - Enter the required report range with > or < signs in the Report Range Comment field. You can enter up to 40 characters maximum. If the Report Range (Low, High) values are not available in the Validation tab or Age/Sex Ranges tab, the Report Range Comment value gets printed in the report.
