Authorize at Issue Store

From this function you can Query the request documents requiring authorization at Issue Store.

This function facilitates screening of requests raised against the Store where issues are to be affected. This screening process mainly involves approving a quantity for issue based on the availability of stocks. This function supports better decision making by alerting you on stocks below the reorder levels and minimum stock levels.

Authorizing Requests at Issue Store

From the ST Transactions > Issue, select the Authorize at Issue Store function.

The Authorize at Issue Store screen gets displayed.

 

  1. Click [Create]; displays the Modify Document Details dialog.

  2. Select the Doc Type, Doc Date, From Store and To Store and click OK; the document details are displayed in white font in the Authorise at Issue Store screen. You can click the Document Type yellow hyperlink to modify the document details, if required.

  3. To add more items to the existing request, enter the required mandatory fields at the bottom of the screen. <Related Field Description>

  4. Click ‘View Batch Details’ hyperlink to view the batch details and the quantity allotted for each batch in a View Batch Details dialog.

    1. Modify the ‘Authorized Qty’ against each batch for authorization.

    2. Click [Close] to confirm the authorized quantity for each batch.

     

    To change the batch details:

     

    1. Click [Change Batch]. The ‘Batch Search’ dialog gets displayed with batch details of the items with their expiry date and bin location.

    2. Select the check box against the item batches that needs to be changed.

    3. Click [Close] to close the ‘Batch Search’ dialog. The selected item batches are displayed in the ‘View Batch Details’ dialog.

    4. Enter the ‘Authorized Qty’ against each batch for authorization.

    5. Click [Close] to close the ‘View Batch Details’ dialog and to confirm the authorization quantity for each batch.

  5. Click [Stock Level] to view the Item Stock details.

  6. Click [Add] to add the request details and the specified details are displayed in the top half of the screen. Click [Cancel] to cancel the selected Item.

  7. On adding the Item, the Item details are displayed. The stock status is displayed using the following color codes in the first column ‘S’:

  1. The Item, Required Quantity, Committed Quantity, Issued Quantity, Pending Quantity, Available Quantity at Requested Store, Available Quantity at Issue Store and Authorize Quantity are displayed on adding the Item. The Stock Value is displayed, if user has access to  view the cost details. The access rights are defined in the 'View Cost Details For User' of Materials Base module.

  2. Click [Apply] to finalize the authorize requests at Issue Store details or [Reset] to clear the entered data.

  3. On clicking [Apply], the Current Pending Requests dialog gets displayed.

  4. Click [Continue]. The Confirm Printing dialog gets displayed. Click [Yes] to print or [No] to cancel the action. The ‘Operation Completed Successfully’ message is displayed.

  5. To close the issue, click Close Upon Issue check box. Entering the Remarks become mandatory.

  6. Click the Remarks hyperlink to enter  the remarks for closing the issue in the Remarks dialog.

  7. Click [Query] to invoke the Query screen.

  8. The Documents for Authorize dialog is displayed, by default. Enter the search criteria  and click [Search].

    Enter the following search criteria:

    1. Requests of – Select the required value from the list. The listed values are ‘Internal Stores’ and ‘External Stores’.

    2. Doc Type – Select the required Document Type from the list. The documents associated to the transaction ‘Requisition Issues’ are displayed in the list. The Document Type, as set in ‘ST Facility Parameter’ is displayed, by default.

    3. Req By Store - Select the Requested by Store from the list. Only those Stores for which you have access and ‘Requisition’ transaction function applicable are displayed.

    4. Req On Store - Select the Requested on Store from the list. Only those Stores are displayed for which you have access and ‘Requisition Issues’ transaction function applicable.

    5. Doc Date [From – To] - The system date is displayed, by default. You can enter the date in date format of dd/mm/yyyy or select the required date using the Calendar function.

    6. Doc No. - The Document Number generation depends on the selected Document Type. If the Document Number has to be generated manually for the selected Document Type, you are informed. Enter the Document Number in numeric format of eight digits maximum length. If the system generates the number, you are not allowed to enter the field value.  

    7. Item Class – Select the required Item class from the list. The Item Class associated to the selected Requested By store and Requested On store are displayed in the list.

    8. Item – Select the required Item using [?] function.

    9. Req Status – Select the required Request Status from the list. The listed values are ‘All’, ‘Authorized at Req by Store’, ‘Partially Authorized at Issue Store’, ‘Partially Authorized/Issued at Store’, ‘Fully Authorized/Partially Issued at Store’. The value ‘All’ is displayed, by default.

  9. The request documents requiring authorization at Issue Store, partially issued are displayed.

  10. Click Doc Type hyperlink of the request documents requiring authorization at issue store and to view requests.

  11. The Item, Required Quantity, Committed Quantity, Issued Quantity, Pending Quantity, Available Quantity at Issue Store, Available Quantity at Requested Store  and Authorize Quantity are displayed on adding the Item. The Stock Value is displayed, if user has access to view the cost details. The access rights are defined in the 'View Cost Details For User' of Materials Base module
