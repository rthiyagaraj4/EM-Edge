Change Expiry Flag

From this function you can Modify the expiry status of an Item.

Before changing the expiry related status of an Item, the number of outstanding transaction(s) (which are not finalized or not posted in Stock Ledger or not interfaced to GL) are selected. If there are any outstanding transactions for the Item, the expiry flag cannot be modified. For a selected Item if the expiry flag is changed from non-expiry to expiry, you can enter expiry date for the available quantity. If the expiry flag is changed from expiry to non-expiry, the system will update the expiry date to receipt date, which is the current date.

Modifying the expiry status of an Item

From the ST Transactions > Expiry Management, select the Change Expiry Flag function.

The Change Expiry Flag screen gets displayed.

  1. Enter the Item Name in alphanumeric format of 60 characters maximum length or select the required Item Name using the [?] function. This is a mandatory field.

  2. On selecting the Item, the Expiry Flag check box is enabled and selected if no pending transactions exist for the selected Item. This check box disabled, when outstanding transactions exist for the selected Item.

  3. The Item Quantity on hand for the selected Item is displayed as read-only text.

  4. Click Transaction tab to view the number of unfinalized/GL uninterfaced transaction.

  5. Click Store Level tab to view the store level unfinalised/GL Uninterfaced transactions.

  6. Click Batch Details tab to view the Batch details. If the expiry flag for the selected Item is selected, an option to change the Expiry Date is available.
