Issue Return

From this function you can Record the details of the Items returned from one Store to another Store. You can Query for the Returned Records and Authorize the Returned Records.

You can record the Item Return details using the Create mode or the User’s having privilege to authorize can authorize the Return Details using Query mode.

Recording the Item Return Details

From the ST Transactions > Issue, select the Issue Return function.

The Issue Return screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Modify Document Details screen is displayed by default or click ‘Document’ hyperlink to modify the document details.

    Enter the following details in the document section:

     

    1. Doc Type – Select the required Document Type from the list. The documents associated to the transaction ‘Issue Return’ are displayed in the list. The Document Type, as set in ‘ST Facility Parameter’ is displayed, by default. This is a mandatory field.

    2. Doc No. - The Document Number generation depends on the selected Document Type. If the Document Number has to be manually generated for the selected Document Type, you are informed. Enter the Document Number in numeric format of eight digits maximum length. If the system generates the number, you are not allowed to enter the field value. This is a mandatory field.

    3. Doc Date - The system date is displayed, by default. You can change the Document Date by entering the date in date format of dd/mm/yyyy. You can modify the date only if the ‘Doc Date Modifiable Y/N’ check box is selected in ST Transaction Type for the transaction Issue Return. This is a mandatory field.

    4. Doc Ref – Enter the Document Reference in alphanumeric format of 15 characters maximum length.

    5. From Store – Select the ‘From Store’ from the list. The Stores for which you have access and ‘Issue Return’ transaction function applicable are displayed. This is a mandatory field.

    6. To Store – Select the ‘To Store’ from the list. The Stores for which you have access and ‘Issue Return’ transaction function applicable are displayed. This is a mandatory field.

    7. Item Class – Select the required Item Class from the list.

  3. On entering the Document Details, click [OK] to record Item Return details or [Reset] to clear the entered data.

  4. If you select Issue Document based return option in the Facility Parameter screen, the Issue Doc Type and Issue Doc No fields are enabled and Item Search is disabled.  

    • Select the Issue Document Type from the list. The listed values are ‘Req Issue’ and ‘Urgent Issue’. This is a mandatory field.
    • Select the Issue Document Number using [?] function. The document numbers are displayed based on the selected Issue Document Type. This is a mandatory field.

  5. Click [Batch Search] to add multiple items. Select one or more batches from the Batch search popup. Add the chosen item/batch to the list of items selected for returning to the left frame of the screen.

The Batch search popup displays a list of all available batches for returns of the chosen item. If the same batch is received against multiple issue documents, the same batch is shown as multiple line items along with the corresponding issue document numbers. For example, 5 Items of Batch A received against Issue Document No. 10 is one line item and 6 items of the same Batch A but received against another Issue Document No. 11 is another line item.

  1. Click [Close] to exit from the batch search screen. The displayed Batch details are refreshed on selecting the new batch.

  2. You can also change the batches of already selected items by clicking on the required item available on the left frame of the screen.

The sum of the quantities of all batches chosen must be equal to the Return Quantity.

  1. The Add button changes into a Modify button in this scenario. After the required batch is selected, you can click Modify to update the changes after which the button reverts to Add to enable adding of more items. Click [Cancel] to cancel the modified changes.

  2. Click [Stock Level] to view the ‘From’ and ‘To’ stock details.

  3. You can select the ‘Delete’ check box to delete the selected record.

  4. Click [Preview] to view the Issue Return Preview details.

  5. Click [Apply] to finalize the Item return details or [Reset] to clear the entered data.

  6. The ‘Confirm Finalize’ dialog is displayed. Click [Yes] to finalize the Item return details or [No] to finalize the Item return details in the query mode.

  7. The ‘Confirm Printing’ dialog is displayed based on the attribute set for the Document Type. Click [Yes] to print the Item return details or [No] to cancel the action.

  8. Click [Query] to invoke the Query screen.

  9. The unauthorized Item return details are displayed based on the default criteria or enter the search criteria and click [Search] to select a specific record.

    Enter the following search criteria:

     

    1. Doc Type – Select the Document Type from the list. The values listed are defined in the function ‘ST Document Type’ and the default value displayed is specified in the function ‘ST Facility Parameter’.

    2. Doc No. – Enter the Document Number in numeric format of eight digits maximum length.

    3. Doc Date - Enter the date in date format of dd/mm/yyyy or select the date using the Calendar function.

    4. From Store – Select the ‘From Store’ from the list.

    5. To Store – Select the ‘To Store’ from the list.

  10. On searching, the unauthorized Item return details are displayed.

  11. Click ‘Doc Type’ hyperlink to authorize the Item return details.

  12. Click [Apply] to finalize the Item return details or [Delete] to delete the selected record.

  13. On clicking [Apply], the ‘Confirm Finalize’ dialog is displayed. Click [Yes] to finalize the Item return details or [No] to finalize the Item return details in the query mode.

  14. On selecting the ‘Delete’ check box, ‘Confirm Delete’ dialog is displayed. Click [Yes] to delete the record or [No] to cancel the action.
