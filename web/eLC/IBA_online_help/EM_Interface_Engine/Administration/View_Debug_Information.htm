View Debug Information

From this function you can View debug information.

View Debug Information function is used for viewing debug information which is generated while creating events (based on EM HIS events), constructing the HL7 messages or processing the query messages that are received from the interfaced applications.

Information in this function helps the user in identifying the reason for any failure. There are two tabs in this function – ‘Triggers’ and ‘Procedures’. The Triggers tab describes the debug information associated with the various triggers. The Procedures tab describes the debug information associated with the various stored procedures.

Viewing Debug Information

From EM Interface Engine > Administration > select View Debug Information function.

The View Debug Information screen gets displayed.

  1. Enter the required details.

    1. Application – Select the required application from the list. The listed values are defined in the ‘Applications’ function of EM Interface Engine module.

    2. Facility – Select the required facility from the list.  The listed values are defined in the Facility function of System Manager module.

    3. Message ID – Enter the Message ID for which the debug information has to be displayed.

    4. Event Type – Select the suitable event type from the list. The listed values are defined in the Event Type function of EM Interface Engine module.

  2. The screen displays two tabs.

Triggers Tab

  1. Operated Date - The date on which the trigger has fired for inserting control log for HL7 communication is displayed.

  2. Operated By - The name of the Oracle user, who has done the transaction is displayed.

  3. Product Name - The product (SQL*Plus, Forms, etc.), which actually fired the trigger is displayed. In case of triggers, it is marked as SQL*Plus, because triggers are fired in the back-end (that is, Oracle database)

  4. Triggering Event - Type of transaction, that is, insert or update.

  5. Triggering Table - Table from which the trigger was fired is displayed.

Click the hyperlinked ‘Operated Date’ and the View Debug Information screen gets displayed.

The Triggering Condition is displayed. Click [Write Log], if you need the system to record the displayed triggering condition. (The file is written into C:EMHISReports in the Application Server).

Procedures Tab

  1. Operated Date - The date on which the procedure is executed is displayed.

  2. Operated By - The name of the Oracle user, who executed the procedure is displayed.

  3. Product Name - The product (SQL*Plus, Forms, etc.), which actually executed the procedure is displayed.

  4. Process ID - The process that executed the procedure is displayed. The backend process that constructs the HL7 message from the control table has no Process ID.

  5. Procedure Name - Name of the procedure executed is displayed.

Click the hyperlinked ‘Operated Date’. The View Debug Information screen gets displayed.

The Procedure Parameters for the procedure are displayed. Click [Write Log] to record the displayed Procedure Parameters. The file is written into C:EMHISReports in the Application Server.

Click [Reset] to reset the selected values.

  1. Click [Reset] to reset the selected values.
