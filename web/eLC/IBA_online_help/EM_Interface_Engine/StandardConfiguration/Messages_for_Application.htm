Messages for Application

From this function you can Associate Applications to Message Type and Event Type.

Select the desired Application and Message Type. The system displays a list of events. Select the suitable Event corresponding to the Application and Message Type.

Communication Types for Events are:

Outbound

Inbound

Both (Outbound and Inbound)

Interfaced application can be defined as ‘Outbound’, when an EM HIS application sends a Message to an interfaced Application using unsolicited interface approach. It is defined as ‘Inbound’, when an interfaced application sends queries to an EM HIS Application for collecting the required information using solicited interface approach.

Defining Messages for Application

From Standard Configuration, select Messages for Applications function.

The Messages for Applications screen gets displayed.

  1. Select appropriate Application from the list. The Applications are defined in the Applications function.

The screen displays two tabs

Messages

Click Messages Tab. The Message Types tab allows selecting various Events for a specified Message Type and Application.

  1. Message Type - Select the Message Type from the system defined list of valid values. On selecting the Message Type, the corresponding Event Types are displayed.

  2. The Events that are interfaced in the ‘Event Types’ function are displayed.

  3. Event ID - The Event ID applicable to the selected Message Type and Application is displayed. The Events that were ‘interfaced’ in the Event Types function get displayed.

  4. Event Name – The Event Name applicable to the selected Message Type and Application is displayed.

  5. Standard Code –

  6. HL7 Version –

  7. Communication Type – Select the suitable Communication TYPE for the event from the list. The available options are – ‘Both’, ‘Inbound’ and ‘Outbound’. The Communication Type cannot be changed if the Communication Type has been set to ‘Outbound’ or ‘Inbound’ in the ‘Event Types’ function.

  8. Select – Select the corresponding check box to enable the Event for the selected Application.

  9. Click [Apply] to save the record or [Reset] to clear the entered data.

Application wise Segments

Click the Application wise Segments. The Application wise Segments tab allows selecting Segments for the selected Event Type and Application.

  1. Event Type – Select the desired Event Type from the list. The list gets populated with the Events that were ‘selected’ in the ‘Messages’ tab.

  2. Segment ID – The Segment ID of the Segment is displayed. The mandatory Segments for a particular Event Type are system-defined; however you have an option to include or delete the optional segments. The Segments ‘selected’ in the global setting in the ‘Segments’ tab of the Event Types function are displayed.

  3. Segment Name – The name of the Segment is displayed.

  4. Optional – The check boxes disabled. Segments may be mandatory or optional in the Message. The system displays the optional segments by placing a check mark in the corresponding check box.

  5. Select - Only optional Segments can be disabled or enabled. Select the corresponding check box to enable the Segments.

  6. Click [Apply] to save the record or [Reset] to clear the entered data.
