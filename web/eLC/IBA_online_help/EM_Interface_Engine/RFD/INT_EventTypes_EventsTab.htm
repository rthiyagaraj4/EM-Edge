Events

This Tab is displayed, by default. The Events tab populates all the Events grouped by Message Types and allows you to enable the events that are to be used at the specific site for the EM Interface Engine.

  1. Message Type - Select the Message Type from the system defined list of valid values. On selecting the Message Type, the corresponding Event Types are displayed.

  2. Event ID – The ID of the Event is displayed. The Events for a particular Message are system-defined. When you select the required Message Type, the system displays a list of the Events with their ID's and names.

  3. Event Name – The name of the Event is displayed.

  4. Communication Type - Select the Communication Type of an Event. The default Communication Type of an event is displayed. The listed values are ‘Outbound’, ‘Inbound’ or ‘Both’. You have to select the ‘Both’ option as the Communication Type can be made more specific in the ‘Messages for Application’ function.

  7. Interfaced - Select the Interfaced check box to enable the Event Type considering the requirements of all applications that are interfaced with CSC using EM Interface Engine. You have to select the check box as this is a global level setting. Only the interfaced messages are displayed under the Messages tab in the ‘Messages For Applications’ function.

  8. Click [Apply] to save the record.

‘Inbound’ indicates that the Event in the selected Message Type can be used for Inbound purposes.

‘Outbound’ indicates that the Event in the selected Message Type can be used for Outbound purposes.

‘Both’ indicates that the Event in the Message Type can be Inbound or Outbound.
