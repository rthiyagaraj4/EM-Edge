Pending Orders Tab

This tab displays orders that require bookings. The specific order is viewed and a booking is done by entering appropriate criteria. OT Orders that were placed through the Clinician Access and those that were not scheduled while placing the order are displayed in this tab.

  1. Click [Search] to display orders that require bookings or enter the required query criteria to search for a required order. Click [Clear] to clear the entered details.

    1. Specialty - Select the Specialty using [?] function.

    2. Surgeon - Select the Surgeon using [?] function.

    3. Patient ID - Select the Patient ID using [?] function. The name of the selected patient is displayed.

    4. From Date/To Date - Enter the date range to display the pending orders for booking. The current system date is displayed, by default. It is mandatory to enter the 'From Date'.

    5. Order ID - Select the required Order ID using the [?] function. An Order ID is generated when an order is placed through the Clinician Access module.

    6. Booking Priority – Select the Booking Priority from the drop-down list. The listed values are Priority-1, Priority-2 and Priority-3.

  2. The screen displays the Patient ID, Patient Name, Gender, Surgery Type, Procedure, Preferred Date, Priority, Pending Days, Specialty and Surgeon.

The pending day column will be blank when the pending day is zero or less than zero. If the pending orders with Priority – 1 are displayed in saffron color. The Color code   is used for pending order with booking priority.

  1. Click Patient ID hyperlink to display Book Appointment, Amend Appointment and Waitlist for the patient by selecting the required option. Click Patient ID hyperlink to Book Appointment or Waitlist for the patient by selecting the required option. Select Waitlist and the Patient are displayed in the Waitlist Tab. Select Book Appointment option to allow booking a new appointment for a single order or for multiple orders selected for a Patient ID. Single appointment is done by selecting the Patient ID arrow where check boxes are not selected in the list. Also you can select a check box and its corresponding Patient ID arrow is enabled in the list,  so that the single appointment is booked in the Schedule Status Tab. For multiple orders booking is done by selecting more than one check boxes for a Patient ID; irrespective of whether the selected orders belong to the same specialty or different specialties. A ‘Select Primary Specialty’ webpage dialog is displayed to allow you to select the default Specialty to be followed across the system.

  2. Select the Amend Appointment option to display Amend Booking dialog to list all the bookings for a patient ID for future time/Date.

    1. Booking Number – The booking number is displayed. You cannot edit the data.

    2. Booking Date/Time – The booking Date/Time is displayed. The number of days in the future within which the existing bookings are retrieved by the system are based on configuration done in the Setup Parameter for facility screen. You can select a single existing booking from the list which will be the default booking to be followed across the system. The operations for the multiple orders selected in pending orders list will be merged into the selected existing booking for the Patient ID on the same day. You cannot edit the data.

    3. Theatre – The operation Theatre name is displayed. You cannot edit the data.

    4. Procedure – The procedure to be taken place in the operation theatre is displayed. You cannot edit the data.

    5. Specialty – The Specialty name is displayed. You cannot edit the data.

    6. Time [From-To] –The surgery time range is displayed as slots.

    7. Select the radio button against the default bookings from among a list of bookings already available in the system for the Patient ID. Upon selecting an existing booking, the Day Schedule for the selected existing booking is displayed in the same screen. You select the existing slot by clicking on the slot start time of the existing booking to either retain the previous slot duration or to extend the existing slot duration if continuous slots are available. You can select the new slot by clicking on it in the Day Schedule to book all the operations into. The slot duration is continuous and meets the operation duration criteria. On clicking a slot, the system will launch the ‘Book Appointment’ screen with the operation details of the existing booking defaulted in the order pane of the screen. All operations of the existing bookings and operations of the selected orders will be displayed in the list view.

    You can modify the operation duration for the operations of the existing bookings and operations of the selected orders. The Total Operation Duration shall not exceed the available slot duration. You can exit the Book Appointment screen and choose another available slot to perform the booking. Once an OT appointment for the operations of the selected single or multiple orders is successfully booked into an existing booking, the system will return to the Pending Orders screen.

     

    If there are no future bookings available within the system for a Patient ID, the system will prompt an error message ‘No future bookings available to amend’ is displayed.

  3. Select Waitlist and the Patient are displayed in the Waitlist Tab. The Waitlist option is not displayed if multiple orders are selected for a Patient ID. The following Primary Specialty details ‘Surgery Type’, ‘Procedure(s)’, ‘Preferred Date’, ‘Specialty’, ‘Surgeon;, ‘Anesthesia Required’, ‘Patient Class’, ‘Encounter ID’, ‘Encounter Date/Time’ are displayed. The details are sorted by Patient ID at first, by Order Date at second time, by procedure at third time. Select the radio button before the Surgery Type to Primary specialty.

  4. Click [OK] to redirect you to the existing Schedule Status tab to select a slot for booking the new appointment.

 Once an OT appointment for the operations of the selected single or multiple orders is successfully booked, the system will return to the Pending Orders screen. After an OT appointment is successfully booked for multiple pending orders, the primary order does not require a consent, secondary order(s) requires a consent and consent is not already captured within the system, the system will display the error message ‘‘Consent is not taken for the patient’ prompt while checking into OR. This will be applicable only for the respective secondary order among the multiple orders booked only when the consent is not already captured in the system. If the consent has already been captured for the secondary order, the consent details are carried to the new order and you will not be required to record the consent again.
