Patient Search

From this function you can Search records of the patients by specifying desired search criteria.

 

Based on the specified search conditions, records are displayed for selection. When the search function retrieves a single record, the control automatically returns back to the called function along with the retrieved data. If the search function retrieves more than one record, then these records are displayed in the result area. You can select the appropriate record. On selecting the required record, the search screen is closed and control shifts back to the called function.  

In MP parameter, if “Accept Patient Name in Local Language” is checked, then patient name is entered in local language as one of the search criteria. If “Maintain Family Link” is set in MP parameter, the Head patient Identifier is specified for searching the required patient. Based on the result formats specified in the function “Patient Search Result Format”, the retrieved details are displayed in the result area.

SEARCHING PATIENT RECORDS:

From the MP Transactions> select the Query function for any transactions.

The Patient Search screen displays

  1. Enter at least one search criteria if not all and click [Search] to display patient records according to the entered criteria. <Related Field Description>

    Patient Records

    1. Result Page – The first set of 10 records are displayed initially. If more than 10 records are retrieved for the search criteria, then each set of 10 records is grouped and numbered. These numbers are hyperlinked. To view the records under a number, click on the hyperlink. The page numbers displayed on the screen is limited to 5 pages. Click on the numbers to view the details of patients in the respective pages.

    2. Order By - By default, Patient records are sorted and displayed in the ascending order of the value specified for the field 'Order By'. Using the column hyperlink, the user can further sort the result set. On clicking the column hyperlink, the displayed result set is sorted and displayed in the ascending order of the column.  

    3. + - The '+' hyperlink is provided against each record. Click '+' to view the demographics of the patient.

    4. Patient ID – Click on the patient ID of the patient is displayed.

    5. Color Coding - Status of the patient is indicated by a unique color code.  The user cannot access these records. The color-codes displayed are:

     (Red) - Indicates the Patient is deceased.

     (Brown) - Indicates the Patient record is Inactive.

     (Grey) – Indicates Patient is suspended.

    1. Sex – The gender of the patient is displayed.

    2. Date of Birth – The Date of Birth of the patient is displayed.

    3. National ID No. – The National ID No. of the patient is displayed.

    4. Alternate ID No. – The Alternate ID No. of the patient is displayed.

    5. Name Prefix - The Name Prefix of the patient is displayed.

    6. First Name - The First Name of the patient is displayed.

    7. Second Name - The Second Name of the patient is displayed.

    8. Third Name - The Third Name of the patient is displayed.

    9. Family Name - The Family Name of the patient is displayed.

    10. Name Suffix - The Name Suffix of the patient is displayed.

    11. Nationality - The Nationality of the patient is displayed.

    12. M. R No. – The MR No. of the patient is displayed. Click Multiple MRNs hyperlink to view the Patient File Numbers of the Patient by invoking Patient File No dialog.

    13. MR Facility – The MR Facility of the patient is displayed.

    14. Old MR No. - The Old MR No of the patient is displayed.

    15. Registration Mode – The registration mode of the patient is displayed.
