Register Patient

From this function you can Register a new patient in a facility by generating a unique identification number and Query for the registered patient.

The details captured while registering a patient are Demographic Details, Contact Addresses, Related Contacts, Documents and Others and Financial Details. This function allows registering patients routed through appointments, referrals and inpatient bookings. Each patient is assigned a Patient Series during the registration process and based on the numbering rules defined for the selected Patient Series, Patient ID is generated. On registration, Medical Record Number is generated based on the MR Section associated to the assigned Patient Series. A permanent file storage location is selected, if Patient File Tracking module is installed.  A patient file is created with appropriate file storage location. A relevant patient file label is also printed for the respective patient file. Online reports applicable to the assigned Patient Series are displayed for selection on generating the Patient File Number.

It is not mandatory to record all details during registration.

Registering a New Patient

From the MP Transactions > select the Register Patient function.

The Register Patient screen gets displayed.

  1. Click [Create] to invoke the Create screen.

 

Based on the MP Parameter Setup, perform any one of the following:

Registering patient using Alternate ID check

If external source is interfaced, on entering the Alternate ID, the patient’s demographics available for the entered ID is displayed in the ‘Register Patient’ screen (the routine interfaces the MP module with the external source).

  1. Click [Create].

  2. If Default Patient Series (MP Parameter > Patient Administration Tab > Patient Series ‘Default’) is set as Alternate ID 1, the Alternate ID entry screen gets displayed.

  3. Enter details in the mandatory (*) fields. <Related Field Description>

  4. Click [OK] to proceed with Register Patient screen or [Cancel] to cancel the Alternate ID entry. The Register Patient screen gets displayed.

 

Registering patient using National ID check

If external source is interfaced, on entering the National ID, the patient’s demographics available for the entered ID are displayed in the ‘Register Patient’ screen (the routine interfaces the MP module with the external source).

  1. Click [Create].

  2. If Default Patient Series (MP Parameter > Patient Administration Tab > Patient Series ‘Default’) is set as National ID, the National ID entry screen (as set in Prompt of MP Parameters) gets displayed.

  3. Enter details in the mandatory (*) fields. <Related Field Description>

  4. Click [OK] to proceed with Register Patient screen or [Cancel] to cancel the National ID entry. The Register Patient screen gets displayed.

 

 

If Default Patient Series (MP Parameter -> Patient Administration Tab -> Patient Series ‘Default’) is set as General, the Register Patient screen displays to register the patient.

The screen displays the following Tabs: Demographic Details, Contact Addresses, Related Contacts, Documents and Others and Financial Details.

 

  1. Select desired value from ‘Populate From’ list. The valid values are Patient Information, Appointment Schedule, Referral and IP Booking. This field is displayed if the ‘Accept Appt Ref No/Referral ID in Register Patient’ field is selected in the MP Parameter.

 

Registering through Patient Information

This function provides an option to perform a quick registration by searching for an already registered patient (eg. accompanying attendant) with same demographic details.

" When the patient with same demographic details is entered the following details are populated. Nationality, Race, Religion, Ethnicity,  Language, Contact Address -  Residence Address, Mailing address and Primary residence number.

The Patient Information value in the 'Populate From' list enables you to enter the Patient ID or search for the Patient ID through patient search screen from which the Demographic details are displayed. The Demographic details can be edited.

Registering through Appointments

  1. Select ‘Appointment Schedule’ from the ‘Populate From’ list.

  2. Click [?] to search Appointment scheduled patients, by invoking the Appointment/Referral Details screen.

 

Registering a Referral Patient

  1. Select ‘Referral’ from ‘Populate From’ list.

  2. Click [?] to search Referral Patients by invoking the Appointment/Referral Details screen.

 

Registering through IP Bookings

  1. Select ‘IP Booking’ from ‘Populate From’ list.

  2. Click [?] to search IP Booking patients by invoking Appointment/Referral Details screen.

 

Registering Patients through Emirates ID card

How it works:

Connect the system to a card reader that can read the Emirates Card of the patient. The data from the Emirates ID will be stored in a temporary file named Emirates_ID.txt and saved in the path C:\SmartIC. The Enterprise Management System will receive the Emirates ID Number, patient demographics and contact details. No information from the EM application will be sent back to the Emirates ID Card.

 

Scenario 1 -  When registering a patient for the first time:

1. Launch EM application and the Patient Registration screen and swipe the Emirates ID card of the patient onto the connected card reader.

2. Click the  button next to the field Emirates ID. The details from the temp file Emirates_ID.txt will be populated onto the relevant fields on the screen. The temporary file in the System will be deleted.

 

Scenario 2 - When the Patient is already registered with the hospital:

1. Launch the “Patient Registration Search screen and swipe the Emirates ID card of the patient onto the connected card reader.

2. Click on the  button; the system will throw an error message as “Duplicate Emirates ID”

 

  1. Enter details in the mandatory (*) fields of the following tab pages.

 

Demographic Details

  2. Enter the details in the mandatory (*) fields. <Related Field Description>

Contact Addresses

  4. Click First to Notify tab to record the details of the first person to notify the incident in case of emergency.

  5. Enter details in the mandatory (*) fields. <Related Field Description>

  6. Click Employment Details tab to record employment details of the patient.

  7. Enter details in the mandatory (*) fields. <Related Field Description>

The details are updated in ‘Change Patient Details’ function.

Documents and Others

  1. Click Documents and Others tab to record the details of documents available with the patient during registration.

  2. Enter details in the mandatory (*) fields. <Related Field Description>

The details are updated in ‘Change Patient Details’ function.

Financial Details

  1. Click Financial Details Tab to record the financial details of the patient.

  2. Enter details in the mandatory (*) fields. <Related Field Description>

 

  1. Click [Apply] to save the Patient Registration details or [Reset] to clear the entered data.

  2. Create Patient File screen gets displayed if file management parameters are set for the facility.

  3. Enter the details in the mandatory (*) fields. <Related Field Description>

  4. Click [OK] to generate file number or [Cancel] to cancel file generation.

  5. If internal reports are defined, the Internal Reports screen gets displayed; click [OK] to generate Internal Reports or [Cancel] to exit from the screen.

  6. Patient ID and File Number is generated and displayed.

  7. Click [Query] to search for existing Patient details.

 

If ‘Generate Patient ID Using Alternate ID1 Rule’ check box is selected and 'Family/Organization' settings are defined in MP Parameter function, the Patient ID is generated based on the rules defined in ' Patient Numbering for Relationship' function. For example, if the Alternate ID of an Organization Member is 000031, and the Prefix values set in 'Patient Numbering for Relationship' function is 1, 0 and 1 with the Separator Value 'R', the Patient ID generated is ‘000031R101’. The Separator Value is placed based on the 'Separator Position' selected in the Patient Series function.
