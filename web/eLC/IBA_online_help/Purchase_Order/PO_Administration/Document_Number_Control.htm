Document Number Control

From this function you can Define the serial number in which the document number needs to be generated for purchase order and purchase request.

This function allows defining the serial number in which the document number needs to be generated when the Autogenerate option is chosen for a transaction type. You will be allowed to enter a unique document number during Purchase Request, Purchase Order transactions, Delivery Order and Quotations if Manual Doc Generation method is selected.

The Document Number Control is a one time setup.

Defining PO Document Number Control

From the PO Admin Setup, select the PO Document Number Control function.

The PO Document Number Control screen gets displayed.

  1. Select the Doc Gen. Method as ‘Autogenerate’ or ‘Manual’ option from the drop-down list for Purchase Order and Purchase Request.  

  2. Select the Prefix Facility, Prefix Purchase Unit, Prefix Year check boxes if Autogenerate option is selected.

  3. Enter the Year for the transactions.

  4. Enter the Starting Doc. No in numeric format of maximum ten character length.

  5. Enter the Next Doc. No in numeric format of maximum ten character length.

  6. The system generated serial number contains Facility code (two characters), Purchase Unit code (six characters), Year (two characters) and system generated serial number (ten characters).

  7. When ‘Manual’ is selected from Doc Gen. Method list, the Prefix Facility, Prefix Purchase Unit and Prefix Year check boxes are disabled. You can enter the document number in alphanumeric format of 20 characters maximum length for Purchase Request and Purchase Order.

  8. Click [Apply] to save the Document Number Control setup or [Reset] to clear the entered data.
