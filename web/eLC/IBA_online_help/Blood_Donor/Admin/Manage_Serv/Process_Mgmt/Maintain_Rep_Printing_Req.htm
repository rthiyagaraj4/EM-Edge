Maintain Report Printing Requests

From this function you can Maintain the report printing requests, Print the reports and Delete the records. You can also View the reports and error logs.

The Maintain Report Printing Requests function is an administrative function that maintains the report printing requests. It also provides options to view the report and error logs, print and delete the records. The statuses of report printing requests are defined in different colors. The indicates that the printing is in process, indicates that there is an error while printing and indicates that the report has been printed successfully.

Maintaining the Report Printing Requests

From the Administration > Manage Services > Process Management, double-click Maintain Report Printing Requests function.

The Maintain Report Printing Requests screen gets displayed.

  1. The details of Report Printing Requests are displayed. <Related Field Description>

  2. Select a report printing request and click [View Report] to view the output file.

  3. Select a report printing request and click [View Error Log]. The system pops-up a window that displays the corresponding printing error message information.

  4. Select a report-printing request and click [Print Report]. The corresponding error messages get deleted (if any) and an active request is made for reprinting. The system highlights the active printing requests in .

  5. Select a processed (highlighted in ) request and click [Purge]. The request gets deleted after the confirmation message is acknowledged. [Purge] removes the report file name from the Operating System. [Purge] does not delete the active records.

  6. Click [Purge All] to delete all the processed (highlighted in ) requests from the queried requests after the confirmation message is acknowledged. Purge All removes the report file name from the Operating System. This does not delete the active records.
