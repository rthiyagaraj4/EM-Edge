Encounter Specific Policy Definition - Services

From this function you can Include or Exclude the services for a particular patient encounter associated to a Policy Type and Query for a desired Encounter Specific Policy Type record.

The function Encounter Specific Policy Definition – Services is used to include or exclude the services for a particular patient encounter in the facility. The coverage charge can be recalculated for specific patient encounter. You can associate additional services to the patient encounter. The services associated to the selected Policy Type, Customer Group and the Customer are defined using the function Setup Service Coverage Detail by Policy. You can associate additional services to the patient encounter other than the services, which are already associated to the Policy Type, Customer Group and the Customer. You can also delete the services from the patient encounter.

Encounter Specific Policy Definition - Services

From the Masters, double-click on Encounter Specific Policy Definition function.

The Encounter Specific Policy Definition screen gets displayed.

  1. Select an Episode Type from the list. The values are system defined - Inpatient, Daycare, Outpatient and Emergency. The policy definition for services is applicable for the selected Episode Type.

  2. Click [List of Values] to select an Episode ID. The Episode ID applicable for the selected Episode Type gets listed. The patient details get displayed based on the Episode ID selected. This is a mandatory field.

  3. The Policy Type details associated to the patient during the specific encounter registration gets displayed. <Related Field Description>

  4. Click [View] adjacent to each record to view the service coverage detail for a specific policy.

  5. You can include additional services to the specific encounter and define the coverage limit for the specific policy using the tabs:

Policy Excluded/Included Service Dtls

Enter the following details:

  1. Exclude/Include Criteria – Select a value from the list. The system defined values are ‘Billing Service’, ‘Payer Service Group’ and ‘Payer Service Group/Line’.

  2. Exclude/Include Code – Click [List of Values] to view and select a Billing Service, Payer Service Group or Payer Service Group/Line.

  3. Special Item – Select the check box, if special items are associated to the specific encounter. Clear the check box and you cannot associate special items to the range of policies.

  4. Excl Incl Ind – Select a value from the list. The system defined values are ‘Excluded’ and ‘Included’.

  5. Override Incl Excl – Select the check box to override the exclude/include indicator selected for the Billing Service associated. The override performs inverse action based on the value selected for ‘Excl/Incl Ind’.

  6. Include Policy Level – Select the check box, if the service exclude/include detail is valid only for the selected policy.

  7. Payment Indicator - Select a Payment Indicator from the list. The system defined values are ‘%age’, ‘Rate’ and ‘Fixed’. This is applicable for co-pay value calculation. On selecting ‘Percentage’, the symbol % is displayed beside it. On selecting ‘Rate’ or ‘Fixed’, currency name is displayed beside it.

  8. Copay Value – Enter the Copay Value in numeric format. The amount to be settled by the Co-payer is calculated based on the Payment Indicator selected and the value specified for Copay value.

  9. Preapproval – Select the check box, if preapproval is necessary to associate the services and the defined co-pay value.

  10. Effective Date – Click Calendar function to select the date. The service details are valid for the selected encounter on the date specified.

  11. Approved – Select the check box and the parameter defined is approved for the selected encounter.

  12. Coverage includes other Payers - Select the check box to define an additional payer for the specific Policy, Payer and Payer Group. Clear the check box, if an additional payer is not required for the selected combination.

  13. Last Payer - Select the check box, if the payer is the last payer. Based on whether the coverage amount is selected and the payer is the last payer or not, the policy coverage amount is distributed as follows:

  14. If the check boxes ‘Coverage includes other Payers’ and ‘Last Payer’ are selected, the amount is sent as discount.

  15. If the check box ‘Coverage includes other Payers’ is selected and the check box ‘Last Payer’ is cleared, the remaining amount has to be paid by the patient.

An alert message is displayed, if the Discharge Bill is generated for the selected Episode. Press ‘Yes’ to continue with the encounter specific policy definition.

Payer Coverage Limits

Enter the following details:

  1. Serv Coverage Limit Crit – Select a value from the list. The system defined values are ‘Billing Service’, ‘Payer Service Group’ and ‘Payer Service Group/Line’.

  2. Exclude/Include Code – Click [List of Values] to view and select a Billing Service, Payer Service Group or Payer Service Group/Line.

  3. Description – The description for the selected Billing Service code gets displayed.

  4. Amt Limit Per Enc – Enter the maximum limit amount covered for the selected billing service associated to the patient encounter in numeric format.

  5. Amt Limit Per Day – Enter the maximum limit amount per day in numeric format. The amount limit is valid for the selected service and is valid for a day and applicable to the selected patient encounter.

  6. No Of Days – Enter the number of days until when the coverage amount limit is valid for the selected patient encounter.

  7. Ovride Limit – Select the check box to override the coverage limit. The override performs inverse action based on the value selected for ‘Amt Limit Per Enc’ or ‘Amt Limit Per Day’.

Click [Insert Record] to associate or dissociate additional services from the specific encounter associated to a Policy Type.

  1. To view the history of the service details with the policy coverage and the payer coverage limits associated to the selected encounter, click on the tabs:

Policy Excluded/Included Service Dtls - History

Using this tab, you can view the services included/excluded from the policy.

Payer Coverage Limits - History

Using this tab, you can view the payer coverage limits recorded for a patient encounter.

  1. Select the check box 'Recalculate Charges' to re-define the charges for the specific encounter. If the discharge bill has been generated, an alert is displayed. The charges are re-calculated based on the transaction.  

  2. Click [Commit Form] or press [F10] to save or click [Clear Record] to clear the entered data.

  3. Click [Execute Query] or press [F8] to search for the existing records within the facility.
