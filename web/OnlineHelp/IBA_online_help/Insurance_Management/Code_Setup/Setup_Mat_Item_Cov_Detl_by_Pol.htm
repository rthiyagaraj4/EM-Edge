Setup Material Item Coverage Detail by Policy

From this function you can View the coverage details defined for the material item and Query for a desired record.

The function Setup Material Item Coverage Detail By Policy is used to view the coverage details defined for the material items associated to the selected Policy. The coverage details for material items are set up using the function Setup Service Coverage Detail by Policy. The header part details are displayed on pressing [F8] or clicking [Execute Query]. You can associate additional services and define Payer Coverage details using this function. You can update the material items associated to a Customer Group, Customer and Policy Type combination in a facility.

Setup Material Item Coverage Detail by Policy

From the Masters, double-click on Setup Material Item Coverage Detail by Policy function.

The Setup Material Item Coverage Detail by Policy screen gets displayed.

  1. Click [Execute Query] or press [F8] to search for the all the existing records within the facility.

OR

  1. Click [Enter Query] or press [F7] to search for a specific record within the facility.

  2. The material item coverage details defined using the function Setup Service Coverage Detail by Policy gets displayed.

  3. Click Policy Excluded/Included Service Details to view or include or exclude the material items from the list.

    Policy Excluded/Included Service Details

    1. Exclude/Include Criteria - Select a value from the list. The system defined values are 'Payer Material Group' and 'Material Item'.

    2. Patient Class – Click [List of Values] to select a Patient Class. The selected material item is either excluded or included for the selected Patient Class. The system defined values are Inpatient, Outpatient, Daycare, Emergency, External and All. Select ‘All’, if the selected material item is either excluded or included in the coverage details for all the patient class in the facility.

    3. Description – The description for the selected Patient Class is displayed.

    4. Code - Click [List of Values] to view and to select a valid material item code.

    5. Description – The description for the selected material item is displayed.

    6. Special Service – Select the check box to associate special services.

    7. Excl/Incl Indicator – Select either Exclude or Include from the list. The special services are associated to the selected Policy based on the value selected from the list.

    8. Exclude/Include Criteria - Select a value from the list. The system defined values are ‘Excluded’ and ‘Included’. This indicates if the material item is eligible for policy coverage or not.

    9. Override Incl/Excl – Select the check box to override the include/exclude criteria for a selected material item. Clear the check box and you cannot override the include/exclude criteria for the selected material item.

    10. Include Policy Level Coverage – Select the check box to include coverage details for the selected material item associated to the Policy.

    11. Inclusion Basis – Select a value from the list. The system defined values are ‘Percentage’,  ‘Rate’ and  ‘Fixed’.

    12. Percentage - Select Percentage to calculate the coverage amount in terms of percentage for the selected material item when the Excl/Incl Indicator is selected as ‘Include’.

    13. Rate - Select Rate to calculate the coverage price as an amount for the selected material item when the Excl/Incl Indicator is selected as ‘Include’.

    14. Fixed - Select Fixed to calculate the coverage amount in terms fixed price for the selected material item when the Excl/Incl Indicator is selected as ‘Include’.

    1. Value – Enter the value in numeric format to define the amount, which has to be settled by the payers for the material items when the Excl/Incl Indicator is selected as ‘Include’.

    2. Pre-Approval Required – Select the check box, if a pre-approval is required to include or exclude the material item to the selected Policy Type, Customer and Customer Group. The approval is required in the billing transaction functions.

    3. Pref Check – Select the check box to associate to associate a specific drug or a generic drug to the material item.

    4. Pref Basis - Select a value from the list. The system-defined values are 'Generic' and 'Specific'. This text box is enabled for selection if the check box 'Pref Check' is selected. Select Specific to associate and define coverage for associated drugs.

    5. Pref Item - Click [List of Values] to associate a Specific Drug item for coverage. You can select an Item only if the 'Pref Basis' is selected as 'Specific'. When a order is placed for the selected drug item in Pharmacy Management module an icon is displayed with an alert message indicating that selected generic or specific item is included or excluded from policy coverage.  

    6. Item Description – The Description for the selected item is displayed.

    7. Pref Description - The Description of the preferred specific item is displayed.

    Items having same Generic or Specific preference checks are prompted with alert for Preferred Items/Drugs. You can click on the icon to select alternate items while placing order for drug items in Pharmacy Management module, if the item is not under insurance coverage. The alternate items are valid, if the ‘Pref Check’ is selected as ‘Specific’. Items with the same generic or specific preference checks are treated as preferred drugs. When an item is set as generic, the drugs associated to the generic name of the selected item are considered as preferred items.

  4. Click Policy Coverage Limits to view or define the coverage limits.  

    Policy Coverage Limits

    1. Exclude/Include Criteria - Select a value from the list. The system defined values are 'Material Item' and 'Payer Material Group'.

    2. Patient Class – Click [List of Values] to select a Patient Class. The selected Material Item is either excluded or included for the selected Patient Class.

    3. Item – Click [List of Values] to view and select an item. The selected item is either included or excluded from the Policy Coverage based on the Exclude/Include Criteria selected.

    4. Description – The description for the selected item gets displayed.

    5. Amt Per Visit – Enter the coverage amount per Visit. The Insurance Company pays the amount specified towards an Outpatient visit for the selected Policy Type.

    6. Amt Per Admission - Enter the coverage amount per Admission. The Insurance Company pays the amount specified towards an Inpatient admission for the selected Policy Type.

    7. Per Inpatient Day – Enter the coverage amount for Inpatients per day. The amount set is specific only to the Inpatients admitted in the facility for the selected Policy Type.

    8. No Of Days – Enter the number of days for which the coverage amount is applicable towards the inpatient stay in the facility.

    9. Override Coverage Limit – Select the check box, if the coverage amount set can be changed during transaction.

  5. Click Coverage Limit By Mapping Code to record the coverage limit for a mapping code.

    Enter the following details:

    1. Patient Class – Click [List of Values] to select a Patient Class. The system-defined values are ‘Inpatient’, ‘Outpatient’, ‘Daycare’, ‘Emergency’, ‘External’ and ‘All’. Select ‘All’, if the Coverage Limit is valid for all patient class. You can select the same Billing Class for different Mapping Codes.

    2. Description - The description of the Patient Class is displayed.

    3. Mapping Code – Click [List of Values] to select the Mapping Code. The values that are defined through the ‘Service/Material Item Mapping’ function are listed.

    4. Mapping Description – The description of the selected Mapping Code is displayed.

    5. Term Set ID – Click [List of Values] to select the Term Set ID. You can associate the same term set ID to different Mapping Codes. This is enabled for selection only if the ‘Physician & Diagnosis Applicable for Coverage’ check box is selected for the respective Mapping Code.

    6. Term Set Description – The description of the selected Term Set is displayed.

    7. Term Code – Click [List of Values] to select the Term Code. The DRG codes that are associated to the selected Term Set ID are displayed. This is enabled for selection only if the ‘Physician & Diagnosis Applicable for Coverage’ check box is selected for the respective Mapping Code.

    8. Term Code Description – The Description of the selected Term Code is displayed.

    9. Physician ID - Click [List of Values] to select the Physician ID. You can select a value only if the ‘Physician & Diagnosis Applicable for Coverage’ check box is selected for the respective Mapping Code.

    10. Level - Click [List of Values] to select a Level. The coverage amount is based on the level selected. For example, if the Level is 1 and the coverage amount is 5,000, the amount covered is 5,000. If level 2 coverage is defined as 7,000, then the total coverage is the sum of [level1 + level2]. This is valid for Mapping Codes with the ‘Level Applicable for Coverage’ check box as selected.

    11. Include/Exclude – Select a value from the list. The system-defined values are ‘Include’ and ‘Exclude’. Select ‘Include’ if the coverage is valid for the selected diagnosis code and physician. Select ‘Exclude’, if the coverage amount is not valid for the associated diagnosis code and physician. You can specify the Copay Value and the Effective Date only if you select the value as ‘Include’.

    12. Rate – Select ‘Rate’ from the list to define Copay Indicator for the Mapping Code.

    13. Copay Value – Enter the Copay Value in numeric format. It defines the coverage amount for the Mapping Code.

  6. Click [Commit Form] or press [F10] to save or click [Clear Record] to clear the entered data.

 

Click [Insert Record] to insert additional records under the Policy Excluded/Included Service Details or Policy Coverage Limits tab.
