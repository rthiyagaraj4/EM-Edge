Enter the following details to generate a Consolidate Invoice:

 

  1. Customer Code [From – To] – Enter the required Customer Code in alphanumeric format of eight characters maximum length or select the required Customer Code from [List of Values]. The Customers defined in the Customer function are displayed for selection. If the code selected is valid, the Customer Name is displayed as read-only text for user verification. The Invoice is generated from that particular Customer. If ‘From’ value is not entered, the system assumes the lowest value for the ‘from’ value. If ‘To’ value is not entered, the system assumes the highest value for the ‘To’ value. When both values are entered, ‘To’ value must be greater than or equal to the ‘From’ value. This is a mandatory field.

  2. Bill Date [From – To] – Enter the Bill Date in the format of dd/mm/yyyy or click the Calendar function to set a valid date. The Consolidated Invoice is generated beginning with the specified date. The ‘To’ date cannot be greater than the current date. The current system date is displayed, by default. This is a mandatory field.

  3. Bill Type/Num [From – To] – Select the required Bill Type and Bill Number from [List of Values]. The entries with Bill Type and Bill Number get populated when the data is posted from the Patient Billing Manager module. If ‘From’ value is not entered, the system assumes the lowest value for the ‘From’ value. If ‘To’ value is not entered, the system assumes the lowest value for the ‘To’ value. When both values are entered, ‘To’ value must be greater than or equal to the ‘From’ value.

Invoice

  1. Doc Type Code – Enter the Document Type code in alphanumeric format of six characters maximum length or select the required Document Type Code from [List of Values]. The Document Type Code set for Invoices in AR Parameter is displayed, by default. This is a mandatory field.

  2. Doc Number/Date – On entering the Document Type Code, the Starting Document Number set for the selected Document Type in Financial Base module is displayed, by default. Click the Calendar function to set a valid date. The current system date is displayed, by default. This is a mandatory field.

 

Credit Notes

  1. Doc Type Code – Enter the Document Type code in alphanumeric format of six characters maximum length or select the required Document Type Code from [List of Values]. The Document Type Code set for Credit Notes in AR Parameter is displayed, by default. This is a mandatory field.

  2. Doc Number/Date – On entering the Document Type Code, the Starting Document Number set for the selected Document Type in Financial Base module is displayed, by default. Click the Calendar function to set a valid date. The current system date is displayed, by default. This is a mandatory field.

  3. Bill Type – Select the required Bill Type from [List of Values]. The entries with Bill Type and its description get populated from the Patient Billing Manager module. This is a mandatory field.

  4. Credit Ref – Enter the Credit Reference in alphanumeric format of 15 characters maximum length. This is a mandatory field.

  5. Post Month/Year – The month and year set in AR Parameter is displayed as Current Accounting Year, by default. You can modify the data if the Post Month and Year is greater than Current Accounting Year. This is a mandatory field.

  6. Facility ID – Select the required Facility ID from [List of Values]. The Facilities defined in the Facility function and associated to the Responsibility of the User in System Manager module are displayed for selection. The value ‘All Facilities’ is displayed on the screen, by default.

  7. Debit Control A/C – Select the check box, to generate the Invoice only for Debit Control accounts.
