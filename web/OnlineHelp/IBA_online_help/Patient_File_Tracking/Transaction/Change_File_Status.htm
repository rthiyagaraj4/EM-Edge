Change File Status

From this function you can Change the current File Status of the patient file.

This function is used by the Permanent File Storage Locations to update the File Status. The status changes allowed are from Active to Lost, Lost to Found and Active to Archive and Not To be Archived to Archived. You can invoke this function with appropriate access rights.

Changing File Status

From the FM Transactions, select the Change File Status function.

The Change File Status function screen gets displayed.

  1. Select the FS Location from the list. Based on the access rights of the logged in user the values are listed. This field is mandatory.

  2. The logged in user name gets displayed, by default, if the 'Security to be Enforced' check box is selected in the FM Parameter Setup. If 'Security to be Enforced' check box is cleared in the FM Parameter Setup, the field is enabled and you can select the required User Name using [?] function. The person responsible for changing the file status is specified.

  3. Enter the Patient ID in alphanumeric format of maximum length as specified in MP Parameter or select the Patient ID using [?] function. This field is mandatory.

  4. On selecting the Patient ID, the File Type(s) associated to the patient is displayed. On selecting the Patient ID, the default File Type associated in the File Type For FS Location is displayed. This field is mandatory.

  5. The active volume number is displayed. Select the required Volume Number from the list.

  6. Click [Add] to view the existing file details or [Clear] to reset the records displayed in the result area.  

  7. The‘Patient Details’ are displayed.

  8. Enter the File status details in the mandatory (*) fields. <Related Field Description>

  9. On entering the new status of the file, click [Apply] to record the file status change or [Reset] to clear the entered data.
