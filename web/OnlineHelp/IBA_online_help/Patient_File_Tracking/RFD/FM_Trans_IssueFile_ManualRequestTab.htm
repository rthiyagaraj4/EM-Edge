Manual Request

Patient files are requested electronically or manually. File can be issued against manual request. The details are recorded under 'Manual Request' Tab. The system validates the access rights of the requesting user. If access rights are not defined for manual Request, an alert message is displayed.

  1. Click Manual Request tab to request patient files manually.

  2. Enter the details in the mandatory (*) fields. <Related Field Description>

  3. Click [Files] to invoke Requested File Details frame or [Clear] to reset the entered data.

  4. Select the required Patient ID. On selecting the Patient ID, the File Type, Volume No, Current FS Location, File Status, and Remarks associated to the Patient File are displayed.

    • Inside status means the file is currently with the owner location

    • Outside status indicates that the file is outside the owner location.

    • In Transit status indicates that the file is issued to the requesting FS Location and it has not confirmed the receipt.

    • Lost status indicates that the file is lost

    • Archived status indicates that the file is inactive

    • Returned status indicates that the file is returned from the requesting location back to the permanent storage location.

  5. Click [Add for Issue] to add patient file to issue list.

To view the files selected for issue, click Issue File tab. The files considered for issue are listed. The files that are currently with the Permanent FS Location can be issued.
