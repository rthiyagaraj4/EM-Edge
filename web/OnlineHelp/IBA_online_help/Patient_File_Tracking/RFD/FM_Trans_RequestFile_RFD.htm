1. Patient Details - The Patient Details consists of Patient Name, Age, Gender and Patient ID.

  2. File Type/Volume No - The selected File Type and Volume Number are displayed.

  3. Request Remarks - Enter the Request Remarks. To enter the request remark, click the Enter Remarks hyperlink to display the Request Remarks screen. Enter the remarks and close the screen. The Request Remarks are entered only for Files with status as Inside or In Transit or In Other Facility. For files with status as Lost or Archived or In Requesting FS Location, remarks cannot be entered.

  4. Permanent FS Location - The Permanent Storage Location of the file is displayed.

  5. Current FS Location -The File Storage Location where the file is currently held is displayed.

  6. File Required On - Enter the date when the file is required in the date format of dd/mm/yyyy.

  7. File status - One of the following is displayed as the file status:

  8. Lost - if the file is lost

  9. Archived - if the file is inactive.

  10. In Transit - if the file is issued to FS Location.

  11. Inside - if the file is with the MR Department.

  12. Outside - if file is outside the owner location.

The files requested are categorized into two areas:

  • Requested file - Files with status as Inside (files that are currently held by the MRD), In Other Facility, In Transit (issued to other FS Locations) or Outside can be requested.

  • Files cannot be requested - Files with status as Lost, Archived, In Requesting FS Location or Returned cannot be requested.

  1. Request? - Select the check box to request for the file. The check box is selected, by default. Only the files with status as Inside or In transit or In Other Facility can be requested. Color-codes are used to differentiate the various file status. The color coding scheme is applicable for the following status.

  2. Transit

  3. Lost

  4. Archived

  5. Outside.

  6. Returned

  7. In Requesting FS Location

  8. In Other Facility

Select the required file from the list displayed by selecting the check boxes in the 'Request?' column.

For the logged in Facility and selected Patient ID, if files do not exist with the requested facility, the message ‘Patient files does not exist in the requested facility’ is displayed in the message area.
