Enter the following details to change the file status:

 

Current Status of the file

  1. Status - The current Status of the file is displayed and this value cannot be modified.

  2. FS Location - The current FS Location is displayed  and this value cannot be modified.

  3. Holder Name - The Holder Name of the file is displayed  and this value cannot be modified.

 

New Status of the file

  1. Status - Enter the new Status of the file. The matrix table below explains the contents of the list that depends on the current status. For example, if the current status is 'Inside', the list is populated with the values as 'Lost', 'Archived' and 'Never To be Archived'. This field is mandatory.

 

The new statuses are:

File Found

To Archive File

Return from Archived

Never To be Archived

 

File status can be changed from Found to Lost or from Lost to Found. Similarly File Status can be changed from Archived to Active or from Active to Archive. File status can also be changed from Never To be Archived to Archived. The new status of the file is dynamically determined based on the current status of the file. Subsequently the new status of the file is decided based on the following matrix table.

 

When the status of the File is in transit or in other facility, changes cannot be made from Found to Lost or Lost to Found, Archived to Active or Active to Archive and Never To be Archived to Archived.   1. FS Location - The FS Location is made default to the value set in the FM Parameter Setup. The changes to the FS Location can be performed based on the File Status.

  2. Updated Date - The current system Date and Time are displayed and this date cannot be modified.

  3. Updated By -Select the name of the person updating the status using [?] function.

  4. Remarks -Enter Remarks in alphanumeric format of 60 characters maximum length.
