Enter the following details:

 

  1. User – Select the User Name by clicking [?] function. This field is mandatory. The lookup function helps in searching for the required user. On clicking the lookup function, the ‘User – Web Page Dialogue’ screen gets displayed. Enter the following search criteria.

  2. Access to All FS Locations – Select the check box to associate the selected user to all the File Storage Locations of the logged in facility. To associate the user to specific File Storage Locations, clear the check box.

  3. FS Location Identity – Select the Identity from the list of valid values. The listed values are Permanent Department, Non-permanent department, Clinic, Nursing Unit, Practitioner, Day Care Unit and Procedure Unit. Depending on the identity selected, the File Storage Locations are populated. For example, if Identity is selected as Permanent Department, all the permanent File Storage Locations of the logged in facility are displayed as values for the field FS Location.

If Access to All FS Locations is selected and a FS Location Identity is selected, the selected user has access to all the File Storage Locations of the Identity. For example, if the Identity is selected as Clinic and Access to All FS Locations is selected, the defined user has access to all the File Storage Locations with Identity as ‘Clinic’ of the logged in facility.

  1. FS Location – The FS Locations displayed in the list of values depends on the FS Location Identity selected. The FS Locations listed are defined in the following table. This field is mandatory.

Access Rights

  1. Assign FS Location – Select the check box if the selected user is to be given rights to execute the function Assign File Storage Location.

  2. Change FS Location – Select the check box if the selected user is to be given rights to execute the function Change File Storage Location.

  3. Change File Return Date – Select the check box if the selected user is to be given rights to execute the function Change File Return Date.

  4. Monitor Deficiency – Select the check box if the selected user is to be given rights to execute the function Monitor File Deficiency.

  5. Change File Status – Select the check box if the selected user is to be given rights to execute the function Change File Status.

  6. Track out files to OPD – Select the check box if the selected user is to be given rights to execute the function Track out to OPD.

  7. Receive File – Select the check box if the selected user is to be given rights to execute the function Receive File at MRD.

  8. Confirm Receipt - Select the check box if the selected user is to be given rights to execute the function Confirm File Receipt.  

  9. Return File - Select the check box if the selected user is to be given rights to execute the function Return to MRD.

  10. Transfer File Outside MRD - Select the check box if the selected user is to be given rights to execute the function Transfer File.

 

Request File:

  1. With In Facility - Select the check box if the selected user is to be given rights to execute the function Request File.

  2. From Other Facilities - Select the check box if the selected user is to be given rights to execute the function Request File. By selecting it, the user can request files from other enterprise facilities for the selected permanent FS Location.

  3. Manual Request – Select the check box if the selected user is to be given rights to request for files manually. By selecting it, the user can request for files manually in the Issue File function. In Issue File function, the system evaluates the rights of the user for manual request.

  4. Cancel Request - Select the check box if the selected user is to be given rights to execute the function Cancel Request.

 

Issue File:

  1. With In Facility - Select the check box if the selected user is to be given rights to execute the function Issue File.

  2. To Other Facilities – Select the check box the selected user is to be given rights to execute the function Issue File. By selecting it, the user can issue files belonging to the selected permanent FS Location to other facilities of the enterprise.

  3. Create File Volume - Select the check box if the selected user is to be given rights to execute the function Create File Volume.

If ‘Access to All FS Locations’ is selected and FS Location Identity is not selected, the selected user has access to all the File storage locations of the facility and rights to execute all the functions. Even though the user has access to all the FS Locations and functions, only those functions that are applicable to the FS Location Identity are available to the user while working with the FS Location.

 

User Access Rights for Permanent Department

For a permanent File Storage Location all the functions are accessible to the user except 'Transfer Files Outside MRD' function.

 

User Access Rights for Non-Permanent Department

For non-permanent department, the user can be assigned access rights for only five functions.

  1. Confirm Receipt

  2. Return File

 

Request File:

  1. Within Facility

  2. Manual Request

  3. Cancel Request

 

User Access Rights for Temporary Locations

For a temporary file storage locations (those locations whose Identity is Clinic, Nursing Unit, Practitioner, Day Care Unit or Procedure Unit) all the functions except 'Transfer Files Outside MRD' function can be assigned to the user.

 

User Access Rights for All FS Locations

For all FS Locations, all the functions can be assigned to the user.
