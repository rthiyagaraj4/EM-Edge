Reason Code

From this function you can Create reasons to be used for entering remarks in Assign Relationship screen. Also, Query for the existing reason codes.

This function allows to create reason codes relevant to be used for remarks in ‘Assign Relationship’ screen when users are accessing patients as non-encounter patients. Also, allows to query and modify the existing reason codes.

User can enable and disable the reason codes as required.

Defining Reason Codes

From the CA Admin Setup > CA Relationship Setup, select the Reason Code function.

The Reason Code screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the details in the mandatory (*) fields. <Related Field Description>

  3. Click [Apply] to save the reason code created or [Reset] to clear the entered data.

  4. Click [Query] to search for existing reason codes.
