Enter the following details to enter Note Parameter:

 

  1. Backdate – Specify the number of days to be backdated in numerical format. This field is mandatory.

  2. XSL Folder - Specify the Location of the stored document by entering the path.

  3. File Size Limit - Enter the Size Limit of the file in numeric format and select the size from the list of values as KB or MB.

  4. Print Discharge Summary Header – Select the check box to enable printing the header of the discharge summary.

  5. Max no of Images allowed per note - Enter the number of images that can be linked to the clinical note in numeric format of two digits maximum length. You can associate multiple images when recording clinical notes based on the limit specified.

  6. Logo Header for All Pages  – Select the checkbox, if the  logo header has to be printed in each page. By default, this field is cleared. This functionality can be configured only if Print Logo Header and Report Header are selected in Note Type.

  7. Report Header for All Pages  – Select the checkbox, if the name of the report has to be printed in each page. By default, this field is cleared. This functionality can be configured only if Print Logo Header and Report Header are selected in Note Type.

  8. Logo Header Size Dimensions (Height and Weight) – Enter the required height and width dimensions of the logo. Only numeric digits are allowed, alpha numeric values are not allowed. By default, the value will be 40 X 40 pixels. The logo dimensions mentioned in the Note Parameter get reflected in the recorded Clinical Note.

• Min value for Height and Width dimensions size – 40 pixels

• Max value for Height and Width dimensions size – 96 pixels

  1. Spell Checker required for Facility – Select the check box to enable spell check function in Clinical Notes for the logged in facility.

  2. Default Spell Checker – This gets enabled only when the Spell Checker required for Facility check box is selected.

    • Spell Checker Pop up window – Select Spell Checker Pop up window option, a pop up window with Ignore / Ignore All / Change / Change All / Add / Undo / Finish buttons are available to correct the spelling.

    • Inline Spell Checker – Select Inline Spell Checker option to display the suggestions for the word wrongly entered. You can use Edit/ Ignore All / Add options to correct the spelling.

  3. Select Text Format by Default for Linked Sections - Select the check box if you want to make the ‘Text’ format as default option while linking Result doc in Clinical Note. By default, this check box is not selected. If you do not select the check box, the ‘Table’ Format appears as default option while linking Result doc in Clinical Note.

  4. Amend Clinical Result Note - Select the check box to allow the referring practitioner to amend the clinical result note in All Notes screen. All review notes reflect in Existing Orders and Order Tracking page with updates done by the referring physician, where the assigned practitioner can select and view the modified clinical result notes.

  5. Disable Reference Range in Link Results – Select the check box to not display the reference range details in the lab results in the Clinical Notes and Link Result Documents dialog invoked from Record Clinical Notes screen. By default, the check box is not selected.

  6. Disable Specimen Type in Link Results – Select the check box to not display the specimen type in the lab results in Clinical Notes and the Link Result Documents dialog invoked from Record Clinical Notes screen. By default, the check box is not selected.

  7. Disable Repeated Date /Time in Link Results - Select the check box to not repeat the Date/Time in the lab results in Clinical Notes and the Link Result Documents dialog invoked from Record Clinical Notes screen. By default, the check box is not selected.

  8. No of Encounters to Copy Previous Notes – Enter the number of encounters to be considered while copying previous notes in Record Clinical Notes function. You can enter only numeric value from 1 to 9. The default value is ‘1’. The value ‘zero’ is not allowed.  For example, if you enter the value as 1, the system displays the notes created against the current encounter in ‘Recording Previous Notes Search’ dialog in Copy Previous Notes function. If you enter the value as 2, the system displays the last created Encounter + current Encounter Notes.

  9. Access Rights applicable to in error – Select the check box to provide access rights to mark the clinical notes as ‘In Error’. The ‘In Error’ button appears in the Clinical Note Details dialog so that you mark the clinical note as in error. By default, the check box is not selected.

  10. Access Rights applicable to unsigned notes – Select the check box to provide access rights to a user to sign the unsigned note which was recorded by another user. By default, the check box is not selected. If you select the check box, the Sign button appears in the Clinical Note Details dialog.

  11. Allow Autosave in Notes – Select the check box to automatically save the notes in Record Clinical Notes screen. By default, the check box is not selected. When the check box is selected, then the documents are automatically draft saved to new column at regularly scheduled times determined by the user. If Notes crashes, or if there is power loss, the user can recover the work that was done prior to the crash or power loss. ‘AutoSave’, ‘Draft Save’ and ‘Delete Draft’ buttons are visible in Clinical Notes only if ‘Allow Autosave in Notes’ check box is selected in Note Parameter screen.

  12. Allow Autosave for every (mts) – This text field gets displayed only if the ‘Allow Autosave in Notes’ check box is selected. This field is mandatory. Enter the minutes i.e., time frequency to autosave while recording in Clinical Notes. You can enter from 1 to 9 minutes, the value zero is not allowed.

  13. To Display Note type title at the time of Notes View & Notes Print – Select the check box to display note type title in the Notes View and Notes Print. By default, the check box is not selected.
