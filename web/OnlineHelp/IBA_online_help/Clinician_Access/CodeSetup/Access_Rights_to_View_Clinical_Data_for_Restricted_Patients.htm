Access Rights to View Clinical Data for Restricted Patients

From this function you can Provide access rights to practitioners to view clinical data of restricted patients.  

This function allows to give access rights for practitioners to view clinical data for restricted patients in PAS and CA.

Authorizing Practitioners to view clinical data of restricted patients

From the CA Code Setup > CA Others Setup, select the ‘Access Rights to View Clinical Data for Restricted Patients’ function

The ‘Access Rights to View Clinical Data for Restricted Patients’ gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Patient Status – Displays the value as ‘Restricted’. This field is disabled and non-editable.

  3. Search By – User can select the search by value as Practitioner ID or Practitioner Name from the drop-down list if required. The default value is ‘Select’.

  4. If user selects either ‘Practitioner ID’ or ‘Practitioner Name’, then the respective text box becomes enabled.  User can enter either Practitioner ID or Practitioner Name in alphanumeric format.

  5. Click the [Search] button. The Practitioner details appear based on selected input value.

  6. If ‘Search By’ value is not selected, then system displays all active practitioners (including ALL practitioner Types) defined in AM Practitioner Setup.

  7. If user selects the ‘search by’ value and enter the required practitioner ID or Name in the text box, then system displays the selected practitioner name if it is active. Else, the message ‘Query caused no records to be retrieved’ gets displayed at the bottom of the screen.

  8. In the search results, a check box is provided against each practitioner. User can select the check box individually or click the ‘Select All’ check box to select all the practitioners at a time.

  9. Select the Date From and To range against each practitioner to set the expiry privilege. Only when the check box against practitioner is selected, the date fields get enabled.

i. Date From Cannot be less than System Date’

ii. Date To Cannot be less than System Date

iii. Date to should be greater than or Equal to Date From

  1. Against each practitioner, hyperlink ‘H’ is displayed next to the ‘Date From - Date To’ field. Click the H hyperlink to invoke the ‘View - Access History to view clinical data for restricted patients’ dialog. The history details include Practitioner name, Start Date, End Date, Record Created By, Record Date, Record Status.

  2. After entering the date range, click [Apply] to give access rights to the selected practitioners to view the clinical data of the restricted patients during the selected date range.

 

Validations:

  1. If the user enters a valid date in ‘Date From’ field and clicks on ‘Apply’ button without defining ‘Date To’, then message ‘Operation completed successfully’ gets displayed. System will consider that there is no expiry date for the selected practitioner.

  2. If the user enters a valid date in ‘Date To’ field and clicks on ‘Apply’ button without defining ‘Date From’, then message ‘Operation completed successfully’ gets displayed. System will consider the current system date as the default from date.

  3. If the privilege expiry dates (From-To dates) are defined as future dates for a practitioner, then that practitioner is considered as unauthorized to access restricted patients related clinical data until reaching the defined privilege dates.

  4. If no expiry date privileges had been defined in ‘Date From - Date To’ fields against the practitioner, then on clicking "H" hyperlink, 'View - Access History to view clinical data for restricted patients' dialog gets displayed without any data.

  5. When only 'Date From' value was defined in previous transaction, now to enable 'Date To' field, practitioner must clear and check the respective Practitioner check box which will delete the existing date in 'Date From' field.

  6. On opening the setup screen, if there are already selected practitioners and the user clicks on ‘Apply’ button without updating or adding any new inputs, system will invoke the confirmation message ‘Operation Completed Successfully’. In History screen, ‘Modified status’ will be updated for all Practitioner whose check box is selected.
