Referral Patient List

From this function you can View patient list based referrals assigned to patients during an active encounter.

The function lists the entire referral patients list on selection of this function from the menu. Practitioners can view all the patients who are directly referred to the logged in practitioner and patients who are not referred, if required. The function provides various query criteria which is used to filter the patient listing.

Viewing Referral Patient List

From the CA Transactions > Clinician Access > Desktop Menu > Patient List, select Referral Patient List.

The Referral Patient List screen gets displayed.

  1. Enter the search criteria to search for specific patients and click [Search] to display patients based on the entered criteria.

    1. Patient Class - Select the Patient Class from the list of valid values. 'All' is displayed, by default. The listed values are Inpatient, Outpatient, DayCare and Emergency.

    2. Location - Enter the Location in the alphanumeric format or select using the [?] function. This is based on the Patient Class selected.

    3. Patient ID - Enter the Patient ID in the alphanumeric format of maximum length as defined in MP Parameter or select using the [?] function to search the patient ID.

    4. Period - Enter the start and end Period in dd/mm/yyyy format. This is to filter patient listing with encounters within the specified period range.

    5. Include - Select a value from the list. The listed values are Self, Specialty Specific and Both. By default, Both gets displayed. When Self is selected, the list of all the patients referred to the logged-in practitioner gets displayed. When Specialty Specific is selected, patients referred to the specialties to which logged-in practitioner is associated gets displayed. When Both is selected, all the patients referred to the logged-in practitioner and to the specialties associated with the logged-in practitioner gets displayed.  

    6. Include Pending Referrals - Select the check box to list of patients with pending referrals.

    7. Status – Select the required status from the Status drop-down list. The list values include All, Open and Closed. By default, the value ‘Open’ is displayed.

    • When the Status value is selected as ‘Open’, the system displays referrals with the status: Unassigned, Pending Referrals and Seen and to be reviewed again.

    • When the Status value is selected as ‘Closed’, the system displays referrals with the status: Seen and Closed, Not Seen and Closed, and Closed

    • When the Status value is selected as ‘All’, the system displays referrals with all the status.

Or

  1. Click [Search] without entering the search criteria to display all referred patients based on the default query criteria.

  2. Click Patient Name hyperlink to display the Patient Chart. First assign the Patient to the Practitioner, if unassigned to proceed.

  3. Click ‘+’ hyperlink to display encounter details. The details displayed are Facility, Admission Date/Time, Specialty, Admission Type, Encounter ID, Discharge Date/Time, Practitioner and No. of Days. The details recorded in the ‘Register Inpatient Referral’ function of the Inpatient and Day Care Management module are displayed.

  4. Click the Referral ID hyperlink to display the Referral Details.

    The referral details of the patient get displayed. On the top part of the screen the patient demographics is displayed. In the other part of the screen the details of Referred From and Referred To gets displayed under two tabs.

    Referred From:

    The Referred From details of the patient is displayed. Click [Clinical Details] to view the Exam and Clinical findings, Referral Diagnosis and Treatment recorded during the referral registration.

    Click [Notes] to view the Referral Notes recorded during the referral registration.

    Referred To:

    The Referred To details of the patient is displayed.

  5. Click Confirm hyperlink to view and Confirm Bedside Referral details.  

    1. The Encounter Details, Nursing Unit Details and Referral Details are displayed.

    2. Enter the Bedside Referral Seen Date in dd/mm/yyyy format or select the Bedside Referral Seen Date using Calendar function. The Bedside Referral Seen Date cannot be greater than the current system date. This is a mandatory field.

    3. Select the Status from the drop-down list. The listed values are ‘Seen and To Be Reviewed Again’ and ‘Seen and Closed’. Select ‘Seen and To Be Reviewed Again’ status if you want to view and access the referral request multiple times; select ‘Seen and Closed’ status to view and complete the request in a single view. The updated status will appear in the Patient Referral List screen. This is a mandatory field.

    4. Click Audit Trail hyperlink to view the modification history of the patient referral requests.

    5. Click [Record] to save the record or click [Close] to exit.

    The ‘Status’ field and ‘Audit Trail’ hyperlink appear only if the ‘Allow Multiple Bedside Referral Review’ check box is selected in the IP Parameter screen.
