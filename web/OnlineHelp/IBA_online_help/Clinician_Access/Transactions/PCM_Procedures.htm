Procedures

From this function you can Record the Procedure details performed for a patient during an encounter in the facility.

This function provides a comprehensive search option, which enables the practitioners to record the procedures with ease. There is also a provision to mark the recorded procedure as error, if there was any mistake during the entry and re-record the procedure. An audit trail is maintained for all the procedures recorded and the procedures marked in error, which may be useful for administrative purposes. Currently, the product supports the coding scheme ICD10, ICD9-CM or CPT4 for recording the procedures. Appropriate parameter is set up in the Medical Records module to identify the procedure coding scheme being followed in the unit. When this function is invoked, the system identifies the coding scheme being followed from the MR Parameter and works accordingly.

Recording Procedures

From the CA Transactions > Clinician Access > Desktop Menu > select required patient > Patient Chart Menu > Outcomes, select Procedures.

The Record Procedures screen gets displayed.

  1. The screen displays Procedure Details already recorded for the patient.

    On invoking the function, the procedure history of the patient is displayed, which includes Procedure Description, Procedure Code, Coding Scheme, Date of the procedure and Practitioner who performed the procedure. This section is empty, if the patient has no procedures recorded in the facility or enterprise.

  2. Enter the details in the mandatory (*) fields under New Procedure. <Related Field Description>

  3. Click [Canned Text] to copy predefined texts to the required field.

    The list of text blocks listed when [Canned Text] is clicked depends on the text blocks created in the ‘Text Block’ master function in Clinician Access module with the flag ‘Applicable for Procedure’ selected. Select the check box against the required text block and click [Select] to copy the text to the required field. Click [Cancel] to exit from the screen.

  4. Click [Record] to record the procedure, [Clear] to clear the entered data or [Cancel] to exit from the Record Procedures screen.

  5. On recording the procedure, the details are displayed on the top half of the screen under Procedure History.

  6. Click Procedure Description hyperlink to modify the recorded details.

    On selecting the Procedure Description hyperlink of the required procedure, the details of the procedure recorded are displayed in the respective fields under the New Procedure section.

    Mark Error

    A procedure recorded is marked as an erroneous record, if it was recorded by mistake. To mark a procedure record as ‘In Error’ choose the procedure from the ‘Procedure History’ section. The [Mark Error] is displayed and on clicking [Mark Error], a small window pops-up asking for the reason for the record to be marked as ‘In Error’. Enter the error remarks in alphanumeric format of 60 characters maximum length.

    View Error List

    Once a procedure is marked as error, [View Error List] is displayed on selecting the patient.

    On clicking [View Error List], all the procedures that have been marked as error and their reasons are displayed along with the other details of the procedure. The Other details displayed are Procedure Code, Procedure Description, Scheme, Error Marked By, Error Marked Date and Remarks. The Remarks field is hyperlinked, which is clicked to view the details of the procedure recorded as error.

  7. Click [Record] to save the updated procedure.
