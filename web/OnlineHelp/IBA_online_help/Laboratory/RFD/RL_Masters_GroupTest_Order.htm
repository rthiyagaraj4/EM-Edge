Order Setup

Order Setup tab enables you to associate the Group Test to an order registered through the ‘Order Entry and Result Reporting’ module.

 

Enter the following details:

  1. Test-Catalog Cross Reference - Enter a valid Order Catalog Code or click [List of Values] function to view a list of order entries created through 'Request and Results' module and select the required order registered for the Lab.

A warning message 'Not a Valid Catalog, Do you want to Create a New One?' appears on creating a new Order Catalog. Click [Yes] to create a new Order Catalog or click [No] to cancel. A warning message 'Test will be linked to the Order Catalog. Do you want to Proceed?' appears on selecting the Order Catalog by clicking [List of Values]. Click [Yes] to link the Order Catalog to the group test or click [No] to exit.

  1. Long/Short Description - Enter the Long Description of Test-Catalog Cross Reference code in alphanumeric format of 60 characters maximum length and Short Description in alphanumeric format of 40 characters maximum length.

Details in the Characteristics and Instruction tabs populates data from 'Requests and Result' modules if you are selecting the Order Catalog from the [List of Values] function else enter the details in the respective field. Refer 'Order Catalog' function in 'Requests and Results' module for more assistance.

 

Characteristics Tab

Synonyms Tab

Synonyms are the different names given to an Order Catalog to be used by various departments. These are basically secondary names categorized as Synonym Type. Primary synonym is the Short Description.

 

Enter the following details:

  1. Type - The types displayed are Direct Care and Ancillary Dept. Ancillary Department refers to Radiology and Pathology, where as Direct Care refers to Practitioner type like Physician, Dentist, Surgeon, etc. The name specified for Direct Care is available for users of Direct Care Department, while Ancillary Department users get the name defined for Ancillary Department while placing an order.

  2. Name - Enter the name for the type in alphanumeric 40 characters maximum length.

  3. Active – Select the check box and the Name is applicable for the selected department.

 

Timing

Some of the values under ‘Timing’ sub-tab are inherited from the selected Order Type. Indexing of an Order Catalog and consent requirement specifications are defined.

  1. Change Qty/Dose - Select the check box, if Quantity/Dose can be changed. Clear the check box and you cannot change the Quantity/Dose. The check box is cleared, by default. On selecting the check box the UOM field gets displayed.

  2. Frequency Required - If the check box is selected, the frequency is mandatory and has to be entered in the 'Settings' tab. If the check box is cleared, frequency value is not mandatory. The field is not applicable for Order Category, Department Diagnostics and Miscellaneous. The check box is enabled, if frequency applicable check box is selected for the selected Order Type.  

  3. Soft Stop for Inpatients - The check box is enabled depending on the Order Category and Order Type selected. If Soft Stop for Inpatient is selected, the duration value is optional. This is applicable only for inpatients. The Soft Stop is applicable only for Order Categories - Dietary, Monitoring, Patient Care and Trmt/Procedure. The Soft Stop for Inpatient check box has to be selected in the Order Type function.

  4. Create as Continuing Order - The value of the field depends on the Order Category and Order Type selected. The values for selected Order Type is inherited. Modification of the value depends on the selected Order Category. If continuous order is applicable for the selected Order Type, a single order is placed for the Order Catalog and it repeats every day. The value for ‘Continuous Frequency Action’ is set to ‘Not Applicable’, by default, if the check box ‘Create as Continuing Order’ is cleared.  If this field is applicable and frequency defined is other than 'Continuous' frequency type, the recurring orders will not be generated on ordering. The field is applicable, if respective fields are selected in order type master function for order category Dietary, Patient Care, Trmt/Procedures and Pharmacy order.

  5. Index – Select the check box to index the defined Order Catalog. If the Order Catalog is indexed, it has to be associated to a service, that is, the defined Order Catalog gets listed in the function 'Index Order Catalog, where it is associated to a Service'. If the Order Catalog is not indexed, it is not available across the services of the facility.

  6. Consent Required – Select the check to associate consent for the created Test.

  7. Consent Note Text – Enter the consent note text in the free text.

 

Duplicate Check

The criteria for checking Duplicate Orders are defined in the tab.

  1. Check for Duplicates - Select the check box, if duplicate check is required for the defined catalog. If duplicate check is not required, clear the check box. On selecting the check box, other fields are enabled.

  2. Duplicate Status - Select the value from the list. The values displayed are  'Active Orderable Status' and 'Any Orderable Status'. This field is enabled only when duplicate check is applicable (that is Required check box is selected). For status as 'Active Orderable Status', the duplicate check is performed on all the active orders of defined catalog (an order is active till it is registered). If 'Any Orderable Status' is selected, duplicate check is performed for any order status of the catalog (that is from pending consent till fully resulted except for cancelled, hold or discontinued orders).

  3. Action Type - Select the action type from the list. The listed values are 'By Orderable' and 'By Activity Type'.

 

Type

  1. Behind Action - Select the required value from the list. The listed values are Reject, Warn and No Check. This field is enabled only when 'Required' is selected. If previous orders exist for the Order Catalog and the order is placed for the same Order Catalog,  the action is performed, based on the Action Type selected. Select 'Reject' and duplicate orders are rejected. Select 'Warn' and a warning message is displayed while placing an order, indicating that the order is a duplicate. You can either proceed to place the order or cancel the duplicate order. On selecting ‘No Check’, duplicate check is not performed for the duplicate order placed.

  2. Period - Enter the period in numeric format of three characters maximum length. Duplicate check is performed for previous orders within the specified period.

  3. Unit - Select the unit from the list. The values displayed are Months, Weeks, Hours, Days and Minutes.

  4. Ahead Action - Select the required value from the list. The values displayed are Reject, Warn and No Check. This field is enabled only when 'Required' is selected. If future order exists for the Order Catalog and it is placed again, action is performed based on the type selected. If 'Reject' is selected, the ordering process is rejected. If 'Warn' is selected, you are warned. Duplicate check is not performed for the order placed again for value as 'No Check'.

  5. Period - Enter the period in numeric format of three characters maximum length. Duplicate check is performed for future orders within the specified period.

  6. Unit - Select the Unit from the list. The values displayed are Months, Weeks, Hours, Days and Minutes. If the status considered is ‘Active Orderable Status’ and ‘Behind action’ is ‘Reject’ with period as ‘two weeks’, duplicate check for orders placed for the defined Order Catalog since last two weeks from the current system date is performed. If any order is placed within these two weeks, new order for the defined Order Catalog cannot be placed, as it is rejected. If behind action is ‘Warn’, you are warned that order is placed for the Order Catalog within the specified period. If behind action is ‘No Check’, you can place order without any hindrance, the same explanation holds for ‘Ahead Action’, but duplicate check is performed for future orders placed.

 

Charting

The reporting characteristics of the Order Catalog are defined under ‘Charting’. The screen gets displayed on clicking the tab.

  1. Department Only - Select the check box, if the Order Catalog is applicable to department. On selecting the check box, the Order Catalog is available only to the practitioners in the Department. If the Department only is not selected then catalog is displayed in new orders.

  2. Complete On Order - Select the check box, if order is to be completed on placing. If selected, the result reporting cannot be performed.

  3. Billable - Select the check box, if this Order catalog is to be billed. The check box appears only if the Patient Billing Manager module is enabled.

  4. Charge Type Applicable - Select Charge type as 'Single' or 'Multi'. The drop down list appears only if 'Billable' is selected. If charge type is selected as 'Multi', different charge types are applicable for the Order Catalog.

  5. Chart Applicable - Select the check box, if resulting is required for the defined Order Catalog. This field is disabled, if 'Complete on Order' is selected. If chart is applicable, the Chart Result Type and Discrete measure are mandatory. If Chart Applicable is cleared, the fields under Charting Characteristics and Procedure Link are disabled.  The characteristics are inherited from the Order Category associated to the Order Catalog.

 

Charting Characteristics

The basic resulting characteristics are defined for the Order Catalog.

  1. Authorization Required - Select the check box, if authorization is required for resulting. This field is disabled, if 'Chart Applicable' is cleared. If the check box is selected, on result reporting for the order, the system will post the result for Result Authorization.

  2. Nature - Select the Nature of the Order Catalog as Individual (Single order catalog) or Panel (discrete measures defined). Panel is displayed, by default.

  3. Chart Result Type - Select the Chart Result Type from the list. The values displayed are Discrete Measure, Result Matrix, Note Type and Charting. If 'Nature' is selected as 'Panel', 'Patient Care Panel' defined in the Application Masters are listed.

  4. Module ID - The name of the module is displayed and disabled, if you have linked the select order catalog to the test code in Laboratory, Radiology or Operation Theatre modules.

 

Procedure Link

  1. Applicable - Select a value from the list. The values are system defined - 'Not applicable', 'Auto-Link' and 'Manual-Link'. If Auto-Link is selected, select the Term Set ID and enter Term Code. If ''Manual Link' is selected, select 'At Resulting' as either ‘Optional’ or ‘Mandatory’. The Order Catalog selected as Manual-Link gets listed in the 'Order Catalog to Procedure link' function. The field is applicable if 'Nature' is selected as 'Individual'. If both Complete on Order and Chart applicable are selected, Auto Report is generated for the Order Catalog. This is not applicable for of the  Order Categories - Laboratory, Miscellaneous, Monitoring and Trmt/Procedure.

 

Instrument

The various instructions used while ordering are defined in the tab.

  1. Prompt Alert - Select the check box to enter the message in alphanumeric format. The alert message is displayed on selecting the Order Catalog while placing orders.

  2. Patient Preps - Enter the preparation instructions in alphanumeric format of 500 characters maximum length. You can view/modify the preparation instructions while placing the order.

  3. Department Instruction - Enter the department instruction in alphanumeric format of 500 characters maximum length. You can view/modify the department instructions while placing the order.

  4. Patient Instruction – Enter the instructions in alphanumeric format. These are the instructions given to patients about the orders.  You have to select Standard or By Age/Sex. On selecting the required type of instruction, click [Instructions] and enter the standard instructions. On selecting Age/Sex, a screen pops-up where you can enter the minimum and the maximum age. The instruction is applicable for the selected Gender in between the age range specified. Click Instrn hyperlink to record the instruction. Click [Select] to record the details and [OK] to confirm the instructions. Click [Clear] to reset the data entered. Click [Cancel] to exit from the [Instructions] screen. The instructions recorded cannot be edited while ordering.
