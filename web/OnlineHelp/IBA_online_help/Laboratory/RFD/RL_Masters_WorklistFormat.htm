Enter the following details:

  1. Worklist Name - Enter Worklist Name in alphanumeric format of 10 characters maximum length. A worklist is specific to a department. A worklist is created, printed, resulted based on worklist name. This is a mandatory field.

  2. Department - The logged in Department Code is displayed, by default. Select required department from the list, if you have access to more than one department.

  3. Number Last Used - Enter the Last Number assigned to the generated worklist in numeric format of five digits maximum length. This is a mandatory field.

  4. Default Specimen No - Enter Default Specimen number in numeric format of 10 digits maximum length. You can set the Default Number for worklist to make use of it in specifying the starting range of specimen while creating worklist. The default specimen number has to match with the ‘Default Specimen Number’ recorded for the respective department in ‘Departmental Parameter for Facility’ function.

  5. Last Specimen No - The last specimen number processed in a worklist is displayed after resulting the specimens in the Result Entry function. You can refer the last specimen number to specify the start range of specimen while creating the worklist.

  6. Field Type - Select the required Field Type from the list. Valid values are 'Assign To', 'Non Assign To' and 'Both'. The selection of specimen is based on the field type chosen.

  7. Spacing Between Cups - Enter the required value in numeric format of two digits maximum length to specify the spacing between cups while printing work list.

  8. Number of Cups - Enter the Number of Cups in numeric format of three digits maximum length. Number of cups indicates the maximum number of specimens to be included in each worklist. The value specified is used in worklist creation and the number of specimens selected during the process is limited to this number. The maximum limit is 250. This is a mandatory field.

 

Work List Tests

You can enter the tests that need to be included in the work list format. The criteria for including the specimen on the work list based on whether a test is present or not on the specimen is defined here. If a test is defined as mandatory and the specimen does not have the specified test though it may have the other tests of the work list, the specimen will not be included in the work list. If tests have been defined as optional and the specimen has any one or more tests associated to it, then the specimen is included in the work list. The maximum number of investigations or tests that can be defined in a single work list format is 25.

  1. Test - Click [List of Values] to select a valid test code. The list displays values defined in ‘Test Codes’ function. The same test code is defined in different work list formats, but duplication is not allowed in the same work list format. Long name of the test code is displayed for user verification on entry of a valid test code. This is a mandatory field.

  2. Description - The corresponding Description of the selected test code is displayed, by default.

  3. Indicator – Select a value from the list. The system-defined values are Pseudo code, Mandatory Test and optional test. ‘Pseudo code’ indicates that the corresponding column does not represent a test and is an extra column. The heading that has to appear is entered in the test column. For example, you can enter any remarks. ‘Mandatory Test’ indicates that a specimen is included in the work list only if the test codes specified as mandatory are ordered for the specimen. ‘Optional Test’ indicates that a specimen is included in the work list if it has any one or more of the test codes specified as optional.

 

Past Results

  1. Past Results Period - Past results, which need to be printed for a period, is specified for the work list test.

  2. Past Results Units - Unit can be in hours, days, weeks, months and Years.

  3. Past Results Max No - Maximum number of times the result needs to be printed can be specified.
