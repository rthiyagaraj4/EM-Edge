Maintain Report Printing Requests

From this function you can Maintain the report printing requests and Query for the existing records.

Maintain Report Printing Requests function enables you to view or print the reports. You can also purge the report printing requests. It can also used to view the generated report. The status of Report Printing requests are highlighted using different colors:

- Indicates requests which are under process

- Indicates requests which are being printed

- Indicates requests which are not processed due to error

Maintaining Report Printing Requests

From Administration > Print Services, select the Maintain Report Printing Requests function.

The Maintain Report Printing Requests screen gets displayed.

  1. The Maintain Report Printing Requests screen displays the reports requested for printing. The color code associated with the request gives the status of the request (Under process, printed or Error). <Related Field Description>

  2. Click [View Report] to view the output file in the defined viewer. Viewer is defined in iSOFT.INI file.

  3. Click [Print Report] to delete the corresponding error messages (if any) and make the request as active request for reprinting. The active printing request is highlighted in   color.

  4. Click [View Error Log] to view the corresponding printing error message information.

  5. Click [Purge] to delete the request after the confirmation message is acknowledged. It removes the report file name from the Operating System. The records, which are being processed, are not deleted.

  6. Click [Purge All] to delete the processed requests from the queried requests after the confirmation message is acknowledged. This removes the report file name from the Operating System.  The records, which are being processed, are not deleted.
