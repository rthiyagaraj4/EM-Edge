Admit Patient

From this function you can Admit patients as Inpatients or Daycare patients to Nursing Units or Daycare Units under the care of an admitting practitioner by assigning a bed for Inpatient/Daycare stay.

Admit Patient function facilitates planned admission for patients with booking, referred patients and patients who walk in directly to the facility. All the patients have to be registered to the facility before admission. During admission, a Nursing Unit/Daycare Unit and Bed Class is assigned to the patient. Based on the assigned Bed Class, deposit is collected (if ‘Billing’ module is installed). The Specialty and admitting Practitioner are specified. Patients’ not allocated beds during admission are assigned beds explicitly using ‘Assign Bed/Mark Patient Arrival’ function. Admission is backdated based on the admission rules set in IP Parameter. On admission, a unique Encounter ID is automatically generated by the system to identify the Inpatient Encounter. Privilege to invoke this function is defined in the Operator Station attached to user in the Application Masters module. A patient file is created with appropriate file storage location. A relevant patient file label is also printed for the respective inpatient file. An Inpatient Wristband label report for each patient is created with the Name, Sex, UHID, Visit and Bar Code.

Admitting a Patient

From the IP Transactions > Admission, select the Admit Patient function.

The Admit Patient screen gets displayed.

  1. Select either ‘Daycare’ or ‘Inpatient’ from the ‘Admission For’ list indicating whether the admission is for a daycare patient or an inpatient. This field is mandatory.

If you had arrived at this screen through a quick admission referral, the ‘Admission For’ field displays “In patient” and cannot be edited.

Admission can take place for patients with Booking Reference Number, Referral Patients or Walk-in patients based on the IP Parameter setup.

Based on the value set for the field ‘Allowed Admission’ in IP Parameters, certain fields are enabled and disabled on selecting the field ‘Admission For’.

  • If ‘Allowed Admission’ is set as ‘Referral’ in IP Parameters, on selecting the ‘Admission For’ field, only the ‘Referral ID’ field is enabled to select referral patients for admission.

  • If ‘Allowed Admission’ is set as ‘Booking’ in IP Parameters, on selecting the ‘Admission For’ field, only the ‘Booking Ref No’ field is enabled to select patients with prior booking for admission.

  • If ‘Allowed Admission’ is set as ‘Booking or Referral’ in IP Parameters, on selecting the ‘Admission For’ field, the ‘Booking Ref No’ and ‘Referral ID’ fields are enabled to select referral patients or patients with prior booking for admission.

  • If ‘Allowed Admission’ is set as ‘Direct’ in IP Parameters, on selecting the ‘Admission For’ field, the ‘Booking Ref No’, ‘Referral Id’ and ‘Patient ID’ fields are enabled to select referral patients and patients with or without prior booking for admission.

 

Admission for Unregistered Patient with Booking Reference Number

  1. To admit a patient who has future bookings for an inpatient admission, system displays Booking Reference List screen prompting about the existing bookings of the patient. Select the Booking Reference Number hyperlink if the booking is for the current admission. The booking details are displayed on the Admit Patient screen. Click [Cancel] on the Booking Reference List screen if the future bookings are not for the current admission. Or

  2. Enter the Booking Reference Number in the ‘Booking Ref No’ field or select the Booking Reference Number using the [?] function. On clicking the [?] function, the Booking Reference Number Lookup screen is displayed.

  3. Specify the required query criteria and search for booking reference details. Select the required Booking Reference Number hyperlink.

  4. The system validates if the patient is registered in the facility and displays an ‘Enter Patient ID’ screen if not registered. If Patient ID is not available, register the patient before admitting the patient by clicking [Register Patient].

  5. On Patient Registration, patient details are displayed on the ‘Admit Patient’ screen.

  6. Proceed with the admission process.

Admission for Registered Patient with Booking Reference Number

  1. To admit a patient who has future bookings for an inpatient admission, system displays Booking Reference List screen prompting about the existing bookings of the patient. Select the Booking Reference Number hyperlink if the booking is for the current admission. The booking details are displayed on the Admit Patient screen. Click [Cancel] on the Booking Reference List screen if the future bookings are not for the current admission. Or

  2. Enter the Booking Reference Number in the ‘Booking Ref No’ field or select the Booking Reference Number using the [?] function. On clicking the [?] function, the Booking Reference Number Lookup screen is displayed.

  3. Specify the required query criteria and search for booking reference details. Select the required Booking Reference Number hyperlink.

  4. The Patient details and Nursing Unit details entered while booking are displayed in the ‘Admit Patient’ screen.

  5. Proceed with the admission process.

Admission for Registered Patient with Referral ID

  1. Enter the Referral ID during admission in the ‘Referral ID’ field or select the Referral ID using the [?] function. On clicking the [?] function, Referral Search screen is displayed. Enter the query criteria to search for a specific registered referral patient and click [Search]. The referral details of the patient are displayed. Click Referral ID hyperlink to display patient details in the Admit Patient screen.

  2. Proceed with the admission process.

Admission for Unregistered Patient with Referral ID

  1. Enter the Referral ID during admission in the ‘Referral ID’ field or select the Referral ID using the [?] function. On clicking the [?] function, Referral Search screen is displayed. Enter the query criteria to search for referral patients and click [Search]. The referral details of the patient are displayed. Click Referral ID hyperlink.

  2. The system validates if the patient is registered in the facility and displays an ‘Enter Patient ID’ screen if not registered. If Patient ID is not available, register the referral patient before admitting the patient by clicking [Register Patient].

  3. On Patient Registration, patient details and referral details are displayed on the ‘Admit Patient’ screen.

  4. Proceed with the admission process.

Admission for Registered Referral Patient without Referral ID

  1. Click the Referral ID [?] function to display the Referral Search screen.

  2. Click [Register Patient Referral] to select a registered patient from the Patient ID [?] function and generate a Referral ID for the patient.

  3. The Patient details and Referral details of the patient are displayed in the Admit Patient screen.

  4. Proceed with the admission process.

Admission for Unregistered Referral Patient without Referral ID

  1. Click the Referral ID [?] function to display the Referral Search screen.

  2. Click [Register Patient Referral] to generate a Referral ID for an unregistered patient.

  3. The system validates if the patient is registered in the facility and displays an ‘Enter Patient ID’ screen if not registered. If Patient ID is not available, register the referral patient before admitting the patient by clicking [Register Patient].

  4. On Patient Registration, patient details and referral details are displayed on the ‘Admit Patient’ screen.

  5. Proceed with the admission process.

Admission for Registered Patient with Booking Reference Number and Referral ID

  1. Enter the Booking Reference Number or Referral ID to display the Booking Reference Number and Referral ID of the registered patient in the respective fields.

  2. Proceed with the admission process.

Admission for Unregistered Patient with Booking Reference Number and Referral ID

  1. Enter the Booking Reference Number or Referral ID to display the Booking Reference Number and Referral ID of the patient in the respective fields.

  2. The system validates if the patient is registered in the facility and displays an ‘Enter Patient ID’ screen if not registered. If Patient ID is not available, register the referral patient before admitting the patient by clicking [Register Patient].

  3. On Patient Registration, patient details and referral details are displayed on the ‘Admit Patient’ screen.

  4. Proceed with the admission process.

Admission for Registered Walk-in Patient

  1. Enter the Patient ID in the ‘Patient ID’ field or select the Patient ID using the [?] function.

  2. Patient details are displayed on the ‘Admit Patient’ screen.

  3. Proceed with the admission process.  

Admission for Unregistered Walk-in Patient

  1. Click the Patient ID [?] function and the Search Patient screen displays to register the patient.

  2. Patient details are displayed on the ‘Admit Patient’ screen.

  3. Proceed with the admission process.

Admission of patients through Quick Admission Referral

If you have arrived at this Admit Patient screen because you have checked the Quick Admission check box in the Register Referral Details page, the following conditions apply:

1. The ‘Admission  For’ field displays “In patient” and cannot be edited.

2. You cannot edit the financial details. The details are displayed by default based on the billing group configured for Citizen and Non-Citizen in billing module.

When you click the Financial details tab or click Apply, the billing page will be launched for viewing and closed automatically.

  1. If patient has any outstanding amount, the system pops-up a screen for Credit Authorization which displays the outstanding amount. If the 'Display Amount' check boxes against the Outstanding, Unbilled, Installment Due and Written Off fields are selected and if the Outstanding, Unbilled, Installment Due and Written Off amounts are greater than the amount set in the BL Parameters function of Patient Billing Manager module, credit authorization details are displayed on the screen.

    1. Patient Remarks – The remarks recorded from the selected patient during bill generation for the previous encounter gets displayed.

    2. Outstanding Amount – The outstanding bill amount from the previous encounters get displayed.

    3. User ID – Enter the User ID. Users with authorization to update the financial details are only allowed to login and modify the billing details. This field is mandatory.

    4. PIN – Enter the PIN for the selected User. This field is mandatory.

    5. Remarks – Enter the remarks in alphanumeric format to record the remarks against the selected patient record.

    Click [Close] to login and update or [Cancel] to exit from the screen.

  2. Enter the Admission Details and Financial Details in the mandatory (*) fields.

Admission Details Tab

Financial Details Tab

  1. Click [Contact Details] to view or modify the contact details of the patient.

  2. Click [Previous Encounters] to view the previous admission details of the patient if the selected patient had previously received inpatient care from the facility.

  3. Click [Referral Details] to view referral details of the patient if the selected patient is referred from another location for admission to the current facility.

  4. Click [Apply] to save the admission or [Reset] to clear the entered data.

  5. On clicking [Apply], Admission Deposit screen gets displayed, if the Admission Deposit parameters are selected for the Nursing Unit in Patient Billing Manager module. <Related Field Description>

  6. On clicking [Apply], the Billing Class Eligibility For Package screen is displayed to select the billing class eligibility, if the Billing Class selected during package subscription and admission do not match.  

    Enter the following details:

    1. Patient's Request - Select the radio button if patient has requested for a separate Billing Class. A warning message 'Package subscribed with Different Billing Class' is displayed, if the Billing Classes do not match.

    2. Hospital's Request - Select the radio button if hospital has requested for a separate Billing Class.

    Click [Cancel] to cancel the transaction.

     

  7. On clicking [Apply], the Patient Charging Basis on Admission screen is displayed to select the charging basis for the Bed Type. The screen is displayed only if the check box ‘Allow Allocation of Higher Bed Type on Hospital Request’ is selected in the BL Parameters function of Patient Billing Manager module.

    Enter the following details:

    1. Patient's Request - Select ‘Patient's Request’ option, if the request for a higher Bed Type is made by the patient.

    2. Request Remarks – Enter the request remarks up to 250 characters maximum. The recorded remarks value will not be displayed anywhere but will be available in the database.

    3. Hospital's Request - Select ‘Hospital's Request’ option, if the request for a higher Bed Type is made by the hospital when the requested Bed Type is not available. An Authorization screen gets displayed on selecting Hospital's Request.

    Enter the User ID and the PIN of the logged in user. Click [OK] to login or [Cancel] to cancel the process. On clicking [OK], the Bed Type valid for the Hospital is listed. Select a Bed Type based on which the bed charge has to be computed.

  8. Create Patient File screen gets displayed if file management parameters are set for the facility.

  9. Enter the details in the mandatory (*) fields. <Related Field Description>

  10. Click [OK] to generate file number or [Cancel] to cancel file generation.

  11. If the future order exist for the selected patient, the Future Order screen is displayed alerting the user to link the previously placed order with future date.

  12. If internal reports are defined, the Internal Reports screen gets displayed. Click [OK] to generate Internal Reports or [Cancel] to exit from the screen.

  13. Encounter ID and File Number is generated and displayed.

 

If Patient Billing Manager module is installed and the discharge bill has not been generated for previous Inpatient encounters, the system displays an alert message and stops you from admitting such patients.

 

An error message is displayed if the patient is re-admitted within the time defined in the Alert Re-admission of the Patient Within ( ) Hours in IP Parameter screen.
