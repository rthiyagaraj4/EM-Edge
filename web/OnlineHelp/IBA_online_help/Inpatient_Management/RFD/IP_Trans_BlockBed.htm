Enter the following details to Block a Bed:

 

  1. Nursing Unit – Enter the Nursing Unit in the alphanumeric format of 15 characters maximum length or select the Nursing Unit using [?] function. This field is mandatory. All the Nursing Units associated to the operator station attached to the logged in user are listed in [?] function.

  2. Bed No – Enter the Bed Number in the alphanumeric format of eight characters maximum length or select the Bed number using [?] function which invokes the ‘Bed Availability Chart’ screen and the beds associated to the selected Nursing Unit are displayed. On selecting the Bed, the Bed Class and Room, number associated to the selected bed is displayed. This is a mandatory field.

  3. Bed Class – The Bed Class is displayed on selection of the bed number.

  4. Room/Cubicle No - The Room/Cubicle Number is displayed on selection of the bed number.

 

Blocking Detail

  1. Reason for Blocking - Select a valid Reason for Blocking the bed from the list. The ‘Blocking Types’ defined in the ‘IP Code Setup’ are displayed in the list. This is a mandatory field.

  2. Override – This field is disabled and cleared, by default. The field value changes to ‘selected’ based on the blocking type selected. If the selected blocking type has its override check box selected (while creating the ‘Blocking Type’ in IP Code Setup), the same is reflected in this field. The override indicator has the following significance: if the override indicator is selected, the blocked bed can be overridden by another booking/blocking/admission transaction. If it is cleared, the blocked bed cannot be overridden by any other transaction (including admission and transfer).

  3. Remarks - Enter the Remarks for blocking the bed in alphanumeric format of 60 characters maximum length. This field is optional if the bed blocking period does not exceed the default blocking period as specified either for the blocking type chosen or based on the values set against the parameter ‘Default period for normal’ in the IP Parameter. If the blocking period exceeds the default blocking period, this field becomes mandatory.

  4. From Date - Indicates the start date and time for blocking the bed. Enter the date and time in date format of dd/mm/yyyy hh:mm. The ‘From’ date must be greater than or equal to the current system date and time, or select the date and time using the Calendar function. This field is mandatory.

  5. Until – Indicates the date and time until when the bed is to be blocked. The system calculates the ‘Until’ date and time based on the default period specified for the blocking type chosen. If the default blocking time is not specified for the chosen blocking type, the system calculates the ‘until’ date based on the ‘Default period for normal’ maintained in the IP Parameter. The ‘Until’ date and time is displayed in the date format of dd/mm/yyyy hh:mm. But the displayed ‘Until’ date and time can be changed provided they do not exceed the ‘maximum allowable period for normal’ set in the IP Parameter setup (system calculates the ‘Until’ date by adding the Bed blocking rules defined for normal/max allowable period for normal with the ‘From’ date), or select the date from the Calendar function. This field is mandatory.

On viewing the bed availability chart, the blocked bed appears with the relevant color.
