Update Diagnosis Profile

  1. Diagnosis Set - Select the Diagnosis Set from the list. The list displays the Terminology Sets as defined in the Terminology Set function that is associated for a Practitioner, Facility and Specialty in Medical Record module. On selecting the Diagnosis Set, the screen is displayed.

  2. Diagnosis – Enter the Diagnosis Code or click [?] function to search and select the desired diagnosis. This field is mandatory. On selecting [?] function, the screen is displayed on clicking the Search Tab from the screen that displays.

On selecting the Search By criteria, enter Search Text. Click [Search] to invoke the screen.

Based on the nature of the diagnosis selected, the system displays a hyperlink for you to select from the associated supporting diagnoses. Based on the nature of the diagnosis selected (Dagger Code or Consequence) the system enables you to associate the Manifestation/External Cause.

The codes as defined for the terminology sets in the function Terminology Codes are displayed. Select Code hyperlink. Based on the nature of the diagnosis selected the system displays a hyperlink for you to select from the associated supporting diagnoses. Based on the nature of the diagnosis selected (Dagger Code or Consequence) the system enables you to associate the Manifestation/External Cause.  

[+] button is enabled if the chosen Diagnosis Code has an indicator set as either ‘D’ or ‘C’ which indicate ‘Dagger’ and ‘Consequence’ respectively. This setup is done in the ‘ICD Code’ master function in the Medical Records module. Select the supporting diagnosis to be associated with the main diagnosis and click [OK].

The selected code is displayed in the Diagnosis Field.

Click [!] corresponding to the diagnosis field to view the description of the diagnosis as set in MR Terminology Code setup. On viewing, click [Close].

  1. Encounter Stage – Encounter Stage field gets displayed if you are recording diagnosis for an inpatient. Select the desired encounter stage from the list. The valid values are ‘Admission’, ’Intermediate’ and ‘Discharge’.

  2. Nature - Select the Nature of diagnosis as Actual, Potential or Wellness. This field is mandatory.

  3. Accuracy – Select Accuracy from the list. The Accuracies are defined in the Accuracy function in the MR Terminology Set. On selecting the Accuracy, Differential Group hyperlink is displayed.

Select the Group 1 radio button or if the diagnosis is to be recorded under a new group, select New Group radio button. On selecting New Group radio button, Group 2 is displayed which can be selected. Click [Close].

  1. Severity – Select Moderate, Mild, Severe or Extreme the severity from the list. This field is mandatory.

  2. Priority – Select priority as Normal, High or Critical. This field is mandatory.

Select ‘As Evidenced By’ hyperlink.

The Defining Characteristics are displayed which can be selected. The characteristics are defined as displayed in ‘Factors for Terminology Code’ function. On selecting, click [Ok] or [Close] to exit from the screen.

  1. Onset Type – Select the Onset Type as Acute or Chronic from the list. This field is mandatory. Not Applicable is displayed, by default.

  2. Onset Date – This field denotes the date from which the Patient is having the problem. It is defaulted to the current system date, which can be changed to an earlier date. To change the date, choose the date form the Calendar function. This field is mandatory.

  3. Since – This field is updated automatically on entering the Onset Date.

  4. Status – Status is displayed as Active, by default. You cannot modify the status while recording the current diagnosis. Other available statuses are Resolved and In Error.

  5. As On – The Current Date is displayed for the status.

  6. Include in Current Encounter – The check box is selected and disabled which indicates that the diagnosis is included in the current encounter.

  7. Remarks – Enter any remarks if any.

On recording the diagnosis, the data entry part of the screen is cleared and the screen is refreshed to include the new problem assessment by the practitioner as part of the ‘Current Problems’ list.
