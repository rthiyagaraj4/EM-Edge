Enter the following details:

 

  1. Reason for Transfer – Select the Reason for Transfer from the list. This field is mandatory. The values in the list are defined in the ‘Transfer Type’ function of IP Code Setup. The details under the ‘Bed Block Details’ are populated if the chosen Transfer Type has a Blocking Type associated.

 

Nursing Unit Details

 

Transfer To

  1. Nursing Unit – The Nursing Unit in which the patient is currently located is displayed. This cannot be changed as transfer takes place within the same Nursing Unit.

  2. Service - Select a Service from the list. This field is mandatory.

  3. Sub-Service – Select a Sub Service from the list that is associated to the selected Service.

  4. Request Bed Class Change – This field appears only if the ‘Request for Bed Class Change’ check box is selected in the IP Parameter. Once the check box is selected, the system allows the change of Bed Class and Bed Type.

If you perform the intra-ward transfer has the privilege of confirming such requests, as defined by the selecting the flag ‘Bed Class Change’ against the operator station to which the logged in user is attached to, the change for bed class request is confirmed in this transaction itself and the transfer of the patient is completed. If the logged in user does not have the privilege all such requests are routed to the ‘Confirm Bed Class Change’ function where an authorized user confirms the bed class change after which the intra-ward patient transfer is completed.

 

  1. Bed Class - The Bed Class associated to the patient is displayed. The value can be changed if the ‘Request for Bed Class Change’ check box is selected.

  2. Bed Type - The Bed Type associated to the patient is displayed. The value can be changed if the ‘Request for Bed Class Change’ check box is selected. The values displayed are based on the Bed Class selected.

  3. Bed No – The Bed to which the patient has to be transferred is entered in alphanumeric format of eight characters maximum length or select the bed from the ‘Bed availability Chart’ by clicking on [?] function.

  4. Room/Cubicle No – The room/cubicle number within which the selected bed is located is displayed.

  5. Specialty – The Specialty to which the patient was admitted is displayed. If the Nursing Unit caters to multiple Specialties, a different Specialty can be selected by entering the Specialty or using [?] function. This is a mandatory field.

  6. Attending Practitioner – The name of the Practitioner currently attending the patient is displayed. The name of the Practitioner can be changed if required. Enter the name of the Practitioner or select the Practitioner from [?] function for the selected specialty. This is a mandatory field.

  7. Medical Team – The Medical Team (if any) associated to the selected practitioner is displayed. The Medical Team can be changed if required. Select the medical team from among the values in the list. The values populated correspond to the selected practitioner.

 

Bed Block Details

  1. Block Type – The Blocking Type is displayed based on the value selected for ‘Reason for Transfer’. This field cannot be changed. If the selected ‘Reason for Transfer’ does not have any Blocking Type associated as defined in ‘Transfer Type’ master function in IP code setup, this field is blank.

  2. Block Period – The default Block Period as defined for the Blocking Type is displayed.

  3. Bed Blocked Until – The date/time until when the bed has to be blocked as defined by the Block Period is displayed. The date/time is calculated by adding the block period to the system date.

  4. Remarks – Enter any Remarks if required for the Blocking in alphanumeric format of maximum length 60 characters.

 

Reserved Bed Details

This section remains disabled if the ‘Restrict Reserving for Intra-Ward Transfer’ check box is  selected in IP Parameter. There by, you will not be allowed to reserve bed during intra-ward transfer.

  1. Reserve Bed – Select the check box to reserve bed for the patient.

  2. Reason for Reserve Bed – Select the reason for reserve bed from the list. This field is mandatory if the Reserve Bed check box is selected.

If the bed class selected is of a higher privilege level, as defined against the billing group of the bed class, than the privilege level of the patient as defined against the billing group of the patient, the system pops-up a window asking you to clarify if the bed class change was due to the patient’s request or the facilities request. If the bed class change was due to the request of the patient, all the charges for the patient are recalculated based on the new billing group associated with the bed class selected. If the bed class change was due to a request from the facility, the user doing the confirmation must authorize the bed class change by providing a Username and pin as defined in System Manager. If the privilege level of the bed class is lesser than the privilege level associated to the patient, by virtue of the billing group associated to the patient, this authorization is not required and hence this window does not pop up.
