MR Batch Process 

From this function, you can  Define the batch process to cancel all the incomplete requests before the cut-off date.

This function allows you to define the batch process to cancel all the incomplete requests in ‘Medical Report’ function that are pending for X number of days. ‘X’ value is defined ‘Max No. of days for Incomplete Requests Cancellation’ in MR Parameter screen. This parameter allows you to calculate the cut-off date for cancelling incomplete requests.

Defining MR Batch Process

From the MR Setup select MR Batch Process function.

The MR Batch Process screen gets displayed.

  1. By default, the Process drop-down field displays the text ‘Auto-Cancel Incomplete Requests’ in read only mode. This value is hardcoded.

  2. By default, the Cut-off Date is displayed. The cut-off date is the difference between Current date (system date) and ‘Max No. of days for Incomplete Requests Cancellation’ value defined in MR Parameter screen. For example, if the ‘Max No. of days for Incomplete Requests Cancellation” is set 3 days and current date is ‘07/07/2017’ then the Cut-off Date is shown as ‘04/07/2017’. Cut-Off date is calculated based on Date only and not based on ‘Date/Time’.

  3. By default, Total Incomplete Requests for Auto Cancel i.e., the total count of incomplete medical report requests to be cancelled before cut-off date gets displayed. The incomplete request is considered for counting based on the date when the ‘Application Status’ was changed to ‘Incomplete’ in ‘Status of Medical Report Request’ screen.

  4. Click [Generate Report] to generate a report showing list of incomplete requests before cut-off date.

  5. Click [Incomplete Requests] to view the list of incomplete medical report requests. The Incomplete Requests details showing Requests ID, Request Date, Patient ID, Patient Name and Gender gets displayed.

  6. Click [Apply]. The incomplete requests get cancelled automatically.

  7. The incomplete requests are cleared from the screen. The ‘Cut-off Date’ and ‘Total Incomplete Requests for Auto Cancel’ fields are recalculated. The count of “Total Incomplete Requests for Auto Cancel” becomes zero.

All the three buttons Generate Report, Incomplete Requests, and Apply button are enabled only if the value of “Total Incomplete Requests for Auto Cancel” field is greater than zero.
