Purchase Requisition

From this function you can Create purchase requests. You can Query for the existing requests.

This function allows the concerned stores to create purchase requests and send it to the concerned purchase units. The purchase units review the request and generate a purchase orders against a supplier if required.

Creating Purchase Requests

From the PO Transactions, select the Purchase Requisition function.

The Purchase Requisition screen gets displayed.

  1. Click [Create] to invoke the Create screen to create a new purchase request.

  2. Enter the details in the mandatory fields (*). <Related Field Description>

  3. Enter the Item or select it using the [?] function. This is a mandatory field.

  4. On selecting the item, the Unit of Measure, Conversion Factor is displayed by default.

  5. Select the Request UOM and enter the Request UOM Quantity in numeric digits up to seven characters in length.

  6. Select the Required By Date by clicking the calendar function.

  7. Click ‘Remarks’ hyperlink to enter the item related remarks.

  8. To view the item batch details, click to open the Batch Search dialog.

  9. Click [Add]. The item details are added in the upper part of the screen.

  10. Click ‘Item Code’ hyperlink to modify the item details. On entering the details, click [Modify] to save the changes or click [Cancel] to cancel the changes.

  11. After adding the required items, click [Apply]. A ‘Confirm Finalize’ dialog is displayed. Click [Yes] to finalize the purchase requests. On clicking [Yes], a ‘Confirm Printing’ dialog is displayed.

  12. Click [Yes] to print the purchase request or click [No] to cancel the printing.

  13. A Doc Number is generated for the purchase request based on the rule setup.

  14. Click [Query] to query for the existing purchase requests.
