Enter the following additional information in the Document section:

 

  1. PO No – Enter the Purchase Order Number against which the goods have been received in numeric format of 10 digits maximum length. If the Purchase Management module of Enterprise Management is operational, this number is validated against Purchase Order Number available in the system.

  2. Supplier – Select the Supplier from whom the goods are received from the list. This is a mandatory field. The Supplier field gets  disabled once PO number is selected.

  3. Currency – Select the foreign currency applicable in which the goods are received from the list. Presently this option is disabled.

  4. Receipt Date – The current system date is displayed, by default. Enter the Receipt Date in dd/mm/yyyy format or select the date using Calendar function. This is a mandatory field.

  5. Entry Completed – Select ‘Entry Completed’ check box to indicate GRN details entry is completed and ready for finalization or clear the ‘Entry Completed’ check box to save the transaction and finalize in the query mode. The check box is cleared, by default.

  6. Remarks - Click 'Remarks' hyperlink to record the goods received remarks. Select the pre-defined remarks using [?] function. The recorded remarks is printed in the GRN report and Reprint Document Report.

  7. DO No - Enter the Delivery Order Number against which the goods have been ordered in numeric format of 20 digits maximum length. This field is enabled only if Purchase Module Installed check box is checked in Entity Parameter screen

  8. Invoice No - Enter the Invoice Number against which the goods have been ordered in numeric format of 30 digits maximum length. This field is enabled only if Purchase Module Installed check box is checked in Entity Parameter screen.

  9. Invoice Received By - Enter the Invoice Received By person name who gets the goods have been ordered in numeric format of 30 digits maximum length. This field is enabled only if Purchase Module Installed check box is checked in Entity Parameter screen.

 

Enter the following details in the Item Entry Area:

 

  1. Item - Enter the required Item in alphanumeric format of 60 characters maximum length or select the Item using [?] function. This is a mandatory field. On selection of an Item, the UOM associated to the Item displayed by default and the Batch Entry section are displayed, which allows entry of batch details.

  1. Normal - Enter the quantity received (against the Purchase Order) in numeric format of 12 digits maximum length. On saving the record, the quantity entered is in terms of Purchase Unit of Measure and the quantity in terms of stock Unit of Measure gets added to the Item's Quantity on Hand. This field is mandatory. When the Purchase Management module of Enterprise Management is operational, the Pending Purchase order quantity is reduced by the normal quantity received and the system ensures that the normal quantity is not greater than the pending Purchase Order quantity.

  2. Bonus - Enter the Bonus Quantity of Item received in numeric format of 12 digits maximum length. The quantity entered is in terms of Purchase Unit of Measure. This field is enabled if ‘Allow Bonus Entry’ check box is selected in MM Parameter or this field is disabled.

  3. Sample – Enter the number of Items in numeric format of 12 digits maximum length. This field is enabled if ‘Allow Sample Entry’ check box is selected in MM Parameter or this field is disabled.

  4. Unit Cost – Enter the cost of the Item for one unit, if known. If not, when the Item is selected the Cost is displayed automatically, which is entered while defining the Item.

  5. Value – Enter the Value or the Value is displayed based on the Unit Cost. The value is the total value on the Items received. The Value calculated is: Unit Cost x Normal.

  6. Label – The check box is made default based on the value selected in ‘Item Master’ in Inventory Management module. On selecting the check box, bar code label is generated for the Item on applying.

  7. No of Labels – The Label value is displayed based on the entered Normal value. You can enter the number of labels required in numeric format of 12 digits maximum length. The check box is enabled, if the ‘label’ printing is applicable. If the quantity entered is in decimals, the Number of Labels will round off to the next number.

  8. Replace on Expiry – Select the check box, if the Supplier agrees to replace the Item on expiry.

  9. Purchase UOM – Select the Purchase UOM from the list when the goods are received other than the stock Unit of Measurement.

  10. Conv Factor/Rule – The Conversion Factor for Purchase UOM is displayed.

 

Batch Entry Area

This area gets displayed on selection of an Item and allows entry of batch details.

The  barcode is captured as a combination of Item code +Batch ID +Expiry Date”  during Allocation/Delivery stage and its batch is selected automatically and dispensed. The Hospital name is also printed along with this Barcode label and affixed in the drug box.   1. Trade Name – Select the required Trade Name from the list. The Trade Names associated to the selected Item in Materials Base module is displayed in the list.

  2. Bin Location – Select the required Bin Location from the list. All the Bin Locations associated to the selected item in ‘Item Store Bin Location’ function is displayed in the list.

  3. Qty – Enter the quantity in numeric format of eight digits maximum length. The entered batch quantity must be equal to the total quantities received or an error message is displayed.

  4. Barcode ID  – Enter the Barcode ID of the item in alphanumeric format of 20 characters maximum. If the barcode of the item is scanned, then the barcode ID is populated automatically during transactions.

It is mandatory to enter values in all the fields in Batch Entry Area.

  1. Sale Price – Enter the sale price.

Sale Price is displayed on the Label for drugs with Setup as Base Prices in ‘Setup/Amend Charge Factor for Material Linked Services’ & Average Price in ‘Material item Charge setup’. Whenever a new item is created, the Pharmacist enters the Base Price using Drug Item Price Setup. The GRN is made after this; The Base price entered appears in the Label printed. This will help the Pharmacist to verify the prices during the time of dispensing of medication. The price will be displayed only from Base price (Standard Price of Hospital/HAAD) and is applicable only for Drug Item.

  1. Uploaded Document – Click [Browse] to open, select and upload the required file from the system. The uploaded file name gets displayed in the text box before the Browse button.  This text box is always in disabled state and is non editable. User can replace the previously uploaded file in the GRN Edit Mode till the GRN Document is not Finalized. Once its finalized, the uploaded GRN Document cannot be changed or a new GRN Document cannot be uploaded. Only one document file can be uploaded per GRN Item + Batch Combination.
