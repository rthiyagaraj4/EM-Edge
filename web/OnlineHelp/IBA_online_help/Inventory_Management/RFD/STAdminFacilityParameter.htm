Enter the following details to create ST Facility Parameter record:

  1. Current Processing Month/Year – This period denotes the Inventory Management module’s processing period. This period must always denote the current processing month and year and not the calendar month and year. This month and year will be automatically updated to the next period during month end processing. The processing month and year cannot be changed, once it is entered and the changes are committed. This is a mandatory field.

  2. Display Lookup in Physical Inventory Entry - Select Display Look Up In Physical Inventory Entry value from the drop-down list. The listed values are ‘Display and allow addition of new batches’, ‘Allow addition only through batch lookup’ and ‘Allow addition of new batch but do not display batch lookup’. By default, ‘Allow addition only through batch look up’ value is selected.

    • If ‘Display and allow addition of new batches’ is selected, you can add batches using batch look up and directly within the Physical Inventory Entry screen.

    • If ‘Allow addition only through batch look up’ is selected, you can add the batch only using batch look up. You cannot add a new batch directly within the Physical Inventory Entry screen.

    • If ‘Allow addition of new batch but do not display batch lookup’ is selected, you can only add new batch using Physical Inventory Entry screen.

  3. Applicable Batches In Physical Inventory Entry - Select 'Applicable Batches In Physical Inventory Entry' value from the list. The listed values are 'All Batches' and 'GRN Batches Only'. This field is enabled if 'Display Batch lookup in Physical Inventory Entry' is selected. If 'All Batches' is selected, on searching all the batches are displayed in the 'Physical Inventory Entry' function. If 'GRN Batches Only' is selected, only GRN batches are displayed, on searching all the batches are displayed in the 'Physical Inventory Entry' function.

  4. Allow Change of Batch During Authorize at Issue Stage – Select this check box to change the batch details during authorization in the ‘Authorize at Issue Store’ function. By default, this check box is cleared.

  5. Allow Authorize of Partial Quantity Without Remarks –  Select this check box to authorize at Issue Store if the authorization quantity is not equal to the requested quantity without entering any remarks. By default, this check box is selected. If this check box is cleared, ‘Remarks’ is mandatory if the authorized quantity is not equal to the requested quantity in the ‘Authorize at Issue Store’ function.

  6. Allow Equivalent UOMs for Physical Inventory Entry - Select this check box to display two new columns  ‘UOM’ and ‘Base UOM Qty’ in Physical Inventory Entry screen. Also, a UOM drop-down list is displayed in the New Batch Details section.. By default this check box is not selected.

  7. Use 'Authorize at Issue Store' privilege to Authorize – If you select this check box, you cannot perform Requisition Issue’ privilege in the Authorize at Isssue Store’ function on the stores. By default, this is not selected and enabled. If this is not selected, you can perform transaction the ‘Authorize at Isssue Store’ function on the stores where you have ‘Requisition Issue’ privilege.

  8. Restrict Requisition of Stock if exceeding Max Quantity - Select the check box to restrict the user from requisitioning stock in ‘Generate Request’ and ‘Create/Review Request’ transactions if the condition ‘Requested Quantity + Quantity on Hand exceeds the Max Quantity’ is met. By default, the check box is not selected.  The system displays the error message, ‘Requested Quantity plus Quantity on Hand exceeds Maximum Quantity by X Qty’ where X = Max Stock Quantity – (Requested Quantity + Quantity on Hand) in the transactions.

 

Acknowledgement For

  1. Issue(Reg /Urg) - Select the check box(es) against this transactions, if you want to enable or disable the acknowledgment at the Facility level.

  2. Issue Return - Select the check box(es) against this transactions, if you want to enable or disable the acknowledgment at the Facility level.

  3. Stock Transfer - Select the check box(es) against this transactions, if you want to enable or disable the acknowledgment at the Facility level.

When you enable or disable the acknowledgement, it is reflected across the facility for all its constituent stores.

Other Defaults

  1. Goods Receipt – Select the default Store from the list. The Stores defined in the Store function with ‘Goods Receipt’ transaction applicable are displayed in the list.

  2. Adjustment Remarks – Select the default Adjustment Remarks using [?] function. The remarks created in the function ‘Transaction Remarks’ of Materials Base module are displayed. The selected remarks is displayed and made default in the Stock Adjustment transaction, which you can modified.

  3. Default Item Search By – Select the default search criteria from the drop-down list based on which the item has to be searched. The listed values are Item Code, Item Name, Machine Code, Part Number, Trade ID, Trade Name and Alpha Code. If the default search criteria is selected as ‘Trade Name’, then the ‘Trade Name’ check box is selected by default in the Item Search dialog.

  4. Item Image/Catalogue Path - Enter the default server path where drug image, catalogue, brochure and other files related to the item are stored. The system supports all file formats.

  5. Default Manufacturing Mode – Select the Default Manufacturing mode value from the list. The listed values are ‘Direct Manufacturing’ and ‘Based on Request’. The default value is ‘Based on Request’.

    • If the ‘Direct Manufacturing’ value is selected, the ‘Direct Manufacturing’ option is selected in the Manufacturing screen by default.

    • If the ‘Based on Request’ value is selected, the ‘By Request’ option is selected in the Manufacturing screen.

  6. Sales Return - Select all items by default when Sale Document is picked – Select the checkbox to select the items by default when the sale document is picked. This check box is enabled and selected by default.

  7. Default Duration for Expiry Date for Disposable Drugs without Expiry Date – Enter the default duration for deriving the expiry date for the stock adjustment for disposable items transaction. The default value is ‘0’ which means the Batch Expiry Date is the system date.

For example, if the number of days entered in the text box is ‘10’ then the system automatically defaults the batch Expiry Date as the system date plus 10 days in the Stock Adjustment for Disposable Items function.

  1. Default Receipt Cutoff Days for consideration of Batches for Physical Inventory - Enter the default cut off days in numeric format of 5 characters maximum length. By default, it is blank. The value mentioned in this field reflects in the ‘Batch Search’ screen in Physical Inventory Entry function.

    • For example, if the default receipt cutoff days is entered as ‘100’, the Batch Search screen fetches all the batches for the last 100 days from the current date and displays in the search result.

    • When the default receipt cutoff days is blank, all batches matching the remaining selected search criteria are displayed.

    • When the default receipt cutoff days is zero, all batches for the current date are considered for searching.

  2. Default Issue Return mode - Select any one of the below options based on which the fields in the Issue Return screen are enabled or disabled.

    • ‘Issue Document based’ return - The Issue Doc Type and Issue Doc No fields are enabled and Item Search is disabled.  

    • ‘Item based’ return - The Item search is enabled and Issue Doc Type and Issue Doc No fields are disabled.

  3. Issue History Days (Max 365) – Enter the number of days to be considered while displaying the history of Previous Issue transactions for the Items when the user performs Create / Review Request, Authorize at Issue Store or Issue (Regular/Urgent) transactions.  User cannot enter more than 365 days. By default, the value 30 is displayed. This field is not mandatory.

  4. Restrict Stores to User Accessible Stores in Reports/Queries – Select the check box to restrict the users from generating Reports and Queries with the data related to stores that the user is not Authorized to access across all Facilities. By default, the check box is cleared.

    The system checks the ‘User Access for Store’ parameter to identify which stores the User has access to and which stores the user is not Authorized to access.
