Period End

From this function you can Close the Current Processing Period, so that, no further Transactions are posted for the Current Processing Period.

The profit/loss and the income incurred by the facility during a certain period are calculated during the end of the accounting period in the facility. The unposted documents or transactions have to be posted to the General Ledger module, Deferred Cash Patients, Accounts Receivable module respectively to process period end. Period End is not processed, if unposted folio transactions exist. The Current Processing Period is incremented by one on closing the period. Based on the period end scope defined in the MM Parameter, an option to specify the ‘Current Processing Month/Year’ is provided at the Entity Level or the Facility Level. You can specify the ‘Current Processing Month/Year’ based on this setup.

Closing the Current Processing Period

From the ST Period End Process, select the Period End function.

The Period End screen gets displayed.

  1. The Current Processing Month/Year specified in the Entity Parameter or Facility Parameter based on the period end scope defined in the MM Parameter is displayed and disabled.

  2. Click [Apply] to close the current processing period and the period would get incremented by one or [Reset] to clear the entered data.
