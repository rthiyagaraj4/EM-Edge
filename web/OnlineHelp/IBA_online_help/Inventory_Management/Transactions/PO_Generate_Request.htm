PO Generate Request

From this function you can Generate Purchase Order requests for multiple Items.

This function should be interfaced with the Oracle Inventory. You can access this function if 'Purchase Order Applicable' check box is selected in the 'Entity Parameter' function. Based on the re-order levels of stock, Purchase Orders are raised by the main stores. The system calculates the Request Quantity based on the Reorder Level, Maximum Stock Level, Reorder Quantity and Quantity in Hand.

Generating Purchase Order Requests

From the ST Transactions > PO Request, select the PO Generate Request function.

The PO Generate Request screen gets displayed.

  1. Enter the details in the mandatory (*) fields. <Related Field Description>

  2. Click [Search] to display the required Items for which the request has to be raised.

  3. On searching, the Items which do not require ‘Special Request’ and for which ‘By manual’ value is not set for ‘Req Gen Base’ in Item for Store function are displayed.

  4. If the requested quantity is not available in the Requested on Store, these Items are indicated by a unique indicator.

  5. Select the check box against the required Item to generate purchase order request for refill of Items. Select the Item check

  6. box in the heading to select all the displayed Items to generate purchase order request for refill of Items.

  7. Enter the Required Quantity in numeric format of 10 digits maximum length. On placing the mouse pointer on the Request

  8. Quantity text box, the Item is selected by default and the Item details are displayed with a unique color.

  9. If the entered Required Quantity value is greater than the ‘Max Stock Quantity’, you will be prompted with an alert message

  10. ‘Req Qty should be less than Max Stock Qty. Proceed?’. Click [OK] to proceed with the request generation or [Cancel] to cancel the request generation for the selected Item.

  11. Click [Apply] to generate the request or [Reset] to clear the entered data.
