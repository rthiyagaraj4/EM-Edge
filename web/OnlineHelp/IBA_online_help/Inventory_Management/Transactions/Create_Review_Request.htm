Create/Review Request

From this function you can Create or Review the requests raised for Items.

The request raised for Items can be reviewed or created irrespective of the Maximum Stock and Reorder Levels. A request document generated earlier is edited using this function before it is authorized. The generated request quantities can be modified, new Items can be added or existing Items can be deleted. A new requisition is entered using this transaction.

 

The system restricts the user from requisitioning stock if the condition ‘Requested Quantity + Quantity on Hand exceeds the Max Quantity’ is met based on the parameter ‘Restrict Requisition of Stock if exceeding Max Quantity’ selected in ST Facility Parameter.

Creating or Reviewing Requests for Items

From the ST Transactions > Issue, select the Create/Review Request function.

The Create/Review Request screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Modify Document Details screen is displayed, by default or click ‘Document’ hyperlink to modify the document details.

  3. On entering the Document Details, click [OK] to create or review the request for Items or [Reset] to clear the entered data.

  4. Enter the Item or click [?] function; displays the Item Search screen.

The item should be in alphanumeric format of 60 characters maximum length. This is a mandatory field.

You can click the information icon next to the Item for viewing the ‘Item Remarks’.

  1. Enter the search criteria and click Search; displays the search results.

  2. On selecting the Item, the Unit of Measure, Conversion factor, and the Quantity on is displayed. If the Item is Stock Item, the Stock Item is selected and disabled. The Kit Template UOM is displayed, where the Kit Template is applicable.

  3. Enter the Request UOM Quantity in multiples of 25. This is a mandatory field.

  4. On entering the Request UOM Quantity value, the Required Quantity is displayed and disabled.

  5. Click [Add] to add the request details and the details are displayed in the top half of the screen or [Cancel] to cancel the selected Item.

  6. On adding the Item, the Item details are displayed with a unique color.

  7. Click ‘Item Code’ hyperlink to modify the Item details. You can modify the Required UOM and Required UOM Quantity. Click [Modify] to modify the updated changes or [Cancel] to cancel the changes.

  8. Select the check box to select the Item as part of the request. The ‘Select’ check box is selected/cleared by default based on the setup done in Kit Template screen.

  9. Click Remarks hyperlink to capture the remarks using the [?] function. The remarks selected in the Transaction Remarks screen are displayed in the lookup function. Click [Close] to exit from the Remarks screen.

 The Remarks field is mandatory if the ‘Header Level Remarks Required’ check box is selected in the Transactions Remarks screen.

  1. Click [Apply] to finalize the Item request details or [Reset] to clear the entered data.

  2. The ‘Confirm Finalize’ dialog is displayed. Click [Yes] to finalize the Item request details or [No] to finalize the Item request details in the query mode.

  3. The ‘Confirm Printing’ dialog is displayed based on the attribute set for the Document Type. Click [Yes] to print the Item request details or [No] to cancel the action.

  4. Click [Query] to invoke the Query screen.

  5. The unauthorized Item request details are displayed based on the default criteria or enter the search criteria and click [Search] to select a specific record.

    Enter the following search criteria:

     

    1. Doc Type – Select the Document Type from the list. The values listed are defined in the function ‘ST Document Type’ and the default value displayed is specified in the function ‘ST Facility Parameter’.

    2. Doc No. – Enter the Document Number in numeric format of eight digits maximum length.

    3. Doc Date - Enter the date in date format of dd/mm/yyyy or select the required date suing the Calendar function.

    4. Req By Store – Select the Requisition by Store from the list. The Stores for which you have access and ‘Requisition’ transaction function applicable are displayed. This is a mandatory field.

    5. Req On Store – Select the Requisition on Store from the list. The Stores for which you have access and ‘Requisition Issues’ transaction function applicable are displayed. This is a mandatory field.

    6. Item Class – Select the required Item Class value from the list. The Item Class associated to the ‘From Store’ are displayed in the list.

  6. On searching, the unauthorized Item request details are displayed.

  7. Click ‘Doc Type’ hyperlink to authorize the Item request details.

  8. Click [Apply] to finalize the Item request details or [Delete] to delete the selected record.

  9. On clicking [Apply], the ‘Confirm Finalize’ dialog is displayed. Click [Yes] to finalize the Item request details or [No] to finalize the Item request details in the query mode.

  10. On selecting the ‘Delete’ check box, ‘Confirm Delete’ dialog is displayed. Click [Yes] to delete the record or [No] to cancel the action.

An alert message window is displayed with duplicate item details of the current pending requests.
