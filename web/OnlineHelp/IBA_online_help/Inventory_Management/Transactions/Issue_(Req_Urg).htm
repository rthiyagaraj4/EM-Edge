Issue (Req/Urg)

From this function you can Create Stock Issue Document and Generate Urgent Issue Document. You can Query for the authorization requests and the urgent issue requests, which are not finalized.

Issue details are entered against an authorized requisition. The authorized requisition document is recalled on the screen of Issue Stock transaction and details of Issuing Stores and Issue Quantity for each Item is entered. The details of Adhoc Issue of Items from one Store to another Store are entered using this function. Urgent Issue is used where there is no issue requisition and authorization; the issue details are directly entered.

Creating and Generating Urgent Issue Document

From the ST Transactions > Issue, select the Issue (Req/Urg) function.

The Issue (Req/Urg) screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. The Modify Document Details screen is displayed by default or click ‘Document’ hyperlink to modify the document details.

  3. On entering the Document Details, click [OK] to create and generate Urgent Issue Document or [Reset] to clear the entered data.

  4. Select Close Upon Issue to close the item on issue.

  5. Click Remarks hyperlink and enter the remarks for closing the issue in the Remarks dialog that opens. This is mandatory.

  6. Click [?] to search for pre-defined remarks.

  7. Scan the barcode of the Item; the barcode is displayed in the text box below Close Upon Issue. Click next to the text box. the application checks if the scanned barcode matches with any of the items listed under the Item Details section. If it matches, then that item is highlighted and the details of the item are displayed in the respective fields. If it does not match, then the application displays a message “The scanned barcode and the barcode of the Listed Item(s) does not match. Please scan the correct barcode.”. The scanned, scanning pending and current record items are displayed by means of color-coded background as mentioned below:

You can scan the item that is requested and authorized by the issuing store in the Issue (Reg/Urg) function.

  1. If there are some more item(s) pending for barcode scanning, the application displays a message “Barcode scanning is not complete. There are item(s) pending for barcode scanning. Do you wish to proceed further?”. However you can proceed further without scanning the rest of the items by clicking ‘Yes’. You can click the information icon next to the Item for viewing the ‘Item Remarks’.

  2. Enter the Item or click [?] function; displays the Item Search screen. Refer <Related Field Descriptions>.

The item should be in alphanumeric format of 60 characters maximum length. This is a mandatory field.

You can click the information icon next to the Item for viewing the ‘Item Remarks’.

  1. Enter the search criteria and click Search; displays the search results.

  2. Enter the Item to create and generate Urgent Issue Document or click [?] function; displays the Item Search screen.  Enter the search criteria in the Item search screen and click Search; displays the search results. The item should be in alphanumeric format of 60 characters maximum length. This is a mandatory field. Refer <Related Field Description>

  3. On selecting the Item, the Unit of Measure, Conversion Factor, Available Quantity and Cost Quantity are displayed in the respective fields in the Issue(Reg/Urg) screen.

  1. Click [Edit/View] to Issue the first expiring batch in the slot. The Item Batch details are displayed. The Issue UOM Quantity entered is displayed as the UOM Quantity. Select the ‘To Store Bin' from the list. The Bin Locations associated to the selected ‘To Store’ are displayed in the list.

  2. Click [Batch Search] to view the Batch details. The selected Item Batch details are displayed. Select the check box to select the Batch and click [Close] to exit from the batch search screen. If batch details are selected using [Edit/View], the selected Batch details are refreshed on selecting the new batch.

  3. On selecting the batch, the batch details are displayed in the Batch Entry area.

  4. Select ‘To Store Master Bin Location’ from the list and enter the Issue Quantity.

  5. Click [Add] to add the Item for Issue or [Cancel] to cancel the selected Item details.

  6. Click [Stock Level] to view the ‘From’ and ‘To’ stock details.

  7. The selected Item details are displayed in the Item Details frame on the left hand side. Click the Information icon near the item name to view either 'Remarks' or 'Specification' or both for the selected item in a popup or as a tool tip. The Information icon  will be available only if either 'Remarks' or 'Specification' or both exists for that item. Click ‘Item Details’ hyperlink to modify the Item Issue details. Click [Modify] to modify the entered details or [Cancel] to cancel the modified changes.

  8. Select the Close Upon Issue check box to close the issued request on making the first issue or this document will be open till all the requested quantities are issued. This check box is selected in case of part issues.

  9. Click Remarks hyperlink to capture the remarks for generating urgent issue document using [?] function. It is mandatory to record the remarks if 'Close Upon Issue' check box is selected. This is a mandatory field when you amend the quantity. Click [Close] to exit from the remarks screen.

The Remarks field is mandatory if the ‘Header Level Remarks Required’ check box is selected in the Transactions Remarks screen.

  1. You can select the Delete check box to delete the selected record. The Delete check box is selected or cleared by default based on the setup done in Kit Template screen.

The items marked as ‘Delete’ go through the ‘Authorize at Issue Store’ screen before being available for issue again.

  1. Select the Hold checkbox next to items planned to be issued later.

You can Issue this partial list of requested items in multiple tranches as they are put on hold and need not go through the Authorize at issue store screen. You can view all items marked as ‘Hold’ of the same request document in Query mode as well.

Whenever one or more items are put on hold, a fresh sequence number is generated against the request  document number.

You can either ‘Delete’ or ‘Hold’ an Item in a single request document.  You cannot mark all the items in a document as ‘Delete’ or ‘Hold’ at the same time.

If the issue quantity of an item is changed during issue, such items cannot be put on hold. The Hold checkbox is disabled in this scenario.

If all items are marked as ‘Hold’ during ‘Issue’, the following error message is displayed:

“Transaction must have at least one detail record.”

  1. Click [Preview] to view the Issue Preview details.

  2. Click [Apply] to finalize the Item issue details or [Reset] to clear the entered data.

  3. The ‘Confirm Finalize’ dialog is displayed. Click [Yes] to finalize the Item issue details or [No] to finalize the Item issue details in the query mode.

  4. The Confirm Printing dialog is displayed based on the attribute set for the Document Type. Click [Yes] to print the Item issue details or [No] to cancel the action.

  5. Click [Query] to invoke the Query screen.

  6. The unauthorized Item issue details are displayed based on the default criteria or enter the search criteria and click [Search] to select a specific record.

    Enter the following search criteria:

     

    1. Doc Type – Select the Document Type from the list. The values listed are defined in the function ‘ST Document Type’ and the default value displayed is specified in the function ‘ST Facility Parameter’.

    2. Doc No. – Enter the Document Number in numeric format of eight digits maximum length.

    3. Doc Date - Enter the date in date format of dd/mm/yyyy or select the required date using the Calendar function.

    4. From Store – Select the ‘From Store’ from the list.

    5. To Store – Select the ‘To Store’ from the list.

    6. Item Class – Select the required Item Class value from the list.

    7. Order By - Select the value 'Urgent Issues' or 'Normal Issues' from the list. On selecting the 'Order By' value, select 'Doc No', 'From Store' or 'To Store' and select 'Ascending' or Descending' from the list.

  7. On searching, the unauthorized Item issue details are displayed.

  8. Click ‘Doc Type’ hyperlink to authorize the Item issue details.

  9. Click [Apply] to finalize the Item issue details or [Delete] to delete the selected record.

  10. On clicking [Apply], the ‘Confirm Finalize’ dialog is displayed. Click [Yes] to finalize the Item issue details or [No] to finalize the Item issue details in the query mode.

  11. On selecting the ‘Delete’ check box, ‘Confirm Delete’ dialog is displayed. Click [Yes] to delete the record or [No] to cancel the action.

 

The Minimum Stock Level or Reorder Level of the selected item is displayed while performing the transaction. If the stock is less than the Minimum Stock Level or Reorder Level those stock levels are yellow highlighted. When the available quantity of a selected item is less than the Minimum Stock Level or Reorder Level in Dispense Medication and Sale/Patient Issue, a link to ‘Create/Review Request’ is displayed to request new stocks. Based on selection of Display Min Level & ROL check box in User Access for Store, this functionality will be enabled.
