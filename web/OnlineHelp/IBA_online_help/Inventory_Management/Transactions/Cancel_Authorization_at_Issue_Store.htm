Cancel Authorization

From this function you can Query and Cancel authorization for all items or specific items in a request

This function allows user to cancel authorization either for the entire document or specific items.

 

For example, if there are 5 items in a request all of them were authorized and 3 items have been issued (partially / fully) only remaining 2 items which are not issued in the request, authorization can be cancelled by selecting individual item. User cannot cancel the authorization of entire request. But if all 5 items in the request have not been issued, the authorization can be cancelled at document level .

 

User cannot cancel the authorization for items in document if they are issued.

Cancelling Authorization

From the ST Transactions > Issue, select the Cancel Authorisation function.

The Cancel Authorization screen gets displayed.

  1. Click [Query] to invoke the Query screen.

  2. Enter the  search criteria  and click [Search] to search for Issue Requests matching search criteria.

    Enter the following search criteria:

    1. Requests Of – Select the required value from the list. The listed values are ‘All’, ‘Internal Stores’ and ‘External Stores’. The value ‘All’ is displayed, by default.

    2. Doc Type – Select the required Document Type from the list. The documents associated to the transaction ‘Requisition Issues’ are displayed in the list. The Document Type, as set in ‘ST Facility Parameter’ is displayed, by default.

    3. Doc No. - Enter the Document Number in numeric format of eight digits maximum length.

    4. Req By Store - Select the Requested by Store from the drop-down list. Only those Stores for which logged user has access will be displayed. The access is provided by selecting ‘Cancel Authorization at Issue Store’ check box in ‘ST User Access for Store’ function.

    5. Req On Store - Select the Requested on Store from drop-down the list. Only those Stores for which logged user has access will be displayed. The access is provided by selecting ‘Cancel Authorisation at Issue Store’ check box in ‘ST User Access for Store’ function.

    6. Item Class – Select the required Item class from the list. The Item Class associated to the selected Requested By store and Requested On store are displayed in the list.

    7. Doc Date [From – To] – By default, current week date. User can enter the date in date format of dd/mm/yyyy or select the required date using the Calendar function.

    8. Item – Enter the stock item name or click [?] to select the item from the Item search dialog.

  3. For cancelling authorization in scenarios, refer the below steps:

    •   To cancel the authorization for all the items in the request at a time, select the ‘Cancel’ check box at the column header level.

If the Cancel Check box at the column header level is checked, all the check boxes at document level (i.e. Issue Requests level) as well as the Line Item level (i.e, Stock item level) are also checked.

    • To  cancel the authorization for single /multiple individual items (provided items not issued), select the respective 'Cancel' check box(es) of the Issue Requests at the Request Issue level.

    • To cancel authorization for an Item, click on a specific Item by clicking ‘+’ hyperlink, provided the item has not been issued (even partially).

  1. After selecting the ‘Cancel’ check box, select the ‘Reason’ from the drop-down list. Or, user can enter the reason in text box (maximum 100 characters).

  2. Once it is saved, the requests for which authorization is cancelled will appear in the 'Authorize at Issue' function with unauthorized status.

  3. The cancelled authorization items will appear in the ‘Cancel Request’ function for cancelling  and in the ‘Reject Request at Issue’ function for rejection.
