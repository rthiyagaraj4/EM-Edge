Burial Permit

From this function, you can Generate the burial permit and print the burial permit when a body has to be released from the mortuary.

A burial permit is a legal document that authorizes a claimant to bury a body after release from the mortuary. A burial permit is generated for a deceased patient even before the deceased is registered to the mortuary using this function. The burial permit number is generated based on the value recorded in 'Next Burial Permit No.' field of the MO Parameter function.

Generating Burial Permit

From MO Transactions, select the Burial Permit function.

The Burial Permit screen gets displayed.

  1. Patients discharged or checked out from AE, IP or OP modules for the current date with the discharge type as 'Deceased' and does not require to record postmortem are listed, default.

  2. Enter the search criteria and click [Search] to select a specific deceased patient. Deceased Patient Details are displayed based on the entered criteria. The records in the screen are displayed in the descending order of the Body Release Date/Time.

    1. Deceased Patient ID - Enter the deceased patient ID in alphanumeric format of 10 characters maximum length or select the patient ID using [?] function.

    2. Deceased Date - Enter the deceased date range of patients to generate the burial permit. This field is mandatory. By default the current date range is displayed. This is a mandatory field.

    3. Location - Select the location from the list where the patient was registered. The listed values are 'Clinic', 'Daycare Unit', 'Procedure Unit', and 'Nursing Unit'. On selecting the required location, the different locations of the selected location type are displayed in the corresponding list to select the specific location.

  3. Click '+' hyperlink to View Patient Details.

  4. Click the Deceased Patient Details hyperlink to record Burial Permit details.

    1. Deceased Patient ID - The patient ID of the selected deceased patient is displayed.

    2. Generate Burial Permit - Select the check box to allow the system to generate the burial permit number for the selected patient ID. Clear the check box to generate the burial permit number manually.

    3. Burial Permit No - Enter the burial permit number in alphanumeric format of 16 characters maximum length. This is a mandatory field. If the 'Generate Burial Permit' field is checked, then this field is disabled. The system generates the burial permit number based on the value set in the 'Next Burial Permit No.' field of MO Parameters function.

    4. Burial Permit Approved By - The practitioner who discharged/checked out the patient as recorded in AE, IP or OP modules is displayed as the practitioner who approved the burial permit. This is a mandatory field.

    5. Burial Permit Approved Date - Enter the date and time in dd/mm/yyyy hh:mm format or select the date and time using the calendar function of when the burial permit was approved. The current date and time is displayed, by default. This is a mandatory field.

  5. Click Location hyperlink to view Previous Encounters of the patient.

  6. Click [Apply] to generate burial permit for the deceased patient or click [Query] to go back to the Burial Permit main screen and query records for which burial permit has not been generated.

 

This function refreshes the patient queue list as per the time limit set in the field 'Refresh Interval' of the MO Parameters function.
