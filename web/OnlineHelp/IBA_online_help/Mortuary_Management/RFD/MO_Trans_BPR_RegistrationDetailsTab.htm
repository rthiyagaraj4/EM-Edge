Registration Details Tab

  1. Body Parts - The body parts displayed are defined in the Body Part function of the MO Code Setup. To register one or more body parts displayed, select the required Body Part and click the button to view the selected body part in the text box provided. Use the button to exclude one or more body parts that have already been included. At least one body part should be mandatorily selected.

  2. Remarks - Enter any remarks for the body part if required.

  3. Body Part Receipt Date Time - Enter the date and time of when the body part was received in the mortuary. The date should not be greater than the system date and time. This is a mandatory field.

  4. Service - Select the service from the list. The services listed are associated to the department specified in the MO parameter. Services are linked to the departments of the logged in facility in Application Masters module. This is a mandatory field.

  5. Referral Type - Select the type of referral from the list. The listed values are 'Login Facility', 'Enterprise' and 'External'.

  6. HealthCare Setting Type - Select the Healthcare setting type from the list. The listed values are defined in the Application Masters module.

  7. Referral Source - Select the Referral Source from the list. The listed values are populated based on the Referral Type Selected. If the 'Referral Type' is selected as Login Facility, the current login facility is populated. If the 'Referral Type' is selected as Enterprise, all the facilities as defined in the 'Systems Manger' module is populated and if the 'Referral Type' is selected as External, all the external facilities as defined in the Application Masters module are populated.

  8. Referred By - Enter the name of the person who referred the patient to the facility in alphanumeric format of 30 characters maximum length.

  9. Unclaimed Body Part - Select the box if the body part is not claimed by anyone like Next of Kin or Employers. On checking this check box, the application allows registration of body parts belonging only to 'deceased unknown patients' in this function.

  10. High Risk Body Part - Select the check box if the body part is a high risk body part.

  11. Medico Legal Case - Select the check box to register the body part as a Medico legal Case in the mortuary. This field is checked by default if it was recorded earlier for the patient. On selecting the check box, [Police Details] is enabled. The 'Postmortem Type' field is disabled and displayed as Medico Legal.

  12. Click [Police Details] to enter the police details which is enabled only if the MLC check box is checked.

    1. Police Report No. - Enter the police report number in alphanumeric format of 20 characters maximum length. The police report number is displayed if it is already entered in Accident and Emergency module. This is a mandatory field.

    2. Station - Enter the name of the station in alphanumeric format of 30 characters maximum length. The police station is displayed if it is already entered in Accident and Emergency module. This is a mandatory field.

    3. Police ID - Enter a unique ID for police in alphanumeric format of 30 characters maximum length. This is a mandatory field.

    4. Police Details - Enter the police details in alphanumeric format of 500 characters maximum length. This is a mandatory field.

    5. Police 61 Received - Select the check box if police 61 is received.

    6. Police 61 Received By - Enter the name of the practitioner who received the police 61 in alphanumeric format of 30 characters maximum length or select the name of the practitioner using [?] function. This field becomes mandatory on checking the 'Police Approval Received' field'.

    7. Police 61 Received Date - Enter the date and time when the police approval was received in dd/mm/yyyy hh:mm format. The date entered cannot be grater than the current system date. This field becomes mandatory on checking the 'Police Approval Received' field.

    8. On entering the police details, click [Close] to save and close the police details window.

  13. Postmortem Required – Select the check box if postmortem is required for the deceased. On selecting the check box, [Consent] is enabled. Also the Burial Permit Tab disappears on selecting the check box. This field appears checked if it was recorded earlier for an unknown deceased patient with Unidentified Body Part.

  14. Click [Consent] to record consent for clinical postmortem. This button is enabled if Postmortem Required field is selected. If MLC and Postmortem fields are selected, this button is disabled.

    1. Consent Given By - Enter the name of the person who gave the consent to conduct clinical postmortem in alphanumeric format of 20 characters maximum length. This is a mandatory field

    2. Relationship with deceased - Select the relationship of the person who gave the consent from the list box. The listed values are populated from the Patient Identity module. This is a mandatory field.

    3. Consent Form - Select the required consent form from the list. The listed values are defined in the "Consent Form Type" function. This is a mandatory field

    4. Description - On selecting the required consent form, the description given while recording the consent form type is displayed. The description cannot be edited.

    5. Consent Form Received By - Enter the name of the practitioner who received the consent form in alphanumeric format of 30 characters maximum length or select the practitioner using [?] function. This is a mandatory field

    6. Consent Form Received Date/Time - Enter the date and time when the consent form was received in dd/mm/yyyy hh:mm format or select the date and time using the calendar function. The date/time entered cannot be less than the date/time entered in the 'Body Receipt Date/Time' field and cannot be greater than the current system date/time. This is a mandatory field

    7. On entering the consent form details, click [Close] to save and close the consent for clinical postmortem window.

  15. Postmortem Type – This field is disabled. This field is displayed as Clinical indicating the type of the postmortem is a clinical postmortem if Postmortem Required field is selected. This field is displayed as Medico Legal indicating that the type of postmortem is a Medico legal postmortem if MLC field is selected. If MLC and Postmortem Required fields are selected, this field is displayed as Medico Legal.

  16. Brought by Police - Select the check box if the body was brought to the facility by the police.

 

Assign Area

  1. Area – Select the area from the list. The listed values are based on the value recorded for the ‘Postmortem Required’ check box. The listed values are populated as defined in the Area function. The validations are explained as follows:

Postmortem Required check box

Applicable Values

Selected

Values listed with area type set as

1. Other Area

2. Postmortem Area

3. Storage Area

Does not populate 'Release Area'

Cleared

Values listed with area type set as

1. Other Area

2. Release Area

3. Storage Area

Does not populate 'Postmortem Area'

 

  1. Location - Select the location using [?] function. Based on the 'Area' selected, the locations are displayed. If the 'Area' is not selected, then this field is disabled. A location can also be reassigned if required, in this case the old location will become vacant and it can be assigned to any other patient.

  2. Assign Date/Time - On selecting the Area, the current system date and time is displayed and disabled.

  3. RFID Tag No - Select the Radio Frequency Identification (RFID) tag number using [?] function.
