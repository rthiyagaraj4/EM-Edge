Enter the following details:

 

Diagnosis Category

  1. Diagnosis Category – Select the required diagnosis category from the drop-down list. The list values include Chronic, Dental, Maternity. By default, the value ‘Select’ is displayed. This field is mandatory.

  2. Patient Copay - Co-pay for Diagnosis Category (Chronic or Dental or Maternity) will be calculated based on the percentage value given in this field. User can enter numbes starting from 0 to 100. This field is mandatory.

  3. Patient Amount limit - Maximum patient copay amount limit will be calculated based on the amount value given in this field. User can enter only numeric characters. This field is mandatory.

  4. Gross/Net - Select the required value from the drop-down list. The default value is ‘Net’. The listed values include Gross, Net. Deductible will be calculated based on Gross or Net selected on this field.

  5. Effective From Date – Select the effective from date by clicking the calendar icon. This field is mandatory.

  6. Effective To Date - Select effective to date by clicking the calendar icon. This field is non-mandatory.

  7. Add Button - click [Add] button to add multiple records.

 

Drug Category

 

  1. Episode/Encounter - Select the required value from the drop-down list. The list values include Episode, Encounter, All, and Select. By default, the value ‘Select’ is displayed. This field is mandatory. If the user tries to navigate to other fields without selecting this field, then system prompts the validation as ‘Please select Episode / Encounter’.

  2. Type - Select the required type from drop-down list. The default value is ‘Select’. Based on the selected Episode / Encounter field, the values appear in the drop-down list.

• When Episode is selected, the drop-down list values include OP, EM, EX, All.

• When Encounter is selected, the drop-down list values include IP, DC and ALL.

• When All is selected, the drop-down list values include ‘ALL’.

  1. Payer Group – Select the required payer group from the lov. This field is mandatory. Note: Payer group drop-down list values include ** (All Payer group).

  2. Payer – Select the required payer from the LOV. This field is mandatory. Based on the Payer group selected, list of values will be displayed against the Payer. Note: Payer lov includes ** ( All Payer) in the list of values.

  3. Policy - Select the required policy from the LOV. This field is mandatory. The Policy LOV displays the policy associated to the Payer group and Payer. Note: ‘** (All Policy)  value will be available in the LOV.

  4. Drug Category - Select the required drug category from the drop-down list. The default value is ‘Select’. This field is mandatory. The drop-down list values include:

• Agreement Items

• Generic Items

• Brand Items (With alternate)

• Brand Items (Without alternate)

  1. Patient Copay - Select the required Co-pay for drugs (drug category) calculated based on the percentage value given in this field. User can enter numbers starting from 0 to 100. This field is mandatory.

  1. Patient Amount Limit (Applicable) – Select whether the patient amount limit is applicable or not. The drop-down list values include ‘Yes’ and ‘No’.  The Patient amount limit defined in the Policy Definition function is applicable against the corresponding record only when the applicable value is selected as Yes. If the value is selected as ‘No’,  patient amount limit will not be applicable. This field is mandatory.

  2. Gross/Net: Select the required value from the drop-down list. The default value is ‘Net’. The listed values include Gross, Net. Deductible will be calculated based on Gross or Net selected on this field.

  3. Effective From Date – Select the effective from date by clicking the calendar icon. This field is mandatory.

  4. Effective To Date - Select the effective to date by clicking the calendar icon. This field is non-mandatory.

  1. Add Button - click [Add] button to add multiple records.

 

Services and Items tab

  1. Episode/Encounter - This is a drop-down field with default dropdown value as ‘Episode’. This field is mandatory.

  2. Type - This a drop-down field with default value as ‘OP’. Only, the value ‘OP’ is available.  This field is mandatory.

  3. Payer Group – Select the required payer group from the LOV. This field is mandatory. Note: Payer group drop-down list values include ** (All Payer group).

  4. Payer – Select the required payer from the LOV. This field is mandatory. Based on the Payer group selected, list of values will be displayed against the Payer. Note: Payer lov includes ** ( All Payer) in the list of values.

  5. Policy - Select the required policy from the LOV. This field is mandatory. The Policy LOV displays the policy associated to the Payer group and Payer. Note: ‘** (All Policy)  value will be available in the LOV.

  6. Category – Select the required category from the drop-down list. The list values include Select, Primary and Specialized. The default value is ‘Select. The values are retrieved from Clinic category master in PAS. This field is mandatory. Without selecting this field, if user tries to navigate to other fields, then system will prompt the validation as "Please select Clinic category".

  7. Clinic/Procedure Unit – Select the required clinic or procedure unit by clicking the [?] icon. This field is mandatory.

  8. Include / Exclude criteria -  This drop-down field will have the value ‘Package Group’ by default.

  9. Service Details - On selecting the LOV, the ‘Package Group’ LOV will be invoked, where user will select the package group from the LOV. This field is mandatory. If the user tries to navigate to other fields, system will prompt the message as ‘Please select package group’.

  10. Referral – Select the required Referral from the drop-down list. The list values include Referral, Non Referral, and Both. The default value is ‘Select’. This field is mandatory.

  11. Patient Copay - Co-pay for services and items will be calculated based on the percentage value given in this field. User can enter numbers starting from 0 to 100. This field is mandatory.

  12. Patient Amount Limit - Enter the patient amount limit in the text field. Only numeric characters is allowed.

  13. Gross / Net – The drop-down list values include Net and Gross. The default value is ‘Net’. Deductible will be calculated based on Gross or Net to be decided based on the selection of this field.

  14. Effective From Date – Select the effective from date by clicking the calendar icon. This field is mandatory.

  15. Effective To Date - Select the effective to date by clicking the calendar icon. This field is non-mandatory.

  16. Add button - click [Add] button to add multiple records.
