Enter the following details to define Customer Base Price:

 

  1. Billing Class - Click [?] function to select and associate a Billing Class. The values defined through the 'Billing Class' function are listed. This is a mandatory field.

  2. Indicator - Select an Indicator from the list. The system-defined values are 'Price' and 'Factor'. The first record should have the indicator as 'Price'. This is mandatory. 'Factor' is valid for selection only for non-order catalog packages.

  1. Price - Enter the price in numeric format at which respective package can be offered for the Billing Class when the selected Customer and Customer Group is associated.

You cannot specify duplicate Customer Base Price for the same Billing Class.

  1. Round To - Enter the value in numeric format. You can select 'Round To' only when the 'Indicator' is selected as ' Price'.

  2. Round Indicator - Select a value from the round indicator. The system-defined values are 'Up' and 'Down'. Select 'Up' to round off the value to the highest nearing value specified for 'Round To'. Select 'Down' to round off the value to the lowest nearing value specified for 'Round To'. For Example, if the package value is 3,515 and the 'Round To' is set as 10 with the 'Round Indicator' as 'Up', the package price is rounded off to 3520.

  3. Deposit - Select the check box if deposit amount has to be accepted for the package. Clear the check box if deposit is not mandatory.

  4. Partial Deposit - Select the check box to define deposit requirement details wither amount or percentage wise. If you select this check box, the Link hyperlink will be enabled. Click this hyperlink to invoke the web dialog to define the partial deposit in terms of amount/percentage against each encounter for across encounter package ..

    Enter the following details:

    1. Encounter Number - Enter the encounter number in which partial deposit has to be collected.

    2. Partial Deposit Type - Select percentage or amount from the drop-down list. The value ‘Percentage’ is selected by default.

    3. Partial Deposit - Enter the partial deposit in a numeric format. The partial deposit will be calculated based on the selection of the Partial Deposit Type.

    If you define the partial deposit amount in percentage or amount, you should define the other partial deposits in the same.

    1. Partial Deposit Mandatory – Select this check box to mandate the partial deposit.

    2. Delete – Select radio button against the partial deposit to be removed.

    3. [Add] – Click this button to add a new in the web dialog to define new partial deposit rule.

    4. [Submit] - Click this button to save the data entered in the web dialog temporarily and web dialog gets closed. You can view the recently added or modified data on click of the Link hyperlink.

    If you close the web dialog without clicking submit button, the entered data will not be saved. On click of the Apply button in the Package Definition screen, the data entered in the web dialog can be saved permanently.

    1. Delete – Click this button to remove the partial deposit records against which the delete radio button is selected.

    The Delete button will be enabled if atleast one row is added.

  5. %/Amount - Select a value from the list. This field is enabled for selection only if the check box 'Partial Deposit' field is selected. You can choose to accept a specific amount as the deposit amount or a percentage of the base price as the deposit amount.

  6. Minimum Deposit Amount - Enter the minimum deposit amount that has to be accepted from the patient in numeric format. The partial deposit amount to be accepted from the patient for the package is calculated based on the value selected for 'Percent/Amount' and 'Minimum Deposit Amount'.

  7. Factor Applicable for Service Limit - Select the check box to define the factor available for service limit. This field is enabled for selection only if the 'indicator' value is selected as 'factor'.

  8. [Add] - Click [Add] to associate the details to the package.
