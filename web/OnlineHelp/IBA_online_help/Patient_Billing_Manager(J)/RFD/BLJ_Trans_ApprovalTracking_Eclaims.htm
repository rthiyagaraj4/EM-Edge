Enter the following details:

  1. Patient ID – Select the patient ID from the lookup.

  2. Patient Name – On selecting the patient ID, the patient name gets displayed.

  3. Encounter ID – Select the required encounter ID of the patient.

  4. Episode Type – Select the required episode type from the drop-down list. The list values include outpatient, Inpatient, Daycare, Emergency, and External.

  5. Payer Group – Select the required payer group from the lookup. This field is mandatory.

  6. Payer – Select the required payer from the lookup.

  7. Policy – Select the required policy from the lookup.

  8. Approval Doc Ref No - Select the approval doc ref no from the lookup. Approval doc ref no is the ID generated upon bills approval in the ‘Approval Bill-New’ function in 2T.

  9. Claim ID - Select the required claim ID from the lookup. The ‘Insurance Status’ will either be on Claim/Bill level or on service level.

  10. Statement Date From–To – By default, two weeks date range is selected. The ‘To Date’ defaults the system date. Date at which Bills were approved through the ‘Approval Bill-New’ function. This field is mandatory.

a. From Date cannot be Future date

b. Selected Date should not be greater than 31 days

  1. Claims/Services Level – Select the required value from the list. The default value is ‘Claims’.

  1. Set as Default – On selecting the check box, the chosen criteria is set as the default for next time so that user uses while accessing the screen.

  2. Insurance Status – Select the required insurance status from the list. The default value is ‘All’. The list values include Pending, Sent to Insurance, Approved, Partially approved, Rejected, Pending.

• Pending - Status provided by EM for records which are approved internally and invoiced but are not submitted to NPHIES yet.

• Sent to Insurance - Status provided by EM for records which are submitted to NPHIES, either as initial submission with no response yet or, Records that are resubmitted after queries received from NPHIES were addressed.

• Pended - Response received From NPHIES indicating that additional documents are required for Processing of Records

• Approved - Response received from NPHIES indicating that the Records are approved.

• Partially Approved - Response from NPHIES indicating that Records are partially approved.

• Rejected - Response from NPHIES indicating that Records are rejected

• All – Shows all records regardless of their status.

 The same Insurance Status terminology is used for claims and services, and these statuses are received from NPHIES and will be displayed as received.
