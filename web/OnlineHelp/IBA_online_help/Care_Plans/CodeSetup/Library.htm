Library

From this function you can Define the Libraries.

Care Plan Libraries are provided with sets of Problems/Diagnosis associated with the ‘Goals’, ‘Interventions’ and ‘Expected Outcomes’. You can define the attributes of the library such as the ‘Version’, ‘Patient Class’ and ‘Age Group’. You can maintain the versions of the Library to track the changes of the components.

Creating Libraries

From the CP Codes Setup > CP Code Setup, select the Library function.

The Library screen gets displayed.

  1. Click [Create] to invoke the Create screen.

  2. Enter the details in the mandatory (*) fields. <Related Field Description>

  3. Enter the details in the mandatory (*) fields in the Characteristics tab.

Version Details

Version Details:

Your can create and maintain versions of the library owing to the dynamic nature of the healthcare needs. At any point in time among the versions created for a library there can be ONLY ONE Final version, which is available for generating the care plans.

 

Enter the following details:

  1. Guidelines – Enter the Guidelines in alphanumeric format of 100 characters maximum length. This is a mandatory field.

  2. Status – Select the status from the list. The listed values are ‘Draft’, ‘Final’ and ‘Inactive’.

    • Libraries in  ‘Draft’ status are only listed in the function ‘Build Library’.

    • Libraries in ‘Inactive’ status are not available for the transactions.

    • Only those libraries with the status ‘Final’ are available in the transactions.

  3. Dependency Level – Select the dependency level from the list. Numbers 1-9 are listed. This is a mandatory field.

  4. Library Basis – Select the Library Basis from the list. The listed values are ‘Case Type’ and ‘Terminology Code’. This is a mandatory field.

Patient Class

Enter the following details:

  1. Patient Class – All the Patient Classes are displayed. The listed values are ‘Daycare’, ‘Emergency’, ‘External’, ‘Inpatient’ and ‘Outpatient’.

  2. Applicability – Select the check box against the Patient Class to which the library defined should be associated.

Age Group

Enter the following details:

  1. Age Group –The age groups are displayed from the Age Group function of Application Masters module.

  2. Sex Specific – The Sex applicable for the age groups as recorded in the Age Group function in the Application Masters module are displayed.

  3. Min Age – The Minimum Age applicable for the age groups as recorded in the Age Group function in the Application Masters module are displayed.

  4. Max Age – The Maximum Age applicable for the age groups as recorded in the Age Group function in the Application Masters module are displayed.

  5. Applicability – Select the check box to select the age group as Applicable. The check box provided in the header level enables you to select all the age groups listed as Applicable.

  1. Click [Apply] to save the changes or [Reset] to clear the entered data.

  2. Click [Query] to invoke the Query screen and to search for the existing records.
