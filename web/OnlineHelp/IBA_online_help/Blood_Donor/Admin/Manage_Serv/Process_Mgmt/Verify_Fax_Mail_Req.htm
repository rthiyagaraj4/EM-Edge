Verify Fax/Mail Request

From this function you can Verify or Reject the pending fax or mail requests made.

The Verify Fax/Mail Request function enables you to verify or reject the pending requests to be faxed or emailed. This function is used when you opt for the option of verifying the faxes and mails before they are sent. All the mails that are sent can be viewed in this function with the status as Pending. Select a mail/fax and choose the option to verify or reject.

Verifying the Fax/Mail Request

From the Administration > Manage Services > Process Management, double-click Verify Fax/Mail Request function.

The Verify Fax/Mail Request screen gets displayed.

  1. Enter the details in the (*) mandatory fields in the following tabs:

Fax

  1. Status – Select the Status form the list. The listed values are Verify, Reject and pending. Select Reject to reject the request. A Reject Reason window pops-up requesting the reason for rejection.

  2. Request Date – The date when the fax/mail request is raised gets displayed.

  3. Priority - The priority code entered while requesting the report is displayed.

  4. Facility – The name of the facility from where the request is raised is displayed.

  5. User ID – The ID of the logged in user who has raised the request is displayed.

  6. Workstation – The workstation from where the request has been raised is displayed.

  7. Module ID – The Module ID is displayed from which the request has been placed gets displayed.

  8. Report – The report type is displayed.

  9. Report File Name – The report file name is displayed.

  10. Destination – The destination code is displayed.

Mail

  1. Status – The value Pending is displayed by default. Select the appropriate value for the action to be done from the list. The listed values are Verify and Reject. When you select Reject, the Reject Reason window gets popped up and you have to enter the reason for rejecting the request. When you select verify, and click [Save], the mails/faxes are sent to the destinations specified.

  2. Request Date – The request date is displayed.

  3. Priority - The priority code entered while requesting the report is displayed.

  4. Facility – The name of the facility from where the request is raised is displayed.

  5. User ID – The ID of the logged in user who has raised the request is displayed.

  6. Workstation – The workstation from where the request has been raised is displayed.

  7. Module ID – The Module ID is displayed.

  8. Report – The report type is displayed.

  9. Report File Name – The report file name is displayed.

  10. Destination – The destination code is displayed.

  1. Click [Verify All] to save or [Clear Record] to clear the entered data.

  2. Click [Execute Query] or press [F8] to search for the existing records within the facility.
