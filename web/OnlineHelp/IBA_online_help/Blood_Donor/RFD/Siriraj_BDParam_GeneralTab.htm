Enter the following details to define BD Parameters:

 

Interface

  1. Interfaced to Blood Transfusion - Select the check box, if the Blood Donor module needs to be interfaced to the Blood Transfusion Services module. If the check box is cleared, you cannot transfer the blood from Blood Donor module to Blood Transfusion Services module.

 

Blood Units Transfer Parameters

  1. Add Blood Units to BT stock on Transfer? - Select the check box to transfer the all the Blood Units to BT Stock only by using the 'Transfer/Issue Blood Unit' function.

  2. Add Blood units to BT stock directly? - Select the check box to add the blood units directly to the BT Stock directly without using the 'Transfer/Issue Blood Unit' function.

  3. Fate Code (Transfer) – Click [List of Values] to select the Fate Code. System displays the value as default one, while transferring the Blood Units to BT Stock or disposing of the blood unit using Dispose Blood Units function.

  4. Override Screening? – Select the check box to override the screening of blood units in the BT stock. Clear the check box to screen blood units before adding them to BT Stock. This check box is introduced so that the screening of the blood units transferred from BD Stock to BT Stock can be ignored, as screening is already done before adding the stock to the BD Stock earlier. This rule is not applicable to the blood units received from the external laboratories.

 

Default Parameters

  1. Print Blood Bag label before adding to Stock? - Select the check box to print the blood bag labels before adding the blood units to the stock. A backend setup controls the printing of the blood bag labels before adding the blood units to the stock.

  2. Display Donor Notes? - Select the check box to display the donor notes in various functions. The donor notes are recorded in the Pre Donation function.

  3. Use ISBT128 Numbering for Blood Bags? - Select the check box to use ISBT128 numbering for blood bags.

ISBT 128 provides for unique identification of any donation worldwide. It does this by using a 13 character identifier built up from three elements, the first identifying the collection facility, the second the year and the third a sequence number for the donation.

  1. Length - Enter the Length value in numeric format of two digits maximum length to specify ISBT 128 numbering for blood bags.

  2. Print Blood Bag Label at Pre-Donation - Select the check box to print the blood bag label during pre-donation analysis in the Pre Donation function.

  3. Number of Labels - Enter numerical value to specify number of labels to be printed in numeric format of two digits maximum length. All the labels printed from the transactions are printed in the entered number.

 

Blood Letting

  1. Generate Unit Number - Select the check box to generate unit numbers automatically. If the check box is cleared, you have to enter the unit numbers while recording the donation.

 The system prompts you to enter the unit number in Collection Centre/Supplier or Products functions, if setup is not defined to generate unit number automatically in BD Parameter.

  1. Length - The Length of the unit number is displayed. It is a one time setup and you cannot modify it.  

  2. Prefix - The Prefix value for the unit number is displayed. It is a one time setup and you cannot modify it.

  3. Next Number - The value for Next Unit Number generation is displayed. It is a one time setup and you cannot modify it.

  4. Cancel Code For Letting - Click [List of Values] to select the fate code/cancel code for Letting. The selected value is set as the default value in the functions wherever it gets displayed.

  5. Number of Labels - Enter the value to specify Number of Labels to be printed after blood letting in two digits maximum length.

  6. Prepare Components based on Blood Bag Weight? - Select the check box, if the components are to be prepared as per the setup in blood bag weight rules. The blood bag weight rules are set up in Blood Bag Weight Rules function.

  7. Blood Bag Weighing Units - Enter the default blood bag weighing unit. The measurement of the blood bags are calculated based on this setup. The list displays the values defined in the Unit of Measurement function of the Application Masters module.

  8.  Mobile Items Entry Sequence - Select the entry sequence for mobile items from the list. The system defined values are Donation Details and Drive Items Register.

  9. Storage Location - Enter a valid Storage Location code for the default storage location of the donated blood units. The list displays the values defined in the Storage Location function. If there is no storage location selected here, you can select the storage location of your choice, whenever you want to store the blood units.

  10. Process Blood Unit Based On - Select either 'Date Range' or 'Batch' to process the blood unit based on 'Date Range' or 'Batch'. The listed values are system defined. If you select 'Date Range' the blood units are processed based on the date of donation and if you select 'Batch' the blood units are processed based on the donation batch.

  11. Reject Storage Location - Select the storage location from the list. The list of values displays the values defined in the Storage Location function. If there is no storage location selected here, you can select the storage location of your choice, whenever you reject the blood units.

  12. Fate Code (Autologous) - Select a default fate code to be used in the autologous type of transactions. The list displays the fate codes defined in the Fates function.

  13. Collection Centre (Pooled Unit) - Select the Collection Centre from the list to store the blood units after pooling. The list displays the values defined in the Collection Centre/Supplier function.

 

Lab Screening Parameters

  1. Collection/Supplier – Click [List of Values] to select the collection/supplier Centre code. The selected collection/supplier Centre is the default collection/supplier Centre for the transactions in Donor Management module.

  2. Source - Click [List of Values] to select the default Source code to be used in the transactions. The list displays the sources defined in the Source function of the Master Patient Index module.

  3. Screen Donor ID - Enter the Screen donor ID. The default Donor ID is used while sending the specimens for testing. The donor's name is masked and this ID is displayed while sending samples for test.

  4. Accept Initial Blood Type Miss match? - Select the check box, if the Initial Blood Type mismatch is to be ignored during the transactions in Blood Unit Screening Result Entry function. If the check box is cleared, you have to change Initial Blood Typing to the resulted Blood Group and Rhesus using the [Change Initial Blood Typing] in Blood Unit Screening Result Entry function.

  5. Consultant – Select the consultant from the list.

 

Reactive Donor Units

  1. Screen Blood Bag? - Select the check box to send the reactive donor’s blood unit for screening automatically. If the check box is selected, the blood units are sent for screening automatically, if the donor reacts negatively during the blood donation process.

  2. Screen Reactive Tests Only? - Select the check box to screen the tests, which are reactive.

  3. Cancel Code for Blood Bag - Click [List of Values] to select the cancel code for the reactive blood bag. The values selected are the default value in all the functions, where the blood bags are to be cancelled.

  4. Max. Time Limit for recall - Enter the maximum time limit to recall the reactive donor and select the unit. This field is enabled only if the ‘Recall Donor?’ check box is selected. Only the donors who are temporally deferred can be recalled.

  5. Recall Donor? - Select the check box to recall the donor for donation. If the check box is selected, on the date when the temporary deferral is over, a recall letter is automatically printed and sent to the reactive donor.

  6. Maximum Acceptable Bleeding Time - Enter the maximum acceptable donation time for a donor for a single blood donation and its unit.
