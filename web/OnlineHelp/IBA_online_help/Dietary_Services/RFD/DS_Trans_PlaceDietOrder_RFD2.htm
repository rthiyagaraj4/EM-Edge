Enter the following details to record the diet order for a patient:

 

  1. Diet Category – Select the Diet category from the list. The system-defined values are ‘Enteral’, ‘Normal Diet’ and ‘Therapeutic Diet’. This is a mandatory field.

  2. Date [From-To] - Click the Calendar function to set the date range for which the Diet Order is to be placed. The system date is displayed as the ‘From’ date, by default.

  3. Referred to Dietician – Select this check box to display the Refer to Dietician column against the diet order in Place Diet Order screen during order search.

  4. Meal Class - Select the Meal Class from the list. The values defined using the ‘Meal Class’ function gets displayed. This is a mandatory field.

  5. Diet Type - Select the Diet Type from the list. On selecting the Diet Category, the Diet Types associated to the Diet Category in ‘Diet Types’ function gets listed in alphabetical order . This is a mandatory field.

  6. Feed Type - Select the Feed Type from the list. The values defined using the ‘Feed Type’ function get listed.

  7. Texture of Diet- Select texture of diet from the drop down list. The diet is served to the patient based on the Texture. This is mandatory field.

  8. Delivery Instruction - Select the Delivery Instruction from the list. The values defined using the ‘Delivery Instructions’ function get listed.

  9. Feeding Instruction – Enter the Feeding Instruction in alphanumeric format to give instructions to feed the patient.

  10. Preparatory Instruction – Enter the Preparatory Instruction in alphanumeric format of 4000 characters maximum length. This indicates additional instruction given to the kitchen staff to prepare food.

  11. Refer to Dietician  - Select the ‘Refer to Dietician’ check box to refer the diet order to dietician. On selecting the check box, the mandatory fields in the Place Diet Order dialog are changed as non-mandatory.

  12. If you place the order without entering any one of the order details in the Place Order dialog, the application displays an alert message ‘Do you want to Refer to Dietician without Diet Order for Patient?’. Click [Yes], the application will allow you to refer the patient to the dietician. The Status for the blank order is highlighted in red and ‘Refer to Dietician’ is highlighted in blue.

  13. When the patient is referred to dietician, the dietician can clear the ‘Refer to Dietician’ check box and place the diet order.

  1. Full Nil By Mouth (NBM) – Select the check box to identify orders as ‘NBM with Diet order’ and ‘NBM without Diet order’. When you select the check box, all the fields except the ‘Nil By Mouth’ field get disabled. The system defaults the ‘From Date’ as ‘Current date and time’ and ‘To Date’ as 23:59 (as per the setup). You cannot modify the ‘From time’ and ‘To time’. By default, the check box is cleared.

 

Assign Diet Type If Different

The Meal Types get listed based on the Timetable for a particular date. You can update the diet type, if the diet type assigned to the patient is different.

 

Nil By Mouth

  1. Date [From-To] – Enter the period in dd/mm/yyyy format or select using Calendar function and select the date range for which the Diet Order placed not to be fed by mouth. When the ‘Full Nil By Mouth (NBM)’ check box is selected, the system defaults the ‘From Date’ as ‘Current date and time’ and ‘To Date’ as 23:59 (as per the setup). You cannot modify the ‘From time’ and ‘To time’. The remaining fields get disabled.

 

Enter the following details:

  1. Morning Tea – Select the Diet Type for Morning Tea from the list. Diet Types associated to the Diet Category and Age Group in ‘Diet Types’ function gets displayed. This is a mandatory field.

  2. Breakfast – Select the Diet Type for Breakfast from the list. Diet Types associated to the Diet Category and Age Group in ‘Diet Types’ function gets displayed. This is a mandatory field.

  3. Lunch – Select the Diet Type for Lunch from the list. Diet Types associated to the Diet Category and Age Group in ‘Diet Types’ function gets displayed. This is a mandatory field.

  4. Evening Tea – Select the Diet Type for the Evening Tea from the list. Diet Types associated to the Diet Category and Age Group in ‘Diet Types’ function gets displayed. This is a mandatory field.

  5. Dinner – Select the Diet Type for Dinner from the list. Diet Types associated to the Diet Category and Age Group in ‘Diet Types’ function gets displayed. This is a mandatory field.

Diet Orders

  1. Show - After entering the required details, select the appropriate order to be shown from the list. The values in the list are ‘All’, ‘Active Orders’, ‘Closed Orders’, ‘Future Orders’ and ‘Cancelled Orders’ based on which the user can view closed, active or future orders placed. Select the hyperlink and the following details are displayed for the set conditions:

  2. button gets displayed only against the active and future orders, so that you can edit, cancel and backdate the orders.

- Click  button to cancel the respective order, an alert message ‘Are you sure you wish to cancel this order?’ is displayed with with Reason for cancellation lookup and text box with OK and Cancel button. You can select reason using lookup or enter the reason in textbox. If you click Yes button, the order is cancelled and the Diet Type in the list will not be selectable.

- If you click  button against the active order where From date is past/current date and To date is blank/future date, the following alert message is displayed with Reason for cancellation lookup and text box with OK and Cancel button. You can select reason using lookup or enter the reason in textbox. ‘This order cannot be cancelled as it is active for the current date. Clicking Yes to this message will update the To date of the order as <<current date>>. Do you wish to continue?’ If you click Yes button, the current date will be updated in ‘To date’ field and X button will not be displayed for this order.

If an existing profile will be redundant due to addition/update of another profile, the message will be displayed with OK and Cancel buttons when Apply button is clicked. If you click OK button, the added/updated profile will be saved and the existing profile will  be automatically cancelled.

  2. Order Date – The date and time when the diet order is placed get displayed.

Select the check box against the diet type to define applicability of the diet type for the patient who has grace period for the day. The check box is displayed only when the order is placed for the discharged patient or deceased patient.
