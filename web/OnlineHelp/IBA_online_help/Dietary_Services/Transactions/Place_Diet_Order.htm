Place Diet Order

From this function you can Place diet orders for Inpatients and their Attendants in a particular Nursing Unit. You can Update the diet orders placed for a patient.

The Place Diet Order function is used to place diet orders from a ward to a kitchen for inpatients requiring meals. Patient’s food allergies can be viewed and updated, while placing orders from inpatient wards to ensure that the patient is not subjected to any allergic food items. The physician defines a Diet Order for a patient based on patient’s condition. Thus, the system creates a diet profile for the patient. Alternately, the dietitian can set up or review the diet profile defined by the physician for a patient and can update the Diet Type based on the patient’s condition. The diet orders are updated on applying the changes. Diet Orders are placed based on the age group of the patient.

Dietitian can specify the mode using which the patient has to be fed. If the patient cannot be fed by mouth for a certain period, the date range can be specified.. For example, when a patient has to undergo a surgery with empty stomach and the food item has to be given to the patient by any other means other than mouth. During this period, the Nil by Mouth option is used to indicate that the patient cannot intake any food by mouth. Special food requirements of the patient along with the serving times and frequency can be set up in the diet profile. Patients can be served with nutritional supplements along with their normal meal.

While placing a diet order for a patient, the Dietitian/Nurse/Clinician has to know the calories that a patient has to consume per day based on the selected food items. The total and subtotal calorie content of each food item is displayed for each meal type. Based on the calculation of the subtotals of the calories, the grand total is calculated and displayed. Based on the calculation of the total calorie content of Meals, Nutrient Supplements and Special Food items, the total amount of calorie content to be consumed by the patient on a day is displayed on the top.  

The system automatically alerts the physician if any item has Drug-Food interaction based on the setup done in ‘Drug Interaction’ screen and ‘Enable Drug Food Interaction Message For IP Patient’ check box selected in the DS Parameter for Facility screen

 

The status of the diet orders is displayed using the following colors:

                             

                  -  Order Placed for current Date

                   -  Order Modified

                  -  Order Pending for patient

                   -  Order Referred to the Dietician

                   - Order Placed For Mother Accompany Child

Placing Diet Orders

From the Transactions, select the Place Diet Order function.

The Place Diet Order screen gets displayed.

  1. Select required Nursing Unit and click [Search] to display all the patients of the selected Nursing Unit. Alternately, enter the query criteria to search for a specific patient with Patient ID. <Related Field Description>

  2. The patient records get displayed based on the search criteria specified.

  3. Select the check box against the Patient ID to place group diet order for multiple patients.

  4. Click [Group Order] to display the Group Order screen.

    Enter the following details to record the group diet order for multiple patients:

    Selected Patients

    The selected Encounter IDs and Patient IDs are displayed. Click the Patient ID hyperlink to display the Diet Order For Patient section. Click remove Patient ID from the worklist.

     

    For Group order, if there is item having drug-food interaction, Remarks hyperlink appears below the patient IDs and it is mandatory. Click Remarks hyperlink to enter the reason for overriding the alert.

    For Restricted items, the system disables the specific patient IDs automatically.

     

    Diet Order For Patient

    1. Spl. Food Items – Select this check box to select the special food Items for the patient. The special food items are food items that are not listed under the food items list.

    2. Food Dislikes - Select this check box to select the food dislikes for the patient.

    3. Diet Category – Select the Diet category from the list. The system-defined values are ‘Enteral’, ‘Normal Diet’ and ‘Therapeutic Diet’. This is a mandatory field.

    4. Date [From-To] - Click the Calendar function to set the date range for which the Diet Order is to be placed. The system date is displayed as the ‘From’ date, by default.

    5. Meal Class - Select the Meal Class from the list. The values defined using the ‘Meal Class’ function gets displayed. This is a mandatory field.

    6. Diet Type – Select the Diet Type from the list. On selecting the Diet Category, the Diet Types associated to the Diet Category in ‘Diet Types’ function gets listed. This is a mandatory field.

    7. Feed Type - Select the Feed Type from the list. The values defined using the ‘Feed Type’ function get listed.

    8. Delivery Instruction - Select the Delivery Instruction from the list. The values defined using the ‘Delivery Instructions’ function get listed.

    9. Feeding Instruction – Enter the Feeding Instruction in alphanumeric format to give instructions to feed the patient.

    10. Preparatory Instruction – Enter the Preparatory Instruction in alphanumeric format of 4000 characters maximum length. This indicates additional instruction given to the kitchen staff to prepare food.

    11. Click [OK] to modify the details.

     

     Assign Diet Type If Different

    The Meal Types get listed based on the Timetable for a particular date. You can update the diet type, if the diet type assigned to the patient is different.

    Enter the following details:

    1. Morning Tea – Select the Diet Type for Morning Tea from the list. Diet Types associated to the Diet Category and Age Group in ‘Diet Types’ function gets displayed. This is a mandatory field.

    2. Breakfast – Select the Diet Type for Breakfast from the list. Diet Types associated to the Diet Category and Age Group in ‘Diet Types’ function gets displayed. This is a mandatory field.

    3. Lunch – Select the Diet Type for Lunch from the list. Diet Types associated to the Diet Category and Age Group in ‘Diet Types’ function gets displayed. This is a mandatory field.

    4. Evening Tea – Select the Diet Type for the Evening Tea from the list. Diet Types associated to the Diet Category and Age Group in ‘Diet Types’ function gets displayed. This is a mandatory field.

    5. Dinner – Select the Diet Type for Dinner from the list. Diet Types associated to the Diet Category and Age Group in ‘Diet Types’ function gets displayed. This is a mandatory field.

     Nil By Mouth

    1. Date [From-To] - Click the Calendar function and select the date range for which the Diet Order placed not to be fed by mouth.

     Diet Orders

    1. Show - After entering the required details, select the appropriate order to be shown from the list. The values in the list are ‘All’, ‘Active Orders’, ‘Closed Orders’ and ‘Future Orders’ based on which you can view closed, active or future orders placed. The following details are displayed for the set conditions:

    2. Diet Type – The diet type for the selected patient is displayed.

    3. Period – The date range for which the diet order is placed gets displayed.

    4. Order Date – The date and time when the diet order is placed get displayed.

    5. Order By – The name of the user who has placed the order gets displayed.

    1. Select Date – Select the date from the calendar function.

    1. Diet Cycle –Select the Diet Cycle for the select Date from the drop-down list. The food items will be supplied based on the diet cycle. The Diet Cycles are defined in the  Menu Cycle function under the Admin Setup.

    Three tabs are available:

     

    Meals

    The Meals tab displays the meal details of the patient for the selected date. You can modify the meal details. Click the hyperlink Edit to display the Edit Menu with following details:

     

    1. Menu For Date – Select the date using the calendar function.

    2. Diet Cycle – Select the Diet Cycle from the drop-down list.

    3. Click [Go] to display the Items.

    4. Item – The Food Item is displayed. You cannot edit the data.

    5. Qty – The food item quantity is displayed. You can edit the quantity.

    6. Serve – Select this check box to serve the food item. Clear the check box to remove the food item from the diet order. This check box is selected by default.

    7. Click [+ADD ITEM] button to add food item.

    8. Select Item – Click [?] to select the Item

    9. Qty – Enter the quantity of the food items in numeric format

    10. Click [ADD] to update the food item.

    11. Click [Update] to modify the details.

    Spl. Food Items

    1. Frequency based – Select this check box to search the item types based on food serving frequency.

    2. From Date – Select the From Date using calendar. This is mandatory field.

    3. Meal Type – Click this hyperlink to display MealTypes dialog to select ‘Breakfast’, ‘Dinner’, ‘Afternoon Tea’ and ‘Lunch’.

    4. Item Type – Select the Item Type from the drop-down list. The Item Types defined in the Setup are listed out. This is mandatory field.

    5. Item – Click [?] to display the Items. The Items are displayed based on the selection of the Item Type. This is mandatory field.

    6. Qty – Enter the quantity of the item in the numeric format of six maximum characters limit. This is mandatory field.

    7. Frequency – Enter the serving frequency of the food item. This field is displayed only if the Frequency Based check box is selected. This is mandatory field.

    8. Period Between – Enter the interval time between the serve timings of the food items. This field gets enabled only if the Frequency Based check box is selected. This is mandatory field.

     

    This hyperlink Edit will be enabled only when any one of the Spl. Food Items is selected.

    Food Dislikes

    Click [?] to select the Items. The Items defined in the Setup are displayed. Click [Select] to display the Food Dislike. You can modify the food dislike items.

     

    On click of the icon in the tab, the tab will expanded and Edit hyperlink will be displayed. On click of the icon the tab will be rewound.

     

    The Group Order button is enabled only when multiple patients are selected. You can select all the check boxes by selecting the check box in the Spl. Food Items and Food Dislikes column. You cannot deselect all the check boxes manually. Atleast one check box is selected.

  5. Click the Patient ID Hyperlink to place an order. Place diet Order screen is displayed.

  6. You can place an order for a patient or for an attendant.

  7. Enter the details in the mandatory (*) fields. <Related Field Description>

  8. Click [Apply] to save a Diet Order or [Reset] to clear the entered details.

  9. Click [Close] to exit from the Place Diet Order screen and the control shifts back to the main screen to select another patient for whom the diet order has to be placed.

 

To place an order for attendant, select the check box ‘Attendant Orders Applicable’ in the ‘DS Parameter for Facility’ function. On selecting the check box, the radio button ‘Attendant’ gets enabled for selection.  If you try to place a diet order for a patient without entering any diagnosis information and click Apply, the following message will be displayed: ‘Diagnosis for the Patient is not entered’.
