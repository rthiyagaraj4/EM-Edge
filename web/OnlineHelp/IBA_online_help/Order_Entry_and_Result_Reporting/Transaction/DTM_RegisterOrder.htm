Register Order

From this function you can Register an Order.

This function is used to register an order having status as ‘Ordered’ to mark the acknowledgement for further processing of the order in the resulting department. This function is used to register orders across categories and this is based on access rights defined against the logged in practitioner. An order, which is ordered, is allowed to be resulted only after registration and this function forms an important step in processing the order and accepting it within the reporting department. This function also allows redirecting orders and sending out external orders. Once results are available from external source, this function is used to acknowledge the receipt of such external results.

 

If the ‘Split  Unbilled orders and create New ID during Register Order’ check box is selected in OR Parameter for Facility screen, the system splits the unbilled orders and creates new order ID during registration. This is applicable only for orders registered from Task List in CA module.  Color indicator s appear in the Register Order screen.

 

Unbilled – Orders to billed/settled before registration are highlighted in blue

Fully/Partially billed or paid orders – Orders which are paid, fully or partially are highlighted in red colour

       No colour indicator is displayed for non-billable orders.

Registering Orders

From the CA Transactions > Clinician Access > Desktop Menu > Pending Activities > Tasks, select the Register Order function.

The Register Order screen gets displayed.

  1. Click [Search] to display orders for registration for default date range or enter the query criteria and click [Search] to display specific orders. Click [Clear] to clear the displayed data.

    1. Order Type - The Order Type is displayed.

    2. Order ID -  The Order ID is displayed.

    3. Order Date -  The date and time of placing Order is displayed in the date format of dd/mm/yyyy hh:mm.

    4. Priority - The priority of the Order is displayed. The values are Routine, Stat and Urgent.

    5. Order Catalog -  The description of Order Catalogs are displayed which could be one or many depending on the total number of catalogs grouped in a single order. The Order Catalog is a hyperlink which allows invoking functions based on access rights of the logged in user.

    6. Patient Details -  The Patient and Encounter details are displayed. The Patient details displayed are Name of the Patient, Gender, Age and Patient ID. The Encounter details displayed are the Encounter ID of the Patient and Location.

    7. Select - A check box is provided against each order. Select the check box to select the orders for registration.

    8. Charge Dtls – Click View/Edit hyperlink to view and record charge details.

  2. Click the Order Catalog hyperlink and select the required option from the list to perform functions based on the selected Task Status. All the pharmacy orders are displayed with dosage and frequency details.

  3. Click View/Edit hyperlink under Charge Details to view and record charge details of the patient.

  4. Select the check box against the required orders to register the orders and click [Register] to confirm registration or [Clear] to clear the selected orders.
