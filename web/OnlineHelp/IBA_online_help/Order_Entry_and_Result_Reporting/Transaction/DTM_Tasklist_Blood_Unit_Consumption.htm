Blood Unit Consumption

  1. Blood Products that are cross matched from Blood Transfusion Services module in Allocate/Xmatch/Issue function are displayed with the status 'Reserved Not Requested'.

  2. Select the Status from the list box and click [Go]. The list box displays Reserved Not Requested, Reserved Requested, Issued and Transfused. Click Filter hyperlink to filter the search.

  3. Select the Unclassified check box to query the unclassified list. This check box is applicable only for the task 'Blood Unit Consumption'.

  4. The request details are displayed on searching.

  5. Click on the Request hyperlink (Order Catalog) to open a menu option. Select the respective menu option to perform the transaction and to view the details.

Status - Reserved Not Requested

  • Request/Print - Click Request/Print to request for blood transfusion. A message 'Print Request has been sent to the Server' is displayed. Click [OK] and the status is changed to 'Reserved Requested'. The status is changed to 'Reserved Requested'. Click the status hyperlink to view the request details.

  • View Order Details - Click View Order Details hyperlink to view the Order Details.

  • Product Details - Move the mouse over the Product Details hyperlink to view the Product Details.

Status - Reserved Requested

  • Reprint – Click Reprint to resend the request. . A message ‘Print Request has been sent to the Server’ is displayed.

  • View Order Details – Click View Order Details hyperlink to view the Order Details.

  • Product Details – Move the mouse over the Product Details hyperlink to view the Product Details.

Status - Issued

Blood Products that are issued from Transfusion Management module in Allocate/Xmatch/Issue function are displayed with the status 'Reserved Not Requested'.

  • Record Consumption - Click Record Consumption to record blood consumption. The Record Consumption screen gets displayed.

    1. Start Date/Time – Select the Start Date/Time from the Calendar function. This is a mandatory field.

    2. End Date/Time – Select the End Date/Time from the Calendar function. This is a mandatory field.

    3. Checked by Practitioner – Select the name of the practitioner who overviews the blood consumption procedure from the [?] function. This is a mandatory field.

    4. Transfused by Practitioner – Select the Practitioner who transfuses the blood unit from the [?] function. This is a mandatory field.

    5. Practitioner ID – Select the Practitioner ID from the [?] function.

    6. Reaction – Select the Reaction from the [?] function. This is a mandatory field.   

    7. Transfusion Remarks – Enter the Transfusion Remarks in alphanumeric format.

    8. Partial Transfusion – Select the check box, if the transfusion is a partial transfusion. On selecting the check box, the Reason field gets enabled.

    9. Reason – Select the reason for Partial Transfusion from the list. The listed values are created in the Reason function in the Blood Transfusion Services module.

    10. Click [Apply] to transfuse the blood unit or [Cancel] to stop the procedure.

  • Antibody - Click Antibody to view the Patient Antibody details.

  • Transfusion History - Click Transfusion History to view the Transfusion History.

  • View Order Details - Click View Order Details to view the Order Details.

Status - Transfused

Blood Products that are transfused from Blood Transfusion Services module in Allocate/Xmatch/Issue function are displayed with the status 'Reserved Not Requested'. Click the status hyperlink to view the transfusion details.

  • View Order Details - Click View Order Details to view the Order Details.

  • Product Details - Move the mouse over the Product Details to view the Product Details.
