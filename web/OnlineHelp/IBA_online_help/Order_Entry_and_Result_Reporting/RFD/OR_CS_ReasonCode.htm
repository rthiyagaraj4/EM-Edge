Enter the following details:

 

  1. Type - Select the Type from the list. The values displayed are Order, Cancel, Hold/Discontinue, and Amend and Not Done.

• On selecting the Type as ‘Order’ the check boxes against the Order Categories are masked and disabled. This indicates that the reason for an order is applicable for all the Order Categories.

• On selecting the Type as ‘Cancel’ or ‘Hold/Discontinue’, the check boxes against the Order Categories are enabled. You can select the check boxes against the selected Order Categories to indicate that the reason is valid for the selected Order Categories. This is a mandatory field.

• On selecting the Type as ‘Not Done, the check boxes against the Order Categories Treatment Procedures and Patient Care are only enabled. You can select the check boxes against the selected Order Categories to indicate that the reason is valid for the selected Order Categories

  1. Code - Enter the Code for the reason in alphanumeric format of four characters maximum length. This is a mandatory field.

  2. Reason - Enter the Reason in alphanumeric format of 255 characters maximum length. This is a mandatory field.

  3. Enabled - Select the check box to enable the reason code in the transaction functions. The check box is selected, by default.

  4. Order Category - The Order Categories are displayed. The categories are system defined.

  5. Applicable - Select the check box against an Order Category to indicate that the defined reason code is applicable to the Order Category.
