Patient Name Derivation Logic

Enter the Patient name Derivation Logic in alphanumeric format of 100 characters maximum length. The logic defined is used to build patient name during Patient Registration. For example, a Patient, ‘Mr. Tom Allen’ has three parts in his name, which are ‘Mr.’, ‘Tom’ and 'Allen’. Define the logic to derive each part of the name and combine them to form a full name. The derivation logic for the name MR. Tom Allen Solly is “PR(MR)+<.>+1N(Tom)+<>+ 2N(Allen)+<>+ FN(Solly)”, where ‘PR’ stands for Prefix, ‘1N’ stands for First Name, 2N stands for Second Name, <> for space and ‘FN’ stands for Family Name. Name derivation logic is based on the parts applicable to the site.

  1. For English - Enter the ‘Patient Name Derivation Logic’ in alphanumeric format of 100 characters maximum length For English.

  2. For Loc Lang - Enter the ‘Patient Name Derivation Logic’ in alphanumeric format of 100 characters maximum length For Local Language. This field is applicable provided 'Accept Patient Name in Local Language' is selected.

  3. Default Patient Name in English – Enter the ‘Default Patient Name’ that is to be displayed in English.

  2. For Loc Lang - Enter the ‘Patient Long Name Derivation Logic’ in alphanumeric format of 280 characters maximum length For Local Language.

Click the hyperlink to record and sign the ‘ Death Register Form’ mandatorily  for deceased patients. This is applicable for users in AAKH.

  12. UHID Valid  For - Enter the number of days in the ‘UHID Valid For’ text box. Based on the entered number of days, the system checks the validity of the UHID/Patient ID. This is applicable for users in AMRI.

Total no. of days = Current System Date – UHID/Patient ID registration Date or UHID renew date

• If the total no of days does not cross the ‘UHID Validity For’ stipulated days then at the time of generating a new encounter the system allows you to continue the transaction without displaying an any error message.

• If the total no of days crosses the ‘UHID Validity For’ stipulated days then at the of generating a new encounter for an OP, AE and IP, the system displays an error message ‘Patient UHID has been expired. Do you want to renew?’ with OK and Cancel buttons.

 

Mandatory fields during Registration

  1. Mandatory fields during Registration – Displays the following options: Race, Religion, Country, Region, Mobile number, Email ID, Area, Town, Patient Category, and Postal . Select the check box against the fields that you want to be mandatory while registering a patient.

 

Auto Cancel Future Appointments/Waitlists/Bookings/Referrals of Deceased Patients

  1. Consider Patients deceased before [ ] Days – Specify the period to consider the deceased patients for cancelling the future appointments, IP booking, waitlist and referral. Enter a numeric value of maximum three digit length i.e., from ‘0’ to ‘999’. If the value zero is entered, the system considers the patients deceased till the current day for auto closure. This field is blank by default.

  1. Reason for Cancellation – Select the Reason for Cancellation from the list. The reason defined in the Application Master for auto cancel of appointments/waitlists/bookings is listed for selection. This field becomes mandatory only when value is entered in the ‘Consider Patients deceased before [ ] Days’ field.

  2. Consider Deceased Patients Released only through Mortuary – This field is available only when Mortuary Management module is available in the site. The system validates the deceased patients released through Mortuary for auto closure only when this check box is selected and a value is entered in the ‘Consider Patients deceased before [ ] Days’ text box.

 

Default

  1. Nationality – Select the Nationality from the list. The selected Nationality is made default during Patient Registration and other functions. The listed values are defined in the ‘Country’ function.

  1. Race - Select the Race from the list. The selected Race is made default during Patient Registration and other functions where it is being referred. The values are listed from ‘Race’ function.

  2. Patient Language - Select the Patient Language using [?] function. The selected Patient Language is made default during Patient Registration and other functions. The listed values are defined in the Patient Language function.

  3. Patient Religion - Select the Patient Religion using [?] function. The selected Religion is made default during Patient Registration and other functions. The listed values are defined in the Religion function.

  4. Patient Search tab – Select the Patient Search tab value from the list. The listed values are ‘Basic’ and ‘Advanced’. The selected values get defaulted in all patient search criteria.

 

Additional Fields tab

  1. Legend (Field1-5) – Enter the legend for the field in alphanumeric format of 15 characters maximum length. The defined field Legends appears in respective section selected for the legend while registering a patient.

  1. Field Size - Enter the size of the field in numeric format of two digits in maximum length.

  2. Section – Select the Section from the list. The Sections defined are Demographics, Contact Addresses, Related Contacts and Others The defined field is displayed under the specified section in the Register Patient function.
