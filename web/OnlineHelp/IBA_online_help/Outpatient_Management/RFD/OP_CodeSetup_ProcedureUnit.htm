Enter the following details to create a Procedure Unit record:

 

  1. Code - Enter the Location code in alphanumeric format of four characters maximum length. A location code is to identify the Location and has to be unique and cannot be changed after creation. This field is mandatory.

  2. Type - Select the required Location Type from the list. The list displays OT (All), OT (Specific) and Procedure Unit. This field is mandatory.

• For Brunei users, a drop-down list appears next to Type field. The categories created in ‘Clinic Category’ master appear in the drop-down list. This field is non-mandatory. The default value is ‘Select’.

  1. Long Description - Enter the long description of the Location in alphanumeric format of 30 characters maximum length. This field is mandatory.

  1. Short Description - Enter the short description of the Location in alphanumeric format of 15 characters maximum length. This field is mandatory.

  2. Physical Location - Enter the Physical Location of the Procedure Unit in alphanumeric format of 40 characters maximum length. A Physical Location is to identify the exact Location of the Procedure Unit.

  3. Enabled - Select the check box to enable the record and clear this check box to disable the record.

  4. Applicable to Radiology – Select the Applicable to Radiology check box to create Procedure Unit for booking radiology appointments. This check box is cleared, by default. If the check box is selected:

    • Primary Resource Class field is defaulted as ‘Equipment’ and disabled.

    • Check in at Identification field is defaulted to ‘Resource’ and disabled.

    • All the fields in the MR Section/ File Storage are disabled.

 

Procedure Unit characteristics can be entered under two tabs, namely, ‘Characteristics’ and ‘Others’ tab.

 

Characteristics Tab

 

Working Days

  1. Working Days - A Location can function on all or specific days of the week. Select the appropriate check boxes to indicate the Location working days. The Visit Registration and appointment processes are used for a Location only on its working days. The default working days are selected based on the day of week defined in System Manager module.

 

Operating Time

  1. Start – Enter the operating start time of the location in time format of hh:mm. This field is optional.

  2. End - Enter the operating end time of the location in time format of hh:mm. This field is optional. Operating End time has to be greater than Operating Start Time.

  3. Base Slot/Slab Time – Base slot/slab time is used to identify the minimum unit of time frame used to book an appointment where multiple Resources are involved by using multiples of the base time defined. Enter the base slot/slab time in time format of hh:mm. This field is optional.

 

Controls

  1. Allow Visit Registration – Select the check box to Allow Visit Registration for the defined Location.

  2. Allow Walk-in – This field is enabled on selecting the ‘Allow Visit Registration’ check box. Select the check box to allow walk-in’s to be entertained to the defined location, or clear the check box.

  3. Allow Appointment - This field is enabled on selecting the ‘Allow Visit Registration’ check box for Procedure Units with level of care set to ‘Ambulatory’. For locations with level of care set to ‘Emergency’ the field is disabled. Select the check box to allow appointments for the defined Location. If the check box is cleared, the time table records cannot be set up for this Location in Appointment Scheduling module and the appointments cannot be booked.

  4. Allow Referral - Select the check box to indicate if referrals are allowed in this Location. This field is enabled on selecting the ‘Allow Visit Registration’ check box. Most locations allow visit registration only on producing a referral identification number. If the check box is cleared, the location does not support registration of visits for referral patients.

  5. Allow Non-Referral - This field is enabled on selecting the ‘Allow Visit Registration’ check box. Select the check box to indicate if non-referrals are allowed in this Location.

  6. Age Group - Select the Age Group of patients who are eligible to receive the services provided from the list of valid values. If this value is set up, the Visit Registration process restricts the patients based on their age and gender depending on the corresponding Age Group parameters set in ‘Other Master Setup’ of Application Master module.

  7. Diagnosis Not Mandatory During Check Out -  Select the check box to make recording of patient diagnosis as non-mandatory during OP check out in spite of Diagnosis is configured as mandatory for OP patient in MDS for patient class setup. By default, this check box is cleared and in enabled mode. You can select it if required. This check box is available only for users in Selayang.

  8. Level Of Care - Select the Level of Care from the list of valid values. This defines the level of care provided by the Procedure Unit. The listed values are ‘Ambulatory’ and ‘Emergency’.

If the Level of Care is set to ‘Ambulatory’, the applicable Visit Type Indicator check box, namely, ‘First’, ‘Follow up’, ‘Routine’, ‘Series’ and ‘Consult’ are selected by default. Clear the required field if necessary.

If the Level of Care is set to ‘Emergency’, the ‘Emergency’ check box in Applicable Visit Type Indicator is selected by default.

 

Others Tab

 

Acct Department/Service

  1. Specialty - Select the Specialty from the list of valid values. All the Specialties created in Application Masters module with enabled check box selected are listed. This field represents the Specialty, which is applicable for the Procedure Unit. This field is mandatory.

  2. Vital Signs Battery – Select the Vital Signs Battery from the list. All the Discrete Measure of type ‘Vital Signs Battery’ setup in Application Master module are displayed in the list. The Vital Sign Battery associated with the Procedure Unit is used to record the Vital Signs of the patients arriving into the Procedure Unit. This field is disabled, by default.

Vital Signs Battery in Outpatient Management module is set at three different levels, namely, ‘Procedure Unit Setup’, ‘Visit Type for Location Setup’ and ‘OP Parameter Setup’. Vital Signs Battery set up is not mandatory while defining the ‘Procedure Unit’ and ‘Visit Type for Location’ but is mandatory while defining ‘OP Parameter’. The system first checks for Vital Signs Battery in the ‘Procedure Unit setup’.  If it is not defined, the Vital Signs Battery Set in ‘Visit Type for Location’ is set as default. If Vital Signs Battery is not defined in Visit Type, the Vital Signs Battery set in ‘OP Parameter’ is displayed as default.

  1. Account Department - Select the Account Department for the Location from the list. The list of Account Departments displayed is limited to the departments of the logged in facility as set in ‘Department in facility’ of Application Masters module. This field is mandatory.

  2. Section – Select the Section for the Location from the list. All the sections associated to the selected Accounting Department in Application Masters module are listed in the list.

  3. Service - Select the Service from the list of valid values. This field represents the Medical/Ancillary Service that is applicable to the Location. The list of Services displayed is limited to the Services offered by the selected department of the logged in facility. This field is mandatory.

  4. MR Section Type – Select the MR Section Type from the values displayed, namely, ‘Central’ or ‘Departmental’. This field is enabled only if File Maintenance is set to ‘Multiple’ in MP Parameter (Logged in facility) of Master Patient Index module or if File Maintenance is set to ‘Single’ in MP Parameter of Master Patient Index module, this field is disabled. The value ‘Departmental’ has to be selected when the Location maintains an exclusive MR Section for the patient records and the value ‘Central’ has to be selected when the Location uses a common centralized MR Section. The MR Section value is displayed based on the value set at the facility level and if these values are not set at the facility level, the values set in MP Parameter are defaulted.

  5. MR Section – Select the MR Section from the list of valid values to be associated to the Location. This is enabled only if MR Section Type is selected as ‘Departmental’. This field represents the Medical Records Section of the facility associated to the Procedure Unit. The list of values displayed is limited to the MR Sections of the logged in facility.

  6. File Storage Location – Select the File Storage Location to request for files from the list. These values are displayed only if the Medical File Management module is installed. On selection of MR Section Type as ‘Central’ and all those File Storage Locations of identity is Procedure Unit and belongs to the logged in facility are displayed. On selection of MR Section Type as ‘Central’ and all those File Storage Locations of identity is department and belongs to the logged in facility are displayed.

  7. Default File Type – Select the applicable File Type from the list of values. The list of values is populated from the File Type function defined in Master Patient Index module and is applicable when File Maintenance is opted for single file maintenance in Master Patient Index module.

  8. Auto File Request - Select the check box to automatically request for files from Permanent MRD on Visit Registration to the defined Procedure Unit, or the files has to be manually requested.

 

Applicable Visit Type Indicator

  1. Emergency/First/Follow-up/Routine/Series/Consult – Select the check boxes to indicate whether each of these Visit Types are applicable for the Location. If ‘Level of Care’ is selected as ‘Emergency’, the ‘Emergency’ Visit Type check box is selected by default. Select the other Visit Types applicable for the Location. If the ‘Level of Care’ is set as ‘Ambulatory, all Visit Types except the ‘Emergency’ Visit Type are selected by default or select the ‘Emergency’ Visit Type if applicable for the Location.

 

Applicable Patient Class

  1. Outpatient/Inpatient/Daycare/External - Select the check boxes to indicate whether of the patient classes are applicable for this Location.

 

Resource Privileges

  1. Primary Resource Class/Type - Select the Resource class and Resource Type from the list. The values populated for Resource Class are ‘Practitioner’, ‘Room’, ‘Equipment’ and ‘Others’ for Locations, which is based on Ambulatory Level of Care, and value is ‘Practitioner’ for Locations, which is based on Emergency Level of Care. Based on the value selected in Resource Class, the values in Resource Type are filtered and populated from Application Masters module.

  2. Open to All Resources - Select the check box to indicate the Procedure Unit is open to all primary resources of the facility. If the check box is not selected, specific resources have to be associated to the Procedure Unit using appropriate function. This field is applicable when Resource Class is set to ‘Practitioner’. If level of care is set to ‘Emergency’, the check box is selected and disabled.

  3. Check in at Identification – Select the required value from the list. If Primary Resource Class is ‘Practitioner’, the values are ‘Non-identifiable’ and ‘Resource’. If Primary Resource Class is ‘Room’, the value is set to ‘Resource’. If value is set to ‘Resource’, the same value has to be identified at the time of Visit Registration. Otherwise, the resource identification becomes optional at the time of Visit Registration and the resources are identified before the consultation process is started. This field is mandatory. If Level of Care is set to ‘Emergency’, the value Non-identifiable is defaulted and disabled.

  4. Select Unassigned Patients - Select the check box to indicate whether unassigned patients can be selected from the Patient Queue for consultation process. Based on the value selected, the attending Practitioner for the encounter is automatically assigned based on who selects the patient. If Level of Care is set to ‘Emergency’, the check box is selected and disabled.

  5. Allow Consultation On – Select the required value from the list. The values displayed are ‘Checked-in’ and ‘Arrived’. Based on the value selected the patient can be selected from the queue for consultation by the Practitioner either on checking in or on arriving at the Procedure Unit. The patient name is listed in the Clinician Access queue based on the value selected.

 

Queue Numbering Controls

  1. Numbering Applicable – Select the check box to assign numbering for the Procedure Unit. Numbering is set according to the setup made in Queue Numbering Control function.

  2. Calculate Wait Time based on Status – Select the required value from the drop-down list. The list values include Checked-In and Arrived At Nursing Station. Based on the selected value, the patient Waiting Time is calculated in the Manage Patient Queue. By default, the value ‘Select’ is displayed. The ‘Calculate Wait Time based on Status’ configuration done in OP Parameter will apply. If the setup is done in Procedure Unit screen, this setup will take precedence. At any point of time, the wait time will be calculated based on the latest configuration.

  3. Contact No – Enter the contact number of the procedure unit in alpha numeric format of maximum 15 characters. Special characters +(Plus) and - (Hyphen) are only allowed.

 

Patient Instructions tab

 

  1. Serial No - Enter the Serial Number in numeric format of two digits maximum length. You can record a maximum of 10 Serial Numbers. This is a mandatory field.

  2. Instruction Description - Enter the Instruction Description in alphanumeric format of 200 characters maximum length or select the Instruction Description using [?] function. This is a mandatory field.

  3. Instruction ID -The corresponding Instruction ID is displayed on selecting the Instruction Description.

Click [Add] to display the selected Patient Instruction to the list at the bottom of the screen.

  1. Default - Select the check box to default the selected patient instructions while booking an appointment for the selected clinic.

Click on Serial Number hyperlink to delete the record. Click [Clear] to clear the selected details. Click [Delete] to delete the record from the list.
