Enter the following Details:

 

In the case of an already registered patient, you can enter the following details in ‘Enter Patient ID’ screen, to establish the Patient ID link to the selected referral patient record:

  1. Patient ID – Enter the Patient ID in alphanumeric format of maximum length as specified in the MP Parameter. You can query for the patient details by invoking the [?] function. On entering the relevant details in the [?] function, the result screen gets populated with patient details that satisfy the entered query details. You can select the patient by clicking on the hyperlink of the patient. The control now shifts back to the ‘Register Patient Referral’ screen with the patient details populated.

On entry of a valid Patient ID, system displays the details of the patient like Patient Name, National ID, Alternate ID, Other Alternate ID (if any), Age, Date of Birth, Gender, Residency, Nationality and address details as part of ‘Register Patient Referral’ screen. The user cannot change these displayed values.

If the patient is not registered, the user has to enter relevant details to record patient referral details.

  1. National ID - Enter the National ID in numeric format as entered while registration. The ‘Prompt’ specified for the National ID in MP Parameter appears as the field legend. The length and format as specified in MP Parameter has to be adhered.

  2. Alternate ID Type(s) - Enter Alternate ID number(s) in the format and length as specified in MP Parameter. Based on the MP Parameter setup, system displays the Alternate ID Type description(s).

  3. Other Alt ID – Select the required Alternate ID from the list. The listed values are defined in the function ‘Alternate ID Type’ of MP module. On selecting the required Alternate ID type, enter the ID in alphanumeric format of 20 characters maximum length. The field ‘Other Alt ID’ is displayed provided ‘Accept Other Alt ID’ is set in MP Parameter (Alt ID checks tab).

The field names are displayed based on the legends for Alternate ID defined in the MP Parameter.

  1. Patient Details – The multi parts of Patient Name are the Prefix, First Name, Second Name, Third Name, Family Name and Suffix. Based on the setup in MP Parameter (Legal Names), the multi parts of patient name are displayed. The displayed titles and order of appearance are specified in the MP Parameter function. On entering the patient’s name, the system builds the patient name based on the derivation logic defined in MP Parameter and displays them in the text area after Patient ID value.   

  2. Birth Place - Enter the Birth Place in alphanumeric format or click [?] function to search for the required Birth Place. All the 'Birth Place' codes created in Master Patient Index module with 'enabled' check box selected are displayed.

  3. Gender - Select the Gender of the unregistered Patient from the list of valid values. The valid values are Male, Female and Unknown. This field is mandatory.

  4. Age - Enter the Age of the Patient in years, months and days, if date of birth is not known. If age is entered, the system automatically calculates the date of birth and displays it in the ‘Birth Date’ field. Either Age or Date of Birth can be entered. On entering one of them, the other field will be read-only and disabled.

  5. Birth Date – Enter the birthplace of the Patient in alphanumeric format of 30 characters maximum length, or use the [?] function to select the required birthplace. If National ID is entered, the birthplace is displayed based on the birthplace logic defined in the routine associated with the National ID. If the required birthplace is not available in the search list, enter the birthplace in alphanumeric format of 30 characters maximum length in the text box provided next to the [?] function.

  6. Residency - Select either ‘Citizen’ or ‘Non-citizen’. ‘Citizen’ is selected, by default. If National ID is entered for the patient, the ‘Citizen’ is selected and disabled, or both the values are available for selection.

  7. Status - Select either ‘Legal’ or ‘Illegal’ as citizen status. ‘Legal’ is selected, by default. If the patient is a ‘Citizen’, ‘Legal’ is selected and disabled. If the patient is a ‘Non-Citizen’, the status can be either ‘Legal’ or ‘Illegal’.

  8. Nationality - The default Nationality set in MP Parameter is displayed and disabled, if National ID is entered, select the required value using the [?] function.    

  9. Race - Select the Race from the list. This field is mandatory. The listed values are defined in ‘Race’ function.

  10. Ethnicity – Select the Ethnicity from the list. The listed values correspond to the selected race. This field is mandatory.

  11. Contact Details - Click on the Contact Details button to enter the contact details for an unregistered patient and a screen gets displayed, or the details captured while patient registration gets displayed for a registered patient.

 

Referred From

  1. Referral Letter No – Enter the referral letter number in alphanumeric format of 30 characters.

  2. Received Date – Enter the received date in dd/mm/yyyy format along with the time in hh:mm format. This date is mandatory. The entered date cannot be less than the Referral date and greater than the system date. The system date is displayed, by default.

  3. Referred Date – The default date displayed is the system date. The user can enter the appropriate date in dd/mm/yyyy format or click on the Calendar function to select the required date. The Referred date cannot be greater than the system date. This field is mandatory.

  4. Preferred Date – Enter the Preferred Date in dd/mm/yyyy format or select the required date using the Calendar function. The preferred date must be greater than referral date. Preferred date refers to the date the patient prefers for consultation.

  5. Reason for Referral – Select the reason for referring from the list of valid values. All the reasons defined with referral check box selected in ‘Other Master setup (Reason)’ of Application Masters  module are displayed.

  6. Referral Type – Select the Referred Type from the list of valid values. The displayed values are Login Facility, Enterprise and External. This field is mandatory. These values are pre-defined.

  7. Healthcare Setting Type – This field gets enabled on selecting the Referral Type as ‘Enterprise’ or ‘External’, or this field gets disabled on selecting the Referral Type as ‘Login Facility’. Select the required healthcare setting where the patient is referred from, from the list. All the healthcare setting types created in System Manager module get listed. This field is mandatory.

  8. Referred To – Select the referral source from the list of valid values. The list of values displayed corresponds to the value selected for the field ‘Referred Type’. If the Referred type is ‘Login Facility’, only the logged in facility is displayed. If the Referred Type is ‘External’, all the external facilities are displayed. If the Referred Type is ‘Enterprise’, all the facilities of the enterprise are displayed. This field is mandatory. Place the mouse on the    icon to display the long name of the External or Referral Source recorded in ‘External/Referral Source’ function of Application Masters module. The  icon appears on the screen if the long name for the External Referral Source is recorded, in ‘External/Referral Source’ function of Application Masters module.

  9. Specialty – Select the required specialty from the list. All the specialties associated to the referred source are displayed.

  10. Location - Select the location type and location from the list. If the Referred Type is ‘Login Facility’ or ‘Enterprise’, two values are to be specified for location field. The first value for the location is the care location type, which are ‘Clinic’, ‘Nursing Unit’, ‘Daycare’ or ‘Procedure Unit’. Depending on the care location type selected, corresponding values are populated in the list next to the selected care location type. You can select the required value from the list. If the referred Type is External, the system allows user to enter the location in alphanumeric format of 30 characters maximum length.

  11. Practitioner – Enter the practitioner name in alphanumeric format of 30 characters maximum length. If referral type selected is external, this field allows only the entry of a practitioner name. However, if referral type is of type login facility or Enterprise, the system allows you to select the required practitioner using [?] function. The practitioners listed are based on the specialty and location selected. The following details are recorded against the selected practitioner.

  12. Tel No – This field displays the telephone number entered if any for the selected practitioner in Application Masters module.

  13. Mobile No - This field displays the mobile number entered (if any) for the selected practitioner in Application Masters module.

  14. Pager No - The telephone number entered (if any) for the selected practitioner in Application Masters module.

  15. [Clinical Details] – Clinical details of the patient referred are entered here. On clicking the clinical details button the Clinical Details screen gets displayed.

  16. [Notes] – Click [Notes] to open a screen to enter the referral notes in alphanumeric format and the Notes screen gets displayed.

Referred To

  1. Specialty – Select the Specialty from the list. This value indicates the specialty to which the patient is referred. This field is mandatory. The values displayed correspond to the Specialty of the logged in facility.

  2. Location - Select the Location type and location from the list of valid values. The valid values for Location Type are ‘Clinic’, ‘Nursing Unit’, ‘Daycare’ and ‘Procedure Unit’. The list next to ‘location’ contains values that correspond to the selected location type.

  3. Service - Select the required service from [?] function. The values displayed depend on the selected specialty.

  4. Practitioner – Select the Practitioner from the [?] function. The Practitioners available for selection are based on the specialty selected.

  5. Referred For IP – Select the required value from the list, if the patient is referred for admission against the selected specialty. The listed values are ‘Admission’ and ‘Booking’. This field is enabled only if location type selected is Nursing Unit.
