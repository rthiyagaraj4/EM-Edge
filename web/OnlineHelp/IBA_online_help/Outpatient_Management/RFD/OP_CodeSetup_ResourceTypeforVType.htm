Enter the following details to create a Resource Type for Visit Type record:

 

  1. Visit Type Indicator – Select the Visit Type Indicator from the list. These values are system defined and the list of values provided are:

  2. First – Initial/new visit for a problem/treatment.

  3. Follow-up – Return/repeat visit as a consequence of the first visit.

  4. Routine – Routine/one-time visit.

  5. Emergency – A visit when a patient requires immediate medical attention; such visits are specific to Clinics, which are set up to handle such emergency situations.

  6. Series – A visit that is part of a recurring series of visits necessary for the healthcare unit/treatment.

  7. Referral for Consult – A visit to the Clinic against an existing encounter pertaining to an Inpatient/Outpatient Service Location.

  1. Visit Type – Select the Visit Type from the list. This field represents the Visit Type, for which Resource Types have to be identified. The values displayed are based on Visit Types of the logged in Facility, pertaining to the selected ‘Visit Type Indicator’ as set in ‘Visit Type’ functions. This field is mandatory.

  2. Resource Class – Select the Resource Class from the list. These values are system defined and the list of values provided are:

  3. Practitioner

  4. Room

  5. Equipment

  6. Others

  1. Resource Type – Select the Resource Type from the list. These values are populated from the Application Masters module based on the Resource Class identified. This field is optional and is used to identify if any specific Resource Type is required to cater to the selected Visit Type.

  2. Required - Select the check box to indicate if the resource identified is mandatory to cater to the selected Visit Type. The check box is selected by default and is cleared to identify the resource as optional. If the check box is selected, at the time of Visit Registration, identification of the resource becomes mandatory to generate an outpatient encounter.
