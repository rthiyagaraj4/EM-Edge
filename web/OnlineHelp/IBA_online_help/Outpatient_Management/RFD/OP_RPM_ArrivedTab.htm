Arrived

On selecting the Record Patient Movement function, ‘Arrived’ tab gets displayed by default. Arrived tab is used to record patient arrival to a care location. The list of patients are populated in Arrived tab if the patients have an open and active encounter, also the patient departure should be recorded from patients previously assigned location.

  1. Select a Location Type from the list. Based on the selected Location Type, select the required Location using [?] function.

  2. Click [Search] to display the arrived patient details or enter the search criteria and click [Search] to select specific arrived patients. Click [Clear] to reset the records displayed in the result area.

    Enter the details to record arrival of a patient to a care location:

    1. Location – Select a Location Type from the list. Based on the selected Location Type, select the required Location from the [?] function.

    2. Encounter - Select the Encounter type from the list. The valid list of values are ‘Outpatient’, ‘Inpatient’, ‘Daycare’ and ‘Emergency’. Based on the selection, enter the valid Encounter ID to filter the patient listing based on Encounter ID.

    3. Patient ID - Enter the Patient ID in alphanumeric format of maximum length as specified in the MP Parameter or query for the patient details using [?] function. On entering the relevant details, the result screen gets populated with patient details that satisfy the entered query details. Select the patient by clicking the hyperlink of the Patient ID. The control now shifts back to the ‘Arrived’ screen with the patient details populated.

    4. Encounter Date – Enter the date of encounter or select it using the calendar function to list all patients who have had encounters on the selected date. System displays current date by default and you can change it, if required.

    5. Order By – Select the order by from the list. This is to group the listing based on either Encounter date or Encounter Type.

    6. All Encounters – Select the check box to include all encounters.

  3. On searching, the encounter details are displayed based on the condition.

    • Patient should have an open and active encounter.

    • Patient departure should have been recorded from patients previously assigned location.

    1. Encounter Date Time – The date and time of generation of the encounter is displayed.

    2. Encounter Type – The type of encounter is displayed.

    3. Encounter ID – The Encounter Identification Number is displayed. This field is a hyperlink which allows selection of the patient for recording arrival details.

    4. Patient ID – The Patient Identification Number is displayed.

    5. Patient Name – The Patient Name is displayed.

    6. Current Location – The Current location of the patient is displayed.

  4. Click Encounter ID hyperlink of a particular patient to display the ‘Arrived’ screen. The Location details are displayed.

    1. Location Type – The Location Type to where arrival has to be marked is displayed.

    2. Location – The Location against which arrival has to be confirmed is displayed.

  5. Click [Apply] to confirm the arrival of the patient or [Close] to exit from the screen. The control shifts to Record Patient Movement Screen.
