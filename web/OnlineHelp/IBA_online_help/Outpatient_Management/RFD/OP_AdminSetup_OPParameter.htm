Enter the following details to define OP Parameter:

 

Visit Controls

  1. Max No. of days for visit revising/canceling - Enter the maximum number of days the details of a visit can be revised or the visit itself can be cancelled after the visit registration has taken place, in numeric format of six digits maximum length. The period is considered based on the date and not on the date/time. This field is mandatory.

  2. Visit Backdate Days - This field is displayed only if Patient Billing Manager module is not installed. Enter the number of days in numeric format of two digits maximum length. Backdated Visit Registration can be performed for the patients who have visited the hospital but not registered within the number of days defined in this field

  3. Allow Visit Regn for Emergency Series Patients – Select the check box to allow Visit Registration for patients registered under Emergency Patient Series in Master Patient Index module.

  4. Allow Visit Registration for Wait List Invitations without Appointments – Select the check box to allow visit registration for patients registered under Wait List Invitation without appointment.

  5. Allow Invitation No. – This tab is enabled if ‘Allow Visit Registration for Wait List Invitations without Appointments’ check box is selected. The listed values are ‘Display’ and ‘Accept’. Select the required value from the list. If ‘Display’ is selected, Invitation Number is displayed in Patient Appointment records. If ‘Accept’ is selected, Invitation List dialog box gets displayed and you can enter the Invitation Number to view the Patient Appointment records.

  6. Override Practitioner for Appointment Case - Select the check box to override Practitioner while registering a visit.

  7. Action on Register Visit when Resource Schedule is blocked -  This drop-down list is used to define the action to be taken while registering a patient’s visit for a practitioner’s blocked schedule. You can select one of below three options, based on which the system action will vary on the Register Visit screen.

• No action – You can go ahead and register the visit with no restrictions about the blocked schedule.

• Warning – A warning message “Resource schedule is blocked. Do you want to continue?” will be displayed. You can either click [OK] and register the visit, or click [Cancel] and not register the visit.

• Stop – A message “Resource schedule is blocked. Cannot proceed” will be displayed, and you will not be able to register the visit.

• You need to set the Check- in at Identification field in Clinic setup as “Resource”. Only then, the field Practitioner will be mandatory in the Register Visit screen. Else, the system alerts about the blocked schedule will be displayed only when you  assign or  re-assign practitioner.

• The conditions for the actions will be applied only for the first time during Register visit or Assign Practitioner or Reassign Practitioner or Revise Visit. If practitioner is not changed in the Reassign Practitioner or Revise Visit, then these conditions will not be applied for any system actions.

 

  1. Allow Visit Registration for Inpatient - Select the check box to allow visit registration for Inpatients. Clear the check box to indicate that the visit registration for Inpatient is not allowed.

  2. Invoke Visit Registration/Manage Patient Queue from CA - Select the check box to allow visit registration or manage patient queue for patient from Clinician Access (CA) module.

  3. Close Patient Chart on Close Consultation/Check Out - Select the check box to close the Patient Chart which is usually displayed on Close Consultation. If the check box is selected, the patient chart is closed and the control goes to the Recently Accessed Patient List in Clinician Access (CA) module after closing consultation.  If the check box is cleared, the patient chart is displayed on Close Consultation.

  4. Visit Details Template – Select the check box to enable the ‘Visit Details Template’ function in the OP Code Setup. If the check box is cleared, you cannot access this function in the OP Code Setup.

  5. Validate existence of National ID during Register Visit - Select the check box to validate the existence of national ID/Civil ID of a patient during visit registration. By default, the check box is selected. This is applicable for users in MOD.

If the check box is selected, then in Register Visit screen, after entering the patient ID in the Patient ID field and on tab out the system validates if whether CIVL ID/National ID has been recorded for the patient or not. If not recorded, an alert message is displayed.

 

Visit Completion

  1. Max No. of days for visit completion - Enter the maximum number of days the Visit Disposition, Notes, Procedures, Diagnosis, etc. can be recorded after the recording took place in numeric format of six digits maximum length. This value has to be greater than the parameter value set for ‘Max No. of days for revising/canceling’ the visit. The period is considered based on the date and not on the time. This field is mandatory.

  2. Prompt for Register Referral on Close Consultation - Select the check box to prompt for Register Referral during Close consultation or OP Check out. Clear the check box if Registering Referral for the patient is not required during Close Consultation or OP Checkout. The system controls the prompt of Register Referral based on this setup during Close Consultation or OP Check out.

  3. Disposition Type for Close Consultation/Check Out – Select the required Disposition value from the drop-down list. The selected value appears as default value in the Disposition Type field in the below completed visits functionalities:

• OP Check Out

• AE Check Out

• OP Close Consultation

• AE Close Consultation

• OP Close Consultation and Chart

• AE Close Consultation and Chart

This value is populated based on the value selected in the ‘Disposition Type’ screen in Application Masters module.

  1. Close Patient Chart on Close Consultation/Check Out - Select this check box to close the patient chart on close consultation or check out activities.

  2. Default Disposition Type for Revoke Death  – By default, the value ‘Select’ is displayed. Select the required Disposition value from the drop-down list. The selected value appears as default value in the Disposition Type field in OP Check Out screen. Then go to ‘Record/Revoke Patient Death’ function to revoke the patient. This change is applicable only to users in MMOH sites.

 

Outpatient Episodes

An Outpatient Episode comprises of one or more outpatient visits to a specific Medical Service/Sub-service/Clinic over a period of relatively continuous care.

  1. Visit Type Deviation Rule - This is a one time setup. Once any of the below option is selected, it cannot be changed again for that facility.

  2. Service Based - If this option is selected, visit types are based on service selected during visit registration/appointment booking.

  3. Practitioner Based - If this option is selected, visit types are based on practitioner and the visit type rule defined as per the practitioners.

  4.  Insurance Group - If this option is selected, the visit types are based on the insurance group during visit registration/appointment.

  1. Default Rule ID – If 'Visit Type Deviation Rule' is selected as 'Practitioner Based', then this option appears.  By default, this field displays 'Select’. Select the required Rule ID from the drop down list values. This field is mandatory.

  2. Override Visit Type Control on Episode - The generation of Episode ID can either be system controlled or manually controlled based on this parameter. On selecting the check box, you are given an option to override the episode build up, if the patient re-visits the same clinic for the same service. There is no visit type control on episode. All the episodes are closed based on the parameter set under 'Minimum days elapsed from Last Visit'. If the check box is cleared, the system considers the 'Visit Type Control' parameter and closes the episode based on the duration and number of follow up visits defined for the clinic and service. The check box is selected, by default.

  3. Duration for Follow-up Visit - This field gets enabled only if the 'Override Visit Type control on Episode' check box is cleared. Enter the duration for follow up visit in numeric format of six digits maximum length. This is applicable for the Clinic and Service for which the setup is not done in 'Visit Type' code master. This is a mandatory field.

  4. Minimum days elapsed from last visit -  Follow up: Enter the number of days to be elapsed after a patient’s visit, to consider the next visit as a follow-up visit.

  5. Episode Closure By – Enter the criteria to be considered for closing a patient’s episode. You can select one of the following options:

• No shows – Patients’ episodes will be closed based on the number of “no shows” recorded against their booked appointments.

• Days – Patient’s episodes will be closed based on the number of days elapsed after the last follow-up

  1. No of Follow-up Visits Allowed - This field gets enabled only if the 'Override Visit Type control on Episode' check box is cleared. Enter the duration of the number of follow up visits in numeric format of six digits maximum length. This is applicable for the Clinic and Service for which the setup is not done in 'Visit Type' code master. This is a mandatory field.

 

Minimum days elapsed from Last visit

This parameter is used to define the closure rule for all open episodes.

  1. Follow-up - This field gets enabled only when the 'Override Visit Type control on Episode' check box is selected. Enter the number of days that must elapse from the date of the last visit in numeric format of six digits maximum length, if the patient has been advised a follow up visit to close forcibly an open episode using a background function. This is a mandatory field.

  2. No Follow-up - This field gets enabled only when the 'Override Visit Type control on Episode' check box is cleared. Enter the number of days that must elapse from the date of the last visit in numeric format of six digits maximum length, if the patient has no follow up advised to close forcibly an open episode using a background function. This is a mandatory field.

 

Queue Management

Patient queues are used to manage the Patient flow in the Facility and to balance workloads. The Queue Management function is used at various stages of interaction of the Patient with the Facility. The following sets of parameters related to Queue Management are maintained:

  1. Refresh Interval (Min) – Enter the Refreshing Interval in minutes for the Queue Management screen in numeric format of four digits maximum length. Based on the time interval set here the system automatically refreshes the Manage Patient Queue screen. This field is mandatory.

  2. On-time Appointment Status - If check-in is at least [] min before appointment time -Enter the value in minutes for On-time appointment status in numeric format of six digits maximum length. If the check-in time has passed the sum of appointment time and the entered grace time, then the check-in is considered to be a late arrival.

  3. No-show Appointment Status - If check-in is not within [] min after appointment time - Enter the value in minutes for No show Appointment status in numeric format of six digits maximum length. An appointment case is considered to be a no show when the current time passes the sum of appointment time and this grace period.

  4. Assign Queue Number By - Select System or Manual from the list. If 'System' is selected from the list, then the system will automatically generate the Queue Number. If ‘Manual’ is selected then you can define Queue Number through Mark Arrival function.

  5. Non-Mandatory Queue No – Select the check box to make ‘Queue No’ field as non-mandatory in Manage Patient Queue function. By default, the check box is not selected. This check box appears only if the ‘Assign Queue Number By’ field is selected as Manual. This field is applicable only for users in MOHML sites.

  6. Include Arrived and Consultation Started Patients – Select the check box to display the legend ‘Include Arrived and Consultation Started Patients’ in ‘Cancel Visit’ function. Clear the check box if the legend ‘Include Arrived and Consultation Started Patients’ and the field will not be displayed in ‘Cancel Visit’ function. The check box is cleared, by default.

  7. Calculate Wait Time based on Status – Select the required value from the drop-down list. The list values include Checked-In and Arrived At Nursing Station. Based on the selected value, the patient Waiting Time is calculated in the Manage Patient Queue. By default, the value ‘Checked-In’ is displayed. The ‘Calculate Wait Time based on Status’ configuration done in OP Parameter will apply for all Clinics. If the setup is done in Clinic screen, then Clinic setup will take precedence. At any point of time, the wait time will be calculated based on the latest configuration.

  8. Colour Legend to Identify Patient if Wait time Exceeds – Enter the allowed waiting for the patients in minutes in the text box. If the patient waiting time exceeds the allowed waiting time, the wait time is displayed in blue or orange colour in the Manage Patient Queue, Record Vital Signs and OP Check Out screens.

  1. Action when unsettled/unbilled amount for the encounter – Defines the action to be taken while marking arrival or starting consultation for a patient, if the patient has any unsettled or unbilled amount in that encounter. You can select one of the three options from the drop-down list.

• No Action – The system will not display any warning or error, and you can proceed with the Mark Arrival and Start Consultation activities for the patient without interruption.

• Warning – The system will display a warning message “Unsettled/Unbilled amount exists for the encounter. Do you want to continue?” both when marking arrival and starting consultation for the patient. You can click OK and still proceed with the activities.

• Stop – The system will display an error message, “Unsettled/Unbilled amount exists for the encounter. Cannot proceed.” You cannot mark arrival or start consultation for the patient.

These messages will be displayed when you try to select the Arrived and Start Consultation options for a patient in the Manage Patient Queue  screen.

 

Auto Completion of Incomplete Visits

  1. Disposition Type – Select the Disposition Type from the list of valid values. The Disposition Type values are displayed from the ‘Disposition Type’ master function in Application Masters module. The Disposition Type values are displayed if 'Use at Visit Conclusion' and 'Use for Maternal Death' check boxes are not selected in 'Disposition Type' master function in Application Masters module. The Disposition Type selected is defaulted while closing the incomplete visits automatically. If no value is defined here, the Disposition Type is defaulted to 'Select' in the OP Check out function. This is a mandatory field, if disposition details have to be captured as part of minimum data set.

 

Vital Signs/Nursing Measures

  1. Group – Select the Vital Signs from the list. The Vital Signs selected are considered as nursing measures during patient visit if you fail to set Vital Signs at the ‘Clinic’ and ‘Visit Type for clinic’ level. This field is mandatory.

 

Billing Parameters

  1. Interfaced to Billing Module - Select the check box if the Patient Billing Manager module is interfaced. The visit charges are automatically generated during registration and updated in the Patient Billing Manager module. Clear the check box if the Patient Billing Manager module is not interfaced. The check box is cleared, by default.

 

Clinic

  1. Category – Select the check box to display the ‘Category’ field in Clinic and Procedure Unit masters. By default, the check box is cleared.

  1. Category Mandatory – The check box is disabled by default. Once the category check box is selected, the ‘Category Mandatory’ check box gets enabled for selection. Select the check box to make the ‘Category’ field as mandatory in Clinic and Procedure Unit functions.

  2. Referral Required - Once the Category check box is selected, Referral Required check box will also get selected automatically. Similarly, if Category check box is cleared, the Referral Required check box gets cleared automatically. The ‘Referral Required’ check box will be always disabled. Select this check box to display the ‘Referral Required’ check box in Add Visit tab in Register Visit function.
