Process Referrals/Non Referrals

From this function you can Modify the Visit Referral Status to ‘Referrals’/‘Non-Referrals’ as required.  .

This function allows user to modify the ‘Referral visits’ to ‘Non-referral’ and ‘Non-referral visits’ to ‘Referral’ post visit registration.

On searching, the data will be displayed only if below conditions are satisfied.

  • Only OP visit encounters will be displayed.

  • Only Primary/Specialized Category visits of both Clinic and Procedure unit will be displayed.

  • Cash and Insurance specific encounters will be displayed. But Cash encounters will be in non-editable field.

  • Also, OP Checked Out visits and Insurance bill posted visits will will be displayed. But the Insurance bill will not be posted from Billing side. If Insurance bill is posted then those records will be displayed, and it will be in non-editable mode.

  • Only first visit encounters will be displayed.

Modifying Visit Referral Status

From the OP Transactions, select the Process Referrals / Non-Referrals function.

The Process Referrals / Non-Referrals screen gets displayed.

  1. Select the Visit Date [From- To] range by clicking the calendar icon. Two weeks date range can be selected.

  2. Episode ID – Enter the encounter ID of the specific visit.

  3. Click [Search] to view patients records matching the entered search criteria.

  4. The search result includes Patient ID, Patient Name, Epsiode ID, Clinic, Practitioner, Visit Date Time, Customer Group, Customer, Policy, Referral, and Referral Details.

  5. Select the ‘Modify’ option against the required patient record to change the visit referral status. The ‘Modify’ option is disabled for cash patients.

  6. On selecting ‘Modify’, the ‘Authorization’ dialog gets invoked if the user is a non-privileged user. Enter the User ID, Pin and Remarks in the dialog and click [OK]. On successful pin verification, user can change the referral status of the encounter. While modifying each record, 'Authorization' is required.

If user is having privileges to access the screen, then the Authorization window will not appear.

  1. Changing ‘Referral’ status to ‘Non-referral’ status - If the current Referral status is ‘Yes’ and user is trying to change it to ‘No’, then after successful authorization, a prompt message will appear to confirm to move status from ‘Referral’ to ‘Non-Referral’. Click OK, system will delink the encounter from the visit and update the visit as non-referral. The subsequent visits of the encounter will also be non-referral.

  1. Changing ‘Non-referral’ to ‘Referral’ status - If the current referral status is ‘No’ and user tries to update it as ‘Yes’, then Referral Search screen will get invoked.

  • In the Referral Search screen, visits details (Specialty, Service, Location, Location type, Patient ID) data will get defaulted in the filters. Click the [Search] button, against the selected data if any referral is already available it will get displayed. Select the referral and the referral ID will be linked to the encounter. The encounter will be considered as referral and the subsequent visits will also be considered as referral.

  • Instead of Search button, user can also click the [Register Patient Referral] button. The Register Patient Referral window gets invoked where user can create a new referral and link it to the encounter.

  • In the Register Patient Referral window, visit details of the encounter (Specialty, Location, Location Type, Service) will get auto defaulted in the ‘Referred To’ tab. User can update the other details and click [Apply]. A new referral ID will be registered and linked to the encounter and the encounter’s referral status will be considered as referral.
