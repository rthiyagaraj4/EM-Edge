Enter the following details:

 

  1. Specialty - Enter the specialty or select the specialty using the [?] function. The specialties created in the Specialties function are listed. This is a mandatory field.

  2. Diagnosis Coding Scheme - The Diagnosis Coding Scheme is displayed and disabled as defined for the selected Specialty.

  3. Operation Code - Enter the Operation code in alphanumeric format of 10 characters maximum length if the Diagnosis Coding Scheme is User Defined. You can select the required Operation Code based on the Diagnosis Coding Scheme if not User Defined. This is a mandatory field.

  4. Enabled - Select the check box to enable this record. Only enabled records are available in the transactions.

  5. Long Description - Enter the Long Description for the Operation in alphanumeric format of 60 characters maximum length. This is a mandatory field.

  6. Short Description - Enter the Short Description for the Operation in alphanumeric format of 40 characters maximum length. This is a mandatory field.

  7. Operation Type - Select the Operation Type to which this operation belongs using the [?] function. The Operation Types are created in the 'Operation Types' function. This is a mandatory field.

  8. Link Catalog - Select the check box to link an order catalog for the operation/procedure.

  9. Order Type - This field is disabled if Link Catalog check box is selected. Select the Order Type from the list. The Order Types associated to the Specialty in 'Order Type for Specialty' function is displayed. This is a mandatory field.  

  10. Catalog Cross Reference - This field is enabled if the 'Link Catalog' check box is selected. Select the required Order Catalog to associate the operation using the [?] function. The Order Catalogs created with Order Category 'OT Surgery' and associated to the Order Types linked to the selected Specialty are displayed for selection. The Order Catalogs are created in the Results and Reports module. This is a mandatory field.

  11. Duration - Enter the duration required for the operation in hh:mm format. This is a mandatory field.  

  12. Side Applicable - Select the check box to indicate that the operation has to be performed on a particular side of the body (right/left). For example, if the operation is Amputation of Leg, the check box must be selected to indicate whether the operation is performed on the left leg or right leg. This is an important parameter that will be invoked in the Booking and OT Slate functions.

  13. Pre-Operative Checklist - Select the Pre-Operative Checklist using the [?] function. The checklists are created in the Checklist function.

  14. Post-Operative Checklist - Select the Post-Operative Checklist using the [?] function. The checklists are created in the Checklist function.

  15. Swab & Instrument Count - Select the Swab and Instrument Count using the [?] function. The swab and instrument counts are created in the Swab Count Template function.

  16. Category - Select the required category using the [?] function. The operation categories are created in the Operation Category function.

  17. Sub Category - Select the sub category using the [?] function. The operation sub categories are created in the Operation Sub-category function.

  18. Standard Instructions - Standard Instructions are displayed on selection of Order Catalog. Order Catalogs are created for all the orderables and the standard instructions for each orderable is defined in the Order Catalog function of the Results and Reports module.

  19. Standard Instructions in Local Language - Standard Instructions in local language are displayed on selection of Order Catalog.
