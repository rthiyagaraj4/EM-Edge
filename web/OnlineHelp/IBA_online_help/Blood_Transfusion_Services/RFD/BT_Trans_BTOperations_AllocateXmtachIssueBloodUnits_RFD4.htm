The following details are displayed:

 

Direct Issue

  1. Request for Issue By – The user who requested for the blood unit is displayed.

  2. Request for Issued Date – The requested Issued Date for the blood unit is displayed.

  3. Print Tag – Select the check box if the unit tag is to be printed while issuing the blood unit.

  4. Issued By – Click [List of Values] to select the user who issued the blood unit. The list displays the list of authorized users who can issue blood units.

  5. Issued To – Click [List of Values] to select the user to whom the blood is issued. The list displays all the authorized users to whom the blood unit can be issued.

  6. Issued Date – Select the date of issue using Calendar function.

  7. Process Code – This field is enabled if ‘Processing of Unit at Issue time’ check box is selected while defining the product in Product code function. Select the process code from the list, which has been implemented on the blood unit before issuing. The list displays the values defined in the Process Code function.

  8. Process Done by – This field gets enabled if ‘Processing of Unit at Issue time’ check box is selected while defining the product in Product code function. Select the user who performed the process on blood unit.

  9. Processed Date – This field gets enabled if ‘Processing of Unit at Issue time’ check box is selected while defining the product in Product code function. Select the blood unit Processed Date using Calendar function.

  10. Expiry Date – The Expiry Date of the unit issued is displayed. You can modify it without exceeding the Expiry date specified for the selected product in the Product function.

  11. Storage Location Code – Click [List of Values] to select the Storage Location where it is stored after issued.

  12. [B] – Click [B] to enter the billing details.
