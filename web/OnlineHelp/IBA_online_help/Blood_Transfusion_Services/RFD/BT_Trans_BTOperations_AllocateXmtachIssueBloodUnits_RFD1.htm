The following details are displayed:

 

  1. Priority – The Priority of the specimen is displayed. The valid values are ‘Stat’, ‘Urgent’ and ‘Routine’.

  2. Patient ID – On selecting the specimen number, the Patient ID and Name of the patient are displayed.

  3. Comments – The comments recorded while requesting the blood unit are displayed. You can associate comments through this function also.

  4. Blood Group – The Blood Group of the patient recorded in the ‘Record Patient Blood Group function’ in Medical Records module is displayed.

  5. Rhesus – The Rhesus factor of the blood group recorded in the ‘Record Patient Blood Group function’ in Medical Records module is displayed.

  6. Sex – The Gender of the patient recorded during patient registration is displayed.

  7. Age – The Age of the patient recorded during the patient registration is displayed. If an asterisk (*) is displayed next to the age, it is a calculated age of the patient and not the actual age.

  8. Regd Date/Time – The Registered Date and Time of the Specimen is displayed.

The following changes are done while allocating, crossmatching and issuing blood units batch wise.

=> Allocation

The scanned Unit is directly allocated with ‘Allocated By’ as the logged in User and ‘Allocated Date’ as System Date and Time. After allocation, the select check box of the unit is automatically selected and disabled. The Expired Units cannot be allocated when scanning.

=> Crossmatch

You can select all required unit numbers and crossmatch them together. Only the results, which are compatible/least incompatible, are accepted for crossmatch by batch. All the Units selected are changed to the crossmatched status. The billing button is disabled when crossmatch is done by batch.

=> Issue

The scanned Unit is directly issued with Issued By as the logged in User and Issued Date as System Date/Time. The billing button is disabled when issue is done by batch. All the Units selected are changed to the issued status.

Revert Issue and Return options of the blood units cannot be performed when the allocation/crossmatch/issue is performed in Batch. Whenever the ‘Select’ check box is selected, it is assumed to be a batch-wise process. When ‘Select all’ check box is selected, all units’ check boxes are selected irrespective of what stage the blood unit is in. All buttons related to the selected units are enabled. When [Xmatch] is clicked, and if the units with multiple statuses are selected, an alert is displayed asking the user whether to check only the units, which are in allocated stage. Similarly, when [Issue] is pressed, and if the units with multiple statuses are selected, an alert is displayed asking the user to check only the units, which are either in compatible or in least incompatible stage.

On selecting the new product, save the record and the new product is displayed in the Allocate/Xmatch/Issue - Blood Units screen. The cursor is active at Barcode Unit Number field and ready for scanning. If the original product is rejected, you can allocate the alternate product.
