The following details are displayed:

 

  1. Patient – The Blood Group and Rhesus code of the patient is displayed.

  2. Test – The Test codes defined in the ‘Test and Allowed Values for Cross Match’ function is displayed.

  3. Result Value – Click [List of Values] to select a valid Result Value defined in the ‘Test and Allowed Values for Cross Match’ function.

  4. Result Indicator – On selecting the Result value, the system displays the Result Indicator defined in the ‘Test and Allowed Values for Cross Match’ function.

  5. [B] – Click [B] to enter the billing details.

  6. Print Tag – Select the check box if the unit crossmatch tag is to be printed after the test is performed.

  7. [Validate] – Click [Validate] to validate all the result values.

  8. Storage Location – The Storage Location of the Blood Unit, where it is stored after performing crossmatch test is displayed.

  9. Reserve Date - The Reserve Date defaults as per the setup in the 'Time Period for Blood Unit Reservation [In Hrs]' field. The Reserve Date can be modified, which can be less than the default date, but it must not exceed the period specified in the 'Time Period to Modify Blood Unit Reserved Date (In Hrs)' field.

  10. Xmatch Result – On validating the result value, the Xmatch Result is displayed in the field. The listed values are Compatible, Incompatible and Least Compatible.

The Mandatory tests are highlighted in   color and the validation status is shown at the bottom of the screen.
