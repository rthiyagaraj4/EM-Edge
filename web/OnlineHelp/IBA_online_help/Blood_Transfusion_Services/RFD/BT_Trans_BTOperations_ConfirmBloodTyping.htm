The following details are displayed:

 

  1. Regd Date/Time – The Registered Date and Time of the specimen are displayed.

  2. Priority – The Priority of the Specimen is displayed. The valid values are ‘Stat’, ‘Urgent’ and ‘Routine’.  The priority is recorded for the specimen in the Registration function in Laboratory module.

  3. Patient ID – The Patient ID and Name of the Patient for whom the blood typing is to be confirmed are displayed.

  4. Sex – The Gender of the patient recorded during patient registration is displayed.

  5. Age – The Age of the patient recorded during the patient registration is displayed.

  6. Comments – Click [List of Values] to select the comment from the list of values. The list displays the values defined in the Crossmatch Comments function.

  7. Blood Group – The Blood Group of the patient is displayed. You have to enter the Blood Group manually to confirm it, if the check box ‘Confirm Blood Typing by Operator’ is selected in ‘BT Parameter’ function. If the blood group you entered is not matching with the patients blood group an error ‘Blood Group is not matching with Patient’s Blood Group’ is displayed.

  8. Rhesus – The Rhesus factor of the blood group is displayed. You have to enter the Rhesus factor manually, if the check box ‘Confirm Blood Typing by Operator’ is selected in ‘BT Parameter’ function.

 

Product/ Categories

The data recorded in the in the Request Blood Units function is displayed.

  1. Product – The Product code of the blood unit is displayed.

  2. Description – The description of the product code is displayed.

  3. Category – The category code of the product is displayed.

  4. Description – The description of the Category is displayed.

  5. Required units – The number of units required for transfusion is displayed.

  6. Date – The date when the blood unit is required for transfusion is displayed.

  7. Expected Transfusion Date – The Expected Transfusion Date is displayed.

  8. Remarks – The remarks are displayed.
