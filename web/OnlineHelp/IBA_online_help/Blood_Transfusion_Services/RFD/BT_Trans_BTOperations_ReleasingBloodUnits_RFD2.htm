The following details are displayed:

 

  1. Patient ID – The Patient ID associated to the specimen number for the specified date range is displayed.

  2. Specimen Number – The Specimen Numbers matching the search criteria are displayed. If you select De-allocation, the allocated specimens are listed. If you select De-reserve, the specimens, which are crossmatched are listed.

  3. Unit Number - The Unit Number that has been allocated or reserved for a particular specimen is displayed.

  4. Product code – The product code of the blood unit that has been allocated or reserved for a particular specimen is displayed.  

  5. Description – On selecting the product code, the Description of the selected product defined in ‘Product Codes’ function is displayed.

  6. Blood Group – The Blood Group of the patient recorded in the ‘Record Patient Blood Group’ function in Medical Records module is displayed.

  7. Rhesus – The Rhesus factor of the blood group recorded in the ‘Record Patient Blood Group’ function in Medical Records module is displayed. Allocate/Reserved Date – The date of allocation or reservation of the blood unit is displayed. The allocation and reservation is done in the ‘Allocate/Xmatch/Issue - Blood Units’ function.

  8. Allocate/Reserved Date - The Allocated or Reserved Date for a particular specimen is displayed.

  9. Storage Location – The Storage Location, where the selected blood unit/specimen is stored is displayed. The values defined in Storage Location function are displayed.
