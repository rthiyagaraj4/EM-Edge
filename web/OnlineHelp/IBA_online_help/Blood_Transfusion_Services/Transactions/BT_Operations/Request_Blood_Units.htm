Request Blood Units

From this function you can Request blood products for transfusion, Add patient requirement details and Modify/Reject the ordered blood products. You can also View the Patient details, Antibodies, Transfusion History and Order Comments. You can Query to view the existing records.

The Request Blood Units function allows you to Request for a new blood product,  Add a new blood product(s) to the existing request and patient requirement details to the existing/new request(s), Modify the ordered blood product and its details such as requested unit, volume, etc.,  and reject the ordered blood products by attaching rejection codes. You can also view patient’s details, antibodies, transfusion history and Order Comments. If a blood unit is requested through the Order Entry and Result Reporting module, and if the Specimen is registered, you can select the specimen number to edit or reject the request. You can request for the blood unit during the specimen registration also.

Requesting Blood Units

From the Transactions > BT Operations, double-click Request Blood Units function.

The Request Blood Units screen gets displayed.

  1. Click [List of Values] to select the Specimen Number. The list displays the Specimen numbers registered in the Registration function in Clinical Laboratory module. This is a mandatory field.

  2. Click [Execute Query] or press [F8] to view the specimen details. <Related Field Description>

  3. Right click on the link Right Click for User Option to select the user. The link has two pop-ups, namely, Patient Chart and Patient Specimen History. Click Patient Chart option to open the Patient Chart Screen from Clinician Access and Patient Specimen History to open the Lab screen from the Laboratory module.

  4. Enter the details in the Remark field to request for the blood units. Select the ‘Reject’ check box to reject the unit, if needed.

  5. Click [Details] to view the patient details by invoking the Patient Details screen. Click [Close] to return to the Request Blood Units screen.

  6. Click [Antibodies] to view the antibodies associated to the patient. Click [Close] to return to the Request Blood Units screen.

  7. Click [Transfusion History] to view the pervious transfusion details of the patient.

  8. Click [Requirements] to view the existing requirements and associate additional transfusion requirements for the patient. Click [Close] to return to the Request Blood Units screen.

    Enter the following:

    1. Code – Click [List of Values] to select the requirement Code from the list of values. The list displays the values defined in Transfusion Requirements function.

    2. Description – On selecting the requirement code, the description of the code is displayed.

  9. Click [BB Comments] to view or add the blood bank comments for the patient. On clicking [BB Comments], the ‘Blood Bank Comments’ dialog gets displayed. Select the BB Comment code from the [List of Values] and click [Save]. The Blood Bank Comment codes defined in the Result Notify/BB Comments master appear in the List of Values.

  10. Click [Order Comments] to view comments recorded by the clinician while ordering blood units.

The selected requirements are verified during allocation or issue of the blood units.

  1. Click [Commit] or press [F10] to record the Blood Unit Request or [Clear Record] to cancel the process.
