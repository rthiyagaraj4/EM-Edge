Patient Name

The Name components applicable for the enterprise as set in the parameter function of Master Patient Index module are displayed as search criteria.

  1. Name in Local Language – Select the check box to include Patient Name in local language as search criteria. This field is displayed, if ‘Accept Patient Name in Local Language’ check box is selected in Legal Names tab of MP Parameter in the Master Patient Index module.

  2. Name - The name components (First/Second/Third/Family Name) set in Master Patient Index module are displayed. Select the check box against each component to include it as search condition.

  3. Soundex Algorithm - Specify the Soundex Algorithm to be used for matching the Name columns. The valid values are ‘Ethnic’ or ‘None’. The value ‘None’ is displayed, by default. If ‘None’ is selected, the Patient names are matched based on the exact spelling.

  4. Previous HN - The Previous HN components set in MP Parameter under 'Alt ID' tab in Master Patient Index is displayed. Select the check box against it to include it as a search condition.

  5. Government No. - The Government No. components set in MP Parameter under 'Alt ID' tab in Master Patient Index is displayed. Select the check box against it to include it as a search condition.

  6. Driving License No. - The Driving License No. components set in MP Parameter under 'Alt ID' tab in Master Patient Index is displayed. Select the check box against it to include it as a search condition.

  7. Expatriate No. - The Expatriate No. components set in MP Parameter under 'Alt ID' tab in Master Patient Index is displayed. Select the check box against it to include it as a search condition.

  8. Other Alt ID - The Alternate ID(s) set in Master Patient Index are listed. Select the Alternate ID from the list.

 

Demographic Data

  1. Nationality - Select the Nationality check box to consider Nationality as a criterion while searching for possible duplicates. The check box is cleared, by default.

  2. Gender - Select the Gender check box to consider Gender of patient as a criterion while searching for possible duplicates. The check box is cleared, by default.

  3. Date of Birth (Year/Month/Day) - Select the corresponding Date of Birth check box to consider year/month/day component of Date of Birth as a criterion while searching for possible duplicates. If the Day is selected, Month and Year are also included in the selection. If the Month is selected, only the Year is included in the selection. The check box is cleared, by default.

  4. Area - Select the Area check box to consider the Area as a criterion while searching for possible duplicates.

  5. Residential Telephone No. - Select the Residence Telephone Number check box to consider Residence Telephone Number as a criterion while searching for possible duplicates. The check box is cleared, by default.

  6. Other Contact No. - Select the Other Contact Number check box to consider Other Contact Number as a criterion while searching for possible duplicates. The check box is cleared, by default.

 

Scope of Search

  1. Search Type - Select the Search Type value from the list of valid values. The listed values are ‘Logged in Facility’ and ‘All’. If ‘All’ is selected, the patients registered across all facilities are considered. If ‘Logged in Facility’ is selected, the patients registered with the logged in facility are considered while searching possible duplicates.

The First Name, Third Name, Local Language Name check boxes display if the fields are enabled in MP Parameter. The fields displayed may vary depending on the MP Parameter Setup.
