Enter the following details to Setup a Daily Schedule record:

 

Location/Resource Details

  1. Location Type – Select the Location Type as Clinic, Daycare Unit or Procedure Unit.

  2. Location – Select the Location based on the selected Location Type using [?] function. This field is mandatory. On selecting the required location type, the Locations listed are based on the following conditions:

  3. All the Clinics/Daycare/Procedure units defined under the selected location type that allows appointments (based on the ‘Allow Appointment’ value set in ‘Clinic’, Daycare Unit’ and ‘Procedure Unit’ setup function of EM Outpatient Management module).

  4. All the defined Clinics/Daycare/Procedure units must be associated to the operator station assigned to the logged in user.

  1. Resource Type – This field is enabled on selecting a location. Select the required Resource Type as Practitioner, Equipment, Room or Other. This field is mandatory.

  2. Resource – Select the required Resource based on the selected Resource Type using [?] function This field becomes mandatory based on the following condition:

  3. If ‘Check in at Identification’ is set to Resource in ‘Clinic’, ‘Day Care’ and ‘Procedure Unit’ setup (OP module), the resource field becomes mandatory while defining the timetable or if ‘Check in at Identification’ is set to Non Identifiable, this field is not mandatory.

 

Day/Time Details

  1. Date – Enter the Date in date format of dd/mm/yyyy for which the timetable is to be defined or click calendar function to select the required date. This field is mandatory.

  2. Timetable Type - Select the Timetable type from the valid list of Timetable Types. This field is mandatory.

  3. Slot

  4. Slab

  5. Free Format (Applicable for all Resource Class)

When you select the Location Type field as ‘Procedure Unit’ created with radiology applicable, the Timetable Type is defaulted as ‘Slot’ and disabled.

  1. Start Time - The operating start time of the selected location as set in ‘Clinic’, ‘Day Care’ or ‘Procedure Unit’ function of Ambulatory Care Management module gets displayed on selecting the required timetable type by default. Enter the Timetable Start Time for the selected Location/Resource/Day combination in hh:mm format. The entered start time must not be less than the location start time and greater than the location end time as set in Outpatient Management module or the system alerts you with a message. This field is mandatory.

  2. End Time - The operating end time of the selected location as set in ‘Clinic’, ‘Day Care’ or ‘Procedure Unit’ function of Outpatient Management module gets displayed on selecting the required timetable type by default. Enter the Timetable End Time for the selected Location/Resource/Day combination in hh:mm format. The entered end time must not be greater than the location end time as set in Outpatient Management module or the system alerts you with a message. The end time must be greater than the Start Time entered. This field is mandatory.

The entered location time span must be multiple of time per patient/slot or slab.

 

Slot Details

  1. Time Per Slot - The base slot/slab time as set in ‘Clinic’, ‘Day Care’ or ‘Procedure Unit’ function of Outpatient Management module gets displayed on selecting the required timetable type, by default. Enter the required time to be allocated to each Patient in hh:mm format. The entered time per slot must be in multiples of the base time set for slot/slab in Outpatient Management module or you are alerted with a message. This field is mandatory only if the Timetable Type selected is ‘Slot’.

  2. Maximum Slots per Day - Total number of slots planned for the day for the entered Location/Resource/Day combination is calculated automatically after the time per Slot/Patient is specified. This value is computed and displayed based upon the Time per Patient (for Slot) values.

  3. Max Patients per Day - Total number of appointments planned for the day for the entered Location/Resource/Day combination is calculated automatically after the time per Patient is specified. This has to be entered manually only if the Time Table Type selected is ‘Free Format’. If the Timetable Type selected is ‘Slot’ or ‘Slab’, this value is computed and displayed based upon the Time per Patient (for Slot) or the Time per Slab and Max Patients per Slab (for Slab) values. This field is mandatory.

 

Slab Details

  1. Time per slab - The base slot/slab time as set in ‘Clinic’, ‘Day Care’ or ‘Procedure Unit’ function of Outpatient Management module gets displayed on selecting the required timetable type, by default. Enter the required time to be allocated to each Patient in hh:mm format. The entered time per slab must be in multiples of the base time set for slot/slab in Outpatient Management module, or you are alerted with a message ‘Time per Slab must be multiples of given duration hh:mm’. This field is mandatory on selection of the Time Table Type as ‘Slab’.

  2. Max Patients per Slab - Enter the total number of appointments planned for a Slab for the entered Location/Resource/Day combination in numeric format of three digits maximum length. This field is mandatory only if the Time Table Type is selected as ‘Slab’.

 

Visit/Overbooking Details

  1. Maximum First Visits - Enter the maximum number of appointments to the Visit Type category as ‘First Visit’, which is allowed for the entered Location/Resource/Day combination, in numeric format of three digits maximum length. The entered value is based on Rule for limiting first/other visits. If the ‘Time Table’ type selected is slot, the number of first visit can be based on ‘Patients’ or ‘Slots’ as set in OA Parameter.

  2. Maximum Other Visits - Enter the maximum number of appointments to the Visit Type category as ‘Others’, which is allowed for the entered Location/Resource/Day combination, in numeric format of three digits maximum length. The entered value is based on Rule for limiting first/other visits. If the ‘Time Table’ type selected is slot, the number of Other visit is based on ‘Patients’ or ‘Slots’ as set in OA Parameter.

  3. Max New Patients - This field is available only if the 'Schedule Rule Applicable' check box is selected in the OA Parameter. Enter the maximum number of new patients planned for visit. This is a mandatory field.

  4. Max Old Patients - This field is available only if the 'Schedule Rule Applicable' check box is selected in the OA Parameter. Enter the maximum number of old patients planned for visit. This is a mandatory field.

  5. Maximum Over bookings - Enter the maximum number of overbookings, which is allowed for the entered Location/Resource/Day combination, in numeric format of three digits maximum length. If the ‘Time Table’ type selected is slot, the entered value is based on the OA Parameter setup. The number of over bookings is based on ‘Patients’ or ‘Slots’ as set in OA Parameter.

The value entered facilitates to accommodate additional patients who need to be scheduled due to unforeseen medical reasons. If the value is not entered, overbookings cannot be made.

 

Other Details

  1. Forced in Bookings allowed – This field is enabled for timetable type ‘Slot’ and ‘Slab’. Select the check box to allow forced bookings for the entered Location/Resource/Day combination, or clear the check box.

  2. No. of Forced in Booking/day – This field is applicable for Time Table of type ‘Slot’ and ‘Slab’ and is enabled on selecting the ‘Forced in Bookings allowed’ check box. You can enter only one forced in booking per slot/slab in numeric format of three digits maximum length, or the system alerts with a message ‘No. of Forced Booking cannot exceed Maximum Patients per Day’ for ‘Slots’ and ‘No. of Forced Booking cannot exceed Maximum Slabs per Day’ for ‘Slabs’.

  3. No. of Global booking per Day – Enter the Number Of Global Booking For Referral in numeric format of three digits maximum length.

  4. Remarks - Enter the Remarks about the timetable in alphanumeric format of 40 characters maximum length.

 

Break Timings

  1. From Time – Enter the Break Start Time for the entered Location/Resource/Day combination in hh:mm format.

  2. To Time - Enter the Break End Time for the entered Location/Resource/Day combination in hh:mm format.
