Enter the following details to register disaster patients:

 

  1. Patient Series – Select the required Patient Series from the list. The default value is displayed based on the Emergency Series selected in MP Parameter function in the Master Patient Index module. This field is mandatory.

  2. Priority Zone – The Priority Zone defined under 'Disaster Management' in 'AE Parameter' is displayed, by default. Select the required Priority Zone from the list. This is a mandatory field.

  3. Patient Details - Enter the total number of patients and average age in numeric format as depicted in the matrix below:

Enter the total number of patients in numeric format of 3 digits maximum length under each age and gender combination.

Estd Age

The age entered in 'Default Unknown Adult Age' field in AE Parameter is displayed and disabled. You cannot modify the estimated age of the adult.

Enter the estimated age of children in numeric format of 2 digits maximum length in ‘Years’.

Enter estimated age of infants in numeric format of 2 digits maximum length in ‘Months’ and ‘Days’.

 

  1. Disaster Registration Reference - Enter the Disaster Registration Reference Number in alphanumeric format of 15 characters maximum length. This is a mandatory field. The Disaster Registration Reference Number entered is displayed in 'Current Emergency Patients' function when disaster patients are listed.

  2. Remarks – Enter the Remarks for disaster patient registration in alphanumeric format.

  3. Create Attendance Details - Select the check box to enable the attendance details for disaster patients. The check box is unselected by default.

  4. Capture Financial  Details – Select the check box to capture the financial details for disaster patients. The check box is unselected by default. The check box is visible only if Billing module is installed. On selecting the ‘Create Attendance Details’ check box, the ‘Capture Financial Details’ check box gets selected and disabled. You can select or unselect independently only if ‘Create Attendance Details’ check box is unselected.

  5. Location – The Location defined under Disaster Management in ‘AE Parameter’ gets displayed. Select the Location from the list. This field represents the Location to which the Patient has to be registered. The locations associated to the logged in users operator station with Level of Care as ‘Emergency’ are displayed. This field is mandatory.

  6. Service - The Service defined under Disaster Management in ‘AE Parameter’ gets displayed. Select appropriate service to be rendered to the patient. The Services associated to a selected Location gets listed. This field is mandatory.

  7. Visit Type - The Visit Type defined under Disaster Management in ‘AE Parameter’ gets displayed. Select the Visit Type from the list. This field represents the Visit Type of Patients. The displayed list is limited to the Visit Types based on the selected Location. This field is mandatory.

  8. Mode of Arrival – Select the mode of arrival of the patient from the list. The values gets listed from the mode of arrival as set in Application Masters module.

The fields ‘Location’, ‘Service’, ‘Visit Type’ and ‘Mode of Arrival’ are disabled by default. These fields  get enabled only when the ‘Create Attendance Details’ check box is selected.
