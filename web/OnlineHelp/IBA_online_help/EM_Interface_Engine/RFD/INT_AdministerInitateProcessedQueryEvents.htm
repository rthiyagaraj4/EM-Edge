The following details are displayed:

 

  1. Event Status (indicated by a color) - The current status of the message is indicated by a colour. ‘Blue’ indicates unprocessed messages; ‘Green’ indicates successfully processed messages; ‘Red’ indicates erroneous messages and ‘Black’ indicates messages that are currently not required.

  2. Message Status – The Message Status is displayed.

  3. Message ID – The Message ID is displayed and hyperlinked.

  4. Event – The Event of which the message is a part is displayed.

  5. Message Date – The date of message generation is displayed.

  6. Facility – The relevant facility is displayed.

  7. Patient ID – The Patient ID, which is displayed in the message.

  8. Episode ID – The Episode ID, which is displayed in the message.

  9. Type – The type of patient is indicated. ‘I’ indicates inpatient and ‘O’ indicates outpatient.

  10. Visit ID – The Visit ID, which is displayed in the message.

  11. Accession Number - The accession number is displayed.

  12. External Accession Number – The External accession number is displayed.

  13. Merged Patient ID – The merged patient ID, which is displayed in the message.

  14. Action Type – ‘I’ indicates whether the message has been inserted and ‘U’ indicates whether the message has been updated.

  15. Last Processed Date – The date when the message was last processed is displayed.

  16. Not required reason – The reason as to why the particular event is not communicated to the application.

  17. Added ID – The name of the user, who has done the HL7 transaction is displayed.

  18. Added Date – The date on which the transaction is carried out is displayed.

  19. Added WS No. – The workstation number from where the transaction was processed is displayed.

  20. Modified ID – The name of the person who has loaded the record.

  21. Modified Date – The date on which the record was loaded for transaction is displayed.

  22. Modified WS No – The workstation number from where the record was loaded is displayed.

  23. Accession number is a combination of primary keys for the internal applications.

  24. External Accession Number is a combination of primary keys for the external applications.

Click the hyperlinked ‘Message ID’ and a screen gets displayed. The following details are displayed:

  1. Reason – Depending on the message triggering event value, some of the messages need not be communicated to the application. In such cases, the ‘Not Required Reason’ is displayed by the system.

Depending on the privileges granted to the logged-in user, the following buttons are displayed:

  • Click [Replay] to replay the messages.

  • Click [Rebuild] to rebuild the messages.

  • Click [Message] to view the message details.

  • Click [Audit Log] to view the audit logs for the event.

  • Click [Debug Log] to view the debug logs for the event.

  • Click [Server Error] to view the background job errors that are generated while formatting the outbound messages.

  • Click [Communication Errors] to view the communication process errors. Both solicited and unsolicited interface process errors are displayed.

  • Click [Reset] to clear the entered data.
