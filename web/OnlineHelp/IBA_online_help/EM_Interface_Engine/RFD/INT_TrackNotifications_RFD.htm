Enter the following details:

 

  1. Notification Events – Select the event type related to the selected application ID from the drop-down menu.  For the application ID ‘laboratory’, the Event Type is ‘Critical Lab Result’.

  2. Notification Type – Select the notification type from the drop-down list. The notification type is ‘SMS’ for critical lab result event type.

  3. Refresh Interval – Select the required interval to auto refresh the records for the user to view the latest one. The default value is 10 mins which is the minimum value.

  4. Last Refresh Date and Time – Displays the last refresh date and time.

  5. Next Refresh Date and Time - Displays the next refresh date and time.

  6. Notification Status – Select the notification status from the drop-down list. The list values include All, Sent, Failed and Pending.

  7. Specimen Released From and To – The date range should not exceed 31 days.

a. ‘To’ date – Defaults the current system date and time.

b. From – Defaults the previous date and time (24 hours prior to "Specimen Released To).

c. From and To date cannot be greater than system date.
