Inpatient Billing Details

From this function you can Define the billing rule for inpatient transactions in the facility and Query to view the existing records in the Facility.

The Inpatient Billing Details function is used to define the charging rule for the services utilized from an Inpatient Nursing Unit. You have to define the charges for each bed type used, transfers from one nursing unit to another and for the transfers from one ward to another within the nursing unit. The amount to be collected from a patient when admitted to the facility has to be defined. The charges for the services utilized by an inpatient from the facility are calculated based on the rules defined.

Defining Inpatient Billing Details

From the Masters > Interfaced Modules Financial Dtls, double-click Inpatient Billing Details function.

The Inpatient Billing Details screen gets displayed.

  1. Enter the details in the mandatory (*) fields found in the following tab pages.

Addl.Charge Rule

Additional Charge Rule tab enables you to define a service charge rule to bill the additional services availed from the Facility. You have to define if a Doctor’s round charge, consumable charge and the workup are valid for an Admission Type. The charges are computed based on the billing service associated. You can also define the additional charges for a specific insurance policy. A patient to whom the respective policy is associated is applicable for the additional charges defined for the specific insurance policy.

Enter the following details to define the additional charges:

  1. Bed Type- Click [List of Values] to select an Admission Type. The values defined using the function Bed Type in the Inpatient and Day Care Management module get listed. The Description for the selected Admission Type gets displayed. This is a mandatory field.

  2. Doctor Rounds Charge – Select the check box, if the Doctor’s round charges has to be calculated. Clear the check box, if the Doctor’s round charges are not applicable.

  3. Billing Service - Click [List of Values] to select a Billing Service. The Doctor’s fee is calculated based on the price defined for the selected Billing Service. The description of the Billing Service is displayed.

  4. Consumable Charge - Select the check box, if the consumable charge has to be calculated for the Bed Type. Clear the check box, if the consumable charges are not applicable for the selected Bed Type.

  5. Billing Service - Click [List of Values] to select a Billing Service. The charges for the consumables are calculated based on the price defined for the selected Billing Service. The description of the Billing Service is displayed.

  6. Workup Charge on Admission Date - Select the check box, if additional workup charges are applicable for the selected Bed Type. Clear the check box, if additional workup charges are not applicable for the selected Bed Type.

  7. Billing Service - Click [List of Values] to select a Billing Service. The workup charge is calculated based on the price defined for the selected Billing Service. The description of the Billing Service is displayed.

  8. ICU Charge - Select the check box, if ICU charges are applicable for the selected Bed Type. Clear the check box, if ICU charges are not applicable for the selected Bed Type.

  9. Billing Service - Click [List of Values] to associate a Billing Service. The ICU charge is calculated based on the price defined for the associated Billing Service. The description of the Billing Service is displayed.

  10. Discharge Day Charge - Select the check box, if discharge day charges are applicable for the selected Bed Type. Clear the check box, if discharge day charges are not applicable for the selected Bed Type. The charge for discharge day is computed based on the Billing Service associated.

  11. Billing Service - Click [List of Values] to associate a Billing Service. The discharge day charge is calculated based on the price defined for the associated Billing Service. The description of the Billing Service is displayed.

  12. Serv Chrg excl adm and dschrg days Charge – Select the check box to charge the patient on all days except the admission and discharge days. The service price recorded for the associated Billing Service is charged. To compute this charge, the patient has to be admitted to the hospital for a minimum of three days.

An additional Billing Service can be associated to Doctor Rounds Charge, Consumable Charge, Workup Charge on Admission Date, ICU Charge, Discharge Day Charge, Service Charge excluding Admission and Discharge days charge. When the additional service is associated, both the service charges are computed for the respective Bed Type.

Policy Specific Additional Charge

You can define additional charges for specific Policy using this tab.

Enter the following details:

  1. Billing Group - Click [List of Values] to select a Billing Group. The values defined using the Billing Group function gets listed.

  2. Payer Group - Click [List of Values] to select a Payer Group . The Payer Groups associated to the selected Billing Group in the Insurance Management module gets listed.

  3. Payer - Click [List of Values] to select a Payer. The Payers associated to the selected Billing Group and the Payer Group in the Insurance Management module gets listed.

  4. Policy Type Code - Click [List of Values] to select a Policy Type. The Policy Types defined in the  Insurance Billing Manager module gets listed.

  5. Description - The Description for the selected Policy Type Code gets displayed.

  6. Charge Type – Select a value from the list. The system defined values are – ‘As per Policy Setup’ and ‘Charge to Patient’.

  7. As per Policy Setup – Select 'As per Policy Setup' to charge additional amount based on the policy coverage defined.

  8. Charge to Patient – Select ‘Charge to Patient’ to charge the additional amount to the patient for the selected policy.

  9. Billing Service - Click [List of Values] to associate a billing service. The additional charge for the Billing Group, Payer Group and Policy Type combination is calculated based on the price defined for the associated billing service.

  10. Description - The Description for the associated Billing Service gets displayed.

  11. ICU Charge - Select the check box, if ICU charges are applicable for the selected Admission Type. Clear the check box, if ICU charges are not applicable for the selected Admission Type.

  12. Billing Service - Click [List of Values] to associate a billing service. The ICU charge is calculated based on the price defined for the associated billing service.

  13. Round Charge Description – The description of the Billing Service selected for the Doctor Rounds Charge gets displayed.

  14. Workup Service Description - The description of the Billing Service selected for the Workup Charge on Admission Date gets displayed.

  15. Consumable Charge Description – The description of the Billing Service selected for the Consumable charges get displayed.

  16. ICU Service Description – The description of the Billing Service selected for the ICU charge get displayed.

Transfer Charge Rule

Transfer Charge Rule tab enables you to define the charges for the transfer occurring within the nursing unit and across nursing units within a Facility. Within the Nursing Unit, an inpatient is transferred from one location to the other or from one bed class to the other. The purpose of Inpatient transfers is to encompass a change in location or a practitioner. The service charge is dependant on the Bed Type. You can also select a Transfer Type as chargeable. The highest charge encountered for the bed, doctor rounds, service charge is used in computing the transfer charges. The highest charge is computed when the transfer takes place from one bed type or ward to another. The following tabs enable you to define the service charge for the transfers and define a transfer type as chargeable:

Service Charge by Bed Type

Transfer Type Applicability

Deposit/Prepayment Rule

The Deposit/Payment Rule tab enables you to enter the deposit applicability details for a Bed Class in a Nursing Unit.

  • Deposit - Deposit means the amount paid to the facility by the patient in advance during registration. The deposit amount can be much more than the actual charge for the service. The additional amount after deducting the service charge is compensated to the patient during discharge.

  • Prepayment - Prepayment defines the amount paid to the facility for the services in advance. The prepayment differs only in a way that the patient is not issued any refund. The amount is stabilized and adjusted towards the service availed.

The Deposit/Payment Rule tab enables you to define the Deposit/Prepayment Rule based on the Bed Class, Nursing Unit Type and Billing Group. You can define the rule using the tabs below:

Deposit Amount by Bed Class

Deposit Applicability by Nursing Unit

Advance Payment by Billing Group

Bill Type for Nursing Unit

Bill Type for Nursing Unit enables you to associate a Bill Type and a Nursing Unit Type to a Nursing Unit. Each Nursing Unit supports several types of patients. A general bill format has to be defined for a particular patient class admitted to the facility.

Enter the following to associate a Bill Type to the Nursing Unit:

  1. Nursing Unit – Click [List of Values] to select a Nursing Unit. The values defined using the Nursing Unit function in the Inpatient and Day Care Management module get listed. This is a mandatory field.

  2. Description - The description for the respective Nursing Unit gets displayed.

  3. Nursing Unit Type – The Nursing Unit Type associated to the selected Nursing Unit gets displayed.

  4. Description - The description for the Nursing Unit Type gets displayed.

  5. Bill Type – Click [List of Values] to select a Bill Type. The values defined using the Bill Type function under the Accounting Entity Setup menu get listed. This is a mandatory field.

  6. Description - The description for the selected Bill Type gets displayed.

  1. Click [Commit Form] or press [F10] to save or [Clear Record] to clear the entered data.

  2. Click [Execute Query] or press [F8] to search for the existing records within the facility.
