Service Markup Definition

From this function you can Define the service markup against the Billing group or Customer.

This function allows you to define markup service against Billing group or Customer for the selected facility. Based on the set up defined against Billing group and customer, the system calculates the Factor value.

The Mark up entry will be shown in a single entry in Discharge Bill Cum Receipt / Undercharge Bill Cum Receipt screen. This change is applicable only for IP / Day Care.  

The Mark-up entry is shown in Draft report / Discharge bill / Undercharge bills. The System date (Date on which bill is generated) is shown against the mark-up entry.

Mark up service will be charge based service.

Defining Service Markup

From the Masters, select Multi Pricing > Service Markup Definition.

The Service Markup Definition screen gets displayed.

  1. Enter the details in the mandatory (*) fields found in the following tab pages:

Billing Group

Billing Group denotes various types of payments the Facility accepts from the patients or the customers paying on behalf of the patients. A Billing Group is assigned to a patient during registration. A patient bill is generated based on the prices set for the billing services associated to a Billing Group. You can define the Billing Group to be valid only for patient registration, for Outpatient visit registration or for an Inpatient admission. Service Level eligibility is defined for Inpatients. A patient is linked to a Billing Group during registration or admission.

 

Enter the following details in the Billing Group tab:

  1. Facility – By default, the logged in facility gets displayed.

  2. Billing Group – Select the required billing group from the lov. The lov populates billing group value based on the data defined in Billing Group masters. This field is mandatory.

  3. Indicator – Select the indicator value from Lov. The indicator can be Service Group, Service Classification and Billing Service. This field is mandatory.

  4. Service - On selecting the Service Group or Service Classification, the system populates the corresponding code in the Service field. You can also enter the service group code or service classification code manually. This field is mandatory.

  5. Description - The corresponding short description of the selected service group / classification gets displayed.  

  6. MarkUp Service – Select the required markup service from the Lov. This field is mandatory.

  7. Mark Up % - Enter the mark up percentage value in numeric format. The minimum value is 1 and the maximum value is 100. You cannot enter decimals and negative values. This field is mandatory.

When an invalid entry is entered and on querying the system displays the message ‘Query caused no records to be retrieved, please re-enter’ in the status bar.

  1. Apply On - The drop-down list values include Gross and Net. You can select either Gross or Net. This field is mandatory.

  2. Valid From – Enter the valid date in DD/MM/YYYY’ date format and select the valid date by clicking the calendar. This field is mandatory. Valid From date should be Future date.

  3. Valid To -  Select the valid to date by clicking the calendar. This field is non-mandatory field. ‘Valid To’ date should be greater than current date and should not allow current date / backdates.

Customer

Enter the following details in the Customer tab:

  1. Facility - By default, the logged in facility gets displayed.

  2. Billing Group – The first billing group in the alphabetical order gets displayed. Select the required billing group from the lov. The lov populates billing group value based on the data defined in Billing Group masters. This field is mandatory.

  3. Indicator - Select the indicator value from Lov. The data is populated from the Service Group / Service Classification masters. On selecting the Service Group or Service Classification, the system populates the corresponding code in the Indicator field. You can also enter the service group code or service classification code manually. This field is mandatory.

  4. Service - Select the required service from the Service Group lov. This field is mandatory.

  5. Description – The corresponding short description of the selected service group / classification gets displayed.  

  6. MarkUp Service - Select the MarkUp Service from the lov. This field is mandatory.

  7. Mark Up % - Enter the mark up percentage value in numeric format. The minimum value is 1 and the maximum value is 100. You cannot enter decimals and negative values. This field is mandatory.

  8. Apply On - The drop-down list values include Gross and Net. You can select either Gross or Net. This field is mandatory. By default, the value 'Gross' is displayed.

  9. Valid From - Enter the valid date in DD/MM/YYYY date format or select the valid date by clicking the calendar. This field is mandatory.

'Valid From' date allows future date and should not allow backdate.

Valid From date should be greater than System date.

  1. Valid To -  Select the valid to date by clicking the calendar. This field is non-mandatory field.

'Valid To' date should be greater than current date and should not allow current date / backdates.

  1. Click [Commit Form] to save the records or [Clear Record] to delete the data entered.

  2. Click [Execute Query] to query for the existing records.
