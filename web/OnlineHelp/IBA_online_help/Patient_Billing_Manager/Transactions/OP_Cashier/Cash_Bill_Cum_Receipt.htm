Cash Bill Cum Receipt

From this function you can Generate visit bills and receipts for an Outpatient, Emergency or an External patient admitted in the facility. You have to confirm the services associated to the patient in order to generate a bill or a receipt.  

Cash Bill Cum Receipt function is used to generate and print visit bills and receipts for an Outpatient, Emergency or an external patient episode conclusion in the Facility. Cashier has to accept the payment against the bills and specify the details in the cash bill-cum-receipt. An authorized User has to have privilege to generate the receipts, so the Cashier can use the function to generate and print the bills for the services availed by an Outpatient, External or an Emergency patient from the facility.  A discount report is also printed along with the bill details, if the patient has availed a discount for the services utilized from the facility. Separate receipts are generated for payment made using various Settlement Types. For example, if the bill amount is 1,000/- and patient chooses to settle 500/- via Credit Card and 500/- using Cash, two separate receipts are generated to indicate multiple payment modes. Consulting Practitioner's name is printed in the bill generated. If there are multiple consulting practitioners, the first practitioner's name is printed. A report is generated on settling the bill amount. The report generated displays the name of the Cashier, receipt type and number generated, mode of payment selected by the patient, cheque or credit card number, amount settled and the total number of receipts generated for various transactions. The bed charge computation rule differs based on the value selected for 'Check Out Basis' in the 'BL Parameter' function. A single bill is generated, if there are multiple payers associated to a patient, listing the coverage amount for each service, agreed to be covered by the payer. It is mandatory to capture additional ID proof details to generate a Payer bill if the respective check boxes are selected in 'BL Parameter' function. You cannot generate a Payer bill, if the details are not captured, but the patient bill can still be printed. The bills printed will contain additional details that are captured. You can print receipts and report in local language. If the local language is not selected, the report and receipts are printed in English, which is set as the default language.

The details are printed from right to left when you select the local language as 'Arabic'.

 

Two separate bills are generated if a part of the bill amount is settled by the Payer. A Payer and Patient Bill generated through this function displays the drug token number, patient's name, name of the hospital, Policy owner's name (name as recorded in the 'Credit Authorization' field), Service Group, total amount, charge amount, total amount, signature and designation of the logged in user, report print date and time and the card type and number through which the amount is settled. The bill/receipt format that is printed during patient discharge is based on the format defined for the respective Episode Type through the Bill Print Format function. If you re-print the bill through the 'Re-print Bills' function, then 'COPY' is displayed on the bill to indicate that the bill generated is a copy of the original bill. The Surgeon's name is printed against each individual service on the report that is generated. You can also view the Hospital Details, Patient's Name, Visit Registration Date, Service Groups, Print Groups, total charge amount, Payer and Patient Payable amount, total amount, discount amount, outstanding amount, remaining deposit amount, policy number and name of the authorized person who has generated the bill is displayed. The generated report displays the subtotal of each Print Service separately.  

This is valid only for a detailed bill and it is not printed, if a summarized bill is generated. The name of the Surgeon captured through the 'OT Slate' function in 'Operation Theatre' module for the respective patient encounter is displayed. The attending surgeon's name is displayed, if the Surgeon's name is not captured in OT Slate function.

 

Process Flow Diagram

Generating Cash Bill-Cum-Receipt

From the Transactions > OP Cashier, double-click Cash Bill Cum Receipt Function.

The Cash Bill Cum Receipt screen gets displayed.

  1. Enter a Patient ID or select the Patient ID using the User Option. The patient details get displayed on selecting the Patient ID. <Related Field Description>

  2. The details get displayed under the following tab pages for the selected patient:

  3. Click [Commit Form] to print the bills/receipts or [Clear Record] to clear the data.

  4. An additional screen 'Additional ID Proof' pops-upon clicking [Commit Form] if the Additional ID capture is set as mandatory in 'BL Parameters' function for payer bill generation.

    Enter the following details to generate a Payer bill:

    1. Alternate ID Type - Click [List of Values] to select an Alternate ID Type for the patient. The 'Default ID Type' selected in 'BL Parameters' function is displayed, by default.

    2. ID - Enter the ID in alphanumeric format of 30 characters maximum length. The data can be recorded in English or Thai language.

    3. Patient Name - Enter the patient's name in alphanumeric format of 100 characters maximum length. The data can be recorded in English or Thai language.

    4. Remarks - Enter any additional details in alphanumeric format.

    5. [Continue] - Click [Continue] to generate a Payer bill.

 

You have to login to the Cash Counter to generate and print visit bills and receipts.
