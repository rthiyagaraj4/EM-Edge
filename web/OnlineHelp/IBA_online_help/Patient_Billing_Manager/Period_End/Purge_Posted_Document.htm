Purge Posted Document   

From this function you can Delete the posted documents from the database. The posted documents are permanently deleted from the database.

Purge Posted Document function is used to delete the posted documents. You have to specify a date and the documents posted until the specified period are deleted from the database. Several validations are carried out before deleting the posted documents from the database.

System checks for the following validations before purging the documents:

  • The documents in Originating Transactions for the period less than or equal to the accepted processing period has to be posted.

  • The documents in Allocating Transactions for the period less than or equal to the accepted processing period has to be posted.

  • There should be no receipts or refund documents for the period less than or equal to the accepted processing period.

  • There should be no distributions in General Ledger interface, which has not been interfaced to General Ledger for the period less than or equal to the accepted processing period.

  • Post the bills dated as per the processing period.

  • If any of the above validation fails, system aborts the purge process. If all the above validations are successful, system proceeds with the following transactions:

  • All posted documents less than or equal to the current processing period are deleted from Originating Transactions, Allocating Transactions, Receipts and Direct Billing Entries.

  • General Ledger distributions until the processing period are deleted.

Purge Posted Document

From the Period End, double-click Purge Posted Document.

The Purge Posted Document screen gets displayed.

  1. Enter the month and the year. The documents posted until the selected month and year gets deleted. This is a mandatory field.

  2. Click [Run Process] to delete the documents permanently from the database. A message pops-up requesting confirmation to purge the documents. Click [Yes] to confirm, or [No] to cancel the process.
