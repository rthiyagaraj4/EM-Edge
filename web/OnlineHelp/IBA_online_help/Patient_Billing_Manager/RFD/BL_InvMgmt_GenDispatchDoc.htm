Enter the following details to generate Dispatch Document:

 

  1. Cut-off Date – Click Calendar function to select a Cut-off Date. System displays the current date, by default. The outstanding bills and the invoice notes associated to a Payer Group, Payer and Policy Type combination gets listed until the date selected.

  2. Payer Group – Click [List of Values] to select the Payer Group. The values defined using the ‘Customer Group’ function under the AR Setup menu gets displayed. The invoices generated for the outstanding bills of the patients associated to the selected Payer Group get listed. This is a mandatory field.

  3. Payer – Click [List of Values] to select the Payer. The values defined using the ‘Customer’ function under the AR Setup menu gets displayed. The invoices generated for the outstanding bills of the patients associated to the selected Payer get listed. This is a mandatory field.

  4. Policy Type - Click [List of Values] to select a Policy Type. The outstanding bill details associated to the selected Policy Type gets listed. You can select the required invoices to generate the dispatch document for the selected invoices.

 

Episode Type

Select an Episode Type to search for the respective Episode Types with the outstanding bill details. You can select from the list of outstanding bills of an Episode Type associated to the Customer to generate the invoices for the selected Episode Type.

  • All – Select ‘All’ to search and to select the invoices of all the Episode Types in the facility.

  • Outpatient – Select ‘Outpatient’ to search and to select the invoices of the Outpatients in the facility.

  • Emergency – Select ‘Emergency’ to search and to select the invoices of the Emergency patients in the facility.

  • Daycare – Select ‘Daycare’ to search and to select the invoices of the Daycare patients in the facility.

  • Inpatient – Select ‘Inpatient’ to search and to select the invoices of the Inpatients in the facility.

  • External – Select ‘External’ to search and to select the invoices of the External patients in the facility.

  1. Breakdown By Policy Type – Select the check box to breakdown the invoices or the debit notes based on the Policy Types. Clear the check box, if the breakdown is not required.

 

Dispatch Letter Format Parameters

  • Body Header – Select the check box, if the ‘Dispatch Letter Body Header’ defined using the ‘Invoice Process Parameters’ function has to be printed in the dispatch document.

  • Body Footer – Select the check box, if the ‘Dispatch Letter Body Footer’ defined using the ‘Invoice Process Parameters’ function has to be printed in the dispatch document.

  • Foot Note – Select the check box, if the ‘Dispatch Letter Foot Note’ defined using the ‘Invoice Process Parameters’ function has to be printed in the dispatch document.

  • Print Text – Select the check box to print text in the dispatch document. Clear the check box, if the text is not to be printed in the dispatch document.

  • Foreign Currency – Select the check box to print Foreign Currency in the dispatch document. Clear the check box, if the Foreign Currency is not to be printed in the dispatch document.

  • Signatory – Select the check box, if the details of the authorised signatory have to be included in the dispatch document.

  1. [Contact Address] – Click [Contact Address] to view or to record the contact address of the Customer.

  2. [Letter Format] – Click [Letter Format] to define the Letter Format for the dispatch document. The format defined using ‘Invoice Process Parameters’ function gets displayed. You can update the letter format to dispatch the document.

  3. [Refresh] – Click [Refresh] to enter the new the details.

 

Invoice/Debit Note

  1. Select All – Select the check box to print dispatch documents for the invoices and the debit notes listed based on the search criteria selected. Clear the check box to print the dispatch document for the selected records.

  2. Select – Select the check box to print the dispatch document for the selected records. Clear the check box, if the dispatch document is not to be printed for the selected record.

  3. Invoice Number – The invoices generated for outstanding bills of the patients associated to the selected Payer Group, Payer and Policy Type combination get listed. An invoice number is automatically generated while generating the invoice for the outstanding bills using the functions ‘Generate Invoice – OP/EM’, ‘Generate Invoice – IP/DC’, ‘Generate Debit Note/Credit Note – OP/EM’ or ‘Generate Debit Note/Credit Note – IP/DC’. You can select the records to print the dispatch document for the selected records.

  4. Document Date – The date when the invoice document was generated gets displayed.

  5. Trx Type – The transaction type selected while generating the invoice gets displayed.

  6. Policy Type Code – If the Policy Type is selected while searching for the records, respective Policy Type gets displayed. If the Policy Type is not selected, the Policy Types associated to the Payer and the Payer Group gets listed.

  7. Due Date – The Due Date defined to clear the payment gets displayed.

  8. Gross Amt – The Gross Amount for each of the Invoice gets displayed.

  9. Markup Amt - The Markup Amount for each of the Invoice gets displayed. The markup amount indicates the amount, which is remaining after the deduction of amount for the services from the total agreed amount.

  10. Discount Amt – The Discount Amount for each invoice is displayed.

  11. Payable Amt – The Amount payable by the Payer Group for the selected invoice is displayed.

  12. Patient ID – The name of the patient who has utilized the services and owes the amount to the facility gets displayed.

  13. Episode ID – The Episode ID against which the invoice is generated for the outstanding bills get displayed.

  14. Dispatch Document Ref – The original document reference associated to the records while generating the invoice gets displayed.

  15. Dispatch Date – The date when the invoice was dispatched to the Payer gets displayed.

  16. Printed - The check box is selected if the invoice records are printed.
