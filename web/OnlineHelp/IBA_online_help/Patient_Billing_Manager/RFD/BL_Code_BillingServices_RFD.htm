Enter the following details to define a Billing Service:

 

  1. Billing Service – Enter the Billing Service code in alphanumeric format of 10 characters maximum length. The minimum character length is one and the maximum length is 10 characters. The Billing Service code is a combination of the Service Groups/Classification code, that is, the initial characters of the Billing Service code has to begin with the Service Groups/Classification code. This is a mandatory field.

  2. Service Group – The Service Group field gets populated automatically when the Billing Service code is set. The Service Group code associated to the first two characters of the Billing Service code is displayed. The Service Groups defined using the Service Groups/Classification function gets displayed.

  3. Service Classification - The Service Classification field gets populated automatically based on the Billing Service code selected.

  4. Long Description - Enter the Long Description in alphanumeric format of 40 characters maximum length. This is a mandatory field.

  5. Short Description - Enter a Short Description for the Billing Service in alphanumeric format of 15 characters maximum length. This is a mandatory field.

  6. UOM – Click [List of Values] to associate Unit of Measure to the Billing Service. The selected Billing Service is charged accordingly based on the Unit of Measure associated to the patient records during transaction functions. The values defined using the function Unit of Measure under the ‘Materials Setup’ menu in the Application Masters module get listed.

  7. Reimbursable – Select the check box, if the service is reimbursable. You can reimburse the amount paid by the patient towards the service, if the check box is selected. The check box is selected, by default. Clear the check box and you cannot reimburse the amount for the selected service.

  8. Reimbursable Type – Select a value from the list. The values for Reimbursable Type are pre-defined:

  9. Full – Select Full to refund the entire amount paid by the patient towards the service.

  10. Partial – Select Partial to refund only a fraction of the service amount. A part of the amount is deducted for the service availed and the remaining amount is refunded to the patient/payer. You have to specify the amount to be reimbursed in the field ‘Reimbursable Amount’ under the ‘General’ tab. The Charge is specified only, if the ‘Charge Type’ is set to ‘Charge’.

  11. Not Applicable – Select Not Applicable, if the service is not reimbursable.

  1. SAC Code - Enter SAC Code in the alphanumeric format of 20 characters maximum length. You cannot enter special characters. You can define single SAC Code for more than one billing service.

  2. Override – Select the check box to override the reimbursable or the non-reimbursable amount for Cash patients. If the Override check box is selected and the ‘Reimbursable’ check box is cleared, the selected service is reimbursable. The reimbursable amount cannot be overridden, if the check box is cleared.

 

Charge Type

Billing Service can be Rate based or Charge based.

 

  1. Rate – Select Rate, if the Billing Service is Rate based. This indicates that the charges for the services are pre-defined. The charge for the services utilized is computed during bill generation based on the pre-defined charges.

  2. Charge – Select Charge, if the Billing Service is Charge based. This indicates that the charges are not pre-defined and the charges vary depending on the quantity of service used. Price setup is defined based on the following pre-defined Multi-Pricing groups:

  3. Customer Specific Prices

  4. Billing Group Specific Prices

  5. Price Class Specific Prices

  6. Standard Price

 

The multi-pricing concept works based on the priority level. On selecting a service, the system checks, if the Customer Specific Price is defined for a service. If the Customer Specific price is defined for a selected service, the charge is calculated based on the price definition for each patient class for a specified date range and the associated facility.

If the Customer Specific Price is not defined, the system checks for Billing Group Specific Price. If the Billing Group Specific Price is defined, the charge is calculated based on the price definition for each patient class for a specified date range and the associated facility.

If the Customer Specific Price and the Billing Group Specific Price are not defined, system checks for Price Class Specific Price. If the Price Class Specific Price is defined, the charge is calculated based on the price definition for each patient class for a specified date range and the associated facility.

If the Customer Specific Price, Billing Group Specific Price and Price Class Specific Price are not defined, the charge is computed based on the Standard Price defined. An error message gets displayed, if the Standard Price is not defined for the selected service.

If the Charge Type is set to Charge, the Multi-Pricing groups cannot be defined; only the Account Department/Location can be set. To define and set the above mentioned Multi-Pricing groups, select Rate under the Charge Type category.
