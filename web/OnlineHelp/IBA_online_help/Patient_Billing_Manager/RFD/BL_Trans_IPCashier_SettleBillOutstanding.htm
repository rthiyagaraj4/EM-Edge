Enter the following details to settle the outstanding details:

 

  1. Patient No – Enter a valid Patient Number. The outstanding bill amount for the relevant patient is displayed. This field is mandatory.

  2. Facility Id – Select a Facility ID from the list.

  3. Option – The user can either select “Receipt” or “Refund” or “Both”.  The relevant option can be selected only if the logged Cash Counter has access to both the options.

  4. Patient Remarks – The remarks for the relevant Patient ID is displayed.

  5. Bill Nature – The user can select either “Patient Bill” or “Payer Bill” or “Both” from the list. Select one of the options from the list.

  6. Sex – The gender of the relevant patient is displayed.

  7. Age – The Age of the patient is displayed.

  8. Downtime Entry – Select this check box to enter the details of the documents when the server is down

  9. Document No – Click [List of Values] to select a Downtime Receipt Document Number. The Document Numbers selected as Receipts in the Document Types function gets listed.

  10. Date – Select the date of bill to be generated using calendar function.

  11. Reason – Enter the comments for bill generation. The Reason lookup values will be populated from Reasons for Billing Actions in Other Masters.

 

Outstanding details

The Receipt Nature, Document Type, Document Number, Customer and Episode Number gets displayed. The description of the Document Type and total outstanding amount are displayed.

  1. Print Bill – Select the check box, if the Settlement details are to be printed.

  2. Print Service Item List – Select the check box, if the Service Item List is to be printed along with the Settlement Details.

  3. Print Specific Service Item List - Select the check box to print a service item list while settling the bill amount. Clear the check box, if printed bill is not required. The check box is accessible only if the 'Print Specific Service Item List' check box is selected under the 'Bill-Cum-Receipt' tab in 'BL Parameters' function.

  4. Select – Select the check box, if the Settled Amount has to be specified. If the check box is cleared, you cannot specify the Settled Amount.

  5. Settled Amount – Enter the amount in numeric format to indicate the amount settled by the patient or the customer.

  6. Rec Type - Click [List of Values] to select a receipt type. Respective Receipt Type is generated with the receipt number when the bill is settled.

 

Settle By Services

Press the [Tab] key to switch to the next tab, Settle by Services. The Outstanding amount for the individual services are displayed. You  can only select the option and specify the Settled Amount for the individual services. Service Date, document reference number recorded for the service, Billing Service associated and the description, token number if any and the net amount are displayed.

  1. Select - Select the check box to settle the bill amount for respective service. Clear the check box, if you do not want to settle the bill amount.

  2. Settled Amount - Enter the amount to be settled against the respective service.

 

Settlement Details

Press the [Tab] key to switch to the next tab, Settlement Details. The Total Outstanding Amount and the Total Settlement Amount is displayed.

 

  1. Total Outstanding Amount - The Total Outstanding Amount of the patient for the services utilized from the facility is displayed.

  2. Total Settlement Amount - The total amount settled by the patient or customer against the bills generated is displayed.

  3. Received Thru Mail - The check box is selected if the payment is received through mail. The check box is selected or cleared based on the details recorded during transaction.

  4. SrNo - Enter the serial number in numeric format of three digits maximum length.

  5. Amount – The amount paid by the patient. The amount should be greater than or equal to the total settlement amount.

  6. Settlement Type - Click [List of Values] to select the Settlement Type. The description for the selected Settlement Type is displayed.

  7. Cash Given – Enter the Cash amount given by the patient.

  8. Cash Returned – Specify the amount refunded after deducting the service amount.

  9. Cheque/Credit Card No – Enter the Cheque or the Credit Card number if the Settlement Type is set to one of these.

  10. Date – Click the Calendar icon to set the date when the settlement is made.

  11. Bank/Credit Card Code – Click [List of Values] to select the Bank or the Credit Card Company Code.

  12. Branch Name – The name of the relevant Bank or the Credit Card Company branch is displayed.

  13. Online Approval – Select the check box, if an Online Approval is suitable for the clearance of the payment.

  14. Batch No. – Enter the Batch Number in alphanumeric format of eight characters maximum length. The Batch Number identifies the set in which the patient settlement was approved.

  15. Approval Code – Enter the Approval Code in alphanumeric format of 30 characters maximum length.

  16. Sale Draft No - Enter the Sale Draft Number in alphanumeric format of 20 characters maximum length.

  17. Payer’s Name – The Payer’s Name recorded during the patient registration gets displayed. Enter the Payer’s name in alphanumeric format of 40 characters maximum length to update the existing Payer’s Name or record a new Payer Name.  

  18. Payer Ref Id/Deposit Ref No – The Payer Reference ID or the deposit reference ID recorded gets displayed. You can update the existing Payer Reference ID in numeric format.

  19. TID No. - Enter the transaction reference ID in alphanumeric format.

  20. CC Issued Bank - Enter the name of the Bank that has issued the credit card.

  21. Total – The total amount entered to be settled under the ‘Outstanding Details’ tab gets displayed. You cannot update the total amount.

  22. Narration – The remarks entered while recording the financial details get displayed. You cannot update the narration.

  23. Doc Type/No – The Document Type and the number generated for the selected transaction gets displayed.

  24. Document Date – The date when the document type and number was generated for the respective transaction gets displayed.

  25. Cash Counter – The Cash Counter ID and the shift to which the user has logged in gets displayed.

  26. Post Mth/Yr – The month and the year when the bill details are posted gets displayed.

 

Outstanding Deposit Details

The document type and number gets displayed along with the deposit amount collected from the patient during registration. The document type and number generated while accepting the deposit from the patient, settlement type and date, adjusted amount and the pending amount are listed.

  1. Document Number – The document type and number generated during deposit collection gets displayed.

  2. Doc Date – The date when the document was generated against the deposit collection gets displayed.

  3. Settlement Type – The Settlement Type using which the deposit was collected gets displayed.

  4. Pending Amt – The amount pending in the deposit gets displayed.

  5. Adjusted Amt – The amount from the deposit which are adjusted towards the services gets displayed.

  6. Adjust Deposit - Select the check box to adjust the bill amount against the deposit amount. Clear the check box, if the bill amount for the services should not be adjusted with the deposit amount. On selecting the check box, the outstanding deposit details of the patient are displayed against which the service amount can be adjusted.

  7. Select - Select the check box to settle the bill amount with the outstanding deposit amount. The outstanding deposit amount details are highlighted in green. On selecting the check box, the amount for the selected service gets displayed under 'Adjusted Amt'. Click [Commit] on entering the details. A message 'Please Confirm the Printer is ready' gets displayed. The receipt type and the receipt number gets displayed on clicking [Confirm]. Click [OK] to save the adjustment details.

  8. Remarks - The deposit remarks captured in the ‘Package Deposit’ and ‘Enter OP Receipts/Refund’ screens is displayed on selecting ‘Receipt’ in the ‘Option’ drop-down list. You can modify the data.

 

The Batch Number, Approval Code, Sale Draft Number, Payer’s Name, Payer’s Reference ID, Document Type/Number, Document Date, Cash Counter, and the Posting Month/Year is displayed automatically. The deposit adjustments done gets reflected in the 'Acknowledge Cashier Transactions' function.

 

Only when you select the ‘Downtime Entry’ check box, the Document No, Date and Reason fields are enabled.
