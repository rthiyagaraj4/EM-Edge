Enter the following details to define the relative weight for a payer:

 

  1. Payer Group - Click [List of Values] to select the Payer Group. The relative weight factor is applicable for the selected Payer Group.  The price for the DRG code is calculated as a product of the quantity of DRG code used and the relative weight factor defined. The values defined using the 'Customer Group' function in the AR Setup menu get listed. This is a mandatory field.

  2. Payer - Click [List of Values] to select the Payer. The relative weight factor is applicable for the selected Payer associated to the Payer Group. The values defined using the 'Customer' function in the AR Setup menu get listed. This is a mandatory field.

  3. Policy - Click [List of Values] to select the Policy. The relative weight factor is applicable for the selected Policy Type associated to the Payer Group and the Payer. The values defined using the 'Policy Types' function in the Insurance Management module get listed. This is a mandatory field.

  4. DRG Code - Click [List of Values] to select the DRG Code. The relative weight factor is applicable for the selected DRG Code. The values defined in the Medical Records function  get listed.  This is a mandatory field.

  5. Description - The description for the selected DRG Code gets displayed.

  6. Effective [From-To] - Click the Calendar function to select the date range. The relative factor is applicable for the date range selected. The 'From' date is mandatory. The 'From' date selected has to be greater than the system date. If the 'To' date is not specified, the relative weight is valid for unlimited period.

  7. RW Factor - Enter the relative weight factor in numeric format with a maximum of six digits. The value has to be of the form 9.9999. This is a mandatory field.
