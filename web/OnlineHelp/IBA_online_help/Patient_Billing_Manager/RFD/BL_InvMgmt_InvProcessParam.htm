Enter the following details to define invoice process parameters:

 

Facility Specific Parameters

Dispatch Letter Body Header

  1. Line (1 – 5) - Enter the text, which is supposed to appear as the header of the letter body in the dispatch letter. Line 1 and Line 2 are mandatory.

 

Dispatch Letter Body Footer

  1. Line (1 – 5) - Enter the text, which is supposed to appear as the footer of the letter body footer in the dispatch letter. Line 1 and Line 2 are mandatory.

 

Dispatch Letter Foot Note

  1. Line (1 – 5) - Enter the text, which is supposed to appear as the footer in the dispatch letter.

 

Pre-Invoice

You can record the default values for pre-invoice generation. This is a one-time setup and you cannot edit the values.

  1. Payer Group - Click [List of Values] to select a Payer Group. The values defined through the 'Customer Group' function are listed. The description of the Payer Group is displayed. This is a mandatory field.

  2. Payer - Click [List of Values] to select a Payer. The description of the Payer is displayed. The values defined through the 'Customer' function are listed.  This is a mandatory field.

  3. Pre Invoice Legend - Enter the legend for pre-invoice. The legend recorded is printed on the Pre-Invoice Report.

  4. Finalize Reason - Click [List of Values] to select a reason for finalizing pre-invoice. The selected value is displayed by default when you finalize the pre-invoice generation.

 

Pre-Invoice Applicable Billing Groups

You can select and associate the Billing Groups that are valid for pre-invoice generation.

  1. Blng Group - Click [List of Values] to select a Billing Group. The selected Billing Group is valid for pre-invoice.

  2. Description - The description of the selected Billing Group is displayed.

 

Invoice

  1. Print Label for DRG - Enter the label for DRG in alphanumeric format of 40 characters maximum length. The label is generated tin the report that is printed during invoice generation. The multiple DRG services are used by the patient are displayed under a single service code.

  2. Print Label for Receipt - Enter the label for receipt in alphanumeric format of 40 characters maximum length.

  3. Print Label for Rounding - Enter the label for receipt in alphanumeric format of 40 characters maximum length.

  4. Invoice Printing Rule - Select a value from the list to define the invoice printing rule. The system-defined values are 'Print Invoice Automatically' and 'Mark Only As Printed'.

  5. Print Invoice - Select a format from the list. The system-defined values are 'Format 1', 'Format 2' and 'Format 3'.

  6. Reimbursable Legend for Manual Adjustment - Enter the legend to be printed for manual reimbursable adjustment.

  7. Compute Agreed Amount as Separate Line - Select the check box to compute and display the agreed amount as a separate line. Clear the check box to compute and display the agreed amount in the same line.

  8. Prothesis and Equipment List to be printed during invoice printing - Select the check box to print a report for the consumables used by the patient, if any. On selecting the check box, the report is printed with the details while generating the invoice or using the 'Print/Reprint Invoice' function. Clear the check box, if the report is not required for the consumables or the equipments used by the patient.

 

Invoice Body Line1-Invoice Body Line3

  1. Line [1 - 3] - Enter the remarks in alphanumeric format that has to be printed in the invoice generated.

 

Invoice Remarks

  1. Line [1-6] - Enter the remarks that has to be printed in the invoice generated. You can record remarks with a  maximum of 200 characters in each line.

 

Dispatch Letter

  1. Authorized Signatory – Enter the name of the Invoice Authorized Signatory in alphanumeric format of 40 characters maximum length. This indicates the name of the authorized person who is in charge of the invoices dispatched to the Customers. This is a mandatory field.

  2. Authorized Signatory Desg - Enter the designation of the Authorised Signatory in alphanumeric format of 30 characters maximum length. This indicates the designation of the authorized person who is in charge of the financial department.

  3. Dispatch Document Printing Rule -  Select the dispatch document printing rule. The system defined values are - 'Print Dispatch Document Automatically' and 'Mark only as Printed'. Select 'Print Dispatch Document Automatically' to print the dispatch document automatically after generation. Select 'Mark only as Printed' to mark the dispatch document as printed.

  4. Generate Dispatch Document for Printed Invoice Only - Select the check box to generate dispatch document for printed invoice. The dispatch document is not generated for printed invoice if you clear the check box.

 

Signatory

Invoice

  1. Authorized Signatory – Enter the name of the Invoice Authorized Signatory in alphanumeric format of 40 characters maximum length. This indicates the name of the authorised person who is in charge of the invoices dispatched to the Customers. This is a mandatory field.

  2. Authorized Signatory Desg - Enter the designation of the Invoice Authorized Signatory in alphanumeric format of 30 characters maximum length. This indicates the designation of the authorised person who is in charge of the invoices dispatched to the Customers.

  3. Print Logged in User as Issuing Signatory - Select the check box, if the logged in user name has to be printed as the issuing signatory.

  4. Issuing Signatory - Enter the name of the Issuing Signatory in alphanumeric format of 40 characters maximum length. The recorded name is printed as the issuing signatory, if the 'Print Logged in User as Issuing Signatory' check box is cleared.

  5. Issuing Signatory Desg - Enter the designation of the Issuing Signatory in alphanumeric format of 30 characters maximum length. The recorded designation is printed along with the Issuing Signatory name, if the 'Print Logged in User as Issuing Signatory' check box is cleared.

 

Debit/Credit Note

  1. Authorized Signatory – Enter the name of the Authorized Signatory for Debit or Credit Notes in alphanumeric format of 40 characters maximum length. This indicates the name of the authorized person who is in charge of the debit/credit notes dispatched to the Customers. This is a mandatory field.

  2. Authorized Signatory Desg - Enter the designation of the Authorized Signatory for Debit or Credit Notes in alphanumeric format of 30 characters maximum length. This indicates the designation of the authorized person who is in charge of the invoices dispatched to the Customers.

  3. Print Logged in User as Issuing Signatory - Select the check box, if the logged in user name has to be printed as the issuing signatory.

  4. Issuing Signatory - Enter the name of the Issuing Signatory in alphanumeric format of 40 characters maximum length. The recorded name is printed as the issuing signatory, if the 'Print Logged in User as Issuing Signatory' check box is cleared.

  5. Issuing Signatory Desg - Enter the designation of the Issuing Signatory in alphanumeric format of 30 characters maximum length. The recorded designation is printed along with the Issuing Signatory name, if the 'Print Logged in User as Issuing Signatory' check box is cleared.

  1. Auto Finalize – Select the check box to finalize the dispatch document automatically. Clear the check box, if the dispatch document has to be manually finalized using the function ‘Finalize Dispatch Document’.

  1. Auto Acceptance – Select the check box to finalize the payer acceptance automatically. Clear the check box to finalize the payer acceptance manually using the function ‘Finalize Payer Acceptance’ when the payer makes the payment based on the invoice dispatched.

  2. Outstd Doc in Annex Only – Select the check box to print the dispatch letters in the format set up. Clear the check box, if the document is to be printed in any other format.

  3. DRG Amount in Detail? - Select the check box to print the DRG amount in detail in the invoice. Clear the check box to print summarized DRG details in the invoice.

  4. Bill Printing Required - Select the check box to print the bill before generating the invoice. The bill generation becomes mandatory before invoice generation, if the check box is selected. Clear the check box, if the bill generation is not mandatory before invoice generation.  

  5. Print Renal Invoice - Click [List of Values] to select the Renal Operation Code. The code selected is displayed, by default while registering the patient for renal treatment.

  6. Adjustment Document - Click [List of Values] to select the Adjustment Document Code. The code selected is displayed while generating the invoice document for the patient.

 

Customer Specific Parameter

You can define the parameters to create the Customer Specific dispatch letters to dispatch the invoices to the Customers. You can define the contents to appear in the header, body and footer of the dispatch letter. If a Customer Specific Parameter is not defined, the invoices are dispatched using the letters with the Facility Parameters defined. You have an option to copy the header, footer and the body contents defined for the Facility to appear in the respective fields under the tabs in the Customer Specific Parameter tab.

 

Enter the following details:

  1. Payer Group - Click [List of Values] to select a Payer Group. The values defined using the ‘Customer Groups’ function under the AR Setup menu gets listed. The Description for the selected Payer Group gets displayed. This is a mandatory field.

  2. Payer - Click [List of Values] to select the Payer. The Customers associated to the selected Customer Group using the function ‘Customer’ under the AR Setup menu gets listed. The Description for the selected Payer gets displayed. This is a mandatory field.

  3. Policy Type Code - Click [List of Values] to select a Policy Type. The policies associated to the selected Payer and the Payer Group using the function ‘Setup Service Coverage Detail by Policy’ function in the Insurance Management module get listed. This is a mandatory field.

 

On clicking to record the Body Header details in ‘Line 1’ a message ‘Confirm Defaulting Facility Parameter Value’ pops-up. Click [OK] to copy the header, footer and the body message recorded for the Facility Parameter. You can update any of the field content, which is copied from the Facility Parameter. Click [Cancel] to record the details.
