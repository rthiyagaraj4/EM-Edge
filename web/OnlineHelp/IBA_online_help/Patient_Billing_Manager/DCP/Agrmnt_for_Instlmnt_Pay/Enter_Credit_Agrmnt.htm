Enter Credit Agreement

From this function you can Enter the new Credit Agreement for a printed bill of a Deferred Cash Patient. You can Query to view the existing records.

The function Enter Credit Agreement records a new credit agreement for a bill printed for an outstanding amount. You can update and delete the existing records only if it is not printed/cancelled/superseded/fully cleared by sponsor/legally referred. You can also post the documents, which are not posted. Authorized Users are only permitted to login and to record the credit agreement. The ‘Installment Agreement Doc. Type Code’ selected under the ‘Receipts/Refund Controls’ tab in the BL Parameters function gets displayed in the ‘Agreement Doc. Type field’. This function is not applicable for inhouse bills. A document reference number is generated on committing the record. The reason code defined using the ‘Reasons for Billing Actions’ function with the ‘Installment Payment’ check box selected gets listed. You have to select a reason to login and record the credit agreement.

Enter Credit Agreement

From the Deferred Cash Patients > Agreement for Installment Payment, double-click on Enter Credit Agreement function.

The Enter Credit Agreement screen gets displayed.

  1. Enter the Patient Number or click [List of Values] to search for a patient number using the lookup screen. You can enter the Patient ID or data in any of the fields. The records matching the entered data get displayed. Double-click on a record to select it. The Patient Number for the selected record gets displayed in the Patient No field in the Enter Credit Agreement screen. The respective patient details get displayed. This is a mandatory field.

  2. Click [Authorize] to login and to record the credit agreement. Only the Authorized users are permitted to login and update the details. Click [OK] to login and to record the credit agreement. Click [Cancel] to exit from the screen.

    Enter the following details:

    1. User ID – Enter the User ID in alphanumeric format. Only the Authorized Users are permitted to login and to record the credit agreement. This is a mandatory field.

    2. PIN No – Enter the PIN Number for the respective User. This is a mandatory field.

    3. Reason – Click [List of Values] to select a Reason. The reasons defined using the function ‘Reasons for Billing Actions’ get listed. The reason indicates the purpose for credit agreement for the selected patient. This is a mandatory field.

  3. Click [Execute Query] or press [F8] to query the details. The Bill and the DCP details associated to the selected patient get displayed. The total outstanding bill amount of the patient also gets displayed. <Related Field Description>

  4. Select the records to post the details and to record the credit agreement. Click [Enter Agreement] to record the details under Guarantor’s Details, Counseling Details and Installment Plan tab.

    1. Installment Start Date – Click Calendar function to select the date. The installment plan is applicable for the selected patient from the date selected. This is a mandatory field.

    2. Monthly Installment Amount – Enter the amount in numeric format. The amount indicates the total amount the sponsor settles every month to settle the outstanding bill amount. This is a mandatory field.

    3. Installment No – Enter the Installment numbering numeric format.

    4. Due Date – Click Calendar function to select the due date. The due date indicates the date within which the payment has to be settled every month.

    5. Installment Amount – Enter the amount to be settled in each installment every month.

    1. Counseling Code - Click [List of Values] to select a valid Counseling Code. The Counseling Codes defined in the masters get listed. This is a mandatory field.

    2. Counseling Descriptor – The Description for the selected Counseling Code gets displayed.

    3. Counseling Value – The value for the set for the relevant Counseling Code using the Counseling Details function get displayed. This is a mandatory field.

    4. Accept – Select one of the values from the list. You can associate either ‘Accept’, ‘Reject’ or ‘Warning’ from the list for the Counseling Code.

    5. UOM – The Unit of Measure associated to the relevant Counseling Code gets displayed.

    1. Sponsor ID – Enter the ID of the sponsor or click [List of Values] to select a valid Sponsor ID. The lookup screen gets displayed. You can select a record from the list.

    2. Res. Address – Enter the Residential Address of the Sponsor. This is a mandatory field.

    3. Res. Phone – Enter the Residential Phone number of the Sponsor. This is a mandatory field.

    4. Sponsor Name – Enter the Name of the Sponsor. On selecting a Sponsor ID from the lookup screen, the name of the sponsor gets displayed. This is a mandatory field.

    5. Office Address – Enter the Official Address of the Sponsor. This is a mandatory field.

    6. Office Phone – Enter the Official Phone number of the Sponsor. This is a mandatory field.

    7. Relationship – Click [List of Values] to select a value indicating the relationship of the sponsor with the patient. The relationships defined in the Master Patient Index module get listed. Listed items are 'Wife', 'Husband' , 'Son', 'Daughter', 'Father', 'Mother', 'Brother', 'Sister', 'Self' and 'Others'.

  5. Click [Commit Form] or press [F10] to save the record.
