Invoice Generation Rule by Payer/Policy

From this function you can Define the rule to generate invoice for a Payer and Policy association. You can Query to view the existing records.

The Invoice Generation Rule by Payer/Policy function enables you to define the invoice generation rule for a combination of the Payer and the Policy associated. You can choose to print an invoice per day, an invoice for encounters or an invoice per encounter. You can select the episodes that has to be covered in the invoice generated for the payer and the policy associated. You can print separate invoice for the special clinic encounters, if any. The invoices for the discharged encounters can be printed based on the selected patient class. The Accident and Emergency and the outpatient encounters of the patient can be clubbed together to print a single invoice.

Defining Invoice generation rule for the Payer and Policy combination

From Invoice Management > Setup, double-click on Invoice Generation Rule by Payer/Policy function.

The Invoice Generation Rule by Payer/Policy screen gets displayed.

  1. The existing records defined for the logged in facility get listed.

  2. The logged in Facility name is displayed, by default. You can define the invoice generation rule for the logged in Facility only. The facilities associated to the logged in user get listed.

  3. Click [Create Record] to insert a define the invoice generation rule. <Related Field Description>

  4. Select the check boxes under Episode Coverage, if the invoice generation rule is valid for the respective patient class.

    Select the following check boxes, if the invoice generation rule is valid for the selected Episode Type:

    1. Outpatient/Emergency - Select the check box, if the rule is valid for outpatients and emergency patients in the facility. Clear the check box, if the invoice generation rule is not valid for outpatients and emergency patients in the facility.

    2. Inpatient/Daycare - Select the check box, if the rule is valid for inpatients and daycare patients in the facility. Clear the check box, if the invoice generation rule is not valid for inpatients and daycare patients in the facility.

    3. External - Select the check box, if the rule is valid for external or referral patients in the facility. Clear the check box, if the invoice generation rule is not valid for external or referral patients in the facility.

  5. Select the values under Invoice Generation Rule to define the rule for the selected Payer, Policy Type and the patient class. <Related Field Description>

  6. Click [Save] or press [F10] to save the records or [Clear Record] to clear the entered data.
